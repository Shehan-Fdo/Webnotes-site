## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**SEO in HTML**

- SEO stands for **search engine optimization**.
- SEO helps search engines understand your content.
- HTML metadata supports search discovery and page presentation.
- The `<head>` section is not displayed in the browser page content.
- The `<head>` holds metadata such as `<title>`, icons, and other page information.
- Search engines use head information to render and interpret the document.

**Core head elements**

- Put metadata in the `<head>` section of the HTML document.
- Use `<title>` to define the page title.
- Use `<meta charset="utf-8">` to set character encoding.
- Use `<meta name="description" content="...">` to provide page description metadata.
- Use `<meta name="robots" content="...">` to control crawl and indexing behavior.
- Use `<link>` for stylesheets and icons when needed.
- Set the document language with the `lang` attribute on the opening `<html>` tag.

**Meta tags and their roles**

- The `<meta>` element provides metadata that other elements do not cover.
- The `name` attribute supplies document-level metadata.
- The `http-equiv` attribute can simulate an HTTP header directive.
- The `charset` attribute declares the document encoding.
- The `itemprop` attribute provides user-defined metadata.
- Search engines may ignore unsupported meta tags.

**Robots meta tag**

- The `robots` value on `<meta name="robots">` controls crawl and indexing instructions.
- Use comma-separated rules in the `content` attribute.
- `noindex` tells cooperative crawlers not to include the page in search indexes.
- Crawlers must access the page to read meta rules.
- A restrictive `robots.txt` file blocks crawling more effectively than indexing rules alone.
- Page-level settings can also be sent through an HTTP header.

**Practical HTML SEO workflow**

1. Add a valid `<head>` section to the page.
2. Set `<meta charset="utf-8">` near the top of the head.
3. Write a clear `<title>` for the page.
4. Add a concise `<meta name="description" content="...">`.
5. Add `<meta name="robots" content="...">` only when you need crawl or index control.
6. Link any required stylesheet or favicon resources.
7. Verify the page language with `lang` on `<html>`.

**Implementation notes**

- Keep metadata specific to the page.
- Use short, descriptive titles.
- Use metadata to help search engines and browsers understand the document.
- Prefer `robots.txt` when you need to block crawling entirely.
- Remember that only cooperative robots follow robots meta instructions.

**Common placement rules**

- Place SEO-related meta tags inside `<head>`.
- Do not put page metadata in visible body content.
- Keep the head focused on document metadata and resource links.
- Use metadata consistently across pages for predictable indexing behavior.
