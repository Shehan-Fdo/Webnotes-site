## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**When tables are the right choice**

- Use a table for **tabular data**.
- Choose tables when data has clear rows and columns.
- Use tables when you need to compare related values.
- Common examples include schedules, price lists, and product details.
- Use tables when a user needs to look up values quickly.

**When tables are not the right choice**

- Do not use tables for **layout**.
- Do not use tables for anything that is not tabular data.
- Prefer other HTML elements when the content is not a data grid.
- Keep presentation separate from structure.

**Basic table structure**

- Build a table with `<table>`.
- Add rows with `<tr>`.
- Add data cells with `<td>`.
- Add header cells with `<th>`.
- Use `<thead>` to group header rows when needed.
- Use `<tbody>` for the main data rows.

**Useful table features**

- Use `colspan` to span multiple columns.
- Use `rowspan` to span multiple rows.
- Use `<colgroup>` and `<col>` to group columns for styling.
- Use CSS to improve readability and design.
- Tables can contain text, images, links, and other elements.

**Practical guidance**

- Start with the simplest table structure that fits the data.
- Add headers when the column meaning is not obvious.
- Use spanning only when the data relationship requires it.
- Keep the table easy to scan and compare.
- Style the table for clarity, not decoration.

**Quick decision check**

- Ask whether the content is data in rows and columns.
- Ask whether the table helps users compare values.
- Ask whether another layout pattern would be clearer.
- If the answer is no, do not use a table.
