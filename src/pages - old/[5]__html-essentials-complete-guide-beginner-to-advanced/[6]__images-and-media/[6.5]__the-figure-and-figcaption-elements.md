## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Purpose of `figure` and `figcaption`**

- Use `figure` to wrap self-contained content.
- Use `figcaption` to provide a caption for that content.
- Treat the pair as a semantic way to connect visuals and their description.
- Use them for images, diagrams, illustrations, photos, charts, and code examples.
- Avoid relying on CSS class names to imply captioned content.

**How `figure` behaves**

- `figure` represents a unit of content.
- The content is typically referenced as a single unit from the main flow.
- You can move a `figure` without changing the document’s meaning.
- A `figure` can include one or more image elements.
- A `figure` can contain an optional `figcaption`.

**How `figcaption` works**

- `figcaption` provides the caption text for the figure.
- Place the caption where it clearly belongs to the grouped content.
- Use it to describe or contextualize the visuals.
- Keep the caption tied to the content it explains.

**Why this matters**

- The elements create a clear connection between an image and its text.
- Without them, the image and caption can read like separate blocks.
- Screen readers can announce the figure as a grouped unit.
- Semantic markup improves structure compared with presentational workarounds.

**Practical usage**

1. Wrap the related media in `figure`.
2. Add `figcaption` when you need a caption.
3. Keep the caption concise and relevant.
4. Use the group for content that stands on its own.

**Notes on context**

- HTML uses markup to annotate content for display in a browser.
- `figure` and `figcaption` help structure content directly in HTML.
- The elements support clearer meaning without depending on styling alone.
