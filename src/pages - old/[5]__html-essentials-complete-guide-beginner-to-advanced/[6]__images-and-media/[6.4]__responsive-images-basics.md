## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Responsive image concept**

- Responsive images work well across different screen sizes, resolutions, and device features.
- They help improve performance across devices.
- They support better user experience by matching image delivery to the rendering situation.

**Core HTML tools**

- `srcset` lists one or more image candidates.
- Each candidate can use a width descriptor or pixel density descriptor.
- `sizes` tells the browser the layout width of the image for each media condition.
- `sizes` works with `srcset` to help the browser choose an appropriate image.
- `sizes` can also use `auto` for lazy-loaded images.
- `<picture>` contains zero or more `<source>` elements and one `<img>` element.
- The browser checks each `<source>` and picks the best match.
- If no `<source>` matches, the browser uses the `<img>` `src` value.
- `<img>` still needs at least one of `src` or `srcset`.
- `alt` is mandatory and provides text for accessibility.

**How the browser chooses an image**

- The browser compares the available candidates against the current layout and capabilities.
- It uses `srcset` and `sizes` together when width descriptors are present.
- It can choose different images for different orientations or aspect ratios.
- It can fall back to the `src` image when needed.

**Practical usage**

- Use responsive images when the same image must serve multiple device contexts.
- Use `srcset` when you provide multiple image versions.
- Use `sizes` when image display width changes with layout.
- Use `<picture>` when you need alternative versions for different scenarios.
- Keep `alt` text meaningful so the image remains understandable if it does not load.

**Accessibility and fallback behavior**

- `alt` text helps screen readers describe the image.
- `alt` text appears if the image cannot load.
- Fallback behavior matters for network errors, content blocking, and broken links.
- The browser still displays the selected image in the space occupied by the `<img>` element.

**Key takeaways**

- Responsive images adapt to device conditions.
- `srcset` provides candidate images.
- `sizes` guides selection based on layout width.
- `<picture>` offers alternate image sources.
- `alt` supports accessibility and fallback clarity.
