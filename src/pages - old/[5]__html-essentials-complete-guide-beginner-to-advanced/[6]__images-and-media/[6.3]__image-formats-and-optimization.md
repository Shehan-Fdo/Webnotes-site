## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Image formats in HTML**

- Choose the format that fits the image type and browser support.
- `AVIF` offers strong compression and supports transparency and animation.
- `WebP` gives good performance and broad support.
- `PNG` works well for lossless images and transparency.
- `JPEG` suits photographs and other complex images.
- `GIF` is common for simple animation, but it is less efficient than newer formats.
- `APNG` is a lossless animation option.
- `SVG` is useful for vector graphics such as icons and illustrations.
- Use `MIME type` and file extension correctly when you serve images.

**Choosing the right format**

- Ask whether the image is necessary before you add it.
- Remove image resources when CSS or other techniques can achieve the same effect.
- Use CSS effects for resolution-independent visuals when possible.
- Use CSS animations when they replace an image cleanly.
- Match the format to the content type, not just the visual style.
- Prefer formats with better compression when quality stays acceptable.
- Use fallbacks when browser support is limited.

**Embedding images**

- Use the `<img>` element to insert image content into a page.
- Set `src` to the image file path or URL.
- Set `alt` to describe the image for accessibility and SEO.
- Include `width` and `height` to help reserve layout space.
- Treat `<img>` as a void element.

**Responsive images**

- Serve images that fit different screen sizes and resolutions.
- Use responsive image techniques to improve performance across devices.
- Provide alternatives for different display conditions when needed.
- Use the `<picture>` element when you need format fallbacks.
- Keep image choices aligned with the layout and viewport.

**Optimization techniques**

- Reduce file size to improve load speed.
- Use compression to lower the bytes sent to the browser.
- Use lazy loading when an image does not need to load immediately.
- Avoid unnecessary media because images can account for a large share of page bytes.
- Optimize images as part of overall web performance work.
- Balance image quality with download cost.

**Practical guidance**

- Use `AVIF` or `WebP` when you want strong compression and modern support.
- Use `PNG` when you need lossless output or transparency.
- Use `JPEG` for photos where smaller file size matters.
- Use `SVG` for logos, icons, and scalable artwork.
- Use `GIF` only when simple animation is acceptable.
- Test image choices on real devices and browsers.
- Keep accessibility and performance in view at the same time.

**Key takeaways**

- Image choice affects quality, performance, and user experience.
- Smaller files usually improve loading speed.
- Responsive delivery helps images work well across devices.
- Good `alt` text supports accessibility and search visibility.
- The best optimization is often not using an image at all.
