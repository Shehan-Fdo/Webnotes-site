## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Core idea**

- **Progressive enhancement** starts with a baseline that works for as many users as possible.
- You add richer behavior only when the browser or device supports it.
- The goal is a simpler but still usable experience on older browsers and limited devices.
- Modern browsers can receive a more compelling, fully featured experience.
- This approach helps you build for unknown user agents and varied connection quality.

**How it works**

- You treat enhancement layers as off by default.
- You build in layers that turn on based on browser capabilities.
- You use **feature detection** to check whether a browser supports a modern feature.
- You can use **polyfills** to add missing features with JavaScript.
- You should provide acceptable alternatives where possible.

**Why it matters**

- It supports essential content and functionality for more users.
- It helps you focus on the best possible website without breaking older environments.
- It is useful when users have old devices, legacy browsers, or limited operating system updates.
- It can improve resilience for poor connections and global audiences.
- It keeps accessibility in view by encouraging usable alternatives.

**Practical patterns**

- Start with semantic HTML that works without extra styling or scripting.
- Add CSS and JavaScript enhancements only after the baseline is complete.
- Use browser support checks before relying on newer features.
- Keep forms and interactions simple when compatibility is uncertain.
- Prefer fallback behavior that still delivers the main task.

**Related concept**

- **Graceful degradation** also aims to preserve essential content and functionality.
- Graceful degradation starts from a modern experience and falls back when needed.
- Progressive enhancement starts from the baseline and builds upward.
- Both approaches are valid and can complement each other.

**Legacy browser mindset**

- Older browsers may not support every HTML input type or modern API.
- Some unsupported inputs fall back to a default type such as `text`.
- Compatibility tables help you check support before you use a feature.
- Simpler form design reduces the risk of failure in legacy environments.

**Feature queries**

- CSS **feature queries** test whether a browser supports specific CSS features.
- They help you apply styles conditionally.
- They fit the progressive enhancement model because unsupported features stay out of the baseline.

**Takeaway**

- Build the essential experience first.
- Enhance only when support is available.
- Keep the result usable, accessible, and resilient across browsers and devices.
