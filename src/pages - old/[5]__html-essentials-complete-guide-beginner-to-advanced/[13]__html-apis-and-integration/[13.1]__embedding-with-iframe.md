## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**What an iframe does**

- The HTML `<iframe>` element embeds another HTML page into the current page.
- It creates a **nested browsing context**.
- The embedded page has its own document.
- The parent page is the **parent browsing context**.
- The topmost page has no parent browsing context.

**Core attributes and API**

- `src` sets the address of the content to embed.
- Removing `src` programmatically loads `about:blank` in the frame.
- `allow` defines the Permissions Policy for the `<iframe>`.
- `allowFullscreen` indicates whether the frame can enter full screen mode.
- `HTMLIFrameElement` provides properties and methods for inline frame elements.
- `align` and `allowPaymentRequest` are deprecated or non-standard in the MDN reference.

**Behavior and navigation**

- Each embedded browsing context can navigate independently.
- Navigations in embedded contexts are added to the top-level session history.
- The `iframe` element is widely available across modern browsers.
- Some related features may still vary in support.

**Basic usage**

1. Add an `<iframe>` element to your HTML.
2. Set the `src` attribute to the page you want to embed.
3. Add a `title` for accessibility and clarity.
4. Set `width` and `height` as needed.
5. Use CSS if you need to override presentation.

**Practical notes**

- Use `iframe` when you need to show external or separate HTML content inside a page.
- Treat embedded content as its own browsing context.
- Check browser support for any advanced frame behavior you rely on.
- Prefer current, standard attributes over deprecated ones.
