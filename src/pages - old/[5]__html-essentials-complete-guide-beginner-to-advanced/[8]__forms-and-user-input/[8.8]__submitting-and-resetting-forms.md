## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Form submission basics**

- HTML forms use the `<form>` element to collect user input.
- Forms support controls such as text fields, emails, passwords, checkboxes, radios, and buttons.
- Forms and buttons are the main tools for user interaction on a website.
- The form submission basics include the `<form>` element and how data is sent.
- Use `name` and `value` attributes when you need form data to be identified and submitted.
- Labels and correct semantics improve accessibility.

**Submit buttons**

- `<input type="submit">` renders as a button.
- Clicking a submit button makes the user agent attempt to submit the form to the server.
- The `value` attribute sets the button label.
- If you omit `value`, the browser chooses a default label such as `Submit`.
- Use a submit button when you want the form to send its data.

**Reset buttons**

- `<input type="reset">` renders as a button.
- Clicking a reset button restores all inputs in the form to their initial values.
- The `reset()` method on `HTMLFormElement` does the same thing as clicking a reset control.
- A control named or identified as `reset` can mask the form’s `reset()` method.
- Reset does not change other input attributes such as `disabled`.
- If you change an attribute with `setAttribute()`, a later `reset()` call keeps that changed attribute value.
- Reset buttons are usually best avoided because users can click them by mistake.

**Practical form behavior**

- Submit controls are for sending data.
- Reset controls are for restoring defaults.
- Keep button labels clear so users understand the action.
- Use reset only when clearing a form is genuinely helpful.
- Prefer simple, accessible form markup with correct labels and control names.
