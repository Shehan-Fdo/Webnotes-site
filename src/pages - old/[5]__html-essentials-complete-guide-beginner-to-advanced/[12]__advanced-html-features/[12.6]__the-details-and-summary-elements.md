## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Overview**

- `<details>` creates a disclosure widget.
- Content stays hidden until the widget opens.
- `<summary>` provides the visible label for the widget.
- The `<summary>` text is what users click to toggle the state.
- The contents of `<details>` provide the accessible description for the summary.

**Core behavior**

- The `open` attribute shows the `<details>` contents.
- Without `open`, the contents remain collapsed.
- The disclosure control often appears with a triangle indicator.
- The triangle changes state when the widget opens or closes.
- Clicking the `<summary>` toggles the parent `<details>` element.

**Grouping details elements**

- The `name` attribute groups multiple `<details>` elements.
- Grouped `<details>` elements act as mutually exclusive items.
- Opening one grouped item closes the others.
- This pattern supports accordion-style interfaces without scripting.
- If multiple grouped items have `open`, only the first in source order renders open.

**Accessibility and structure**

- Use `<summary>` inside `<details>` as required label content.
- Keep the summary short and descriptive.
- Put the expandable content inside `<details>`.
- Use the hidden content to add supporting information, not the main label.
- Treat the summary as the control and the details as the content region.

**Styling notes**

- You can style `<details>` and `<summary>` with CSS.
- The `::details-content` pseudo-element targets the expandable content.
- `details[open]` helps you style the open state.
- Use styling to clarify state changes, not to replace the native behavior.

**Practical usage**

- Use `<details>` for small blocks of optional information.
- Use it for content that should stay out of the way until needed.
- Use grouped `<details>` when you want one open panel at a time.
- Use native disclosure behavior before adding custom scripting.
- Keep the structure simple so the toggle remains predictable.

**Key terms**

- **Disclosure widget**: a control that reveals or hides content.
- **Open state**: the expanded state of a `<details>` element.
- **Mutually exclusive**: only one item in a group can stay open at once.
- **Accessible description**: the content that helps explain the summary.
