## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Keyboard navigation basics**

- Keyboard access lets users operate a page without a mouse.
- It supports users of screen readers and users with limited pointing-device access.
- Interactive elements should be focusable and usable by keyboard.
- Source order usually controls the default focus order.

**Semantic HTML and focus**

- Use semantic HTML for the right element and the right job.
- Most interactive elements are focusable by default.
- If an element can receive focus, it should usually perform an action.
- Keep focusable elements aligned with their interactive purpose.

**Using `tabindex`**

- `tabindex` makes elements focusable or controls sequential focus.
- `tabindex="0"` makes an element focusable by keyboard and script.
- `tabindex="-1"` allows script focus but removes the element from tab order.
- Positive `tabindex` values change the default order.
- Avoid positive `tabindex` unless you have a strong reason.

**Tab order behavior**

- Elements with positive `tabIndex` come first in tab order.
- Elements with the same positive value follow document order.
- Elements with `tabIndex="0"` follow after positive values.
- Disabled elements do not participate in tabbing order.
- Very large values may be trimmed by browsers.

**Practical accessibility guidance**

- Use keyboard navigation to verify that all important controls are reachable.
- Keep the focus sequence predictable.
- Use `tabindex` only when native HTML does not meet the need.
- Prefer plain language and clear labels to support accessibility.

**Common use cases**

- Use `tabindex="-1"` for programmatic focus on dynamic content.
- Use `tabindex="0"` for custom widgets that need keyboard access.
- Use positive `tabindex` only in exceptional cases.
- Make custom JavaScript widgets behave like their desktop counterparts when needed.

**Key takeaways**

- Keyboard navigation is a core accessibility requirement.
- Semantic HTML reduces the need for manual focus management.
- `tabindex` changes focus behavior, so use it carefully.
- Source order remains the safest default for navigation.
