## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Semantic HTML overview**

- Semantic HTML uses elements that describe meaning, not just appearance.
- The goal is to give content a machine-readable structure.
- This helps browsers, assistive technologies, and search engines understand the page.
- Use the right element for the right job.

**Why semantic elements matter**

- Semantic elements improve **accessibility**.
- Screen readers can better interpret page regions and content roles.
- Semantic structure supports **search engine optimization**.
- Clear element meaning improves readability for developers.
- HTML is not only for layout; it also defines content and structure.

**Common semantic structure elements**

- Use `<header>` for introductory content or a group of navigational aids.
- Use `<nav>` for a section that provides navigation links.
- Use `<main>` for the primary content of the page.
- Use `<section>` to group related content.
- Use `<article>` for self-contained content.
- Use `<aside>` for content related to the main content but separate from it.
- Use `<footer>` for closing content for a page or section.

**Using `<nav>` correctly**

- `<nav>` represents a navigation section.
- Common examples include menus, tables of contents, and indexes.
- Use it for major navigation areas, not every group of links.
- Keep navigation links clear and purposeful.

**Choosing semantic elements over generic containers**

- Use semantic elements when they match the content role.
- Avoid using `<div>` everywhere when a meaningful element exists.
- Generic containers do not describe purpose on their own.
- Semantic tags make the document easier to read and maintain.

**Document structure habits**

- Place semantic elements in their appropriate places.
- Build a clear page structure before styling.
- Keep the document order logical.
- Use headings, lists, and sections to support the page meaning.

**Accessibility and semantics**

- Correct HTML elements provide built-in accessibility hooks.
- Semantic markup supports keyboard and screen reader use.
- Use plain language where possible.
- Define terms when abbreviations or jargon are necessary.

**Practical reminders**

- Choose elements based on meaning first.
- Use structure to reflect the content hierarchy.
- Keep navigation separate from main content.
- Prefer semantic markup when it improves clarity and accessibility.
