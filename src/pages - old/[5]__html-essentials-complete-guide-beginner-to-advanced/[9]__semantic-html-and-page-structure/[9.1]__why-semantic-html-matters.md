## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**What semantic HTML means**

- Semantic HTML uses elements that describe **meaning**, not just appearance.
- It tells browsers and developers what content does or where it belongs in the page.
- A semantic element like `<header>` or `<footer>` carries a clear role.
- Non-semantic elements like `<div>` do not describe content purpose by themselves.
- HTML is the structure and content layer of a webpage.

**Why it matters**

- Semantic HTML improves **accessibility** because assistive technologies can interpret content more accurately.
- It supports **SEO** by giving search engines clearer page meaning.
- It helps you build maintainable code because structure is easier to read.
- It makes content more machine-readable for browsers and other tools.
- It helps you use built-in browser features more effectively.

**Common semantic elements**

- `<header>` represents introductory content.
- `<nav>` represents navigation links.
- `<main>` represents the main content of the page.
- `<footer>` represents closing or footer content.
- `<article>` and `<section>` help organize content into meaningful blocks.

**How semantic HTML helps accessibility**

- Use the right element for the right job.
- Keep source order logical.
- Use plain language where possible.
- Add definitions when you must use abbreviations.
- Use accessible patterns such as alt text, good link text, form labels, and table headings.

**Practical coding habits**

1. Choose semantic elements before defaulting to `<div>`.
2. Match each element to its content role.
3. Keep document structure clear from top to bottom.
4. Use semantic markup consistently across pages.
5. Review the page with accessibility and readability in mind.

**Key takeaway**

- Semantic HTML gives content meaning.
- Meaning improves accessibility, search visibility, and long-term maintainability.
- The browser and assistive tools can do more when your markup is descriptive.
