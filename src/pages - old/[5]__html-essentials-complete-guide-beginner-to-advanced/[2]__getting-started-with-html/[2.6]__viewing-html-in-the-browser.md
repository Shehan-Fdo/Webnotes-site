## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Viewing HTML locally**

- You can open a local `.html` file by double-clicking it.
- You can also drag and drop the file into a browser window.
- You can use `File > Open...` and select the HTML file.
- A local file uses the `file://` URL scheme.
- The browser renders the HTML file directly from your computer.

**Linking CSS for preview**

- Save the stylesheet with the `.css` extension.
- Keep the HTML file and `style.css` in the same folder.
- Add `<link rel="stylesheet" href="style.css" />` in the HTML `<head>`.
- Open the HTML file in a browser to see the linked styles.
- Some text editors can run the page in a browser for you.

**Why local files can be limited**

- Some web features do not work well on `file://`.
- `file://` does not provide a clear origin separation.
- Modern web platform features may be disabled on local files.
- Some examples need a server to behave correctly.
- Browser behavior can differ when content is loaded locally versus over `http://` or `https://`.

> Use a local server when you need behavior that local file viewing does not support.

**Using a local testing server**

- A local server helps you test pages during development.
- It lets you view files through a server instead of `file://`.
- In VS Code, you can use the Live Server extension.
- After installation, open your folder in VS Code.
- Click `Go Live` to start the server.
- The server shows the port it uses.
- Live Server can refresh the page automatically after changes.
- This reduces manual refreshes while you edit.

**Browser behavior and performance context**

- Browsers aim to load pages quickly and stay responsive.
- Latency affects how fast a page appears to load.
- Browsers are mostly single-threaded.
- Understanding browser behavior helps you test and optimize pages more effectively.

**Practical workflow**

1. Save your HTML file.
2. Open it in a browser.
3. Check whether linked CSS loads as expected.
4. If a feature fails on `file://`, switch to a local server.
5. Re-test after each change.

**Key takeaways**

- `file://` is fine for simple previews.
- A local server is better for modern web development.
- CSS links work when paths and filenames match.
- Browser preview is part of the development loop.
- Testing in the right environment avoids false errors.
