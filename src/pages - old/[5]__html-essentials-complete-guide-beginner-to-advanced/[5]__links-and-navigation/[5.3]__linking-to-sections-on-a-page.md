## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Linking to sections with fragments**

- Use a URL fragment to point to a specific part of a page.
- The fragment starts with `#`.
- The browser processes the fragment on the client side after loading the resource.
- The fragment is not sent to the server.
- A fragment can identify a section of a document or another position in a resource.

**Anchor links in HTML**

- Use the `<a>` element with an `href` attribute to create a hyperlink.
- The link target can be a web page, file, email address, or a location in the same page.
- Put clear link text inside `<a>` so the destination is obvious.
- Pressing `Enter` activates a focused `<a>` element when `href` is present.

**Linking to a section by `id`**

- Give the target section an `id` value.
- Link to that section with `#` plus the `id`.
- The browser scrolls to the element that matches the fragment.
- Use this pattern for in-page navigation and jump links.

**Example pattern**

- Target section: `<h2 id="examples">Examples</h2>`
- Link to it: `<a href="#examples">Go to Examples</a>`
- The link jumps to the section with `id="examples"`.

**Text fragments**

- Text fragments link directly to specific text in a page.
- They do not require you to add an `id`.
- They use special syntax in the URL fragment.
- They are useful when you link to content you do not control.
- Browsers may highlight the linked text after scrolling it into view.

**Practical notes**

- Use section links for long pages with multiple topics.
- Use descriptive link text for better usability.
- Keep fragment names stable if other pages depend on them.
- Check browser behavior if you rely on text fragments.

**Related terms**

- **Fragment identifier**: the part of a URL after `#`.
- **Anchor element**: the `<a>` element used for hyperlinks.
- **Hash**: the `#fragment` portion of an anchor or URL.
