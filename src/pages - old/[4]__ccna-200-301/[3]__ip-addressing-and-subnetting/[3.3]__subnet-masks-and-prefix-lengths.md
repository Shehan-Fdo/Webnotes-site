## **Course:** 200-301 CCNA

**Subnet mask basics**

- A subnet mask separates the **network portion** from the host portion of an IPv4 address.
- The mask helps you identify which bits belong to the network and which bits belong to hosts.
- Subnetting divides one larger network into smaller, manageable networks.
- Smaller networks can improve efficiency, simplify routing, and conserve IPv4 addresses.

**Prefix length notation**

- A prefix length uses a slash notation such as `/24`.
- The prefix length tells you how many bits are used for the network portion.
- The remaining bits are available for host addresses.
- Prefix length and subnet mask describe the same boundary in different formats.

**Host and subnet quantities**

- The number of host addresses depends on how many host bits remain.
- More network bits mean fewer host bits.
- Fewer host bits mean fewer usable host addresses.
- Subnetting changes both the number of subnets and the number of hosts per subnet.

**Analyzing subnets**

- Use the mask or prefix length to determine the size of each subnet.
- Compare the address against the subnet boundary to find the network address.
- Use the host portion to identify valid host addresses within that subnet.
- Check that each subnet is unique when assigning IP addresses.

**Practical CCNA focus**

- CCNA subnetting questions often ask you to split a network into smaller parts.
- You should be able to read both dotted-decimal masks and prefix lengths.
- You should be able to relate the notation to host capacity and subnet count.
- You should use subnetting to support addressing and routing design.

> A prefix length and subnet mask are equivalent representations of the same IPv4 network boundary.
