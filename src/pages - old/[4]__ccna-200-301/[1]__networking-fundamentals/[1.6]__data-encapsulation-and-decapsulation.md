## **Course:** 200-301 CCNA

**Encapsulation overview**

- **Encapsulation** is the process of adding protocol information as data moves down the stack.
- Each layer adds its own control information.
- The data unit changes as it moves through the layers.
- The TCP/IP and OSI models help you describe this layered process.
- A protocol stack separates communication functions into manageable layers.

**Decapsulation overview**

- **Decapsulation** is the reverse process at the destination.
- Each layer removes its related protocol information.
- The receiving device uses the remaining data to pass information upward.
- This process continues until the application receives the data.

**Protocol data units**

- A **PDU** is a protocol data unit.
- A PDU name can change as data moves between layers.
- Layered names help you track what the data looks like at each stage.
- The exact PDU names depend on the model and layer being discussed.

**How the models help**

- The TCP/IP model provides a common frame of reference for Internet communications.
- The OSI model also helps you map communication functions to layers.
- Models make it easier to understand where encapsulation and decapsulation happen.
- Models also help you relate captured packets to specific layers.

**Practical flow**

1. The application creates data.
2. The transport layer adds transport information.
3. The network layer adds network information.
4. The data link layer adds local delivery information.
5. The physical layer sends bits across the medium.
6. The destination reverses the process during decapsulation.

**Key takeaway**

- Encapsulation adds information for delivery.
- Decapsulation removes that information after delivery.
- Layered models explain both directions clearly.
- Understanding the process helps you trace packets through a network.
