## **Course:** 200-301 CCNA

**NTP overview**

- **NTP** stands for Network Time Protocol.
- It synchronizes time between hosts on a TCP/IP network.
- It helps devices agree on the system clock.
- Small time differences can cause problems for network services.
- Accurate time matters for logging and SNMP.

**How NTP works**

- NTP uses a **client-server** model.
- The client periodically contacts an NTP server.
- The server responds with its current time.
- The client adjusts its clock using that response.
- One server can distribute time to many clients.

**Time sources**

- NTP servers usually get time from an authoritative source.
- Common reference sources include GPS and atomic clocks.
- These sources provide Coordinated Universal Time, or **UTC**.
- A reference clock is the top time source in the hierarchy.
- Stratum 0 refers to the reference clock itself.
- Stratum 1 devices connect directly to stratum 0 sources.
- Higher stratum numbers mean more steps away from the reference source.

**Stratum hierarchy**

- **Stratum** describes how far a device is from an authoritative time source.
- Lower stratum values indicate closer proximity to the reference clock.
- Each hop adds another level in the hierarchy.
- The hierarchy helps distribute accurate time across the network.

**Why NTP matters**

- Many network functions depend on correct time.
- Logs are easier to correlate when devices share the same time.
- Time-sensitive services can fail or behave unpredictably when clocks drift.
- Authentication and directory access can depend on time alignment.

**Verification focus**

- Check whether a device is synchronized to the expected time source.
- Review NTP associations to confirm peer relationships.
- Verify that the device is using the intended stratum and server.
