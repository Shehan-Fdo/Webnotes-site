## **Course:** 200-301 CCNA

**Overview**

- Basic device configuration is the starting point for Cisco IOS device setup.
- You configure hostnames, passwords, interfaces, and save the configuration.
- You also need to navigate the CLI correctly and know which mode you are in.
- Common lab tasks include configuring a router and a switch.

**Cisco IOS navigation**

- Cisco IOS can feel confusing at first.
- Use the correct command mode before entering configuration commands.
- `show` commands help you check device state and troubleshoot.
- `show version` displays hardware, software version, boot image, and startup details.
- `show running-config` displays the active configuration.
- `show interfaces` displays interface status and details.
- `show logging` displays log messages.

**Initial router configuration**

- Set the **hostname** early so the device is easy to identify.
- Configure password access for management and privileged access.
- Enter global configuration mode before making device-wide changes.
- Apply interface settings as part of the base setup.
- Bring interfaces up when the lab requires connectivity.

**Interface and addressing tasks**

- Configure router interfaces with the required IP settings.
- Use the correct physical or logical interface for the lab topology.
- Some labs use a loopback interface if the router lacks enough FastEthernet ports.
- Some labs use a serial link between routers instead of extra Ethernet ports.
- Verify interface status after configuration.

**Switch configuration**

- Configure a Cisco switch as part of the basic setup.
- Apply the same core management steps you use on a router.
- Set the switch hostname and access passwords.
- Confirm the switch is ready for end-device connectivity.

**Saving configuration**

- Save the running configuration after you finish setup.
- This keeps the configuration available after a reload.
- Do not assume changes persist until you save them.

**Lab context**

- A basic lab may include routers, switches, and PCs.
- One example topology uses two routers, two switches, and four PCs.
- A point-to-point link may use a `/30` subnet.
- Lab equipment can include console and Ethernet cables.

**Practical reminders**

- Work in the correct mode before typing commands.
- Check your work with `show` commands.
- Save the configuration before ending the session.
- Keep the setup simple and verify each step as you go.
