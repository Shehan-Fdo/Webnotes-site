## **Course:** 200-301 CCNA

**Device hardening overview**

- Device hardening reduces the chance of unauthorized access, tampering, and service disruption.
- Focus on routers, switches, firewalls, wireless controllers, and related infrastructure.
- Protect the **management plane** first because attackers often target administrative access.
- Use secure operations and configuration management to keep devices consistent and monitored.

**Core hardening practices**

- Use **authentication, authorization, and accounting (AAA)** to control access and track activity.
- Centralize log collection and monitoring.
- Use secure protocols when possible.
- Gain traffic visibility with NetFlow when you need operational insight.
- Keep device software and images current to reduce exposure to known issues.

**Password and access controls**

- Configure strong passwords.
- Enforce strong password selection where the platform supports it.
- Use login password retry lockout to slow repeated guessing.
- Disable password recovery when appropriate for your security policy.
- Set an **EXEC timeout** so idle sessions close automatically.
- Display warning banners to state access restrictions.

**Management plane protection**

- Disable unused services.
- Limit access to management interfaces.
- Use infrastructure access control lists (**ACLs**) to restrict who can reach the device.
- Filter ICMP packets when needed.
- Filter IP fragments and IP options when the platform supports it.
- Apply TTL-based filtering where supported.
- Control `vty` and `tty` lines to limit remote and local access paths.
- Secure the console and auxiliary ports.
- Use memory and CPU threshold notifications to detect abnormal conditions.

**Secure management sessions**

- Encrypt management sessions with **SSHv2**.
- Prefer secure protocols over cleartext management methods.
- Control transport for `vty` and `tty` lines so only approved access methods work.
- Use authentication and authorization features for remote administration.
- Use TACACS+ or RADIUS where your design requires centralized access control.

**Operational safeguards**

- Monitor Cisco security advisories and responses.
- Use configuration management to track changes.
- Centralize logs so you can correlate events across devices.
- Keep Network Time Protocol (**NTP**) aligned so logs and events are easier to compare.
- Use keepalives for TCP sessions when you need session health visibility.

**Practical hardening checklist**

1. Review management access paths.
2. Remove unused services.
3. Enforce strong passwords and lockout settings.
4. Restrict access with ACLs.
5. Encrypt administrative sessions with SSHv2.
6. Enable logging, monitoring, and time synchronization.
7. Verify the configuration after each change.

**Key terms**

- **AAA** controls who can log in, what they can do, and what gets recorded.
- **ACL** limits traffic based on defined rules.
- **SSHv2** encrypts remote management sessions.
- **NTP** keeps device clocks synchronized for accurate logs.
- **vty** lines handle remote terminal access.
