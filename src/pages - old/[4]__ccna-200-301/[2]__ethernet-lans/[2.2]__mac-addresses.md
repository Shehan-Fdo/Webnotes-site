## **Course:** 200-301 CCNA

**MAC address basics**

- A **MAC** (Media Access Control) address is a Layer 2 identifier used in Ethernet communication.
- It is 48 bits, or 6 bytes, long.
- It is usually written as 12 hexadecimal digits.
- Each hexadecimal character represents 4 bits.
- The first 3 bytes are the **OUI** (Organizationally Unique Identifier).
- The last 3 bytes are unique to the device.
- The address is associated with the network interface hardware.

**Switch MAC address learning**

- A switch learns MAC addresses from incoming Ethernet frames.
- It records the source MAC address and the switch port where the frame arrived.
- It uses this information to build the **MAC address table**.
- The table maps learned MAC addresses to switch ports.
- The switch uses the destination MAC address to choose the outgoing port.
- If the destination MAC address is known, the switch forwards the frame to the matching port.
- If the destination MAC address is unknown, the switch floods the frame out all ports except the incoming port.

**MAC address table behavior**

- The MAC address table helps the switch make forwarding decisions based on destination MAC address.
- Dynamic entries come from MAC learning.
- Static entries are added manually by an administrator.
- The table can include the VLAN ID when present.
- The table reduces unnecessary flooding when the destination is known.

**Finding a MAC address**

- On Windows, use `ipconfig /all` and look for `Physical Address`.
- On Linux, use `ifconfig` to view the interface MAC address.

**Key takeaways**

- MAC addresses identify devices at Layer 2.
- Switches learn source MAC addresses from frames.
- The MAC address table links addresses to ports.
- Unknown destinations trigger flooding.
- Known destinations trigger targeted forwarding.
