## **Course:** 200-301 CCNA

**Overview**

- Switch port security helps protect access-layer ports from unauthorized devices.
- By default, Cisco switch interfaces are enabled and forward traffic for connected devices.
- You use port security when you know which device should connect to a specific port.
- Port security associates allowed **MAC addresses** with an interface.
- It can block traffic from unauthorized devices.
- It can also shut down the port or apply other violation actions.

**Why it matters**

- Access switches sit at the network edge.
- The edge is often the weakest part of the network.
- Anyone with physical access may plug into a wall jack.
- A rogue device can then try to join the local network.
- Port security reduces that risk by limiting which devices can use the port.

**Core behavior**

- The switch forwards frames only for permitted MAC addresses.
- Frames from other devices are discarded.
- If the allowed MAC limit is exceeded, a security violation occurs.
- A port can be limited to a single MAC address or a small set of addresses.
- Limiting to one device can reserve the full port bandwidth for that device.

**Basic configuration steps**

1. Set the interface to access mode with `switchport mode access`.
2. Enable port security with `switchport port-security`.
3. Allow specific MAC addresses with `switchport port-security mac-address <MAC_ADDRESS>`.
4. Optionally use `switchport port-security mac-address sticky` to learn MAC addresses dynamically.

**Violation handling**

- `shutdown` is the common default response in many scenarios.
- `restrict` keeps the port up and generates log messages.
- `protect` drops unauthorized traffic without shutting down the port.
- Choose the action based on whether you want availability, logging, or strict enforcement.

**Secure MAC address types**

- **Static** secure MAC addresses are configured manually.
- **Sticky** secure MAC addresses are learned dynamically and saved in the running configuration.
- Dynamic learning supports simpler setup when devices are expected to remain on the port.

**Practical notes**

- Port security is most useful on access ports, not uplinks.
- It works best when the connected device is known in advance.
- You should verify the allowed MAC addresses before applying strict violation actions.
- If a sticky MAC address must be removed, you may need to clear it from the port configuration.

**Key commands**

- `switchport mode access`
- `switchport port-security`
- `switchport port-security mac-address <MAC_ADDRESS>`
- `switchport port-security mac-address sticky`
- `switchport port-security maximum <number>`
- `switchport port-security violation shutdown`
- `switchport port-security violation restrict`
- `switchport port-security violation protect`

**Exam focus**

- Know that port security binds MAC addresses to a switch port.
- Know that unauthorized traffic can be dropped or trigger a violation.
- Know the difference between `shutdown`, `restrict`, and `protect`.
- Know that sticky MAC addresses are learned dynamically.
- Know that access ports are the usual place to apply port security.
