## **Course:** 200-301 CCNA

**Wireless security overview**

- Wireless traffic is inherently different from wired traffic because devices in range can hear frames over the air.
- WLANs need security to limit access to authorized users and devices.
- Wireless security helps protect data from unauthorized reading and access.

**Authentication basics**

- **Authentication** verifies the identity of a device before it connects to the network.
- Wireless authentication matters because open-air transmission increases exposure to unauthorized access.
- The original 802.11 Wi-Fi standard defined two basic authentication methods:
  - Open System Authentication (`OSA`)
  - Shared Key Authentication (`SKA`)

**Common wireless security terms**

- Wireless security includes many short acronyms.
- Common terms include `WEP`, `PSK`, `TKIP`, `EAP`, `WPA`, `WPA2`, and `WPA3`.
- Understanding the concept behind each acronym helps you remember the term more easily.

**Practical CCNA focus**

- CCNA wireless security topics connect authentication, encryption, and access control.
- You should recognize that wireless networks need stronger protection than wired networks because the medium is open.
- You should associate wireless security with preventing eavesdropping, tampering, and unauthorized access.

**Study reminders**

- Learn the purpose of each authentication method before memorizing the acronym.
- Treat wireless security as a core part of network security, not a separate topic.
- Review wireless security terms alongside wireless network configuration and troubleshooting.
