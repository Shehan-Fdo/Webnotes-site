## **Course:** 200-301 CCNA

**Data serialization overview**

- **Data serialization** converts structured data into a format you can store or share.
- It helps different systems exchange information in a standardized way.
- Format choice depends on human readability, data complexity, storage, and speed needs.

**JSON**

- **JSON** means JavaScript Object Notation.
- It is a lightweight data-interchange format.
- It uses a short, text-based syntax.
- It is easier for humans to read and write than many other structured formats.
- It uses a limited set of data types.

**XML**

- **XML** means Extensible Markup Language.
- It represents structured data with tags.
- It is commonly used when systems need a structured, machine-readable format.
- It appears in network automation contexts such as `NETCONF`, where XML is used for data exchange.

**JSON vs. XML in network automation**

- JSON and XML both support automated data exchange.
- JSON is often associated with lightweight interchange.
- XML is often associated with tagged, structured representation.
- Cisco automation and programmability topics use these formats in modern network workflows.

**Practical CCNA takeaways**

- Use JSON when you want compact, readable structured data.
- Use XML when you need tagged data representation for system exchange.
- Recognize both formats as part of network automation and programmability.
- Expect these formats in APIs and automation-related tools.

**Key terms**

- **API**: Application Programming Interface.
- **NETCONF**: Network Configuration Protocol.
- **RESTCONF**: RESTful protocol for configuration and state data access.
- **YANG**: A data modeling language used with network automation.
