## **Course:** Comptia Network Plus

**DNS**

- **DNS** stands for Domain Name System.
- DNS translates human-readable names into IP addresses.
- DNS uses a hierarchical structure.
- The root is represented by a single period `.`.
- Top-level domains include `com`, `org`, and `net`.
- Country-code top-level domains use two-letter codes such as `us`, `ca`, and `uk`.
- DNS records store information used for name resolution.
- An `A` record maps a hostname to an IPv4 address.
- An `AAAA` record maps a hostname to an IPv6 address.

**DHCP**

- **DHCP** stands for Dynamic Host Configuration Protocol.
- DHCP assigns IP addresses to devices automatically.
- DHCP scopes define ranges of addresses available from a DHCP server.
- A DHCP lease is temporary.
- A DHCP reservation maps a specific MAC address to a specific IP address.
- A reservation keeps the same address available for the same device.
- The Internet Protocol Helper, or `IP Helper`, API can retrieve and modify local network configuration settings.

**NTP**

- **NTP** stands for Network Time Protocol.
- NTP synchronizes clocks across network devices.
- Synchronized time helps keep log files aligned.
- Accurate time supports authentication and outage analysis.
- NTP uses clients and servers.
- NTP uses stratums to describe time hierarchy.
- Workstations can be configured to use NTP for time updates.

**Network Service Relationships**

- DNS supports name resolution for network communication.
- DHCP supports address assignment for network connectivity.
- NTP supports consistent time across devices.
- These services help with deployment, monitoring, and troubleshooting.

**Practical Study Points**

- Check DNS when a name does not resolve.
- Check DHCP when a device does not receive an address.
- Check NTP when device times drift or logs do not match.
- Use reservations when a device needs a consistent IP address.
- Use synchronized time when you compare logs across devices.
