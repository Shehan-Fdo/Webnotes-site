## **Course:** Comptia Network Plus

**Cloud service models**

- Cloud service models define who manages each layer of the stack.
- **IaaS** means Infrastructure as a Service.
- With IaaS, you manage the operating system, applications, and data.
- The provider manages the hardware, virtualization, and network.
- Use IaaS when you need OS control or custom software.
- **PaaS** means Platform as a Service.
- With PaaS, you manage applications and data.
- The provider manages the OS, runtime, and hardware.
- Use PaaS when you want to focus on code, not infrastructure.
- **SaaS** means Software as a Service.
- With SaaS, the provider manages the full stack.
- You usually manage only data, and sometimes even that is limited.
- Use SaaS for pre-built applications.

**Deployment models**

- Deployment models define who can access the cloud environment.
- **Public cloud** uses shared infrastructure.
- Public cloud supports multi-tenancy.
- Use public cloud for cost efficiency, scalability, and agility.
- **Private cloud** uses dedicated infrastructure for one organization.
- Private cloud supports single-tenancy.
- Use private cloud for security, compliance, and control.
- A private cloud can be hosted by a provider.
- The key distinction is dedicated use by one customer.
- **Hybrid cloud** combines public and private cloud.
- Use hybrid cloud for flexibility, burst capacity, or migration.
- **Community cloud** is shared by organizations with common needs.
- Use community cloud for groups such as healthcare or government consortia.

**Exam focus**

- Match each service model to the layer you manage.
- Match each deployment model to the access pattern it describes.
- Do not assume private cloud means on-premises.
- Focus on single-tenancy versus multi-tenancy when comparing deployment models.
