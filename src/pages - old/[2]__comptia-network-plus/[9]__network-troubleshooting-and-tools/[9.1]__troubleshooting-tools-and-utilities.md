## **Course:** Comptia Network Plus

**Troubleshooting focus**

- Troubleshooting is a major part of the Network+ exam.
- Real-world network work often starts with fast diagnosis and targeted fixes.
- The exam emphasizes using appropriate tools to resolve connectivity issues.

**Command-line tools**

- Many detailed troubleshooting tools are available at the command line.
- `ping` checks whether a device is reachable.
- `ping` uses the **ICMP** (Internet Control Message Protocol) protocol.
- `ping` is often one of the first checks you run during troubleshooting.
- `traceroute` helps you trace the path to a destination.
- `nslookup` helps you query DNS-related information.
- `dig` is another DNS lookup utility.
- `tcpdump` captures and inspects network traffic.
- `netstat` shows network connections and related status.

**Using `ping` effectively**

- Start with `ping` when you want to confirm basic reachability.
- Use a hostname or IP address as the target.
- If the host does not respond, use the result as a clue for the next step.
- Test from different systems when you need to isolate the problem.

**Practical troubleshooting approach**

1. Check whether the device is reachable with `ping`.
2. Use `traceroute` if you need to see where traffic stops.
3. Use `nslookup` or `dig` if the issue may involve name resolution.
4. Use `tcpdump` or `netstat` when you need deeper traffic or connection details.

**Study emphasis**

- Focus on what each tool tells you.
- Match the tool to the symptom.
- Use command-line utilities to narrow the problem before changing settings.
