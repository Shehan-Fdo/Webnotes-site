## **Course:** Comptia Network Plus

**Network topology basics**

- **Network topology** is the arrangement of devices and links in a network.
- It describes how devices connect and how data flows between them.
- Topology choices affect performance, cost, reliability, security, and troubleshooting.
- Network+ coverage includes comparing topologies, architectures, and types.

**Physical and logical topology**

- **Physical topology** is the actual layout of cables and devices.
- **Logical topology** is how data moves across the network.
- A physical map shows device placement and cable connections.
- A logical map shows how the network connects at a higher level.
- Use the right map for the documentation task you need.

**Common topology types**

- **Point-to-point** connects two nodes directly.
- It is the simplest communication model.
- It can provide high bandwidth.
- **Bus** uses one main cable as a backbone.
- All nodes connect to that main cable.
- **Star** connects all devices to a central device such as a switch or hub.
- All communication passes through the center.
- **Ring** connects devices in a loop.
- **Mesh** uses multiple interconnections between nodes.
- **Tree** combines hierarchical branches of connected devices.
- **Hybrid** combines two or more topology types.

**Network type considerations**

- Wired topologies are commonly grouped as bus, ring, star, tree, mesh, and hybrid.
- Network type selection depends on the layout you need to support.
- The right design helps with installation and maintenance.
- The right design also affects scalability and fault detection.

**Documentation focus**

- Create a physical network map when you need exact device and cable placement.
- Create a logical network map when you need a higher-level view of connectivity.
- Use topology documentation to support troubleshooting and network planning.
