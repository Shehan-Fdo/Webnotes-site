## **Course:** Comptia A+ 220-1201

**Hardware Troubleshooting Overview**

Hardware problems often show up as startup errors, blank screens, no power, sluggish performance, overheating, or Windows stop errors.

- **POST** means Power-On Self-Test.
- POST checks core components at startup.
- It verifies the CPU, video, keyboard, and memory.
- If POST finds a problem, you may see an error message or hear beeps.
- Beep codes and on-screen messages can vary by manufacturer.

**Startup and POST Checks**

Use POST results to narrow the fault before you replace parts.

1. Turn on the system and watch for any on-screen messages.
2. Listen for beep patterns from the motherboard.
3. Identify the BIOS or UEFI vendor if you need the correct beep code chart.
4. Match the beep pattern to the vendor documentation.
5. Focus on the component that POST identifies.

> Beep code meanings are not universal, so you must use the correct vendor chart.

**Blank Screen and No Power**

A blank screen or no-power condition can come from several hardware issues.

- Check the power source first.
- Verify internal power connections.
- Confirm the display is working if the system powers on.
- Use the symptoms to separate a power issue from a video issue.

**Overheating and Sudden Shutdowns**

A system that suddenly turns off without warning often points to excessive heat.

- Heavy CPU use can expose cooling problems.
- Gaming can trigger heat-related failures.
- Check fans for operation.
- Check airflow through the case.
- Confirm heat sinks are installed and working correctly.

**Sluggish Performance**

Slow performance can indicate a hardware problem, especially when the issue appears during demanding tasks.

- Compare the slowdown with the workload.
- Watch for symptoms during CPU-intensive applications.
- Check whether cooling or other hardware limits are affecting performance.
- Use the pattern of failure to isolate the likely component.

**Practical Troubleshooting Approach**

Use a simple, repeatable process when you diagnose hardware issues.

1. Identify the symptom.
2. Check for startup indicators such as POST beeps or messages.
3. Test the most likely hardware path first.
4. Verify cooling, power, and basic component operation.
5. Replace or repair the failed part after you confirm the cause.
