## **Course:** Comptia A+ 220-1201

**Virtualization Overview**

Virtualization lets you run multiple operating systems on the same physical system.

- A **virtual machine (VM)** behaves like a separate computer.
- Each VM can have its own CPU, memory, storage, and network connection.
- You can use virtualization to run different operating systems on one device.

**Host-Based Virtualization**

Host-based virtualization runs virtual machines on top of a primary operating system.

- The host operating system stays active on the physical device.
- The guest operating systems run inside virtual machines.
- This setup is common on desktop systems.

**Hypervisors**

A **hypervisor** is the software that manages virtual machines and their interaction with the physical system.

- It allocates CPU cycles to virtual machines.
- It allocates memory to virtual machines.
- It separates networking resources between virtual machines.
- It manages the virtual environment on top of the hardware.

**Virtualization Services**

Virtualization depends on supporting features and use cases.

- Modern CPUs may include virtualization support.
- Some environments use **virtual desktop infrastructure (VDI)** for centralized desktops.
- **Containerization** is another method used to isolate applications.
- Virtualization can support legacy software and cross-platform use.

**Practical Uses**

You use virtualization when you need flexibility without adding more physical hardware.

- You can test software in isolated environments.
- You can run legacy operating systems for compatibility.
- You can use different operating systems on one computer.
- You can separate workloads for easier management.
