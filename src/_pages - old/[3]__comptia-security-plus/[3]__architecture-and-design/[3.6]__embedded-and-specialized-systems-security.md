## **Course:** CompTia Security Plus

**Specialized systems overview**

- Specialized systems prioritize operational function over IT security.
- They often run for long periods with limited maintenance.
- Security controls must fit device constraints and business uptime needs.
- Embedded systems are common in devices that perform a dedicated function.
- Security issues can affect both data and physical processes.

**IoT and embedded device characteristics**

- **IoT** stands for Internet of Things.
- IoT devices are network-connected and often include sensors.
- Examples include smart thermostats, cameras, medical devices, and wearables.
- Many IoT devices have limited processing power.
- Some devices cannot run antivirus software.
- Firmware updates may be infrequent or impossible.
- Default credentials are a common risk.
- Encryption may be absent on some devices.

**ICS and SCADA**

- **ICS** stands for Industrial Control Systems.
- **SCADA** stands for Supervisory Control and Data Acquisition.
- ICS and SCADA control physical processes in manufacturing, power, water, and transportation.
- SCADA provides centralized monitoring and control.
- These systems often run for decades.
- Proprietary protocols are common in these environments.
- Patching can disrupt operations, so updates require caution.

**Security challenges in specialized environments**

- Operational availability is often the top priority.
- Security changes can create downtime risk.
- Legacy hardware may not support modern protections.
- Limited compute resources restrict security tooling.
- Long device lifecycles increase exposure to outdated configurations.
- Vulnerabilities can affect both cyber and physical safety.

**Network segmentation**

- Segmentation limits which devices can communicate.
- It helps isolate sensitive systems from broader network traffic.
- You can segment physically, logically, or virtually.
- **VLANs** are one common segmentation method.
- Screened subnets can separate public-facing and internal systems.
- Intranets and extranets support different trust boundaries.
- Segment application instances when they need isolation or high throughput.
- Keep users away from direct access to sensitive database servers when possible.

**Secure boot and device trust**

- Secure boot helps ensure trusted software starts on embedded devices.
- It reduces the risk of unauthorized code loading at startup.
- Use trusted startup checks when device integrity matters.
- Secure boot is especially relevant for devices that are hard to patch.

**Practical study focus**

- Know the difference between IoT, embedded systems, ICS, and SCADA.
- Match security controls to device limitations.
- Expect tradeoffs between security and uptime.
- Use segmentation to reduce blast radius.
- Treat physical impact as part of the risk picture.
