## **Course:** Comptia Network Plus

**Spanning Tree Protocol overview**

- **Spanning Tree Protocol** (`STP`) prevents loops in switched Ethernet networks.
- It builds a loop-free logical topology for Layer 2 networks.
- It keeps backup links available for fault tolerance.
- It disables links that are not part of the active tree.
- It helps avoid broadcast storms and network instability.

**Why loops matter**

- Multiple switch connections can create redundant paths.
- Redundant paths can form a loop at Layer 2.
- Frames can circulate repeatedly in a loop.
- A loop can overwhelm switch resources quickly.
- A loop can bring network connectivity down.

**Core STP behavior**

- STP uses an algorithm to create a tree structure across the Layer 2 topology.
- STP lets switches communicate to discover physical loops.
- STP selects a single active path between network nodes.
- STP leaves backup links in place for failover.
- STP reacts when the network changes.

**Root bridge and path selection**

- A **root bridge** is necessary to manage redundant paths.
- Switches use STP information to determine forwarding roles.
- Path cost helps influence which path STP prefers.
- Root port selection depends on the best path toward the root bridge.

**Loop protection**

- **Loop protection** improves STP behavior on interfaces.
- It helps prevent ports from moving into a forwarding state that would create a loop.
- It acts when BPDUs are not received on a nondesignated interface.
- It adds protection for STP, RSTP, and MSTP deployments.

**Practical troubleshooting focus**

- Check for unintended redundant links when users lose connectivity after switch expansion.
- Verify that a root bridge exists in the spanning tree.
- Review port roles and path costs when STP behavior seems incorrect.
- Use redundancy carefully so it supports availability without creating loops.

**Key terms**

- **BPDU**: Bridge Protocol Data Unit.
- **RSTP**: Rapid Spanning Tree Protocol.
- **MSTP**: Multiple Spanning Tree Protocol.
- **Layer 2**: The data-link layer where switching and MAC-based forwarding occur.
