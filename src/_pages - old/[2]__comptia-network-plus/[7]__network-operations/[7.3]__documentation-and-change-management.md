## **Course:** Comptia Network Plus

**Documentation in network operations**

- Documentation supports network management and reduces time and cost.
- You use it to record design, processes, and procedures.
- It helps you understand how the network was built and how it changes over time.
- It is especially useful when devices are added, removed, or reconfigured.
- It includes both technical details and internal operating procedures.

**Common documentation types**

- **Logical network maps** show how devices and segments relate.
- **Physical network maps** show the actual layout and connections.
- **Asset management** tracks network equipment and related resources.
- **IP address documentation** records assigned addresses and related details.
- Internal procedures document how you handle routine and exceptional tasks.

**Internal operating procedures**

- Procedures should match the organization’s business needs.
- Notification steps may differ by company.
- A system outage may require an email alert or a direct phone call.
- Facilities issues should have a defined response process.
- Specific tasks should also have written procedures.

**Change management purpose**

- Change management gives network changes a formal process.
- It helps reduce risk when you upgrade software or modify configurations.
- It applies to switch changes, firewall updates, and routing changes.
- It prevents unexpected network behavior from causing enterprise impact.
- It requires clear, well-documented policies.

**Change control planning**

- Define how often changes occur.
- Define when changes are allowed to happen.
- Set a change window for the work.
- Document the exact process for making the change.
- Document how you back out the change if needed.

**Practical change workflow**

1. Review the requested change and confirm the need.
2. Check the approved change window.
3. Follow the documented procedure.
4. Verify the change after implementation.
5. Use the rollback plan if the change fails.

**Why documentation matters**

- Good documentation supports consistency.
- It helps you respond faster during incidents.
- It makes network changes easier to manage.
- It reduces confusion when multiple people support the environment.
- It improves long-term maintainability.

> Change control can be difficult to implement because many questions must be answered before a change is approved.
