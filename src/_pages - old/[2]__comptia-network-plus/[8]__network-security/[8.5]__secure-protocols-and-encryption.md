## **Course:** Comptia Network Plus

**Why secure protocols matter**

- Network traffic can be viewed by others along the path.
- Unsecured wireless traffic is especially easy to capture.
- Sensitive data may travel inside packets without you noticing.
- You should encrypt data in transit to reduce exposure.

**Secure communication choices**

- Use **encrypted protocols** for network communication.
- Use a **VPN** (Virtual Private Network) tunnel when you need protected remote traffic.
- Use secure services instead of plain-text alternatives.
- Choose protocols that protect usernames, passwords, and session data.

**Remote access protocols**

- Use **SSH** (Secure Shell) for encrypted remote terminal access.
- SSH uses TCP port `22`.
- SSH protects command-line traffic in transit.
- Avoid **Telnet** for remote administration.
- Telnet uses TCP port `23`.
- Telnet provides no encryption.

**Common secure protocol examples**

- **HTTPS** protects web traffic with encryption.
- **SSL** (Secure Sockets Layer) is a common secure protocol name you may see in study material.
- **IPSec** is used to secure network communications.
- Secure protocols help protect data, credentials, and management sessions.

**Practical guidance**

1. Identify whether the traffic carries sensitive information.
2. Replace unencrypted services with secure alternatives.
3. Use SSH for remote device management.
4. Use VPNs when you need encrypted transport across untrusted networks.
5. Verify the protocol and port before connecting to a service.

**Key takeaways**

- Encryption helps protect data from packet capture.
- Secure protocols reduce risk on public and wireless networks.
- SSH is the secure replacement for Telnet.
- Protocol choice matters as much as device configuration.
