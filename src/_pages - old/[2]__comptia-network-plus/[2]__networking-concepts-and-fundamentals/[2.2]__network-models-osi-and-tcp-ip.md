## **Course:** Comptia Network Plus

**Reference models**

- The OSI (Open Systems Interconnection) model is a **conceptual framework** for network communication.
- It standardizes how different networking systems communicate.
- It divides networking functions into seven layers.
- The TCP/IP (Transmission Control Protocol/Internet Protocol) model is a **protocol suite** used in production.
- TCP/IP is a set of protocols that work together to support inter-communication among network devices.
- Both models help you understand and structure how networks function.

**OSI model overview**

- The OSI model has **seven layers**.
- Each layer has a distinct responsibility.
- The model helps you understand the logical process that happens when devices communicate.
- The model is useful for troubleshooting and exam study.
- Layered thinking helps you isolate where a network problem occurs.

**OSI layers**

- Layer 1 is the **Physical** layer.
- It transmits raw bits over a physical medium.
- Examples include Ethernet cables, fiber optic cables, hubs, and repeaters.
- A broken cable is a Layer 1 issue.
- Layer 2 is the **Data Link** layer.
- It handles node-to-node data transfer.
- It also supports error detection and correction.
- Examples include MAC addresses and switches.
- Layer 3 is the **Network** layer.
- It handles logical addressing and routing.
- Layer 4 is the **Transport** layer.
- It supports end-to-end communication.
- Layer 5 is the **Session** layer.
- It manages sessions between systems.
- Layer 6 is the **Presentation** layer.
- It handles data representation.
- Layer 7 is the **Application** layer.
- It provides services used by applications.

**TCP/IP model**

- The TCP/IP stack is the practical model used in real networks.
- It is less theoretical than the OSI model.
- It groups networking functions into fewer layers than OSI.
- It is commonly treated as a protocol stack.
- Use it to understand how protocols operate together in production environments.

**How to study the models**

- Learn the OSI layers in order.
- Match each layer to its main function.
- Compare OSI and TCP/IP by layer count and purpose.
- Use the models to trace where communication breaks down.
- Start with the lowest layer when you suspect a physical problem.

**Key exam focus**

- Know that OSI is a reference model.
- Know that TCP/IP is a protocol suite.
- Know that OSI has seven layers.
- Know that TCP/IP is used in production.
- Know that each layer has a specific role in communication.
