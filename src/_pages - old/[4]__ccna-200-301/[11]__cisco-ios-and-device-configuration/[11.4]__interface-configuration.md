## **Course:** 200-301 CCNA

**Interface basics**

- Use interface configuration mode to change settings on a specific port.
- Apply changes to the selected interface only.
- Keep interface notes locally significant.
- Use descriptions to document the interface role.
- A description can be up to 240 characters.
- Use `description` from interface submode.
- Remove a description with `no description` or `no desc`.

**Common interface commands**

- Use `shutdown` to disable an interface.
- Use `no shutdown` to enable a disabled interface.
- `shutdown` marks the interface unavailable.
- A shut interface appears as `administratively down` in status output.
- Use `show interfaces` to confirm shutdown state.
- Use `show ip interface` for IP-related details.
- Use `show ip interface brief` for a compact status view.
- `show ip interface` can reveal secondary addresses.

**Verification and documentation**

- Check `show running-config` to confirm interface descriptions.
- Look for the `description` line under the interface entry.
- Use descriptions to make later troubleshooting easier.
- Keep interface labels consistent with the network role.
- Document links, WAN paths, or management ports clearly.

**Switch management note**

- A switch can forward frames without extra configuration.
- Configure an IP address on a logical interface for management access.
- VLAN 1 is commonly used as the management interface in basic setups.
- Use the management IP for network access and tools such as SNMP.

**Quick workflow**

1. Enter the interface configuration mode.
2. Add a description if needed.
3. Configure the interface state with `shutdown` or `no shutdown`.
4. Verify the result with `show ip interface brief` or `show running-config`.
