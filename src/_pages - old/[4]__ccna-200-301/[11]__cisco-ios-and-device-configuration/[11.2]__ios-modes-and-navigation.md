## **Course:** 200-301 CCNA

**Cisco IOS command hierarchy**

- Cisco IOS separates sessions into **user EXEC** and **privileged EXEC** access levels.
- User EXEC gives you basic monitoring commands only.
- Privileged EXEC gives you access to all router commands.
- Privileged EXEC can be password protected.
- After you enter EXEC, IOS uses a hierarchical command structure.
- Command options depend on your current mode.

**Main command modes**

- User EXEC is the initial login mode.
- User EXEC has the fewest commands.
- You can use `?` to list available commands.
- Common user EXEC tasks include status checks and basic troubleshooting.
- Privileged EXEC lets you access all command modes.
- Global configuration mode is where you make device-wide changes.
- Cisco IOS also includes interface configuration mode, subinterface configuration mode, router configuration mode, and line configuration mode.

**Prompt and navigation**

- The prompt shows the device hostname.
- The prompt also shows the current mode.
- `enable` moves you from user EXEC to privileged EXEC.
- `disable` turns off privileged commands and returns you to user EXEC.
- `exit` leaves the current mode.
- `end` or `Ctrl-Z` returns you from any configuration mode to privileged EXEC.
- `exit` from interface, line, router, IPX-router, or route-map mode returns you to global configuration mode.
- `exit` from subinterface configuration mode returns you to interface configuration mode.

**Command behavior by mode**

- Configuration commands are not available until you reach the correct configuration mode.
- Some commands work in all command modes.
- `exit` works in all command modes.
- `exit` closes an active terminal session when used at the EXEC level.
- `show` displays running system information in user EXEC.
- `ping` sends echo messages in user EXEC.
- `traceroute` traces a route to a destination in user EXEC.

**Practical navigation habits**

- Check the prompt before you enter a command.
- Use `?` when you need help with available commands.
- Move to privileged EXEC before entering configuration mode.
- Return to the correct parent mode when you finish a submode.
- Use the hierarchy to avoid entering commands in the wrong place.
