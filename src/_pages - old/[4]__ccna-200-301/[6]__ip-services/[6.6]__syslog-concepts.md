## **Course:** 200-301 CCNA

**Syslog overview**

- **Syslog** is a standard for message logging.
- Cisco devices generate syslog messages when events occur.
- Examples include an interface going down or a port security violation.
- Syslog helps you monitor device health and network incidents.

**Default message display**

- Cisco devices send log messages to the console by default.
- The `logging console` command is enabled by default.
- Console users can see messages as they occur.
- SSH and Telnet users do not see these messages unless you enable terminal monitoring.

**Viewing logs in remote sessions**

- Use `terminal monitor` in EXEC mode to display syslog messages in an SSH or Telnet session.
- This command lets you see real-time messages on the current terminal.
- Without it, remote sessions may miss important device events.

**Common logging locations**

- Cisco devices can store or display syslog messages in several places.
- The **logging buffer** stores messages in RAM.
- The console line displays messages locally.
- Terminal lines can display messages for remote sessions.
- A syslog server stores messages centrally.
- Flash memory can also store logs in some configurations.

**Centralized logging**

- Store logs on a central **syslog server** for easier monitoring.
- Central logging helps you keep a history of device events.
- It also improves troubleshooting and visibility across the network.
- Cisco devices can send logs to a server with the `logging` command and a destination IP address.

**Transport and delivery**

- Syslog messages sent to a server use UDP port `514`.
- UDP is connectionless, so delivery is not guaranteed.
- Use the server for collection and analysis, not for reliable transport assumptions.

**Why syslog matters**

- Syslog gives you visibility into device events.
- It supports faster troubleshooting when something fails.
- It helps you track security-related incidents.
- It gives administrators a practical way to monitor network behavior.
