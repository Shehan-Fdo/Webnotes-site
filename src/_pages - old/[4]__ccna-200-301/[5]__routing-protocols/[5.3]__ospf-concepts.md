## **Course:** 200-301 CCNA

**OSPF overview**

- OSPF stands for **Open Shortest Path First**.
- It is an **Interior Gateway Protocol (IGP)**.
- It is a **link-state** routing protocol.
- It is an **open standard** protocol.
- It is classless and supports **VLSM** and **CIDR**.
- It uses the **SPF** algorithm to choose best paths.
- It uses interface cost as its metric.
- The default administrative distance is `110`.

**How OSPF works**

- Routers form **neighbor** relationships before exchanging routes.
- Neighbors exchange topology information, not full routing tables.
- Each router builds a topology view of the network.
- Each router runs SPF to calculate best routes.
- This design reduces routing loops.
- OSPF sends partial updates instead of full-table updates.
- OSPF converges fast compared to distance-vector protocols.

**Areas and topology**

- OSPF uses the concept of **areas**.
- Routers in the same area share the same topology table.
- Area design helps scale larger networks.
- Manual summarization can be done on **ABRs** and **ASBRs**.
- Equal-cost load balancing is supported.

**OSPF updates and addresses**

- OSPF uses multicast addresses `224.0.0.5` and `224.0.0.6` for routing updates.
- `224.0.0.5` reaches all OSPF routers.
- `224.0.0.6` reaches designated routers and backup designated routers.
- OSPF uses incremental updates after the initial exchange.

**OSPF packet types**

- OSPF packet exchange includes **Hello**, **DBD**, **LSR**, **LSU**, and **LSAck**.
- **Hello** packets discover and maintain neighbors.
- **DBD** means Database Description.
- **LSR** means Link-State Request.
- **LSU** means Link-State Update.
- **LSAck** means Link-State Acknowledgment.
- The DBD packet carries an abbreviated list of the sender’s **LSDB**.
- **LSDB** means Link-State Database.

**Basic single-area configuration**

- Enable OSPF with `router ospf PROCESS-ID`.
- Advertise interfaces with `network IP_ADDRESS WILDCARD_MASK AREA_ID`.
- The OSPF process ID does not have to match on all routers.
- Neighboring routers must use the same area ID to form adjacency.
- For CCNA, focus on single-area OSPFv2 with IPv4.

**Default route injection**

- A default route is `0.0.0.0/0` in IPv4.
- A default route is `::/0` in IPv6.
- Use `default-information originate` to inject a default route into OSPF.
- This is useful when you do not want to learn every specific route.
- It is common in stub networks and Internet edge designs.

**Key takeaways**

- OSPF is a classless link-state protocol.
- It uses topology exchange, not route-table exchange.
- It relies on areas, neighbors, and SPF calculations.
- It supports fast convergence and partial updates.
- It is a core CCNA routing topic.
