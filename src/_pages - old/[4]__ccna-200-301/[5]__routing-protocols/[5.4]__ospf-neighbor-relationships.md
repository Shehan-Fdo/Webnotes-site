## **Course:** 200-301 CCNA

**OSPF neighbor basics**

- OSPF stands for **Open Shortest Path First**.
- OSPF is a link-state **Interior Gateway Protocol** (IGP).
- Routers must form neighbor relationships before they exchange routes.
- OSPF neighbors exchange topology information, not full routing tables.
- Each router runs the **SPF** (Shortest Path First) or Dijkstra algorithm to choose best routes.
- OSPF uses multicast addresses `224.0.0.5` and `224.0.0.6` for routing updates.
- The default OSPF administrative distance is `110`.

**Neighbor formation and adjacency**

- OSPF neighbor formation starts with **hello packets**.
- Hello packets help discover neighbors and build adjacencies.
- OSPF neighbor states describe progress toward full synchronization.
- The common state sequence is `Down`, `Init`, `2-Way`, `ExStart`, `Exchange`, `Loading`, `Full`.
- `Down` means no hello packets have been received.
- `Init` means a hello packet is received, but the router has not yet seen itself listed as a neighbor.
- `2-Way` means both routers see each other in hello packets.
- `ExStart` begins database description exchange.
- `Exchange` carries **DBD** (Database Description) packets.
- `Loading` involves **LSR** (Link-State Request) and **LSU** (Link-State Update) packets.
- `Full` means the neighbors are fully synchronized.

**Packets used in OSPF neighbor exchange**

- **Hello** packets discover neighbors and maintain the relationship.
- **DBD** packets summarize link-state information.
- **LSR** packets request specific link-state records.
- **LSU** packets send requested link-state updates.
- **LSAck** packets acknowledge received link-state advertisements.

**Settings that must match**

- Hello and dead timers must match for OSPF neighborship.
- OSPF neighbors also need compatible interface and network settings to form adjacency.
- If key settings do not match, the neighbor relationship does not complete.

**CCNA study focus**

- OSPF is an open standard, so multiple vendors can implement it.
- OSPF supports classless routing.
- OSPF supports VLSM, CIDR, manual route summarization, and equal-cost load balancing.
- OSPF uses incremental updates instead of sending full routing tables repeatedly.
- OSPF stores topology information in a link-state database.
- The topology table reflects the network topology and SPF results.

**Practical takeaways**

- Check `hello` and `dead` timer values first when neighbors fail to form.
- Verify that both routers reach the `Full` state before you expect stable route exchange.
- Remember that OSPF adjacency depends on shared topology information, not route-table exchange.
- Use the neighbor state to identify where the formation process stops.
