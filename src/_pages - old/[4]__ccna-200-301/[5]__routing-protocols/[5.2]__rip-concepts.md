## **Course:** 200-301 CCNA

**RIP overview**

- RIP stands for **Routing Information Protocol**.
- It is an **Interior Gateway Protocol (IGP)**.
- It uses a **distance vector** approach.
- It is simple to configure and maintain.
- It is usually used on small networks.
- It lacks some advanced features found in protocols like OSPF or EIGRP.

**How RIP chooses routes**

- RIP uses **hop count** as its metric.
- The route with the fewest hops is preferred.
- Routers learn routes from neighbors.
- Routers exchange routing information periodically.
- RIP sends the full routing table at regular intervals.
- This periodic exchange can consume bandwidth.
- It can also slow convergence.

**RIP versions**

- RIP has two common versions: **RIPv1** and **RIPv2**.
- Both versions use hop count.
- Both versions have an administrative distance of `120`.
- RIPv2 is **classless**.
- RIPv1 is **classful**.
- RIPv2 advertises subnet mask information.
- RIPv1 does not advertise subnet masks.
- RIPv2 uses multicast for updates.
- RIPv1 uses broadcast for updates.
- RIPv2 is backward compatible with RIPv1.

**RIPv1 vs RIPv2**

| Feature | RIPv1 | RIPv2 |
|---|---|---|
| Routing style | Classful | Classless |
| Subnet mask info | Not sent | Sent |
| Update method | Broadcast | Multicast |
| Compatibility | Baseline version | Backward compatible with RIPv1 |

**Operational notes**

- RIP relies on secondhand route information.
- That makes it less accurate than protocols with more advanced behavior.
- It can be enhanced with techniques such as split horizon and poison reverse.
- Route summarization is a relevant RIP topic in Cisco discussions.
- Cisco documentation includes RIP configuration, RIPv2 authentication, split horizon, route filtering, route summarization, and route redistribution.

> RIP is not always listed as a major 200-301 CCNA exam topic, but it can still appear in exam-related study material.

**Key terms**

- **Distance vector**: A routing method where routers learn destinations from neighbors.
- **Hop count**: The number of routers a packet crosses to reach a destination.
- **Classful**: Does not carry subnet mask information in updates.
- **Classless**: Carries subnet mask information in updates.
