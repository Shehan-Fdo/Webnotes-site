## **Course:** 200-301 CCNA

**Overview**

- **Distance vector** protocols learn routes from neighbors.
- They do not build a full topology map.
- They advertise routes as a distance and a direction.
- The direction is the next-hop router.
- The distance is a metric such as hop count.
- This approach is often described as routing by rumor.
- **Link state** protocols build a view of the network topology.
- They use that topology view to compute best paths.
- OSPF uses the SPF, or Dijkstra Shortest Path First, algorithm.
- Link state protocols generally scale better as networks grow.

**Distance Vector**

- Routers exchange routing information with directly connected neighbors.
- A router depends on neighbor information instead of full network knowledge.
- Distance vector protocols can suffer from temporary routing loops.
- They may converge slowly in some situations.
- They are often used in smaller networks with fewer routers.
- RIP is a common distance vector example.

**Link State**

- Routers share link state information to build a topology database.
- The database gives a broader view of the network.
- The router then runs a shortest-path calculation.
- OSPF is a link state routing protocol.
- Link state protocols are common in networks of many sizes.
- They are generally preferred when faster convergence and better scaling matter.

**Key Differences**

| Aspect | Distance Vector | Link State |
|---|---|---|
| Route knowledge | Learns from neighbors | Builds topology view |
| Path selection | Uses advertised distance and direction | Computes best path from database |
| Convergence | Can be slower | Typically faster |
| Loop risk | More prone to temporary loops | Less prone with topology-based design |
| Scaling | Better for small networks | Better for larger networks |

**CCNA Takeaways**

- Remember that distance vector relies on neighbor updates.
- Remember that link state relies on a topology database.
- Use **RIP** as a distance vector reference point.
- Use **OSPF** as a link state reference point.
- Treat OSPF as link state in normal CCNA study.
- Focus on how each protocol learns, stores, and uses route information.
