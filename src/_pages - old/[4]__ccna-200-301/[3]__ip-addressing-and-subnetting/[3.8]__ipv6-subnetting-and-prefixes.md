## **Course:** 200-301 CCNA

**IPv6 address structure**

- IPv6 uses **128 bits**.
- The large address space reduces the need to conserve addresses like IPv4.
- IPv6 subnetting follows different design habits than IPv4.
- A prefix length shows how many leading bits identify the network.

**Prefix and subnet basics**

- The prefix is written with CIDR notation, such as `/64`.
- The prefix length separates the network portion from the host portion.
- In IPv6, the subnet mask recommendation for end hosts is typically `/64`.
- A `/64` leaves 64 bits for the interface identifier.

**Why `/64` matters**

- `/64` is the standard boundary for individual IPv6 subnets.
- Many IPv6 features are designed around this fixed boundary.
- **SLAAC** (Stateless Address Autoconfiguration) depends on `/64`.
- Hosts can form their own interface identifier on a `/64` network.

**Subnetting mindset**

- IPv6 subnetting is not mainly about conserving addresses.
- You plan prefixes for structure and scalability.
- You usually do not need to worry about running out of host addresses inside a `/64`.
- The huge address space makes waste less of a concern than in IPv4.

**Practical design notes**

- Use `/64` for normal host networks.
- Treat the prefix as the key planning value.
- Keep subnet design simple and consistent.
- Avoid applying IPv4 conservation habits directly to IPv6.

**Key terms**

- **CIDR**: Classless Inter-Domain Routing.
- **SLAAC**: Stateless Address Autoconfiguration.
- **Interface identifier**: The host portion of an IPv6 address.
