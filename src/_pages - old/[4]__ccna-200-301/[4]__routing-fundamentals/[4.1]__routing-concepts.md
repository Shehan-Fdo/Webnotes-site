## **Course:** 200-301 CCNA

**Routing overview**

- Routing moves IP packets from a source network to a destination network.
- A host sends traffic to the **default gateway** when the destination is outside its subnet.
- The host uses the destination IP address of the final device and the MAC address of the router interface.
- The router examines the destination IP address and chooses an outgoing interface.

**Static routing**

- **Static routing** uses manually configured paths.
- The network administrator defines the route.
- Static routing works well in small networks with stable topology.
- Static routing becomes harder to maintain as the network grows or changes often.

**Dynamic routing**

- **Dynamic routing** lets routers exchange route information with other routers.
- Routers use routing protocols to learn and update routes.
- Dynamic routing reduces manual changes when topology changes or links fail.
- Dynamic routing is better suited to larger networks.

**Routing table**

- The router uses the **routing table** to make forwarding decisions.
- A routing table entry needs a destination and a pointer to reach it.
- The pointer can reference a directly connected interface or a next-hop router.
- The next hop eventually resolves to an outgoing interface.

**Route selection**

- Routers may learn the same destination from multiple sources.
- **Administrative distance** helps the router choose between routes from different sources.
- Lower administrative distance is preferred.
- **Metric** helps compare routes within a routing protocol.
- If two routes exist, the router uses the best available path based on these values.

**Common route sources**

- `C` means connected.
- `S` means static.
- `R` means RIP.
- `D` means EIGRP.
- `O` means OSPF.

**Verification**

- Use `show ip route` to view the routing table.
- Check the route source, destination, and next hop.
- Confirm the router has a valid path before troubleshooting forwarding issues.

**Practical takeaways**

- Use static routes for simple, stable networks.
- Use dynamic routing for networks that change often.
- Verify route selection when multiple paths exist.
- Remember that the router forwards based on the destination IP address, not the destination MAC address.
