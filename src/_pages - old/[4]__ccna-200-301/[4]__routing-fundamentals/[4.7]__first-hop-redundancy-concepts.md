## **Course:** 200-301 CCNA

**First Hop Redundancy Protocols**

- **First Hop Redundancy Protocol (FHRP)** protects the default gateway.
- It uses a **virtual IP address** and **virtual MAC address**.
- You deploy two or more routers for gateway redundancy.
- One router forwards traffic as the active device.
- Another router waits as the standby or backup device.
- If the gateway router fails, users keep connectivity through the remaining router.

**Why FHRP matters**

- A single gateway router creates a **single point of failure**.
- Gateway failure can interrupt user access to the Internet or other networks.
- FHRP reduces service interruption during router failure.
- It also helps when you need to change the default gateway without long downtime.

**Core FHRP behavior**

- Hosts use the virtual IP address as the default gateway.
- The virtual MAC address stays associated with the gateway role.
- Routers share responsibility for the same gateway identity.
- Traffic moves to the backup router when the active router stops responding.

**Common FHRP models**

- **HSRP** uses an active and standby router model.
- **VRRP** uses a master and backup router model.
- **GLBP** supports gateway redundancy and load balancing.
- HSRP and GLBP are Cisco proprietary.
- VRRP is an open standard.

**Protocol comparison**

| Protocol | Role model | Key idea |
| --- | --- | --- |
| HSRP | Active / standby | One router forwards traffic while another waits |
| VRRP | Master / backup | One router owns the gateway role with backups available |
| GLBP | Gateway / load balancing | Multiple routers can forward traffic |

**Practical takeaways**

- Use FHRP when you want high availability for LAN users.
- Expect seamless failover when the active gateway fails.
- Remember that FHRP is a Layer 3 redundancy concept.
- Focus on the virtual gateway identity, not the physical router address.

**Study focus**

- Know why a default gateway needs redundancy.
- Know the purpose of the virtual IP address.
- Know the difference between active, standby, master, and backup roles.
- Know that GLBP adds load balancing beyond basic redundancy.
