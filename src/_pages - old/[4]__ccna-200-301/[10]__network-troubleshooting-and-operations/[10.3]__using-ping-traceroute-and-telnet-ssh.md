## **Course:** 200-301 CCNA

**Troubleshooting tools**

- `ping` and `traceroute` are command-line tools used for network troubleshooting.
- Use them to diagnose connectivity problems in IPv4 and IPv6 environments.
- `ping` tests reachability between devices.
- `traceroute` helps you trace the path traffic takes across the network.
- These tools support a systematic troubleshooting process.

**Cisco troubleshooting approach**

- Cisco recommends a methodical process for resolving network issues.
- Define the problem first.
- Gather detailed information next.
- Consider the probable cause.
- Devise a plan to solve the problem.
- Implement the plan.
- Observe the results.
- Repeat the process if needed.
- Document the changes you make.

**Ping**

- `ping` sends test traffic to check whether a destination responds.
- Use it when you want a quick reachability check.
- A successful reply suggests basic connectivity.
- A failed reply means you should investigate the path, addressing, or device state.

**Traceroute**

- `traceroute` shows the route packets follow toward a destination.
- Use it to identify where traffic stops or changes path.
- It helps narrow down the point of failure during troubleshooting.
- Compare the observed path with the expected path.

**Telnet**

- `Telnet` is a remote-access protocol for communicating with a device.
- It uses TCP port `23` by default.
- You need a Telnet client on your local system.
- The remote device must run a Telnet server.
- Telnet sends usernames and passwords in clear text.
- Avoid Telnet when security matters.

**SSH**

- `SSH` stands for Secure Shell.
- It is used to remotely access and manage devices.
- SSH uses encryption.
- Prefer SSH over Telnet for secure remote administration.
- Use SSH when you need remote CLI access with better protection for credentials and session data.

**Using remote access tools**

- Use `telnet IP_ADDRESS` or `telnet HOSTNAME` to start a Telnet session on Windows.
- Use SSH for secure remote access to Cisco devices.
- Verify that the target device allows the service before you troubleshoot the client.
- If you cannot reach a device with `ping`, remote access tools may also fail.

**Practical troubleshooting flow**

1. Test basic reachability with `ping`.
2. Trace the path with `traceroute` if `ping` fails or the path is unclear.
3. Try `SSH` for secure remote access when the device is reachable.
4. Use `Telnet` only when required and when security risk is acceptable.
5. Document what you changed and what you observed.

> Telnet is insecure because it sends credentials in clear text. Prefer `SSH` for routine administration.

**Key takeaways**

- `ping` checks reachability.
- `traceroute` reveals the path.
- `Telnet` provides remote access but is insecure.
- `SSH` provides remote access with encryption.
- Use these tools as part of a structured troubleshooting method.
