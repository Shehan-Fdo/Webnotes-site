## **Course:** 200-301 CCNA

**STP overview**

- **Spanning Tree Protocol (STP)** prevents Layer 2 loops in switched networks.
- STP is standardized as IEEE 802.1D.
- STP blocks some switch ports on redundant links.
- STP keeps a loop-free logical topology.
- STP lets you keep redundant links for **fault tolerance**.

**Why STP matters**

- Redundant links can create a **bridging loop**.
- A loop can cause frames to circulate endlessly.
- Broadcast traffic can multiply across switches.
- A **broadcast storm** can overload the network.
- STP prevents this by placing one port in a blocking state.

**How STP works**

- STP allows only one active path between two stations.
- STP selects which ports forward traffic.
- STP blocks at least one redundant path.
- The blocked port does not forward frames.
- The blocked port still helps preserve redundancy if the active path fails.

**Port roles and behavior**

- A **root port** is the best path to the root bridge on a non-root switch.
- A **designated port** forwards traffic on a segment.
- The root bridge has all ports as designated ports.
- An **alternate port** provides a backup path to the root bridge.
- A disabled port is administratively shut down and does not participate in STP.

**Port states**

- STP uses port states to control forwarding safely.
- A blocking state stops a port from forwarding traffic.
- A listening state prepares the port for participation.
- A learning state allows the switch to learn MAC addresses.
- A forwarding state allows normal traffic flow.

**Practical outcome**

- STP gives you redundancy without unstable loops.
- STP improves network resilience in switched Ethernet designs.
- You should expect one or more ports to remain blocked in a redundant topology.
