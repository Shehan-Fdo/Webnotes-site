## **Course:** 200-301 CCNA

**EtherChannel overview**

- EtherChannel bundles multiple physical links into one logical link.
- It increases **bandwidth** between devices.
- It adds **redundancy** if one link fails.
- It helps prevent a single link from becoming a bottleneck.
- It is used between switches or between a switch and a router.

**Core idea**

- The bundled links act as one logical connection.
- Traffic can use multiple links in the bundle.
- The network keeps forwarding if one member link goes down.
- In Cisco documentation, `port-channel` refers to the logical interface for the bundle.

**Protocols and modes**

- **PAgP** stands for Port Aggregation Protocol.
- PAgP is Cisco proprietary.
- **LACP** stands for Link Aggregation Control Protocol.
- LACP is an IEEE open standard.
- PAgP uses `auto` and `desirable` negotiation modes.
- `auto` responds to negotiation but does not start it.
- `desirable` actively starts negotiation.
- EtherChannel can also be configured statically with `on`.

**Configuration consistency**

- Member ports must match key settings.
- Keep **speed** and **duplex** the same on all member interfaces.
- Keep the same switching mode on all member interfaces.
- Use the same access VLAN on access ports.
- Use the same native VLAN and allowed VLANs on trunk ports.
- Keep spanning tree settings consistent.

**Restrictions and limits**

- All ports in an EtherChannel must belong to the same VLAN or be trunk ports.
- You cannot configure a voice VLAN on a port-channel or member interface.
- You cannot use Layer 2 settings on a Layer 3 port.
- Some platform features are not supported on EtherChannel interfaces.
- Cisco notes that some load-balancing options depend on platform support.

**Verification focus**

- Check that the logical bundle forms correctly.
- Check that all member links match the required settings.
- Check that traffic can continue when one link fails.
- Check the port-channel interface, not just the physical ports.

**Practical takeaway**

- Use EtherChannel when you need more capacity and resilience.
- Treat the bundle as one logical link for design and troubleshooting.
- Match interface settings before you try to form the channel.
