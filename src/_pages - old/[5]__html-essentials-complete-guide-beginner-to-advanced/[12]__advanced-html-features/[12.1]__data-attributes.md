## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**What data attributes are**

- `data-*` attributes store extra information on standard HTML elements.
- They let you associate data with an element without adding visual content.
- They are useful when the data has no defined HTML meaning.
- They help avoid non-standard attributes and extra DOM properties.

**Syntax and naming**

- Any attribute whose name starts with `data-` is a data attribute.
- Use the `data-` prefix on the element you want to annotate.
- Keep the rest of the name descriptive and consistent.
- In HTML, attribute names use lowercase letters after conversion rules apply.
- Allowed characters include letters, numbers, dashes, periods, colons, and underscores.

**Common use cases**

- Store IDs, indexes, or other metadata on elements.
- Attach product numbers or record identifiers to content.
- Keep machine-readable values alongside human-readable text.
- Use them when the page needs hidden structured data for scripts.

**Example pattern**

- `<article data-columns="3" data-index-number="12314" data-parent="cars">` shows multiple custom values on one element.
- `<li data-id="10784">` stores a single identifier on a list item.
- `<data value="398">Mini Ketchup</data>` links visible text to a machine-readable value.
- The `<data>` element is different from `data-*` attributes.
- Use `<data>` when you need a machine-readable translation of content.

**JavaScript access**

- Read custom data attributes through `HTMLElement.dataset`.
- `dataset` provides read/write access to `data-*` values as a `DOMStringMap`.
- The `dataset` property itself is read-only.
- Write to individual `dataset` properties instead of replacing the whole object.
- You can also use `getAttribute()` with the full attribute name.

**Name mapping between HTML and JavaScript**

- In HTML, the attribute name includes the `data-` prefix.
- In JavaScript, the property name drops the `data-` prefix.
- The HTML and JavaScript names map to the same underlying data.
- Use consistent naming so the mapping stays predictable.

**Practical guidance**

- Use `data-*` for proprietary or app-specific information.
- Keep the values small and focused on the element they describe.
- Do not use them for content that needs visible semantic meaning.
- Use `time` for time- or date-related content instead of `<data>`.

**Key takeaways**

- `data-*` attributes extend HTML without changing page appearance.
- `dataset` is the standard way to access them in scripts.
- `<data>` is a separate element for machine-readable content values.
- Choose the simplest option that matches your data needs.
