## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**What the DOM is**

- The **Document Object Model** (DOM) represents an HTML document in memory.
- It connects web pages to scripts and programming languages.
- The DOM models the document as a logical tree.
- Each branch ends in a node.
- Nodes contain objects that scripts can work with.

**Why HTML authors should care**

- The DOM lets you change document structure, style, and content.
- Browser APIs expose the DOM for programmatic access.
- A web page is the same document whether you view it in the browser or as HTML source.
- HTML authors use the DOM to understand how markup becomes interactive content.

**Core DOM relationships**

- The DOM tree organizes nodes by relationship.
- A **root** node starts the tree.
- A **parent** node contains child nodes.
- A **child** node belongs to a parent.
- **Sibling** nodes share the same parent.
- A **descendant** is any node below another node in the tree.

**HTML DOM API**

- The **HTML DOM API** defines interfaces for HTML elements and related types.
- It covers access to and control of HTML elements through the DOM.
- It includes form data manipulation.
- It includes interaction with `canvas` content.
- It includes media handling for `<audio>` and `<video>`.
- It also supports drag-and-drop and browser history access.
- It connects with other web platform features such as Web Components, Web Storage, Web Workers, WebSocket, and server-sent events.

**Common DOM scripting tasks**

- You can get references to DOM nodes.
- You can create new nodes.
- You can add and remove nodes.
- You can add and remove attributes.
- You can change CSS styles with JavaScript.
- You can attach event handlers to nodes.
- Event handlers run when the related event fires.

**Practical browser context**

- The browser exposes key objects such as `Window`, `Document`, and `Navigator`.
- `Document` gives you access to the page structure.
- `Window` represents the browser environment around the document.
- `Navigator` provides browser-related information.

**Working approach for HTML authors**

1. Identify the element or node you need.
2. Use browser APIs to read or change it.
3. Update structure, attributes, or styles as needed.
4. Attach events when the page needs interaction.
5. Test the result in the browser.

**Key takeaway**

- The DOM is the bridge between HTML markup and dynamic behavior.
- Understanding the tree helps you write HTML that is easier to inspect, script, and maintain.
