## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**What the `title` element does**

- The `<title>` element defines the document title.
- Browsers show the title in the tab or title bar.
- The element belongs inside the `<head>` block.
- It contains text only.
- HTML tags inside `<title>` are treated as plain text.

**Why the title matters**

- A descriptive title helps search engine optimization (**SEO**).
- Search engines may use title content when ranking pages.
- The title is often the first text users see in search results.
- Longer, descriptive titles usually perform better than short, generic ones.

**Where to place it**

- Put `<title>` inside `<head>`.
- Keep metadata in the head, not in the visible page content.
- Use the head for information browsers and search engines need to render and describe the page.

**Working with the document title in JavaScript**

- `document.title` gets or sets the document title text.
- `document.querySelector('title')` returns the `<title>` element object.
- Do not use `document.title.text`.
- Set the title with `document.title = "Hello world!"` when you want to change it.

**Related `title` attribute use**

- The `title` global attribute provides advisory information for the element it belongs to.
- The `title` attribute can label `<iframe>` elements for assistive technology.
- The `title` attribute can label controls in data tables.
- The `title` attribute on `<link rel="stylesheet">` can create an alternate stylesheet.
- Use `title` carefully because it is not the same as the document `<title>` element.

**Quick usage reminders**

- Use one clear page title per document.
- Make the title specific to the page content.
- Keep the title text concise and descriptive.
- Treat the title as metadata, not visible page content.
