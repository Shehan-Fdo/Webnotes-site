## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**What social metadata does**

- Social metadata helps platforms interpret your page when someone shares a URL.
- It lives in the HTML `<head>` with other metadata such as `<title>` and description.
- It supports richer link previews instead of plain text shares.
- Open Graph is a protocol for turning a web page into a rich object in a social graph.
- Twitter Cards add Twitter-specific sharing metadata.

**Where to place it**

- Add the tags in the `<head>` of your HTML document.
- Keep the page metadata close to other document metadata like title, description, and keywords.
- Use the `lang` attribute on the opening `<html>` tag as part of good document metadata practice.

**Open Graph basics**

- Open Graph uses `<meta>` tags with `property` attributes.
- The protocol is designed to be simple for developers to implement.
- The required Open Graph properties include `og:title` and `og:type`.
- Other properties may be required depending on the object type.
- Common Open Graph tags also include `og:description`, `og:url`, and `og:site_name`.

**Twitter Cards basics**

- Twitter Cards use structured markup in the page head.
- They help control the snippet shown when your URL is shared on Twitter.
- They can improve the image, title, and description shown in the preview.
- Twitter Cards are separate from Open Graph, but they serve a similar sharing goal.

**Practical implementation**

1. Add the core metadata tags in the `<head>`.
2. Set `og:title` to the page title you want shared.
3. Set `og:type` to describe the content type.
4. Add `og:description` and `og:url` for clearer previews.
5. Add Twitter Card tags for Twitter-specific sharing behavior.
6. Verify that the shared preview shows the intended title, description, and image.

**Why it matters**

- Rich previews can make shared links more informative.
- Better previews can improve how content appears on social platforms.
- Metadata also helps search engines and browsers understand the document.

**Implementation notes**

- Some systems can auto-generate Open Graph and Twitter Card tags from page metadata.
- You can override generated values when you need custom sharing text or images.
- If a platform does not use a tag exactly as expected, keep the metadata consistent and practical.

> Use conservative metadata values when you are unsure, especially for title, description, and image.
