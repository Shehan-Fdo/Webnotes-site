## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Semantic structure**

- HTML gives a page structure and meaning.
- Semantic elements describe what content does or where it belongs.
- `header`, `nav`, `main`, and `footer` help browsers and developers read the page layout.
- These elements replace generic containers when you need clearer meaning.

**`<header>`**

- `<header>` represents introductory content.
- It often contains a heading, logo, search form, or navigational aids.
- You can use it at the top of the page body or inside a section.
- A section can have its own `<header>`.

**`<nav>`**

- `<nav>` represents a section for navigation links.
- Use it for menus, tables of contents, or indexes.
- It can link within the current document or to other documents.
- Not every group of links needs `<nav>`.

**`<main>`**

- `<main>` contains the primary content of the page.
- It should hold content that is unique to that document.
- It excludes repeated site content such as navigation, logos, and copyright information.
- Use only one `<main>` per page.

> Do not place more than one `<main>` element on a page.

**`<footer>`**

- `<footer>` represents the footer for its nearest sectioning ancestor.
- It often contains author information, copyright data, or related links.
- You can include an `<address>` element inside a footer for author details.
- A page can have more than one footer when sections need their own closing content.

**Practical page layout**

- Put shared top content in `<header>`.
- Put site or section links in `<nav>`.
- Put unique page content in `<main>`.
- Put closing or reference content in `<footer>`.
- Keep the structure consistent so the page reads clearly.

**Why semantic placement matters**

- Semantic elements improve readability for developers.
- They help browsers understand page purpose.
- They support cleaner, more maintainable HTML.
- They make page regions easier to identify.
