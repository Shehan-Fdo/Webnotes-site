## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**What the DOCTYPE declaration is**

- The DOCTYPE is the required `<!doctype html>` preamble at the top of an HTML document.
- It tells the browser the document uses HTML.
- The declaration is case-insensitive.
- Common forms include `<!doctype html>` and `<!DOCTYPE html>`.

**Why you include it**

- The DOCTYPE helps prevent the browser from switching into **quirks mode**.
- It encourages the browser to use a standards-compliant rendering mode.
- Without it, older browser behavior can affect layout and compatibility.
- Including it is a safe default for modern HTML pages.

**Browser rendering modes**

- Browsers use different rendering modes to handle modern and legacy pages.
- **Quirks mode** emulates older browser behavior.
- **No-quirks mode** follows current standards more closely.
- The DOCTYPE supports the browser’s best-effort attempt to follow specifications.

**Basic placement**

- Put `<!doctype html>` at the very top of the document.
- Place it before the `<html>` element.
- Keep it in every HTML file you create.
- Use the lowercase form in examples if you want to match common convention.

**Practical notes**

- The DOCTYPE is not a visible page element.
- It is part of the document’s opening structure.
- It does not replace valid HTML structure.
- It works alongside the rest of the page markup.

**Quick reminder**

- Use `<!doctype html>` as the first line of your HTML document.
- Treat it as a standard starting point for modern web pages.
