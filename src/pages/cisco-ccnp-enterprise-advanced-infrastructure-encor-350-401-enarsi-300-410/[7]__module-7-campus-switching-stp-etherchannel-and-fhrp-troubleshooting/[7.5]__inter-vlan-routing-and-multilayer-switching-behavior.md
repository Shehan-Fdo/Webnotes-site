## **Course:** Cisco CCNP Enterprise Advanced Infrastructure (ENCOR 350-401 & ENARSI 300-410)

**Inter-VLAN routing fundamentals**

Inter-VLAN routing lets devices in different VLANs communicate through a Layer 3 device. A VLAN is a Layer 2 broadcast domain, so traffic cannot cross VLAN boundaries without routing. On Cisco networks, you typically use one of three designs:

- A **legacy router with physical interfaces**, where each router interface connects to one VLAN.
- **Router-on-a-stick**, where one router physical interface carries multiple VLANs over an 802.1Q trunk.
- A **multilayer switch**, where switched virtual interfaces (SVIs) or routed ports perform the Layer 3 function.

The key exam idea is simple: if the device only switches at Layer 2, it cannot route between VLANs. If it has Layer 3 capability, it can make a routing decision and then rewrite the Layer 2 header for the egress VLAN or next hop.

**How a multilayer switch decides to bridge or route**

A multilayer switch can do both Layer 2 switching and Layer 3 routing. The forwarding path depends on the destination and the interface type:

- If the destination MAC address belongs to a host in the same VLAN, the switch bridges the frame at Layer 2.
- If the destination IP address belongs to a different subnet, the switch routes the packet at Layer 3.
- If the ingress interface is a routed port, the switch treats the traffic as Layer 3 from the start.
- If the ingress interface is an access port or trunk carrying VLANs, the switch first classifies the frame into a VLAN, then decides whether to bridge or route.

In practice, the switch receives an Ethernet frame on an ingress port assigned to a VLAN or on a trunk subinterface context. It does not “receive on a VLAN” physically; VLAN is the logical forwarding domain used for classification.

A useful mental model is:

1. Ingress frame arrives.
2. The switch identifies the VLAN or routed interface context.
3. The switch checks whether the destination is local in that VLAN.
4. If not local, the switch performs a routing lookup.
5. The switch rewrites the Layer 2 header for the next hop and forwards the frame.

**SVIs and routed ports**

A switched virtual interface, or **SVI** (Switched Virtual Interface), is a Layer 3 interface tied to a VLAN. It is commonly used as the default gateway for hosts in that VLAN.

Example:

```cisco
interface vlan 10
 ip address 10.10.10.1 255.255.255.0
 no shutdown
```

An SVI is not a physical port. It represents the Layer 3 gateway for the VLAN. Hosts in VLAN 10 use `10.10.10.1` as their default gateway, and the switch routes traffic from that SVI to other VLANs or routed destinations.

A **routed port** is a physical switch port converted to Layer 3 with `no switchport`:

```cisco
interface gigabitEthernet1/0/48
 no switchport
 ip address 10.1.1.2 255.255.255.252
 no shutdown
```

Use routed ports when you want a point-to-point Layer 3 link, such as to a router, firewall, or another switch. On some platforms or interface types, routed-port support may be limited, so you should verify platform capability before designing around it.

**Layer 2 switch versus multilayer switch behavior**

A Layer 2 switch forwards frames within a VLAN only. It cannot route between subnets unless you add an external router. A multilayer switch can do both, but only if Layer 3 routing is enabled.

Important distinction:

- On a **Layer 2 switch**, `ip default-gateway` is used for management traffic only.
- On a **Layer 3 switch**, `ip routing` enables routing between interfaces and SVIs.
- When `ip routing` is enabled, `ip default-gateway` is ignored for forwarding decisions.

Example management-only configuration on a Layer 2 switch:

```cisco
interface vlan 99
 ip address 192.0.2.10 255.255.255.0
 no shutdown
ip default-gateway 192.0.2.1
```

If you later enable `ip routing`, the switch stops using `ip default-gateway` for routed traffic. That is a common exam trap.

**Complete inter-VLAN design on a multilayer switch**

A typical end-to-end design includes access ports, a trunk uplink, SVIs, and host gateway settings.

Example topology:

- Host A in VLAN 10: `10.10.10.50/24`
- Host B in VLAN 20: `10.10.20.50/24`
- Multilayer switch:
  - `VLAN 10` SVI: `10.10.10.1/24`
  - `VLAN 20` SVI: `10.10.20.1/24`
  - Trunk to access switch or another switch

Configuration example:

```cisco
vlan 10
 name USERS
vlan 20
 name SERVERS

interface gigabitEthernet1/0/1
 switchport mode access
 switchport access vlan 10
 spanning-tree portfast

interface gigabitEthernet1/0/2
 switchport mode access
 switchport access vlan 20
 spanning-tree portfast

interface gigabitEthernet1/0/48
 switchport mode trunk
 switchport trunk allowed vlan 10,20

interface vlan 10
 ip address 10.10.10.1 255.255.255.0
 no shutdown

interface vlan 20
 ip address 10.10.20.1 255.255.255.0
 no shutdown

ip routing
```

Host gateway settings:

- VLAN 10 hosts use `10.10.10.1`
- VLAN 20 hosts use `10.10.20.1`

When Host A sends traffic to Host B, Host A first sends the frame to its default gateway MAC address, not directly to Host B. The switch receives the frame on an access port in VLAN 10, routes the packet to VLAN 20, and then forwards it out the VLAN 20 SVI context with a new Layer 2 header appropriate for the egress next hop.

> The switch does not simply “rewrite the destination MAC to Host B’s MAC” in every case. It rewrites the Layer 2 header to the resolved next-hop MAC on the egress VLAN or routed adjacency, which may be Host B only if Host B is directly connected and the adjacency resolves that way.

**Packet flow through a multilayer switch**

For same-VLAN traffic:

- Host A ARPs for Host B.
- The switch learns the MAC addresses in VLAN 10.
- The frame is switched at Layer 2.
- No routing lookup occurs.

For inter-VLAN traffic:

- Host A sends the frame to the default gateway MAC for VLAN 10.
- The switch receives the frame on an access port in VLAN 10.
- The switch strips the Layer 2 header, examines the IP packet, and performs a routing lookup.
- The switch selects the outgoing interface, often an SVI for VLAN 20.
- The switch resolves the next-hop MAC using ARP if needed.
- The switch rewrites the Layer 2 header and forwards the packet.

This is why multilayer switching is fast: the routing decision and adjacency rewrite are typically done in hardware.

**Router-on-a-stick**

Router-on-a-stick is the classic design for inter-VLAN routing on a Layer 2 switch. The switch cannot route by itself, so it sends all inter-VLAN traffic to a router over a trunk link. The router uses subinterfaces, one per VLAN.

Switch side:

```cisco
interface gigabitEthernet1/0/24
 switchport mode trunk
 switchport trunk allowed vlan 10,20
```

Router side:

```cisco
interface gigabitEthernet0/0
 no ip address
 no shutdown

interface gigabitEthernet0/0.10
 encapsulation dot1Q 10
 ip address 10.10.10.1 255.255.255.0

interface gigabitEthernet0/0.20
 encapsulation dot1Q 20
 ip address 10.10.20.1 255.255.255.0
```

Key points:

- The physical router interface often has `no ip address`.
- Each subinterface maps to one VLAN with `encapsulation dot1Q`.
- The switch port facing the router must be a trunk.
- Hosts still use the router subinterface IP as their default gateway.

Use router-on-a-stick when you have a Layer 2 switch and no multilayer switching capability. It is simple, but it can become a bottleneck because all inter-VLAN traffic shares one physical link.

**Legacy external-router design**

Before router-on-a-stick became common, a router could use separate physical interfaces, one per VLAN. Each router interface connected to an access port in a single VLAN.

Characteristics:

- One router interface per VLAN.
- No trunking required.
- Simple to understand, but inefficient in port usage.
- Limited scalability because every VLAN consumes a router port.

This design still matters conceptually because it reinforces the rule that a Layer 2 switch cannot route on its own.

**`ip routing` and forwarding behavior**

On Cisco multilayer switches, `ip routing` controls whether the switch performs Layer 3 forwarding between interfaces.

- If `ip routing` is disabled, the switch can still have SVIs for management, but it does not route between them.
- If `ip routing` is enabled, the switch can route between SVIs and routed ports.
- Some platforms may default differently, so always verify the running configuration.

Useful verification commands:

```cisco
show running-config | include ^ip routing
show ip interface brief
show ip route
show interfaces vlan 10
```

Interpretation:

- `show ip interface brief` confirms whether SVIs are up/up.
- `show ip route` confirms that connected routes for VLAN subnets exist.
- `show interfaces vlan 10` helps you verify line protocol state and encapsulation details.

**SVI state and common dependencies**

An SVI can be administratively up but line protocol down. That usually means the VLAN exists but the switch does not see active Layer 2 membership for that VLAN.

Typical conditions for an SVI to come up:

- The VLAN exists in the VLAN database.
- At least one port in that VLAN is active, or the VLAN is active on a trunk.
- The SVI is not shut down.

Example troubleshooting sequence:

```cisco
show vlan brief
show interfaces status
show interfaces trunk
show ip interface brief
```

If `interface vlan 20` stays down/down or up/down, check:

- Whether VLAN 20 exists.
- Whether any access ports in VLAN 20 are up.
- Whether VLAN 20 is allowed and active on the trunk.
- Whether the SVI is `shutdown`.

**Proxy ARP and troubleshooting**

**Proxy ARP** allows a router or multilayer switch to answer ARP requests on behalf of another host or subnet. It can make misconfigured hosts appear to work even when the default gateway is wrong or the subnet mask is incorrect.

Why this matters:

- It can hide host configuration errors.
- It can complicate troubleshooting because traffic may partially work.
- It may create unexpected Layer 2 and Layer 3 behavior in labs and exams.

In modern designs, you often disable proxy ARP on routed interfaces when you want stricter behavior, but you should know that default behavior can vary by platform and interface type.

**First-hop redundancy on SVIs**

First-hop redundancy protocols provide a resilient default gateway for hosts. The switch or router pair presents a virtual IP address, and hosts use that virtual IP as their gateway.

Common protocols:

- **HSRP** (Hot Standby Router Protocol)
- **VRRP** (Virtual Router Redundancy Protocol)
- **GLBP** (Gateway Load Balancing Protocol)

Behavior summary:

- In **HSRP**, one device is active and another is standby.
- In **VRRP**, one device is master and others are backups.
- In **GLBP**, multiple devices can share forwarding load.

Example HSRP on an SVI:

```cisco
interface vlan 10
 ip address 10.10.10.2 255.255.255.0
 standby 10 ip 10.10.10.1
 standby 10 priority 110
 standby 10 preempt
```

Host behavior:

- Hosts ARP for the virtual IP.
- The active device replies with the virtual MAC address.
- If failover occurs, the new active device takes over the virtual MAC behavior according to the protocol.

Important exam distinction:

- The active or master device does not “own all traffic” in a generic sense.
- It forwards traffic for the virtual gateway according to the protocol state.
- The exact MAC and failover behavior differs among HSRP, VRRP, and GLBP.

**IPv6 inter-VLAN routing**

IPv6 uses the same basic design principles, but the gateway and neighbor discovery behavior differ from IPv4.

Common IPv6 SVI example:

```cisco
ipv6 unicast-routing

interface vlan 10
 ipv6 address 2001:db8:10::1/64
 no shutdown

interface vlan 20
 ipv6 address 2001:db8:20::1/64
 no shutdown
```

Key points:

- `ipv6 unicast-routing` enables IPv6 forwarding.
- Hosts use the SVI IPv6 address as the default gateway.
- Neighbor Discovery Protocol (NDP) replaces ARP for address resolution.
- First-hop redundancy for IPv6 can also use HSRP, VRRP, or GLBP variants depending on platform support.

For IPv6 troubleshooting, verify:

```cisco
show ipv6 interface brief
show ipv6 route
show ipv6 neighbors
```

**Practical troubleshooting workflow**

When inter-VLAN routing fails, check the problem from the host outward.

1. Verify the host IP address, subnet mask, and default gateway.
2. Confirm the access port is in the correct VLAN.
3. Confirm the trunk allows the VLANs that need to pass.
4. Verify the SVI exists and is up/up.
5. Confirm `ip routing` is enabled on a multilayer switch.
6. Check ARP or neighbor tables.
7. Verify the routing table contains the connected VLAN subnets.
8. Test with `ping` and `traceroute`.

Useful commands:

```cisco
show vlan brief
show interfaces trunk
show ip interface brief
show ip route
show arp
ping 10.10.20.50
traceroute 10.10.20.50
```

If the host can ping its gateway but not another VLAN, the issue is often:

- Missing `ip routing`
- SVI down
- Wrong host gateway
- VLAN not allowed on trunk
- ACL blocking traffic
- Incorrect subnet mask

**Exam-relevant distinctions**

- A **Layer 2 switch** cannot perform inter-VLAN routing without an external router.
- A **multilayer switch** can route between SVIs when `ip routing` is enabled.
- A **router-on-a-stick** design requires a trunk between the switch and router.
- A **routed port** uses `no switchport` and behaves like a Layer 3 interface.
- A **default gateway** on hosts points to the SVI or router subinterface in the same VLAN.
- **Proxy ARP** can mask configuration mistakes.
- **First-hop redundancy** protects the gateway, not the end-to-end path.

**Quick comparison**

| Design | Where routing happens | Typical use | Limitation |
|---|---|---|---|
| Layer 2 switch + external router | Router | Small or legacy networks | Consumes router ports |
| Router-on-a-stick | Router subinterfaces | Simple VLAN routing on L2 switch | Single trunk bottleneck |
| Multilayer switch with SVIs | Switch | Campus access/distribution | Requires Layer 3 capability |
| Routed ports | Switch or router | Point-to-point L3 links | Not for host access VLANs |

**Configuration checklist**

Before you declare inter-VLAN routing complete, verify:

- VLANs exist on the switch.
- Access ports are assigned to the correct VLANs.
- Trunks carry the required VLANs.
- SVIs have IP addresses and are not shut down.
- `ip routing` is enabled when needed.
- Hosts use the correct default gateway.
- Any first-hop redundancy protocol is consistent across peers.
- ACLs and security features are not blocking traffic.

**Summary**

Inter-VLAN routing is the process of moving traffic between separate Layer 2 broadcast domains using a Layer 3 device. On Cisco platforms, that device may be a router, a router-on-a-stick subinterface, or a multilayer switch using SVIs and routed ports. The most important exam concept is the forwarding decision: the switch or router routes based on the destination IP, then rewrites the Layer 2 header for the next hop before forwarding. Understanding when traffic is bridged, when it is routed, and how `ip routing`, SVIs, trunks, and host gateways interact will help you troubleshoot almost every VLAN connectivity issue.
