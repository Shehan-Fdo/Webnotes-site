## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202) — V15

**Authentication, authorization, and accounting overview**

<abbr title="Authentication, Authorization, and Accounting">AAA</abbr> is a security framework for controlling and tracking user access within a computer network. It helps network administrators maintain security while still giving users access to the resources they need.

AAA supports three related goals:

- **Access control** by deciding who can use a resource
- **Policy enforcement** by applying rules to that access
- **Auditing** by recording what users do

The model follows a clear sequence: authentication first, then authorization, then accounting.

**Authentication**

Authentication identifies the user before access is granted. In practice, this usually means entering a valid username and password, but the key idea is verifying identity.

- **Identity check** confirms who the user is
- **Credentials** are the proof used during sign-in
- **Access request** happens only after identity is verified

Authentication answers the question: “Who are you?”

**Authorization**

Authorization happens after authentication succeeds. It determines what the verified user is allowed to do on the system or network.

- **Permissions** define allowed actions
- **Policies** control access to resources
- **Least access** is the practical goal: give users only what they need

Authorization answers the question: “What can you do?”

**Accounting**

Accounting tracks user activity after access is granted. It measures resource use and records actions for auditing, reporting, and billing when needed.

- **Usage tracking** records how much of a resource a user consumes
- **Audit data** supports security review and visibility
- **Billing support** is possible when services must be charged

Accounting answers the question: “What did you do?”

**AAA in practice**

AAA gives administrators control and visibility over user activity. It is important for effective access management, network management, and security because it helps prevent unauthorized access while documenting legitimate use.

> AAA is a framework, not a single login feature. You use it to verify identity, limit access, and record activity in a predictable order.

**Related protocol example**

Some systems use <abbr title="Terminal Access Controller Access-Control System Plus">TACACS+</abbr> to support AAA functions. In the provided evidence, the UFM Enterprise Appliance switch supports TACACS+.

**Quick sequence**

1. Authenticate the user.
2. Authorize the user’s permitted actions.
3. Account for the user’s activity.

**Memory aid**

- **Authentication** = identify
- **Authorization** = allow
- **Accounting** = record
