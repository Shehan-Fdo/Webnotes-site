## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202) — V15

**Encryption at rest**

Encryption scrambles data so only someone with the proper key can read it. In this topic, the focus is on protecting data at rest, such as data stored on hard disks, external drives, and thumb drives.

Encryption is useful for reducing the impact of theft or improper device disposal, but it does not remove every risk. You still need to manage keys and choose the right level of protection for the device and use case.

- **Data at rest** refers to stored data, not data moving across a network.
- Full-device encryption protects an entire volume instead of individual files.
- File-level encryption gives finer control, but it requires more attention to how you handle each file.

**BitLocker basics**

BitLocker is a Windows security feature that encrypts entire volumes. It helps protect data on lost, stolen, or improperly decommissioned devices by making the contents harder to access without authorization.

BitLocker is especially relevant when a device could be physically removed from your control. If an attacker can take the drive and connect it to another system, encryption helps reduce the chance of unauthorized access.

- **BitLocker** protects entire volumes on supported Windows systems.
- It helps mitigate exposure from stolen devices and removed drives.
- It strengthens file and system protections for data stored on the volume.

**Trusted Platform Module**

A Trusted Platform Module, or **TPM** (Trusted Platform Module), is a microchip on the motherboard that provides security-related functions, especially for encryption keys. It can create cryptographic keys and encrypt them so the keys can only be decrypted by the TPM.

This key protection is often described as wrapping or binding a key. The practical benefit is that the key is tied to the device’s hardware, which helps protect it from disclosure.

- **TPM** is hardware-based and communicates with the system over a hardware bus.
- It can store or protect keys so they are not easily exposed.
- It helps mitigate dictionary attacks and supports secure key handling.

**TPM-based BitLocker**

BitLocker commonly uses TPM-based protection. In TPM-only mode, the system can unlock the drive automatically when the expected hardware state is present.

That convenience comes with a trade-off: physical-access attacks may target recovery or reset workflows if the drive is not relocked as expected. For exam purposes, remember that TPM improves key protection, but it is still part of a broader security design.

> TPM-only mode is convenient, but it is not the same as having no physical attack surface. Secure recovery and device handling still matter.

**Encrypted hard drives**

Encrypted hard drives are self-encrypting drives that perform encryption in hardware. They combine the management benefits of BitLocker with hardware-level encryption, and they are designed to be transparent to the user.

Because the cryptographic work happens in hardware, these drives can improve BitLocker performance and reduce CPU usage and power consumption. That makes them useful in enterprise deployments where you want encryption with minimal productivity impact.

- **Self-encrypting drives** handle encryption on the drive itself.
- Hardware offload can reduce CPU load and power use.
- Faster encryption can make large-scale deployment easier.

**Granularity and trade-offs**

Encryption choices involve trade-offs between convenience, performance, and control. Encrypting each file individually gives more granular protection, while encrypting the whole device is simpler to manage and better suited to lost-device scenarios.

For A+ purposes, focus on matching the method to the risk:

- Use file-level protection when you need tighter control over specific data.
- Use full-disk protection when you want broad protection for a device.
- Use hardware-backed options when performance and deployment scale matter.

**Quick comparison**

| Method | Scope | Main benefit | Main trade-off |
|---|---|---|---|
| File encryption | Individual files | Fine-grained control | More management overhead |
| Full-disk encryption | Entire volume | Strong lost-device protection | Depends on key handling and device state |
| Hardware-encrypted drive | Drive hardware | Lower CPU use and better performance | Still needs proper management and policy |

**Practical exam takeaways**

When you see a question about protecting a lost laptop, think about full-volume encryption such as BitLocker. When you see a question about hardware-backed key protection, think about TPM. When you see a question about reducing encryption overhead, think about encrypted hard drives.

- **BitLocker** is the Windows volume-encryption feature to remember.
- **TPM** protects keys by binding them to the device.
- **Encrypted hard drives** offload encryption to hardware for efficiency.
- Encryption protects stored data, but key management remains critical.
