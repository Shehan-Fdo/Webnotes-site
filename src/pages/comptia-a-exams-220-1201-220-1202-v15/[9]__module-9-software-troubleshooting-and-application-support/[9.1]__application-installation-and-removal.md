## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202) — V15

**Application installation and removal overview**

Windows gives you multiple ways to remove apps and programs, and the path you use depends on where the app appears and how it was installed. Some apps are built into Windows and cannot be uninstalled. If an app is not working correctly, you can often try a repair before removing it.

- **Built-in apps** may not support removal.
- **Desktop apps** can often be removed from Settings or Control Panel.
- **Microsoft Store apps** may use different recovery options than desktop programs.

**Common uninstall paths in Windows**

You can remove apps from the Start menu, Settings, or Control Panel. If one method does not show the app, try another location.

1. Open `Start > All apps`, find the app, then press and hold or right-click it.
2. Select `Uninstall` from the Start menu if the option is available.
3. Open `Start > Settings > Apps > Installed apps`.
4. Select the app, choose `More > Uninstall`, and follow the prompts.
5. If the app does not uninstall from Settings, open `Control Panel`.
6. Go to `Programs > Programs and Features`, then right-click the program and select `Uninstall`.

**Troubleshooting blocked installs or removals**

If Windows blocks installation, uninstallation, or updating, built-in recovery options can help. Microsoft notes that incomplete installs or corrupted app data can cause these problems. A practical troubleshooting sequence starts with a restart and then moves to app repair or reset when available.

- Restart the PC from `Start > Power > Restart`.
- In `Settings > Apps > Installed apps`, open the app’s `Advanced options`.
- Select `Repair` first if the app supports it.
- Use `Reset` if repair does not solve the issue.
- Not all desktop apps support repair or reset.

> Use the Microsoft troubleshooting path for Windows programs. If the app came from Microsoft Store, use the Store-specific fix process instead.

**When an app will not uninstall normally**

Some uninstall problems require extra steps beyond the standard Windows interface. Vendor uninstall tools, Safe Mode, or a troubleshooter can help when the program is stuck, partially removed, or still running in the background.

- Close the app before uninstalling it if it is still active.
- Try the manufacturer’s uninstall tool when one is provided.
- Use Safe Mode if normal removal fails.
- Run the Microsoft Install or Uninstall troubleshooter when prompted.
- Third-party uninstallers can remove leftover traces, but use them carefully.

**Practical removal workflow**

A simple workflow helps you choose the least disruptive option first and escalate only when needed.

1. Try `Repair` if the app is malfunctioning but still installed.
2. Try `Uninstall` from `Settings`.
3. If needed, try `Control Panel > Programs and Features`.
4. If the app is still blocked, restart and try again.
5. Use Safe Mode, a vendor tool, or the Microsoft troubleshooter if the uninstall remains stuck.

**Key exam takeaways**

Application removal in Windows is not limited to one menu. You should know the common paths in `Start`, `Settings`, and `Control Panel`, plus the basic recovery steps for failed installs and removals. For troubleshooting, start with the simplest fix and move toward more advanced tools only when the standard uninstall path does not work.
