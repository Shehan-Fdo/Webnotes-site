## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202) — V15

**Productivity suite overview**

Microsoft 365 provides everyday productivity apps that work together for work, school, and personal use. The suite includes familiar tools such as Word, Excel, PowerPoint, Outlook, OneNote, Access, Publisher, Project, and Visio, depending on the plan and version. Microsoft also offers free web access to core apps, which is useful when you need browser-based editing without a full desktop install.

- **Word** supports document creation and editing.
- **Excel** supports data organization, formulas, tables, and analysis.
- **PowerPoint** supports slide creation and presentation design.
- **Outlook** supports email and related productivity workflows.
- **OneDrive** provides cloud storage for files and sharing.

Microsoft 365 for the web lets you use Word, Excel, and PowerPoint in a browser. It also supports real-time collaboration from any device, which makes it practical when users need quick access without installing software.

**Access and licensing**

Microsoft 365 can be used through a subscription or, for some features, through free web access with a Microsoft account. The free web experience includes web and mobile app access, plus 5 GB of cloud storage in OneDrive. Users can also access apps across browser, iOS, and Android devices.

> If a user cannot access an app or feature, check the account, subscription, and device context first. Access problems are often tied to sign-in state or licensing rather than the app itself.

**Common support tasks**

Support for productivity suites often starts with basic app health checks. Microsoft documents repair options for Office applications, and Microsoft Learn provides troubleshooting guidance for Microsoft 365 Apps issues such as damaged documents, blank Excel openings, and account-related sign-in conflicts.

1. Confirm the user is signed in with the correct Microsoft account.
2. Check whether the issue affects one app or the whole suite.
3. Verify whether the app is current and whether updates are available.
4. Use the Office repair option if the app will not start or behaves unpredictably.
5. Escalate to deeper troubleshooting if the issue persists after repair.

**Repair and recovery**

Microsoft Support provides repair options for Office applications across many Microsoft 365 and Office versions. Repair is a standard response when apps fail to launch, crash, freeze, or show repeated errors. In practice, you use repair to restore missing or damaged program files and settings without immediately reinstalling everything.

```text
Control Panel > Programs > Programs and Features > Microsoft Office > Change
```

- **Quick Repair** is the faster first attempt.
- **Online Repair** is more thorough when Quick Repair does not resolve the issue.
- **Update checks** help when crashes or freezes appear after a version change or patch issue.
- **Add-ins** can cause instability, so disabling them is a useful test when an app misbehaves.

**Troubleshooting patterns**

Microsoft Learn groups common Microsoft 365 Apps issues by product area, which helps you narrow the problem quickly. For example, Word issues may involve damaged documents or unsaved work, while Excel issues may involve blank launches or workbook-saving problems. PowerPoint issues can involve damaged presentations or export settings.

- **Word**: recover unsaved documents or work with damaged files.
- **Excel**: investigate formulas, blank launches, or save failures.
- **PowerPoint**: check presentation damage, fonts, or export behavior.
- **OneNote**: verify notebook location and file-size limits.
- **Access**: review relationships, normalization, and database errors.

**Practical support approach**

When you troubleshoot a productivity suite issue, start with the simplest explanation and move outward. Many problems are client-side, such as account permissions, local app corruption, or sync issues. That means you should verify the user context before assuming a service outage or product defect.

- Check sign-in and account selection.
- Confirm whether the issue is local to one device.
- Test the web version if the desktop app fails.
- Repair the app if the problem persists.
- Use product-specific troubleshooting when the symptom points to one app.

**Key takeaways**

Microsoft 365 support combines app knowledge, account verification, and repair-based troubleshooting. For A+ support scenarios, focus on whether the user needs web access, desktop access, or a repaired installation. The most useful first steps are usually account checks, update checks, and Office repair.
