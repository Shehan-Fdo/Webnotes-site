## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202) — V15

**File access and privacy controls**

Windows apps may request access to your file system so they can work with the same files and folders you can access. You decide whether to allow or deny that access, and you can turn it off later if needed. This matters when an app can no longer reach files it previously used or when a workplace-managed device restricts access.

- **File system access** controls whether an app can reach your files and folders.
- In Windows 10, check `Start > Settings > Privacy > File system`.
- In Windows 11, check `Start > Settings > Privacy & security > File system`.
- Work devices or devices with a work account may already have this access turned off.

> If an app cannot open or save files as expected, verify both the app’s permission and the user’s access to the target folder.

**Permissions and shared folder access**

Permission problems often appear when a user can open a folder locally but gets denied from another computer. In shared-folder scenarios, the share permissions and the underlying folder permissions both matter. Granting broad access does not always fix the issue if the effective permissions still block the remote user.

- Check that the intended user has access to both the share and the folder.
- Test access from the remote computer, not only from the server or host machine.
- Avoid using an administrator account for routine access when a standard user account should work.
- If access works locally but not remotely, focus on the share path, user account, and effective permissions.

**File associations and default programs**

A file extension identifies the file type, and Windows uses file associations to decide which program opens it by default. Problems happen when the extension is wrong or when the correct extension opens in the wrong application. For example, a Word document may open in a PDF reader if the association is incorrect.

- A file extension such as `.pdf`, `.docx`, or `.jpg` helps Windows identify the file type.
- A file can fail to open correctly if the extension was renamed incorrectly.
- A file can have the correct extension but still be linked to the wrong default app.
- Fix the extension first when the filename itself is wrong, then correct the default program if needed.

**Troubleshooting approach**

Use a simple sequence so you do not confuse file corruption, permissions, and association problems. Start with the file name and extension, then verify access rights, and finally confirm the default application.

1. Check whether the file extension matches the actual file type.
2. Confirm the user has permission to access the file or shared folder.
3. Verify the file opens with the correct default application.
4. Change the association if Windows keeps opening the file in the wrong program.
5. Re-test from the affected device or user account.

**Practical examples**

A downloaded file that opens in the wrong app usually points to a file association issue. A file that appears inaccessible over the network but works on the host machine usually points to permissions or share configuration. A file with the wrong extension may need to be renamed before any application can open it correctly.

- Wrong extension: rename the file to the correct extension.
- Wrong app: change the default program for that file type.
- Access denied: review share and folder permissions.
- App cannot reach files: review Windows file system privacy settings.
