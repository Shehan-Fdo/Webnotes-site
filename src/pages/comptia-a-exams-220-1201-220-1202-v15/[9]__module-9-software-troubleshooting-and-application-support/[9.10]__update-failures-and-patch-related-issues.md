## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202) — V15

**Windows Update failure basics**

Windows Update keeps Windows secure, stable, and current, but updates can fail to install. In A+ troubleshooting, you focus on practical checks first: confirm the system is ready for the update, then use built-in repair tools before moving to recovery options.

- **Windows Update** problems can block security and feature updates.
- A failed patch can affect both system stability and protection against vulnerabilities.
- Start with the least disruptive fix and move toward more invasive recovery steps only if needed.

**Pre-update checks**

Before you attempt another update, verify the system is ready. A stable setup reduces the chance of repeat failures and helps you separate environmental issues from update corruption.

- Make sure the device meets the update requirements.
- Confirm you have enough free disk space; a practical target is `10-15 GB`.
- Check that the internet connection is stable.
- Back up important personal files before making changes.
- Disconnect external devices if they are not needed.
- Keep laptops on AC power during the update process.

**Built-in troubleshooting tools**

Windows includes automated troubleshooting that can identify and repair common update issues. Microsoft recommends starting with the Windows Update troubleshooter in the Get Help app on Windows 11.

- Use the **Windows Update troubleshooter** first when updates fail.
- Let the tool run diagnostics and apply fixes automatically when available.
- If the issue remains, review the most common Windows Update problems and follow the matching guidance.
- On older Windows versions or mobile devices, use the general troubleshooting steps instead.

> If the troubleshooter cannot resolve the issue, continue with broader troubleshooting rather than repeatedly rerunning the same step.

**Common repair approach**

The built-in troubleshooter can correct some update database or corruption-related issues. Dell’s guidance notes that the troubleshooter can identify and repair common problems that prevent Windows Update from working properly.

1. Open the troubleshooting area in Windows.
2. Select the **Windows Update** troubleshooter.
3. Open **Advanced** options if available.
4. Enable **Apply Repair Automatically**.
5. Run the troubleshooter and review the results.
6. Apply any recommended fixes and retry the update.

**Rollback and recovery**

If a recent update causes problems, rollback or recovery may be the practical next step. Microsoft Q&A references the idea of returning to a previous version, but availability depends on the device state and the time since the update.

- Use rollback only when the update itself is the likely cause of the issue.
- Recovery options may not always be available after the rollback window passes.
- If rollback is unavailable, continue with other repair or recovery methods supported by the system.

**Troubleshooting mindset for A+**

For exam scenarios, treat update failures as a structured troubleshooting problem. You first verify prerequisites, then use built-in tools, and finally consider rollback or recovery if the patch introduced the issue.

- **Document** what changed before the failure.
- **Test** one fix at a time so you can identify what worked.
- **Escalate** only after basic checks and automated repair fail.
- **Protect data** before attempting invasive recovery actions.

**Key takeaways**

Windows update issues are usually handled by a layered approach: prepare the system, run the troubleshooter, and then move to rollback or recovery if needed. This keeps the process efficient and reduces the risk of unnecessary changes.

- Start with readiness checks and backups.
- Use the built-in Windows Update troubleshooter early.
- Retry the update after each successful fix.
- Consider rollback when a recent patch clearly caused the problem.
