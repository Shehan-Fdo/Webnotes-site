## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202) — V15

**Desktop virtualization overview**

Desktop virtualization separates the desktop environment from the physical device. Instead of installing the operating system and applications on one local computer, you access a desktop that runs on centralized infrastructure over a network.

This model supports remote work and flexible device access. Users can connect from a laptop, tablet, smartphone, or other endpoint while the desktop itself remains hosted in a data center or cloud environment.

- **Centralized desktop** hosting keeps the operating system and apps on server infrastructure.
- **Network delivery** lets users reach the desktop from a remote physical machine.
- **Device flexibility** allows access from almost any endpoint with network connectivity.
- **Security benefits** come from keeping data in a central location rather than on many local devices.

**Virtual Desktop Infrastructure (VDI)**

<abbr title="Virtual Desktop Infrastructure">VDI</abbr> is a desktop virtualization model that centrally creates and manages virtual desktops. A virtual desktop is a virtualized instance of a desktop computer delivered over a network to an end user.

VDI is useful when you need to quickly provision many desktops for internal business applications and services. It also helps organizations scale desktop infrastructure more cost-effectively than expanding many individual physical PCs.

- **VDI** stands for Virtual Desktop Infrastructure.
- **Virtual desktops** run on central servers instead of local hardware.
- **Scalability** improves when you need to add desktops quickly.
- **Management** is simpler because IT handles desktops from one place.

**How VDI works**

A VDI environment uses server-side resources to host desktop operating systems and deliver them to users. The desktop environment is separated from the physical hardware, so the screen you use is effectively a remote session to a server-based desktop.

The architecture typically includes host servers and software that divides server resources into smaller desktop instances. A control layer directs traffic so users can connect to the correct virtual desktop.

- **Host servers** provide the compute resources for desktops.
- **Virtualization software** splits server capacity into multiple desktop instances.
- **Traffic control** routes users to the correct desktop session.
- **Separation from hardware** means the desktop is not tied to one physical machine.

**Benefits of VDI**

VDI supports secure remote access to internal business applications and services. It also simplifies IT operations because updates, security settings, and desktop configuration can be managed centrally.

Because the desktops are hosted centrally, organizations can scale up without major hardware investments. This makes VDI practical for changing workforce needs and flexible work models.

- **Secure access** supports remote users connecting to internal resources.
- **Central management** reduces the effort of maintaining many endpoints.
- **Lower hardware pressure** helps avoid large local PC expansion.
- **Flexible work** is easier when users can work from virtually anywhere.

**VDI versus traditional desktop deployment**

Traditional desktop deployment installs the operating system and applications directly on the hard drive of a specific computer. In VDI, the desktop environment lives on centralized infrastructure and is delivered over the network.

| Aspect | Traditional desktop | VDI |
|---|---|---|
| OS location | Local hard drive | Central server or cloud |
| Hardware dependency | Tied to one device | Separated from endpoint |
| Access model | Local use | Network-delivered remote access |
| Management | Per device | Centralized |

**Practical exam takeaways**

Desktop virtualization is about moving the desktop experience away from the local machine and into centrally managed infrastructure. VDI is the most important term to know in this topic because it describes the common model for delivering virtual desktops to users.

> Keep the distinction clear: VDI is not the same as a traditional local desktop, because the desktop runs on shared infrastructure and is accessed over a network.

**Quick review**

- Desktop virtualization delivers a desktop environment from centralized infrastructure.
- VDI creates and manages virtual desktops on central servers.
- Users can access virtual desktops from many device types.
- Centralized hosting improves management, security, and scalability.
