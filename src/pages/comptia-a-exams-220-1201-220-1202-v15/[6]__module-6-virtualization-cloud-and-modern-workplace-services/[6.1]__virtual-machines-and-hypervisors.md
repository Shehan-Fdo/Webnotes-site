## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202) — V15

**Virtualization overview**

Virtualization is the abstraction of physical hardware into multiple simulated environments. A single physical system can host multiple **virtual machines** (VMs), each with its own operating system and applications. This improves resource use because one machine can support several isolated workloads instead of leaving capacity idle.

A **hypervisor** is the software layer that manages this process. It sits between the hardware and the VMs, allocates resources such as CPU, memory, and networking, and keeps the guest systems separated.

- **Host hardware** is the physical computer or server.
- **Guest VM** is the software-based computer running inside the virtualized environment.
- **Isolation** helps one VM run without directly affecting another VM.

**Hypervisor types**

Hypervisors are commonly grouped into two types. The main difference is where they run and what they depend on.

| Type | Description | Typical host support |
|---|---|---|
| **Type 1** | Runs directly on the hardware | Common in server and infrastructure environments |
| **Type 2** | Runs on top of a host operating system | Common on desktop systems |

Type 1 hypervisors are often described as bare-metal because they interact directly with the hardware. Type 2 hypervisors depend on an existing operating system, which makes them easier to use in many desktop and lab scenarios.

> A hypervisor can run on many systems, but modern CPUs often include virtualization support that improves how efficiently VMs use hardware resources.

**How a hypervisor manages VMs**

The hypervisor controls the interaction between the physical system and each guest VM. It assigns resources and keeps workloads separated so that each VM behaves like an independent computer.

- It allocates **CPU cycles** to each VM.
- It assigns **memory** based on VM needs.
- It manages **networking** so VMs can communicate as configured.
- It maintains **separation** between guest systems.

This design lets you run different operating systems on the same physical machine while keeping them logically isolated.

**Common use cases**

Virtualization is a core technology in modern IT and cloud environments. It supports efficient resource use, flexible testing, and scalable infrastructure.

- Run multiple operating systems on one computer.
- Test software in isolated environments.
- Consolidate workloads onto fewer physical systems.
- Support cloud-style infrastructure where many virtual servers share hardware.

**Choosing between hypervisors**

Different hypervisors can meet different needs. For example, Hyper-V and VirtualBox are both used to run and manage VMs, but they differ in type and host support. Hyper-V is a type 1 hypervisor, while VirtualBox is a type 2 hypervisor. VirtualBox supports Windows, Linux, and macOS hosts, while Hyper-V is associated with Windows hosts.

When you choose a hypervisor, focus on the host operating system, the environment you need to support, and whether you want a desktop-friendly lab tool or a more infrastructure-oriented platform.

**Exam focus**

For A+, make sure you can explain the following:

- What virtualization does.
- What a hypervisor is and where it sits.
- The difference between **type 1** and **type 2** hypervisors.
- How VMs share CPU, memory, and networking resources.
- Why virtualization is useful for isolation and efficient hardware use.

**Quick comparison**

| Concept | Key idea |
|---|---|
| **Virtual machine** | A software-based computer with its own OS and apps |
| **Hypervisor** | Software that manages VMs and hardware access |
| **Type 1** | Runs directly on hardware |
| **Type 2** | Runs on top of a host OS |
