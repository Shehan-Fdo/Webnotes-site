## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202) — V15

**Motherboard firmware basics**

The motherboard firmware starts before the operating system loads. The **Basic Input/Output System** (BIOS) is the early startup software that initializes hardware and prepares the system to continue booting. You may also see it described as firmware, System BIOS, or ROM BIOS, with storage in ROM or flash memory.

BIOS checks that core components are present and functioning so the system can proceed. That includes verifying the CPU and memory during startup. In practical terms, firmware is the first control layer that makes the rest of the boot process possible.

- **BIOS** starts the system before Windows, Linux, or macOS loads.
- It initializes hardware and checks for required components.
- It can exist in ROM or flash memory.
- It is sometimes referred to as firmware or System BIOS.

**BIOS and UEFI**

**Unified Extensible Firmware Interface** (UEFI) is the newer firmware environment commonly discussed alongside BIOS. In exam scenarios, treat BIOS and UEFI as the firmware layer you use to configure startup behavior and hardware-related settings.

Firmware settings are useful for optimization, hardware management, and troubleshooting. Common options include boot order, system time and date, integrated peripheral configuration, and power management. You can also enable or disable specific hardware components and adjust CPU and memory-related settings.

> Be careful when changing firmware settings. Incorrect values can cause instability, reduced performance, or hardware problems.

**Accessing firmware setup**

You usually enter BIOS or UEFI setup during startup by pressing a special key before the operating system begins loading. Common keys include `Delete`, `F1`, and `F2`, and some systems use combinations such as `Ctrl` + `S` or `Ctrl` + `Alt` + `S`.

Virtual machines can also expose firmware setup. Some desktop hypervisors let you interrupt the boot process and open the VM’s BIOS settings.

1. Restart the computer or virtual machine.
2. Watch for the startup prompt that indicates the setup key.
3. Press the required key before the operating system begins loading.
4. Make the needed change in the firmware menu.
5. Save the configuration before exiting.

**Common firmware settings**

Firmware menus often control startup and device behavior. The most common settings are the ones that affect how the system boots and which hardware features are available.

- **Boot order** determines which device the system tries first.
- **System time and date** are stored in firmware settings.
- **Integrated peripherals** can be enabled or disabled.
- **Power management** settings affect how the system handles energy use.
- **CPU and memory** options may be available for performance tuning.

**TPM and Secure Boot**

**Trusted Platform Module** (TPM) is a security-related hardware feature used by modern operating systems. TPM 2.0 is required for Windows 11 and supports features such as Windows Hello and BitLocker. Some systems ship with TPM-capable hardware, but the feature may need to be enabled in firmware.

**Secure Boot** is another firmware-related security setting commonly associated with modern boot protection. When troubleshooting Windows 11 readiness or security-related startup issues, check whether TPM and Secure Boot are enabled in firmware.

- **TPM 2.0** supports security features such as identity protection and data protection.
- It may be present but disabled by default.
- **Secure Boot** is configured in firmware settings.
- Manufacturer instructions may vary by motherboard model.

**Motherboards and chipsets**

The motherboard is the main board that connects system components and provides the platform for firmware configuration. The chipset helps coordinate how hardware components interact with the motherboard and system resources. In practice, motherboard and chipset design influence which firmware options are available and how the system supports installed hardware.

When troubleshooting, use firmware settings to confirm whether a device is detected and whether the relevant controller or peripheral is enabled. If a drive is not detected, the first practical step is often to check the physical connections before assuming a firmware or operating system problem.

**Troubleshooting mindset**

Firmware changes should be deliberate and reversible. If a setting causes a problem, return to the firmware menu and restore the previous value or load the default configuration if appropriate. Keep changes minimal and document what you adjust so you can isolate the cause of a boot or hardware issue.

- Verify the setting you changed.
- Confirm the hardware is enabled and detected.
- Check boot order when a system starts from the wrong device.
- Review TPM and Secure Boot when security requirements are involved.
