## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202) — V15

**Wireless and network printer basics**

Wireless and network printers connect to a wired or wireless network so multiple devices can print without a direct USB connection. In a home or small office environment, this reduces cable clutter and lets you place the printer where it is convenient, as long as it can reach the network.

A multifunction device (MFD) may also include scanning, copying, faxing, and mobile printing features. Because these devices depend on both the printer hardware and the network path, setup issues often come from addressing, wireless association, or driver and sharing configuration rather than from the printer engine itself.

- **Wireless printer**: connects through Wi-Fi and appears on the network like other devices.
- **Network printer**: is reachable over the local network and can be shared by multiple systems.
- **MFD**: a multifunction device that combines printing with other functions.

**Network addressing and DHCP**

<abbr title="Dynamic Host Configuration Protocol">DHCP</abbr> automatically assigns IP addresses on many home and office networks. A printer can show that it is connected to the correct SSID, yet still have an IP address that does not match the expected router range if it received an address from another network or a different DHCP source.

When a printer appears offline but its wireless light is solid and its internal network test passes, check the IP address first. If the address is outside the local subnet, the workstation may not be able to reach the printer even though the printer believes it is connected.

> If the printer has a valid wireless connection but an unexpected IP address, treat it as a network addressing problem before assuming a hardware failure.

- Verify the printer’s current IP address from the printer menu or status page.
- Compare that address to the router’s DHCP range and subnet.
- Confirm the printer is connected to the intended SSID.
- Reconnect the printer to the correct wireless network if the address is inconsistent.

**Setup and sharing considerations**

In a shared environment, a printer attached to one computer can stop responding if the host system’s address changes and other devices still point to the old location. This is a common issue in DHCP-based networks when a shared printer depends on a computer that does not keep a stable address.

For reliable access, keep the printer’s network path consistent. If the printer is network-capable, connect it directly to the network instead of relying on a shared USB printer on a changing host system. If the printer is shared from a workstation, make sure the host remains reachable and that the sharing configuration stays intact.

- Use a stable network path for the printer whenever possible.
- Avoid depending on a changing host IP address for printer access.
- Reinstall or re-add the printer on client systems if the connection target changes.
- Confirm the printer share is still available if the printer is attached to another PC.

**Troubleshooting workflow**

1. Check the printer’s wireless status and confirm the Wi-Fi indicator shows an active connection.
2. Print or view the printer’s network test results to confirm the device can see the network.
3. Compare the printer’s IP address with the router’s subnet and DHCP range.
4. Verify the printer is on the correct SSID and not associated with another network.
5. Re-add the printer on the client if the address or share path has changed.
6. Use Windows printer troubleshooting if the printer still shows offline or does not print.

Windows includes built-in printer troubleshooting for connection and printing problems. Use it when the printer is visible but jobs do not complete, the device shows offline, or the system cannot maintain a stable connection.

**Practical setup checks**

A successful setup depends on matching the printer’s network identity with the client’s expectations. The printer must be on the correct wireless network, have a reachable address, and be configured in a way that the operating system can consistently find.

- Confirm the printer is on the intended network before adding it in Windows.
- Use the printer’s network menu to verify connection details.
- Keep the printer and client on the same local network segment when possible.
- Recheck the printer after router changes, since DHCP assignments can shift.

**Common symptoms and likely causes**

| Symptom | Likely cause | Practical check |
|---|---|---|
| Printer shows offline | Wrong IP address or unreachable network path | Compare the printer IP to the router subnet |
| Wi-Fi light is on but printing fails | Connected to a different network or stale printer entry | Verify SSID and re-add the printer |
| Shared printer stops responding | Host computer address changed | Confirm the host is reachable and sharing is active |
| Printer test passes but Windows cannot print | Client-side connection issue | Use Windows printer troubleshooting |

**Key takeaways**

Wireless and network printer setup is mostly about connectivity, addressing, and consistency. If the printer is online but unreachable, start with the IP address and network association. If the printer is shared from another computer, make sure the host path stays stable so client systems can keep printing without reconfiguration.
