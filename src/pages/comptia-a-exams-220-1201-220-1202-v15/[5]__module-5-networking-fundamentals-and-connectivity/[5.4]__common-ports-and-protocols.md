## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202) — V15

**Port numbers and why they matter**

Applications use **well-known port numbers** so devices know which service should receive traffic. A port number helps differentiate applications as data moves across the network, and it also gives firewalls a simple way to allow or block traffic.

These ports are important in troubleshooting because a service can be reachable only if the client, server, and any intervening firewall agree on the same port. In practice, you use port knowledge to identify what service is supposed to be running and where communication may be failing.

- **TCP** and **UDP** are the transport protocols that commonly carry application traffic.
- **Well-known ports** are the standard ports associated with common services.
- **Firewalls** often filter traffic based on port number.

**Common port and protocol pairs**

The following ports are the ones most often associated with A+ networking topics. Memorize the service name, the port number, and whether the protocol is typically TCP or UDP when that is part of the service’s normal use.

| Port | Protocol | Common use |
|---|---|---|
| `20` | TCP | FTP data transfer in active mode |
| `21` | TCP | FTP control connection |
| `22` | TCP | SSH remote access |
| `23` | TCP | Telnet remote access |
| `25` | TCP | SMTP email sending |
| `53` | TCP/UDP | DNS name resolution |
| `67` | UDP | DHCP server |
| `68` | UDP | DHCP client |
| `80` | TCP | HTTP web traffic |
| `110` | TCP | POP3 email retrieval |
| `123` | UDP | NTP time synchronization |
| `143` | TCP | IMAP email retrieval |
| `161` | UDP | SNMP management queries |
| `162` | UDP | SNMP traps |
| `389` | TCP/UDP | LDAP directory services |
| `443` | TCP | HTTPS secure web traffic |
| `445` | TCP | SMB file and printer sharing |
| `3389` | TCP | RDP remote desktop |

**Service behavior to remember**

Some services are easy to confuse because they serve related purposes. Focus on what each one does, not just the number.

- **FTP** uses separate ports for control and data.
  - `21` handles control.
  - `20` handles data in active mode.
- **HTTP** uses `80`, while **HTTPS** uses `443`.
  - HTTPS adds encrypted communication for web traffic.
- **DNS** commonly uses `53` for both TCP and UDP.
  - UDP is common for standard queries.
  - TCP is also used when needed for larger transfers or specific operations.
- **DHCP** uses `67` and `68`.
  - The server listens on `67`.
  - The client uses `68`.
- **SMB** uses `445` for file and printer sharing on modern networks.
- **RDP** uses `3389` for remote desktop access.

**How to study these ports**

You do not need to treat port numbers as isolated facts. Group them by function so they are easier to recall during troubleshooting.

- **Web services**
  - `80` for HTTP
  - `443` for HTTPS
- **Remote access**
  - `22` for SSH
  - `23` for Telnet
  - `3389` for RDP
- **Email**
  - `25` for SMTP
  - `110` for POP3
  - `143` for IMAP
- **Infrastructure**
  - `53` for DNS
  - `67` and `68` for DHCP
  - `123` for NTP
- **Management and sharing**
  - `161` and `162` for SNMP
  - `389` for LDAP
  - `445` for SMB

**Troubleshooting perspective**

When a service fails, ask whether the issue is with the application, the port, or the firewall. A service can be installed correctly and still fail if the expected port is blocked or if the client is trying the wrong protocol.

> If a connection fails, verify the service name, the expected port, and whether the traffic uses TCP or UDP before changing other settings.

**Quick recall cues**

- **FTP**: `20` data, `21` control
- **SSH**: `22`
- **Telnet**: `23`
- **SMTP**: `25`
- **DNS**: `53`
- **DHCP**: `67`/`68`
- **HTTP**: `80`
- **HTTPS**: `443`
- **SMB**: `445`
- **RDP**: `3389`
