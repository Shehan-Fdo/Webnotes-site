## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202) — V15

**IP addressing fundamentals**

An <abbr title="Internet Protocol">IP</abbr> address identifies a network-equipped device on a <abbr title="Transmission Control Protocol / Internet Protocol">TCP/IP</abbr>-based network. Each device needs a unique address so it can communicate correctly with other systems.

- **IPv4** uses four numbers separated by dots, such as `192.168.1.165`.
- **IPv6** is the successor to IPv4 and expands addressing capacity.
- **Unique addressing** matters because every device on the network must have its own IP address.
- **Static addressing** is manually configured, while **dynamic addressing** is assigned automatically by network services such as DHCP.

**Subnet basics**

A subnet mask works with the IP address to show the local network portion of the address. The device uses this information to decide whether traffic stays on the local network or must go to another network.

- **Subnet mask** is required with an IP address for proper communication.
- A common example is `255.255.255.0`.
- The mask is used by the local device and is not transmitted across the network like the IP address itself.
- **Classless subnetting** is part of basic TCP/IP addressing knowledge and helps define network boundaries more precisely.

> If the device does not have a valid subnet mask, it cannot reliably determine its network scope.

**Default gateway**

The default gateway is the router or next-hop address a device uses when it needs to reach destinations outside its local network. It is commonly included in IP configuration settings along with the IP address and subnet mask.

- **Default gateway** provides the path off the local subnet.
- You typically configure it together with the IP address and subnet mask.
- If the gateway is wrong or missing, local communication may still work, but external connectivity fails.

**DHCP**

<abbr title="Dynamic Host Configuration Protocol">DHCP</abbr> automatically provides IP configuration to devices when they join a network. Instead of typing in addressing details manually, you connect the device and the network supplies the needed values.

- **DHCP server** assigns network settings to clients.
- DHCP commonly provides the IP address, subnet mask, and default gateway.
- **Reservations** let a device receive the same address consistently.
- DHCP reduces manual configuration and helps avoid typing errors.

**DNS**

<abbr title="Domain Name System">DNS</abbr> translates human-friendly names into IP addresses so users can reach services by name instead of by number. This makes browsing and application access easier to use and manage.

- **Name resolution** is the core DNS function.
- DNS improves usability by letting you use names instead of memorizing IP addresses.
- DNS is often part of the basic services provided by a home gateway or network infrastructure.
- When DNS fails, devices may still have network connectivity but cannot resolve names correctly.

**NAT**

<abbr title="Network Address Translation">NAT</abbr> maps internal private addresses to a public address for communication with external networks. It is commonly used on home gateways and routers to let multiple devices share one public IPv4 address.

- **Private IP addresses** stay inside the local network.
- **Public IPv4 address** is the address seen on the Internet side.
- NAT helps preserve IPv4 addressing by reducing the need for one public address per device.
- Home gateways often combine routing, DHCP, DNS, and NAT in one device.

**How the services work together**

These technologies usually appear together in a small office or home network. DHCP gives the device its configuration, DNS resolves names, the subnet mask defines the local network, the default gateway provides the route out, and NAT translates internal traffic for Internet access.

- **DHCP** supplies the settings.
- **Subnet mask** defines local reach.
- **Default gateway** handles off-subnet traffic.
- **DNS** resolves names.
- **NAT** translates private addresses for external communication.

**Practical configuration flow**

1. Connect the device to the network.
2. Obtain IP settings from DHCP or enter them manually if static addressing is required.
3. Verify the subnet mask and default gateway.
4. Test name resolution with DNS.
5. Confirm that NAT is allowing access to external resources through the gateway.

**Key troubleshooting checks**

When connectivity fails, start with the addressing basics before moving to deeper network issues. A valid IP address alone is not enough if the subnet mask, gateway, or DNS settings are wrong.

- **IP address** must be valid and unique.
- **Subnet mask** must match the network design.
- **Default gateway** must point to the correct router.
- **DNS** must resolve names successfully.
- **NAT** must be functioning on the gateway for Internet access.
