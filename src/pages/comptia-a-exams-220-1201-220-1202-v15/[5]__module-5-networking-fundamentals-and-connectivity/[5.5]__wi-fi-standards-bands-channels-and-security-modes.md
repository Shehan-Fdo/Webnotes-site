## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202) — V15

**Wi-Fi standards overview**

Wi-Fi is based on the IEEE 802.11 family of wireless LAN (local area network) standards. These standards define how devices communicate over radio waves and evolve through amendments that add speed, efficiency, and compatibility improvements.

For A+ purposes, focus on the relationship between the standard, the band, and the device capability:

- **802.11** is the umbrella standard for Wi-Fi networking.
- Newer amendments such as `802.11n`, `802.11ac`, `802.11ax`, and `802.11be` represent newer generations with improved performance.
- Older and newer devices can coexist, but the network often runs at the pace of the slowest supported mode on that segment.
- Mixed environments are common in homes and small offices, so compatibility matters as much as raw speed.

**Common Wi-Fi generations**

The exam typically expects you to recognize the major generations and their general progression. Newer standards usually improve throughput, efficiency, and support for more devices, but the exact performance depends on the client, the access point, and the band in use.

| Standard | Common name | General note |
|---|---|---|
| `802.11b` | Legacy Wi-Fi | Older 2.4 GHz support with low data rates |
| `802.11g` | Legacy Wi-Fi | 2.4 GHz standard with better speed than `802.11b` |
| `802.11n` | Wi-Fi 4 | Works on `2.4 GHz` and `5 GHz` |
| `802.11ac` | Wi-Fi 5 | Primarily `5 GHz` |
| `802.11ax` | Wi-Fi 6 / 6E | Improves efficiency; `6E` extends into `6 GHz` |
| `802.11be` | Wi-Fi 7 | Newer generation with further performance improvements |

> Use conservative exam language: newer Wi-Fi generations generally improve speed and efficiency, but real-world results depend on interference, client support, and channel conditions.

**Bands and coverage**

Wi-Fi bands are the frequency ranges used for wireless communication. The most common bands in A+ material are `2.4 GHz`, `5 GHz`, and `6 GHz`.

- `2.4 GHz` usually offers better range and wall penetration.
- `5 GHz` usually offers higher performance and more channels, but shorter range than `2.4 GHz`.
- `6 GHz` is associated with newer Wi-Fi deployments and can reduce congestion by providing additional spectrum.
- Band choice affects both coverage and which wireless modes are available on the access point.

A practical way to think about the bands is that lower frequencies travel farther, while higher frequencies often provide more capacity at shorter distances. In dense environments, the cleaner band is often more useful than the theoretically fastest one.

**Channels and channel width**

A channel is a smaller slice of a Wi-Fi band. Access points and clients must use compatible channels to communicate effectively.

- Channel selection helps reduce interference from nearby networks.
- Channel width affects throughput and congestion.
- Common widths include `20 MHz`, `40 MHz`, `80 MHz`, and wider options on newer standards.
- Wider channels can increase speed, but they also consume more spectrum and may be more sensitive to interference.

In practice, channel planning is about balance. A wide channel can help performance in a clean environment, but a narrower channel may be more stable in crowded areas.

**Wireless modes and compatibility**

Wireless modes describe which 802.11 standards an access point supports on a given band. Some devices support only one standard, while others support mixed modes for compatibility.

- `2.4 GHz` commonly supports `802.11b`, `802.11g`, and `802.11n`.
- `5 GHz` commonly supports `802.11a`, `802.11n`, and `802.11ac`.
- Mixed protocol modes help older devices connect.
- Supporting legacy devices can reduce overall performance on the network.

When you configure a wireless network, you often choose between maximum compatibility and maximum efficiency. If you must support older clients, mixed mode is practical. If all clients are modern, a more restrictive mode can improve performance and reduce overhead.

**Security modes**

Wi-Fi security protects the wireless network from unauthorized access and helps keep traffic private. The evidence specifically highlights encryption methods as an important comparison point across newer Wi-Fi generations, so security mode selection matters alongside speed and band choice.

- Use the strongest security mode your devices support.
- Avoid weak or outdated security when newer options are available.
- Security compatibility can affect whether older devices can join the network.
- Encryption choice should match both the access point capability and the client capability.

A secure network is not just about encryption strength. It also depends on using current firmware, strong credentials, and a configuration that does not force unnecessary legacy support.

**Practical exam takeaways**

For troubleshooting and configuration questions, connect the standard, band, and security mode to the symptom.

1. Identify the client’s supported Wi-Fi standard.
2. Check which band the client is using and whether the band matches the expected range or speed.
3. Review channel width and nearby interference if performance is poor.
4. Confirm the security mode is compatible with the client and is not forcing a weaker legacy setting.
5. If older devices must connect, use mixed mode carefully and expect some performance tradeoffs.

**What to remember**

- `802.11` is the Wi-Fi standard family.
- Newer standards generally improve speed and efficiency.
- `2.4 GHz` favors range; `5 GHz` favors performance; `6 GHz` adds newer spectrum.
- Channels and channel width affect congestion and throughput.
- Mixed modes improve compatibility but can reduce performance.
- Security mode must balance protection with device support.
