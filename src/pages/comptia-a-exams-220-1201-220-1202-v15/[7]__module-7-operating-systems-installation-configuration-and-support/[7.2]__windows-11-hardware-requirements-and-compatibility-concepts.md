## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202) — V15

**Windows 11 minimum requirements**

Windows 11 uses minimum system requirements to support a reliable, secure, and compatible experience across devices. Microsoft states that these requirements are intended to deliver a good computing experience now and in the future, not just to make installation possible.

The most commonly emphasized compatibility checks are the processor, memory, storage, TPM 2.0, and Secure Boot capability. If a device misses one of these requirements, it may not be considered compatible for a standard Windows 11 installation.

- **Processor** compatibility is a primary gate for installation.
- **Memory** and storage must meet the minimum thresholds.
- **TPM 2.0** and Secure Boot are key security-related checks.
- Feature-specific requirements can exceed the base system requirements.

**TPM 2.0 and security**

<abbr title="Trusted Platform Module">TPM</abbr> 2.0 is a hardware-based security component that can be built into a dedicated chip or firmware. Microsoft identifies it as a required building block for Windows 11 security features.

TPM 2.0 helps protect sensitive data by securely storing encryption keys, certificates, passwords, and other secrets. It also supports cryptographic operations such as generating random numbers, encrypting and decrypting data, and verifying digital signatures.

- **Windows Hello** uses TPM 2.0 for identity protection.
- **BitLocker** uses TPM 2.0 for data protection.
- Many PCs shipped in the last 5 years can support TPM 2.0, but it may need to be enabled in firmware settings.
- If TPM 2.0 is supported but not enabled, you may need manufacturer-specific guidance to turn it on.

> TPM 2.0 is not just a compatibility checkbox; it is part of Windows 11’s security design.

**Compatibility concepts to check before upgrading**

Windows 11 compatibility is not only about raw performance. Microsoft frames the minimum requirements around three principles: reliability, security, and compatibility. That means a device can be fast enough but still fail the upgrade if it lacks a required security feature or supported platform component.

When evaluating a PC, focus on the full hardware picture rather than a single specification.

- A supported **64-bit processor** is required.
- The device must have at least **4 GB RAM**.
- The system needs enough **storage** for installation and normal use.
- **Secure Boot** capability matters for supported installation paths.
- Some apps, peripherals, or features may require more than the minimum system baseline.

**How to approach a Windows 11 readiness check**

1. Verify the processor is on a supported platform.
2. Confirm the system has at least 4 GB of memory.
3. Check that storage meets the minimum requirement.
4. Confirm TPM 2.0 is present and enabled.
5. Verify Secure Boot capability in firmware settings.
6. Review any feature-specific requirements for the apps or hardware you plan to use.

**Practical upgrade guidance**

If a PC does not meet Windows 11 requirements, the issue is often tied to firmware settings or unsupported hardware rather than a software problem. TPM 2.0 may exist but be disabled, and some systems need setup changes before they can qualify.

For troubleshooting, start with the device manufacturer’s support information. That is the safest path when you need model-specific instructions for enabling TPM or checking Secure Boot support.

**Windows 10 context**

Microsoft notes that Windows 10 support ended on October 14, 2025. The practical result is that Windows 10 devices still function, but they no longer receive free updates, technical assistance, or security fixes from Windows Update.

That makes Windows 11 compatibility checks especially important when planning upgrades, device refreshes, or long-term support decisions.
