**Command-line shells in Windows**

Windows includes two main command-line shells: the **Command shell** and **PowerShell**. A shell is a software program that gives you direct communication with the operating system or an application, which makes it useful for automating IT operations and support tasks.

The Command shell is the older built-in option and is commonly used for routine automation such as user account management and backups. PowerShell is a more modern shell that supports command-line work and scripting, and it is often used alongside other tools when you need more flexible administration.

- **Command shell**: Traditional Windows command-line environment for built-in console commands.
- **PowerShell**: Scriptable shell used for administration and automation.
- **Automation**: Repeating tasks through scripts instead of manual clicks.

**Why command-line tools matter in support**

Command-line tools are useful when you need fast, lightweight diagnostics or when graphical tools are slow or unavailable. They can help you inspect the system, move through folders, manage files, and run repeatable tasks during troubleshooting.

In support work, the command line is especially valuable because it provides direct access to the operating system. That makes it practical for quick checks, scripted actions, and basic recovery steps.

- Use it for **diagnostics** when you need quick system feedback.
- Use it for **automation** when the same task repeats often.
- Use it for **support** when GUI tools are unavailable or inconvenient.

**Windows Terminal and console hosting**

In Windows 11 22H2, the default app used to host console windows is **Windows Terminal**. After the October 2022 update, Command Prompt, Windows PowerShell, and other console apps appear inside Windows Terminal by default.

This can create compatibility issues for some apps, especially those that mix graphical elements with text-mode elements. If needed, you can opt out and use the classic Windows Console Host instead.

1. Open **Start** > **Settings** > **System** > **For developers**.
2. Under **Terminal**, select **Windows Console Host**.
3. Or open **Windows Terminal**, go to the **Startup** tab, and set **Default terminal application** to **Windows Console Host**.
4. If a console app is already open, right-click the title bar, select **Properties**, and choose **Windows Console Host**.

> Some older or specialized apps may behave better outside Windows Terminal if they have compatibility issues.

**Basic navigation and file tasks**

PowerShell examples from introductory command-line use show common support actions such as listing directories, changing locations, creating folders, deleting items, copying files, moving files, and reading or writing file content. These tasks are foundational because they let you inspect and organize data without relying on the file explorer.

Common navigation and file-management actions include:

- `Get-ChildItem` to view directory contents.
- `Set-Location` to move between directories.
- `mkdir` to create a new directory.
- `Remove-Item` to delete files or folders.
- `Copy-Item` and `Move-Item` to duplicate or relocate files.
- `Get-Content` to read file contents.
- `Write-Output` and redirection to write data to files.

```powershell
Get-ChildItem
Set-Location C:\Users
mkdir SupportLogs
Copy-Item .\report.txt .\SupportLogs\
Get-Content .\report.txt
```

**Working efficiently from the command line**

You can move faster by using shortcuts and patterns that reduce repetitive typing. Wildcards help you target multiple files at once, and help commands let you learn more about available cmdlets and parameters.

Useful support habits include:

- Use `ls` or `gci` as shorter aliases when appropriate.
- Use `cd` or `Set-Location` to move quickly between folders.
- Use `*` as a wildcard when you need to match multiple files.
- Use `Get-Help` to learn syntax and parameters before running a command.
- Use `Ctrl` + `C` to stop a command that is running too long or looping.

**Practical support mindset**

Command-line work is most effective when you treat it as a precise tool rather than a replacement for every GUI task. It complements other administration tools by giving you a lightweight way to verify paths, inspect files, and perform repeatable actions.

For A+ support scenarios, focus on understanding what each command does, when to use it, and how to stop or adjust it if the result is not what you expected. That approach helps you troubleshoot faster and reduces mistakes during live support.
