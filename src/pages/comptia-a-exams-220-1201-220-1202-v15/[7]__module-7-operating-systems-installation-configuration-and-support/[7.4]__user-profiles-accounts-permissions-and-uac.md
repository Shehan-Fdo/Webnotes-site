## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202) — V15

**User profiles and account basics**

A **user account** identifies who signs in to Windows, while a **user profile** stores that user’s settings and data. In practice, the account controls access and the profile holds the personalized environment you see after sign-in.

Windows supports multiple accounts on the same device. You may see a local account, a Microsoft account, or a work or school account, and each can affect sign-in behavior and access to resources.

- A **local account** stays on the device and does not require a Microsoft sign-in.
- A **Microsoft account** links the Windows sign-in to Microsoft services.
- A **work or school account** connects the device to an organization-managed identity.

If a profile becomes corrupted, Windows recommends creating a new user account, configuring it as an administrator, and then copying files from the old profile folder.

**Permissions and access control**

Permissions determine what a user can open, change, or administer. When you switch account types or move files between profiles, access can change because ownership and permissions do not automatically follow every identity change.

A common symptom of permission problems is an `Access denied` message when opening files or folders. This usually means the current account does not have the required rights, even if the user previously had access under a different sign-in.

- **Ownership** affects who controls a file or folder.
- **Permissions** define allowed actions such as read, write, or modify.
- **Administrator access** can grant broader control, but it does not guarantee access to every file if permissions are restricted.

> If you change from a local account to a Microsoft account and lose access to documents, treat it as a permissions or ownership issue first, not automatically as a hardware failure.

**User Account Control (UAC)**

**User Account Control** is a Windows security feature that helps prevent unauthorized changes. When an action requires administrator-level permission, UAC prompts you to approve or deny the change.

UAC reduces the ability of malicious software to run with elevated privileges and helps you make informed decisions before system changes occur. Unless UAC is disabled, malicious software is prevented from disabling or interfering with UAC settings.

You can change UAC behavior from `Control Panel > System and Security > Change User Account Control settings`, then move the slider to the desired notification level and select `OK`.

**UAC notification levels**

| Setting | Behavior |
|---|---|
| Always notify | Prompts when programs try to install software or make changes, and when you make changes to Windows settings. Other tasks are frozen until you respond. |
| Default level | Notifies you only when apps try to make changes to your computer. |
| Lower notification levels | Reduce prompts, but also reduce protection. |

The most cautious setting is useful if you routinely install new software or visit unfamiliar websites. In general, more prompts mean more visibility into changes, but also more interruptions.

**Practical troubleshooting approach**

When you troubleshoot account or permission issues, start by identifying the sign-in type and the scope of the problem. Then check whether the issue affects one file, one folder, or the entire profile.

1. Confirm which account is signed in.
2. Check whether the account is local, Microsoft, or work or school.
3. Test access to a different file or folder.
4. If the profile appears damaged, create a new administrator account.
5. Copy needed files from the old profile folder after signing in with the new account.

This approach helps you separate profile corruption from permission problems and from UAC prompts that are working as designed.

**Key takeaways**

- **Profiles** store user-specific settings and data.
- **Accounts** control identity and sign-in.
- **Permissions** control access to files and folders.
- **UAC** protects Windows by requiring approval for elevated changes.
