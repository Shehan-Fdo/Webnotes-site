## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202) — V15

**Overview**

Windows includes built-in utilities that help you inspect a system, troubleshoot problems, and manage administrative settings. These tools are useful when you need a quick view of hardware, drivers, services, performance, or configuration on a machine you have not used before.

Administrative tools are available through Windows folders such as **Windows Tools** in Windows 11 and **Administrative Tools** in Windows 10. The exact tools shown can vary by Windows edition, but the purpose stays the same: give you access to system-management utilities in one place.

**Microsoft Management Console**

The **Microsoft Management Console** (MMC) is a blank framework that you customize by adding snap-ins. It lets you combine multiple tools into one console so you can work from a single view instead of opening each utility separately.

Common snap-ins mentioned in the evidence include:

- **Event Viewer** for viewing system events
- **Shared Folders** for checking shared resources
- **Local Users and Groups** for account management
- **Performance Monitor** for performance tracking
- Other administrative tools you choose to add

To build a custom console, you open MMC and use the File menu to add or remove snap-ins.

> The available snap-ins depend on the Windows edition and the tools installed on the system.

**System Information**

**System Information** gives you a fast overview of a computer’s configuration. It is useful when you need to understand the internals of a device quickly during troubleshooting.

You can launch it with `msinfo32.exe`.

The utility organizes information into categories such as:

- **Hardware resources**
  - Memory
  - DMA settings
  - Interrupts
  - Conflicts
- **Components**
  - Display settings
  - Network configuration
- **Software environment**
  - Installed drivers
  - Pending print jobs
  - Running tasks

**Resource Monitor**

**Resource Monitor** helps you inspect how system resources are being used. It is part of the set of Windows troubleshooting utilities and is useful when you need more detail about current resource activity.

Use it when you want to observe system behavior rather than just static configuration.

**System Configuration**

**System Configuration** is another Windows utility used for troubleshooting and startup-related management. It appears in the list of useful administrative tools for system support tasks.

**Disk Cleanup and Defragmentation**

Windows includes maintenance utilities such as **Disk Cleanup** and **defrag**. These tools support system upkeep by helping you manage disk-related maintenance tasks.

- **Disk Cleanup** helps remove unnecessary files.
- **Defragmentation** is used for disk organization on supported storage types.

**Registry Editor**

The **Registry Editor** (`regedit`) is a Windows tool used for registry management. It is part of the core set of utilities you may use when working with advanced configuration and troubleshooting tasks.

> Make registry changes carefully. Incorrect edits can affect system behavior.

**Run dialog shortcuts**

The **Run** dialog gives you quick access to tools and commands. You can open it with <kbd>Win</kbd> + <kbd>R</kbd> and then launch utilities directly by name.

Common examples include:

- `cmd` to open Command Prompt
- `powershell` to open Windows PowerShell
- `msinfo32` to open System Information
- `dxdiag` for DirectX diagnostics
- `ncpa.cpl` to manage network connections
- `appwiz.cpl` to open Programs and Features

**Practical use in troubleshooting**

These utilities are most useful when you need to identify what is installed, what is running, or what is misconfigured. A typical troubleshooting flow is to gather system details first, then move to the specific tool that matches the symptom.

1. Open **System Information** to identify the system’s hardware and software environment.
2. Use **Resource Monitor** or **Performance Monitor** to check current activity.
3. Review **Event Viewer** inside MMC for logged issues.
4. Use **System Configuration** or other tools to isolate startup or configuration problems.
5. Apply maintenance tools such as **Disk Cleanup** or registry changes only when the issue calls for them.

**Key takeaways**

- **MMC** is a customizable console for grouping administrative snap-ins.
- **System Information** quickly summarizes hardware and software details.
- **Run** provides fast access to utilities through commands like `msinfo32` and `regedit`.
- Windows administrative tools vary by edition, but the core troubleshooting purpose remains the same.
