## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202) — V15

**Installation media basics**

Installation media gives you a way to start a system when the installed operating system is missing, damaged, or not suitable for the task. In practice, you boot the computer into a bare-bones installer environment and use it to set up Windows.

A common approach is a bootable USB flash drive. The motherboard must support booting from USB, and the drive must contain the operating system installation files.

- **Bootable USB** media is a practical choice for local installs.
- The installer runs before the full operating system is present.
- You use it when you need to install or reinstall Windows on a system.

**Creating installation media**

Microsoft provides a way to create Windows installation media with its Media Creation Tool. The tool downloads the current installation files and prepares bootable media for setup.

> Use the correct Windows version and installation source for the target system. A mismatched installer can create avoidable setup problems.

Typical creation flow:

1. Go to the Windows download page.
2. Find the section for creating Windows installation media.
3. Download the tool and follow the prompts.
4. Write the installation files to a USB flash drive.

**Recovery media and recovery drives**

Recovery media helps you troubleshoot or repair a Windows system when normal startup fails. A Windows recovery drive can boot the machine into repair tools, and it may also include Windows installation files so you can reinstall Windows if needed.

Recovery media is useful when you want a portable repair option before a failure happens. It gives you a way to access basic troubleshooting tools on a Windows system without relying on the installed OS.

- **Recovery Drive** media supports basic troubleshooting repairs.
- Adding installation files can extend the drive’s usefulness for reinstalling Windows.
- You create it ahead of time so you are prepared for future failures.

**Deployment basics**

Deployment is the process of installing operating systems across one or many systems in a controlled way. In small environments, you may use removable media and visit each machine. In larger environments, you may use methods that reduce manual work.

Professor Messer’s course material highlights several deployment considerations:

- **Boot methods** determine how the system starts the installer.
- **Installation options** affect how Windows is placed on the device.
- **Partition types** and formatting choices matter during setup.
- **Zero-touch deployment** reduces hands-on installation work.

Deployment planning also depends on hardware and software compatibility. You need the right operating system version, enough memory, storage, and other resources before you install applications or complete setup.

**Practical comparison**

| Media type | Primary purpose | Typical use |
|---|---|---|
| Installation media | Install or reinstall Windows | Fresh setup or repair reinstall |
| Recovery media | Access repair tools | Troubleshooting startup or system issues |
| Deployment media/process | Roll out systems efficiently | Repeated installs across multiple devices |

**Key setup considerations**

When you prepare media or plan deployment, focus on the target device and the install path you need. A USB installer is convenient, but it is not the only option. In some environments, physically visiting every computer is inefficient, so deployment methods that scale better become more useful.

- Check that the device can boot from the chosen media.
- Match the installer to the intended Windows version.
- Confirm that the system has the required resources before installation.
- Use recovery media proactively, not only after a failure.

**Why this matters for A+**

These topics connect installation, recovery, and operational readiness. You should recognize when to use installation media for setup, when to use recovery media for repair, and when deployment methods help you install systems efficiently in real environments.
