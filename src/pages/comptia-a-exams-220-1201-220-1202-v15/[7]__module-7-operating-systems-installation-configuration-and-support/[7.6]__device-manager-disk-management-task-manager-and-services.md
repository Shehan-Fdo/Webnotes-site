## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202) — V15

**Windows administrative tools overview**

Windows includes built-in tools that help you troubleshoot hardware, storage, performance, and background processes. For A+ support work, these tools are useful because they let you inspect system state quickly and make targeted changes without installing extra software.

- **Device Manager** helps you view and manage hardware devices and drivers.
- **Disk Management** helps you manage disks and volumes.
- **Task Manager** helps you monitor running apps and system performance.
- **Services** helps you control background Windows services.

These tools are commonly used during setup, troubleshooting, and recovery tasks. If one tool does not open normally, that can itself point to a broader Windows problem.

**Device Manager**

Device Manager is the Windows tool you use to review installed hardware and driver status. Microsoft notes that you can open it by selecting Start and entering `device manager`, then choosing Device Manager from the search results.

Use Device Manager when you need to:

- Check whether hardware is detected correctly.
- Review device status and driver behavior.
- Update, disable, or uninstall a device driver when troubleshooting.

A practical A+ workflow is to open Device Manager first when a peripheral, adapter, or internal component is not working as expected. It gives you a fast view of whether Windows recognizes the device and whether a driver issue may be involved.

**Disk Management**

Disk Management is a built-in Windows tool for managing disks and volumes. Microsoft states that you can use it to initialize new drives, create and format volumes, change drive letters, and extend or shrink existing volumes.

Common uses include:

- Initializing a new drive before use.
- Creating partitions or volumes.
- Formatting a volume for use in Windows.
- Changing a drive letter.
- Extending or shrinking an existing volume.

This tool is especially useful after installing a new drive or when storage layout needs to change. If a disk is present but not usable, Disk Management is one of the first places to check.

> If Device Manager or Disk Management will not open, treat that as a troubleshooting clue rather than a separate inconvenience. It may indicate a deeper Windows issue affecting administrative tools.

**Task Manager**

Task Manager is the Windows utility for checking what is running and how the system is performing. In support scenarios, you use it to identify resource-heavy applications, confirm whether a program is responding, and observe overall system load.

Use Task Manager to:

- Review running applications and background activity.
- Check CPU, memory, disk, and network usage.
- End an unresponsive task when needed.
- Compare current performance against normal behavior.

Task Manager is especially helpful when a system feels slow, an app freezes, or you need a quick snapshot of resource usage during troubleshooting. It gives you immediate visibility into what may be consuming system resources.

**Services**

Services are background components that Windows and installed software use to perform ongoing tasks. The Services console lets you view service status and control whether a service starts or stops.

Use Services when you need to:

- Check whether a required service is running.
- Start or stop a service during troubleshooting.
- Review whether a service is configured to run automatically.

Services are important because many Windows features depend on them. If a feature is not working, the related service may be stopped, disabled, or failing to start.

**How these tools fit together**

These tools often work as a sequence during troubleshooting. You start by identifying the symptom, then choose the tool that matches the problem area.

- Hardware issue: check **Device Manager**.
- Storage or partition issue: check **Disk Management**.
- Performance or app freeze: check **Task Manager**.
- Background feature failure: check **Services**.

This approach keeps troubleshooting focused. Instead of guessing, you use the tool that matches the layer of the problem.

**Quick access and practical notes**

You can reach these tools through Windows search or administrative shortcuts, depending on the environment. For A+ purposes, the important point is not just where they are, but what each one tells you.

- Use **Device Manager** for hardware and driver visibility.
- Use **Disk Management** for disk and volume changes.
- Use **Task Manager** for live performance and process checks.
- Use **Services** for background service control.

When a tool fails to open, that may suggest a permissions issue, a system corruption issue, or a broader Windows management problem. In that case, document the symptom carefully and continue troubleshooting from a higher level.
