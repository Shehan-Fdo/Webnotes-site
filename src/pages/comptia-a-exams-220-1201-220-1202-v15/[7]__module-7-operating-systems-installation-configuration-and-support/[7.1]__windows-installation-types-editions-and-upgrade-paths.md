## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202) — V15

**Windows installation overview**

Installing an operating system (OS) starts by booting to installation media or another deployment method because the target computer may not yet have a usable OS. In practice, you choose the method based on the environment, the device state, and whether you need a fresh install or an upgrade.

- A **bootable USB drive** is a common installation method.
- Installation media contains the setup files needed to start the OS installation process.
- Zero-touch deployment is useful when you need to install many systems without visiting each one manually.
- The installation approach must match the hardware’s boot support and the deployment workflow.

**Installation types and deployment choices**

Windows installation options include different boot methods, partition choices, and formatting options. These decisions affect how the OS is placed on the device and how the storage is prepared for use.

- **Boot method** determines how you start the installer.
- **Partition type** affects how the disk is organized for installation.
- **Formatting** prepares the selected partition for the file system and OS files.
- A bare-bones installation environment is often used to launch the setup process before the full OS is present.

**Windows editions and edition changes**

Windows comes in multiple editions, and edition changes are supported when the upgrade path is valid. Microsoft notes that you can move from one edition to another on the same release, and that edition changes require entering a license or product key for the new edition.

- **Edition upgrade** means changing from one Windows edition to another.
- Supported edition changes apply to Windows 10 and Windows 11.
- You use a license or product key for the new edition before completing the change.
- Downgrading an edition is handled separately and is tied to license expiration guidance.

> If the edition change is not supported, do not assume the upgrade will complete successfully. Verify the path before you begin.

**Supported upgrade paths**

Windows upgrade paths cover moving from a previous supported version to a currently supported version, as well as upgrading from one release to a later release of the same version. Microsoft also supports migrating between editions of the same release when the path is supported.

- You can upgrade only from previous versions that are still supported.
- You can move from one Windows release to a later release of the same version.
- You can change editions within the same release when the path is supported.
- For edition-specific changes, use the edition upgrade guidance rather than the general upgrade path guidance.

**Planning a Windows 11 upgrade**

Before upgrading to Windows 11, confirm that the device meets the required system requirements and choose the installation method that fits the environment. Microsoft provides guidance for creating installation media, using a product key, and troubleshooting upgrade issues.

- Check device readiness before starting the upgrade.
- Create installation media when you need a clean or repeatable deployment method.
- Use a product key when the upgrade path requires edition activation or edition change.
- Review upgrade troubleshooting guidance if the process fails or stalls.

**Practical exam takeaways**

For A+ purposes, focus on the difference between installing Windows, upgrading Windows, and changing editions. These are related but not identical tasks, and each one has its own supported path and prerequisites.

- **Installation** places Windows on a device, often from bootable media.
- **Upgrade** moves to a later supported version or release.
- **Edition change** switches between Windows editions on the same release.
- Always verify support before you begin, especially when moving to Windows 11.
