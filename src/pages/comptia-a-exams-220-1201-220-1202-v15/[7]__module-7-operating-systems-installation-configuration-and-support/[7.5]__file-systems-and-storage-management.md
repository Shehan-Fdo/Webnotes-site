**File systems overview**

A file system is the structure an operating system uses to organize data on storage media. It creates the rules for how files are stored, named, tracked, and retrieved. When you format a partition, you choose the file system that will manage that space.

Different file systems support different features, so the choice affects compatibility, permissions, and advanced storage behavior. In Windows environments, NTFS is the common default, while FAT32 and exFAT are often used for removable media and cross-platform compatibility.

- **NTFS** is the Windows file system with the broadest feature support in the comparison data.
- **exFAT** is designed for simpler storage use and supports fewer advanced features than NTFS.
- **FAT32** remains widely recognized for compatibility, but it has limited feature support.
- **UDF** appears in the comparison as another Windows-supported file system with some features similar to NTFS.

**Common file system capabilities**

Windows file systems differ in the metadata and storage features they support. Some features are basic, such as timestamps and case preservation, while others are more advanced, such as hard links, sparse files, and named streams.

- **Creation time stamps** are supported by NTFS, exFAT, UDF, and FAT32.
- **Last access time stamps** are supported by exFAT, UDF, and FAT32, while NTFS does not support them in the comparison table.
- **Case-sensitive** behavior is optional in NTFS and supported in UDF, but not in exFAT or FAT32.
- **Case-preserving** behavior is supported by all four file systems.
- **Hard links** are supported by NTFS and UDF, but not by exFAT or FAT32.
- **Soft links** are supported by NTFS only in the comparison table.
- **Sparse files** and **named streams** are supported by NTFS only in the comparison table.
- **Extended attributes** are supported by NTFS and UDF, but not by exFAT or FAT32.

> Use the file system that matches the device’s purpose. A removable drive may prioritize compatibility, while a system volume may need NTFS features such as permissions and advanced file handling.

**Disk Management in Windows**

Disk Management is a built-in Windows utility for advanced storage operations. It shows each drive, its partitions, and details such as allocated and used space. You use it to manage storage without third-party software.

It is useful for initial disk setup, partition changes, and troubleshooting. The utility also helps you adjust drive letters and volume sizes when storage needs change.

- **Initialize** a new disk before you use it.
- **Extend** a basic volume into unclaimed space on the same drive.
- **Shrink** a basic volume or partition to make room for another volume.
- **Change** or assign a drive letter.
- **View** drive and partition details, including space usage.

**Typical Disk Management tasks**

1. Open Disk Management from Windows administrative tools or the management console environment.
2. Review the list of disks, partitions, and volume status.
3. Choose the task you need, such as initializing a disk, changing a drive letter, or resizing a volume.
4. Confirm the change and verify that the storage layout matches your plan.

**Storage administration considerations**

Disk Management supports routine administration and troubleshooting, but storage changes can affect data access. Partitioning, formatting, and resizing should be done carefully because they change how the operating system sees the disk.

When you work with removable drives or new disks, the file system choice matters as much as the partition layout. A compatible file system can improve portability, while a more capable file system can provide better support for Windows-specific features.

**Key takeaways**

- A file system defines how the operating system organizes and accesses data on a partition.
- NTFS provides the richest feature set in the Windows comparison data.
- exFAT and FAT32 favor compatibility over advanced features.
- Disk Management lets you initialize, partition, resize, and relabel storage devices in Windows.
- Choose the file system and volume layout based on the device’s role and the features you need.
