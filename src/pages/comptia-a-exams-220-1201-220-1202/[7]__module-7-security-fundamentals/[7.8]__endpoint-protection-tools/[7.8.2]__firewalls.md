## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Firewall fundamentals**

A firewall filters network traffic entering and leaving a device or network. In A+ terms, think of it as a control point that helps you allow trusted communication and block unwanted connections. Firewalls can use criteria such as IP address, protocol, and port number to make those decisions.

- **Traffic filtering** can be based on source and destination IP address.
- **Protocol filtering** can target specific IP protocols.
- **Port filtering** can allow or block traffic by source or destination port number.
- **Application control** can restrict traffic to approved services and apps.

A firewall is often described as a verb as much as a device: you firewall traffic by deciding what is permitted. In practice, that can happen at the edge of a network, such as in a SOHO (small office/home office) router, or on the endpoint itself.

**Host-based firewalls**

Windows Firewall is a built-in, host-based firewall. It protects the local device by filtering inbound and outbound traffic and is part of the Windows security stack. Microsoft’s Windows Security app exposes firewall and network protection settings, and the firewall can also be managed through Control Panel.

- **Host-based** means the firewall runs on the device it protects.
- Windows Firewall is designed to stay on by default.
- You need elevated permissions to turn it on or off.
- It helps restrict communication to only explicitly allowed apps and services.

Because it runs on the endpoint, Windows Firewall is useful for protecting laptops, desktops, tablets, and other Windows devices from unwanted inbound connections and from unauthorized outbound communication.

**Windows Firewall profiles**

Windows uses separate firewall profiles for different network types. This lets the system apply different rules depending on where the device is connected.

- **Domain** profile applies on a domain network.
- **Private** profile applies on trusted private networks.
- **Public** profile applies on less trusted public networks.

These profiles help you match the firewall’s behavior to the trust level of the network. Public networks generally require the most restrictive posture, while private and domain networks may allow more communication depending on policy.

**Common configuration ideas**

Windows Firewall settings let you control whether the firewall is enabled and how it handles traffic. The practical goal is to permit only the traffic you need while keeping the rest blocked.

- Enable or disable the firewall when troubleshooting.
- Review profile-specific settings for the current network type.
- Allow only required applications and services.
- Use the Windows Security app for quick access to firewall controls.

> If you disable the firewall for troubleshooting, do so only temporarily and restore protection as soon as possible.

**How to access firewall settings in Windows**

You can reach firewall controls from the Windows Security app. This is the most direct path for checking firewall status and adjusting network protection settings.

1. Open Windows search.
2. Type `security`.
3. Select the Windows Security app.
4. Open `Firewall & network protection`.
5. Review the active profile and its status.
6. Change settings only if you have administrator permissions.

**What to remember for the exam**

Firewall questions on A+ usually focus on purpose, placement, and basic management rather than deep enterprise policy design. Be ready to identify what a firewall blocks, where it can be placed, and how Windows Firewall behaves on different network types.

- Firewalls filter traffic based on rules.
- Windows Firewall is a built-in host-based firewall.
- Windows uses domain, private, and public profiles.
- Administrative rights are required for on/off changes.
- The Windows Security app is a common management path.

**Quick comparison**

| Concept | What it means |
|---|---|
| Network firewall | Protects traffic at the network edge |
| Host-based firewall | Protects a single device |
| Windows Firewall | Built-in firewall for Windows devices |
| Profile | Rule set tied to network trust level |

**Key takeaway**

Use firewalls to reduce exposure by allowing only necessary traffic. For A+ purposes, focus on the basic filtering model, the Windows Firewall interface, and the difference between domain, private, and public profiles.
