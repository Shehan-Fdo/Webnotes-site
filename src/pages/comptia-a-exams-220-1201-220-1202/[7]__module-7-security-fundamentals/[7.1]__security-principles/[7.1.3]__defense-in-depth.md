## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Defense in depth overview**

Defense in depth is a layered security strategy. You do not rely on a single control to protect an environment; instead, you combine multiple controls so one failure does not expose everything.

This approach matters because attackers can bypass one safeguard, but they still have to get through the next layer. In practice, defense in depth improves resilience by spreading protection across the organization.

- **Layered security** uses multiple controls together rather than one tool alone.
- The goal is to protect systems at different points of attack.
- If one layer fails, other layers still help prevent, detect, or limit damage.

**Common control layers**

A defense-in-depth plan usually includes physical, technical, and administrative controls. These layers work together and should be chosen based on the environment you are protecting.

Physical controls help stop unauthorized access to equipment and spaces.

- Door locks restrict entry to rooms and buildings.
- Fences create a boundary around a site or data center.
- Rack locks protect hardware inside cabinets.
- Cameras help monitor activity and deter tampering.

Technical controls protect systems and traffic through hardware and software.

- Firewalls filter traffic at the network edge or between internal segments.
- Authentication controls verify who can access resources.
- Disk encryption helps protect data on storage media.
- Network segmentation limits how far an attacker can move inside a network.

Administrative controls define how security is planned and enforced.

- Policies and procedures document how controls should be used.
- Separation of duties reduces the risk of one person having too much control.
- Security design should support the way the organization actually operates.

**Network-focused examples**

Defense in depth often shows up in network design. You can place controls at the perimeter and also inside the network to reduce trust in any single zone.

- A firewall at the edge helps control traffic entering and leaving the organization.
- Internal firewalls can separate floors, departments, or data center devices.
- Screened subnets add another barrier between public-facing and internal systems.
- Network access control can help limit which devices connect to the network.
- Honeypots can act as decoys to observe suspicious activity.

A layered design is especially useful because modern threats do not always arrive through one obvious path. Some attacks target users, some target exposed services, and some try to move laterally after initial access.

**How to think about implementation**

When you build defense in depth, start by identifying where an attacker could enter and where damage would spread. Then place controls so each layer supports the next one.

1. Protect the physical environment first with locks, fences, and monitoring.
2. Add technical controls such as firewalls, authentication, and encryption.
3. Use policies and procedures to define how the controls are managed.
4. Segment the network so one compromise does not expose every system.
5. Review the design regularly and adjust controls as the environment changes.

> A single control rarely stops every threat. The strength of defense in depth comes from overlapping layers that reduce risk even when one layer is bypassed.

**Exam focus**

For A+ purposes, remember the core idea: defense in depth means layered protection. If you see a question about improving security with multiple safeguards, think about combining physical, technical, and administrative controls instead of depending on one solution.

- Look for answers that mention multiple layers of protection.
- Associate firewalls, locks, cameras, encryption, and policies with this strategy.
- Remember that internal segmentation is part of the same layered mindset.
