## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Zero trust overview**

Zero trust is a security model that assumes no user, device, or process is trusted by default. Instead of relying on a strong internal perimeter, you verify access every time a resource is requested.

This approach matters because modern networks include many more connected devices, including <abbr title="Internet of Things">IoT</abbr> devices, which expand the attack surface. A perimeter-only mindset is no longer enough when risk exists on all sides of the firewall.

- **Default deny** means access is not granted until approval is verified.
- **Least privilege** means users get only the access they need for their work.
- **Continuous verification** means each request is checked, not just the first login.
- **Post-perimeter** thinking treats the internal network as untrusted too.

**How zero trust works**

In a zero trust model, you do not assume that being inside the network makes a user safe. You authenticate or prove identity each time access is requested, and you apply controls to users, devices, and processes.

This model often includes stronger sign-in checks and tighter access rules. For example, multi-factor authentication can help confirm identity, and encryption can protect data as it moves or is stored.

- Access is granted per resource, not broadly across the network.
- Authentication happens repeatedly as users move between resources.
- Security checks apply to every device and every process, not only to users.
- Entitlements define what a user can reach after approval.

**Why organizations adopt it**

Zero trust helps reduce the damage caused by compromised credentials, unauthorized movement, and unmanaged devices. It is especially useful when many endpoints and IoT devices connect to the same environment.

The goal is not to make access impossible. The goal is to make access deliberate, limited, and verified so that users can do their jobs without unnecessary exposure.

- More connected devices increase potential vulnerabilities.
- Internal access is not automatically safe.
- Limiting access reduces the blast radius of an incident.
- Verification supports safer access in distributed environments.

**Practical exam takeaways**

For exam purposes, connect zero trust with these ideas:

- Trust is never implicit.
- Access is checked at each request.
- Users receive only the permissions they need.
- The model shifts security away from a single protected edge.
- It applies to users, devices, and processes.

> Zero trust does not mean “no one can access anything.” It means access is approved, limited, and continuously validated.

**Quick comparison**

| Traditional perimeter model | Zero trust model |
|---|---|
| Trust often increases after login or after crossing the firewall | Trust is not assumed at any point |
| Internal network access can be broad | Access is restricted to needed resources |
| Security focuses on the edge | Security is applied throughout the environment |
| Movement inside the network may be easy | Each access request is checked |

**Remember**

If you see a question about a network where users must authenticate for each resource, or where access is limited to only what is needed, think zero trust. If the question mentions a move away from perimeter-only security, that is also a strong clue.
