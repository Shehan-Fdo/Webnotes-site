## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Principle overview**

The principle of **least privilege** means giving users or processes only the access they need to do their job. It is a core security idea in logical security and host security because it reduces unnecessary exposure to data and system functions.

When you apply least privilege, you limit what a person can see, change, or execute. That helps protect both the user account and the applications running under that account.

- Grant the **minimum access** needed for the task.
- Avoid broad permissions that exceed job requirements.
- Treat user rights as a security control, not just an administrative setting.

**Why it matters**

Least privilege helps reduce the impact of common security problems. If a user account is compromised, the attacker inherits only the permissions assigned to that account. If malware runs under a limited account, it is also constrained by those same permissions.

This approach also supports system stability by limiting access to critical resources and services. It lowers the chance of accidental data loss, unauthorized actions, and misuse of administrative capabilities.

- Reduces risk from **insider threats**.
- Limits damage from **malware**.
- Helps prevent accidental changes or deletions.
- Supports controlled access to sensitive resources.

**How it is applied**

You usually enforce least privilege through access control methods that match job roles and responsibilities. Role-based or group-based access helps you assign permissions consistently, while explicit permission assignment gives you tighter control over specific resources.

Auditing is also important. Reviewing user and process activity helps you confirm that access remains appropriate over time and that permissions do not drift beyond what is required.

1. Identify the task or job function.
2. Assign only the permissions needed to complete that task.
3. Use role-based or group-based access where practical.
4. Review access and audit usage regularly.

**Practical examples**

A standard user should not run with administrative rights unless there is a clear need. Administrative access gives users and applications more control than necessary, which increases the chance of unauthorized changes and expands the impact of malicious software.

Least privilege applies to both people and software. A user account should have access only to the data required for the job, and an application should not receive more rights than it needs to function.

- A help desk user may need access to support tools, but not full system administration.
- A business application may need read access to a database, but not write access.
- A malware infection under a limited account has less room to spread or modify the system.

**Exam focus**

For A+ questions, connect least privilege to the idea of minimizing access. If the question asks how to reduce risk, look for answers involving limited permissions, controlled access, or removing unnecessary administrative rights.

> If a choice gives a user more access than needed, it is usually the wrong answer for least privilege.
