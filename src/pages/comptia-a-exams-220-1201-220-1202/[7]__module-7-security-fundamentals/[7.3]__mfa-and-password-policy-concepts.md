## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Authentication and MFA basics**

Authentication is the process of identifying and verifying that a user or system has the right approvals to access resources. In practice, you prove identity with one or more factors. <abbr title="Multifactor authentication">MFA</abbr> adds another layer by requiring more than one type of evidence before access is granted.

The core idea is simple: a password alone is often not enough. MFA reduces the chance that a stolen password leads to unauthorized access, and it is commonly used to protect internet-enabled services and sensitive data.

- **Something you know** includes passwords, PINs, and security questions.
- **Something you have** includes an authentication app, one-time code token, certificate, smart card, email, SMS, or phone call verification.
- **Something you are** includes fingerprints, facial recognition, and iris scans.
- **Somewhere you are** can use source IP address, geolocation, or geofencing.
- **Something you do** can include behavioral profiling, keystroke dynamics, mouse dynamics, or gait analysis.

> Requiring multiple instances of the same factor, such as a password plus a PIN, does not provide true MFA. The factors should be independent.

**Why MFA matters**

MFA helps prevent unauthorized access even when one factor is compromised. It is a practical defense against common threats such as phishing, brute force attempts, malware infections, and credential theft.

A useful way to think about MFA is that it makes an attacker solve more than one problem at once. If a password is exposed, the second factor still blocks access unless the attacker can also satisfy that additional verification step.

- MFA adds an extra verification step before access is granted.
- It is especially valuable for accounts that protect sensitive data or administrative access.
- It works best when the factors are independent and not vulnerable to the same attack.

**Password policy concepts**

Password policy is the set of rules that governs how passwords are created, used, and protected. Strong password practices support account security, but they work best when paired with MFA and user awareness.

CompTIA emphasizes that passwords remain critical to IT security, even though they are often inconvenient. Training users to recognize unsafe behavior, including phishing attempts, helps reduce risk and supports better password hygiene.

- Use passwords as one part of a broader security strategy.
- Treat password reuse as a risk because one compromised password can affect multiple accounts.
- Support users with security awareness training so they can spot suspicious messages and report concerns.

**Practical exam takeaways**

For exam purposes, focus on the relationship between authentication factors and the reason MFA is stronger than a single password. Also remember that password policy is not just about complexity; it is about reducing exposure and improving user behavior.

- MFA means more than one type of evidence is required.
- Passwords are a form of “something you know.”
- A second factor can come from a device, app, or biometric trait.
- Security awareness helps users avoid phishing and other password-related attacks.
