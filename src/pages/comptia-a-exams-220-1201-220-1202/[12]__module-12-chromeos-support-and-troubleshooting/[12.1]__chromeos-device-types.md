## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**ChromeOS device family**

ChromeOS is the operating system used on Google’s ChromeOS devices. For A+ purposes, focus on the device types you are most likely to see in the field and on how the platform is maintained over time.

- **Chromebook**: laptop-style ChromeOS device.
- **Chromebox**: desktop-style ChromeOS device.
- **Chromebase**: all-in-one ChromeOS device.
- ChromeOS devices are preinstalled with the operating system and are designed to run applications within that platform.
- The operating system acts as the link between hardware components and the applications you use.

ChromeOS is part of the broader operating system landscape alongside Windows, macOS, Linux, Android, and iOS. The key exam idea is not just the name of the OS, but the device form factor and how the OS supports the hardware.

**Updates and support**

ChromeOS devices receive automatic updates that improve features, security, and stability. Google states that ChromeOS devices receive 10 years of updates from the platform release date.

- Updates apply across the operating system, browser, and hardware support layers.
- Automatic updates help keep the device secure and useful for longer.
- Some features and services may not be supported on older devices that receive extended updates.

> ChromeOS update behavior can vary by channel and device platform, so use the device’s support status when planning maintenance or troubleshooting.

**Release channels**

ChromeOS uses different release channels to balance stability and testing. The Stable channel is the normal production path, while other channels provide earlier access to changes.

- **Stable**: full OS updates arrive about every 4 weeks.
- **Minor updates**: security fixes and software updates happen every 2–3 weeks.
- **Beta**: gives a 4–6 week preview of features coming to Stable.
- **Dev**: updates once or twice weekly and shows what the Chrome team is working on.
- **LTS**: long-term support channel with a slower release cadence than Stable.

**Long-term support behavior**

The long-term support (LTS) channel is designed for organizations that need more predictable change control. It still receives frequent security fixes, but feature updates are cumulative and arrive every 6 months.

- Use LTS when you need less frequent feature change.
- Use Stable when you want the standard release cadence.
- LTS helps administrators and developers test and manage devices with more time between feature changes.

**What to remember for the exam**

ChromeOS device types are mainly about the hardware form factor and the operating system that ships with it.

1. Identify the device type: `Chromebook`, `Chromebox`, or `Chromebase`.
2. Remember that ChromeOS devices update automatically.
3. Know that ChromeOS has multiple release channels, including Stable, Beta, Dev, and LTS.
4. Associate LTS with slower feature updates and ongoing security fixes.
