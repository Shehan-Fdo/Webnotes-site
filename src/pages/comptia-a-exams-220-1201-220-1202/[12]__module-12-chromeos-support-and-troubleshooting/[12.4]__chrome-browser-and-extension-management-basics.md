## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Browser security basics**

A browser is a major application environment, so you treat it like any other software you install and maintain. The safest approach is to get the browser from a legitimate source, not from random download sites or links in email.

Browser security also includes checking the integrity of what you download. One practical control is **hash verification**, which helps confirm that the installer on your system matches the file published by the developer.

- Download the browser directly from the developer site.
- Avoid third-party download chains and email links.
- Verify the downloaded file when hash information is available.
- Keep the browser patched so you reduce exposure to known issues.

> Do not trust a browser installer just because it looks familiar. Attackers can distribute fake browser versions to gain access to your system.

**Extension installation and trust**

Chrome extensions add features to the browser, but they also expand what the browser can access. You install extensions from the Chrome Web Store, and you should only approve extensions you trust.

Chrome limits some extension actions in special browsing modes. You cannot add extensions while browsing in Incognito mode or as a guest. In managed environments, your organization may block certain extensions.

- Open the Chrome Web Store.
- Select the extension you want.
- Choose `Add to Chrome`.
- Review any requested permissions or data access.
- Confirm with `Add extension` only if you trust the extension.

If you use Chrome on a phone, you must be signed in with your Google Account. After you add the extension, Chrome on your computer may show that it was installed and may ask you to enable it if permissions are required.

**Extension usage and permissions**

Extensions often need explicit permission before they can work. That permission prompt is important because it tells you what the extension can access or change.

Use the extension from the icon to the right of the address bar after installation. If the extension requests additional permissions later, review them before enabling the extension.

- Check the permission request before approving.
- Use the extension from the toolbar icon.
- Enable only the access you understand and need.
- Treat unexpected permission requests as a warning sign.

**Managed Chrome environments**

In work or school environments, Chrome may be managed by policy. That means extension availability is not fully under your control, and some extensions may be blocked by the organization.

This matters for troubleshooting because a missing extension is not always a user error. The browser may be enforcing a policy restriction.

- Confirm whether the device is managed.
- Check whether the organization blocks the extension.
- Use policy-aware troubleshooting before reinstalling or reconfiguring.

**Basic extension development context**

A Chrome extension starts with a directory of files and a `manifest.json` file. The manifest describes the extension’s capabilities and configuration.

A common starting point is a simple extension that shows a message when the user clicks the toolbar icon. The manifest often includes an `action` key, which defines the icon and the popup page Chrome uses.

```json
{
  "manifest_version": 3,
  "name": "Hello World",
  "version": "1.0",
  "action": {
    "default_icon": "icon.png",
    "default_popup": "popup.html"
  }
}
```

- Create a new directory for the extension files.
- Add `manifest.json` to describe the extension.
- Define the extension action if you want a toolbar icon and popup.
- Load the extension locally during development.

**Practical exam takeaways**

For exam prep, focus on the difference between safe browser sourcing, extension trust, and policy-based restrictions. The key idea is that browser security is not only about the browser itself; it also includes what you install into it and how you verify it.

- Use trusted sources for browser downloads.
- Verify files when possible.
- Approve only trusted extensions.
- Remember that managed devices can block extensions.
- Know that `manifest.json` is the core extension configuration file.
