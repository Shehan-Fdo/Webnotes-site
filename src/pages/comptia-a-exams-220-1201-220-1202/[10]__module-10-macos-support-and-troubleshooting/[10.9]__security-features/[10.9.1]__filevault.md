## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**FileVault overview**

FileVault is a built-in macOS disk encryption feature. Its purpose is to protect the data on a Mac’s startup drive so unauthorized users cannot read it if they gain physical access.

- It encrypts the entire startup drive, including the operating system, applications, and user files.
- It uses **XTS-AES-128** encryption with a 256-bit key.
- It helps protect data when the Mac is powered off or otherwise locked at startup.

**How FileVault access works**

FileVault ties disk access to authorized user accounts. At startup, a user must authenticate before the disk unlocks.

- An authorized user account can unlock the disk with its password.
- After the Mac is unlocked, users do not need to keep re-entering passwords for normal use.
- This makes FileVault a full-disk protection layer rather than a per-file prompt system.

**Recovery key**

When you enable FileVault, macOS provides a recovery key. You use this key if you forget the password needed to unlock the disk.

- You can store the recovery key with Apple.
- You can also keep it yourself in a secure location.
- Treat the recovery key as sensitive access information because it can restore access to encrypted data.

**Practical exam takeaway**

FileVault is a macOS security feature that protects data at rest by encrypting the startup drive. In A+ terms, focus on what it protects, how users unlock it, and why the recovery key matters.

- It is a **disk encryption** feature, not a file-sharing tool.
- It protects against unauthorized access to the drive contents.
- It depends on user authentication and recovery planning.
