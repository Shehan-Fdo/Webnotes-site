## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Overview**

Gatekeeper is a built-in macOS security feature that helps protect users from suspicious applications, malicious software, and other untrusted code. It focuses on controlling what gets installed, which makes it an important part of the macOS security model.

In practice, Gatekeeper can also block legitimate software if the app is less common or not recognized by macOS. For that reason, you should understand both its protective purpose and the common troubleshooting approach when a trusted app fails to install.

**What Gatekeeper Does**

Gatekeeper is designed to reduce the risk of malware by limiting installation of lesser-known or untrusted applications. It acts as a checkpoint before an app is allowed onto the system.

- **Security control**: Helps prevent installation of malware and unwanted software.
- **Install-time focus**: It deals with installation, not the running of applications.
- **User impact**: It may stop legitimate apps if macOS does not trust the source or package.

> Re-enable Gatekeeper after installing a legitimate app if you temporarily disable it.

**When You May Encounter It**

You are most likely to notice Gatekeeper when macOS displays a warning or blocks an application from installing. This often happens with software downloaded outside the most trusted distribution paths.

Common situations include:
- A legitimate app is blocked during installation.
- A user wants to install software from an unfamiliar developer.
- A security prompt appears because macOS does not fully trust the app.

**Troubleshooting and Temporary Bypass**

If you need to install a trusted application that Gatekeeper blocks, you may temporarily disable it, complete the installation, and then turn it back on. The evidence describes this as a practical workaround, but it should be used carefully.

1. Open the Apple menu and launch `System Preferences`.
2. Go to `Security & Privacy` and select the `General` tab.
3. If the lock is closed, click it and authenticate with your username and password.
4. Select `Anywhere` under `Allow applications downloaded from:`.
5. Re-lock the setting if needed and close `System Preferences`.
6. Re-enable Gatekeeper after the installation is complete.

**Key Exam Points**

- Gatekeeper is part of macOS security.
- It helps prevent installation of suspicious or untrusted applications.
- It can block legitimate software as well as malicious software.
- It only affects installation, not application execution.
- Temporary disabling is a troubleshooting step, not a permanent best practice.

**Security Context**

Gatekeeper is one layer in Apple’s effort to keep macOS users safer from untrusted code. It is not a guarantee that every harmful app is blocked, and research has shown that flaws can exist in security controls. For exam purposes, focus on its intended role: reducing exposure to risky software during installation.
