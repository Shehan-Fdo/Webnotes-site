## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Startup issue patterns**

Startup problems often show up as a system that halts before you can reach the desktop, or as an operating system that becomes unstable during boot or shutdown. These issues can point to hardware, drivers, or software conflicts, so you should treat the symptom as a clue rather than a final diagnosis.

- A **blue screen of death** indicates a critical failure that stops the system immediately.
- Startup or shutdown failures can be related to bad hardware.
- Driver problems can prevent normal boot behavior.
- An application can also trigger a startup issue if it conflicts with system resources.
- On macOS, a spinning beach ball can indicate an app waiting on an unavailable resource or competing for the same resource.

**Common recovery options**

When Windows fails during startup, you can use built-in recovery paths to return the system to a known working state. These options are useful when the failure appears after a recent change and you want to reverse that change without rebuilding the machine.

- Use **Last Known Good** to roll back to a previous configuration.
- Use **System Restore** to return to an earlier restore point.
- Choose a specific recovery option when you need a targeted rollback.

**macOS startup symptoms**

macOS startup issues can present differently from Windows, but the troubleshooting mindset is the same: identify whether the problem is tied to a missing resource, a damaged startup volume, or a failed boot path. A flashing folder with a question mark is a strong sign that the Mac cannot find a valid startup disk.

- A spinning beach ball suggests the system is waiting on a resource or encountering contention.
- A flashing folder with a question mark indicates a startup disk problem.
- If the startup volume is missing or damaged, the Mac may not boot normally.

**Practical troubleshooting approach**

Start with the most recent change and work backward. If the issue began after a driver install, application install, or system change, focus on undoing that change first. If the system will not boot far enough to troubleshoot normally, use recovery tools or startup repair options that can restore a known good state.

1. Identify when the failure occurs: startup, shutdown, or after login.
2. Check whether the symptom suggests hardware, driver, or application conflict.
3. On Windows, try **Last Known Good** or **System Restore**.
4. On macOS, look for signs of a missing or damaged startup volume.
5. Use the least disruptive recovery option first.

**What to remember for exam prep**

The exam usually expects you to match the symptom to the most likely cause or recovery action. Focus on the difference between a system-wide boot failure and an application-level hang, and remember that startup issues are often caused by recent changes.

- **BSOD** points to a serious system failure.
- Boot-time failures can involve hardware or drivers.
- macOS spinning beach ball behavior often reflects resource contention.
- A question-mark folder on macOS points to startup disk trouble.
- Windows recovery features can help you revert to a working configuration.
