## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Storage-related performance symptoms**

Storage problems often show up as general slowness rather than a single obvious failure. A system may boot slowly, take longer to open apps, or feel sluggish after sign-in. On macOS, these issues can be caused by low disk space, background items, or aging software that gradually stretches system resources.

- **Low disk space** can make a device feel sluggish even when CPU and memory usage do not look extreme.
- **Background apps** and login items can add startup delay and keep the system busy after boot.
- **Accumulated small issues** often combine into noticeable performance loss over time.

A practical troubleshooting approach is to check storage first when a machine feels slow but otherwise appears functional. That gives you a fast way to rule out one of the most common causes of performance complaints.

**macOS storage and maintenance tools**

macOS includes built-in tools that help with storage management and disk repair. Time Machine is the built-in backup feature, and it keeps hourly backups for the last 24 hours, daily backups for the last month, and weekly backups after that. It continues until the backup drive fills up, then it removes the oldest backups as new ones are created.

Disk Utility is the macOS tool for disk maintenance. It is typically used as needed rather than on a fixed schedule. If you suspect drive problems, you can use the First Aid function in Disk Utility to check and repair the disk.

- **Time Machine** provides rolling backups that let you recover earlier versions of data.
- **Disk Utility** supports on-demand maintenance for storage-related issues.
- **First Aid** is the repair-oriented option when a drive shows signs of trouble.

> Use disk maintenance tools when symptoms point to the drive itself, not as routine busywork.

**macOS file types and install packages**

macOS uses file extensions to identify file types, just like other operating systems. Two important extensions are `.dmg` and `.pkg`.

- `.dmg` is an Apple disk image.
- `.pkg` is an installer package.
- A mounted `.dmg` appears like a drive in Finder.

These file types matter during troubleshooting because installation media and app distribution often rely on them. If you understand what each extension represents, you can better explain how software is installed or mounted on a Mac.

**Troubleshooting workflow**

When a user reports a slow Mac, start with the simplest checks and move toward more specific causes. Focus on storage, startup items, and software state before assuming a hardware failure.

1. Check available disk space.
2. Review login items and background apps.
3. Confirm the system and apps are up to date.
4. Use Disk Utility if the drive appears to need maintenance.
5. Verify backups with Time Machine before making major changes.

This sequence keeps you practical and minimizes unnecessary disruption. It also helps you separate a storage problem from a broader performance issue.

**Key takeaways**

Storage issues often present as slow boot times, delayed app launches, and general sluggishness. On macOS, Time Machine protects data with rolling backups, while Disk Utility and First Aid help with disk maintenance. File extensions such as `.dmg` and `.pkg` are part of everyday macOS software handling and are useful to recognize during support tasks.
