## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Network troubleshooting overview**

Network issues often show up as missing connectivity, unstable wireless performance, or degraded application quality. A practical troubleshooting approach starts with simple checks that tell you whether the problem is local to the device, the cable, the wireless link, or the broader network path.

- Check for a **link light** on a wired Ethernet connection.
- If the light is off, suspect a bad cable or a broken connection to the device on the other end.
- Use a loopback test by pinging `127.0.0.1` to confirm the local TCP/IP stack responds.
- Treat intermittent wireless connections as a separate symptom from total loss of connectivity.
- Watch for performance symptoms such as high latency, high jitter, and poor VoIP quality.

**Basic verification steps**

You use verification to narrow the fault domain before changing settings. Start with the simplest observable indicators, then move toward software and configuration checks if the physical layer looks healthy.

1. Confirm the device has a physical network connection if it uses Ethernet.
2. Verify the link light is present and stable.
3. Ping `127.0.0.1` to test the local network stack.
4. If the local test works, continue checking the path beyond the device.

> If the link light is absent, do not assume the network is down. A local cable or port issue can produce the same symptom.

**Common symptoms and what they suggest**

Different symptoms point to different parts of the network path. Matching the symptom to the likely cause helps you avoid unnecessary changes.

- **Lack of connectivity** can indicate a local adapter, cable, or upstream path problem.
- **Intermittent wireless connections** often point to unstable radio conditions or roaming issues.
- **High latency** affects responsiveness and can make applications feel slow.
- **High jitter** can disrupt real-time traffic.
- **Poor VoIP quality** often appears when latency or jitter is elevated.
- **Port flapping** suggests a link that repeatedly goes up and down.

**macOS network-related checks**

macOS provides built-in configuration areas that help you review and adjust network access. The System Preferences app acts as a central place for operating system settings, including network adapters.

- Use **System Preferences** to access network-related configuration.
- Review adapter settings when you need to change how the Mac accesses the network.
- Use the same area to manage other system settings that may affect connectivity indirectly.

macOS also includes tools that support maintenance when a problem is not purely network-based. `Disk Utility` can help with drive-related issues, and its `First Aid` function is used when you need to check a disk for problems.

- Use `Disk Utility` as needed rather than as a routine scheduled task.
- Use `First Aid` when drive problems may be affecting system behavior.
- Use `Time Machine` for rolling backups if you need recovery from system or data issues.

**Practical troubleshooting mindset**

A good workflow is to separate physical connectivity, local stack health, and service quality. That keeps you from chasing the wrong layer.

- Physical layer: check cables, ports, and link lights.
- Local host layer: confirm the device responds to `ping 127.0.0.1`.
- Service quality layer: look for latency, jitter, and voice degradation.
- Platform layer: on macOS, review `System Preferences` and use `Disk Utility` when needed.

**Key takeaways**

Network troubleshooting is most effective when you start with visible, low-cost checks and move outward only when each layer looks healthy. The link light and loopback ping give you fast evidence about whether the issue is local, while symptoms like jitter and poor VoIP quality point to performance problems rather than total failure.
