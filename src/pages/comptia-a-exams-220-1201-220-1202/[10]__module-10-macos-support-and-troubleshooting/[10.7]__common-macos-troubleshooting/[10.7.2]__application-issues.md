## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Application and file-type basics**

Application issues often start with the file itself, the app that opens it, or the system state around it. On macOS, file extensions help identify what a file is used for, and some extensions have specific meanings that matter during troubleshooting.

- A file extension is the suffix at the end of a file name that often indicates the file type.
- `.dmg` is an Apple disk image that behaves like a mounted drive on macOS.
- `.pkg` is an installer package used to install software on macOS.
- `.zip` is a compressed archive format you may encounter across platforms.
- File-type awareness helps you decide whether the issue is with the document, the installer, or the application itself.

When you troubleshoot, start by confirming that the file type matches the app you expect to use. A file that will not open may simply need the correct application, while an installer may need to be mounted or launched in the proper format.

**Basic troubleshooting approach**

Most application problems respond best to simple, low-risk checks first. Practical troubleshooting starts with the easiest causes before you move to deeper repair steps.

- Restart the system to clear temporary application state.
- Force-quit an unresponsive app if it stops responding.
- Check available storage space, since low storage can contribute to instability.
- Verify that macOS and the affected apps are up to date.
- Use safe mode to help determine whether third-party software is involved.
- Create a new user account if you suspect the issue is tied to user-specific settings or preferences.

> If the app works in safe mode, third-party software is a likely factor. If a new user account behaves normally, the original account data or preferences may be the cause.

**Backup and recovery tools**

Backups matter when application issues lead to lost files or when you need to recover after a repair attempt. macOS includes Time Machine, which automatically backs up files and supports file recovery through a familiar Finder-style interface.

- Time Machine stores backup history so you can restore a file from a specific point in time.
- You can move through dates on the right side of the restore view to find the version you want.
- On a laptop away from the backup drive, macOS can save snapshots and update the backup later when you reconnect.
- Backup planning reduces risk before you attempt larger fixes such as reinstalling software or the operating system.

**Deeper repair options**

If basic steps do not resolve the issue, move to recovery-oriented tools. Recovery Mode gives you access to repair and reinstall options that can address more persistent problems.

1. Boot the Mac into Recovery Mode.
2. Run Disk Utility First Aid if you suspect disk-related corruption.
3. Reinstall macOS if the problem persists and you need a deeper system repair.
4. Back up data before reinstalling so you do not lose important files.

These steps are more invasive than routine troubleshooting, so use them after you rule out simpler causes. They are most useful when the system or application environment appears damaged rather than when the problem is limited to a single temporary glitch.

**Exam-focused takeaways**

Application issues are often a mix of file handling, software state, and user environment. For exam prep, focus on matching the symptom to the least disruptive fix first.

- Confirm the file type and the app expected to open it.
- Check for updates before assuming the app is broken.
- Use safe mode and a new user account to isolate the source of the problem.
- Use Time Machine for recovery when files need to be restored.
- Use Recovery Mode when you need disk repair or a reinstall path.
