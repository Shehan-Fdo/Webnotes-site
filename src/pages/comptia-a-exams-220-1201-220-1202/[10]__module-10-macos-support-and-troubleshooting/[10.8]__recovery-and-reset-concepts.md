## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Recovery and reset overview**

Recovery and reset concepts focus on getting a system back to a usable state after data loss, corruption, or software problems. For A+ Core 2, you should recognize that recovery methods vary by operating system, but the goal stays the same: restore data, repair the system, and return to normal operation with minimal disruption.

CompTIA A+ Core 2 includes backup and recovery methods as part of the operating-system skill set. That means you should understand both the tools that restore files and the environments that help you repair a system when it will not boot normally.

- **Backup** supports recovery by preserving data before a failure occurs.
- **Restore** returns files or system data from a backup source.
- **Recovery environment** gives you a way to repair or troubleshoot when the normal desktop is unavailable.
- **Reset** concepts usually involve returning a system to a known working state.

**macOS recovery tools**

macOS includes built-in tools that support file recovery and system repair. One key example is Time Machine, which automatically backs up files and lets you restore them through a familiar Finder-style interface. You can move through backup dates and choose the version you want to recover.

Time Machine is especially useful because it can keep working even when you are away from the backup drive. On a laptop, macOS can save snapshots locally and update the backup later when the system reconnects to the backup destination.

- **Time Machine** provides automatic backups and file restoration.
- Backup browsing uses dates so you can return to a specific point in time.
- Local snapshots help protect data when the backup drive is not connected.
- Restores use a simple file-selection workflow rather than a complex recovery utility.

> If you are restoring data, choose the correct backup point before you overwrite newer changes.

**Recovery modes and offline repair**

When a system cannot start normally, recovery modes provide a controlled environment for troubleshooting. On macOS, recovery features are a newer addition compared with earlier approaches, but they are now a standard part of system repair workflows.

Older macOS and Mac OS X systems also used Single User Mode (SUM) as a command-line recovery option. In that mode, you could run file-system repair commands such as `fsck` and then restart the system if the repair succeeded. This shows the general recovery principle: use a minimal environment to repair the startup volume when the full operating system is unavailable.

- **Single User Mode** is a command-line recovery environment used on older macOS systems.
- `fsck` checks and repairs file-system issues.
- Recovery tools are useful when the startup volume needs repair before normal booting can continue.
- Offline recovery is valuable when the installed operating system cannot load fully.

**Practical recovery workflow**

1. Identify whether the issue is data loss, file corruption, or a boot problem.
2. Use the appropriate recovery source, such as a backup or recovery environment.
3. Restore the needed file or repair the affected volume.
4. Verify that the system boots and the data is usable.
5. Reconnect any backup source so pending snapshots or updates can sync.

**What to remember for A+**

A+ Core 2 expects you to connect recovery tools to real support tasks. You should know that macOS includes Time Machine for backup and restore, and that recovery environments help when the system cannot boot normally. You should also understand that backup and recovery are part of broader operating-system management across macOS, Linux, and Windows.

- **Time Machine** is the primary macOS file-recovery tool highlighted in the evidence.
- **Recovery mode** helps repair or restore a system outside the normal desktop.
- **Offline recovery** is important when the installed OS is not usable.
- **Backup and recovery methods** support data integrity and system continuity.
