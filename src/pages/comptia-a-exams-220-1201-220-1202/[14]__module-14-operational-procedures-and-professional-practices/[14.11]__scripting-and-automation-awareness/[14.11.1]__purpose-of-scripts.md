## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**What scripts do**

Scripts automate a series of commands so you do not have to type the same steps interactively at the command line. They are useful when a task repeats often, needs to run quickly, or should happen without a person watching it.

- **Automation** reduces repetitive typing and duplicated work.
- Scripts can run while you are away from the computer or even while you are sleeping.
- They help you monitor an environment and respond when problems occur.
- They reduce the chance of human error from manual command entry.
- They execute faster than a person can type the same commands.

**Why scripts matter in IT**

In IT, scripts are a practical way to handle routine operations across one system or many systems. A script can be reused reliably, which makes it a dependable way to perform the same set of actions whenever needed.

- Scripts are especially useful when multiple systems need the same commands.
- They free you to focus on other work instead of repeating the same tasks.
- They support consistent results because the process is defined ahead of time.
- They can be triggered manually, at startup, or by another program.

**Common use cases**

Scripts are often used for everyday administrative tasks that would otherwise take time and attention from a technician.

- Restarting computers
- Mapping network drives
- Installing applications
- Running automated backups
- Monitoring systems to confirm they are operating as expected

These use cases show the main value of scripting: you define the action once, then let the computer repeat it as needed.

**How scripts improve reliability**

Scripts help standardize work. When a task is written as a script, the same instructions run the same way each time, which is useful for repeatable maintenance and recovery actions.

- They can run without human intervention.
- They avoid mistakes caused by typing commands manually.
- They are dependable for recurring tasks.
- They can be used to resolve issues automatically when conditions are met.

> Scripts are not mainly about writing software from scratch; they are about automating specific tasks efficiently and consistently.

**Key idea to remember**

A script is a reusable set of instructions that automates work. For the A+ exam, focus on the purpose: save time, reduce errors, and perform routine tasks automatically.
