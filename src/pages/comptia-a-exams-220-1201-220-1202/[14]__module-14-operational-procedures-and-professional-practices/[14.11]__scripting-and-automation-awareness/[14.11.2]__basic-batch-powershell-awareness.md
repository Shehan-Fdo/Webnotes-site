## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Scripting basics**

Automation and scripting are core IT skills because they let you repeat tasks without manual effort. For A+ awareness, focus on recognizing common scripting options and choosing the right one for the task and operating system.

- **Batch files** are a Windows scripting option used at the command line.
- Batch files commonly use the `.bat` file extension.
- The Windows command interpreter `cmd.exe` is the traditional environment for batch scripting.
- Batch scripting has long roots in DOS and OS/2-era Windows administration.
- **PowerShell** is another Windows scripting and automation tool.
- Script choice depends on the task, the application, and the operating system.

A practical way to think about scripting is that you are matching the tool to the job. Windows tasks may use batch or PowerShell, while other environments may use shell scripts, JavaScript, or Python depending on the platform and purpose.

**Batch files in Windows**

Batch files are simple command files that run in Windows from the command line. They are useful when you want to chain commands together or automate repetitive administrative steps.

- Batch scripts are associated with `cmd.exe`.
- They are part of the Windows command-line workflow.
- They are a basic automation method rather than a full programming environment.
- They are still relevant for legacy support and simple task automation.

> Batch files are the fundamental scripting language of Microsoft Windows in many A+ study contexts, so you should recognize them quickly by name and extension.

**PowerShell awareness**

PowerShell is a Microsoft scripting environment used for automation and administration. It is more modern than batch scripting and is commonly associated with Windows management tasks.

- PowerShell supports automation through scripts and command-line use.
- It is part of the scripting tools you should recognize for A+.
- PowerShell is available in multiple versions, including stable, LTS (Long Term Servicing), and preview releases.
- Microsoft maintains LTS releases with critical security updates and servicing fixes.
- PowerShell scripts commonly use the `.ps1` extension.

PowerShell is not just a replacement for batch files; it is a separate tool with its own syntax and management model. For exam awareness, the key point is knowing when PowerShell is the likely Windows automation choice and identifying its script files.

**Choosing a scripting language**

The best scripting language depends on what you need to do and where the script runs. In practice, you choose based on the operating system and the task.

- Use Windows-oriented scripting when you are automating Windows tasks.
- Use platform-appropriate scripting when the environment is not Windows.
- Expect to see multiple scripting languages in IT, including batch, PowerShell, shell scripts, JavaScript, and Python.
- Match the script to the application or system you are automating.

This is a common exam theme: you do not need deep programming knowledge, but you do need to recognize the purpose and environment of each scripting option.

**Quick recognition points**

| Item | What to remember |
|---|---|
| Batch file | Windows command-line script, usually `.bat` |
| `cmd.exe` | Traditional Windows command interpreter |
| PowerShell | Microsoft automation and scripting environment |
| `.ps1` | Common PowerShell script extension |
| Script selection | Depends on task and operating system |

**Exam focus**

For A+ purposes, you should be able to identify basic scripting terminology and understand the role of batch and PowerShell in Windows automation.

- Know that scripting helps automate repetitive IT tasks.
- Know that batch files are tied to Windows command-line use.
- Know that PowerShell is a separate Windows automation tool.
- Know that the operating system and task determine the best script choice.
- Know that multiple scripting languages exist, even if the exam only expects awareness rather than deep coding skill.
