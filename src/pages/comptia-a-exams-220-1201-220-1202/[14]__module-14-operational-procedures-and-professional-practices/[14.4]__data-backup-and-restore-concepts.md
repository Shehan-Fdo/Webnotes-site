## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Backup and restore fundamentals**

A backup is a separate copy of data, files, and configurations that you can use to recover from loss, corruption, hardware failure, or system issues. Restore is the process of retrieving that backup and putting the data back on the original system or a different system.

Backups support availability and business continuity. They reduce downtime when accidental deletion, equipment failure, or a cyberattack affects data.

- **Data backup** means the copy is stored separately from the original.
- **Recovery** means you return usable data after an incident.
- **System backup** can include files, settings, and configurations, not just user documents.
- **Restore target** may be the same device or a replacement system.

**Common backup types**

Backup methods differ in how much data they copy and how often they need to run. The evidence specifically identifies full, incremental, and differential backups as common approaches.

- **Full backup** copies all selected data.
- **Incremental backup** copies changes since the last backup.
- **Differential backup** copies changes since the last full backup.

These methods help you balance speed, storage use, and recovery effort. In practice, the right choice depends on how much data changes and how quickly you need to recover.

**Where backups are stored**

Backups can live on local media or remote storage. Local options are useful for quick access, while remote options help protect against site-level loss.

- **Local storage** can include removable or attached devices.
- **Network-attached storage (NAS)** provides network-based storage on a local environment.
- **Cloud storage** keeps backup data offsite through a service provider.

Using more than one location improves resilience. If one copy is unavailable, another copy can still support recovery.

**Restore planning**

Restore is not just about having a copy; it is about being able to use that copy effectively when needed. You should know what data you are restoring, where it goes, and whether the destination system matches the backup.

1. Identify the data or system that needs recovery.
2. Select the appropriate backup copy.
3. Restore the data to the original system or a new system.
4. Verify that the restored data is usable.

> A backup is only valuable if you can restore it successfully when data loss occurs.

**Operational considerations**

Backup and recovery planning should focus on practical protection against common failure scenarios. The goal is to preserve critical data and minimize downtime.

- Protect against **data loss** from deletion, corruption, or failure.
- Plan for **hardware failure** and **system corruption**.
- Keep backups accessible enough to support timely recovery.
- Match the backup method to the importance and change rate of the data.

**Exam focus reminders**

For A+ exam prep, remember the core relationship between backup type, storage location, and restore outcome. Questions often test whether you can distinguish the method used, the place the backup is stored, and the purpose of recovery.

- Know the difference between **full**, **incremental**, and **differential** backups.
- Recognize local versus remote storage choices.
- Understand that restore can target the original system or a different one.
- Tie backup strategy to continuity and reduced downtime.
