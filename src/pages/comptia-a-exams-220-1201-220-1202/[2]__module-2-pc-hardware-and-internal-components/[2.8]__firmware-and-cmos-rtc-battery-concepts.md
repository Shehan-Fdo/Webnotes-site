## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Firmware basics**

Firmware is the low-level software that starts the computer and manages early hardware initialization. On PCs, this is commonly referred to as the **BIOS** (Basic Input/Output System), and modern systems may use **UEFI** (Unified Extensible Firmware Interface) instead of a legacy BIOS.

The firmware is stored in flash memory on modern systems, which means you can update it with software rather than replacing chips. During startup, firmware performs the **POST** (Power-On Self-Test) to verify core hardware before handing off to a boot loader.

- BIOS initializes memory and checks the system during startup.
- POST looks for key hardware such as the CPU, memory, display, and keyboard.
- If hardware passes diagnostics, the system looks for a boot loader on a storage device.
- Legacy BIOS and UEFI are both firmware, but UEFI is the newer approach.

**CMOS and RTC battery concepts**

The **CMOS** (Complementary Metal-Oxide-Semiconductor) area is associated with storing system configuration data such as date, time, hard drive information, and boot priorities. You may also see this described as **RTC** (Real-Time Clock), **NVRAM** (Non-Volatile RAM), or CMOS RAM in older references.

A small coin-cell battery powers this memory when the computer is off. Its job is to preserve settings and the clock, not to power the entire system. In many modern systems, settings are integrated into other chipset components, but the battery concept remains important for exam troubleshooting.

- The battery maintains system time and configuration when power is off.
- Common stored values include date, time, drive information, and boot order.
- Older systems used dedicated CMOS RAM more directly; newer systems often integrate settings elsewhere.
- A typical CMOS battery can last around 10 years, but lifespan varies by environment and usage.

**What happens when the battery fails**

When the CMOS/RTC battery weakens or fails, the system may lose stored settings after shutdown. That can lead to incorrect time, reset BIOS settings, or boot problems if the machine no longer remembers its configuration.

Troubleshooting often starts with checking the symptoms, then testing or replacing the battery if needed. If the system shows date/time errors, BIOS setting loss, or booting issues, the battery is a practical suspect.

- Date and time may reset after the system is powered off.
- BIOS settings may revert to defaults.
- Boot behavior can change if stored configuration is lost.
- Incorrect settings can prevent hardware detection or cause boot failure.

**BIOS setup and configuration impact**

You edit CMOS-related settings through the BIOS setup utility. The BIOS reads these settings during boot to determine how the PC should function, including boot priority and hardware behavior.

Incorrect configuration can create problems even when the hardware is healthy. For exam purposes, connect the symptom to the likely cause before changing parts.

- BIOS setup is where you change stored system settings.
- Boot priorities affect which device the system tries first.
- Wrong settings can stop the system from booting or recognizing devices.
- Firmware settings are part of the startup path, so they matter early in troubleshooting.

**Troubleshooting approach**

1. Check for symptoms such as wrong time, lost settings, or boot errors.
2. Verify whether the BIOS/UEFI configuration still matches the expected hardware.
3. Test the CMOS/RTC battery if the system forgets settings after power loss.
4. Replace the battery if it no longer holds configuration.
5. Re-enter BIOS setup and restore the required settings.

> If the system repeatedly loses date, time, or boot order, treat the battery as a likely cause before assuming a deeper motherboard failure.

**Exam focus**

For A+ questions, separate these ideas clearly:

- **Firmware** starts the machine and performs early hardware checks.
- **BIOS/UEFI** stores and applies startup configuration.
- **CMOS/RTC battery** preserves settings and time when power is off.
- Battery failure usually shows up as lost settings, incorrect time, or boot issues.
