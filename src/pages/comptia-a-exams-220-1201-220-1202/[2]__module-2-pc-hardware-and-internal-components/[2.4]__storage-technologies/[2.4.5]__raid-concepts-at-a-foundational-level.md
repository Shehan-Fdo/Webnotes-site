## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**RAID overview**

RAID stands for **redundant array of independent disks**. You use RAID to combine multiple storage devices for better performance, redundancy, or both. Not every RAID level provides fault tolerance, so you need to match the level to the goal.

Common RAID levels on A+ exams include:

- **RAID 0**: striping for performance
- **RAID 1**: mirroring for redundancy
- **RAID 5**: striping with parity
- **RAID 10**: a nested RAID that combines striping and mirroring

> RAID is not automatically redundant. Some levels improve speed only, while others protect data if a drive fails.

**Core RAID concepts**

RAID uses different techniques to distribute or duplicate data across disks. The main ideas are:

- **Striping** spreads data across drives to improve performance.
- **Mirroring** writes the same data to more than one drive.
- **Parity** stores calculated information that helps recover data after a drive failure.
- **Nested RAID** combines RAID levels into one array design.

A practical exam takeaway is that the RAID number does not tell you everything by itself. You still need to know whether the array favors speed, redundancy, or a balance of both.

**Common RAID levels**

| RAID level | Main idea | Typical exam takeaway |
|---|---|---|
| RAID 0 | Striping | Improves performance, but does not provide redundancy |
| RAID 1 | Mirroring | Provides redundancy by duplicating data |
| RAID 5 | Striping with parity | Balances performance and redundancy |
| RAID 10 | Stripe of mirrored drives | Nested RAID that combines both approaches |

RAID 0 is useful when speed matters more than protection, but it is not appropriate for critical data because it does not survive a drive failure. RAID 1 is the simplest redundancy-focused option because each disk has a mirror. RAID 5 adds parity so the array can tolerate a drive failure while still using storage efficiently. RAID 10 combines mirrored pairs and striping, so it is often described as a stripe of mirrors.

**How to reason about RAID questions**

When you see a RAID question, identify the requirement first:

1. Decide whether the goal is performance, redundancy, or both.
2. Check how many disks are available.
3. Match the RAID level to the requirement.
4. Watch for wording such as `striping`, `mirroring`, `parity`, or `fault-tolerant`.

This approach helps you avoid common exam traps. For example, a question that asks for the fastest option may point to RAID 0, while a question that asks for protection against a single disk failure points toward a redundant level.

**Exam-ready reminders**

- **RAID 0** gives speed, not safety.
- **RAID 1** gives redundancy through duplication.
- **RAID 5** uses parity to balance efficiency and fault tolerance.
- **RAID 10** is a nested design that combines striping and mirroring.
- The term **independent disks** is the modern wording for RAID.

**What to remember for the exam**

RAID is a storage strategy, not a single technology. The key is to recognize the tradeoff between performance and redundancy. If a scenario emphasizes faster reads and writes, think striping. If it emphasizes surviving a disk failure, think mirroring or parity-based protection. If it combines both, think nested RAID such as RAID 10.
