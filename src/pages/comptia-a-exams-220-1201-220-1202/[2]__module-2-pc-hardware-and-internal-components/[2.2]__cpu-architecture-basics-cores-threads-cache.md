## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**CPU architecture overview**

A CPU (central processing unit) is the main processing chip in a computer. It handles instructions, performs calculations, and coordinates system activity. Modern CPUs include multiple design features that affect performance, including clock speed, architecture, bus width, cache size, and operating system optimization.

- **Clock speed** is the rate a CPU runs at, commonly shown in gigahertz (`GHz`) on modern systems.
- Older CPUs may list speed in megahertz (`MHz`), but the same idea applies: it is the processor’s operating frequency.
- Overall system performance is not determined by CPU speed alone.
- CPU architecture and other components also influence real-world performance.

**Cores and parallel processing**

Older CPUs were designed to process one transaction at a time. Modern processors increase throughput by using multiple cores inside a single chip. Each core acts like an independent processing unit, so the CPU can handle more than one task at the same time.

- A **core** is an individual processing unit inside the CPU.
- Common configurations include dual-core, quad-core, and higher core counts.
- More cores allow more simultaneous transactions.
- Core count does not automatically guarantee faster performance for every workload, but it improves multitasking and parallel processing potential.

**Threads and logical processing**

A thread is a unit of work that the operating system schedules for execution. CPU features such as hyper-threading can make a single physical core appear as multiple logical processors to the operating system.

- **Threads** help the CPU manage multiple execution paths.
- Hyper-threading is an example of a technology that increases the number of usable CPU threads.
- Threads improve efficiency when workloads can be split across multiple execution paths.
- A logical processor is not the same as a physical core, so do not treat them as identical.

**Cache and memory hierarchy**

CPU cache is small, fast memory located close to the processor core. It stores copies of frequently used data so the CPU does not need to access main memory as often. This reduces the average time needed to retrieve data.

- **Cache** improves performance by reducing memory access delays.
- Cache is typically organized into levels such as `L1`, `L2`, and `L3`.
- `L1` and `L2` cache may be dedicated to each core.
- `L3` cache is often shared across multiple cores.
- Larger cache sizes can improve responsiveness, especially when the same data is reused frequently.

> Cache is faster than main memory, but it is also more expensive and limited in size, so CPU designs balance speed, capacity, and chip area.

**How these features work together**

CPU performance comes from the combination of architecture, core count, thread handling, cache design, and clock speed. A processor with a high clock speed may still perform differently from another processor with the same speed if the architecture or cache design is different.

- Compare CPUs by looking at more than one specification.
- Use core count and thread support to estimate multitasking capability.
- Use cache size and cache hierarchy to understand how efficiently the CPU can reuse data.
- Treat clock speed as one factor, not the only factor.

**Quick review**

1. Identify the CPU’s clock speed in `GHz` or `MHz`.
2. Check the number of physical cores.
3. Determine whether the CPU supports multiple threads per core.
4. Review cache levels and whether any cache is shared.
5. Evaluate the full design instead of relying on a single specification.

**Exam focus**

For A+ exam prep, be ready to explain the difference between cores and threads, describe what cache does, and recognize that CPU speed is only one part of overall system performance.
