## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Motherboard firmware basics**

The BIOS, or <abbr title="Basic Input/Output System">BIOS</abbr>, is the first software that runs when you power on a computer. It starts before the operating system and prepares the system to continue booting. You may also see it described as firmware, system BIOS, or ROM BIOS.

BIOS is stored on the motherboard, typically in flash memory on modern systems. It does not depend on an installed operating system to run, so it can initialize hardware even when no storage device or OS is present.

- **BIOS** starts the boot process before Windows, Linux, or macOS loads.
- **Firmware** is the general term for software stored on hardware that controls startup behavior.
- **Flash memory** commonly stores the BIOS on current motherboards.
- **ROM BIOS** is a legacy name that refers to read-only memory storage.

**POST and startup checks**

When the system powers on, the BIOS performs the Power-On Self-Test, or <abbr title="Power-On Self-Test">POST</abbr>. This is the hardware check that confirms the machine is ready to load an operating system.

POST verifies core components such as the CPU and memory. If a required component is missing or not working properly, the system may fail to continue booting.

- **POST** checks hardware before the operating system starts.
- **CPU** detection is part of the initial startup validation.
- **Memory** is also checked during the boot sequence.
- Startup problems often appear before any OS-specific screen loads.

**UEFI and modern motherboard setup**

<abbr title="Unified Extensible Firmware Interface">UEFI</abbr> is the modern replacement for legacy BIOS on many systems. It provides a more user-friendly graphical interface and mouse support on supported motherboards.

On ASUS systems, you can enter UEFI setup by pressing `DEL` or `F2` when the ASUS logo appears. The motherboard’s technical specifications should confirm whether UEFI BIOS support is available.

- **UEFI** is the modern firmware interface used on many current motherboards.
- **Graphical interface** and mouse support improve setup usability.
- **DEL** and `F2` are common keys for entering setup on ASUS systems.
- **Tech specs** help you confirm UEFI support for a specific motherboard model.

**Chipset and BIOS menu organization**

Motherboard firmware setup often groups options into menus such as Main, Advanced, Chipset, Boot, Security, Performance, and Exit. These menus help you find settings without searching through a single long list.

The chipset-related area is where you expect to find platform-level configuration options. Advanced menus may also include CPU, SATA, PCI, power management, and wake-event controls.

- **Chipset menu** groups motherboard platform settings.
- **Advanced menu** often contains CPU and storage configuration.
- **Boot menu** controls startup order and boot behavior.
- **Security menu** handles firmware-level protection options.
- **Exit menu** is where you leave setup after saving or discarding changes.

**Practical exam focus**

For A+ questions, connect the firmware term to the stage of startup and the hardware it checks. If a question describes a computer that has not yet reached the operating system, think BIOS or UEFI first.

A useful way to remember the sequence is: power on, firmware starts, POST runs, and then the operating system loads.

1. Identify whether the question is about pre-OS startup behavior.
2. Match the description to BIOS, UEFI, or POST.
3. Look for hardware checks such as CPU and memory.
4. Use the motherboard model or setup prompt to infer how firmware is accessed.

> If a system cannot reach the operating system, the issue may be happening during firmware initialization or POST rather than inside Windows or another OS.

**Quick comparison**

| Item | What it does | Notes |
|---|---|---|
| BIOS | Starts the system and initializes hardware | Legacy term still used in exam questions |
| UEFI | Modern firmware interface | Often includes GUI and mouse support |
| POST | Tests hardware during startup | Runs before the OS loads |
| Chipset menu | Organizes motherboard settings | Often includes platform-specific controls |
