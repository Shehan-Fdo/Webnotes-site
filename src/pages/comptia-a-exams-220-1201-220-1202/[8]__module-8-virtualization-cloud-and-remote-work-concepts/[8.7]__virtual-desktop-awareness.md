## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Virtual desktop fundamentals**

Virtual desktop infrastructure (VDI) is a virtualization approach that hosts desktop environments on centralized servers and delivers them to users over a network. Instead of running the operating system and applications locally, you access a virtual desktop from a remote device.

A **hypervisor** manages the virtual machines that make this possible. It sits between the hardware and the virtual systems, allocating resources such as CPU cycles, memory, and networking while keeping virtual machines separated.

- VDI hosts desktops centrally in a data center or cloud.
- Users can access a virtual desktop from nearly any device with an internet connection.
- The desktop experience can be similar to using a local machine.
- Hypervisors manage the interaction between the physical system and the virtual world.

**How VDI works**

VDI creates desktop instances on a server using virtualization. The server runs the desktop environment, and the user connects remotely to interact with it. This model supports multiple operating system or application instances on the server.

Common characteristics include:

- Centralized desktop hosting
- Remote access over a network
- Server-side generation of virtual desktops
- Separation of user desktops from endpoint hardware

**Why organizations use VDI**

VDI gives IT teams centralized control over desktops and can simplify administration. It also supports security and operational goals by keeping data and applications off the endpoint device.

Typical advantages include:

- **Centralized management** for desktops, updates, patches, and backups
- **Improved accessibility** from different devices and locations
- **Reduced hardware costs** because endpoints do less local processing
- **Enhanced security** by limiting data stored on user devices
- Faster deployment of desktops for new users

These benefits make VDI useful when an organization needs to manage many desktops consistently and efficiently.

**Remote desktop access vs. VDI**

Remote desktop technologies let you view and interact with another desktop over the network. Windows Remote Desktop Protocol (RDP) is a common example and can provide access to an entire desktop or a single application.

VDI and remote desktop access are related, but they are not identical:

| Topic | Focus | Typical use |
|---|---|---|
| VDI | Hosts the desktop itself on centralized servers | Delivering a full virtual desktop environment |
| RDP | Connects to a remote desktop session over the network | Viewing and controlling a desktop or application remotely |

RDP clients are available on multiple operating systems, including Windows, macOS, Linux, and iPhone platforms.

**Key exam takeaways**

- **VDI** delivers desktops from centralized servers instead of local devices.
- A **hypervisor** manages the virtual machines that run those desktops.
- VDI supports remote access, centralized administration, and easier maintenance.
- **RDP** is a remote access technology used to connect to an existing desktop session.
- Virtualization may also rely on CPU support and network connectivity to function effectively.
