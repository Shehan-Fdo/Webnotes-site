## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Hypervisor basics**

A <abbr title="virtual machine">VM</abbr> is a digital replica of a physical machine. A hypervisor is the software layer that makes virtualization possible by managing and allocating physical resources to VMs while keeping them isolated from one another and from the underlying hardware.

- Hypervisors manage resources such as CPU cycles, memory, and networking information.
- They keep multiple operating systems separated on the same physical machine.
- They are foundational to virtualization and cloud environments.

**Type 1 hypervisors**

A Type 1 hypervisor runs directly on the hardware. It sits on top of the bare metal server and has direct access to the system resources it controls. This architecture is often described as a bare metal hypervisor.

- It becomes the base layer of the system.
- It manages hardware resources without a host operating system in between.
- It is associated with strong separation between VMs and efficient use of server hardware.

**Type 2 hypervisors**

A Type 2 hypervisor runs on top of a regular operating system. This makes it a hosted hypervisor because the host OS sits between the hypervisor and the hardware.

- It is easier to deploy in many general-purpose environments.
- It depends on the host operating system for hardware access.
- It can reduce performance compared with a bare metal approach because of the extra software layer.

**Type 1 vs Type 2 at a glance**

| Feature | Type 1 | Type 2 |
|---|---|---|
| Location | Runs directly on hardware | Runs on top of an operating system |
| Common name | Bare metal hypervisor | Hosted hypervisor |
| Hardware access | Direct | Through the host OS |
| Typical effect | Better efficiency | Simpler deployment, potentially lower performance |

**How to remember the difference**

Think about where the hypervisor starts.

1. If it starts on the hardware itself, it is Type 1.
2. If it starts inside an operating system, it is Type 2.

> For exam questions, focus on the placement of the hypervisor layer. That is the fastest way to identify the type.

**Why the distinction matters**

The type you choose affects performance, control, and how resources are shared. Bare metal hypervisors are often preferred when you want maximum efficiency and direct hardware control. Hosted hypervisors are useful when convenience and simpler setup matter more than raw performance.

**Exam focus**

You should be able to identify these clues:

- **Type 1** means bare metal and direct hardware access.
- **Type 2** means hosted and runs on top of an operating system.
- Hypervisors manage VMs by allocating CPU, memory, and networking resources.
- Virtualization keeps systems isolated while sharing the same physical machine.
