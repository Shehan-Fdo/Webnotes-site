## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**SaaS overview**

Software as a Service (**SaaS**) is a cloud delivery model where the application is provided to you as a service. In the cloud-model context, it sits alongside Infrastructure as a Service (**IaaS**) and Platform as a Service (**PaaS**), but SaaS is the most application-focused of the three.

With SaaS, you use the software without managing the underlying hardware or the application stack. The provider handles the service delivery, while you focus on using the application itself.

- You access the software over the network, often through a browser.
- You do not install or maintain the server hardware.
- You do not manage the operating system for the service.
- You rely on the provider for availability and maintenance of the application.

**How SaaS differs from other cloud models**

The key exam idea is responsibility. SaaS shifts more of the operational burden to the provider than IaaS does. In IaaS, you still manage the operating system, applications, and data. SaaS removes most of that management from your side.

This makes SaaS useful when you want quick access to a ready-to-use application rather than a platform or raw infrastructure.

| Model | What you get | What you manage |
|---|---|---|
| IaaS | Hardware resources | Operating system, applications, data |
| PaaS | Application platform | Your application and data |
| SaaS | Finished application | Mostly usage and user-side settings |

**Why organizations use SaaS**

SaaS supports flexible use of software without large upfront infrastructure investment. It also fits situations where users need access from different locations or devices.

Common practical advantages include:

- **Accessibility** from anywhere with network access.
- **Scalability** when usage needs change.
- Reduced need for local installation and maintenance.
- Faster deployment of applications compared with building and hosting them yourself.

**Support and customer experience considerations**

Evidence about SaaS support emphasizes that the service is not only about the software itself. Ongoing support affects whether users stay satisfied and continue using the product.

Good SaaS support typically depends on:

- Clear onboarding so users can start effectively.
- Consistent help when issues arise.
- Visibility into user problems so support can respond well.
- Feedback from support teams that can inform product improvements.

If support tools are fragmented or teams lack customer context, responses can become slow or inconsistent. That can affect retention and the overall value of the service.

**Exam focus points**

For A+ purposes, focus on these SaaS traits:

- The application is delivered as a service.
- The provider manages the underlying infrastructure and most maintenance.
- You mainly consume the software rather than build or host it.
- SaaS is commonly associated with easy access and reduced administrative overhead.

> SaaS is about using a complete application in the cloud, not managing the servers that run it.

**Quick recall**

- **IaaS**: you manage more.
- **PaaS**: you manage your app on a platform.
- **SaaS**: you use the finished app.
