## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Virtualization overview**

Virtualization lets you run multiple virtual machines (VMs) on a single physical host. A VM behaves like an independent computer, but the hypervisor manages the shared hardware behind the scenes.

A hypervisor is also called a **virtual machine monitor (VMM)**. It abstracts the underlying hardware and keeps each VM isolated so workloads do not interfere with one another.

- **CPU** cycles are allocated by the hypervisor.
- **Memory** is divided and assigned to each VM.
- **Storage** and **networking** resources are also managed centrally.
- Isolation helps each VM operate as if it has dedicated hardware.

**Hypervisor role and value**

The hypervisor is the software layer that sits between the physical system and the virtual machines. It coordinates access to hardware and keeps the virtual environment organized.

This model improves hardware utilization because one machine can host multiple operating systems at the same time. It also simplifies management and can reduce the need for additional physical servers.

- Hypervisors support multiple operating systems on one host.
- They help maximize efficiency of compute, storage, and network resources.
- They are foundational in cloud computing, data centers, and edge computing environments.

**Type 1 and Type 2 hypervisors**

Hypervisors are commonly grouped into two types. The key difference is where they run relative to the host operating system.

| Type | Placement | Practical idea |
|---|---|---|
| Type 1 | Runs on bare metal | Direct access to hardware |
| Type 2 | Runs on top of a host OS | Depends on the host operating system |

Type 1 hypervisors run directly on the physical machine. Type 2 hypervisors run as software on an existing operating system. Both can run VMs, but the placement changes how they interact with the host.

- **Type 1** is closer to the hardware.
- **Type 2** depends on the host OS.
- Both manage VM resource allocation and isolation.

**Hardware and platform requirements**

Modern hypervisors can take advantage of CPUs built with virtualization support in mind. That support helps the hypervisor manage VMs more effectively.

Hyper-V on Windows and Windows Server has specific hardware requirements, and those requirements can exceed the general minimum requirements for the operating system because virtualization needs more computing resources.

> Hyper-V requirements can be higher than standard OS minimums because a virtualization environment needs additional resources.

- Check hardware support before planning a virtualization deployment.
- Expect virtualization to require more CPU, memory, and related resources than a non-virtualized system.
- Existing hardware may already be suitable if Hyper-V is already in use.

**How the hypervisor separates workloads**

The hypervisor keeps each VM separated from the others while still sharing the same physical host. That separation is what makes VMs feel independent even though they rely on common hardware.

This separation is important for stability and efficiency. If one VM has a problem, the hypervisor still maintains boundaries between that VM and the rest of the environment.

- Each VM gets controlled access to shared resources.
- The hypervisor maintains separation between workloads.
- Shared hardware use improves overall utilization.

**Study focus**

When you review this topic, focus on the relationship between the host, the hypervisor, and the VMs. Be able to explain what the hypervisor does, why virtualization is useful, and how Type 1 and Type 2 hypervisors differ.

1. Identify the hypervisor as the software layer that manages VMs.
2. Distinguish Type 1 from Type 2 by where each runs.
3. Remember that virtualization requires adequate CPU and other system resources.
4. Connect hypervisors to resource allocation, isolation, and efficient hardware use.
