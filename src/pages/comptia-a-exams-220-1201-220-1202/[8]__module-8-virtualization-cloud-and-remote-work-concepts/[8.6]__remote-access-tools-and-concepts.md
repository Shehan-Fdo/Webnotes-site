## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Remote access overview**

Remote access tools let you view, control, or share resources on another device across a network. In A+ contexts, you should recognize both the user experience and the security implications of each tool.

- **Screen sharing** lets you see and interact with a remote desktop.
- **Resource sharing** can expose applications or sessions on another system.
- **Network access** is the transport layer that makes these tools work across devices.

**Remote Desktop Protocol (RDP)**

Remote Desktop Protocol, or **RDP**, is one of the most common remote access technologies in Windows. It lets you connect to another desktop and interact with it from your own device.

RDP is useful when you need full desktop access or access to a single application on the remote system. It is widely available in Windows environments, and clients also exist for macOS, Linux, iPhone, and other operating systems.

- You use RDP to view and control a remote Windows desktop.
- You can connect to an entire desktop or a single application.
- Remote desktop services are available in many versions of Windows.
- RDP is a common choice for Windows administration and support tasks.

> RDP can take over the active session, so the remote user may see a blank screen while you control the system.

**Remote Assistance and support workflows**

Remote access tools are often used for troubleshooting and maintenance. Remote Assistance is part of that support workflow because it helps an IT professional access a system to diagnose or resolve issues.

These tools are valuable when you need to work on a remote machine without being physically present. They are especially useful in help desk and network management scenarios.

- Use remote access to troubleshoot systems from another location.
- Use support tools when you need to observe or control a user session.
- Choose the tool based on whether you need collaboration or full control.

**Telnet and SSH**

**Telnet** stands for Telecommunication Network. It provides a console-based view into another device over TCP. It is a legacy remote access method and is not commonly used in modern environments.

**SSH**, or Secure Shell, is another remote access technology commonly associated with secure command-line access. It is typically preferred over Telnet because it supports secure remote administration.

- Telnet provides text-based remote access.
- SSH provides secure command-line remote access.
- Both are useful to recognize as remote access concepts in exam questions.

**Security considerations**

Remote access expands convenience, but it also expands risk. You should treat remote sessions as sensitive administrative connections and use the most appropriate tool for the job.

- Prefer secure methods when available.
- Limit access to authorized users only.
- Be aware that remote control can expose active sessions and data on the screen.
- Match the tool to the task instead of using the most convenient option by default.

**Quick comparison**

| Tool | Typical use | Key idea |
|---|---|---|
| RDP | Remote desktop control | Full desktop or single application access |
| Remote Assistance | Troubleshooting and support | Helps an IT professional assist a user |
| Telnet | Legacy console access | Text-based access over TCP |
| SSH | Secure command-line access | Secure remote administration |

**Exam focus**

For A+, focus on what each tool does and when you would use it. Questions often test whether you can distinguish full desktop control from console access, and secure access from legacy access.

- Identify **RDP** as Windows remote desktop access.
- Identify **Telnet** as a console-based legacy protocol.
- Identify **SSH** as secure remote command-line access.
- Recognize remote access as a troubleshooting and management tool.
