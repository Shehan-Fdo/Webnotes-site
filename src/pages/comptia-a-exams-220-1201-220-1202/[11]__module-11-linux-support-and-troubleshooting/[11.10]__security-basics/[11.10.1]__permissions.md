## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Permissions overview**

Linux and macOS use file permissions to control who can interact with files and directories. The model is simpler than the very granular control found in NTFS, but it still gives you practical control over access.

The core idea is the same on both systems: permissions are evaluated against three identity buckets.

- **Owner**: the user who owns the file.
- **Group**: users who share the file’s group.
- **Everyone**: all other users.

This structure helps you reason about access quickly. If you know who owns the file and which group it belongs to, you can predict who can read, write, or execute it.

**Viewing permissions**

You typically inspect permissions from the command line. On Linux, `ls -l` shows the file mode in the first column along with the owner and group.

```bash
ls -l /path/to/directory
```

A typical listing includes entries such as:

```bash
drwxr-xr-x 2 archie archie 4096 Jul 5 21:03 Desktop
-rw-r--r-- 1 archie archie 3339 Jun 27 08:28 todo
-rwxr-xr-x 1 archie archie 2048 Jul 6 12:56 myscript.sh
```

- The first character indicates the item type, such as a directory or regular file.
- The remaining characters show permissions for owner, group, and everyone.
- The owner and group names appear after the mode string.

> Permissions protect against attacks launched from the booted system, but they do not replace data-at-rest encryption for physical theft scenarios.

**Understanding the mode string**

The mode string is the fastest way to read access at a glance. It is arranged in three sets of three characters after the leading type character.

- The first triplet applies to the **owner**.
- The second triplet applies to the **group**.
- The third triplet applies to **everyone**.

Common symbols include:

- `r` for read
- `w` for write
- `x` for execute
- `-` for no permission in that position

For example, `-rw-r-----.` means the owner can read and write, the group can read, and everyone else has no access in those positions.

**Changing permissions**

You change permissions with `chmod`. A common exam-style task is granting execute access to the group without changing existing read or write permissions.

```bash
chmod g+x maint.sh
```

That command adds execute permission for the group only. It preserves the current read and write settings.

When you work with permission changes, keep the scope narrow:

- Use `g` to target the group.
- Use `u` to target the owner.
- Use `o` to target others.
- Use `a` to target all categories.

**Practical interpretation**

Permissions are useful because they let you separate everyday access from administrative control. In a shared environment, you can allow a group to collaborate on a file while still limiting access for everyone else.

- Use ownership to identify the primary user responsible for the file.
- Use group membership to share access with a defined set of users.
- Use execute permission when a file must run as a script or program.

**Quick exam focus**

For A+ exam prep, focus on these takeaways:

- Linux and macOS permissions use the owner/group/everyone model.
- `ls -l` is the standard way to view file modes.
- `chmod` changes permissions.
- `g+x` adds execute permission for the group without altering other bits.
- Permissions are not the same as encryption.
