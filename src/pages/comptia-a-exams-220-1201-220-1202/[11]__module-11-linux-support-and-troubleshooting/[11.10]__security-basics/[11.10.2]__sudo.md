## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**What `sudo` does**

`sudo` stands for **superuser do**. You use it to run a command with administrative privileges when your normal user account does not have enough permission.

It is a common Linux command-line tool for system-level tasks. Instead of logging in directly as the `root` user, you temporarily elevate privileges for a single command.

- **Temporary elevation** lets you perform protected actions without staying logged in as `root`.
- **Controlled access** helps limit who can run privileged commands.
- **Auditing** is easier because privileged commands can be logged.

> `sudo` is preferred over direct `root` login for routine administration because it reduces unnecessary exposure to full administrative access.

**When you use it**

Use `sudo` when a command needs higher permissions than your current user has. Typical examples include reading protected locations or editing system files.

```bash
sudo ls /root
sudo nano /etc/hosts
```

- `sudo ls /root` attempts to list a directory that is normally restricted.
- `sudo nano /etc/hosts` opens a protected system file for editing.
- The command after `sudo` runs with elevated privileges only for that execution.

**How it fits into Linux administration**

Linux administration often happens from the command line through a terminal application such as `Terminal` or `XTerm`. Many Linux commands are also similar to those in macOS, so `sudo` is useful across Unix-like environments.

`sudo` is especially helpful when you need to complete a task quickly without switching accounts. That makes it practical for exam scenarios and everyday troubleshooting.

**Key points to remember**

- **`sudo`** grants temporary administrative rights.
- It helps you avoid logging in directly as `root`.
- It is used for commands that require elevated permissions.
- It supports safer, more controlled system administration.

**Exam focus**

For A+ purposes, know the purpose of `sudo` and recognize it as the command used to run administrative tasks from a normal user session. If a command fails with a permissions issue, `sudo` is often the first tool to consider.
