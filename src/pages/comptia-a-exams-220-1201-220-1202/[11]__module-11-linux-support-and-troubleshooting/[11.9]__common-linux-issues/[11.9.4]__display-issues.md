## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Display issue symptoms**

Display problems often show up as a black screen, a `no signal` message, fuzzy images, flashing screens, or a monitor that does not wake up. In practice, you start with the simplest checks first because many failures come from basic connection or power problems rather than the panel itself.

- **Black screen** can mean the monitor is not receiving video or power.
- **No signal** usually points to a disconnected, loose, or incorrect video path.
- **Fuzzy image** and **flashing screen** suggest a display quality or stability problem that needs further isolation.
- **Dead pixels** and **burn-in** are visible panel defects that affect image quality.

**First checks to make**

Before you assume a hardware failure, verify the monitor setup. A loose cable or missing power is common and easy to miss. Check both the video path and the monitor’s power source, then confirm the display is actually turned on.

1. Check that the video cable between the computer and monitor is fully seated.
2. Confirm the monitor has power and can turn on.
3. Verify the correct input source is selected on the display.
4. Test with another known-good cable or monitor if the issue remains.

> If the monitor is not connected properly, you can waste time troubleshooting the wrong component.

**Common display problems**

Some issues are tied to the display hardware itself, while others come from the signal source or adapter. Keep the symptom tied to the most likely cause so you can narrow the fault quickly.

- **Input source problems** happen when the monitor is set to the wrong input.
- **LCD projector bulb** issues can prevent proper projection output.
- **Burn-in** leaves a persistent image on the screen.
- **Dead pixels** appear as fixed dark or stuck points.
- **Fuzzy images** may indicate a poor connection or display configuration issue.

**Linux display and graphics troubleshooting**

On Linux, you can also use command-line tools for monitoring, management, and troubleshooting. The `ip` command helps you inspect and change network adapter settings, which can matter when remote display access or network-dependent workflows are involved.

```bash
ip address
ip route
```

- `ip address` shows interface-based IP configuration.
- `ip route` shows routing tables.
- `ip` can also enable or disable interfaces and update network settings.

For graphics-related problems on Linux, practical symptoms can include a black screen, failure to wake, login problems after suspend, or external monitor issues. Those symptoms often point to a graphics driver or display path problem rather than a simple cable fault.

- **External monitor** problems include no detection, glitches, or failure to work at all.
- **Suspend/resume** issues can prevent the system from returning to a usable display state.
- **Brightness controls** may stop responding when the graphics stack is not behaving normally.

**Troubleshooting approach**

Use a simple, repeatable process so you can isolate the fault without changing too many variables at once. Start with the monitor, then the cable, then the source device, and finally the graphics or driver layer if the basic checks pass.

1. Confirm the monitor has power.
2. Confirm the video cable is connected securely.
3. Confirm the correct input source is selected.
4. Test with alternate hardware if available.
5. Move to software or driver troubleshooting if the hardware path checks out.

**Key takeaways**

Display issues are often caused by basic connection problems, but the visible symptom helps you decide where to look next. A black screen or `no signal` message usually means you should verify power, cabling, and input selection first. If the hardware path is sound, then investigate the graphics stack, especially on systems showing wake, suspend, or external monitor problems.
