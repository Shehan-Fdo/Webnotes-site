## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Permission model basics**

Linux permissions use three access levels and three identities. The access levels are read (`r`), write (`w`), and execute (`x`). The identities are user/owner (`u`), group (`g`), and all others (`o`).

Permissions behave differently for files and directories:

- **Read** lets you view file contents or list directory contents.
- **Write** lets you modify a file or create and remove items in a directory.
- **Execute** runs a file such as a script, and it also lets you enter a directory with `cd`.

A permission problem usually comes down to one of two questions:

- Who needs access?
- What action do they need to perform?

**Working with `chmod`**

Use `chmod` to change standard Linux permissions. The command can use absolute or symbolic mode, but the general pattern is the same: you specify the access change and the target resource.

```bash
chmod {access-level} {resource}
```

For example, this command applies numeric permissions to a target path:

```bash
chmod 764 /opt/projects
```

Depending on system configuration, you may need `sudo` to change permissions.

**Symbolic and numeric changes**

Symbolic mode is useful when you want to adjust one permission without changing the rest. Numeric mode is useful when you want to set the full permission pattern at once.

- `chmod g+x maint.sh` adds execute permission for the group.
- `chmod g=x maint.sh` sets the group permissions to execute only.
- `chmod 640 maint.sh` sets a complete permission pattern in one step.

> Be careful when changing permissions on shared files or directories. A broad change can affect access for the owner, group, and everyone else.

**Troubleshooting permission errors**

When you see a permission error, start by identifying the correct user and the current owner of the file or directory. The user name is not always the most useful identity reference; the system tracks identity through user IDs as well.

A practical troubleshooting flow is:

1. Identify the user who needs access.
2. Check the current owner and permission pattern.
3. Decide whether the fix is a permission change, an ownership review, or both.
4. Apply the smallest change that restores the needed access.

This approach helps you avoid over-permissioning while still resolving the issue quickly.

**Access control lists**

Access control lists, or <abbr title="Access Control Lists">ACLs</abbr>, provide another way to manage access when standard permissions are not enough. They extend the basic owner-group-others model with more specific access rules.

Use ACLs when you need more granular control than `chmod` alone provides. Standard permissions remain the first thing to check, but ACLs are useful when multiple users need different access to the same resource.

**Exam focus**

For A+ exam prep, focus on these high-value ideas:

- Know the three permission types: read, write, execute.
- Know the three identities: user, group, others.
- Remember that execute means different things for files and directories.
- Use `chmod` for standard permission changes.
- Recognize when ACLs provide more specific access control.
