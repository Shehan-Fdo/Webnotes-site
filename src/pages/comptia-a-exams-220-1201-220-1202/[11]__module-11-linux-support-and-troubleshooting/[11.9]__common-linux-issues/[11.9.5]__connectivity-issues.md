## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Connectivity troubleshooting overview**

When you troubleshoot connectivity issues, start by confirming the device has a valid IP address configuration, then test basic reachability, then trace the network path if needed. This sequence helps you separate local configuration problems from broader network or routing issues.

Many administrators use command-line tools for these checks, but graphical tools can also show the same core information. On A+ exams, you should recognize the purpose of each tool and choose the one that matches the symptom.

- **IP configuration** tells you whether the device has an address, subnet information, and related network settings.
- **Basic connectivity** checks whether one host can reach another host.
- **Path tracing** shows the route to a remote destination.
- **Path quality** checks help you evaluate delay or loss along the route.

**Common commands and what they do**

The evidence highlights a small set of commands that appear often in troubleshooting questions. Windows and Linux use different commands for some tasks, but the goal is the same.

| Tool | Operating system | Function |
|---|---|---|
| `ipconfig` | Windows | Display IP address configuration |
| `ip addr` | Linux | Display IP address configuration |
| `ping` | Windows and Linux | Test connectivity |
| `tracert` | Windows | Trace the path to a remote destination |
| `traceroute` | Linux | Trace the path to a remote destination |
| `pathping` | Windows | Display path quality information |

Use `ipconfig` or `ip addr` first when the issue may involve addressing, interface status, or missing network settings. If the device has a valid configuration, move to `ping` to confirm that traffic can reach another system. If the connection works but the problem seems to be somewhere along the route, use `tracert` or `traceroute` to identify where the path goes.

> Check the physical layer first when the symptoms suggest a local network problem. Link lights and adapter status can quickly confirm whether the device is even connected.

**Linux-focused checks**

Linux troubleshooting often uses the `ip` command family to inspect and manage networking. The evidence specifically notes that `ip address` shows the current IP configuration, and `ip route` shows routing tables. These commands help you verify whether the interface is up and whether the system knows where to send traffic.

```bash
ip address
ip route
ping 8.8.8.8
traceroute example.com
```

- `ip address` shows addresses by interface.
- `ip route` shows routing information.
- `ping` confirms reachability to a host or IP address.
- `traceroute` helps you see the path to a remote destination.

**Troubleshooting flow**

1. Check the device’s IP configuration.
2. Confirm the physical connection if the device appears offline.
3. Use `ping` to test basic connectivity.
4. Use `tracert` or `traceroute` to inspect the path to the destination.
5. Use `pathping` on Windows when you need more detail about path quality.

This flow keeps you from jumping too quickly to advanced diagnostics. If the device cannot even reach a nearby host, the issue is usually closer to the endpoint than the internet path.

**What to remember for exam questions**

- Match the tool to the operating system.
- Use `ping` for simple reachability checks.
- Use `tracert` or `traceroute` for route inspection.
- Use `pathping` when you need path quality details on Windows.
- Start with IP configuration before assuming a routing problem.
