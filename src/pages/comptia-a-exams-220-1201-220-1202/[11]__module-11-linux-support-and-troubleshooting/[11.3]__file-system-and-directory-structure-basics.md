## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**File system fundamentals**

A file system defines how data is organized, stored, and retrieved on a system. In Linux, the directory layout follows the <abbr title="Filesystem Hierarchy Standard">FHS</abbr>, which describes where different types of files and directories belong. Linux also treats everything as a file, including directories and devices, so the structure is consistent across many system components.

The root directory is represented by `/` and serves as the starting point for the entire hierarchy. Even when files are stored on different physical or virtual devices, they still appear under this root-based tree.

- **Root directory**: `/` is the top of the Linux directory tree.
- **Hierarchy**: directories are arranged in a structured, nested layout.
- **FHS**: the standard that guides directory placement on Unix-like systems.
- **File model**: Linux treats files, directories, and devices as file-like objects.

**Why directory structure matters**

You use directory structure to find system files quickly, identify where user data lives, and understand what a path means. This is especially useful when working from the command line, where navigation depends on knowing the directory tree.

Linux command-line access commonly uses a terminal program such as `Terminal` or `XTerm`. The command line gives you direct control over the operating system, so knowing the layout helps you move through the system efficiently.

- **Navigation**: directory knowledge makes command-line work faster.
- **Support tasks**: you can locate configuration, user, and system areas more easily.
- **Portability**: similar conventions appear in other Unix-like systems such as BSD and macOS.

**Common Linux directory concepts**

The evidence emphasizes the root-based layout and the role of standard directories, but it does not require memorizing a full directory map here. Focus on the idea that each directory has a purpose and that Linux separates system structure from user-specific content.

<dl>
<dt><strong>System directories</strong></dt>
<dd>Store operating system components, binaries, and configuration-related content.</dd>
<dt><strong>User directories</strong></dt>
<dd>Store personal files and settings for individual accounts.</dd>
<dt><strong>Device representation</strong></dt>
<dd>Linux can represent hardware devices as files within the file system model.</dd>
</dl>

- Keep the purpose of each directory in mind rather than memorizing names without context.
- Use the root `/` as your reference point when reading paths.
- Expect Unix-like systems to share similar layout ideas, even if details vary.

**Working from the command line**

When you use Linux commands, you typically interact through a terminal. This is where directory structure becomes practical: you need to know where you are, where files live, and how paths are built from the root.

```bash
pwd
ls /
cd /home
```

- `pwd` shows your current location in the directory tree.
- `ls /` lists top-level directories under the root.
- `cd /home` changes into a common user-oriented location.

> Use conservative path assumptions during exam prep: if a directory’s exact role is not given, rely on the root-based hierarchy and the general purpose of the location.

**Exam focus**

For A+ exam prep, prioritize these ideas:

1. Identify `/` as the root directory.
2. Explain that Linux uses a hierarchical directory structure.
3. Recognize that the FHS guides directory organization.
4. Understand that Linux treats many system components as files.
5. Connect directory structure to terminal-based navigation.

**Quick recall**

- Linux directory structure is hierarchical.
- `/` is the root of the file system.
- FHS describes standard directory organization.
- Terminal access is common for command-line navigation.
- Unix-like systems often share similar directory conventions.
