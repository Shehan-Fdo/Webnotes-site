## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Overview**

Linux command-line tools are common in support work because they let you inspect, change, and troubleshoot a system quickly from a terminal. In A+ contexts, you should recognize the purpose of core commands and know when to use them for navigation, file management, permissions, process control, networking, and basic maintenance.

Many of these commands also appear in macOS because both environments share Unix-like roots. That makes the command set useful beyond Linux alone, but the exam focus is on practical support tasks rather than deep administration.

**Core navigation and file management**

You use basic commands to move around the file system and work with files efficiently. These are the first tools you reach for when you need to locate data, copy it, or remove it.

- `ls` lists directory contents.
- `pwd` shows the current working directory.
- `cd` changes directories.
- `mv` moves or renames files and directories.
- `cp` copies files and directories.
- `rm` removes files and directories.

A support technician often starts with `pwd` and `ls` to confirm location before making changes. That reduces mistakes when you are working in unfamiliar paths.

```bash
pwd
ls -l
cd /home/user/Documents
cp report.txt report-backup.txt
mv report-backup.txt /tmp/
rm oldfile.txt
```

**Searching and viewing information**

When you need to find text inside files or identify system details, use commands that filter output or display targeted information. `grep` is especially useful for searching for matching text in logs or configuration files.

- `grep` searches for patterns in text.
- `find` locates files and directories by name or other criteria.
- `man` opens the manual page for a command.
- `help` may provide built-in command assistance in some shells.

These tools help you narrow down large amounts of output instead of reading everything manually. In support work, that saves time when you are tracing errors or confirming configuration values.

**Permissions and ownership**

Linux permissions control who can read, write, or execute files. Support tasks often involve checking access problems or applying the correct ownership and mode.

- `chmod` changes file permissions.
- `chown` changes file ownership.
- `sudo` runs a command with elevated privileges.
- `passwd` changes a user password.

Use `sudo` carefully because it grants administrative rights for a single command or session action. If a user cannot access a file or service, permissions and ownership are common places to check first.

> Be cautious with `rm`, `chmod`, and `chown`; incorrect use can remove data or break access unexpectedly.

**Process and system control**

Support work often includes checking whether a program is running, stopping a stuck process, or restarting a system when required. Process commands help you identify what is active and take action on it.

- `ps` displays running processes.
- `kill` terminates a process by process ID.
- `shutdown` powers off or restarts a system, depending on options.

Use process tools when an application is unresponsive or when you need to confirm that a service-related task is still active. `kill` is a direct action, so verify the process ID before using it.

**Networking and connectivity**

Some support tasks require checking interface information or validating wireless and network settings. The evidence highlights commands used to identify network interface details and troubleshoot connectivity.

- `ifconfig` shows network interface configuration on many systems.
- `iwconfig` is used for wireless interface information.
- `ip address` can identify interface details on modern Linux systems.

`ip address` is a practical modern command to know because it reveals interface state and addressing information. In troubleshooting, you use these commands to confirm whether the system has the expected network configuration.

```bash
ip address
ifconfig
iwconfig
```

**Package and maintenance tasks**

Linux support work may also include installing tools or maintaining the system. The evidence specifically mentions `apt-get` as a package management command and `rsync` as a file synchronization tool.

- `apt-get` manages packages on Debian-based systems.
- `rsync` synchronizes files between devices.
- `dd` is used for low-level disk and image-related operations.

`rsync` is useful for backups and restores because it can synchronize data efficiently between systems. `dd` is powerful and should be used carefully because it works at a low level and can affect disks or images directly.

**Practical support workflow**

1. Identify the task you need to perform, such as locating a file, checking permissions, or verifying network status.
2. Use a discovery command first, such as `pwd`, `ls`, `ps`, or `ip address`.
3. Confirm the target before making changes.
4. Apply the needed action with a command such as `cp`, `chmod`, `kill`, or `sudo`.
5. Recheck the result to verify that the system now behaves as expected.

This workflow keeps you from making blind changes. In support scenarios, verification matters as much as the command itself.

**Exam focus**

For A+, you should be able to match each command to its support purpose:

- `ls`, `pwd`, `cd` for navigation.
- `cp`, `mv`, `rm` for file handling.
- `grep` and `find` for locating data.
- `chmod`, `chown`, `sudo`, `passwd` for access and administration.
- `ps`, `kill`, `shutdown` for process and system control.
- `ifconfig`, `iwconfig`, `ip address` for network checks.
- `apt-get`, `rsync`, `dd` for maintenance and recovery-related tasks.

The goal is recognition and practical use, not memorizing every possible switch. If you know what each command does, you can usually choose the right one in a support question.
