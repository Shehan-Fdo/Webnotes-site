## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Service and process awareness**

Linux systems often rely on background services and scheduled tasks to stay secure, updated, and maintainable. For A+ exam prep, focus on what services do, how processes behave, and why you should disable anything unnecessary.

- A **process** is a running program that uses system resources.
- A **daemon** is a background service that runs without direct user interaction.
- The `init` system or `systemd` commonly acts as the root process in modern Linux systems.
- Unneeded services can waste resources and increase security risk.
- Maintenance tasks often include backups, disk maintenance, and system updates.

**Processes and daemons**

A process represents an active instance of a program. In Linux, process management includes understanding how programs start, run, and stop. Daemons are long-running background processes that provide services such as scheduling, logging, or synchronization.

<dl>
<dt><abbr title="Process ID">PID</abbr></dt>
<dd>The unique identifier for a process.</dd>
<dt><abbr title="Parent Process ID">PPID</abbr></dt>
<dd>The identifier that links a process to its parent in the process tree.</dd>
</dl>

The process tree shows how child processes branch from parent processes. In modern Linux systems, `systemd` is commonly the first process and serves as the root of that tree.

- Use process awareness to monitor what is running on the system.
- Track parent-child relationships when you need to understand how a program started.
- Treat background services as part of system management, not just application behavior.

**Services, runlevels, and startup control**

Linux services are often tied to startup behavior and system state. Older Linux terminology uses runlevels, while modern systems commonly use `systemd`-based service management. The practical goal is the same: start what you need and stop what you do not.

> Stop unnecessary services when you want to improve security and reduce resource use.

- Disable services that do not support the system’s role.
- Keep only required services active during normal operation.
- Review startup behavior when troubleshooting performance or security issues.

**Scheduled maintenance tasks**

Linux provides multiple ways to automate maintenance. You can schedule tasks from a graphical interface or from the command line, and many distributions include built-in utilities for common jobs.

- `tar` is commonly used for backups and restores.
- `tar` stands for Tape Archive, but it works with many storage devices.
- `rsync` synchronizes data between devices.
- `rsync` can copy data immediately or on a schedule.

```bash
# Example backup concepts only
tar -cvf backup.tar /path/to/data
rsync -av /source/ /destination/
```

These tools support repeatable maintenance workflows. `tar` is useful when you want a straightforward archive, while `rsync` is useful when you want to keep two locations synchronized.

**Practical exam focus**

When you see a question about Linux service and process awareness, identify whether the task is about runtime behavior, startup control, or maintenance automation.

1. Determine whether the item is a process, a daemon, or a scheduled task.
2. Decide whether the goal is to improve security, save resources, or maintain data.
3. Match the tool or concept to the task, such as `tar` for archiving or `rsync` for synchronization.

**Key takeaways**

Service awareness helps you keep a Linux system secure and efficient. Process awareness helps you understand what is running and how it is related to other running components. Together, they support better troubleshooting, maintenance, and system control.
