## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**What SFC does**

System File Checker, or **SFC**, is a Windows utility that checks system files for corruption and helps fix problems that can cause Windows to malfunction. Use it when you suspect missing, damaged, or corrupted operating system files are affecting stability.

Microsoft describes symptoms that can point to this kind of issue, including:

- Windows features not working properly
- System errors
- Windows becoming unresponsive or freezing

SFC is part of the built-in repair tools available in Windows. In practice, it is a troubleshooting command you run from an elevated command prompt.

**When to use it**

Use SFC when Windows behavior suggests core file damage rather than a hardware failure. It is a software repair tool, so it focuses on protected system files and Windows components.

- It helps restore missing or damaged Windows files.
- It is useful after updates, crashes, or unexplained instability.
- It is commonly run from the command line with administrator privileges.

> Run the command prompt as an administrator before using SFC. Elevated permissions are required for system-level repair tasks.

**How to run SFC**

Microsoft’s documented workflow uses an elevated command prompt and, in some cases, DISM (Deployment Image Servicing and Management) before SFC.

1. Open Command Prompt as an administrator.
2. Run DISM to repair the Windows image if needed:
   ```bash
   DISM.exe /Online /Cleanup-image /Restorehealth
   ```
3. After DISM completes successfully, run SFC:
   ```bash
   sfc /scannow
   ```
4. Wait for verification to finish.
5. Type `exit` when the scan completes.

The `sfc /scannow` command performs a full scan and attempts to repair detected corruption. Microsoft notes that DISM may take a few minutes to start and complete before you run SFC.

**Key command details**

- `sfc` is the System File Checker command.
- `/scannow` tells SFC to scan all protected system files and repair problems it finds.
- `DISM.exe /Online /Cleanup-image /Restorehealth` prepares the Windows image so SFC can use healthy source files when repairs are needed.

**What to remember for the exam**

SFC is a Windows repair utility, not a general disk-checking tool. It focuses on system file integrity and is used when Windows itself is unstable or behaving incorrectly.

- **SFC** checks and repairs corrupted Windows system files.
- It is run from an elevated command prompt.
- `sfc /scannow` is the standard repair command.
- DISM may be used first to support the repair process.
- It is a common troubleshooting step for Windows malfunction symptoms.

**Quick comparison**

| Tool | Primary purpose | Typical use |
|---|---|---|
| `sfc` | Checks and repairs Windows system files | Fixes corruption in protected OS files |
| `DISM` | Repairs the Windows image | Prepares the component store for SFC repairs |

**Practical takeaway**

If Windows is freezing, throwing system errors, or losing built-in features, SFC is one of the first built-in tools to try. Run it with administrator rights, and use DISM first if the repair process needs a healthy Windows image source.
