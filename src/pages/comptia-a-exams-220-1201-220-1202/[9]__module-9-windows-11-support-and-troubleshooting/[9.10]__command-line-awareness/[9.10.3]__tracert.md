## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**What `tracert` does**

`tracert` is a Windows command-line utility that traces the path an IP packet takes to reach a destination. It helps you see each hop along the route, which makes it useful when a resource is unreachable or when you need to identify where communication stops.

The tool works by sending ICMP (Internet Control Message Protocol) echo packets with increasing Time-To-Live (TTL) values. Each router along the path decrements the TTL by at least 1. When the TTL reaches zero, the router returns an ICMP `Time Exceeded` message, which reveals that hop.

- Use `tracert` when you need to see the path to a host, not just whether the host responds.
- It helps you narrow down whether a failure occurs on the local network, at an intermediate router, or farther along the route.
- It is a standard troubleshooting tool on Windows systems.

**How to run it**

You run `tracert` from a command prompt and provide a destination as a host name or IP address. The destination can be a website name or another reachable network resource.

```cmd
tracert <destination>
tracert -d <destination>
```

The `-d` option skips name resolution, which can make the command respond faster. Use it when you want the route information without waiting for IP addresses to be translated into names.

1. Open a command prompt.
2. Type `tracert <destination>` or `tracert -d <destination>`.
3. Replace `<destination>` with the host name or IP address you want to trace.
4. Review the hop-by-hop output to see where the path changes or stops.

**How to interpret the output**

`tracert` shows each hop in order, so you can compare the route against the point where connectivity fails. If the trace stops early, the problem may be closer to your device or the first few routers. If the trace reaches several hops and then fails, the issue may be farther along the path.

- Each hop represents a device that forwarded the packet.
- The output helps you identify the path being taken to the destination.
- A trace can provide useful information about where communication is failing.

> `tracert` is most useful for path discovery and fault isolation, not for measuring application performance by itself.

**Practical troubleshooting use**

When a user cannot reach a resource, `tracert` gives you a quick view of the network path. That makes it a good companion to other command-line checks used in network troubleshooting, especially when you need to confirm whether the problem is local or somewhere upstream.

- Use it after confirming basic connectivity checks.
- Compare the trace from different systems if you suspect a routing or path-specific issue.
- Use the output to guide the next troubleshooting step rather than guessing.

**Key takeaways**

- `tracert` traces the route to a destination using ICMP and TTL behavior.
- It is a Windows command-line tool for diagnosing TCP/IP path problems.
- The `-d` option avoids name resolution and can speed up results.
- The output helps you locate where a network path fails.
