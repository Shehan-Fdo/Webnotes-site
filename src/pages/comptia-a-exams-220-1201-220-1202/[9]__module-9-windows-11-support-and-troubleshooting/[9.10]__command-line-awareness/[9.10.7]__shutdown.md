## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Overview**

The `shutdown` command is a Windows command-line tool used to shut down or restart local or remote computers. It is useful when you need more control than the graphical power menu provides.

In A+ exam prep, focus on what the command does, where it runs, and which switches change its behavior. The command prompt can be used for administration and troubleshooting, and `shutdown` is one of the practical utilities you should recognize.

- It can shut down or restart a computer.
- It can target a local or remote system.
- It supports switches that change timing, sign-out behavior, and boot flow.

**Core behavior**

Microsoft documents `shutdown` as a command that enables you to shut down or restart local or remote computers, one at a time. That makes it a general-purpose control command rather than a simple power-off shortcut.

The command is available from the Windows command line. In practice, you use it when you need repeatable control, scripted actions, or options that are not exposed in the normal user interface.

- **Local control** affects the current machine.
- **Remote control** can target another computer with the proper permissions.
- **One at a time** means it is not a bulk-management tool.

**Common switches**

The exact syntax includes several switches, but the most testable ones are the ones tied to everyday administration tasks.

```text
shutdown [/i | /l | /s | /sg | /r | /g | /a | /p | /h | /e | /o] [/hybrid] [/fw] [/f] [/m \\computer][/t xxx][/d [p|u:]xx:yy [/c "comment"]]
```

- `/i` opens the Remote Shutdown dialog box.
- `/l` signs out the current user immediately.
- `/s` performs a full shutdown.
- `/r` performs a full shutdown and restart.
- `/h` hibernates the local computer.
- `/a` aborts a pending shutdown.
- `/m \\computer` targets a remote computer.
- `/t xxx` sets a timeout in seconds.
- `/c "comment"` adds a message for the shutdown event.

> `/l` works independently and cannot be combined with other parameters.

**Practical exam points**

The command is often tested as part of Windows command-line knowledge. You should be able to match a switch to its effect without needing to memorize every possible parameter.

- `shutdown /s` means full shutdown.
- `shutdown /r` means restart after shutdown.
- `shutdown /h` means hibernate.
- `shutdown /i` gives you a graphical remote-shutdown interface.
- `shutdown /a` is the cancel option when a shutdown is pending.

**When to use it**

Use `shutdown` when you need a controlled power action from the command line. It is especially helpful when you want to automate a shutdown, restart a system remotely, or trigger a specific boot-related behavior.

It is also useful when the normal Windows interface does not provide the exact option you need. Command-line tools can expose more precise control than menus.

**Quick reference**

| Switch | Result |
|---|---|
| `/s` | Shut down the computer |
| `/r` | Shut down and restart |
| `/h` | Hibernate the local computer |
| `/l` | Sign out the current user |
| `/a` | Abort a pending shutdown |
| `/i` | Open the Remote Shutdown box |

**Study reminder**

If you see a question about Windows command-line tools, look for the action being requested:
- power off
- restart
- hibernate
- sign out
- remote shutdown
- cancel a scheduled shutdown

Matching the switch to the action is usually the fastest path to the correct answer.
