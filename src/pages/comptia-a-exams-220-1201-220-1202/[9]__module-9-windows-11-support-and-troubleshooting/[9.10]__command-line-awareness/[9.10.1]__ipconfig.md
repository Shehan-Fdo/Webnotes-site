## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**What `ipconfig` does**

`ipconfig` is a Windows command-line tool for viewing current TCP/IP network configuration values. It is useful when you need to confirm that a device has a valid address, subnet mask, and default gateway, or when you want to inspect adapter-level network details during troubleshooting.

It also helps you check whether the system received its configuration from <abbr title="Dynamic Host Configuration Protocol">DHCP</abbr> and whether <abbr title="Domain Name System">DNS</abbr> settings are present. Because it runs from the command line, it is often faster than using the graphical interface when you need a quick answer.

- Use it to display IPv4 and IPv6 configuration.
- Use it to verify the hostname, subnet mask, and default gateway.
- Use it to identify the DNS server and DHCP server in use.

**Why it matters in troubleshooting**

When a system cannot be reached across the network, `ipconfig` gives you a fast way to inspect the local configuration. If the address shown is `169.254.x.x`, that is an <abbr title="Automatic Private IP Addressing">APIPA</abbr> or link-local address. This usually means the device did not successfully lease an address from a DHCP server.

That clue changes your next step. Instead of guessing, you can focus on DHCP connectivity, adapter status, or other causes of failed address assignment.

> A `169.254.x.x` address is a warning sign that the device likely did not obtain a DHCP lease.

**Common output to look for**

`ipconfig` without parameters shows the core settings you usually need first. The full output with `ipconfig /all` provides more detail for deeper troubleshooting.

| Command | What you get |
|---|---|
| `ipconfig` | Basic IP information for active adapters |
| `ipconfig /all` | Full TCP/IP configuration for all adapters |
| `ipconfig /release` | Releases a DHCP lease |
| `ipconfig /renew` | Requests a new DHCP lease |
| `ipconfig /flushdns` | Clears the DNS cache |
| `ipconfig /displaydns` | Shows cached DNS entries |
| `ipconfig /registerdns` | Updates DNS records |

**Useful switches and what they help with**

`/all` is the most common switch when you need more than the basic address summary. It exposes details such as the MAC address, DHCP server, DNS server, and lease information.

Other switches support common recovery tasks:

- `/release` and `/renew` help refresh DHCP-assigned settings.
- `/release6` and `/renew6` manage IPv6 address leases.
- `/flushdns` clears cached name-resolution data when records are stale or incorrect.
- `/displaydns` lets you inspect what the system has cached.
- `/registerdns` forces the system to update its DNS records.

**How to use it**

1. Open a command prompt or terminal with access to Windows networking commands.
2. Run `ipconfig` to check the current address, subnet mask, and default gateway.
3. Run `ipconfig /all` if you need adapter details, DHCP information, or DNS server data.
4. If the address looks wrong or missing, use `ipconfig /release` followed by `ipconfig /renew`.
5. If name resolution seems stale, use `ipconfig /flushdns`.

```powershell
ipconfig
ipconfig /all
ipconfig /release
ipconfig /renew
ipconfig /flushdns
```

**What to remember for the exam**

- `ipconfig` is a Windows network troubleshooting command.
- It displays current TCP/IP configuration values.
- It helps you verify IPv4 and IPv6 settings.
- `ipconfig /all` gives deeper adapter and server details.
- `169.254.x.x` usually indicates a failed DHCP lease and an APIPA address.

**Practical takeaway**

Use `ipconfig` early in network troubleshooting because it quickly tells you what the system believes its network settings are. That makes it easier to decide whether the problem is with addressing, DHCP, DNS, or basic adapter configuration.
