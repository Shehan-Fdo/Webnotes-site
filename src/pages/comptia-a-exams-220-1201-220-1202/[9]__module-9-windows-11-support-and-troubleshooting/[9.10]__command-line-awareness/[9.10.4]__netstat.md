## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**What `netstat` does**

`netstat` is a Windows command-line utility for checking network activity on a local system. It helps you see what the computer is doing on the network right now, which makes it useful when you troubleshoot slow connectivity, application issues, or unexpected network behavior.

It can display:

- **Active TCP connections**
- **Listening ports**
- **Ethernet statistics**
- **IP routing table**
- **IPv4 and IPv6 statistics**

Used without parameters, `netstat` shows active TCP connections.

> `netstat` is available only if the Internet Protocol (TCP/IP) protocol is installed as a component in the network adapter properties.

**Why you use it**

When a system is not behaving as expected, `netstat` gives you a quick view of connection state and port usage. That makes it helpful for confirming whether a service is listening, whether a connection is established, or whether there is unexpected network activity on the machine.

Common troubleshooting uses include:

- Checking whether a local service is listening on the expected port
- Reviewing active connections for a specific application
- Looking for unusual or unexpected network sessions
- Verifying basic network activity during incident response or malware investigation

**Basic syntax**

```powershell
netstat [-a] [-b] [-e] [-n] [-o] [-p <Protocol>] [-r] [-s] [<interval>]
```

**Common options**

- `-a` shows all connections and listening ports.
- `-n` displays addresses and ports in numeric form.
- `-o` shows the owning process ID.
- `-p <Protocol>` filters by protocol.
- `-r` displays the routing table.
- `-s` shows protocol statistics.

**How to read the output**

`netstat` output typically includes the local address, foreign address, and connection state. That structure helps you identify where traffic is coming from, where it is going, and whether the session is established or waiting.

Useful fields to watch:

- **Local address**: the local IP address and port
- **Foreign address**: the remote IP address and port
- **State**: the current connection state

**Practical troubleshooting approach**

1. Open Command Prompt.
2. Run `netstat` to view active TCP connections.
3. Add options such as `-a`, `-n`, or `-o` when you need more detail.
4. Compare the results with the service or application you are investigating.

**Exam focus**

For A+ purposes, remember that `netstat` is a built-in Windows network troubleshooting command. It is most associated with viewing active connections, listening ports, and network statistics rather than changing network settings.
