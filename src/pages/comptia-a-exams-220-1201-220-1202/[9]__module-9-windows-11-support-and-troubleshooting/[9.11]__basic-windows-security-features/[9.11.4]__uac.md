## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**UAC overview**

User Account Control, or **UAC** (User Account Control), is a Windows security feature that helps protect the operating system from unauthorized changes. It does this by limiting how applications and processes run and by prompting you when an action needs administrator-level permission.

UAC is part of Windows security and is designed to reduce the risk of malware gaining elevated privileges. In practical terms, it helps you decide whether a change should be allowed before the system applies it.

- UAC prompts appear when an app tries to make a system-level change.
- The prompt gives you the choice to approve or deny the action.
- UAC helps prevent malicious code from running with administrator privileges.

**How UAC works**

Windows uses UAC to keep applications running with the least permissions needed. When a process needs more access, it requests elevation. Microsoft notes that each application requiring the administrator access token must prompt for consent, with a limited exception for parent and child processes that share the same integrity level.

This means UAC is not just a warning dialog; it is a control point for privilege elevation. If the action affects system settings, software installation, or other protected areas, Windows asks for approval before continuing.

- Applications run with limited permissions by default.
- Elevated access requires user consent.
- Parent and child processes can share an access token when they meet the same integrity level requirement.

**Secure Desktop prompt**

Some UAC prompts appear on the **Secure Desktop**, which dims the rest of the screen and isolates the prompt from normal desktop interaction. This is meant to reduce interference while you decide whether to allow the change.

A legacy application may trigger a Secure Desktop prompt when it tries to access protected resources such as the registry. If the application does not handle the prompt well, it may stop working correctly while the UAC message is displayed.

> If an application breaks when UAC prompts appear, the issue is often with how the app handles elevation, not with Windows itself.

**Adjusting UAC settings**

You can modify UAC behavior for an individual user through Control Panel. The evidence points to the **User Account Control** settings area rather than general user accounts or administrative tools.

A practical exam takeaway is that UAC settings are user-specific and are adjusted from the UAC control panel entry.

1. Open `Control Panel`.
2. Select `User Account Control`.
3. Adjust the UAC configuration for the user as needed.

**What to remember for the exam**

- UAC protects Windows from unauthorized changes.
- It prompts for approval when administrator permission is needed.
- It helps block malware from silently elevating privileges.
- UAC settings can be modified from `Control Panel / User Account Control`.
- Legacy apps may have trouble when UAC prompts interrupt their workflow.

**Quick terminology**

<dl>
<dt>Elevation</dt>
<dd>The process of granting a program higher permissions, usually administrator-level access.</dd>
<dt>Integrity level</dt>
<dd>A Windows security boundary used to help control how processes interact and what they can access.</dd>
<dt>Secure Desktop</dt>
<dd>A protected desktop used for sensitive prompts such as UAC approval dialogs.</dd>
</dl>
