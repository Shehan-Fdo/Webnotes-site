## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Firewall overview**

A firewall is a security feature that helps protect a device by filtering network traffic that enters and exits it. In Windows, the firewall is host-based, which means it runs on the individual computer rather than only at the network edge.

Windows Firewall can filter traffic using several criteria:

- source and destination IP address
- IP protocol
- source and destination port number
- installed services and applications

This lets you restrict communication to only the apps and services that are explicitly allowed. In practice, that makes the firewall useful for both security and troubleshooting.

**Host firewall vs. network firewall**

A network firewall sits at the edge of the network and can inspect traffic for many devices. A host firewall sits on a single system and can make decisions based on what that specific computer is trying to send or receive.

That host-level visibility is important because it can control traffic per device and per application. For A+ purposes, remember that Windows Firewall is built into the operating system and is designed to stay on by default.

- **Network firewall**: protects traffic for a network segment or perimeter
- **Host firewall**: protects one computer directly
- **Application-aware control**: can allow or block specific programs

**Windows Security and Control Panel management**

You can manage Windows Firewall through the Windows Security app or through Control Panel. When you change firewall state or certain settings, you need elevated permissions.

The Windows Security app includes the **Firewall and network protection** area. Control Panel also provides access to turning Windows Defender Firewall on or off and adjusting related settings.

> You need administrator permissions to enable or disable the firewall.

**Network profiles**

Windows uses separate firewall profiles for different network types. The common profiles are:

- **Domain**
- **Private**
- **Public**

These profiles let Windows apply different rules depending on where the device is connected. That matters because a trusted office network and an untrusted public network should not use the same level of exposure.

A practical exam takeaway is that firewall behavior can vary by profile, so troubleshooting should include checking which profile is active.

**Common configuration options**

Windows Firewall includes options that change how strict the filtering is. One notable setting is the ability to block all incoming connections, including those that are normally allowed.

Other common actions include:

- turning the firewall on or off
- changing notification settings
- configuring exceptions for allowed applications
- adjusting settings per network profile

These options are often used during troubleshooting, but disabling protection should be temporary and intentional.

**Troubleshooting mindset**

Firewall settings can prevent unwanted connections, but they can also block legitimate traffic. If an application cannot connect, the firewall is one of the first places to check.

Use a careful approach:

1. Confirm which network profile is active.
2. Check whether the firewall is enabled.
3. Review allowed apps or exceptions.
4. Look for stricter settings such as blocking all incoming connections.
5. Restore protection after testing is complete.

**Key exam points**

- Windows Firewall is a built-in security feature.
- It is a **host-based firewall**.
- It filters inbound and outbound traffic.
- It can use IP address, protocol, and port-based rules.
- It supports separate profiles for domain, private, and public networks.
- Administrative rights are required for on/off changes.
