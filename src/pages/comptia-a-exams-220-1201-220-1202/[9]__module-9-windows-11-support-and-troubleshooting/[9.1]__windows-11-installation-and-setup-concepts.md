## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Installation choices**

Windows 11 setup starts with choosing the right installation method for the device and the outcome you want. The evidence highlights two primary approaches: an **in-place upgrade** and a **clean install**.

An in-place upgrade installs Windows 11 over an existing operating system. This is the practical choice when you want to keep the current environment and move forward with minimal disruption. A clean install replaces the existing setup and is better when you want a fresh start or need to rebuild the system from scratch.

- **In-place upgrade** keeps the existing OS in place during the transition.
- **Clean install** removes the old setup and installs Windows 11 anew.
- Your choice depends on the current system state and the goal of the deployment.

**System requirements and compatibility**

Before you install, verify that the device meets the minimum Windows 11 requirements. Microsoft states that these requirements are designed around **reliability**, **security**, and **compatibility**. If the device does not meet them, installation may not be possible, and a new PC may be the practical option.

Microsoft also notes that if you are upgrading, the device must be running Windows 10, version 2004 or later. If you are unsure about compatibility, you can check with the PC original equipment manufacturer (**OEM**) or use the **PC Health Check** app on a Windows 10 system.

- Minimum requirements help ensure a stable user experience.
- The PC Health Check app helps assess compatibility on Windows 10 devices.
- Feature-specific requirements can exceed the baseline system requirements.
- Some devices may need hardware or settings changes before they can upgrade.

> If the device does not meet the minimum requirements, do not assume the upgrade will succeed. Verify compatibility first and plan for alternatives if needed.

**Preparing for setup**

Preparation reduces installation problems and helps you choose the correct path. The evidence emphasizes planning the installation method, preparing the boot method, setting up partitions, and verifying the system after installation.

You should also understand that Windows 11 installation can involve different tools and checks depending on whether you are upgrading or performing a clean install. Careful preparation helps avoid interruptions and makes the setup process smoother.

- Confirm the installation method before you begin.
- Prepare the boot method for the selected installation path.
- Set up partitions as needed for the target system.
- Verify the installation after setup completes.

**Windows 11 editions and deployment context**

Windows 11 is part of the modern Windows landscape, where many desktops and portable computers still run Windows 10 or Windows 11. Microsoft support for Windows 10 ends in October 2025, which makes upgrade planning relevant in real environments.

This context matters because installation decisions are not only technical; they also affect supportability and lifecycle planning. When you prepare for Windows 11, you are often balancing compatibility, user impact, and the long-term operating system strategy.

- Windows 10 remains common in organizations, even as Windows 11 adoption grows.
- Support timelines influence upgrade planning.
- Installation decisions should align with the device’s role and future use.

**Practical setup workflow**

A simple workflow helps you stay organized during installation and setup. Use it as a checklist rather than a rigid script.

1. Confirm whether you need an in-place upgrade or a clean install.
2. Check that the device meets Windows 11 minimum requirements.
3. Verify upgrade eligibility, including the Windows 10 version if applicable.
4. Prepare the boot media or installation method.
5. Complete the installation and review the system afterward.

**Key takeaways**

Windows 11 installation is mainly about choosing the right method, confirming compatibility, and preparing the device correctly. The most important exam-ready ideas are the difference between upgrade and clean install, the role of minimum requirements, and the need to verify the system before and after setup.
