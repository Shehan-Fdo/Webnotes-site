## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**What performance degradation means**

Performance degradation is the general experience of a system becoming slower or less responsive over time. In Windows, the slowdown can come from active processes, background services, software behavior, or hardware limits. The practical goal is to identify where the bottleneck occurs before you change settings or replace components.

A good first response is to observe the system while the slowdown is happening. That gives you real-time evidence instead of guessing based on symptoms alone.

- Check whether the slowdown affects the whole system or only one app.
- Look for high resource use before making changes.
- Start with simple actions such as restarting if the system has not been rebooted recently.

**Initial checks**

Begin with the most visible and least disruptive causes. Task Manager is the main starting point because it shows current CPU, memory, disk, and network activity. It also helps you identify unresponsive applications and processes that may be consuming resources.

If the system has been running for a long time, background programs and startup items can contribute to slow performance. Security tools can also create temporary slowdowns while scanning.

- Use **Task Manager** to view active apps and services.
- Sort by CPU, memory, disk, or network usage to find heavy consumers.
- End unresponsive processes only when they are clearly causing the issue.
- Review startup programs and background tasks if the slowdown is persistent.

> If the system is slow after a long uptime, a restart can clear temporary resource pressure and is worth trying early.

**Using Task Manager**

Task Manager gives you a live view of system activity. It helps you narrow the problem by showing which process or application is using the most resources at the moment the slowdown occurs. The Performance tab adds a graphical view, which is useful when you want to see how usage changes over time.

This makes Task Manager useful for both quick checks and deeper observation. You can confirm whether the issue is tied to CPU load, memory pressure, disk activity, or network traffic.

- Use the **Processes** view to identify resource-heavy apps.
- Use the **Performance** tab to review trends in CPU, memory, disk, and network usage.
- Compare what the user reports with what the system is actually doing.

**Performance Monitor**

Performance Monitor is a built-in Windows tool for collecting performance data over time. It uses counters to record system behavior, which helps you narrow down the bottleneck that makes the computer slow. This is more useful than a quick glance when the issue is intermittent or hard to reproduce.

Use it when you need evidence beyond a momentary snapshot. It helps you track patterns and confirm whether the slowdown is tied to a specific resource.

- Use **Performance Monitor** to record performance data.
- Focus on counters related to CPU and memory usage.
- Use collected data to narrow the bottleneck instead of relying on assumptions.

**Common causes to consider**

Slow performance is often caused by a mix of software activity and system limitations. Evidence points to background programs, startup items, security scans, and hardware constraints such as low memory or slower storage.

Power state can also matter on some devices. If a laptop is not connected to AC power, performance may be reduced.

- **Background programs** can consume CPU and memory in the background.
- **Startup programs** can slow the system as soon as it boots.
- **Hardware limits** such as insufficient memory or HDD storage can reduce responsiveness.
- **Power settings** may affect performance on portable devices.

**Practical troubleshooting flow**

1. Confirm the slowdown and note whether it affects the whole system or one application.
2. Open Task Manager and check CPU, memory, disk, and network usage.
3. Sort by resource usage to identify the most active process or service.
4. Restart the system if it has not been rebooted recently.
5. Review startup items, background programs, and security scans.
6. Use Performance Monitor if you need longer-term data to isolate the bottleneck.
7. Check whether hardware limits or power conditions may be contributing.

**Exam focus**

For A+ troubleshooting, the key skill is matching the symptom to the right tool. Task Manager is best for immediate visibility, while Performance Monitor is better for collecting data over time. You should also remember that not every slowdown is caused by malware or a failing component; many issues come from ordinary resource contention or configuration.

Keep your approach structured:
- Observe first.
- Measure resource use.
- Change one likely cause at a time.
- Verify whether the system returns to normal.
