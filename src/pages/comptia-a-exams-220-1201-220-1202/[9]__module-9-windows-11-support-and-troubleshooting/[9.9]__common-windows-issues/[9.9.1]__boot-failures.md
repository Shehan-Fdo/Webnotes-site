## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Boot failure overview**

A boot failure happens when the system cannot complete the startup path and reach Windows or another operating system. The boot process begins with basic hardware checks, then moves through firmware-controlled loading steps before the operating system starts.

The key idea for troubleshooting is to identify *where* startup stops. That tells you whether the issue is likely in firmware, the boot loader, the operating system loader, or Windows itself.

- **POST** (Power-On Self-Test) checks basic hardware first.
- The **bootstrap loader** in BIOS (Basic Input/Output System) starts the next stage.
- A second-stage boot loader may be involved, such as `Winload`, `GRUB Legacy`, `GRUB 2`, or `LILO`.
- Windows startup issues can be separated into phases such as `PreBoot`, `Boot Manager`, `OS Loader`, and `Kernel`.

**Boot process stages**

The boot sequence is multi-step, so a failure at one stage does not mean every stage is broken. You should use the symptom to narrow the problem instead of treating all boot issues the same.

<dl>
<dt>POST</dt>
<dd>The system verifies core hardware before trying to load an operating system.</dd>
<dt>Bootstrap loader</dt>
<dd>Firmware code that starts the next program in the boot chain.</dd>
<dt>Boot loader</dt>
<dd>The component that begins loading the operating system.</dd>
<dt>OS loader</dt>
<dd>The stage that loads Windows or another operating system into memory.</dd>
</dl>

- If the system fails before Windows starts, focus on the boot chain rather than application-level troubleshooting.
- If Windows begins loading but does not reach the desktop, the issue is later in the startup path.
- If Windows stops with a blue screen, the failure may involve hardware, drivers, or an application-related condition.

**Windows boot failure symptoms**

Windows boot problems often show up as a system that cannot find Windows, cannot continue loading, or stops with an error before the desktop appears. A blue screen of death (BSOD) is a common stop condition when Windows determines it cannot safely continue.

> A stop error means Windows has halted startup because it detected a condition that prevents the operating system from continuing.

- A boot failure can be caused by bad hardware.
- A boot failure can be caused by bad drivers.
- A boot failure can be caused by damaged system files or incorrect boot configuration.
- A boot failure can also be tied to disk errors.

**Troubleshooting approach**

You troubleshoot boot failures by matching the symptom to the startup phase and then using the appropriate recovery path. Microsoft describes startup troubleshooting as a phase-based process, which helps you isolate the affected area.

1. Identify the point of failure during startup.
2. Decide whether the issue is in `PreBoot`, `Boot Manager`, `OS Loader`, or `Kernel`.
3. Use the available startup or recovery tools for that phase.
4. Re-test the boot path after each change.

- Startup troubleshooting is most effective when you work from the earliest failure point forward.
- Windows startup settings can help you troubleshoot, optimize startup, or change startup behavior.
- Advanced boot options can provide specialized boot modes for diagnosis and repair.

**Startup settings and recovery options**

Windows startup settings are useful when you need controlled access to the operating system for troubleshooting. These options support diagnosis without requiring a full normal boot.

- **Startup Repair** attempts to fix common startup problems.
- **Startup Settings** provides specialized boot modes for troubleshooting.
- **Debugging mode** supports advanced troubleshooting with tools such as `WinDbg`.
- Recovery options are useful when Windows cannot start normally.

```text
Typical troubleshooting focus:
- damaged system files
- incorrect boot configuration
- disk errors
- driver issues
```

**Practical exam takeaways**

For exam prep, remember that boot failures are about the startup chain, not just “Windows won’t load.” The most useful skill is recognizing which stage failed and choosing the right recovery path.

- Use the boot phase to guide your next step.
- Treat BSODs as a sign that Windows started but encountered a fatal stop condition.
- Keep the distinction between firmware, boot loader, and Windows startup clear.
- Use startup tools when normal booting is not possible.
