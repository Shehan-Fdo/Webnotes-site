## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**User accounts and permissions**

Windows uses user authentication to assign rights and permissions. When you sign in, you use a username and password to access the operating system with the permissions tied to that account.

- **Administrator** accounts have the most control in Windows.
- **Guest** accounts provide very limited access and are commonly disabled by default.
- **Regular user** accounts have assigned rights without full administrative control.

A user account is not just a login name. It defines what you can do in the system, which is why account type matters during setup and daily support.

**Local users, groups, and access**

Windows combines user accounts with groups to help define access. Managing local users and groups is part of controlling rights and permissions on a standalone system.

- **Local users** are accounts stored on the computer itself.
- **Groups** collect users so you can apply permissions more efficiently.
- Account choice affects how the system behaves in home, small office, and enterprise environments.

When you are setting up Windows, account planning matters. In some environments, you may need a specific domain account rather than a local account, especially when connecting to centralized services such as Active Directory.

**Workgroups, domains, and account context**

A workgroup is a simpler setup, while a domain supports centralized account management. The account type you use should match the network environment.

- **Workgroup** setups are common in smaller or simpler environments.
- **Domain** environments use centralized authentication and account control.
- A domain account may be required when a user connects to domain-managed resources.

Keep the environment in mind before you create or assign accounts. The wrong account type can create access problems later.

**User profiles**

A user profile stores the user-specific settings and files associated with an account. If a profile becomes corrupted, Windows support guidance recommends creating a new user account and then copying files from the old profile.

- Create a new account and configure it as an administrator.
- Sign in with the new account before copying data.
- Copy files from the old profile folder under `C:\Users\<Old_Username>`.

> Do not change File Explorer to show hidden files or hidden protected operating system files when following the recovery steps.

**Recovering a corrupted profile**

If a profile is damaged, the practical recovery path is to move the user to a new profile and preserve their files.

1. Create a new user account.
2. Make the new account an administrator.
3. Sign in with the new account.
4. Open File Explorer and go to `This PC`.
5. Open the Windows drive, then the `Users` folder.
6. Locate `C:\Users\<Old_Username>`.
7. Copy the needed files and folders to the new profile.

This approach restores access to user data without trying to repair the damaged profile in place.

**Key exam takeaways**

- User accounts control access through assigned rights and permissions.
- Administrator, guest, and regular user accounts serve different access levels.
- Groups help organize permissions across multiple users.
- Workgroup and domain environments affect how accounts are managed.
- A corrupted profile is often handled by creating a new account and copying data from the old profile.
