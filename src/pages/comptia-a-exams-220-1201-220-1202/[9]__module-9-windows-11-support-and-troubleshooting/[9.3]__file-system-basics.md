## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**File systems and formatting**

A file system defines how an operating system stores, names, and retrieves data on a partition or volume. Before you can store files on a newly created partition, you format it with a file system. Different operating systems often support multiple file systems, and the best choice can depend on the OS, the media, and the intended use.

- **Partition**: a storage area that must be formatted before use.
- **File system**: the structure that organizes files and folders on that storage.
- Windows commonly uses `NTFS` or `FAT32`.
- Linux and macOS may use file systems optimized for those platforms.
- Operating systems can often recognize more than one file system type.

**Common file systems**

The evidence highlights several file systems you should recognize for A+ study. `FAT32` is part of the older FAT family, while `NTFS` is the default file system for modern Windows-based operating systems. Linux commonly uses `ext3` and `ext4`, and Apple systems use `APFS`.

- `FAT` stands for **File Allocation Table**.
- `FAT32` is the most recent version mentioned in the FAT family.
- `NTFS` stands for **New Technology File System**.
- `ext3` and `ext4` are common Linux file systems.
- `APFS` is used on Apple systems.

> `NTFS` is associated with modern Windows systems and adds features such as security descriptors, encryption, disk quotas, and rich metadata support.

**Windows file structures and paths**

Windows uses a specific naming convention for drives and paths. Drives are referenced by a letter from `A` to `Z`, followed by a colon. The main storage device is usually `C:`. Paths use backslashes to separate folders.

- Drive letters are written as `A:` through `Z:`.
- `C:` is typically the primary drive.
- Optical drives may also appear as drive letters.
- Backslashes separate folders in Windows paths.
- Windows file and folder references follow a consistent structure.

**Practical path examples**

Use the drive letter first, then the path to the file or folder. This makes it easier to identify where data lives on the system.

```text
C:\Users\Public\Documents
D:\Installers\setup.exe
```

- `C:\Users\Public\Documents` shows a folder path on the `C:` drive.
- `D:\Installers\setup.exe` shows a file stored on another drive.
- The colon identifies the drive.
- The backslash separates each directory level.

**NTFS characteristics to remember**

`NTFS` is the most important Windows file system in the evidence set. It is designed for reliability and advanced management on modern systems. It also uses metadata and supports recovery-oriented features.

- `NTFS` is the default file system for modern Windows-based operating systems.
- It supports security descriptors.
- It supports encryption.
- It supports disk quotas.
- It uses rich metadata.
- It improves reliability with transaction-based logging and checkpoint information.

**Study focus**

For exam prep, focus on matching the file system to the platform and recognizing how Windows references storage locations. You do not need to memorize every implementation detail, but you should know the common names and their typical use cases.

- Match `NTFS` with modern Windows.
- Match `FAT32` with the FAT family.
- Match `ext3` and `ext4` with Linux.
- Match `APFS` with Apple systems.
- Read Windows paths from drive letter to folder hierarchy.
