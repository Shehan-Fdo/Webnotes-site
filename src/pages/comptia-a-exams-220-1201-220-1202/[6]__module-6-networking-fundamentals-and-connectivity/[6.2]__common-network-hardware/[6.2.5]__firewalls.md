## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Firewall fundamentals**

A firewall controls network traffic by allowing or blocking connections based on defined criteria. In practice, you use it to reduce unauthorized access and limit risky traffic between networks or devices.

- A firewall can filter by **IP address**, port number, time of day, or website keywords.
- The most common placement is at the edge of the network, often in a SOHO (small office/home office) router.
- Firewalls protect both sides of the connection: your network from the internet and the internet from unwanted traffic originating from your systems.

**Windows Defender Firewall**

Windows includes an integrated firewall called Windows Defender Firewall. It should stay enabled by default to help prevent unauthorized access.

- You can change firewall settings when troubleshooting, but you need elevated permissions.
- Windows uses different security postures for **public** and **private** networks.
- Public networks should use stricter protection than private networks because they are less trusted.
- You can create customized exception rules when a specific app or service needs access.

> Disable or relax firewall settings only when you have a clear troubleshooting need and the required permissions.

**Firewall behavior and rule concepts**

A firewall is often described as a verb because it is about the act of blocking traffic according to policy. That policy can be simple or highly granular depending on the platform.

- Rules can block specific incoming ports.
- Rules can block outbound traffic as well as inbound traffic.
- Some firewall products support rules imported from common open source formats.
- Managed services may scale automatically with traffic and reduce the need to deploy your own infrastructure.

**AWS Network Firewall overview**

AWS Network Firewall is a managed, stateful network firewall and intrusion detection and prevention service for a virtual private cloud (VPC). It filters traffic at the perimeter of the VPC.

- It can inspect traffic going to and from an internet gateway, NAT gateway, VPN, or Direct Connect.
- It uses Suricata for stateful inspection and supports Suricata-compatible rules.
- AWS describes it as a service that helps deploy essential network protections across VPCs with fine-grained control.
- It can help block specific traffic patterns, such as outbound Server Message Block (SMB) requests.

**Firewall types and placement**

Firewalls are commonly categorized by where they sit in the network and what they inspect. The evidence emphasizes placement at the network edge and modern filtering across multiple layers.

- Edge firewalls protect the boundary between internal and external networks.
- Cloud firewalls protect virtual networks such as a VPC.
- Modern firewalls may inspect traffic across multiple layers, from network to application behavior.
- The right firewall depends on the environment, traffic patterns, and control requirements.

**Practical exam takeaways**

For A+ purposes, focus on what a firewall does, where it is commonly placed, and how Windows handles basic firewall configuration.

1. Identify whether the network is public or private before choosing a security posture.
2. Keep Windows Defender Firewall enabled unless troubleshooting requires a temporary change.
3. Use elevated permissions when modifying firewall settings.
4. Recognize that managed cloud firewalls can provide scalable protection without local infrastructure management.

**Key terms**

- **Stateful**: Tracks connection state so filtering can consider the context of traffic.
- **VPC**: Virtual Private Cloud, an isolated cloud network environment.
- **SMB**: Server Message Block, a network protocol often associated with file sharing.
