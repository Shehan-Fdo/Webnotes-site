## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**IPv4 fundamentals**

Internet Protocol version 4 (**IPv4**) is the long-standing addressing scheme used on modern networks. It is a Layer 3 address in the Open Systems Interconnection (**OSI**) model and is commonly written in dotted decimal notation.

- An IPv4 address has **32 bits** total.
- It is shown as **four decimal numbers** separated by periods, such as `192.168.1.131`.
- Each section is an **octet** of 8 bits.
- Each octet ranges from `0` to `255`.

This format is human-readable, but the device still processes the address in binary. Knowing the decimal form and the binary structure helps you recognize valid addresses and understand why each octet has a fixed range.

**IPv6 fundamentals**

Internet Protocol version 6 (**IPv6**) is the newer IP protocol and is included with many modern operating systems. It exists alongside IPv4 on current networks, so you should recognize both formats.

- IPv6 uses a much larger address space than IPv4.
- It is written in a different style than dotted decimal IPv4.
- It is important to know how to configure IPv6 on a workstation when the environment uses it.

IPv6 appears in modern deployments because IPv4 address space is limited. In practice, you may encounter systems that support both protocols at the same time.

**IPv4 vs. IPv6 awareness**

You do not need to treat IPv4 and IPv6 as interchangeable. They solve the same basic problem—addressing devices on a network—but they differ in structure and size.

| Feature | IPv4 | IPv6 |
|---|---|---|
| Address length | 32-bit | 128-bit |
| Common display | Dotted decimal | Colon-separated format |
| Typical status | Widely used | Newer, widely supported |

A useful exam habit is to identify the protocol from the address shape before you think about configuration details. If you see four decimal groups, think IPv4. If you see the newer expanded format, think IPv6.

**Addressing concepts you should recognize**

When configuring TCP/IP on a Windows computer, the settings commonly include an IP address, a subnet mask, and a default gateway. These values work together to let the device communicate on the local network and beyond.

- **IP address** identifies the device on the network.
- **Subnet mask** helps define the network and host portions.
- **Default gateway** provides a path to other networks.

These concepts matter because IP addressing is not just about assigning a number. You also need to understand how the address fits into the network and how traffic leaves the local segment.

**Public, private, and automatic addressing**

IPv4 addressing includes both public and private use cases. Public addresses are used for broader network communication, while private addresses are used inside local networks.

- **Private IP addressing** is common inside home and business networks.
- **Public IP addressing** is used when a device needs to be reachable on the wider internet.
- **Automatic Private IP Addressing (APIPA)** can provide limited connectivity when normal address assignment fails.

APIPA is a fallback behavior, so it is useful to recognize it during troubleshooting. If a workstation cannot obtain a proper address, APIPA may allow local communication instead of complete isolation.

**Classful IPv4 awareness**

Classful addressing was an early IPv4 method that divided the address space into fixed classes. It is mainly useful as background knowledge because modern networks rely on more flexible addressing methods, but the class labels still appear in networking discussions.

- **Class A**, **Class B**, and **Class C** were used for unicast communication.
- **Class D** was reserved for multicast.
- **Class E** was intended for experimental and research use.

This historical model helps explain older references you may still encounter in study material or legacy documentation.

**Practical recognition tips**

When you look at an address, focus on format first and meaning second.

1. Check whether the address uses four decimal octets or the newer IPv6 style.
2. Confirm whether the value fits the expected range for IPv4 octets.
3. Identify whether the address is likely private, public, or a fallback address such as APIPA.
4. Match the address type to the configuration task on the workstation.

> If a workstation is configured with an address that does not match the network’s expected protocol version, connectivity problems are likely even when the physical connection is working.

**Quick memory cues**

- IPv4 = **32-bit**, four octets, dotted decimal.
- IPv6 = **newer**, larger address space, modern operating system support.
- TCP/IP configuration commonly needs an address, subnet mask, and default gateway.
- APIPA can provide limited connectivity when normal assignment fails.
