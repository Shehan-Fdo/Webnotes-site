## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Subnetting overview**

Subnetting is the process of dividing a large IP network into smaller pieces called subnets. At a conceptual level, the goal is to make large networks easier to manage and to keep communication organized as the number of devices grows.

The Internet contains billions of devices, so a single device cannot reasonably track every possible destination on its own. Subnetting helps by breaking the network into smaller logical sections that are easier to route and maintain.

- **Subnet**: a smaller logical network created from a larger IP network.
- **IP address**: the identifier used to locate a device on a network.
- **Routing**: the process of moving traffic between networks.

**Why subnetting exists**

Subnetting exists because networks become difficult to manage when everything shares one large address space. Splitting a network into smaller parts reduces complexity and supports more practical routing decisions.

It also improves how organizations organize traffic. For example, different parts of a network can be separated into their own subnets instead of forcing every device to share the same broadcast domain.

- Smaller networks are easier to understand and administer.
- Traffic stays more organized when devices are grouped logically.
- Large address spaces are used more efficiently when divided into smaller blocks.

> Subnetting is conceptually about organization and scale, not just math.

**Classful and classless ideas**

Legacy class-based subnetting used address classes such as Class A, Class B, and Class C as a starting point for IP addressing. In that model, the default subnet mask depended on the class.

- Class A used `255.0.0.0`
- Class B used `255.255.0.0`
- Class C used `255.255.255.0`

This class-based style is no longer the modern approach, but the terms still appear in casual conversation. You may still hear someone describe an address as a “Class A” or “Class C” address even though modern networking uses classless methods.

Classless subnetting is the modern approach. It does not rely on fixed class boundaries, which gives networks more flexibility when they are designed and routed.

**How subnetting helps the network**

Subnetting supports practical network design by keeping traffic separated into smaller logical groups. Devices in the same subnet can communicate more directly, while traffic between subnets is handled through routing.

This separation helps with:

- **Manageability**: you can organize devices by department, location, or purpose.
- **Performance**: smaller groups reduce unnecessary network chatter.
- **Security**: separating groups makes it easier to control traffic flow.
- **Scalability**: you can grow a network without treating every device as part of one giant segment.

**Conceptual model to remember**

Think of subnetting as taking one large network and splitting it into smaller sections that still belong to the same overall environment. Each section can operate more independently, but the sections remain connected through routing.

A simple way to remember the idea:

1. Start with a large IP network.
2. Divide it into smaller logical networks.
3. Route traffic between those networks when needed.

**What to focus on for A+**

For CompTIA A+, you mainly need the concept, not deep binary calculations. You should understand why subnetting exists, what a subnet is, and how it supports network organization.

- Know that subnetting divides a large network into smaller logical networks.
- Know that classful subnetting is legacy terminology.
- Know that classless subnetting is the modern approach.
- Know that routing connects separate subnets.
