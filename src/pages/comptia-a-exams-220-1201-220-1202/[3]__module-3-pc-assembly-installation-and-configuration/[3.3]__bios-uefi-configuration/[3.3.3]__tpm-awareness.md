## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**TPM overview**

A <abbr title="Trusted Platform Module">TPM</abbr> is a specialized hardware chip, usually on the motherboard, that supports security-related cryptographic functions. It is commonly associated with encryption and helps protect sensitive data by keeping certain operations inside dedicated hardware rather than relying only on software.

- **Hardware-based** security means the TPM can perform cryptographic tasks in a more isolated environment.
- It is used for **encryption** and related security features.
- The chip is designed to be tamper-resistant, which helps protect stored security material.

**What a TPM does**

The TPM can generate, store, and limit the use of cryptographic keys. It also supports device authentication and access control in Windows environments. Because the TPM operates outside the host operating system, malware has a harder time interfering with its security functions.

> A compromised operating system does not automatically expose the data protected by the TPM.

- It can securely store:
  - passwords
  - certificates
  - encryption keys
- It helps protect data even when software on the system is attacked.
- It supports trusted boot-related security checks by helping verify that startup begins from a trusted hardware and software combination.

**Why TPM matters**

For A+ purposes, focus on TPM as a security component, not a storage device or performance feature. Its main value is protecting cryptographic material and strengthening platform trust.

- It is a **secure crypto-processor**.
- It supports modern platform security requirements.
- It is part of the security foundation used by operating systems such as Windows.

**Exam focus**

When you see TPM on the exam, connect it to these ideas:

- motherboard-based security hardware
- cryptographic operations
- encryption key protection
- resistance to software tampering
- device authentication and access control
