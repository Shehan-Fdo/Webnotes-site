## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Firmware update basics**

Firmware is the low-level software that starts the system and helps it reach the point where an operating system can load. On PCs, this is commonly the BIOS (Basic Input/Output System) or UEFI (Unified Extensible Firmware Interface). Modern systems usually store this firmware in flash memory, which allows updates through software instead of replacing chips.

Firmware updates are not routine maintenance. You usually apply them for a specific reason, such as fixing bugs, improving stability, resolving hardware conflicts, addressing security vulnerabilities, or occasionally improving performance.

- **BIOS** updates change the startup firmware on legacy and modern systems that still use BIOS terminology.
- **UEFI** updates serve the same general purpose on newer systems.
- **Flash memory** makes software-based updates practical on current hardware.
- **Device firmware** can also be updated on items such as video adapters and gaming consoles.

**Why you update firmware**

A firmware update is a targeted change, not a regular cycle task. The evidence points to a few common motivations:

- Fixing bugs in the startup code or device behavior
- Improving system stability
- Resolving hardware compatibility or conflicts
- Closing security vulnerabilities
- Providing occasional performance improvements

> Take caution during firmware updates: incorrect installation can prevent the system from starting correctly.

**Typical update workflow**

The exact process depends on the manufacturer, but the general pattern is consistent. You first identify the correct firmware for the exact system model, then follow the vendor’s instructions carefully.

1. Visit the manufacturer’s website.
2. Find the latest firmware version for your exact system or motherboard model.
3. Download the correct update file.
4. Check the vendor’s instructions for the required update method.
5. Boot into the BIOS/UEFI utility if the process requires it.
6. Select the update option and load the file from a USB drive or other supported storage.
7. Restart the computer after the update completes.
8. Verify that the update succeeded.

A careful download step matters because the wrong file can cause the update to fail. The manufacturer’s documentation is the best source for the required file type, update utility, and verification steps.

**Best practices and precautions**

Planning reduces risk. Before you start, make sure you know what hardware you have and what the vendor expects.

- Confirm the motherboard or system model before downloading anything.
- Use the vendor’s documentation to determine the correct update file and checksum guidance if provided.
- Follow the update instructions exactly as written.
- Avoid updating firmware just because a newer version exists.
- Verify the result after rebooting.

**Accessing BIOS or UEFI settings**

You often enter the BIOS or UEFI setup during startup by pressing a special key. Common examples include `Delete`, `F1`, `F2`, or key combinations such as `Ctrl` + `S` or `Ctrl` + `Alt` + `S`.

Some virtual machines also let you interrupt boot and open firmware settings through the hypervisor. This is useful when you need to review boot options or other firmware settings in a lab environment.

**What to remember for the exam**

- Firmware is the startup software that initializes hardware and begins the boot process.
- BIOS and UEFI updates are usually done to solve a problem, not as routine maintenance.
- Always use the correct manufacturer file for the exact system model.
- Follow the vendor’s update procedure carefully to reduce the risk of a failed update.
- Verify the update after the system restarts.
