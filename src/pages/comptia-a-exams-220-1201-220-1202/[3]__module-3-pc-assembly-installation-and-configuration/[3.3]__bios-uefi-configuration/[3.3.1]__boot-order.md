## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Boot order basics**

Boot order is the sequence firmware uses to decide which storage device to check first when a system starts. After the power-on self-test (POST), the firmware looks for a device with an installed operating system and attempts to boot from it.

Common boot candidates include:

- **Hard disk drives** and **solid-state drives (SSDs)**
- **Optical media**
- **M.2 storage**
- **USB thumb drives**

The practical goal is to place the device you want to boot from first in the list. For example, a removable USB device may need to be first when you are installing or repairing an operating system.

**BIOS and UEFI access**

You usually change boot order inside the system firmware setup, either BIOS (Basic Input/Output System) or UEFI (Unified Extensible Firmware Interface). To enter setup, you press a special key during startup before the operating system begins loading.

Common keys mentioned in the evidence include:

- `Delete`
- `F1`
- `F2`
- `Ctrl` + `S`
- `Ctrl` + `Alt` + `S`

> The exact key depends on the motherboard or system manufacturer.

Some systems also provide a boot selection menu or firmware settings through a virtual machine environment. Hypervisors such as Hyper-V, VMware Workstation, and VMware Fusion can expose firmware settings for virtual machines.

**Why boot order matters**

Boot order controls which device gets priority when multiple bootable devices are present. That matters when you have several drives installed, because the system may otherwise start from the wrong one.

Use boot order to:

- Start from an installation USB when deploying an operating system
- Boot from the internal drive for normal use
- Test alternate boot media during troubleshooting

If the wrong device is first, the system may not start the expected operating system or may boot into the wrong media.

**Changing the boot device**

A typical workflow is to enter firmware setup, locate the boot options, and adjust the priority list. On some systems, you can also use a one-time boot menu instead of permanently changing the order.

1. Restart the system.
2. Press the correct firmware key during startup.
3. Open the boot options or boot order settings.
4. Move the desired device to the top of the list.
5. Save the changes and exit.

**Troubleshooting considerations**

If you cannot reach firmware setup, the startup window may be very short. Try again and press the key repeatedly as the system powers on. If you are working in a virtual machine, use the hypervisor’s controls to interrupt startup and access firmware settings.

Keep in mind that boot behavior varies by manufacturer, so the exact menu names and key sequence are not universal. The exam-relevant idea is simple: the system checks boot devices in priority order until it finds a bootable one.
