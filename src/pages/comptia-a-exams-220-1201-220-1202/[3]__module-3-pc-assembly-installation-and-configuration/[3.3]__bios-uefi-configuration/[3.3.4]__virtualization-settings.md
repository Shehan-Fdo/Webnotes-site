## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Virtualization settings overview**

Virtualization settings control whether your system firmware exposes hardware-assisted virtualization to the operating system and to virtual machine software. In practice, you usually enable these options in the BIOS or UEFI setup utility before a hypervisor can use them effectively.

- **BIOS/UEFI** access usually starts during boot with a setup key such as `Delete`, `F1`, `F2`, or a key combination like `Ctrl` + `S`.
- **Virtualization** support is commonly enabled in CPU or advanced settings.
- **Hyper-V**, VMware Workstation, and VMware Fusion can provide a way to work with VM firmware settings without changing the physical machine’s BIOS.
- VirtualBox is noted as lacking direct BIOS access in this context, so you may need another method or a BIOS simulator for practice.

**What the setting does**

Enabling virtualization allows the system to run multiple operating systems concurrently and in isolation on the same hardware. The evidence points to common hardware features such as Intel Virtualization Technology (`VT-x`) and AMD-V, which are enabled from firmware menus rather than from inside the guest operating system.

- **VT-x** is Intel’s virtualization technology.
- **AMD-V** is AMD’s virtualization support.
- **SVM Mode** is the AMD firmware option referenced for enabling virtualization on some boards.
- **VMX** appears in some Intel motherboard menus as `Intel(VMX) Virtualization Technology`.

> If virtualization is disabled in firmware, the hypervisor may not be able to use the CPU features it needs for efficient VM operation.

**How you enable it in firmware**

The exact menu names vary by motherboard vendor, but the workflow is consistent: enter setup during boot, find the CPU or advanced configuration area, enable the virtualization option, then save and reboot.

1. Restart the computer and enter BIOS or UEFI setup with the vendor’s setup key.
2. Open the advanced or CPU configuration menu.
3. Enable the virtualization option:
   - `Intel(VMX) Virtualization Technology` on Intel systems
   - `SVM Mode` on AMD systems
4. Save changes with `F10` and confirm the reboot.

```text
Typical firmware path:
Advanced -> CPU Configuration -> Virtualization Technology -> Enabled
```

**Working with virtualization in Windows**

If you are using Windows, you can often test or manage VM firmware settings through a desktop hypervisor instead of changing the host machine’s BIOS directly. The evidence specifically mentions `Hyper-V` and VMware Workstation as options for accessing VM BIOS settings.

- **Hyper-V** can be used on Windows 10 and Windows 11 for VM-related firmware access.
- **VMware Workstation** can also expose VM BIOS settings.
- **VMware Fusion** is mentioned for macOS environments.
- Use this approach when you want to practice BIOS navigation without altering the physical system.

**Practical exam takeaways**

For A+ purposes, focus on the relationship between firmware settings and virtualization software. The key idea is that the CPU feature must be enabled in BIOS or UEFI before the hypervisor can use it.

- Know the common setup keys: `Delete`, `F1`, `F2`, `Ctrl` + `S`, and `Ctrl` + `Alt` + `S`.
- Recognize vendor-specific labels for the same feature.
- Understand that firmware virtualization support improves how VM platforms use hardware resources.
- Remember that some hypervisors let you access a VM’s BIOS-like settings directly.
