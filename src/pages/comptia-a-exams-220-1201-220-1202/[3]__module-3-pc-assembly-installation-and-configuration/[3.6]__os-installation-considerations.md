## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Installation planning**

Operating system (OS) installation starts with choosing a boot method that can launch the installer before the target OS exists. In practice, you use bootable media or a network-based deployment path, then follow the installer prompts for setup choices.

- **Boot media** must be usable by the system firmware.
- A **bootable USB drive** is common because it is portable and easy to reuse.
- **CD-ROM/DVD-ROM** media still appears in some environments, but it is less common.
- **PXE** (Preboot Execution Environment) lets you boot from a central server in larger deployments.
- Some environments use **zero-touch deployment** to reduce hands-on installation work.

> If the system cannot boot the installer, you cannot begin the installation process.

**Installation types and setup choices**

The installer usually asks you to make decisions that shape the final system. These choices include the installation type, partition layout, and file system selection. Keep the target use case in mind before you proceed.

- **Clean install** replaces the existing OS and is useful when you want a fresh start.
- **Upgrade** preserves more of the existing environment than a clean install.
- **Repair** focuses on restoring a damaged installation rather than replacing it.
- You may also choose language, default applications, and other setup options during installation.
- Drivers and additional software may be needed after setup so hardware works correctly.

**Partition style considerations**

Partition style must match the boot mode and firmware expectations of the computer. A mismatch can block installation even when the media and installer are working.

- **GPT** (GUID Partition Table) supports UEFI-based installation scenarios.
- **MBR** (Master Boot Record) is associated with legacy BIOS-compatibility mode.
- If Windows Setup reports that the disk is not the correct partition style, check whether the system booted in **UEFI** mode or legacy BIOS-compatibility mode.
- You can resolve the mismatch by rebooting in the matching mode or by reconfiguring the drive for the required partition style.

**Formatting and file system choices**

Formatting prepares the partition for use by the operating system. The installer may prompt you to select a file system as part of the setup flow.

- Choose the file system that matches the installation requirements.
- Formatting decisions affect how the OS stores and accesses data.
- Partitioning and formatting are separate steps, even though installers often present them together.

**Practical installation workflow**

1. Create or obtain bootable installation media.
2. Confirm the computer can boot from that media or from the network.
3. Start the installer and select the installation type.
4. Choose the partition and formatting options.
5. Complete the setup prompts.
6. Install drivers and any required software after the OS finishes installing.

**Exam focus**

For A+ exam prep, focus on matching the installation method to the environment. USB is the most common local method, PXE is useful for centralized deployment, and partition style must align with the boot mode.

- **USB** is the most common direct-install option.
- **PXE** is useful when you want to avoid visiting each machine individually.
- **UEFI** systems generally expect **GPT**.
- Legacy BIOS-compatibility mode aligns with **MBR**.
