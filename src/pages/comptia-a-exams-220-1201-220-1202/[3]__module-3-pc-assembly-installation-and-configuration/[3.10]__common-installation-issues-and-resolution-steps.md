## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Installation planning and data protection**

Before you install or upgrade an operating system, you need to inspect the target drive and understand what is already stored there. Installation can overwrite existing data, so the safest approach is to verify the drive contents first and back up important files when needed.

A clean install and an upgrade are not the same thing. An upgrade moves from one operating system version to another, while a clean install replaces the existing setup. That difference matters because the risk to existing data is much higher during a clean install.

- Check whether the storage drive is empty or already in use.
- Back up user data if the drive contains important files.
- Expect partitioning and formatting to happen during installation.
- Treat the install process as a point where data can be overwritten.

**Compatibility and selection issues**

Hardware selection affects whether the system boots and operates correctly after installation. CPU choice must match the type of system you are building, such as a workstation or server, and the processor must fit the motherboard and platform requirements.

Some older motherboards use a locking screw instead of an arm for CPU installation. If that lock is not secured, the system may not boot. This is a practical reminder that installation issues are often mechanical, not just software-related.

- Match the processor to the intended system type.
- Verify the CPU is installed and locked correctly.
- Confirm the motherboard mechanism is fully secured.
- Use the correct hardware for the platform you are setting up.

**Troubleshooting approach**

IT support commonly uses tiered levels, with Level 1, Level 2, and Level 3 handling different kinds of issues. In practice, installation problems often start with basic support and move upward if the issue needs deeper technical analysis.

A structured troubleshooting methodology helps you avoid random guesswork. You identify the problem, check likely causes, and resolve the issue in a logical order. This approach is especially useful when installation failures could come from hardware, software, or configuration mistakes.

- Start with the simplest likely cause.
- Escalate when the issue exceeds first-line support.
- Use a structured process to isolate the failure.
- Keep the focus on verifiable symptoms and configuration details.

**Practical resolution steps**

When an installation fails or behaves unexpectedly, begin with the environment and the target system. Confirm the drive state, verify the hardware installation, and then review whether the issue is related to the install method or the selected components.

If the problem involves a Windows move to a newer version, decide whether an upgrade or a clean install is appropriate. If the problem involves a CPU or motherboard mismatch, correct the hardware before retrying the installation.

1. Inspect the target drive and identify any existing data.
2. Back up files that must be preserved.
3. Confirm the installation type you need: upgrade or clean install.
4. Verify hardware compatibility, especially CPU and motherboard fit.
5. Check that all locking mechanisms and installation steps are complete.
6. Retry the installation after correcting the identified issue.

**Troubleshooting mindset for exam prep**

The exam focus is less about memorizing one fix and more about recognizing the category of problem. Data loss risk, hardware mismatch, and incomplete installation steps are all common reasons an install can fail or produce a bad result.

A good answer on the exam usually reflects careful preparation first, then methodical correction. That means you protect data, confirm compatibility, and use a support process that narrows the cause before you act.
