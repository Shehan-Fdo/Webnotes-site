## **Course:** CompTIA A+ (Exams 220-1201 & 220-1202)

**Keyboard basics**

A keyboard is an input device, and A+ questions often focus on how it connects, how it feels to use, and how it behaves during startup. You should be able to distinguish common keyboard types and recognize when a keyboard problem is part of a larger boot or hardware issue.

- **Mechanical** keyboards use individual switches for each key and often provide tactile or audible feedback.
- **Membrane** keyboards use pressure pads and are usually quieter and more affordable.
- **Hybrid** designs and **scissor** mechanisms combine compact design, durability, or noise control in different ways.
- Keyboard choice often comes down to comfort, efficiency, and personal preference.

**Connection types**

Keyboard connectivity affects compatibility, stability, and convenience. Wired options avoid battery concerns, while wireless options improve mobility. Some connection types are also tied to older hardware or specific setup needs.

- **USB** wired keyboards are widely compatible and provide a stable connection.
- **PS/2** keyboards are associated with older systems and may be used for BIOS setup.
- **Bluetooth** keyboards provide wireless freedom.
- **2.4 GHz** wireless keyboards use a dongle for connectivity.
- **Lightning** and **USB Type-C** appear in some modern keyboard ecosystems.

> A keyboard connection choice can affect troubleshooting. If the keyboard is not responding, confirm the interface, power source, and whether the system supports that connection type.

**Startup and POST checks**

The power-on self-test, or **POST** (power-on self-test), runs diagnostic checks when you first turn on a PC. It verifies that key components are available, including the CPU, video, memory, and keyboard. If POST detects a problem, it may signal the issue with beeps or on-screen messages.

- POST checks for a working **keyboard** during startup.
- Beep codes can indicate a hardware problem before the operating system loads.
- On-screen messages may help identify the failing component.
- If video is unavailable, you may hear beeps without seeing the error details.

**Troubleshooting input problems**

When you troubleshoot a keyboard issue, separate a simple input failure from a broader system problem. If the keyboard does not respond during boot, POST behavior helps you narrow the cause. If the system is otherwise working, focus on the keyboard’s connection type and compatibility.

1. Verify that the keyboard is connected using the correct interface.
2. Check whether the keyboard has power or a wireless link if it is not wired.
3. Observe POST behavior for beeps or messages that point to a hardware fault.
4. Test with a known-good keyboard if you need to isolate the issue.

**Choosing the right keyboard**

Keyboard selection is not only about typing feel. It also affects portability, compatibility, and how the device fits the work environment. A practical choice balances the switch type, connection method, and intended use.

- **Comfort** matters when you type for long periods.
- **Compatibility** matters when you need the keyboard to work with specific devices or older systems.
- **Portability** matters when you want a wireless or compact setup.
- **Noise** matters in shared spaces where quieter input is preferred.

**Quick exam reminders**

- Know the difference between mechanical and membrane keyboards.
- Match the connection type to the device and use case.
- Remember that POST can report keyboard-related problems before the OS loads.
- Use beep codes and error messages as clues when video output is limited.
