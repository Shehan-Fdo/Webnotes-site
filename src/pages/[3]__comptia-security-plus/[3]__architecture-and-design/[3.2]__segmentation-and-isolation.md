## **Course:** CompTia Security Plus

**Segmentation basics**

- **Network segmentation** separates parts of a network into different zones.
- You can use devices like firewalls, switches, and routers to separate zones.
- Segmentation can be physical, logical, or virtual.
- It applies in data centers and on-premises environments.
- Segmentation helps protect data and reduce security risk.

**Why segmentation matters**

- Segmentation helps mitigate potential risks.
- Segmentation can reduce the severity of security incidents.
- Segmentation can limit **lateral movement** after a compromise.
- Segmentation can improve performance for high-bandwidth applications.
- Segmentation can keep users away from sensitive internal systems.

**Common controls**

- Use **VLANs** to separate network traffic.
- Use firewalls to control traffic between zones.
- Use **ACLs** to enforce access rules.
- Use separate devices or virtualized networks when needed.
- Use resource access controls in virtual environments.

**Isolation in virtualization**

- Isolation helps each virtual machine run independently.
- Isolation helps protect one VM when another VM is compromised.
- Configure resource limits to reduce resource exhaustion.
- Apply network design controls to support VM separation.
- Use VLANs, firewalls, and ACLs to strengthen isolation.

**Design goals**

- Start with segmentation to shrink the attack surface.
- Apply least privilege to network access.
- Allow only the protocols needed for the application.
- Separate users from core systems when direct access is not required.
- Choose the simplest control that reduces risk.

**Practical takeaways**

- Segment by function, sensitivity, or trust level.
- Isolate critical assets from general user traffic.
- Monitor traffic between segments.
- Review whether a control supports security, performance, or both.
- Use segmentation to support secure network architecture.
