## **Course:** CompTia Security Plus

**Secure application development**

- Secure application development focuses on building software with security in mind from the start.
- You should integrate security into the **SDLC** (software development life cycle).
- You should identify and mitigate application threats during development.
- Secure coding practices help reduce risk in the finished application.
- Application security also extends to cloud, container, and API environments.

**Development lifecycle models**

- A development lifecycle describes how software moves from requirements to support.
- Common phases include requirements, planning, design, coding, testing, deployment, and support.
- **Waterfall** follows a linear sequence.
- Waterfall works phase by phase and offers limited flexibility for changing requirements.
- Security concerns need to be addressed early in Waterfall.
- **Agile** uses small, rapid iterations.
- Agile supports change more easily than Waterfall.

**Security baselining**

- **Baselining** means establishing security standards for systems in your network.
- A security baseline defines expected controls for a system type.
- Baselines can cover services, file permissions, Registry permissions, and authentication protocols.
- You should create separate baselines for different system roles.
- Examples include domain controllers, file servers, print servers, application servers, and clients.

**Deployment and cloud concepts**

- Cloud deployment models describe how services are delivered and managed.
- **IaaS** (Infrastructure as a Service) lets you provision and run workloads without managing the underlying infrastructure.
- **PaaS** (Platform as a Service) lets you create and host applications.
- **SaaS** (Software as a Service) lets you use provider-hosted applications over the Internet.
- **SecaaS** (Security as a Service) delivers security functions as a service.

**Mobile security controls**

- Mobile devices often need direct security controls during deployment.
- A **screen lock** uses a password to restrict access to the device.
- A **proximity lock** automatically locks the device when you move away from it.
- These controls help reduce unauthorized access to device contents.

**Practical exam focus**

- Expect questions about secure design choices, not just coding.
- Match the lifecycle model to the project’s change tolerance.
- Apply baselines to keep system configurations consistent.
- Choose the cloud model based on how much control you need over the environment.
