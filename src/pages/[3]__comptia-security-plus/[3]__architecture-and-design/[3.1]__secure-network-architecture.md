## **Course:** CompTia Security Plus

**Secure network architecture overview**

- Secure network architecture designs network infrastructure with security as a foundational element.
- It uses multiple layers of defense to protect organizational assets.
- It supports secure systems, networks, and applications.
- CompTIA Security+ expects you to design secure network architectures and apply secure protocols.

**Core design concepts**

- **Network segmentation** divides a network into distinct zones.
- Segmentation limits lateral movement after a breach.
- Segmentation helps contain damage.
- Common implementation methods include VLANs, subnets, and physical separation.
- A **DMZ** (demilitarized zone) holds public-facing services while protecting internal networks.
- **Defense in depth** places security controls at multiple network layers.
- Firewalls, intrusion detection systems, intrusion prevention systems, secure routers, and proxies can work together.
- **Zero Trust Architecture** follows a “never trust, always verify” model.
- Zero trust requires authentication and authorization regardless of location.

**Network devices and technologies**

- Firewalls enforce traffic rules between network zones.
- Routers and switches need secure configuration.
- Load balancers can support resilient service delivery.
- Proxies and web security gateways help mediate access to web resources.
- VPN concentrators support secure remote connectivity.
- Network intrusion detection systems and network intrusion prevention systems monitor and block suspicious activity.
- Protocol analyzers help inspect network traffic.
- Spam filters, web application firewalls, URL filtering, content filtering, and content inspection add layered protection.

**Secure network administration**

- Apply secure administration principles to network devices.
- Use secure configurations on routers and switches.
- Review firewall rules for intended access.
- Protect wireless networks with encryption and proper SSID handling.
- Implement common secure protocols such as `SSL` and `IPSec`.
- Identify common ports and services such as `HTTP`, `HTTPS`, and `FTP` when planning controls.

**Frameworks and reference architectures**

- Frameworks and reference architectures provide conceptual blueprints for IT systems.
- They help align IT and security with business strategy.
- Frameworks are more generic than architectures.
- Regulatory frameworks apply in regulated industries.
- Non-regulatory frameworks are technology-focused.
- `NIST` Cybersecurity Framework (`CSF`) is an example of a non-regulatory framework.

**Practical exam focus**

- Expect questions on how design choices reduce risk.
- Expect questions on how segmentation limits exposure.
- Expect questions on how layered controls support resilience.
- Expect questions on how zero trust changes access decisions.
- Expect questions on secure device configuration and secure protocol use.
