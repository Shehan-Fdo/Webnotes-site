## **Course:** CompTia Security Plus

**Cloud security basics**

- Cloud computing lets you deploy application instances quickly and scale them up or down as demand changes.
- Cloud services can provide access to applications from anywhere.
- Cloud planning must include how you develop, deploy, recover, and retire resources.

**Cloud service models**

- **IaaS** stands for Infrastructure as a Service.
- In `IaaS`, the provider supplies the hardware, storage, and networking.
- You manage the operating system and applications in `IaaS`.
- `IaaS` still leaves you responsible for data security on the system.
- `PaaS` stands for Platform as a Service.
- `SaaS` stands for Software as a Service.
- Cloud deployment can also use public or private models.

**Shared responsibility**

- Cloud security depends on the **shared responsibility** model.
- The provider and the customer split security tasks.
- You must know which party handles each control before you deploy.

**Virtualization security risks**

- Virtualization makes it easy to create many systems quickly.
- This speed can lead to **VM sprawl**.
- VM sprawl happens when you create more virtual machines than you can track or justify.
- Untracked virtual machines make cleanup and deprovisioning harder.
- A **VM escape** lets an attacker move out of a virtual machine boundary.
- A virtual machine manager can be a useful starting point for an attacker.
- Resource reuse can create security concerns in cloud-based virtual environments.

**Virtualized environment management**

- Virtual machines may be created and torn down during the workday.
- Different virtual machines may use different CPU, memory, or storage configurations.
- These differences make consistent security management harder.
- You should use a formal provisioning process for application instances.
- You should also use a formal deprovisioning process when instances are no longer needed.
- You should return unused resources to the pool.

**Practical security focus**

- Track what you deploy.
- Remove what you no longer need.
- Protect the virtualization layer as part of the overall cloud security posture.
- Plan for automation without losing control of system lifecycle.
