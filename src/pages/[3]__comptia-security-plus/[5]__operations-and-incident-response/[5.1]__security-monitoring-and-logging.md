## **Course:** CompTia Security Plus

**Security monitoring goals**

- Monitor networks continuously because attackers constantly try to gain access.
- Review authentications and logins to verify activity is legitimate.
- Track services, remote access, and account access for unusual behavior.
- Use real-time monitoring to react to security events as they happen.
- Roll up monitoring data into dashboards for quick status checks.

**Log data and aggregation**

- Collect logs from servers, devices, applications, endpoints, and operating systems.
- Use **log aggregation** to combine logs from multiple sources into one stream.
- Correlate logs across systems to connect related events.
- Review firewall logs for allowed and blocked traffic flows.
- Use intrusion prevention logs to identify exploit attempts.
- Check DNS sinkhole traffic for signs of malicious internal activity.

**Security Information and Event Management**

- **SIEM** stands for Security Information and Event Management.
- Use SIEM to centralize security-related data for analysis and response.
- Combine event data with other logs to improve visibility.
- Use SIEM outputs to support detection and investigation workflows.

**Monitoring data and integrity**

- Use **file integrity monitoring** to detect unexpected changes to important files.
- Treat rarely changing executables and libraries as high-value monitoring targets.
- Use Windows `SFC` (System File Checker) to scan critical operating system files on demand.
- Monitor for file changes that may indicate tampering or compromise.
- Use data loss prevention, USB blocking, and related controls to reduce exposure.

**Alerts, scanning, and reporting**

- Use alerting to notify you when monitored conditions change.
- Tune alerts so they support response without creating unnecessary noise.
- Add scanning when you need more detail about suspicious activity.
- Use reporting to summarize security posture and observed events.
- Keep monitoring focused on systems, access paths, and network activity that matter most.

**Practical monitoring focus**

- Watch for blocked and allowed traffic to understand network behavior.
- Check URL categories when a workstation shows suspicious browsing patterns.
- Review authentication events to spot abnormal access attempts.
- Use monitoring data to document traffic flows and support incident analysis.
- Prefer centralized views when you need to compare activity across multiple sources.
