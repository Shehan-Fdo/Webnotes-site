## **Course:** CompTia Security Plus

**AAA overview**

- **AAA** stands for authentication, authorization, and accounting.
- It is a framework for network security and access control.
- It supports secure login and resource access.
- It is commonly used when you log in to a network, VPN, or switch command line.

**Authentication**

- Authentication proves you are who you say you are.
- You usually start with a username.
- You then provide a password or other authentication factors.
- Additional factors strengthen identity proof.
- Identification is the claim of a user identity before verification.

**Authorization**

- Authorization determines what access you have after authentication.
- It limits resources based on your role or account.
- A shipping and receiving user should access shipping systems, not finance data.
- Authorization enforces access boundaries for authenticated users.

**Accounting**

- Accounting records who logged in.
- It logs the date and time of access.
- It can track when a session ends.
- It supports auditing and usage review.

**Practical use**

- AAA helps control access in centralized environments.
- Network devices can request access decisions from AAA services.
- AAA is useful for monitoring user activity across systems.
- It helps separate identity verification from access decisions and logging.

**Security takeaways**

- Use authentication to verify identity.
- Use authorization to restrict access.
- Use accounting to record activity.
- Treat AAA as a core part of secure access control.
