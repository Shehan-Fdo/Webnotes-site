## **Course:** CompTia Security Plus

**Identity and Access Management overview**

- **IAM** matches the right permissions to the right people at the right time.
- It controls what users can and cannot access.
- It helps restrict sensitive data and functions to authorized users and resources.
- It starts when a user needs access and ends when you remove that access.
- It applies across desktop, laptop, mobile, on-premises, and cloud environments.
- It supports employees, vendors, contractors, and customers with different access needs.

**Core IAM activities**

- Provisioning creates user accounts and access.
- Deprovisioning removes user accounts and access.
- Permission assignment defines what a user can do after authentication.
- Access management controls which users can perform which actions.
- You should align permissions with business need and current role.

**Authentication concepts**

- Authentication verifies identity before access decisions.
- Password-based authentication remains a core method.
- **MFA** means multifactor authentication.
- MFA adds more than one factor to strengthen login security.
- Password best practices include length, complexity, reuse limits, expiration, and age controls.
- Password managers help users store and use strong passwords.
- Passwordless solutions reduce reliance on passwords.

**Federation and single sign-on**

- **SSO** means single sign-on.
- SSO lets a user access multiple systems after one successful sign-in.
- Federation connects identity across different systems or organizations.
- **LDAP** means Lightweight Directory Access Protocol.
- **OAuth** supports authorization for delegated access.
- **SAML** means Security Assertions Markup Language.
- Interoperability helps different identity systems work together.
- Attestation supports trust in identity-related assertions.

**Access control models**

- Access control models define how users, systems, and applications get access to resources.
- They support least privilege.
- They support separation of duties.
- **DAC** means discretionary access control.
- In DAC, the resource owner grants or denies access.
- **MAC** means mandatory access control.
- MAC uses labels or clearances tied to classifications.
- **ABAC** means attribute-based access control.
- ABAC uses attributes such as user role, resource type, and environment.

**Practical IAM focus**

- Use IAM to reduce unauthorized access.
- Use IAM to protect sensitive data and functions.
- Use IAM to manage access consistently across hybrid and remote work environments.
- Review access when roles change.
- Remove access promptly when it is no longer needed.
