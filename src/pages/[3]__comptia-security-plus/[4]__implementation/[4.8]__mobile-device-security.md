## **Course:** CompTia Security Plus

**Mobile device security overview**

- Mobile devices move outside direct organizational control.
- You must protect company data on devices used for work.
- **BYOD** means bring your own device.
- BYOD creates a mix of personal and corporate data on one device.
- Organizations often use **MDM** (Mobile Device Management) to manage mobile devices centrally.
- MDM helps track devices, data, and policy settings.

**Mobile Device Management controls**

- MDM can enforce application-based policies.
- MDM can control where data is stored.
- MDM can manage camera use.
- MDM can apply security settings to mobile devices.
- MDM is especially useful when users bring personal devices into the workplace.
- MDM helps reduce risk without requiring full ownership of the device.

**Deployment models**

- **BYOD** uses a personal smartphone or tablet for both personal and corporate use.
- BYOD requires a clear way to separate personal and corporate data.
- **COPE** means corporate-owned, personally enabled.
- Corporate-owned devices give the organization more direct control.
- **VDI** (Virtual Desktop Infrastructure) and **VMI** (Virtual Mobile Infrastructure) are deployment options for mobile access.
- Deployment choice affects how you manage data and security.

**Mobile enforcement and restrictions**

- Third-party app stores can still contain risky applications.
- Some apps may leak data or contain vulnerabilities.
- Rooting and carrier unlocking weaken device control.
- Firmware over-the-air updates can affect device security posture.
- Accessory use can be restricted by policy.
- You should enforce device rules that match the organization’s risk tolerance.

**Hardware-backed protection**

- A **hardware security module** (HSM) provides cryptographic features.
- A **microSD HSM** brings HSM functions to mobile devices in a smaller form factor.
- MicroSD HSMs can support encryption, key generation, digital signatures, and authentication.
- Hardware-backed storage can keep encryption and decryption keys secure.
- Hardware-backed protection ties cryptographic functions to the physical device.

**Mobile application and endpoint management**

- **Mobile Application Management** focuses on controlling mobile apps.
- **Unified Endpoint Management** helps manage multiple device types from one system.
- These tools support broader control across phones, tablets, and other endpoints.
- Mobile security often combines device, app, and endpoint controls.

**Practical exam takeaways**

- Use MDM to manage company and personal devices centrally.
- Use PIN enforcement when you need basic device access protection.
- Use remote wipe carefully when personal data must be preserved.
- Choose controls that protect company data without overreaching on personal data.
- Match the control to the problem before selecting a stronger action.
