## **Course:** CompTia Security Plus

**Secure protocol purpose**

- Secure protocols add built-in security mechanisms to network and application communication.
- They help protect confidentiality, integrity, and availability.
- They reduce risk from spoofing, interception, and replay attacks.
- They support interoperable communication between systems that follow the same rules.

**DNSSEC**

- DNSSEC stands for Domain Name System Security Extensions.
- It extends **DNS** to add origin authentication.
- It helps protect DNS data from spoofing.
- DNS normally uses UDP on port `53`.
- DNS requests and replies are often sent in plaintext without DNSSEC.
- DNSSEC can expose zone content in some cases, so you should consider that tradeoff.

**SRTP**

- SRTP stands for Secure Real-time Transport Protocol.
- It is the secure version of RTP, or Real-time Transport Protocol.
- It protects voice and video traffic in transit.
- It uses `AES` for encryption.
- It adds authentication, integrity, and replay protection.
- It uses `HMAC-SHA1` for message authentication.

**Common secure services**

- `HTTPS` protects web traffic with SSL/TLS.
- `SSH` secures remote command-line access.
- `S/MIME` secures email.
- `LDAPS` secures directory service communication.
- `FTPS` secures file transfer with SSL/TLS.
- `SFTP` provides secure file transfer.
- `SNMPv3` adds security to network management.
- `IPsec` secures IP traffic at the network layer.
- `AH` stands for Authentication Header.
- `ESP` stands for Encapsulated Security Payload.
- `ESP` supports tunnel and transport modes.

**Practical exam focus**

- Know what each protocol is used for.
- Know the relevant port when the protocol has one.
- Match the protocol to the service it protects.
- Distinguish secure versions from their insecure counterparts.

**Study reminders**

- Use secure protocols whenever you communicate sensitive data over a network.
- Choose the protocol that matches the service and security need.
- Remember that some exam questions focus on use case rather than implementation detail.
