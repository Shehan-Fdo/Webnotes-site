## **Course:** CompTia Security Plus

**Application and web attack basics**

- Web application attacks target websites, web applications, and related services.
- A single exposed form, API endpoint, or admin panel can become an entry point.
- Attackers often use weak input handling, weak session handling, or broken access control.
- These flaws can affect confidentiality, integrity, and availability.

**Common attack types**

- **SQL injection** uses unsanitized input to alter database queries.
- **Cross-site scripting (XSS)** injects malicious script into a page or browser context.
- **Cross-site request forgery (CSRF)** tricks a user into sending an unwanted request.
- **Server-side request forgery (SSRF)** makes a server send requests on the attacker’s behalf.
- **MIME sniffing** can cause a browser to interpret content in an unsafe way.
- **Request smuggling** abuses differences in how systems parse HTTP requests.
- **Broken access control** lets a user reach data or actions outside their permissions.

**Web security controls**

- Use **HTTPS/TLS** to encrypt client-server traffic.
- Use **content security policy (CSP)** to limit which resources a browser loads.
- Use input validation to reject unexpected data early.
- Use output encoding to prevent injected content from executing.
- Use sanitization to remove unsafe characters or markup.
- Use parameterized queries to reduce SQL injection risk.
- Use anti-CSRF tokens to verify legitimate requests.
- Use `HTTPOnly`, `Secure`, and `SameSite` cookie flags to improve cookie protection.
- Use a **web application firewall (WAF)** to filter malicious traffic before it reaches the app.

**Malware and web-delivered threats**

- **Ransomware** denies access to systems or data until payment.
- It can spread through phishing email or a compromised website.
- **Worms** spread themselves through networks, email, and messages.
- **Trojans** pretend to be harmless applications.
- **Rootkits** provide backdoor access to a system.
- **Keyloggers** record keystrokes from an infected machine.
- **Adware** shows unwanted ads and pop-ups.
- **Spyware** installs itself to spy on the infected machine.

**Practical defense approach**

1. Validate all user input.
2. Encode output before rendering it in a browser.
3. Use least privilege for accounts, APIs, and database access.
4. Protect sessions with secure cookie settings.
5. Test web forms, APIs, and admin panels for logic flaws.
6. Scan for misconfigurations and weak access controls.
7. Add layered controls such as CSP, WAF, and MFA.

**Exam focus**

- Know the difference between injection, scripting, and request-forgery attacks.
- Match each attack to its likely control.
- Recognize that web flaws often start with small input or access mistakes.
- Remember that practical defense combines secure coding and protective controls.
