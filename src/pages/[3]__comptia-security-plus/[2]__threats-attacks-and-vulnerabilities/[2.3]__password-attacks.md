## **Course:** CompTia Security Plus

**Password attack basics**

- Attackers use multiple techniques to discover passwords.
- Weak passwords increase the success of password attacks.
- Password reuse increases risk across services.
- Plain-text password storage exposes credentials if an attacker finds the database.

**Common attack methods**

- **Brute force** tries every possible password until one works.
- **Dictionary attacks** try likely words and common password patterns.
- **Password spraying** tries a small set of common passwords against many accounts.
- **Credential stuffing** uses credentials exposed in one breach against other services.
- **Rainbow tables** use precomputed hashes to speed up password cracking.
- **Hashing** protects stored passwords by converting them into a non-readable form.
- **Salt** adds unique data before hashing to make precomputed attacks harder.

**Storage and protection**

- Do not store passwords as plain text.
- Use strong password hashing for stored credentials.
- Add a unique **salt** to each password hash.
- Enforce strong password length and complexity.
- Limit repeated login attempts to slow automated attacks.

**Practical security impact**

- Weak rate limiting makes brute force attacks more effective.
- Reused passwords make credential stuffing more dangerous.
- Plain-text storage creates immediate exposure if a database is compromised.
- Strong password policy reduces the value of stolen credentials.

**Study focus**

- Match each attack type to its method.
- Recognize that brute force tests all possibilities.
- Recognize that dictionary attacks use likely guesses.
- Recognize that password spraying targets many accounts with few passwords.
- Recognize that rainbow tables depend on unsalted hashes.
