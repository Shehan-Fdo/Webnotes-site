## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Anchor tag basics**

- The `<a>` element creates a **hyperlink**.
- Use it to link to web pages, files, email addresses, phone numbers, or locations in the same page.
- Put the destination in the `href` attribute.
- Keep the visible text meaningful so it describes the link target.
- The `href` value can be a URL or another valid link target.

**How `href` works**

- `href` stands for the link destination.
- If `href` is missing, the anchor has no link target.
- In the DOM, `HTMLAnchorElement.href` returns an absolute URL when the value resolves as a valid URL.
- Setting the `href` property updates the element’s `href` attribute.
- An empty string is a valid relative URL and resolves to the document base URL.

**Common link targets**

- Use `https://...` for external web pages.
- Use `mailto:` for email links.
- Use `tel:` for phone links.
- Use fragment links like `#section-id` to jump within the same page.
- Use relative paths for files inside your site.

**Practical behavior**

- Pressing `Enter` activates a focused `<a>` element when `href` is present.
- The link text should indicate the destination.
- Browsers treat the anchor as a built-in navigation control.
- The element supports global HTML attributes.

**`<a>` versus `<link>`**

- `<a>` creates a clickable hyperlink in the page content.
- `<link>` connects the current document to an external resource.
- Use `<link>` mainly in the `<head>` for stylesheets and icons.
- Use `<a>` in the document body for user navigation.

**`rel` and related attributes**

- `rel` describes the relationship between linked resources and the current document.
- `rel` is valid on `<a>`, `<link>`, `<area>`, and `<form>`.
- Use `rel` values that match the link purpose.
- Keep relationship values semantically meaningful.

**Good authoring habits**

- Make link text clear and specific.
- Use the correct URL scheme for the destination.
- Check that the `href` value points to the intended resource.
- Prefer simple, readable link markup.
- Use links to improve navigation and user experience.
