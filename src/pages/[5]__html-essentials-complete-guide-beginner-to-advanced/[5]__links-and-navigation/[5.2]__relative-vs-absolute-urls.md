## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**URL basics**

- A **URL** (Uniform Resource Locator) is the address of a resource on the web.
- Browsers use URLs to retrieve pages, images, stylesheets, and other files.
- A URL can point to a resource that no longer exists or has moved.

**Absolute URLs**

- An **absolute URL** includes the full address.
- It typically includes the protocol, domain, path, and file name.
- Example form: `https://www.example.com/category/topic.html`
- Use an absolute URL when you want a complete, explicit destination.

**Relative URLs**

- A **relative URL** includes only part of the full address.
- It points to a resource based on the current document or base URL.
- Example form: `../images/pic.jpg`
- Relative references can include current or parent directory paths.
- Use a relative URL for links within the same site structure.

**How resolution works**

- The browser resolves a relative URL against a **base URL**.
- Resolution is not simple string concatenation.
- The `URL()` constructor can resolve a relative reference and a base URL to an absolute URL.
- If the first argument is relative, the base is required.
- If the first argument is already absolute, the base is ignored.

**`<base>` element**

- The `<base>` element sets the base URL for all relative URLs in a document.
- A document can contain only one `<base>` element.
- Scripts can read the used base URL with `Node.baseURI`.
- Changing the base affects how relative links resolve.

**Practical differences**

- Absolute URLs are explicit and self-contained.
- Relative URLs are shorter and depend on context.
- Relative paths can break if you move files or change folder depth.
- Absolute URLs reduce ambiguity when the destination must stay fixed.

**Linking and site structure**

- Internal linking often uses either absolute or relative URLs.
- Your choice affects maintainability and how links behave across environments.
- Use the same approach consistently within a project when possible.

**Common examples**

- Absolute link: `<a href="https://www.example.com/category/topic.html">Topic</a>`
- Relative link: `<a href="../category/topic.html">Topic</a>`
- Base URL example: `new URL("articles", "https://developer.mozilla.org/some/path")`

**Key takeaways**

- Absolute URLs include the full location.
- Relative URLs depend on a base path.
- The browser resolves relative URLs before use.
- The `<base>` element changes relative URL behavior for the whole document.
