## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Navigation menus in HTML**

- Navigation helps users find their way around a site.
- Use semantic HTML to describe menu structure clearly.
- The `nav` element identifies a major navigation region.
- A list-based structure improves readability and organization.
- A menu built with `<nav>`, `<ul>`, and `<li>` is clearer than nested generic `<div>` elements.
- HTML gives structure and meaning; it does not style the menu.

**Accessible menu markup**

- Accessible navigation supports keyboard and screen reader users.
- Keep menu items in a logical order.
- Use links for destinations and keep labels clear.
- Avoid over-engineering the markup when a simpler structure works.
- ARIA (Accessible Rich Internet Applications) attributes can help, but unnecessary ARIA can make navigation harder to use.

**Building a main navigation**

- Start with a basic setup.
- Add features layer by layer.
- Keep the implementation practical for most websites.
- Balance simplicity and functionality.
- Test the menu with keyboard navigation.

**Common structure pattern**

- Place the main menu inside `nav`.
- Group links in an unordered list with `ul`.
- Put each navigation item in an `li`.
- Use `a` elements for clickable destinations.
- Keep the structure easy to scan and maintain.

**Practical accessibility checks**

1. Verify that you can reach each menu item with `Tab`.
2. Verify that links activate with `Enter` or `Space` when appropriate.
3. Verify that screen reader output describes the menu clearly.
4. Verify that submenu behavior does not block keyboard access.

**Key takeaways**

- Semantic markup improves clarity.
- Lists make navigation easier to read and maintain.
- Accessibility matters for keyboard and screen reader users.
- Simple, well-structured navigation is often better than complex markup.
