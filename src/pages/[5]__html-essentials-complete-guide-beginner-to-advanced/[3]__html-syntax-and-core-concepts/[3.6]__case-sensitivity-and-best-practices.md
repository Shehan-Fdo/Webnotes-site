## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**HTML syntax basics**

- HTML stands for HyperText Markup Language.
- HTML uses **elements** to structure web content.
- An element typically includes an opening tag, content, and a closing tag.
- Some elements are **void elements** and do not need a closing tag.
- HTML documents need correct nesting to stay valid and predictable.
- The `<!doctype html>` declaration belongs at the top of a complete HTML document.

**Case sensitivity and consistency**

- Use consistent markup so browsers can interpret your page reliably.
- Keep tag and attribute spelling exact when you write HTML.
- Follow one formatting style across a project.
- Use the same naming and indentation patterns in repeated structures.
- Prefer clear, readable code over clever shortcuts.

**Semantic HTML**

- Use the right element for the right job.
- Choose semantic elements to describe content meaning, not just appearance.
- Semantic HTML helps browsers understand page structure.
- Semantic HTML also supports accessibility hooks built into the browser.
- Use plain language where possible.
- Define abbreviations when plain language is not enough.

**Accessibility best practices**

- Add `alt` text for images when the image conveys information.
- Write link text that describes the destination or action.
- Label form controls clearly.
- Use row and column headings for tables when needed.
- Keep source order logical.
- Support keyboard access for interactive content.

**Document structure**

- Start with a proper document skeleton.
- Place page content inside the `body` container.
- Use headings to organize sections in a logical order.
- Keep related content grouped together.
- Use attributes only when they add useful information or behavior.

**Writing and maintenance habits**

- Keep examples simple and focused.
- Use consistent indentation in code samples.
- Avoid unnecessary complexity in markup.
- Review pages for structure, readability, and accessibility.
- Treat HTML as the foundation for web content structure.
