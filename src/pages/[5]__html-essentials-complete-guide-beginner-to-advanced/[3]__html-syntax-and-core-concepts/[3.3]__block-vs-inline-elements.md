## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Block elements**

- Block-level content participates in **block layout**.
- A block element starts on a new line.
- It typically occupies the full horizontal space of its parent in horizontal writing modes.
- Block boxes stack vertically in normal flow.
- Vertical margins between adjacent block-level boxes can collapse.
- Common examples include `div`, `p`, `section`, and `article`.

**Inline elements**

- Inline-level content participates in **inline layout**.
- Inline elements flow within text instead of starting a new line.
- They are laid out in line boxes.
- Inline boxes align with surrounding content, often by text baselines.
- Common examples include `span`, links, and many text-level elements.
- Some replaced elements, such as `input`, can also behave as inline-level content.

**How they differ**

- Block elements create larger layout sections.
- Inline elements fit inside a line of text.
- Block elements usually take the available width.
- Inline elements usually take only the space their content needs.
- Block elements can contain block and inline content.
- Inline content is used for text-level structure and emphasis.

**CSS `display`**

- The `display` property controls whether an element behaves as a block or inline box.
- `display` also affects how an element’s children are laid out.
- Modern CSS defines the outer and inner display types separately.
- Use `display: block` or `display: inline` when you need to change default behavior.
- Layout behavior is now specified by CSS, even though HTML historically used block and inline categories.

**Practical layout notes**

- Use block elements for page sections and structural grouping.
- Use inline elements for text fragments inside a line.
- Choose the element type based on the layout behavior you need.
- Remember that normal flow places block boxes vertically and inline boxes within line boxes.

**Quick comparison**

| Aspect | Block | Inline |
|---|---|---|
| Line behavior | Starts on a new line | Stays in the current line |
| Width | Usually fills the container | Usually fits content width |
| Layout role | Section-level structure | Text-level content |
| Typical use | Layout containers | Words, links, emphasis |

**Key takeaway**

- Block and inline describe how content participates in layout.
- CSS `display` controls that behavior in modern styling.
- Pick the element type that matches the structure and flow you need.
