## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Nesting and hierarchy**

- HTML uses nested elements to show parent-child relationships in a document.
- Structure content with semantic elements such as `<main>`, `<section>`, `<article>`, `<header>`, `<nav>`, and `<footer>`.
- Use semantic elements in the right place instead of relying on `<div>` for every block container.
- Good structure improves accessibility and helps browsers and assistive technologies interpret content.
- Headings from `<h1>` to `<h6>` create a clear content hierarchy.
- Use heading levels in order to reflect the importance of sections.
- Keep nested markup readable so the document structure is easy to scan.

**Forms and nesting rules**

- Start a form with `<form>` and nest the form contents inside it.
- Use form structure to give meaning to different parts of the form.
- Proper form structure helps usability and accessibility.
- Do not nest a form inside another form.
- Nested forms can behave unpredictably.

**CSS nesting and selector relationships**

- CSS nesting lets you write styles that are easier to read and maintain.
- Nesting reduces repeated selectors and can reduce file size.
- CSS nesting is parsed by the browser, unlike precompiled preprocessors such as Sass.
- The `&` nesting selector makes the parent-child relationship explicit.
- Use `&` when joining selectors, combining selectors, or adding pseudo-classes.
- `&` can also help with backwards compatibility and readability.
- Without `&`, a nested rule targets child elements directly.
- The specificity of `&` is similar to `:is()` and uses the highest specificity in the selector list.

**Stacking context and visual hierarchy**

- Stacking context describes how elements layer along the z-axis.
- It controls the visual order of overlapping content.
- Elements inside one stacking context stack independently from elements outside it.
- Each stacking context is self-contained.
- After stacking, the whole element acts as one unit in its parent stacking context.
- Child elements are stacked by `z-index` within their own context.
- Nested stacking contexts only matter within their parent context.

**Practical structure habits**

- Plan the page structure before writing markup.
- Use nesting to represent real relationships in content.
- Keep containers semantic and purposeful.
- Use clean hierarchy to support accessibility and maintainability.
- Treat HTML structure as the foundation for later styling with CSS.
