## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**HTML comments overview**

- HTML comments let you add notes inside markup.
- Browsers ignore comments when rendering the page.
- Comments help you explain code or temporarily hide content.
- Use comments to make HTML easier to read and maintain.

**Comment syntax**

- Write an HTML comment with `<!--` at the start and `-->` at the end.
- Place the comment anywhere in the HTML document where markup is allowed.
- Keep the comment text inside the delimiters.
- Do not use comment syntax as visible page content.

**Practical uses**

- Add reminders for yourself or other developers.
- Mark sections of a page for easier navigation in the source.
- Temporarily disable content without deleting it.
- Keep notes close to the related HTML elements.

**Working with comments**

- Comments do not change the page layout.
- Comments do not display in the browser window.
- Use them for explanation, not for user-facing text.
- Keep them short and relevant to the surrounding code.

**Related note**

- The `<small>` element can represent side comments and small print in content.
- HTML comments are different from the `<small>` element because comments are not rendered.
- Use the right tool based on whether the note should appear on the page.

**Best practices**

- Write clear comments that explain intent.
- Avoid leaving outdated comments in finished code.
- Do not overuse comments for obvious markup.
- Keep comments accurate when you change the HTML.
