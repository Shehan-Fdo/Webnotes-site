## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Form basics overview**

- HTML forms are a main way users interact with a website.
- Forms collect data for a server or update the interface on the client side.
- The `<form>` element contains interactive controls for submission.
- Forms and buttons work alongside links as core interaction tools.
- Start with simple cases before moving to advanced form features.

**Core form elements**

- Use `<form>` to wrap the controls that belong together.
- Use `<input>` for many common data-entry controls.
- Use `<label>` to connect text with a control.
- Use `<textarea>` for multi-line text entry.
- Use `<button>` for clickable actions inside forms.
- Use `<fieldset>` and `<legend>` to group related controls.

**Common input types**

- `type="text"` collects plain text.
- `type="email"` supports email entry.
- `type="password"` hides typed characters.
- `type="checkbox"` supports on/off choices.
- `type="radio"` supports one choice from a set.
- `type="file"` lets users choose a file.
- `type="submit"` sends the form.
- `type="reset"` clears form values.
- `type="hidden"` stores data without showing a control.
- `type="image"` uses an image as a submit control.

**Important attributes**

- `name` identifies the submitted field.
- `value` sets the submitted value or button label.
- `id` links a control to a `<label>`.
- `required` makes a field mandatory.
- `minlength` and `maxlength` limit text length.
- `size` affects visible width for some inputs.
- `action` sets where the form sends data.
- `method` sets how the form submits data.

**Accessibility basics**

- Always pair controls with labels.
- Use correct semantics for each control.
- Group related controls when it improves clarity.
- Make the form easy to understand before styling it.
- Accessible forms help users and assistive technologies.

**Submission basics**

- A submit control triggers form submission.
- `input type="submit"` renders as a button.
- The button label comes from its `value` attribute.
- If `value` is missing, the browser supplies a default label.
- After validation, the form can be sent to the server.

**Practical form workflow**

1. Decide what data you need.
2. Choose the right form controls for each field.
3. Add labels and useful attributes.
4. Set `action` and `method` on the `<form>`.
5. Test submission and basic validation.

**What to remember**

- Forms are the standard HTML pattern for user input.
- Input type choice affects behavior and usability.
- Labels and semantics are essential, not optional.
- Validation and submission are part of the same workflow.
- Keep the first version simple and functional.
