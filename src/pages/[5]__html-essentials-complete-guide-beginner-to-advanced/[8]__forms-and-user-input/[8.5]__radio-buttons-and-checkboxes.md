## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Form control purpose**

- Radio buttons and checkboxes are common **form controls** for collecting user input.
- They help users make choices in web forms, surveys, quizzes, and settings.
- Forms and buttons are core tools for user interaction in HTML.

**Radio buttons**

- Radio buttons let users select one option from a predefined set.
- Only one radio button in a same-named group can be selected at a time.
- A selected radio input clears other radio inputs with the same `name`.
- Radio buttons work well when the choice must be exclusive.
- They appear as small circles and show selection when filled.
- Users can navigate radio groups with arrow keys and move on with `Tab`.

**Checkboxes**

- Checkboxes let users turn a value on or off.
- Users can select multiple checkboxes in the same form.
- A checkbox is useful when more than one option may apply.
- Checkboxes appear as boxes that are checked when activated.
- Users can operate a checkbox with the `Spacebar` key or the mouse.

**Selection behavior**

- Radio buttons enforce a single choice.
- Checkboxes allow multiple selections.
- Use radio buttons for one-of-many decisions.
- Use checkboxes for check-all-that-apply decisions.

**Labels and semantics**

- Associate each control with a specific `label`.
- Missing labels can make forms harder to understand.
- Correct grouping improves clarity and usability.
- Proper semantics help screen reader users and keyboard users.

**State and attributes**

- The `checked` property shows whether a radio button or checkbox is currently selected.
- The `checked` attribute sets the default checked state.
- For checkboxes, `checked` does not track later user changes.
- For radio groups, checking one input makes the others in the same group false.
- The `name` attribute matters for grouping radio buttons.

**Accessibility and usability**

- Accessible controls support keyboard navigation and screen readers.
- Clear option text reduces confusion.
- Poor grouping can frustrate users.
- Unclear choices can lead to bad form input.
- Good implementation improves form usability and data collection.

**Practical use cases**

- Use radio buttons for account settings with one required choice.
- Use radio buttons for surveys and quizzes with one answer.
- Use checkboxes for lists of items where multiple selections apply.
- Use checkboxes for optional features or preferences.

**Implementation notes**

- `<input type="radio">` and `<input type="checkbox">` are standard HTML input types.
- Browser support is broad across modern desktop and mobile browsers.
- If a browser does not support these types, it may fall back to a text field.
- Keep the control type aligned with the user decision you want to collect.
