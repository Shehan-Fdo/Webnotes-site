## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Form validation basics**

- HTML form validation checks user input before submission.
- Client-side validation gives immediate feedback and improves user experience.
- Validation helps ensure required fields are filled and values match expected formats.
- HTML5 adds built-in constraint validation and semantic input types.
- You can use attributes to enforce common rules without JavaScript.

**Common validation attributes**

- `required` makes a field mandatory.
- `minlength` sets the minimum text length for `<input>` and `<textarea>`.
- `maxlength` sets the maximum text length for `<input>` and `<textarea>`.
- `pattern` requires the value to match a regular expression.
- `type="email"` applies email-specific input handling and validation.
- `multiple` with `type="email"` allows a list of email addresses.

**`minlength` and `maxlength`**

- `minlength` accepts an integer value of `0` or higher.
- `maxlength` accepts an integer value of `0` or higher.
- Length is measured in UTF-16 code units.
- `minlength` does not imply `required`.
- An empty value can still submit when the field is not required.
- `maxlength` must be greater than or equal to `minlength`.
- A value below `minlength` triggers `tooShort`.
- A value above `maxlength` triggers constraint validation failure.

**`pattern` attribute**

- `pattern` works on text, `tel`, `email`, `url`, `password`, and `search` inputs.
- `pattern` uses a regular expression.
- The value must match the pattern to pass validation.
- A mismatch sets `patternMismatch` to `true`.
- Use `pattern` for format rules such as usernames or PIN codes.

**Constraint validation behavior**

- Constraint validation runs when the user changes the value.
- Browsers may show built-in error messages after submission fails.
- Validation attributes cover basic checks without JavaScript.
- More complex rules can use the Constraint Validation API.
- Client-side validation is an initial check, not a security boundary.

**Server-side validation**

- Always validate on the server side too.
- Client-side checks can be bypassed.
- Users can modify HTML in developer tools.
- Users can send crafted HTTP requests without using the form.
- Server validation protects data integrity and security.

**Practical usage**

- Combine `required`, `minlength`, `maxlength`, and `pattern` for stronger input rules.
- Use `type="email"` for email fields instead of plain text.
- Keep validation rules consistent with the data you expect.
- Use labels and correct semantics to support accessible forms.
