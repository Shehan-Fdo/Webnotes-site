## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Overview**

- The `<input>` element creates interactive controls for web forms.
- You use different `type` values to collect different kinds of user data.
- The same element can behave very differently based on its `type` attribute.
- HTML forms often use inputs for text, email, passwords, file uploads, and date selection.

**Core idea**

- The `type` attribute controls the inputâ€™s behavior and widget.
- If `type` is omitted, the browser uses the default input type.
- The `<input>` element supports many attribute combinations.
- Some input types have their own reference pages because their behavior differs.

**Common input types**

- `text` is the general-purpose input for short text entry.
- `email` is for email addresses.
- `radio` is for selecting one option from a group.
- Other common form controls include buttons and text areas, though they are separate controls.

**Email input**

- `type="email"` lets the user enter and edit an email address.
- With `multiple`, the field can accept a list of email addresses.
- The browser validates the value before form submission.
- Invalid email content triggers a validation error.
- `:valid` and `:invalid` can reflect the field state in CSS.
- Use `required` when the field must not be empty.

**Radio input**

- `type="radio"` is used in a radio group.
- Only one radio button in a group can be selected at a time.
- Radio buttons represent related options.
- Use the same `name` value to group related radio inputs.
- A `fieldset` and `legend` can provide structure and context.

**Attributes and validation**

- `name` identifies the submitted field data.
- `value` provides the data sent with the form.
- `required` makes the field mandatory.
- `minlength` and `maxlength` constrain text length.
- `size` affects the visible width of the field.
- `pattern` can enforce a custom format.

**Accessibility and semantics**

- Use `<label>` with `for` and `id` to connect text to the control.
- Correct labels improve usability and accessibility.
- Form controls help users interact with a site and submit information.
- Clear semantics matter when you build forms for real users.

**Practical notes**

- Native input types give you built-in browser behavior.
- Browser support is broad for common input types.
- Some parts of input behavior can vary by browser or device.
- Custom controls may be needed when native appearance does not match design needs.

**Takeaway**

- Choose the input type that matches the data you want.
- Rely on built-in validation where possible.
- Group related controls clearly.
- Keep forms simple, semantic, and accessible.
