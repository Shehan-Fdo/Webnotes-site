## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Sandboxing overview**

- **Sandboxing** limits what embedded content can do.
- It helps isolate potentially hostile content from the parent page.
- The HTML standard treats origins as a core security boundary.
- Different origins are isolated from each other to varying degrees.

**`<iframe>` sandboxing**

- An `<iframe>` creates a nested browsing context.
- The `sandbox` attribute adds extra restrictions to the embedded page.
- The `sandbox` property on `HTMLIFrameElement` reflects the attribute.
- You can inspect or update sandbox tokens with DOM methods like `add()`, `remove()`, `replace()`, and `toggle()`.
- Example tokens include `allow-same-origin` and `allow-scripts`.
- Use the `title` attribute on an `<iframe>` for accessibility context.

**Content Security Policy sandbox**

- The `Content-Security-Policy` (CSP) `sandbox` directive applies similar restrictions at the HTTP level.
- It can prevent popups.
- It can prevent plugin execution.
- It can prevent script execution.
- It can enforce a same-origin policy.
- It is not supported in the `<meta>` element.
- It is not supported in the `Content-Security-Policy-Report-Only` header.

**Same-origin policy**

- The same-origin policy restricts how a document or script from one origin interacts with another origin.
- An origin uses the protocol, host, and port.
- This policy reduces attack vectors by isolating potentially malicious documents.
- It helps stop a malicious site from reading data from another signed-in service or intranet site.

**Practical security guidance**

- Treat embedded third-party content as untrusted by default.
- Use sandboxing when you need to reduce the capabilities of embedded pages.
- Add only the permissions you actually need.
- Avoid broad access when a narrower sandbox setting works.
- Remember that security depends on origin boundaries as well as sandbox flags.

**Key takeaways**

- `sandbox` is a browser-level restriction mechanism for embedded content.
- CSP `sandbox` extends similar protection to a requested resource.
- The same-origin policy is a foundational web security control.
- Safer embedding means limiting script, popup, and cross-origin access.
