## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**What the `meta` element does**

- The `meta` element provides metadata that other elements do not cover.
- It belongs in the document `head`, which is not displayed in the browser page content.
- Browsers use head metadata to render the document correctly.
- Metadata can describe the page, its author, or important keywords.

**Common `meta` uses**

- `charset` declares the document character encoding.
- `name` provides document-level metadata for the whole page.
- `http-equiv` provides browser instructions that act like some HTTP headers.
- `itemprop` provides user-defined metadata.
- `viewport` helps control how the page scales on different devices.

**`charset` and document encoding**

- Use `charset` to tell the browser how the document is encoded.
- Set encoding early in the document head.
- This helps the browser interpret text correctly.

**`name="viewport"`**

- `viewport` defines viewport-related behavior through the `content` attribute.
- `content` uses comma-separated key-value pairs.
- `width=device-width` matches the viewport to the device screen in CSS pixels.
- `height` can also be set when needed.
- `viewport` supports responsive design behavior on mobile devices.

**`http-equiv`**

- `http-equiv` simulates selected HTTP header directives.
- `content` supplies the matching value.
- `Refresh` can tell the browser to reload after a delay.
- `content-type` declares the document media type and character encoding.
- `content-language` is deprecated; use `lang` on the opening `<html>` tag instead.

**Practical placement**

- Put metadata in the `<head>` section.
- Use `lang` on `<html>` for the document language.
- Use `meta` for information that should apply to the whole page.
- Use `meta` when the metadata cannot be represented by `<base>`, `<link>`, `<script>`, `<style>`, or `<title>`.

**Key takeaways**

- `meta` supports page metadata, browser behavior, and responsive hints.
- `charset` handles encoding.
- `viewport` supports device-aware layout.
- `http-equiv` covers limited browser directives.
