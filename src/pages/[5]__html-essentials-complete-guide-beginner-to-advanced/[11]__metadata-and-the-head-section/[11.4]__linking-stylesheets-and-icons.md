## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**HTML head purpose**

- The `head` element holds metadata that is not shown in the page body.
- It can include the page `title`, stylesheet links, icon links, and other metadata.
- Browsers use this information to render the document correctly.

**Linking stylesheets**

- Use the `<link>` element to connect the current document to an external resource.
- Place stylesheet links inside the `head`.
- Set `rel="stylesheet"` to identify the relationship.
- Set `href` to the stylesheet path, such as `main.css`.
- The `style` attribute on an element can still override external CSS rules.

**Linking icons**

- Use `<link>` to establish site icons for browsers and mobile platforms.
- A favicon is a small icon shown in places like tabs and bookmarks.
- Favicons help users recognize a site and support brand consistency.
- Common favicon formats include `GIF`, `PNG`, and `ICO`.
- A typical favicon size is `16x16` pixels, though icon sizes can vary.

**`rel` and `sizes` attributes**

- The `rel` attribute defines the relationship between the linked resource and the current document.
- On `<link>`, `rel` values depend on the resource type.
- For icons, `rel="icon"` identifies the resource as an icon.
- The `sizes` attribute lists icon dimensions as space-separated values.
- Use the `any` keyword when the icon size is not fixed.
- `sizes` is relevant for `rel="icon"` and similar icon types.

**Practical markup pattern**

- Put stylesheet and icon links in the `head`.
- Use `href` for the resource location.
- Use `rel` to describe the resource purpose.
- Add `sizes` when the icon file supports specific dimensions.

**Key takeaways**

- The `head` stores metadata, not visible page content.
- `<link>` connects the page to stylesheets and icons.
- `rel`, `href`, and `sizes` work together to describe external resources.
- Favicons improve recognition across browser UI surfaces.
