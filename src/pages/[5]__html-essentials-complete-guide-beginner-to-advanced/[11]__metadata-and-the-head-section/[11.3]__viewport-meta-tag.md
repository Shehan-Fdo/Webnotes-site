## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Viewport meta tag overview**

- The viewport is the browser area used to view a document.
- The `<meta name="viewport">` tag gives hints about viewport sizing and behavior.
- Place it in the `<head>` section because metadata belongs there.
- The tag does not render visible content on the page.
- It helps browsers handle mobile layouts more predictably.

**Why it matters**

- Mobile browsers may scale pages in ways that make layouts look too wide.
- `width=device-width` tells the page to match the device screen width in CSS pixels.
- `initial-scale=1.0` sets the initial zoom level.
- Without a proper viewport setting, pages can appear broken on small screens.
- The tag supports responsive design without changing visible content directly.

**Basic syntax**

- Use a `<meta>` element with `name="viewport"`.
- Put viewport settings in the `content` attribute.
- Separate multiple settings with commas.
- Common values include `width=device-width` and `initial-scale=1.0`.
- Keep the syntax inside the document `<head>`.

**Common behavior controls**

- `width` controls the minimum viewport width.
- `height` controls the minimum viewport height.
- `initial-scale` controls the starting zoom level.
- The `content` value uses key-value pairs.
- The browser uses these hints to size the page view.

**How it fits with HTML metadata**

- The HTML `<head>` stores metadata for the document.
- Metadata can include the page title, icons, stylesheets, and other page information.
- The `<meta>` element provides metadata that other elements do not cover.
- `name`-based metadata applies to the whole page.
- The viewport tag is one of the most important metadata entries for mobile behavior.

**Practical usage**

1. Open the `<head>` section of your HTML document.
2. Add a viewport meta tag.
3. Set `width=device-width`.
4. Set `initial-scale=1.0` if you want the page to start at normal zoom.
5. Test the page on a mobile screen or emulator.

**Notes and cautions**

> The viewport tag affects how the page is displayed, not the actual content structure.

- The tag is widely supported across modern browsers.
- The exact viewport behavior can vary by browser and device.
- Use conservative settings when you want predictable mobile rendering.
- Keep the tag simple unless you need a specific viewport behavior.

**Related metadata concepts**

- The `<meta>` element can also define character encoding.
- The `<meta>` element can provide document-level metadata for search engines.
- The `<head>` section is the correct place for metadata, not visible page content.
- The `lang` attribute on `<html>` also helps browsers interpret the document correctly.
