## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Web overview**

- The web is a system of connected documents and resources accessed through the internet.
- A **browser** is the client-side software you use to request and view web content.
- A **server** stores website files and sends them to clients on request.
- HTML defines the content and structure of a web page.
- CSS and JavaScript often accompany HTML to shape presentation and behavior.

**Clients and servers**

- Computers connected to the web act as clients or servers.
- The client usually initiates the request.
- The server responds with the requested resource.
- A web page is typically downloaded from the server to the client machine.
- A complete document can include text, layout instructions, images, videos, and scripts.

**HTTP basics**

- **HTTP** stands for Hypertext Transfer Protocol.
- HTTP is the protocol used to fetch resources such as HTML documents.
- HTTP is a client-server protocol.
- The client sends a **request**.
- The server sends a **response**.
- HTTP can also fetch images, videos, and other resources.
- HTTP can post form results back to a server.

**URLs and DNS**

- A **URL** identifies a resource on the web.
- A URL commonly includes a protocol, domain, subdomain, and path.
- The browser uses the URL to locate the target resource.
- **DNS** stands for Domain Name System.
- DNS helps translate human-readable domain names into network addresses.

**Web servers**

- A web server can mean hardware, software, or both.
- On the hardware side, it is a computer that stores website component files.
- Those files can include HTML documents, images, CSS stylesheets, and JavaScript files.
- On the software side, an HTTP server understands URLs and HTTP.
- The server delivers hosted content through the website’s domain name.

**What happens when you load a page**

- You enter a URL in the browser.
- The browser uses DNS to find the server.
- The browser sends an HTTP request.
- The server returns a response with the needed resources.
- The browser assembles the resources into a page you can view.

**Why this matters**

- Understanding the web helps you write better HTML.
- It also helps you understand how browsers receive and use content.
- It supports better decisions about structure, performance, and accessibility.
- HTML should express meaning in a machine-readable way when written properly.

**Key takeaways**

- Clients request resources.
- Servers store and deliver resources.
- HTTP moves messages between them.
- DNS maps names to addresses.
- HTML provides the structure browsers use to build pages.
