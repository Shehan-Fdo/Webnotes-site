## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Images in HTML**

- Use the `<img>` element to embed an image in a document.
- Provide `src` to point to the image file.
- Include `alt` text as a textual replacement for the image.
- Treat `alt` as mandatory for accessibility.
- Use `alt` to help screen readers describe the image.
- Use `alt` to show fallback text when the image does not load.
- Keep the description aligned with the image’s purpose.

**Accessible image text**

- Write `alt` text that matches the image’s function.
- Use concise wording for informative images.
- Leave out unnecessary phrases like “image of” when the meaning is already clear.
- Use plain language where possible.
- Define abbreviations if you must use them in text alternatives.

**Semantic HTML and accessibility**

- Use semantic HTML for the right element and purpose.
- Choose elements that give browsers built-in accessibility hooks.
- Keep source order logical.
- Use good link text.
- Add form labels where relevant.
- Use table row and column headings for data tables.
- Scope table headers when needed.

**Audio and video media**

- Integrate audio and video with HTML media elements.
- Add captions for accessible video content.
- Use transcripts when users need a text version of spoken content.
- Distinguish between captions and subtitles when planning media accessibility.
- Treat synchronized media as content that combines audio and video.

**Practical accessibility habits**

- Use simple, clear language.
- Avoid slang and unexplained abbreviations.
- Support keyboard accessibility.
- Make content understandable without relying on visuals alone.
- Check that media still communicates meaning when images fail to load.

**Key takeaways**

- HTML supports accessible media when you use the right element for the job.
- `alt` text gives images a meaningful fallback.
- Captions and transcripts improve access to audio and video.
- Semantic structure strengthens accessibility across the page.
