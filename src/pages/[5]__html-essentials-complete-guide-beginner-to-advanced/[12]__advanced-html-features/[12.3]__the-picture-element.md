## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**What the `picture` element does**

- The `picture` element provides alternative image versions for different display or device scenarios.
- It contains zero or more `source` elements and one `img` element.
- The browser evaluates each `source` and picks the best match.
- If no `source` matches, the browser uses the `img` `src` value.
- The selected image displays in the space occupied by the `img` element.

**How the selection works**

- The browser checks each `source` in order.
- It uses `srcset`, `media`, and `type` to choose a compatible image.
- You use `picture` to support responsive images across different screen sizes and capabilities.
- This approach helps improve performance on devices with different layouts and resolutions.

**Required fallback behavior**

- The `img` element acts as the fallback image.
- You should always include an `img` inside `picture`.
- At least one of `src` or `srcset` must be provided on `img`.
- The `alt` attribute on `img` is mandatory and provides text for accessibility.
- `alt` text also appears if the image fails to load.

**Related image elements**

- The `img` element embeds an image into the document.
- The `source` element supplies media resources for `picture`, `audio`, and `video`.
- `source` is a void element, so it has no closing tag.
- `source` commonly helps provide multiple file formats for compatibility.

**Practical use cases**

- Use `picture` when you need different image versions for different conditions.
- Use `source` to target layout changes such as orientation.
- Use `img` to guarantee a usable default image.
- Use responsive images to better fit varying screen sizes and device features.

**Implementation notes**

- Keep image files small when possible.
- Use `width` and `height` to reduce layout shifts after loading.
- Check licensing before using media assets in projects.
- `picture` is widely available across modern browsers.
