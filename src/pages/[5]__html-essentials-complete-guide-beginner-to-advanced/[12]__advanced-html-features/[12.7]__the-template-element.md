## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**What the `template` element does**

- The `<template>` element holds HTML fragments for later use.
- Its contents are not rendered when the page loads.
- You can use it to avoid repeating the same markup structure.
- It works well with JavaScript-driven insertion.

**How it behaves in the DOM**

- The element exposes its contents as a `DocumentFragment`.
- The `content` property returns the template contents.
- The template content lives in a separate document context.
- Use `document.importNode()` when you need to clone content into the current document context.

**Common use cases**

- Reuse markup for repeated page structures.
- Populate the shadow DOM of a web component.
- Store fragments that you append later with JavaScript.
- Support component-based patterns with templates and slots.

**Shadow DOM and related attributes**

- The `shadowrootmode` attribute creates a shadow root for the parent element.
- `open` exposes the internal shadow root DOM to JavaScript.
- `closed` hides the internal shadow root DOM from JavaScript.
- The parser can create a `ShadowRoot` or an `HTMLTemplateElement` depending on the `<template>` attributes.

**JavaScript interaction**

- You can select the template with `document.getElementById()`.
- You can read the fragment from `template.content`.
- You can append the fragment to the document after cloning or importing it.
- `cloneNode()` copies a node in the node’s own document context.
- `importNode()` copies a node in the calling document context.

**Practical notes**

- The `template` element is widely available across browsers.
- It has been available since November 2015.
- Some related shadow DOM behavior may vary by browser support.
- Web components commonly combine custom elements, shadow DOM, and HTML templates.
