## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Custom attributes in HTML**

- HTML elements use **attributes** to add extra values that configure behavior or store information.
- `data-*` attributes let you attach extra information to standard HTML elements.
- Use custom data when the information has no direct visual meaning.
- Keep the attribute name after `data-` valid for HTML naming patterns.
- `data-*` attributes are commonly called **custom data attributes**.

**Practical uses**

- Store element-specific metadata on items such as articles or list entries.
- Pass values from HTML to JavaScript without extra hidden markup.
- Keep semantic HTML clean while still associating data with elements.
- Support interactive features that read element data at runtime.
- Use custom data when you need script-accessible information tied to one element.

**Example pattern**

- Add custom data directly in markup, such as `data-columns="3"` or `data-id="10784"`.
- Use the attribute on standard elements like `<article>` or `<li>`.
- Keep values short and meaningful.
- Use the same pattern across repeated items when the data represents a shared field.

**JavaScript access**

- Read custom data with `getAttribute()` when you need the full attribute name.
- Use `HTMLElement.dataset` for direct access to `data-*` values.
- Treat the stored value as element-associated data, not visible content.
- Use script access when the page needs filtering, labeling, or behavior based on the attribute value.

**Global attribute context**

- Some attributes are **global attributes**, so they can appear on all HTML elements.
- Global attributes may have no effect on some elements.
- `accesskey` is a global attribute that provides a keyboard shortcut hint.
- `aria-*` attributes support accessibility-related states and properties.
- Deprecated global attributes such as `xml:lang` and `xml:base` remain for compatibility.

**Practical guidance**

- Prefer `data-*` for custom information that scripts need to read.
- Avoid non-standard attributes when a standard custom data attribute works.
- Keep custom data separate from content that users must see.
- Use attributes to describe or enhance elements, not to replace visible text.
- Choose the simplest attribute pattern that supports your use case.
