## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Responsive images**

- `srcset` lets you provide one or more image candidates.
- Each candidate can use a URL plus a width or pixel density descriptor.
- The browser uses these candidates to pick an image for the current rendering situation.
- `srcset` reflects the `<img>` element `srcset` content attribute.
- `srcset` works with `sizes` to support responsive image selection.

**The `sizes` attribute**

- `sizes` tells the browser the layout width of the image for each media condition.
- `sizes` can also use `auto` for lazy-loaded images.
- `sizes` helps the browser choose among images listed in `srcset`.
- `sizes` can support images with different orientations or aspect ratios.
- `sizes` reflects the `<img>` element `sizes` content attribute.
- `sizes` is only present when `srcset` uses width descriptors.

**How the browser uses them**

- The browser compares the layout context with the candidate list.
- The browser selects the image that best matches the rendering situation.
- This approach helps improve performance across devices.
- This approach supports devices with different screen sizes and resolutions.

**Practical usage**

- Use `srcset` when you want multiple image options.
- Use `sizes` when image display width changes with layout.
- Use both together for responsive websites.
- Keep the fallback `src` for the default image source.

**Key terms**

- **Image candidate**: one possible image resource in `srcset`.
- **Width descriptor**: a size marker such as `360w`.
- **Pixel density descriptor**: a density marker that helps with high-resolution displays.
- **Media query**: a condition that helps describe layout width in `sizes`.

**Notes**

- `srcset` and `sizes` are widely available browser features.
- They are part of the HTML image workflow for responsive design.
- Use them to match image choice to layout size instead of serving one fixed file.
