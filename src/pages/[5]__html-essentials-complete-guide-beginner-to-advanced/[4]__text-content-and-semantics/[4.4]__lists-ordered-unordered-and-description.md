## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**List types in HTML**

- HTML supports three list types: **unordered**, **ordered**, and **description**.
- Lists organize related content in the document body.
- You can nest lists inside other lists.

**Unordered lists**

- Use `<ul>` for items where order does not matter.
- Use `<li>` for each list item.
- Browsers typically render unordered lists as bulleted lists.
- Nested unordered lists are valid.

**Ordered lists**

- Use `<ol>` for items that follow a sequence.
- Use `<li>` for each step or item.
- Browsers typically render ordered lists as numbered lists.
- Ordered lists fit directions, recipes, and other step-based content.

**Description lists**

- Use `<dl>` for grouped terms and descriptions.
- Use `<dt>` for each term.
- Use `<dd>` for the matching description.
- Description lists work well for glossaries and key-value pairs.

**Core list syntax**

- `<li>` is the item element used in ordered and unordered lists.
- `<ul>` and `<ol>` group the `<li>` items.
- `<dl>` groups term-description pairs instead of `<li>` items.
- Introductory text can appear before a description list.

**Common usage**

- Use unordered lists for shopping items or other unordered collections.
- Use ordered lists for instructions that must stay in sequence.
- Use description lists for definitions and metadata.
- Lists also support broader content structures, such as navigation menus.

**Attributes and notes**

- `ol` supports `start` to begin counting from a chosen number.
- `ol` supports `reversed` to count from high to low.
- `compact` is deprecated and browser-specific.
- `type` on `ul` is deprecated; use CSS for styling instead.

**Practical reminders**

- Keep list items short and focused.
- Choose the list type based on meaning, not appearance.
- Use CSS when you need visual styling changes.
- Use nested lists only when the content structure requires it.
