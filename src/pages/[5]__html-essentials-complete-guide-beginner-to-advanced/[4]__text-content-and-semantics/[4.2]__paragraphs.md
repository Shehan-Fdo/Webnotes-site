## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Paragraph basics**

- The `<p>` element represents a **paragraph**.
- Use paragraphs to group related text into readable blocks.
- HTML paragraphs are **block-level** elements.
- Browsers usually separate paragraphs with blank lines or spacing.
- Paragraphs can also group related content such as images or form fields.

**How paragraphs behave**

- A `<p>` element belongs to **flow content**.
- Its content model is **phrasing content**.
- The end tag can be omitted in some cases when another block-level element starts.
- The element closes automatically if a block-level element appears before `</p>`.
- The `align` attribute on `<p>` is **deprecated** and should not be used.

**Writing semantic content**

- Use `<p>` for real text structure, not for visual spacing.
- Break content into paragraphs to make the page easier to read.
- Use semantic HTML to describe meaning, not just appearance.
- Keep heading levels logical when paragraphs sit under headings.
- Use CSS for font size and spacing instead of changing structure for presentation.

**Practical usage**

- Write each paragraph as one related idea.
- Keep paragraph text concise when possible.
- Use separate `<p>` elements for separate thoughts.
- Avoid using obsolete alignment markup.
- Treat paragraphs as part of the document structure, not decoration.

**Key points to remember**

- `<p>` is the standard HTML paragraph element.
- Paragraphs help browsers and users understand content structure.
- The element supports global attributes.
- Paragraphs are widely supported across browsers.
- Good paragraph structure improves readability and semantic clarity.
