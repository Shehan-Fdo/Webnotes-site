## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Quotes in HTML**

- HTML uses semantic elements to mark up quotations and citations.
- `blockquote` is for extended quotations.
- `q` is for inline quotations.
- `cite` identifies the source of a quotation or a cited work.
- Use quotation elements to keep content structure clear and meaningful.

**`blockquote` for long quotations**

- `blockquote` indicates an extended quotation.
- Browsers usually render it with indentation.
- You can include a `cite` attribute with a source URL.
- You can place paragraphs and other block content inside it.
- Use it when the quoted text stands apart from surrounding content.

**`q` for inline quotations**

- `q` marks a short quotation inside a sentence.
- It fits within normal text flow.
- Use it when the quote does not need separate block formatting.
- Keep the surrounding sentence readable and concise.

**`cite` for source attribution**

- `cite` gives a text representation of a source.
- Use it with a quoted work, author, or title when attribution is needed.
- It helps connect the quote to its origin.
- Use it alongside quotation elements for clearer referencing.

**Practical usage**

- Choose `blockquote` for longer quoted passages.
- Choose `q` for short quoted phrases inside a paragraph.
- Add `cite` when you want to identify the source.
- Keep the markup semantic instead of relying on visual styling alone.

**Key points**

- `blockquote` is the block-level quotation element.
- `q` is the inline quotation element.
- `cite` supports source naming and attribution.
- Semantic quotation markup improves clarity and structure.
