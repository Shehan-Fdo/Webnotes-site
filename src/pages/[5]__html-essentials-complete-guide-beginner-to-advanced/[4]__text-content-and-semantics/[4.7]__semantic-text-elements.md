## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Overview**

- HTML defines content structure and meaning in a machine-readable way.
- Semantic text elements help browsers, search engines, and assistive technologies interpret content.
- Use semantic markup for meaning, not just visual styling.

**Core semantic text elements**

- `<strong>` indicates **strong importance**, seriousness, or urgency.
- Browsers typically render `<strong>` as bold text.
- Use `<strong>` for warnings or key points, not for styling alone.
- `<mark>` highlights text for reference or relevance in context.
- Use `<mark>` when the marked text matters to the surrounding passage.
- `<small>` represents side comments and small print, such as copyright or legal text.
- `<small>` changes presentation by default, but its meaning is about side information.
- `<sup>` displays inline text as superscript for typographical reasons.
- Use `<sup>` for conventions like exponents or raised notation.
- Do not use `<sup>` just to create a visual effect.
- Semantic elements can improve accessibility and search engine optimization.

**Usage guidance**

- Choose the element that matches the meaning of the text.
- Keep visual styling separate when CSS can handle the presentation.
- Use `<strong>` for content with real importance, not generic bolding.
- Use `<small>` for fine print or side notes that are still part of the document.
- Use `<mark>` to show relevant text in context, such as search matches.
- Use `<sup>` when the text position must follow typographic standards.

**Practical distinctions**

- `<strong>` signals importance.
- `<mark>` signals relevance or highlight.
- `<small>` signals side comments or legal text.
- `<sup>` signals superscript formatting for typographic meaning.

**Best practices**

- Write semantic HTML so the document meaning stays clear.
- Prefer CSS for pure visual changes.
- Use inline semantic elements only when the text meaning changes.
- Keep markup minimal and accurate.
