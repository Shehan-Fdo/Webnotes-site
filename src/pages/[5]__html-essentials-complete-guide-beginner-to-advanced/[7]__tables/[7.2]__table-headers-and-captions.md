## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Table purpose**

- Use tables for **tabular data**.
- Use rows and columns to show related values.
- Do not use tables for layout.
- A table helps users look up values quickly.

**Table headers**

- Use `<th>` for header cells.
- Use `<td>` for regular data cells.
- Place `<th>` inside a `<tr>` like other table cells.
- Use headers to show the meaning of rows or columns.
- Define the header relationship with `scope` when needed.
- Use `scope="col"` for column headers.
- Use `scope="row"` for row headers.
- Use `scope="rowgroup"` when one header applies to a grouped section.

**Captions**

- Use `<caption>` to name the table.
- Put the caption inside the `<table>` element.
- Treat the caption as the table’s accessible name or description.
- Keep the caption concise and specific.
- Do not repeat the same text in a separate summary if you add one.

**Table structure for accessibility**

- Use `<thead>` for header rows.
- Use `<tbody>` for the main data rows.
- Use `<tfoot>` for footer or summary rows.
- Grouping sections helps users understand complex tables.
- Add `colspan` or `rowspan` only when cells must span multiple columns or rows.
- Use `id` and `headers` when you need stronger cell-to-header associations.

**Practical markup pattern**

- Start with `<table>`.
- Add `<caption>` near the top of the table.
- Add a header row with `<th>` cells.
- Fill the body with `<td>` cells and row headers where needed.
- Add sectioning elements when the table needs clearer structure.

**Accessibility reminders**

- Make the table easy to scan visually.
- Make header relationships clear for assistive technologies.
- Use captions to help users identify the table’s purpose.
- Use scoped headers to reduce ambiguity in complex data.
- Keep the structure aligned with the data, not the page layout.
