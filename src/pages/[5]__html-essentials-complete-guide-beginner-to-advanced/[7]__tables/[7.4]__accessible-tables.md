## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Accessible table basics**

- Use tables for **data tables**, not layout.
- Mark up tabular data with `<table>`, `<tr>`, `<th>`, and `<td>`.
- Give users a programmatic way to match cells with row and column headers.
- Screen readers rely on correct structure to announce table relationships.

**Captions and table names**

- Add a `<caption>` to provide a table title or accessible name.
- Place `<caption>` as the first child after the opening `<table>` tag.
- Use the caption to describe the table’s content briefly.
- Keep the caption and any longer description from repeating the same text.

**Table sections**

- Group header rows in `<thead>`.
- Group main data rows in `<tbody>`.
- Group totals or summary rows in `<tfoot>`.
- Use these sections to make the table structure clearer for assistive technology.

**Header cells and scope**

- Use `<th>` for header cells.
- Use `scope="col"` for column headers.
- Use `scope="row"` for row headers.
- Add `scope` so assistive technology can associate headers with the right cells.

**Complex table associations**

- Use `id` and `headers` when simple `scope` is not enough.
- Use these attributes to create stronger links between specific headers and cells.
- Add a summary or description for complex tables when needed.
- Keep the summary focused on structure, not a duplicate of the caption.

**Practical accessibility checks**

- Make sure each data table has clear headers.
- Make sure the table title helps users understand the content quickly.
- Check that row and column relationships are obvious without visual scanning.
- Verify that the table still makes sense when read cell by cell.

**Example structure**

- A common pattern is `<caption>`, then `<thead>`, then `<tbody>`, then `<tfoot>`.
- Put column headers in the table head.
- Put row labels in the first cell of each data row when appropriate.
- Use semantic structure instead of visual formatting alone.
