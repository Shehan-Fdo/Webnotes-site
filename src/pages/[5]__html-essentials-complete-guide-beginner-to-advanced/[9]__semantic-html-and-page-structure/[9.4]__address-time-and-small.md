## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Semantic purpose**

- HTML defines the structure and meaning of web content.
- Semantic elements help browsers and assistive tools understand content.
- Use these elements when you want the markup to describe the content clearly.

**`<address>` element**

- Use `<address>` for contact information for a person, people, or an organization.
- Include the name of the person or organization the contact details refer to.
- Put appropriate contact details inside it, such as an email address, phone number, URL, physical address, or social media handle.
- Keep it focused on contact information, not general postal content.
- Use links like `mailto:` and `tel:` when they fit the contact details.

**`<time>` element**

- Use `<time>` for dates or times that benefit from machine-readable meaning.
- The `datetime` attribute provides a machine-readable form of the date or time value.
- Use it when you want browsers or software to interpret the value reliably.
- Keep the visible text readable for people.
- Match the displayed value with the `datetime` value.

**`<small>` element**

- Use `<small>` for side comments, disclaimers, or fine print.
- Use it for content that is less prominent than the main text.
- Keep the meaning intact; do not use it only for visual styling.
- Use it when the content is supplementary rather than central.

**Practical usage**

- Use semantic HTML to improve accessibility and document clarity.
- Choose the element that matches the meaning of the content.
- Keep contact details, time values, and fine print separate when they serve different purposes.
- Use inline elements like these where the content fits naturally in the flow.

**Related HTML context**

- HTML uses tags to mark up content for display in a browser.
- Elements such as links, text, and metadata work together to structure a page.
- Good structure supports search engines, browsers, and accessibility tools.
