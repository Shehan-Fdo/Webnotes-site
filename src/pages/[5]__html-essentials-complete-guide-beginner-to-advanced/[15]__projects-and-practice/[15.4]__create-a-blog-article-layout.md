## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Blog article layout basics**

- HTML provides the structure for web content.
- Use semantic elements to describe page regions clearly.
- A blog article layout usually includes a header, navigation, main content, article content, sidebar, and footer.
- Semantic structure helps browsers, search engines, and assistive technologies understand the page.
- Avoid using `<div>` for every block when a semantic element fits better.

**Core semantic elements**

- `<header>` holds introductory content for the page or article.
- `<nav>` contains primary navigation links.
- `<main>` wraps the main content of the page.
- `<section>` groups related content within a page or article.
- `<article>` represents a self-contained piece of content.
- `<aside>` holds supporting or related content.
- `<footer>` contains closing content for the page or article.

**Why semantic layout matters**

- Semantic HTML improves accessibility.
- Semantic HTML supports maintainability.
- Semantic HTML gives search engines clearer content meaning.
- Semantic HTML makes the document easier to read and edit.

**Planning the page structure**

- Start by identifying the major content areas.
- Place the main article inside `<main>`.
- Use `<article>` for the blog post itself.
- Use `<section>` for grouped parts such as related topics or content blocks.
- Use `<aside>` for supporting information like related links or extra notes.

**Building the layout**

1. Create the document skeleton with `<!DOCTYPE html>`, `<html>`, `<head>`, and `<body>`.
2. Add a `<header>` for the site or article introduction.
3. Add `<nav>` for the main menu if the page needs navigation.
4. Place the blog post inside `<main>`.
5. Wrap the post content in `<article>`.
6. Add `<section>` elements for logical content groups.
7. Add `<aside>` for secondary content.
8. Finish with a `<footer>` for closing information.

**Practical content choices**

- Use headings and paragraphs to organize the article body.
- Use lists when you need grouped points.
- Use tables only when you need side-by-side data.
- Use images and other media where they support the article.
- Keep the structure consistent across pages for easier reuse.

**Good layout habits**

- Use semantic elements in the right place.
- Keep the markup clean and readable.
- Choose elements based on meaning, not appearance.
- Use a text editor and browser to test the structure as you build it.
- Review the page in the browser to confirm the content order makes sense.

**Key takeaway**

- A blog article layout works best when the page structure is meaningful.
- Semantic HTML gives the page a clear hierarchy.
- A well-planned layout improves readability and accessibility.
