## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Multi-page website basics**

- HTML structures web content and lets you organize text, images, and interactive elements.
- A multi-page website uses separate HTML files that users move between through links.
- Links, or **hyperlinks**, are the core mechanism for web navigation.
- The `<a>` tag creates a link to another page, an external site, a section, or another resource.
- Use clear link text so users understand where the link goes.

**Linking local pages**

- Link pages in the same project with the `<a>` tag and an `href` value that points to a relative file path.
- Relative paths help you connect files inside the same website directory.
- Use `href` with a path such as `/about.html` when linking to a local page.
- A relative reference is resolved against a base URL, so it is not just string concatenation.
- Use `.` and `..` path syntax when you need to move within folder levels.

**Navigation structure**

- Build navigation with semantic HTML so the site is easier to understand and use.
- Keep the main navigation simple and organized.
- Add features layer by layer instead of over-engineering the markup.
- Use accessible link text to support screen reader users.
- Use ARIA attributes carefully, because unnecessary use can hurt usability.

**External resources and document links**

- Use the `<link>` element in the `<head>` to connect external resources such as stylesheets.
- The `rel` attribute describes the relationship between the document and the resource.
- The `<link>` element is also used for site icons and app icons.
- Use the `href` attribute to point to the external file or resource.

**Practical setup**

- Start with a basic development environment that includes a text editor and a web browser.
- Make sure character encoding is set correctly before you build pages.
- Use consistent file names and folder structure so links stay predictable.
- Test each link after you create it to confirm navigation works.

**Common link behavior**

- Links usually appear underlined and blue unless CSS changes the style.
- A link can wrap text or an image.
- Link states such as `:hover`, `:focus`, `:visited`, and `:active` help define interaction feedback.
- Use block or inline links as needed for the layout.

**Procedure**

1. Create separate HTML files for each page in your site.
2. Add an `<a>` tag on each page that points to the next page with a relative `href`.
3. Place shared resources like stylesheets in the `<head>` with `<link>`.
4. Check every path in the browser and fix broken links.
5. Refine the navigation so users can move through the site easily.
