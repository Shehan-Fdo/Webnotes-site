## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**HTML document basics**

- HTML stands for HyperText Markup Language.
- HTML defines the **meaning** and structure of web content.
- Browsers use HTML to understand how to display page content.
- HTML uses elements wrapped in tags such as `<head>`, `<title>`, and `<body>`.
- HTML is the most basic building block of the Web.

**Core document skeleton**

- Start documents with `<!doctype html>`.
- Place the `<html>` root element after the doctype.
- Nest the `<head>` and `<body>` inside `<html>`.
- Keep the document structure correctly nested.
- Use the `<body>` for visible page content.
- Use the `<head>` for document metadata.

**The head section**

- The `<head>` element contains machine-readable information.
- Put the page `<title>` in the head.
- Include metadata in the head for search engines and social media.
- Add icons and browser-related settings in the head.
- Add stylesheets and scripts in the head when needed.
- Use only one `<head>` element per HTML document.

**Semantic page structure**

- Use semantic elements to describe page regions.
- Common structural elements include `<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, and `<footer>`.
- Use `<main>` for the main content area.
- Use `<nav>` for navigation links.
- Use `<header>` for introductory or top-level page content.
- Use `<footer>` for closing or supporting page content.
- Prefer semantic elements over using `<div>` for every block container.

**Why structure matters**

- Semantic structure improves accessibility.
- Semantic structure helps search engines understand content.
- Semantic structure helps browsers use built-in features more effectively.
- The head is not visible to the visitor, but it still affects page behavior and presentation.
- The doctype helps prevent browsers from switching into quirks mode.

**Practical document planning**

- Plan the page before writing the HTML.
- Identify the major regions of the page first.
- Map content into semantic containers.
- Keep the structure simple and consistent.
- Use the document skeleton as the base for most web pages.
