## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Document structure**

- The `<html>` element is the root element of an HTML document.
- All other elements must be descendants of `<html>`.
- Use only one `<html>` element in a document.
- The `<head>` element contains machine-readable **metadata**.
- The `<body>` element represents the visible content of the document.
- Use only one `<head>` element and one `<body>` element in a document.

**What belongs in `<head>`**

- Put the document `<title>` in `<head>`.
- Put linked stylesheets and scripts in `<head>` when they need to load as metadata.
- Use `<meta charset="UTF-8">` to define character encoding.
- Use `<meta name="viewport" content="width=device-width">` for responsive layout support.
- Keep `<head>` focused on information for software, not page content.

**What belongs in `<body>`**

- Put the content users read and interact with in `<body>`.
- Use `<body>` for text, images, and other visible page elements.
- Keep page content out of `<head>`.
- The `<body>` element can include global attributes.
- The `<body>` element can also include event attributes.

**Headings and page hierarchy**

- Use `<h1>` to `<h6>` for six levels of section headings.
- `<h1>` is the highest section level.
- `<h6>` is the lowest section level.
- Headings create block-level boxes by default.
- Headings start on a new line and take the full available width.
- Use headings to structure content, not to resize text.

**Practical page skeleton**

1. Start with `<!doctype html>` when creating a basic HTML document.
2. Add the `<html>` root element.
3. Place `<head>` before `<body>`.
4. Add metadata such as `<meta charset="UTF-8">` and `<title>`.
5. Place visible page content inside `<body>`.

**Key takeaways**

- `<html>` defines the document root.
- `<head>` stores metadata for browsers and other software.
- `<body>` stores the content users see.
- Headings help organize the document into readable sections.
