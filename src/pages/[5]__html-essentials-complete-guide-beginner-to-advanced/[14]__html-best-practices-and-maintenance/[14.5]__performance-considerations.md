## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Performance goals**

- HTML is the foundation for structuring web content.
- Keep HTML fast to preserve a good first response and smooth rendering.
- Optimize only what affects your page in practice.

**Why HTML performance matters**

- A page starts with an HTML request.
- Faster HTML delivery helps reduce **Time to First Byte** (`TTFB`).
- High `TTFB` makes it harder to reach good loading metrics.
- HTML should stay fast and accessible by default.
- Large embeds or blocking scripts can slow critical page elements.

**Practical optimization focus**

- Identify what actually needs optimization before you change code.
- Avoid applying every technique everywhere.
- Use targeted improvements for the parts that affect load time.
- Keep critical content available early.
- Reduce anything that blocks rendering.

**Lazy loading**

- Treat non-critical resources as **lazy loaded**.
- Load them only when needed.
- Use it to shorten the **Critical Rendering Path**.
- Trigger loading on user actions such as scrolling or navigation.
- Delay cart or similar section resources until the user reaches them.

**Common HTML performance risks**

- Large `video` embeds can increase file size.
- JavaScript parsing can block rendering.
- Heavy images can slow the initial page load.
- Extra non-critical assets can delay first render.

**Optimization mindset**

- Preserve HTML’s fast and accessible nature.
- Focus on the resources that affect the first view.
- Use performance techniques where they create measurable value.
- Keep the page responsive for users and devices.
