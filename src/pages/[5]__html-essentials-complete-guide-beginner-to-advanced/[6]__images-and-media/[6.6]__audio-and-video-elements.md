## **Course:** HTML Essentials: Complete Guide (Beginner to Advanced)

**Audio and video in HTML**

- Use `<audio>` to embed sound content in a document.
- Use `<video>` to embed a media player for video playback.
- Prefer native HTML elements over older plugin-based approaches.
- Use JavaScript APIs when you need programmatic control.

**Core element behavior**

- Add `controls` when you want the browser to show playback controls.
- Use `autoplay` only when the experience truly needs it.
- Use `loop` when media should repeat automatically.
- Use `muted` when you need silent playback by default.

**Using multiple media sources**

- Provide one or more sources with `src` or `<source>`.
- Let the browser choose the most suitable source.
- Use `<source>` inside `<audio>` or `<video>` for fallback options.
- Use `type` to describe the media format when needed.

**Audio element notes**

- `<audio>` supports embedded sound content.
- `<audio>` can also work as a destination for streamed media.
- The `Audio()` constructor creates a new `HTMLAudioElement`.
- `new Audio(url)` can start loading the file asynchronously.

**Video element notes**

- `<video>` supports embedded video playback in the document.
- `<video>` can also be used for audio content, but `<audio>` is often more appropriate.
- You can set `width` and `height` to control display size.
- You can combine multiple `<source>` entries for format fallback.

**Captions and subtitles**

- Add text tracks when you need captions or subtitles.
- Use captions to improve accessibility for video content.
- Keep media alternatives available when possible.

**Practical usage**

1. Choose `<audio>` or `<video>` based on the media type.
2. Add `controls` if the user should interact with playback.
3. Provide fallback sources with `<source>` when multiple formats exist.
4. Test the media in the browser you expect users to use.

**Key takeaways**

- Native media elements are the standard HTML solution.
- Multiple sources improve compatibility across browsers.
- Playback attributes shape the user experience.
- Captions and subtitles support accessibility.
