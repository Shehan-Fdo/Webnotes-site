**Steganography overview**

Steganography is the practice of hiding data inside another file, message, or medium so the existence of the hidden content is not obvious. Unlike encryption, which makes data unreadable, steganography aims to make data invisible to casual inspection. In security work, you use it to understand how attackers can conceal payloads, exfiltrate data, or embed command-and-control artifacts inside ordinary-looking content.

A useful way to think about it is:

- **Encryption** hides meaning.
- **Steganography** hides existence.
- They are often combined to increase attacker stealth.

Common goals include:

- Concealing malware configuration or payload fragments.
- Hiding stolen data inside images, audio, or documents.
- Embedding covert communication in network traffic or media files.
- Bypassing simple content filters that inspect only file type or visible content.

**Core concepts and terminology**

Steganography relies on a **carrier** file or medium and a hidden **payload**. The carrier is the visible object, such as a JPEG image, PNG image, WAV audio file, MP3 file, video, PDF, or text document. The payload is the secret data embedded within it.

<dl>
<dt><strong>Carrier</strong></dt>
<dd>The innocent-looking file or medium that holds hidden data.</dd>
<dt><strong>Payload</strong></dt>
<dd>The secret information embedded into the carrier.</dd>
<dt><strong>Stego object</strong></dt>
<dd>The modified carrier after data has been embedded.</dd>
<dt><strong>Steganalysis</strong></dt>
<dd>The process of detecting, extracting, or analyzing hidden data.</dd>
</dl>

Typical properties of a good steganographic method:

- The carrier still appears normal to human observers.
- The hidden data survives common handling, such as copying or compression, when possible.
- The embedding method introduces minimal statistical anomalies.
- The payload capacity is sufficient for the attacker’s goal.

**How steganography works**

Most digital steganography techniques modify redundant or less noticeable parts of a file. The exact method depends on the carrier type.

- **Images** often use pixel-level changes, especially in the least significant bits.
- **Audio** may hide data in sample values, phase, or frequency-related properties.
- **Video** can hide data across frames, motion vectors, or audio tracks.
- **Text** can hide data through spacing, formatting, punctuation, or character choice.
- **Network traffic** can hide data in packet timing, header fields, or protocol misuse.

A common example is **Least Significant Bit (LSB)** embedding. In an image, the least significant bit of a pixel channel can be changed with minimal visible impact. For example, changing a blue channel value from `10110110` to `10110111` alters the value by only 1, which is usually imperceptible. Attackers may encode a message by replacing many such bits across the file.

```python
# Simple illustration of LSB replacement in a byte
original = 0b10110110
hidden_bit = 1
stego_byte = (original & 0b11111110) | hidden_bit
print(bin(stego_byte))  # 0b10110111
```

**Common carrier types**

Different carriers have different strengths and weaknesses.

| Carrier | Typical hiding method | Notes |
|---|---|---|
| Image | LSB, palette manipulation, metadata abuse | Very common because images are widely shared |
| Audio | LSB, echo hiding, phase coding | Compression can reduce reliability |
| Video | Frame-based embedding, motion data | Large capacity, but more complex to analyze |
| Text | Whitespace, formatting, synonym substitution | Low capacity and often fragile |
| Document | Metadata, embedded objects, hidden layers | Useful for covert delivery |
| Network | Timing channels, header fields, protocol abuse | Often called covert channels rather than classic file steganography |

> <mark>Lossy compression</mark> can destroy hidden data, especially when the method depends on exact bit patterns. JPEG recompression, audio transcoding, or video re-encoding may break simple steganographic payloads.

**Steganography versus related techniques**

Steganography is often confused with other concealment methods. The distinctions matter in exams and in real analysis.

| Technique | Primary purpose | Visibility | Example |
|---|---|---|---|
| Encryption | Protect confidentiality | Data looks unreadable | AES-encrypted archive |
| Steganography | Hide existence of data | Data looks ordinary | Secret text inside a PNG |
| Obfuscation | Make analysis harder | Code or data is confusing | Renamed variables in a script |
| Encoding | Transform data representation | Usually reversible and obvious | Base64 text |
| Covert channel | Exfiltrate data through unintended path | Often hidden in protocol behavior | DNS timing channel |

A file can be both encrypted and hidden. In practice, attackers often encrypt the payload first, then embed the ciphertext into a carrier. This reduces the usefulness of extraction unless the analyst also obtains the key.

**Detection and steganalysis**

Steganalysis is the process of finding hidden content. You usually look for anomalies rather than the payload itself. Detection can be manual, statistical, or tool-assisted.

Common indicators include:

- Unexpected file size for the apparent content.
- Metadata that does not match the file’s origin or creation history.
- Unusual entropy patterns.
- Repeated pixel or sample anomalies.
- Compression artifacts inconsistent with the file type.
- Hidden appended data after the normal file end.
- Suspicious strings, headers, or embedded archives.

A practical workflow is:

1. Identify the file type and verify it matches the extension.
2. Inspect metadata and basic file properties.
3. Compare file size against similar benign samples.
4. Check entropy and structure for anomalies.
5. Search for appended data, embedded archives, or alternate streams.
6. Attempt extraction with steganography tools if suspicion remains.

**Useful CLI commands for initial triage**

These commands help you inspect suspicious files quickly on Linux.

```bash
file suspicious.png
ls -lh suspicious.png
sha256sum suspicious.png
strings -n 8 suspicious.png | head
exiftool suspicious.png
```

For binary structure checks:

```bash
hexdump -C suspicious.png | head -n 40
binwalk suspicious.png
```

For archive or embedded content detection:

```bash
binwalk -e suspicious.jpg
7z l suspicious.docx
```

For entropy inspection, you can use tools such as `binwalk`, `ent`, or custom scripts. High entropy may suggest compression or encryption, but it is not proof of steganography by itself.

```bash
ent suspicious.bin
```

**Image steganography detection**

Images are the most common steganography carriers in basic security exercises. Detection often starts with visual and structural checks.

Look for:

- Slight color noise or banding in flat regions.
- Unexpected differences between similar images.
- Metadata inconsistencies.
- File size that is too large for the image dimensions and format.
- Hidden data appended after the image end marker.

Useful checks:

```bash
exiftool image.png
identify -verbose image.png | head -n 40
pngcheck -v image.png
```

For JPEG files, inspect markers and appended data:

```bash
jpeginfo -c image.jpg
binwalk image.jpg
```

For PNG files, verify chunk structure:

```bash
pngcheck -v image.png
```

If you suspect LSB embedding, specialized tools may help:

```bash
zsteg image.png
steghide info image.jpg
steghide extract -sf image.jpg
```

> `steghide` is commonly used in labs and challenges, but extraction usually requires the correct passphrase if the file was protected.

**Audio and video detection**

Audio steganography can be harder to spot because small sample changes are not obvious. You often need spectral or waveform analysis.

Useful checks:

```bash
file audio.wav
sox audio.wav -n stat
ffmpeg -i audio.mp3
```

Potential indicators:

- Unexpected noise floor changes.
- Irregular sample patterns.
- Metadata mismatches.
- Large file size for short duration.
- Hidden data surviving only in uncompressed formats such as WAV.

Video files may hide data in frames, subtitles, or audio tracks. You can inspect streams with:

```bash
ffprobe video.mp4
ffmpeg -i video.mp4 -map 0
```

If a video contains multiple streams, subtitles, or unusual attachments, investigate whether they are legitimate.

**Text and document steganography**

Text-based steganography has low capacity but can be effective in simple channels. It may use:

- Extra spaces or tabs.
- Line breaks or indentation patterns.
- Zero-width characters.
- Synonym substitution.
- Formatting differences in documents.

Document formats such as PDF, DOCX, and XLSX can also hide data in:

- Embedded objects.
- Comments.
- Revision history.
- Metadata.
- Hidden sheets or layers.

Useful commands:

```bash
exiftool report.pdf
pdfinfo report.pdf
pdftotext report.pdf -
strings report.docx | head
```

For Office documents, remember that `.docx` and `.xlsx` are ZIP-based containers. You can inspect them like archives:

```bash
unzip -l suspicious.docx
```

**Network-based covert channels**

Although not always classified as file steganography, network covert channels are relevant to detection concepts. Attackers may hide data in packet timing, DNS queries, HTTP headers, or unused protocol fields.

Examples include:

- DNS tunneling for data exfiltration.
- HTTP header abuse with encoded payload fragments.
- ICMP payload misuse.
- Timing-based channels where packet intervals encode bits.

Detection ideas:

- Look for unusual query lengths or high-frequency DNS requests.
- Compare traffic patterns against normal baselines.
- Inspect packet captures for repeated encoded-looking values.
- Correlate traffic with host processes and destinations.

Useful commands:

```bash
tcpdump -i eth0 -nn port 53
tshark -r capture.pcap -Y dns
```

**Practical analysis workflow**

When you receive a suspicious file, use a layered approach.

1. Confirm the file type with `file`.
2. Check size, hash, and metadata.
3. Inspect the binary structure with `hexdump` or `binwalk`.
4. Search for appended data or embedded archives.
5. Use format-specific tools such as `pngcheck`, `exiftool`, or `pdfinfo`.
6. Try extraction tools if the file type suggests steganography.
7. Validate any extracted content in a safe environment.

Example triage sequence:

```bash
file suspicious.png
exiftool suspicious.png
binwalk suspicious.png
strings -n 8 suspicious.png | less
zsteg suspicious.png
```

If `binwalk` reveals an embedded archive, extract it and inspect the contents separately.

```bash
binwalk -e suspicious.jpg
cd _suspicious.jpg.extracted
ls -la
```

**Indicators of compromise in steganography cases**

Steganography often appears alongside other malicious behavior. Watch for:

- Files exchanged through unusual channels.
- Repeated image or document downloads from the same source.
- Hidden payloads inside seemingly harmless attachments.
- Password-protected archives nested inside media files.
- Malware droppers that unpack hidden content at runtime.

In incident response, steganography matters because it can delay detection. A file may pass email filters, endpoint scanning, or casual review while still carrying malicious instructions or stolen data.

**Limitations and false positives**

Not every anomaly means steganography. Many legitimate files have:

- High entropy because they are compressed or encrypted.
- Large metadata blocks from editing software.
- Embedded thumbnails or previews.
- Multiple streams in media containers.
- Appended data from benign tools.

Be careful not to overstate findings. A strong conclusion usually requires multiple indicators, such as suspicious structure plus successful extraction or corroborating behavioral evidence.

**Exam-focused takeaways**

- **Steganography** hides the existence of data, not just its content.
- **Steganalysis** is the detection and analysis of hidden data.
- Images, audio, video, text, documents, and network traffic can all act as carriers.
- LSB embedding is a common concept, especially in image-based examples.
- File inspection tools such as `file`, `exiftool`, `binwalk`, `strings`, and format-specific validators are essential first steps.
- Compression, re-encoding, and format conversion can destroy hidden payloads.
- High entropy alone is not proof of steganography.

Steganography is a concealment technique that embeds hidden data inside ordinary-looking carriers.
