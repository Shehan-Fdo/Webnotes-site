**Learning paradigms overview**

Machine learning (ML) systems differ mainly by the kind of feedback they receive during training. The learning paradigm determines what data you need, how the model improves, and what kinds of problems it can solve well. For security and AI operations, this choice affects data labeling cost, model explainability, drift sensitivity, and how quickly you can adapt to new threats.

At a high level:

- **Supervised learning** uses labeled examples.
- **Unsupervised learning** finds structure in unlabeled data.
- **Semi-supervised learning** combines a small labeled set with a larger unlabeled set.
- **Reinforcement learning** learns by trial and error from rewards.
- **Deep learning** is a model family that often uses one of the above paradigms, usually with neural networks.

**Supervised learning**

Supervised learning trains on input-output pairs. You provide features and the correct label, and the model learns a mapping from inputs to outputs. This is the most common paradigm for classification and regression tasks.

Typical security use cases include:

- **Malware classification** based on file features or behavior.
- **Phishing detection** using message content, headers, and sender reputation.
- **Spam filtering** with labeled email examples.
- **Risk scoring** where the output is a probability or score.

Common model types include:

- **Linear regression** for continuous values.
- **Logistic regression** for binary classification.
- **Decision trees** and **random forests** for interpretable classification.
- **Support vector machines (SVMs)** for margin-based classification.
- **Neural networks** for complex nonlinear patterns.

A simple supervised workflow looks like this:

```text
Collect labeled data -> split into train/validation/test -> train model -> tune hyperparameters -> evaluate -> deploy -> monitor
```

Practical example in Python with scikit-learn:

```python
from sklearn.model_selection import train_test_split
from sklearn.ensemble import RandomForestClassifier
from sklearn.metrics import classification_report

X_train, X_test, y_train, y_test = train_test_split(features, labels, test_size=0.2, random_state=42)

model = RandomForestClassifier(n_estimators=200, random_state=42)
model.fit(X_train, y_train)

predictions = model.predict(X_test)
print(classification_report(y_test, predictions))
```

Key characteristics:

- **Requires labels** that are accurate and representative.
- **Performs well** when the target outcome is known in advance.
- **Can overfit** if the model memorizes training data instead of generalizing.
- **Depends on label quality**; noisy labels reduce performance.

Use supervised learning when you need a specific prediction and you can define the correct answer for historical data. In security, this is often the best choice for known threats, but it is weaker against novel attacks that do not resemble past examples.

**Unsupervised learning**

Unsupervised learning uses unlabeled data. The model searches for patterns, clusters, anomalies, or latent structure without being told the correct output. This is useful when labels are unavailable, expensive, or incomplete.

Common use cases include:

- **Anomaly detection** in network traffic or user behavior.
- **Clustering** endpoints by similar activity.
- **Dimensionality reduction** to simplify high-dimensional telemetry.
- **Topic discovery** in logs or incident reports.

Common techniques include:

- **K-means clustering** for grouping similar records.
- **Hierarchical clustering** for nested groupings.
- **DBSCAN** for density-based clusters and outlier detection.
- **Principal component analysis (PCA)** for dimensionality reduction.
- **Autoencoders** for compression and anomaly scoring.

Example with clustering:

```python
from sklearn.cluster import KMeans

model = KMeans(n_clusters=4, random_state=42, n_init="auto")
clusters = model.fit_predict(X)
```

Example with anomaly detection:

```python
from sklearn.ensemble import IsolationForest

model = IsolationForest(contamination=0.02, random_state=42)
anomaly_flags = model.fit_predict(X)
```

Key characteristics:

- **No labels required**, which makes it practical for large telemetry datasets.
- **Outputs structure**, not necessarily a direct business answer.
- **Can be hard to validate** because there is no ground truth for many tasks.
- **Sensitive to feature scaling** for many algorithms, especially distance-based methods.

Unsupervised learning is valuable for discovery and triage. In security operations, it often helps surface suspicious behavior that analysts can investigate, but it usually needs human review before you treat results as confirmed threats.

**Semi-supervised learning**

Semi-supervised learning combines a small labeled dataset with a larger unlabeled dataset. This approach is useful when labeling is expensive but unlabeled data is abundant. It often improves performance over purely supervised learning when labels are scarce.

Typical scenarios include:

- **Email classification** with a few thousand labeled messages and millions of unlabeled ones.
- **Malware family grouping** where only some samples are labeled.
- **Image or document classification** with limited analyst annotation.

Common strategies include:

- **Pseudo-labeling**, where the model labels unlabeled data and retrains on high-confidence predictions.
- **Self-training**, where confident predictions are added back into the training set.
- **Consistency regularization**, where the model should produce stable predictions under small input changes.
- **Graph-based methods**, where similar samples influence each other.

A practical workflow:

1. Train an initial model on labeled data.
2. Predict labels for unlabeled data.
3. Keep only high-confidence predictions.
4. Retrain with the expanded dataset.
5. Validate carefully to avoid reinforcing mistakes.

Important risks:

- **Error amplification** if the model assigns wrong pseudo-labels with high confidence.
- **Bias propagation** from the small labeled set.
- **Confidence calibration issues** where probability scores are not reliable.

Semi-supervised learning is often a strong compromise in security environments because analysts can label a small set of high-value examples, then let the model leverage the much larger pool of unlabeled logs, alerts, or artifacts.

**Reinforcement learning**

Reinforcement learning (RL) learns by interacting with an environment. The agent takes actions, receives rewards or penalties, and updates its policy to maximize long-term reward. Unlike supervised learning, RL does not need a fixed labeled dataset for every action; instead, it learns from feedback over time.

Core RL terms:

- **Agent**: the decision-maker.
- **Environment**: the system the agent interacts with.
- **State**: the current situation or observation.
- **Action**: a choice the agent can make.
- **Reward**: feedback signal after an action.
- **Policy**: the strategy for choosing actions.
- **Episode**: one complete interaction sequence.

Security-adjacent examples include:

- **Adaptive defense tuning** where a system chooses response actions based on observed outcomes.
- **Resource allocation** for dynamic scaling or throttling.
- **Game-like simulations** for adversarial training or red-team exercises.

A simplified RL loop:

```text
observe state -> choose action -> receive reward -> update policy -> repeat
```

Example pseudocode:

```python
state = env.reset()
done = False

while not done:
    action = agent.choose_action(state)
    next_state, reward, done, info = env.step(action)
    agent.learn(state, action, reward, next_state, done)
    state = next_state
```

Key characteristics:

- **Optimizes long-term outcomes**, not just immediate accuracy.
- **Explores and exploits**; it must try new actions while using known good ones.
- **Can be unstable** and data-hungry.
- **Requires careful reward design**; a poorly designed reward can produce unintended behavior.

RL is powerful when decisions affect future states, but it is harder to train and validate than supervised methods. In security contexts, you should be cautious with autonomous actions because reward hacking or unintended side effects can create operational risk.

**Deep learning**

Deep learning uses multi-layer neural networks to learn hierarchical representations from data. It is not a separate learning signal by itself; rather, it is a model approach that can be trained with supervised, unsupervised, semi-supervised, or reinforcement learning.

Why deep learning matters:

- It can learn complex nonlinear relationships.
- It often performs well on unstructured data such as text, images, audio, and sequences.
- It reduces the need for manual feature engineering in many cases.

Common deep learning architectures include:

- **Feedforward neural networks** for tabular or structured data.
- **Convolutional neural networks (CNNs)** for images and spatial patterns.
- **Recurrent neural networks (RNNs)** and **LSTMs** for sequences, though transformers are now more common.
- **Transformers** for language, code, and multimodal tasks.
- **Autoencoders** for compression, denoising, and anomaly detection.

Example of a simple neural network in Keras:

```python
from tensorflow import keras
from tensorflow.keras import layers

model = keras.Sequential([
    layers.Dense(64, activation="relu", input_shape=(X_train.shape[1],)),
    layers.Dense(32, activation="relu"),
    layers.Dense(1, activation="sigmoid")
])

model.compile(optimizer="adam", loss="binary_crossentropy", metrics=["accuracy"])
model.fit(X_train, y_train, validation_split=0.2, epochs=10, batch_size=32)
```

Deep learning strengths:

- **Feature learning** from raw or minimally processed data.
- **High accuracy potential** on large datasets.
- **Good fit for unstructured inputs** such as text and images.

Deep learning tradeoffs:

- **Needs more data** and compute than many classical models.
- **Can be less interpretable** than tree-based or linear models.
- **May be harder to secure and govern** because of model complexity.
- **Can be vulnerable to adversarial examples** and prompt or input manipulation depending on the system.

In security work, deep learning is often useful for detection pipelines, content analysis, and behavioral modeling, but you should pair it with monitoring, explainability tools, and human review for high-impact decisions.

**Comparison of the paradigms**

| Paradigm | Training signal | Best for | Main limitation |
|---|---|---|---|
| Supervised | Labeled examples | Prediction with known targets | Needs quality labels |
| Unsupervised | No labels | Discovery, clustering, anomaly detection | Harder to validate |
| Semi-supervised | Few labels + many unlabeled samples | Label-scarce environments | Can amplify mistakes |
| Reinforcement | Rewards from actions | Sequential decision-making | Hard to train safely |
| Deep learning | Model family, not a signal | Complex unstructured data | Data, compute, and interpretability costs |

A useful exam distinction is that **deep learning is not the same as supervised learning**. Deep learning describes the architecture and representation style, while supervised, unsupervised, semi-supervised, and reinforcement learning describe how the model learns.

**Practical selection guidance**

Choose the paradigm based on the problem, the data, and the operational risk.

- Use **supervised learning** when you have reliable labels and a clear target.
- Use **unsupervised learning** when you need to find unknown patterns or anomalies.
- Use **semi-supervised learning** when labels are scarce but unlabeled data is abundant.
- Use **reinforcement learning** when actions influence future states and you can define a reward.
- Use **deep learning** when the data is complex, high-dimensional, or unstructured.

A security operations center might combine them:

- Supervised model for known phishing detection.
- Unsupervised model for anomaly detection on login behavior.
- Semi-supervised pipeline to expand a small set of analyst-labeled alerts.
- Deep learning model for email body and attachment analysis.
- Reinforcement learning in a simulation for adaptive response research.

**Data preparation and evaluation**

The learning paradigm does not remove the need for good data engineering. Most model failures come from poor data quality, leakage, or weak evaluation rather than from the algorithm itself.

Important practices:

- **Split data correctly** into training, validation, and test sets.
- **Avoid data leakage** by keeping future information out of training.
- **Normalize or scale features** when the algorithm depends on distance or gradient behavior.
- **Track class imbalance** because security datasets are often skewed.
- **Monitor drift** because attacker behavior changes over time.

Common evaluation metrics:

- **Accuracy** for balanced classification problems.
- **Precision** when false positives are costly.
- **Recall** when missing threats is costly.
- **F1 score** when you need a balance of precision and recall.
- **ROC-AUC** for ranking quality across thresholds.
- **Silhouette score** for clustering quality, with caution.
- **Reward curves** and cumulative reward for reinforcement learning.

Example of a classification report:

```python
from sklearn.metrics import classification_report, confusion_matrix

print(confusion_matrix(y_test, predictions))
print(classification_report(y_test, predictions))
```

**Security and governance considerations**

AI systems used in security must be treated as operational controls, not just models. Each paradigm introduces different risks.

- **Supervised learning**
  - Can inherit label bias from historical decisions.
  - Can be poisoned if training data is compromised.
- **Unsupervised learning**
  - Can generate noisy alerts and analyst fatigue.
  - Can miss novel attacks if features are weak.
- **Semi-supervised learning**
  - Can spread early labeling errors across large datasets.
  - Needs confidence thresholds and review.
- **Reinforcement learning**
  - Can optimize the wrong reward.
  - Needs sandboxing before real-world deployment.
- **Deep learning**
  - Can be difficult to explain to auditors or incident responders.
  - Can be sensitive to adversarial manipulation.

> Treat model outputs as decision support unless you have strong validation, monitoring, and rollback controls.

**Quick memory cues**

- **Supervised** = labeled answers.
- **Unsupervised** = hidden structure.
- **Semi-supervised** = few labels, many unlabeled.
- **Reinforcement** = reward-driven actions.
- **Deep learning** = layered neural networks.

**Exam-focused takeaways**

- Know that **labels** distinguish supervised from unsupervised learning.
- Know that **semi-supervised** uses both labeled and unlabeled data.
- Know that **reinforcement learning** uses rewards, not labels.
- Know that **deep learning** is a neural-network-based approach that can be used with multiple learning paradigms.
- Expect questions about which paradigm fits a scenario such as anomaly detection, phishing classification, or adaptive response.
