## **Course:** CompTIA Security+ (Exam SY0-701)

**Wireless security fundamentals**

Wireless networks send data through the air, so anyone within range may be able to observe traffic if you do not protect it. That makes Wi-Fi security the set of controls that protect devices and networks in a wireless environment. The main goals are to prevent unauthorized access, reduce eavesdropping, and protect sensitive data in transit.

- **Eavesdropping** is a core wireless risk because radio signals extend beyond the intended user.
- Open or unsecured networks allow nearby devices to connect without meaningful protection.
- Wireless security matters in homes, public spaces, and enterprise environments.

A practical wireless security posture starts with encryption, authentication, and careful device configuration. You should assume that anything sent in the clear can be captured by an attacker within range.

**Wireless security protocols**

Wi-Fi security protocols have evolved over time to improve encryption and authentication. Older protocols are weaker, while newer ones address flaws found in earlier designs.

| Protocol | Key idea | Security note |
|---|---|---|
| WEP | Wired Equivalent Privacy | Uses RC4 with a static key; obsolete and easy to crack |
| WPA | Wi-Fi Protected Access | Introduced TKIP for per-packet keying and message integrity |
| WPA2 | Improved Wi-Fi security | Commonly used as a stronger successor to earlier standards |
| WPA3 | Newer Wi-Fi security | Designed to strengthen wireless protection further |

Use the strongest protocol your environment supports. If you must support older devices, treat that as a compatibility constraint and not a preferred security choice.

> Avoid relying on obsolete wireless security standards when stronger options are available.

**Authentication and access control**

Wireless security is not only about encryption. You also need to make sure only authorized users can join the network. Security+ wireless settings commonly include authentication options and supporting infrastructure such as the AAA framework, RADIUS, 802.1X, and EAP.

- **AAA** stands for authentication, authorization, and accounting.
- **RADIUS** helps centralize authentication for network access.
- **802.1X** supports port-based network access control.
- **EAP** provides extensible authentication methods.

In practice, you use these controls to verify identity before granting access. That reduces the chance that an unauthorized device can join the wireless network and reach internal resources.

**Common wireless threats**

Wireless threats often target the trust users place in familiar network names and nearby access points. Rogue access points, evil twins, and spoofed networks are especially dangerous because they can look legitimate while redirecting traffic or capturing credentials.

- A **rogue access point** is an unauthorized AP on or near the network.
- An **evil twin** imitates a legitimate wireless network name to trick users.
- A **spoofed** wireless network can impersonate a trusted SSID to lure connections.

These attacks work because users often connect based on the network name alone. If you do not verify the AP and the network identity, you may expose credentials or sensitive traffic to an attacker.

**Basic protection practices**

You improve wireless security by combining protocol choice, authentication, and device hardening. One simple but important step is changing default passwords on wireless devices and routers.

1. Change default administrator credentials on wireless equipment.
2. Enable strong encryption and authentication for the network.
3. Use centralized access controls where appropriate.
4. Avoid open networks unless there is a clear business need.
5. Verify the legitimacy of the SSID before connecting.

These steps do not eliminate all risk, but they reduce the most common exposure points. In particular, default passwords and unsecured networks create easy opportunities for unauthorized access.

**Exam focus**

For Security+ SY0-701, focus on the relationship between wireless convenience and wireless risk. You should be able to explain why wireless traffic is easier to intercept than wired traffic, identify malicious access point types, and recognize the purpose of encryption and authentication controls.

- Wireless data travels over the air and can be observed within range.
- Strong security depends on both encryption and identity verification.
- Rogue APs and evil twins exploit user trust in network names.
