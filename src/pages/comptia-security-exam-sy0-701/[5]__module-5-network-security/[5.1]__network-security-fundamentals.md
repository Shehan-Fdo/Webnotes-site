## **Course:** CompTIA Security+ (Exam SY0-701)

**Network security overview**

Network security protects communication infrastructure from cyberattacks, unauthorized access, and data loss. It focuses on keeping legitimate traffic moving while reducing exposure to threats across the network edge and inside the perimeter.

A practical view of network security includes technologies, policies, people, and procedures. The goal is not only to block attacks, but also to preserve the **CIA triad**: confidentiality, integrity, and availability.

- Networks connect users, applications, workloads, and services.
- Security controls reduce risk and help prevent lateral movement if an attacker gets in.
- A layered approach is more resilient than relying on a single control.

**Core networking models and traffic flow**

The **OSI** (Open Systems Interconnection) model and **TCP/IP** (Transmission Control Protocol/Internet Protocol) model help you understand how data moves from applications to cables. Layered models are foundational because they separate responsibilities and make troubleshooting easier.

Protocol choice also matters. **TCP** (Transmission Control Protocol) emphasizes reliable delivery, while **UDP** (User Datagram Protocol) favors speed and lower overhead. You choose based on the application’s tolerance for delay, loss, or retransmission.

- OSI and TCP/IP are reference models for network communication.
- TCP is connection-oriented and reliability-focused.
- UDP is connectionless and speed-focused.

**Addressing, ports, and common services**

**IP** (Internet Protocol) addressing identifies devices on a network. You should recognize both **IPv4** and **IPv6** formats, along with private address ranges such as `192.168.x.x`.

Common ports help you identify services and support firewall rules, diagnostics, and application configuration. Service awareness is essential because many security decisions depend on what should or should not be reachable.

- IP addressing identifies hosts and networks.
- Private ranges are commonly used inside internal networks.
- Port numbers map traffic to specific services.

| Item | Purpose | Example |
|---|---|---|
| IP address | Identifies a device | `192.168.1.10` |
| Port | Identifies a service | `22` for SSH |
| Protocol | Defines communication behavior | TCP or UDP |

**Network services and protocols**

Several services and protocols appear repeatedly in network security work. **DNS** (Domain Name System), **DHCP** (Dynamic Host Configuration Protocol), **ARP** (Address Resolution Protocol), and **HTTP** (Hypertext Transfer Protocol) keep networks discoverable, addressable, and usable.

These services are operationally important because security controls often depend on them. If name resolution, address assignment, or local address mapping fails, users may see symptoms that look like application or connectivity problems.

- DNS resolves names to addresses.
- DHCP assigns network configuration dynamically.
- ARP maps local network addresses.
- HTTP carries web traffic.

**Security devices and control points**

Common network security devices include **firewalls**, **IDS** (Intrusion Detection Systems), **IPS** (Intrusion Prevention Systems), and **proxies**. These tools help control traffic, detect threats, and protect endpoints.

A firewall is a primary safeguard at the perimeter or between internal segments. IDS focuses on detection, while IPS can block or prevent malicious activity. Proxies sit between clients and destinations to mediate traffic and add control.

- Firewalls filter traffic based on rules.
- IDS alerts on suspicious activity.
- IPS can stop malicious traffic in transit.
- Proxies mediate requests and can enforce policy.

> Use the right control for the job: detection, prevention, filtering, or mediation. Do not assume one device replaces the others.

**Useful networking commands**

Basic commands help you validate connectivity, trace paths, and resolve issues quickly. The evidence highlights `ping`, `trace`, `scan`, and `resolve` as go-to tools for quick insight and fixes.

Use these commands to confirm whether a problem is local, network-wide, or service-specific. They are especially useful when a host is unreachable, a service is not responding, or name resolution appears broken.

```bash
ping 192.168.1.10
traceroute example.com
nslookup example.com
nmap 192.168.1.0/24
```

- `ping` checks reachability.
- `trace` shows the path to a destination.
- `scan` helps identify open services.
- `resolve` verifies name-to-address lookup.

**How to study this topic**

Focus on relationships rather than isolated definitions. If you can explain how a packet moves, which protocol carries it, which port it uses, and which control inspects it, you are building exam-ready understanding.

1. Review the OSI and TCP/IP models.
2. Match common protocols to their roles.
3. Memorize key ports and address ranges.
4. Compare firewall, IDS, IPS, and proxy behavior.
5. Practice basic troubleshooting commands.

**Quick recall**

- Network security protects traffic and network-accessible assets.
- Layered models explain how data moves through a network.
- TCP prioritizes reliability; UDP prioritizes speed.
- DNS, DHCP, ARP, and HTTP are core services to recognize.
- Firewalls, IDS, IPS, and proxies are common security tools.
