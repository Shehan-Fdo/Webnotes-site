## **Course:** CompTIA Security+ (Exam SY0-701)

**Segmentation as a security control**

Network segmentation divides a network into smaller sections or subnets so each segment acts like its own network. This improves control over traffic flow and helps reduce the impact of a compromise.

Segmentation supports both security and organization. By separating resources, you make it harder for unauthorized users to reach high-value assets such as customer data, financial records, and intellectual property.

- **Smaller scope** limits how far an incident can spread.
- **Traffic control** lets you apply different rules to different network areas.
- **Performance** can improve when traffic is organized into separate segments.
- **Compliance** may require separation for sensitive data, such as cardholder information.

**Common segmentation approaches**

Segmentation can be physical or logical. Physical segmentation separates devices with separate infrastructure, while logical segmentation separates traffic on shared infrastructure.

Virtual LANs (VLANs) are a common form of logical segmentation. In cloud and virtual machine environments, virtual segmentation serves a similar purpose by isolating workloads into separate network areas.

> Use segmentation to reduce exposure, not as a replacement for authentication or authorization.

**DMZs and exposed services**

A demilitarized zone, or **DMZ**, is a perimeter network between a protected internal network and an untrusted network such as the internet. It gives public-facing services a place to live without placing the rest of the internal network directly at risk.

Organizations commonly place externally reachable services in a DMZ, including:

- Web servers
- Domain Name System (DNS) servers
- Mail servers
- File Transfer Protocol (FTP) servers
- Proxy services
- Voice over Internet Protocol (VoIP) services

A DMZ isolates these systems and gives them limited access to the internal local-area network (LAN). That design helps you expose internet-facing functionality while keeping internal systems more protected.

**Access control lists and allow lists**

Access control lists, or **ACLs**, define which traffic is permitted or denied. They are a practical way to enforce segmentation rules between network zones.

Application allow lists are another control that limits what software or services can run or communicate. In the context of segmentation, allow lists help you narrow permitted activity to only what is needed.

- **ACLs** control network traffic between segments.
- **Allow lists** restrict approved applications or services.
- **Least privilege** keeps access as narrow as possible.
- **Policy enforcement** becomes easier when zones have clear rules.

**Zero Trust and micro-segmentation**

Zero Trust Network Access (ZTNA) treats devices as untrustworthy by default and requires each transaction to be authenticated, authorized, and encrypted. This approach aligns with segmentation because it assumes no implicit trust based on location.

Micro-segmentation takes segmentation further by creating very small trust boundaries. It is useful when you want tighter control over east-west traffic inside the network.

**Practical exam takeaways**

When you see a scenario with sensitive systems, public-facing services, or a need to contain breaches, think about segmentation first. The goal is to separate assets so that one compromise does not automatically expose everything else.

1. Identify the asset or service that needs isolation.
2. Choose the right boundary, such as a VLAN, DMZ, or micro-segment.
3. Apply ACLs or allow lists to restrict traffic.
4. Verify that only required communication paths remain open.
5. Reassess the design when the environment changes.

**What to remember**

- Segmentation reduces the blast radius of an attack.
- A DMZ protects internal systems from untrusted traffic.
- VLANs and virtual segmentation are common logical controls.
- ACLs and allow lists enforce what can cross boundaries.
- ZTNA and micro-segmentation strengthen the idea of no implicit trust.
