## **Course:** CompTIA Security+ (Exam SY0-701)

**Cloud security service mapping**

Cloud platforms expose similar security capabilities under different product names. For exam prep, focus on the function first, then map the vendor-specific service. This helps you recognize equivalent controls across AWS, Google Cloud Platform (GCP), and Azure.

- **Infrastructure protection** services reduce exposure to distributed attacks.
  - AWS Shield
  - Google Cloud Armor
  - Azure DDoS Protection
- **Web application firewall (WAF)** services filter malicious web traffic.
  - AWS WAF
  - Google Cloud Armor
  - Azure WAF
- **Monitoring and threat detection** services centralize security visibility.
  - AWS Security Hub
  - GCP Security Command Center
  - Azure Security Center
- **Identity and access management (IAM)** services control authentication and authorization.
  - AWS IAM
  - GCP IAM
  - Azure Active Directory
- **Logging and auditing** services support investigation and compliance.
  - AWS CloudTrail
  - GCP Cloud Audit Logs
  - Azure Audit Logs
- **Data protection** services manage encryption keys and hardware-backed key storage.
  - AWS KMS / CloudHSM
  - GCP KMS / CloudHSM
  - Azure Key Vault / Managed HSM

A practical study approach is to memorize the security function, then attach the cloud provider label. That pattern is more useful than trying to learn each product in isolation.

**Cross-cloud security operations**

Cloud security tools can also work across environments. One example is using Microsoft security solutions to help secure AWS by applying defense-in-depth controls aligned with Zero Trust principles. The goal is consistent visibility, enforcement, and protection even when the workload runs outside the native cloud stack.

- **Microsoft Entra** can provide single sign-on (SSO), lifecycle automation, conditional access, identity protection, and Privileged Identity Management.
- **Microsoft Sentinel** can ingest and analyze AWS security logs for centralized monitoring.
- **AWS security logs** such as CloudTrail, CloudWatch Logs, VPC Flow Logs, and GuardDuty findings can be delivered into Sentinel.
- **Centralized analysis** helps you detect security-related events faster than reviewing each service separately.

> Use conservative wording on the exam: cross-cloud integration does not replace native controls. It extends visibility and policy enforcement across environments.

**Infrastructure as Code and security as code**

Infrastructure as Code (IaC) means you define infrastructure in machine-readable policy files instead of configuring each resource manually. Security as code applies the same idea to security controls. Both approaches support repeatable deployments and reduce configuration drift.

- **Consistency** improves because the same policy file can configure many resources the same way.
- **Automation** reduces human error during provisioning and change management.
- **Traceability** improves because changes are easier to track and review.
- **Governance** becomes more predictable when policy enforcement is built into deployment workflows.

IaC is especially relevant in cloud environments because public cloud breaches often stem from misconfiguration rather than attacks on the underlying infrastructure. That makes secure configuration and change control a core defensive skill.

**Why IaC matters for security**

IaC supports security at scale by making configuration repeatable across environments. It helps you avoid the drift that appears when teams manually change resources over time.

- **Repeatable deployments** make it easier to keep dev, test, and production aligned.
- **Policy enforcement** can be applied consistently instead of relying on manual checks.
- **Audit readiness** improves when infrastructure changes are defined and reviewed as code.
- **Operational risk** drops when privilege sprawl and inconsistent settings are reduced.

**Key takeaways**

Representative cloud technologies are easiest to remember when you group them by security purpose:

- **Protect** traffic with DDoS protection and WAF services.
- **Detect** threats with centralized security monitoring.
- **Control** access with IAM and identity services.
- **Record** activity with cloud logging and auditing.
- **Protect data** with key management and hardware security modules.
- **Automate** secure configuration with IaC and security as code.
