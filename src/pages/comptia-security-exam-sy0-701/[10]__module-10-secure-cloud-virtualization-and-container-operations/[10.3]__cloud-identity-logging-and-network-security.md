## **Course:** CompTIA Security+ (Exam SY0-701)

**Cloud identity and audit visibility**

Cloud security depends on both identity visibility and activity logging. Audit logs help you answer who did what, where, and when across cloud resources, while also supporting security, auditing, and compliance monitoring.

- **Cloud Audit Logs** record administrative activities and resource access in Google Cloud.
- **Data Access** audit logs are recommended to stay enabled because they help support troubleshooting and investigation.
- Audit logs provide transparency similar to on-premises environments, but they are tied to cloud services and control planes.
- In multicloud environments, you may centralize logs for analysis across providers.

A practical exam takeaway is that identity and logging work together: permissions define what should be possible, and logs show what actually happened.

**Cloud logging use cases**

Cloud logging platforms often extend beyond simple retention. They can ingest activity from other clouds, normalize fields, and support investigation workflows.

- Microsoft Defender for Cloud can collect **AWS CloudTrail** management events.
- CloudTrail ingestion improves visibility into identity operations, permission changes, and other control-plane activity.
- Activity-based signals help identify unused permissions, misconfigured roles, dormant identities, and possible privilege escalation paths.
- CloudTrail data can also strengthen configuration drift detection and security recommendations.

When you see a logging question, focus on whether the tool is providing visibility, retention, correlation, or investigation support.

**Virtual firewall controls in the cloud**

Cloud providers use virtual firewall features to filter traffic at the packet level. These controls are not the same as traditional perimeter firewalls, because they are built into the cloud platform and applied to cloud resources.

- Azure **network security groups** (NSGs) filter inbound and outbound traffic between Azure resources in virtual networks.
- NSGs contain security rules that allow or deny traffic.
- Oracle Cloud Infrastructure offers **network security groups** as one of two virtual firewall features, alongside security lists.
- Both NSGs and security lists use security rules.
- NSGs are supported only for specific services in Oracle Cloud Infrastructure.

A useful comparison point is that cloud firewalls are resource-aware and policy-driven, so you usually manage access closer to the workload rather than only at the edge.

**Zero Trust Packet Routing and layered enforcement**

Oracle Cloud Infrastructure also supports **Zero Trust Packet Routing** (ZPR), which can be used along with or in place of network security groups to manage access.

- ZPR policies govern how resources communicate with each other.
- You add security attributes to resources to apply ZPR behavior.
- If an endpoint has a ZPR security attribute, traffic must satisfy ZPR policies and also all NSG and security list rules.
- This creates layered enforcement rather than replacing existing controls.

> If ZPR is enabled, do not assume it overrides other rules. Traffic still has to pass every applicable policy and firewall rule.

**How to reason about cloud network security questions**

When you are asked about cloud network filtering, identify the control by its scope and rule model.

1. Determine whether the control filters traffic between cloud resources or at a broader perimeter.
2. Check whether the rule set is allow/deny based and whether it applies inbound, outbound, or both.
3. Look for platform-specific terms such as NSG, security list, or ZPR.
4. Match the control to the cloud provider and the type of resource it protects.

**Key terms to remember**

- **Audit log**: A record of administrative activity and resource access.
- **Control plane**: The management layer where identity and configuration changes occur.
- **NSG**: A virtual firewall group that applies security rules to cloud resources.
- **CloudTrail**: AWS logging for activity and management events.
- **CIEM**: Cloud Infrastructure Entitlement Management, which analyzes identity and permission risk.

**Exam focus summary**

Cloud identity and logging questions usually test whether you can connect permissions, observed activity, and investigation value. Network security questions usually test whether you can identify the cloud-native firewall feature and understand how rules are applied.

- Use logs to answer what happened and who did it.
- Use identity data to understand what should be allowed.
- Use NSGs and similar controls to restrict traffic between cloud resources.
- Remember that layered controls can all apply at once.
