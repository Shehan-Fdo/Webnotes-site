## **Course:** CompTIA Security+ (Exam SY0-701)

**Remote access VPN overview**

A remote access virtual private network (<abbr title="Virtual Private Network">VPN</abbr>) lets an individual user connect to a private network from an off-site location. It is commonly used by employees, telecommuters, mobile users, and other remote clients who need secure access to internal resources.

A remote access VPN creates an encrypted tunnel over the internet so traffic is protected from unauthorized access. Once connected, the user can reach network resources as if they were on-premises.

- **Remote user** access is individual and client-focused.
- **Encryption** protects data in transit.
- **Tunneling** carries private traffic across a public network.
- **Full access** to network resources may be available after connection, depending on policy.

**Remote access VPN vs. site-to-site VPN**

Remote access VPNs are designed for one host at a time. Site-to-site VPNs connect entire networks together. For exam purposes, focus on the user-centric nature of remote access VPNs and the fact that they are built for off-site endpoints rather than permanent network-to-network links.

| Aspect | Remote access VPN | Site-to-site VPN |
|---|---|---|
| Primary user | Individual host or client | Network or office location |
| Typical use | Telecommuting, mobile access, extranet access | Connecting two networks |
| Access model | User connects to corporate resources | Networks communicate through a tunnel |

**Security controls used with remote access VPNs**

Remote secure access VPNs often combine several controls to protect the session and the endpoint. The goal is not only to encrypt traffic, but also to verify the user and the device before access is granted.

- **Multi-factor authentication** helps confirm the user’s identity.
- **Endpoint compliance scanning** checks whether the device meets policy requirements.
- **Encryption** protects transmitted data during the session.
- **TLS** can secure client-to-server traffic in a single session.
- **IPsec-IKEv2** and SSL-based options are used in some remote access deployments.

> Remote access VPNs improve confidentiality and reachability, but they also expand the attack surface because remote users connect from outside the trusted network.

**Client and deployment concepts**

Remote access VPNs may use dedicated client software or a web-based client. Some environments deploy the client from a secure gateway when the user connects, while mobile apps may be installed from a platform app store.

In practice, the client handles negotiation with the security gateway and establishes the protected connection. This reduces the need for administrators to manually install and configure software on every remote device.

- **Client software** may be installed locally on the endpoint.
- **Web-based access** can reduce installation requirements.
- **Secure gateway** devices terminate and manage the VPN connection.
- **Policy wizards** may be used to set up basic remote access VPN settings before further customization.

**Protocols and transport**

Remote access VPNs can use different protocols depending on the platform and security design. The evidence highlights SSL/TLS-based connections and IPsec-IKEv2 as common options.

- **SSL VPN** uses TLS or DTLS for secure communication in some implementations.
- **IPsec-IKEv2 VPN** is another supported remote access option in some environments.
- **Protocol choice** affects how the client connects and how the tunnel is established.

**Practical exam takeaways**

When you see a scenario, identify whether the need is for a single remote user or for connecting two networks. Then match the control set to the goal: encryption for confidentiality, authentication for identity, and endpoint checks for device trust.

1. Determine whether the user is connecting from off-site to internal resources.
2. Confirm that the connection is encrypted and tunneled.
3. Look for additional controls such as multi-factor authentication or endpoint compliance scanning.
4. Distinguish remote access VPNs from site-to-site VPNs based on scope.

**Key terms to remember**

- **VPN**: A private connection established over a public network.
- **Tunnel**: The protected path that carries traffic between endpoints.
- **TLS**: Transport Layer Security, used to secure client-to-server communication.
- **IPsec-IKEv2**: A VPN approach used in some remote access deployments.
- **Endpoint compliance scanning**: A check that verifies device posture before access.
