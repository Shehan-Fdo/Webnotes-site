## **Course:** CompTIA Security+ (Exam SY0-701)

**Identity threats and attacks**

Identity-based attacks target user identities rather than perimeter defenses. They focus on usernames, passwords, authentication tokens, and other trust artifacts to gain unauthorized access to systems or data. These attacks often succeed because they exploit the relationship between users and services, then reuse legitimate access to move deeper into an environment.

- **Identity-based attacks** use stolen, forged, or misused credentials to bypass normal access controls.
- Common techniques include phishing, credential stuffing, session hijacking, MFA bypass, and token theft.
- Attackers often aim for access that looks legitimate, which makes detection harder than with noisy malware-only attacks.

**Credential theft and token theft**

Credential theft and token theft are related but not the same. Credential theft captures usernames and passwords, often through phishing, keyloggers, malware, or breach databases. Token theft captures active session tokens or OAuth refresh tokens, which can act like keys to an already authenticated session.

> Stolen tokens can bypass MFA because they represent an existing authenticated session.

- **Credential theft** usually still requires the attacker to authenticate after stealing the secret.
- **Token theft** can provide immediate access without re-entering credentials.
- Token theft is especially important in SaaS environments because it can preserve access until the token expires or is revoked.

**Multi-factor authentication interception**

Multi-factor authentication (MFA) improves security, but attackers still target the authentication process itself. MITRE ATT&CK describes MFA interception as adversaries targeting MFA mechanisms such as smart cards or token generators to obtain the information needed to authenticate.

- Attackers may use a **keylogger** to capture passwords associated with hardware tokens or smart cards.
- Interception can let an attacker proxy authentication from an infected system.
- MFA reduces risk, but it does not eliminate identity attacks when the authentication flow is compromised.

**Active Directory identity attacks**

Active Directory environments are common targets because stolen identity material can be reused for lateral movement and privilege escalation. Pass-the-Hash and Pass-the-Ticket are two important examples.

- **Pass-the-Hash (PtH)** reuses stolen NTLM hashes without cracking the password.
- **Pass-the-Ticket (PtT)** uses stolen Kerberos tickets, such as Ticket Granting Tickets (TGTs) or Ticket Granting Service (TGS) tickets.
- These attacks can support lateral movement, persistence, and domain compromise.

A practical attack path in Active Directory often follows this sequence:

1. Initial access.
2. Credential dumping, often from LSASS (Local Security Authority Subsystem Service).
3. Pass-the-Hash or Pass-the-Ticket.
4. Lateral movement.
5. Privilege escalation.
6. Domain takeover.

**Phishing and modern identity attack patterns**

Phishing remains a core identity threat because it targets the human side of authentication. Modern phishing techniques increasingly try to defeat stronger controls such as MFA, passwordless authentication, and email protections.

- **Adversary-in-the-middle (AiTM)** phishing can capture authentication data during a legitimate-looking login flow.
- Device code phishing can target users inside an organization by abusing authentication workflows.
- Lures remain effective because they exploit trust, urgency, and routine behavior.

**Detection and alert context**

Security tools may surface identity attacks through alerting systems that normalize detections into different formats. For example, Microsoft Defender for Identity can generate alerts in both a Defender format and a classic format, even when both are based on the same underlying sensor detections.

- Check the **Detection source** field to identify the alert format.
- Alert names may differ across formats even when the underlying detection is the same.
- Consistent alert naming and categorization help you correlate identity events across tools.

**What to remember for exam prep**

Identity threats are about abusing trust in authentication and authorization. Focus on how the attacker gets the identity artifact, how they use it, and whether the access is pre-authentication or post-authentication.

- **Pre-compromise** attacks steal credentials before login.
- **Post-compromise** attacks reuse sessions, tokens, or tickets after access exists.
- Strong identity security depends on layered controls, but attackers still look for the weakest point in the authentication chain.
