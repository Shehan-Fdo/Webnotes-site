## **Course:** CompTIA Security+ (Exam SY0-701)

**Core security model**

The **CIA triad** is a foundational model for security systems. It breaks security into three focal points: confidentiality, integrity, and availability. Use it to think clearly about vulnerabilities and the controls that reduce them.

- **Confidentiality** protects information from unauthorized access.
- **Integrity** ensures information stays accurate and unaltered.
- **Availability** keeps systems and data accessible when needed.

When you evaluate a control or a risk, ask which part of the triad it supports. Strong security improves all three areas together, but tradeoffs are common in real environments.

**Confidentiality**

Confidentiality focuses on preventing unauthorized disclosure of information. It matters whenever data must stay private, whether at rest, in transit, or during access by users and services.

- Limit access to only approved users and processes.
- Protect sensitive data from exposure through weak permissions or poor handling.
- Use controls that reduce the chance of accidental or malicious disclosure.

A practical way to study confidentiality is to connect it to access decisions. If a control reduces who can see data, it supports confidentiality.

**Integrity**

Integrity means data remains correct, complete, and trustworthy. If information changes unexpectedly, the result can be bad decisions, failed transactions, or unreliable records.

- Detect unauthorized changes to files, messages, or records.
- Preserve the original meaning of data during storage and transmission.
- Verify that the received information matches what was sent.

Integrity is especially important when data moves between systems. If the recipient cannot trust the content, the security value of the data drops quickly.

**Availability**

Availability ensures systems and information are accessible when users need them. A secure environment is not useful if legitimate users cannot reach critical services.

- Keep systems running and resilient against disruption.
- Reduce single points of failure where possible.
- Plan for incidents that could interrupt access.

Availability is often tested through outages, attacks, or resource exhaustion. Good design aims to keep essential services usable even under stress.

**Extended security principles**

Zero Trust is a security approach built around the idea of **“never trust, always verify.”** It is not a single product. Instead, it is a collection of security capabilities that support controls, visibility, and orchestration across the environment.

- Do not assume trust based only on network location.
- Verify access requests before granting access.
- Apply security consistently across modern environments, including cloud-based systems and third-party relationships.

Zero Trust fits well with the CIA triad because it helps reduce unauthorized access, improve oversight, and limit the impact of compromise.

**How the concepts work together**

The CIA triad gives you the three main security goals. Extended principles such as Zero Trust help you apply those goals in modern environments.

- Confidentiality is strengthened when access is verified instead of assumed.
- Integrity improves when systems can detect and limit unauthorized changes.
- Availability improves when security design supports resilience and controlled access.

In practice, you use these principles to guide security decisions rather than treat them as isolated definitions.

**Exam focus reminders**

For Security+ study, be ready to identify the triad element affected by a scenario.

- A privacy issue usually points to **confidentiality**.
- A tampering or alteration issue usually points to **integrity**.
- An outage or service interruption usually points to **availability**.
- A “never trust, always verify” scenario usually points to **Zero Trust**.

> Keep the distinction clear: the CIA triad is a security model, while Zero Trust is an approach for applying security controls in a modern environment.
