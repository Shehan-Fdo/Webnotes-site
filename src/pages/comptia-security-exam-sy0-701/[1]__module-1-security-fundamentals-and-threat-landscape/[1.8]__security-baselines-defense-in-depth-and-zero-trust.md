## **Course:** CompTIA Security+ (Exam SY0-701)

**Security baselines**

A security baseline is the minimum set of security settings and controls you apply consistently across systems. It gives you a known starting point for secure configuration and helps you compare systems against an expected standard.

Baselines matter because they reduce configuration drift. When systems are built and maintained from the same reference point, you can spot exceptions faster and manage risk more predictably.

- Use a baseline to define the expected secure state for a system or service.
- Compare current settings against the baseline to identify deviations.
- Update baselines when business needs or threat conditions change.
- Treat baselines as a practical control for consistency, not a one-time setup.

**Defense in depth**

Defense in depth uses multiple layers of protection instead of relying on a single barrier. The idea is that if one control fails, another control still helps protect the asset.

This approach fits modern threats because perimeter defenses alone are not enough. Multidimensional protection gives you better resilience against advanced threats and reduces the chance that one weakness leads to full compromise.

- Layer controls across people, processes, and technology.
- Combine preventive, detective, and corrective measures.
- Protect assets at multiple points, not only at the network edge.
- Assume one control can fail and design for overlap.

**Zero Trust**

Zero Trust is a security model built on the idea that nothing is trusted by default. That includes actors, systems, networks, and services both inside and outside the perimeter. You verify every access attempt before granting access.

A Zero Trust strategy focuses on safeguarding people, devices, apps, and data while improving visibility and reducing risk. It also supports compliance by using risk-based access controls and automated policies.

<dl>
<dt>Core idea</dt>
<dd>Never trust by default; always verify.</dd>
<dt>Access model</dt>
<dd>Grant only the access needed for the task and manage privileges actively.</dd>
<dt>Monitoring focus</dt>
<dd>Track devices and other access points continuously.</dd>
</dl>

- Verify identity and context before allowing access.
- Manage employee privileges so users get only what they need to do their jobs.
- Monitor devices used to access resources.
- Use risk-based access controls to adapt to changing conditions.
- Apply automated policies to reduce manual enforcement gaps.

> Zero Trust does not mean “trust nothing ever.” It means you do not assume trust based on location or network position.

**How the models work together**

Security baselines, defense in depth, and Zero Trust support each other. Baselines establish a secure starting point, defense in depth adds layered protection, and Zero Trust enforces continuous verification for access decisions.

Together, they help you reduce exposure in modern environments where perimeter-only thinking is not enough. A strong baseline makes layered controls easier to manage, and Zero Trust helps ensure that access remains conditional and monitored.

| Concept | Primary purpose | Practical emphasis |
|---|---|---|
| Security baseline | Standardize secure configuration | Consistency and drift reduction |
| Defense in depth | Add overlapping protections | Layered resilience |
| Zero Trust | Verify every access attempt | Continuous validation |

**Study approach**

When you review these topics, focus on the relationship between them rather than memorizing isolated definitions. Security baselines define the expected state, defense in depth protects across multiple layers, and Zero Trust controls access through verification and least privilege.

1. Identify the baseline for the system or service.
2. Add layered controls that protect the asset at different points.
3. Require verification before access is granted.
4. Monitor and adjust controls as risk changes.

**Exam focus**

For exam questions, look for clues about trust assumptions, layered controls, and configuration consistency. If a scenario mentions verifying users and devices before access, think Zero Trust. If it emphasizes multiple protective layers, think defense in depth. If it focuses on standard secure settings, think security baseline.
