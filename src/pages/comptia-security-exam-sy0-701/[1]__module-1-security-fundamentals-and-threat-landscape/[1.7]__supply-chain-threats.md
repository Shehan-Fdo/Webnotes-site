## **Course:** CompTIA Security+ (Exam SY0-701)

**Supply chain threat overview**

Supply chain threats target the path a product or service takes from creation to delivery. In security terms, you care about every stage where an attacker can introduce malicious code, tamper with components, or gain access through a trusted relationship.

Modern supply chains are highly interconnected. That means a single weakness in a vendor, supplier, software package, or hardware component can affect many downstream systems at once.

- **Third-party risk** grows as you depend on more vendors, platforms, and service providers.
- **Downstream impact** can spread quickly when a trusted component is poisoned before it reaches users.
- **Trust assumptions** are dangerous when equipment or software is accepted without verification.

**How supply chain compromise happens**

Attackers often target the places organizations trust most. They may compromise a maintainer account, alter a package before release, or insert malicious code into a component during manufacturing or distribution.

A software supply chain attack can bypass normal defenses because the malicious content arrives through a legitimate update or dependency. In one reported case, attackers compromised the official Axios package on the npm registry and released poisoned versions that executed immediately when installed.

- **Maintainer credentials** are a high-value target because they can be used to publish malicious updates.
- **Poisoned dependencies** can execute during installation or build processes.
- **Persistence** may follow initial compromise if the payload installs remote access tooling.

> A trusted update path can become an attack path if integrity is not verified.

**Software supply chain threats**

Software supply chain attacks exploit the trust placed in libraries, packages, build systems, and vendors. These attacks can steal credentials, install malware, and create long-term access to affected systems.

The evidence shows that malicious code can steal cloud access keys, database passwords, and API tokens. It can also install a Remote Access Trojan (RAT), which gives an attacker persistent access to the compromised machine.

- **Open-source libraries** can have broad downstream reach because they are reused across many applications.
- **Build pipelines** are exposed when automated systems install compromised packages.
- **Credential theft** is a common objective because stolen secrets enable follow-on access.

**Hardware and firmware supply chain threats**

Hardware supply chains are multi-layered and globalized. Design, sourcing, manufacturing, assembly, and logistics may all involve different organizations and locations, which increases exposure to tampering and compromise.

Firmware is especially sensitive because it can execute below the operating system level. Malicious firmware updates may evade many security tools and remain hidden from normal inspection.

- **Hardware tampering** can occur at multiple stages before a device reaches the customer.
- **Firmware attacks** can execute with more privilege than the operating system kernel.
- **Boot-level compromise** is dangerous because it can survive standard endpoint defenses.

**Why these threats matter**

Supply chain compromise can lead to network access, stealthy persistence, data theft, and surveillance. Because the compromised component is often trusted, defenders may not notice the intrusion right away.

The practical risk is not limited to one device or one application. A single compromised supplier or package can cascade across an entire business environment and create operational, financial, and reputational damage.

- **Stealth** makes detection harder when the attacker uses legitimate channels.
- **Scale** increases when one component is reused across many systems.
- **Business impact** can include disruption, data exposure, and loss of trust.

**Defensive priorities**

You reduce supply chain risk by verifying what you install, limiting trust, and monitoring for unexpected behavior. The evidence emphasizes the need to detect and prevent compromise across vendors, software providers, and hardware sources.

Focus on practical controls that reduce exposure to poisoned components and unauthorized changes.

- **Verify integrity** of packages, updates, and delivered components.
- **Review third-party risk** before trusting vendors and software providers.
- **Monitor credentials** and secrets for signs of theft or misuse.
- **Inspect firmware and hardware sources** where feasible in your environment.

**Exam focus**

For Security+ SY0-701, remember that supply chain threats are about compromise through trusted relationships and delivery paths. The key idea is that the attacker does not always attack the target directly; instead, they exploit the path the target already trusts.

Use these cues to identify the topic on the exam:

- **Vendor compromise** affects software, hardware, or services before delivery.
- **Malicious dependency** can execute when installed by developers or automation.
- **Firmware tampering** can persist below the operating system.
- **Cascading impact** can spread from one weak link to many systems.
