## **Course:** CompTIA Security+ (Exam SY0-701)

**Virtualization basics**

Virtualization lets one physical host run multiple isolated computing environments. In practice, this means you can share hardware while keeping workloads separated.

A **virtual machine (VM)** emulates a physical computer so each guest operating system can run independently on the same hardware. A **hypervisor** is the software layer, also called a virtual machine monitor (VMM), that creates and manages those VMs.

- VMs virtualize hardware, not just applications.
- Each VM has its own operating system.
- The hypervisor allocates resources such as CPU, memory, and storage.
- Isolation helps one VM’s issues stay contained from others.

**Hypervisor types**

Hypervisors come in two common forms:

| Type | Description | Typical use |
|---|---|---|
| Type 1 | Runs directly on hardware without an underlying operating system | Enterprise data centers and cloud environments |
| Type 2 | Runs on top of a host operating system | Smaller-scale environments and personal use |

Type 1 hypervisors are associated with higher efficiency and performance. Type 2 hypervisors are easier to install and support a wider range of hardware.

**Containerization fundamentals**

A container packages an application with its dependencies and configuration so it can run consistently across environments. Containers provide a lightweight runtime that improves portability and repeatability.

- Containers focus on the application and its dependencies.
- They support cross-platform use across operating systems and cloud platforms.
- They help keep development, testing, and deployment behavior consistent.
- They are useful when you want a portable runtime with less overhead than a full VM.

> Containers share infrastructure and can be multi-tenant by design, so isolation matters for both security and stability.

**Container isolation**

Container isolation keeps containers separated from each other and from the host system. This is important because containers often share a host kernel and shared resources.

Windows containers support two runtime isolation modes:

- **Process isolation**
  - Multiple container instances run on the same host.
  - Isolation is provided through the host operating system.
- **Hyper-V isolation**
  - Containers still use the same images and management model.
  - The isolation boundary is stronger between the container, host OS, and other containers.

Both modes create, manage, and function similarly from the outside, but they differ in how much isolation they provide.

**Kubernetes overview**

Kubernetes is a portable, extensible platform for managing containerized workloads and services. It supports declarative configuration and automation, which helps you manage containers at scale.

Kubernetes is useful when you need the platform to handle container behavior for you, such as starting another container when one fails.

- It manages containerized workloads and services.
- It supports automation and declarative configuration.
- It helps reduce downtime by replacing failed containers.
- It has a large ecosystem of tools and support.

**Virtual machines vs. containers**

Use the right model based on the level of isolation and the workload shape you need.

| Aspect | Virtual machines | Containers |
|---|---|---|
| Abstraction level | Hardware | Application and dependencies |
| Guest OS | Each VM has its own OS | Containers share the host kernel |
| Overhead | Higher | Lower |
| Isolation | Strong separation through the hypervisor | Lighter-weight separation, depends on runtime and host controls |
| Typical strength | Running multiple independent OS environments | Portable application deployment |

**Security considerations**

Virtualization and containerization both improve efficiency, but they also change the security model.

- Strong isolation boundaries matter when multiple workloads share the same host.
- Patch management and orchestration complexity can affect both VM and container environments.
- You should understand what is shared, what is isolated, and where the control plane lives.
- Container environments often need extra attention to runtime isolation and host security.

**Exam focus**

For Security+ purposes, focus on these distinctions:

- A **hypervisor** manages VMs.
- A **VM** includes its own operating system.
- A **container** packages an app with dependencies and shares the host kernel.
- **Kubernetes** manages containerized workloads and services.
- Isolation is a key security concept in both virtualization and containerization.
