## **Course:** CompTIA Security+ (Exam SY0-701)

**Security model fundamentals**

Security models provide a formal way to define access rules and protect systems. In Security+, you use them to reason about how a system supports the **CIA triad**: confidentiality, integrity, and availability.

These models are technology-independent. They can be implemented in operating systems, networks, database management systems, and application platforms. The key exam idea is that a model describes *how access should work*, while a policy or implementation applies that model in practice.

- **Confidentiality** focuses on preventing unauthorized disclosure.
- **Integrity** focuses on preventing unauthorized or improper modification.
- **Availability** focuses on keeping systems and data usable when needed.

**Classic models and their focus**

The classic models most often associated with Security+ are Bell-LaPadula and Biba. They are useful because they emphasize different security goals.

Bell-LaPadula focuses on **confidentiality** and controlled access to classified information. It uses layered secrecy and is commonly described with the rule set that limits reading and writing across levels.

- **Simple security rule**: no read up.
- **Star security rule**: no write down.
- Subjects can read at their own level and below, but not above.
- Subjects can write at their own level and above, but not below.

Biba focuses on **integrity** rather than confidentiality. It is often summarized as the reverse of Bell-LaPadula in terms of information flow.

- **No read down** helps prevent contamination from lower-integrity data.
- **No write up** helps prevent a lower-integrity subject from modifying higher-integrity objects.
- It is useful when data correctness matters more than secrecy.

> Use the model’s goal to remember it: Bell-LaPadula protects secrecy, while Biba protects trustworthiness of data.

**Access control model concepts**

Access control models define the rules for who can access what. They are independent of the specific technology used to enforce them.

Common access control approaches include:

- **Discretionary Access Control (DAC)**: users control access to resources they own.
- **Mandatory Access Control (MAC)**: access is based on assigned security labels and enforced rules.
- **Role-Based Access Control (RBAC)**: access is tied to a user’s role in the organization.
- **Attribute-Based Access Control (ABAC)**: access decisions use attributes such as user, resource, and context.
- **Relationship-Based Access Control (ReBAC)**: access depends on relationships between entities.

DAC is flexible and supports collaboration, but it is not ideal for highly sensitive data. MAC is stricter because users and resources are assigned security classifications. RBAC and ABAC are common in modern systems because they scale better than purely user-by-user control.

**How to compare models**

When you study security models, compare them by the security property they protect and the way they make access decisions.

| Model | Primary focus | Core idea |
|---|---|---|
| Bell-LaPadula | Confidentiality | Control information flow to prevent disclosure |
| Biba | Integrity | Prevent lower-integrity data from corrupting higher-integrity data |
| DAC | User control | Owners decide who gets access |
| MAC | Enforced policy | Labels and rules determine access |
| RBAC | Organizational roles | Permissions follow job function |
| ABAC | Attributes and context | Access depends on multiple conditions |

A practical exam approach is to identify the goal first, then match the model. If the question emphasizes secrecy, think Bell-LaPadula. If it emphasizes data correctness, think Biba. If it emphasizes ownership, roles, or attributes, think access control models.

**Study approach for Security+**

To answer exam questions quickly, focus on the relationship between the model and the security objective.

1. Identify whether the scenario is about confidentiality or integrity.
2. Look for clues about who controls access.
3. Match the access pattern to the model’s rule set.
4. Eliminate choices that protect the wrong security property.

A short memory aid helps:

- **Bell-LaPadula**: confidentiality.
- **Biba**: integrity.
- **DAC**: owner decides.
- **MAC**: system decides.
- **RBAC**: role decides.
- **ABAC**: attributes decide.

**Why these models matter**

Security+ expects you to understand how organizations protect sensitive data and enforce access rules. These models support secure systems and help align technical controls with business requirements. They also show up across operating systems, networks, and application environments, so the concepts are broadly useful beyond the exam.
