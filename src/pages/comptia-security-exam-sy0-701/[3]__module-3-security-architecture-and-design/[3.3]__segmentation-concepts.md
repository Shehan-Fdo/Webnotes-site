## **Course:** CompTIA Security+ (Exam SY0-701)

**Segmentation overview**

Network segmentation divides a network into isolated zones so you can control traffic flow, limit access, and contain breaches. The core idea is to stop an attacker from moving freely once they get inside.

Modern security design treats the internal network as untrusted. That matters because attackers often enter through a phished credential, a compromised device, or a vulnerability, then move laterally across a flat network.

- **Lateral movement** is the movement of an attacker from one internal system to another.
- **Blast radius** is the amount of damage an attacker can cause before containment stops them.
- **Flat network** design increases risk because internal systems are too easy to reach.

> Do not rely on a perimeter firewall alone. Security must exist within the network, not just at the edge.

**Why segmentation matters**

Segmentation reduces the attack surface by limiting which systems can talk to each other. It also helps protect high-value assets such as domain controllers, databases, and other sensitive internal services.

A segmented design gives you practical containment. If one host is compromised, the attacker should not automatically gain access to every other host in the environment.

- It slows intruders down.
- It limits exposure between user, server, and sensitive zones.
- It supports a defense strategy based on containment rather than trust.

**Segmentation and isolation**

Segregation and segmentation both describe ways to separate systems, but the practical goal is the same: isolate critical resources from less trusted parts of the network or from public access.

Common isolation methods include:

- Firewalls that restrict traffic between zones
- Routers that enforce path control
- Air-gapped environments that block outside network access
- Virtual local area networks (<abbr title="Virtual Local Area Network">VLANs</abbr>) that separate traffic logically

**Micro-segmentation**

Micro-segmentation is a more granular form of segmentation. Instead of separating only large network zones, it narrows access further so workloads and services have only the connections they need.

This approach is useful when you want tighter control over east-west traffic, which is traffic moving between internal systems.

- **East-west traffic** is internal traffic between hosts, servers, and services.
- Micro-segmentation helps contain compromise inside a smaller boundary.
- It is a modern extension of the same containment goal used in broader segmentation.

**Network design perspective**

Security is part of network design at every layer, including local area network (<abbr title="Local Area Network">LAN</abbr>), data center, and wide area network (<abbr title="Wide Area Network">WAN</abbr>) environments. The goal is not just to place controls at the perimeter, but to build security into the architecture itself.

A practical design uses segmentation to create obstacles that slow an attacker long enough for detection and response.

**Implementation focus**

When you implement segmentation, think in terms of zones and policy enforcement.

1. Identify the systems or data that need stronger isolation.
2. Group assets into logical zones based on trust and function.
3. Apply controls such as VLANs, firewalls, or routing rules to restrict traffic.
4. Test whether allowed communication still works and blocked communication stays blocked.
5. Review the design regularly as systems and risk change.

**Exam takeaways**

- Segmentation limits access and contains breaches.
- Micro-segmentation adds finer-grained control.
- Internal networks are not automatically trusted.
- The main security benefit is reducing lateral movement.
- VLANs and firewall rules are common tools for enforcing separation.
