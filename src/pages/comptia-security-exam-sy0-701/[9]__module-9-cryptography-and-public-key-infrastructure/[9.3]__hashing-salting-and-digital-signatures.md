## **Course:** CompTIA Security+ (Exam SY0-701)

**Hashing fundamentals**

A cryptographic hash turns data into a fixed-length output that acts like a message digest or fingerprint. You use it to represent data, not to hide it for later recovery. That distinction matters: hashing is **not encryption**, and you cannot reconstruct the original data from the hash alone.

Hashing supports integrity checks in practical workflows. For example, you can compare a downloaded file’s hash against the publisher’s expected value to confirm the file has not changed.

- **Integrity** is the main goal of hashing in Security+ contexts.
- Hashes are one-way functions.
- A hash can represent a document, password, or other data without revealing the original content.
- SHA-256 is a common example of a hashing algorithm.

**SHA family overview**

The Secure Hash Algorithms (SHA) are a family of cryptographic hash functions published by the National Institute of Standards and Technology (NIST) as a Federal Information Processing Standard (FIPS). Security+ focuses most often on SHA-2 in modern use.

- SHA-1 is a 160-bit hash function.
- SHA-1 has known cryptographic weaknesses and is no longer approved for most cryptographic uses after 2010.
- SHA-2 includes related hash functions such as SHA-256 and SHA-512.
- SHA-256 uses 32-bit words, while SHA-512 uses 64-bit words.
- Truncated variants also exist, including SHA-224, SHA-384, SHA-512/224, and SHA-512/256.

> Use conservative exam language: treat SHA-1 as legacy and avoid it for modern security design.

**Password hashing and salting**

Password hashing converts a plaintext password into a fixed-length string using a hashing algorithm. The goal is to store passwords in a form that is difficult to reverse, even if an attacker gains access to the stored hashes.

Salting adds randomness to the password hashing process. A salt makes identical passwords produce different hashes, which helps defend against rainbow table attacks.

- **Password hashing** protects stored credentials.
- **Salting** adds uniqueness and randomness.
- Salting makes precomputed attack tables less effective.
- Common password-hashing choices include bcrypt and Argon2.
- Password-hashing algorithms are designed to be computationally intensive.

**Digital signatures**

Digital signatures use hashing as part of the signing process. You hash the data first, then sign the hash with cryptographic key material. This gives you a way to verify that the data came from the expected source and was not altered.

Digital signatures provide three core security properties:

- **Authentication** confirms the signer.
- **Integrity** confirms the data has not changed.
- **Non-repudiation** helps prevent the signer from denying the action later.

A digital signature is not the same as a hash. The hash is only one input to the signature process, and the signature adds trust and verification through cryptographic signing.

**How to think about the relationships**

Hashing, salting, and digital signatures solve different problems, but they often appear together in security systems.

- Hashing checks whether data changed.
- Salting strengthens password storage.
- Digital signatures prove origin and support integrity.
- Encryption protects confidentiality, which is a different goal from hashing.

**Exam-ready distinctions**

Use these distinctions to avoid common Security+ confusion:

- A hash is a fixed-length representation of data.
- A hash cannot be reversed into the original input.
- Salting is used with password hashing, not with general file integrity checks.
- Digital signatures rely on hashing but add authentication and non-repudiation.
- Encryption and hashing are different processes with different goals.

**Practical review points**

When you see a scenario, match the tool to the goal:

- If the goal is to verify a file download, think **hashing**.
- If the goal is to store passwords more safely, think **hashing plus salting**.
- If the goal is to prove who signed data and whether it changed, think **digital signatures**.
- If the goal is to keep data secret, think **encryption**.
