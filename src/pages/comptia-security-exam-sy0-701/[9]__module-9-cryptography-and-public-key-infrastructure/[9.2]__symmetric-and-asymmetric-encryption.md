## **Course:** CompTIA Security+ (Exam SY0-701)

**Encryption overview**

Encryption converts readable data, or *plaintext*, into unreadable data, or *ciphertext*, so only authorized parties can recover the original information. In Security+, the key distinction is whether the same key is used on both sides or whether a key pair is used.

- **Symmetric encryption** uses one shared key for encryption and decryption.
- **Asymmetric encryption** uses a public/private key pair.
- Encryption supports confidentiality and is commonly used for data at rest and data in transit.

**Symmetric encryption**

Symmetric encryption is the faster and more efficient option for large amounts of data. Because both parties must use the same secret key, the main challenge is secure key sharing.

- **One key** handles both encryption and decryption.
- It is practical for bulk data, file encryption, VPNs, and secure storage.
- Common algorithms include `AES`, `DES`, and `Blowfish`.
- It is less secure in practice if key distribution is weak, because the shared key must be protected during transfer.

> Use symmetric encryption when performance matters and you can safely manage the shared secret.

**Asymmetric encryption**

Asymmetric encryption uses two related keys: one public and one private. The public key can be shared, while the private key stays protected by the owner. This design reduces the need to exchange a secret key directly, but it adds computational overhead.

- **Public key** encrypts or verifies, depending on the use case.
- **Private key** decrypts or signs, depending on the use case.
- It is slower than symmetric encryption and is not ideal for encrypting large data sets directly.
- It is commonly associated with key exchange, certificates, and trust relationships in public key infrastructure (PKI).

**Symmetric vs. asymmetric at a glance**

| Aspect | Symmetric | Asymmetric |
|---|---|---|
| Keys used | One shared key | Public/private key pair |
| Speed | Faster | Slower |
| Best for | Large data volumes | Key exchange and trust workflows |
| Main challenge | Safe key sharing | Higher computing overhead |

**How they work together**

In practice, systems often combine both methods. Asymmetric encryption helps establish trust or exchange a secret, and symmetric encryption then handles the bulk data because it is more efficient.

- Use asymmetric methods to support secure setup and identity trust.
- Use symmetric methods for the actual data transfer when speed matters.
- This approach balances security and performance.

**Security+ exam focus**

For exam questions, look for clues about data size, device limits, and performance.

- If the scenario mentions limited computing resources or large data, choose **symmetric encryption**.
- If the scenario emphasizes identity, certificates, or secure key exchange, think **asymmetric encryption**.
- If the question asks about public trust relationships, connect the concept to **PKI**.

**Key takeaways**

Encryption protects data by making it unreadable to unauthorized users. Symmetric encryption is fast and efficient but depends on secure key sharing. Asymmetric encryption is more flexible for trust and key exchange, but it costs more processing power. Security+ questions often test your ability to choose the right method for the job.
