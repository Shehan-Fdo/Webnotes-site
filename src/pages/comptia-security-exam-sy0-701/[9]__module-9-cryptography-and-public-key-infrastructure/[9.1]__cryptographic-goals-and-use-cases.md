## **Course:** CompTIA Security+ (Exam SY0-701)

**Cryptography goals**

Cryptography is the use of mathematical methods to protect information in transit and at rest. In Security+, the main goals are to preserve **confidentiality**, support **integrity**, and strengthen **trust** in digital communications and transactions.

It works by transforming readable data, or plaintext, into unreadable data, or ciphertext, so only intended recipients can recover it. This protects sensitive data from unauthorized access and helps secure communication across networks.

- **Confidentiality** keeps data hidden from unauthorized parties.
- **Integrity** helps detect tampering or unauthorized changes.
- **Authenticity** supports confidence that data or a sender is genuine.
- **Availability** is part of the broader security goal set, even though cryptography mainly focuses on protecting data and communications.

**Core cryptographic use cases**

Cryptography appears in many everyday security controls. It protects digital identities, transactions, communications, and infrastructure against prying eyes and tampering.

Common use cases include:

- **Encrypted communication** over networks, such as secure transport for data in motion.
- **Encryption at rest** for stored files, databases, and devices.
- **Digital trust** for systems that need to verify identity or message origin.
- **Compliance support** when organizations must protect sensitive information such as personal or financial data.
- **Key management** to control how cryptographic keys are created, stored, and used.

**How cryptography supports security**

Cryptography is a foundation for modern cybersecurity because it helps keep information private while also making it harder to alter without detection. It is especially important when organizations exchange sensitive information across public or untrusted networks.

A practical way to think about it is:

- Data is protected while it moves between systems.
- Data is protected while it is stored.
- Trust is established when parties can verify what they receive.
- Security controls become more effective when cryptography is paired with good operational practices.

**Common cryptographic approaches**

Different cryptographic methods solve different problems. Security+ expects you to recognize the role each one plays.

| Approach | Main purpose | Typical use |
|---|---|---|
| Symmetric-key cryptography | Fast encryption and decryption with one shared key | Bulk data protection |
| Asymmetric-key cryptography | Uses a public/private key pair | Secure key exchange and digital signatures |
| Hash functions | Produce a fixed output for data integrity checks | Detecting changes to data |

Symmetric-key cryptography is efficient for large amounts of data because the same key is used for both encryption and decryption. Asymmetric-key cryptography is useful when two parties need to exchange information securely without first sharing a secret key. Hashing does not encrypt data; instead, it helps verify whether data has changed.

**Threats cryptography helps address**

Cryptography is not a complete defense by itself, but it reduces the impact of common attack types. Passive attacks involve monitoring or intercepting data without changing it, while active attacks involve modifying, deleting, or disrupting data.

- **Passive attacks** include eavesdropping and traffic analysis.
- **Active attacks** include data modification and denial-of-service attempts.
- **Encryption** helps limit the value of intercepted data.
- **Integrity controls** help reveal unauthorized changes.

> Cryptography protects data, but it does not replace access control, secure configuration, or strong key handling.

**Exam focus reminders**

For Security+, focus on the relationship between the cryptographic goal and the problem it solves.

- Use **confidentiality** when the goal is to keep data private.
- Use **integrity** when the goal is to detect tampering.
- Use **authentication** or **authenticity** when the goal is to verify identity or origin.
- Use **encryption** for readable data that must become unreadable to outsiders.
- Use **hashing** when you need a change-detection mechanism rather than secrecy.

**Quick mental model**

Think of cryptography as a set of tools that protect information at rest, in transit, and during verification. The right tool depends on whether you need secrecy, tamper detection, identity assurance, or secure exchange.
