## **Course:** CompTIA CySA+ Blue Team Operations Labs (Exam CS0-003)

**ATT&CK Navigator overview**

MITRE ATT&CK<sup><a href="#fn1">1</a></sup> is a globally accessible knowledge base of adversary tactics and techniques based on real-world observations. The ATT&CK Navigator is a web-based tool that helps you visualize that data in a more practical way.

Use it when you need to:

- Explore tactics, techniques, and procedures (TTPs) in a structured view.
- Create customizable visual representations of ATT&CK data.
- Support planning, red team operations, and defensive analysis.
- Highlight where your security controls align with attacker behavior.

The Navigator is designed for analysis and communication, not just browsing. A layer turns ATT&CK data into an annotated view that you can save, share, and reuse.

**What a layer is**

A layer is a set of annotations on the ATT&CK matrix for a specific technology domain. It can also store view settings such as sorting and visible platforms.

In practice, a layer helps you answer questions like:

- Which techniques matter most for this environment?
- Where do you already have coverage?
- Which areas still need attention?

A layer is useful because it adds context to the matrix instead of showing only the raw ATT&CK layout.

**Layer creation workflow**

When you build a layer, you typically start with the ATT&CK matrix and then add annotations that reflect your environment or analysis goals.

1. Open the ATT&CK Navigator and choose the matrix or domain you want to work with.
2. Add annotations for techniques that are relevant to your analysis.
3. Adjust the view settings, such as sorting or visible platforms, if needed.
4. Export the annotations into a layer file when you want to save or share the result.
5. Import a layer file later if you want to review or reuse the same view.

The Navigator supports both exporting annotations into layer files and importing layer files for viewing. That makes it practical for repeatable analysis and collaboration.

**Coverage mapping**

Coverage mapping is the process of comparing your defensive capabilities against ATT&CK techniques. The goal is to see where your controls, monitoring, or detection logic provide visibility and where gaps remain.

A coverage-focused layer can help you:

- Mark techniques that are covered by existing controls.
- Identify techniques that need additional monitoring.
- Prioritize defensive work based on observed or expected adversary behavior.
- Communicate security posture in a visual format.

This fits CySA+ work because the certification emphasizes incident detection, prevention, response, continuous security monitoring, and vulnerability management. ATT&CK-based mapping supports those activities by making defensive coverage easier to discuss and prioritize.

**Practical exam focus**

For exam-style questions, connect the tool to its purpose:

- ATT&CK provides a framework for understanding attacker behavior.
- Navigator provides a way to visualize and annotate that framework.
- Layers provide the saved, customized view.
- Coverage mapping uses those layers to compare defenses against techniques.

If a question asks what the Navigator helps you do, think in terms of visualization, annotation, and coverage analysis rather than raw threat intelligence storage.

**Quick comparison**

| Concept | Purpose |
|---|---|
| ATT&CK framework | Describes adversary tactics and techniques |
| ATT&CK Navigator | Visualizes and annotates ATT&CK data |
| Layer | Saves a customized annotated view |
| Coverage mapping | Compares defenses to techniques and highlights gaps |

**Key takeaways**

- Use ATT&CK Navigator to create a visual, customizable view of ATT&CK data.
- Use layers to store annotations and view settings for a specific domain.
- Use coverage mapping to compare your controls against techniques and identify gaps.
- Treat the tool as a planning and analysis aid for defensive operations.
