## **Course:** CompTIA Network+ Hands-On Infrastructure (Exam N10-009)

**DHCP relay overview**

Dynamic Host Configuration Protocol (DHCP) automatically assigns IP configuration to clients, including the address, subnet mask, DNS settings, and default gateway. In routed networks, DHCP relay lets clients reach a DHCP server that is not on the local subnet. This is useful when you want one central server to serve multiple VLANs or network segments.

A relay agent forwards client requests between the local broadcast domain and the DHCP server. On Cisco devices, this is commonly configured with an IP helper address. The relay function reduces the need to place a DHCP server on every subnet.

- **DHCP** automates address assignment and related network settings.
- **Relay agent** forwards DHCP traffic across subnets.
- **IP helper address** is the common Cisco relay configuration method.
- **Centralized DHCP** simplifies administration across multiple VLANs.

> DHCP relay is only needed when the client and server are separated by a router or Layer 3 boundary. On the same subnet, clients can usually reach the server without relay.

**How DHCP lease assignment works**

When a client requests an address, the DHCP server checks its configured scope or pool. The scope defines the range of addresses available for assignment, along with exclusions and other options. The server also tracks lease duration, which determines how long a client can keep the assigned address before renewal.

Professor Messer’s Network+ material highlights the main server-side items you should recognize:
- **Scope or pool** defines the available address range.
- **Exclusions** keep specific addresses out of the pool.
- **Lease duration** controls how long the assignment remains valid.
- **Options** can include DNS servers, default gateways, and other settings.

A practical way to think about lease assignment is that the server offers an address from the pool, the client accepts it, and the lease timer starts. If the lease is renewed successfully, the client keeps the same address for another period.

**Configuring DHCP relay**

On Cisco IOS, you configure relay on the interface that receives the client broadcast. The relay agent then forwards the request toward the DHCP server. The exact syntax can vary by platform and IOS version, but the core idea is the same: point the interface to the server or servers that should receive the forwarded DHCP traffic.

```bash
interface vlan 10
 ip helper-address 192.0.2.10
```

Use this pattern when:
- Clients are on a different subnet than the DHCP server.
- You want one DHCP server to serve multiple VLANs.
- You need to reduce broadcast-based DHCP limitations across routed networks.

**Verifying lease assignment**

After configuration, verify that clients receive an address from the expected scope. You should confirm both the relay path and the resulting lease on the client side. If the client gets an address, the relay path and server scope are likely working together correctly.

1. Connect a client to the target subnet and request network settings.
2. Check that the client receives an IP address, subnet mask, gateway, and DNS information.
3. Confirm the address comes from the intended DHCP scope or pool.
4. If the client does not receive a lease, review the relay configuration and the server-side scope.

Useful verification checks include:
- Client address is in the correct subnet.
- Default gateway matches the local network design.
- DNS settings match the DHCP options you configured.
- Lease time appears reasonable for the environment.

**Troubleshooting focus**

If lease assignment fails, start with the relay interface and the DHCP server scope. A relay misconfiguration prevents requests from reaching the server, while a scope problem prevents the server from offering an address even when requests arrive.

Common practical checks:
- Confirm the helper address points to the correct DHCP server.
- Confirm the client-facing interface is the one with relay enabled.
- Confirm the DHCP scope includes usable addresses.
- Confirm exclusions do not remove the entire usable range.

**Exam takeaway**

For Network+ hands-on tasks, focus on the relationship between routed subnets, relay forwarding, and lease verification. You are not just configuring DHCP; you are proving that clients on one network can obtain valid leases from a server on another network.
