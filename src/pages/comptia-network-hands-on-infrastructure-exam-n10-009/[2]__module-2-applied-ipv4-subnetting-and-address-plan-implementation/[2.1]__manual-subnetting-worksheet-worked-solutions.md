## **Course:** CompTIA Network+ Hands-On Infrastructure (Exam N10-009)

**Subnetting worksheet goal**

Manual subnetting practice helps you move from memorizing masks to solving address ranges quickly. For each target IP and CIDR notation, you work out the network ID, first host IP, last host IP, broadcast IP, and next network address.

The practice flow is consistent across worksheet-style questions:

- Start with the target IP and prefix length.
- Identify which octets are part of the subnet calculation.
- Solve the address boundaries for the subnet.
- Verify the next network by moving to the following block.

**What you are expected to calculate**

Subnet practice tools for N10-009 commonly ask you to determine five values for each target address:

- **Network ID**: the first address in the subnet.
- **First Host IP**: the first usable address.
- **Last Host IP**: the last usable address.
- **Broadcast IP**: the final address in the subnet.
- **Next Network**: the first address of the following subnet.

These questions may present a random IP address and CIDR value each time, so the skill you build is pattern recognition, not memorization of one fixed example.

**How the worksheet logic works**

A subnetting worksheet is easiest when you treat the prefix as the boundary that separates network bits from host bits. The subnet mask tells you which part of the address stays fixed for the subnet and which part changes across hosts.

Useful practice behaviors include:

- Filling in the non-interesting octets automatically when the prefix makes them obvious.
- Checking each row only after all four octets are entered.
- Using keyboard shortcuts in browser-based practice tools to move faster during drills.

> When you practice manually, always confirm that the broadcast address is the last address in the subnet and that the next network begins immediately after it.

**Worked-solution approach**

Use the same process every time so your answers stay consistent under exam pressure.

1. Read the target IP and CIDR prefix.
2. Determine which octet contains the subnet boundary.
3. Find the network ID by setting host bits to zero.
4. Find the broadcast IP by setting host bits to one.
5. Derive the first host and last host from those boundary addresses.
6. Compute the next network by adding one block past the broadcast address.

This method works well because it keeps the calculation order stable, even when the address changes from question to question.

**Practice environment features**

Some subnetting practice tools are designed to reduce repetitive typing and speed up review.

- **Auto-fill non-interesting octets** pre-fills octets outside the subnet calculation.
- **Auto-check** validates a row after all four octets are entered.
- **Keyboard shortcuts** let you enter answers faster in desktop practice sessions.

These features support repetition, but they do not replace the need to understand the address math.

**Exam-focused reminders**

Subnetting appears in practice tests and study materials for Network+ N10-009, so you should be comfortable with both the binary mask view and the address-range view. The exam may test your ability to interpret subnet masks, identify ranges, and reason about usable host addresses.

- Know how to read a CIDR prefix.
- Know how to identify the network and broadcast boundaries.
- Know how to derive usable host range from those boundaries.
- Know how to move from one subnet to the next.

**Quick self-check**

Use this checklist after each worksheet problem:

- The network ID is the first address in the subnet.
- The broadcast IP is the last address in the subnet.
- The first host is one address after the network ID.
- The last host is one address before the broadcast IP.
- The next network starts after the broadcast IP.

**Study habit**

Short, repeated drills are more effective than long one-time sessions. A configurable subnetting quiz helps you practice many address combinations, which builds speed and confidence for exam-style questions.
