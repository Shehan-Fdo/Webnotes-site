## **Course:** CompTIA Network+ Hands-On Infrastructure (Exam N10-009)

**Addressing conflicts overview**

An IP address conflict happens when two devices use the same IP address on the same network. This breaks normal delivery because the network cannot reliably determine which device should receive traffic.

Subnet mask errors create a different but related problem. Devices may be on the same physical network but calculate different network boundaries, so one host sends traffic to the gateway while the other tries to send directly.

- **IP conflict**: two devices share one address on the same network.
- **Subnet mask**: defines which part of the address is the network and which part is the host.
- **Default gateway**: becomes important when a host believes the destination is remote.
- **Asymmetric behavior**: one host may send traffic directly while the other routes it.

**Why conflicts happen**

The evidence points to a few common causes of duplicate addressing and mask problems. Static configuration mistakes are a frequent source of duplicate IPs, and preset addresses on devices such as IoT equipment can also collide if they are not changed before deployment.

Mask errors often come from entering the wrong subnet mask or pairing a network address with a mask that does not match the intended subnet. On Cisco devices, this can trigger `% Inconsistent address and mask` when the network portion in a route command is not valid for the mask.

- Manual static assignment can duplicate an address.
- Default or preset device addresses can overlap.
- A wrong mask can make hosts disagree about local versus remote destinations.
- A route command can fail if the network address and mask do not align.

**How mask mismatches affect communication**

A subnet mask mismatch can make two hosts interpret the same IP range differently. For example, one device may treat `192.168.1.20` as local with a `/24` mask, while another may treat the same destination as remote with a `/16` mask. That mismatch leads to inconsistent forwarding decisions and no reliable communication.

The practical takeaway is simple: if hosts on the same physical segment cannot talk, verify that they agree on the subnet mask before you assume a cabling or switching problem.

- Compare the configured mask on each host.
- Confirm that the mask matches the intended subnet size.
- Check whether one device is sending to the gateway while the other is trying direct delivery.
- Verify the router or switch interface address if the issue involves infrastructure.

**Verification commands**

Use platform-specific commands to inspect the current addressing state. The goal is to confirm the IP address, subnet mask, and interface configuration before making changes.

```bash
# Linux
ip addr show eth0
ifconfig eth0

# Windows
ipconfig /all | findstr "Subnet Mask"

# Cisco IOS
show interfaces GigabitEthernet0/0
```

- `ip addr show` displays the interface address and prefix on Linux.
- `ipconfig /all` shows the subnet mask on Windows.
- `show interfaces` confirms the configured address and prefix on Cisco devices.
- `nmap -sn` can help you scan a subnet and compare live hosts when you suspect a mismatch.

**Troubleshooting workflow**

1. Identify the affected devices and confirm whether the issue is a duplicate IP or a mask mismatch.
2. Check each device’s configured IP address and subnet mask.
3. Compare the values against the intended network design.
4. Correct any duplicate static assignments or preset device addresses.
5. Re-test connectivity after the change.

> If a Cisco route command returns `% Inconsistent address and mask`, recheck the network address and subnet mask combination before retrying the configuration.

**Practical interpretation of the errors**

An IP conflict usually points to uniqueness problems: two devices claim the same address. A mask error usually points to boundary problems: devices disagree on which addresses belong to the local subnet.

That distinction helps you troubleshoot faster:

- Duplicate address issues focus on uniqueness.
- Mask issues focus on subnet definition.
- Both can produce connectivity failures even when the physical link is up.

**Exam focus**

For Network+ troubleshooting, be ready to:

- Recognize symptoms of duplicate IP addressing.
- Explain how subnet masks affect local and remote traffic decisions.
- Use basic commands to verify addressing on common platforms.
- Interpret a Cisco `% Inconsistent address and mask` message as a sign of invalid network and mask pairing.
