## **Course:** CompTIA Network+ Hands-On Infrastructure (Exam N10-009)

**Subnet calculations become lab addressing**

Subnetting gives you the math; lab addressing turns that math into a usable plan for devices, interfaces, and test networks. In practice, you map calculated subnets to specific segments, then assign host addresses, gateways, and loopbacks in a way that is easy to document and expand.

A good lab plan uses private address space, avoids overlap, and keeps the structure consistent across similar segments. That makes it easier to configure devices, troubleshoot conflicts, and scale the design without renumbering.

- Use **private address space** from RFC 1918 when building lab and enterprise plans.
- Keep each site, building, or zone in a **summarizable block**.
- Leave **growth room** by allocating more space than you need today.
- Reuse the same subnet size for the same purpose across sites.
- Reserve ranges that do not conflict with partners or VPN peers.

**Planning the address pool**

When you convert subnet calculations into a lab layout, start with the larger structure before assigning individual hosts. The evidence emphasizes hierarchical planning: choose a block, divide it by site or function, and keep the resulting prefixes easy to summarize in routing tables.

<dl>
<dt>Network part</dt>
<dd>The prefix portion of an IP address that identifies the subnet.</dd>
<dt>Host part</dt>
<dd>The portion used for device addresses inside that subnet.</dd>
<dt>Subnetting</dt>
<dd>Borrowing bits from the host portion to create smaller networks.</dd>
</dl>

- Pick a private block that fits the lab’s scale.
- Divide it into subnet sizes that match the required host counts.
- Keep related segments aligned so they can be summarized cleanly.
- Document which subnet belongs to each device role or topology segment.

**Turning subnet math into assignments**

A subnet calculation is only useful once you map it to actual interfaces. In lab work, that usually means assigning addresses to router interfaces, switch management interfaces, host NICs, and any loopbacks used to simulate extra networks.

> Do not assign addresses ad hoc. If you skip the plan, you increase the risk of overlap, IP conflicts, and confusing troubleshooting later.

Typical lab addressing follows a repeatable pattern:

- Assign the first usable address to the default gateway when that helps keep the plan consistent.
- Reserve predictable ranges for infrastructure devices.
- Use loopback interfaces to simulate additional LANs when the lab requires them.
- Keep host assignments within the correct subnet boundary.

**Example workflow for a lab topology**

1. Identify the required subnets from the host and segment counts.
2. Choose a private base network that leaves room for expansion.
3. Allocate one subnet per segment or site.
4. Assign interface IP addresses from the correct subnet.
5. Record the subnet mask and default gateway for each host.
6. Test connectivity and adjust only if the plan does not match the topology.

**Common design habits that help in labs**

Structured IP planning is not just for large enterprises. The same habits make hands-on labs easier to complete and verify. A consistent scheme reduces mistakes when you configure multiple devices and helps you spot incorrect masks or overlapping ranges faster.

- Keep LAN, WAN, and data center-style segments separate when the topology calls for them.
- Use the same addressing pattern for similar devices.
- Leave unused addresses available for future hosts or interfaces.
- Verify that each subnet supports the number of hosts you expect.

**Troubleshooting the converted plan**

If a lab fails after you assign addresses, check the plan before changing device settings. Most issues come from mismatched masks, wrong gateways, or overlapping subnets rather than from the routing protocol itself.

- Confirm the address belongs to the intended subnet.
- Check that the subnet mask matches the calculated prefix.
- Verify the default gateway is in the same subnet as the host.
- Look for duplicate addresses or ranges that overlap.

**Key takeaway**

Subnet calculations tell you how to divide the network; lab addressing tells you how to apply that division to real interfaces. The best plans are structured, summarizable, and consistent, with enough spare space to support future growth.
