## **Course:** CompTIA Network+ Hands-On Infrastructure (Exam N10-009)

**Script-based configuration overview**

Scripts help you generate device commands or apply configuration changes consistently across routers and switches. In the evidence, Python and PowerShell are used to create Cisco IOS command sets, and Python is also presented as a practical way to automate network device configuration tasks.

This approach is useful when you want to reduce repetitive manual work and keep configuration patterns consistent. It is especially relevant when you are building templates for devices that share similar settings.

- **Python** can generate commands for network devices.
- **PowerShell** can also be used to build configuration output.
- Templates help you reuse a known-good configuration pattern.
- Scripted workflows support repeatable changes across multiple devices.

**Why automation matters**

Manual configuration is slower and more error-prone than scripted generation. The evidence shows that automation is used to simplify network engineering tasks and to create configuration commands for Cisco routers and switches running IOS.

A practical benefit is that you can prepare configuration text before applying it to a device. That makes it easier to review changes, standardize settings, and reduce typing mistakes.

- Scripts support repeatability.
- Generated output is easier to review before deployment.
- Configuration templates reduce per-device effort.
- Automation is useful even when you are not using a larger orchestration platform.

**Secure remote access for applying changes**

When you apply configuration remotely, use a secure management method. The evidence contrasts Telnet with Secure Shell (SSH), noting that Telnet does not encrypt traffic while SSH encrypts information and provides authentication.

SSH is commonly used to log in to a remote device and execute commands. It can also support file transfer through related tools, which makes it useful for operational workflows beyond interactive access.

> Use SSH instead of Telnet when you need encrypted remote administration and authenticated access.

- **SSH** protects credentials and configuration data in transit.
- Telnet exposes management traffic to interception.
- Remote command execution is a common SSH use case.
- Secure access is important when scripts push or trigger configuration changes.

**Typical script-driven workflow**

1. Define the configuration goal for the device or device group.
2. Build a reusable template or script that generates the required commands.
3. Review the generated output before applying it.
4. Connect to the device using a secure method such as SSH.
5. Apply the configuration and verify the result.

This workflow keeps generation and application separate. That separation helps you catch mistakes before they reach the device.

**What to watch for**

Scripted configuration is powerful, but it still depends on correct input and careful review. If a template is wrong, the script can reproduce the same mistake across many devices.

- Validate variable values before generating output.
- Check device-specific differences before reuse.
- Confirm that the remote access method is secure.
- Review generated commands before execution.

**Practical exam focus**

For Network+ purposes, focus on the relationship between automation, configuration consistency, and secure management. You should understand that scripts can generate or apply device configurations, and that SSH is the secure remote access method highlighted in the evidence.

- Scripts can generate configuration commands.
- Scripts can help apply changes consistently.
- SSH is the secure remote access protocol in the evidence.
- Telnet is insecure because it does not encrypt traffic.
