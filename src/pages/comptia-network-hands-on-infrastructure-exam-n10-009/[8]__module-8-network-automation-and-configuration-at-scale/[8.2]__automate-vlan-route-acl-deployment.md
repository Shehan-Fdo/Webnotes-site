## **Course:** CompTIA Network+ Hands-On Infrastructure (Exam N10-009)

**Automation goals for network deployment**

Infrastructure as Code (IaC) describes network technology in code so you can build, version, and redeploy configurations instead of entering them manually. For VLANs, routes, and access control lists (ACLs), the main benefit is repeatability: you define the intended configuration once and apply it consistently across devices.

Automation tools in the evidence include **Netmiko** and **NAPALM**, which use SSH connections to push configuration changes to Cisco devices. The practical value is reduced manual effort, faster setup, and fewer mistakes when you need to make the same change on many devices.

- **VLANs** separate broadcast domains on a switch.
- **Routes** can be deployed as part of a broader infrastructure configuration.
- **ACLs** can be generated and applied automatically, including interface and direction placement.
- **Source control** helps you keep versions of the configuration and redeploy changes safely.

**Key automation concepts**

Declarative automation focuses on the desired end state rather than the step-by-step process. In practice, that means you describe what the network should look like after the change, and the automation system calculates the needed difference.

Idempotent configuration means you can apply the same configuration more than once without creating unintended extra changes. This matters when you automate VLAN, route, or ACL deployment because you want the result to stay stable even if you rerun the script or playbook.

<dl>
<dt>Declarative</dt>
<dd>You define the final state you want.</dd>
<dt>Imperative</dt>
<dd>You define the exact steps to reach the state.</dd>
<dt>Idempotent</dt>
<dd>Repeated application produces the same intended result.</dd>
</dl>

> Use conservative change control when automating network policy. A script that is fast but inaccurate can create outages just as quickly as manual work.

**VLAN deployment patterns**

VLAN automation is a common example of repeatable network configuration. The evidence shows Python with Netmiko used to automate VLAN creation, which fits the broader IaC model of describing and applying network state as code.

When you automate VLAN deployment, you typically want to reduce repetitive switch work and keep the configuration consistent across sites or devices. This is especially useful when the same VLAN structure must be created on multiple switches.

- Automate creation instead of entering VLANs one by one.
- Keep VLAN definitions consistent across devices.
- Use versioned configuration files so changes are traceable.
- Reapply the same configuration only when the intended state changes.

**ACL deployment patterns**

ACL automation is useful when you need to block or permit many entries at once. The evidence describes a Python and Netmiko workflow that reads IP addresses from a file, builds an extended ACL with sequence numbers, applies it to the correct interface and direction, and saves the configuration.

That pattern is practical because ACL changes are easy to mistype manually and can become time-consuming when the list is large. Automation helps you turn a long list of policy entries into a repeatable deployment process.

1. Read the source list of addresses or rules from a file.
2. Build the ACL entries in the required order.
3. Apply the ACL to the target interface and direction.
4. Save the configuration after verification.

**Automation workflow example**

A simple automation workflow usually starts with a source file or configuration template, then pushes the desired state to the device over SSH. The toolkit evidence shows this approach for Cisco devices using Netmiko and NAPALM.

```python
# Conceptual workflow based on the evidence
from netmiko import ConnectHandler

device = {
    "device_type": "cisco_ios",
    "host": "192.0.2.10",
    "username": "admin",
    "password": "password",
}

commands = [
    "vlan 50",
    "name new_example_vlan",
]

with ConnectHandler(**device) as conn:
    conn.send_config_set(commands)
    conn.save_config()
```

- Use SSH-based automation when the device supports it.
- Keep commands grouped by the change you want to deploy.
- Save the configuration after successful application.
- Prefer scripts that are easy to review and rerun.

**What to remember for exam prep**

The exam focus is not on one specific tool alone. It is on understanding why automation is used and how it changes the deployment model.

- **IaC** lets you describe network changes as code.
- **Netmiko** and **NAPALM** are examples of tools used to automate Cisco device configuration.
- **Declarative** and **idempotent** approaches reduce configuration drift.
- VLANs, routes, and ACLs are all good candidates for repeatable deployment.
- Automation is most valuable when manual configuration is slow, repetitive, or error-prone.
