## **Course:** CompTIA Network+ Hands-On Infrastructure (Exam N10-009)

**Troubleshooting focus**

Network automation with Python often uses libraries such as Paramiko, Netmiko, and NAPALM to connect to devices and run commands. These tools help you automate repetitive tasks, but failures usually fall into a few practical categories: script syntax, authentication, and transport behavior.

CompTIA Network+ emphasizes the ability to securely establish, maintain, and troubleshoot networks. In automation workflows, that means you need to separate code problems from connection problems and from device-response problems.

- **Script syntax** issues come from invalid Python structure, bad string handling, or incorrect library usage.
- **Authentication** issues happen when credentials, keys, or device access settings do not match.
- **Transport** issues involve the connection path, session behavior, or timing between your script and the device.

**Script syntax failures**

Python automation scripts can fail before they ever reach the network device. When that happens, focus on the code itself first. The evidence highlights Python as a common choice for network automation, but it also implies that library usage must match the task.

Typical checks include:

- Confirm that the script imports the right library for the job.
- Verify that command strings and parameters are written correctly.
- Check that the code path matches the library’s expected method, such as command execution versus interactive shell handling.
- Review any device-specific options passed into the library.

A practical example from NAPALM shows that a script can connect successfully and still fail later during command handling. In that case, the connection worked, but `send_command` did not detect the expected prompt pattern.

```python
# Example of a troubleshooting pattern
# Connection succeeds, but command execution waits for a prompt that never appears
device = {
    "hostname": "R4",
    "global_delay_factor": 5,
}
```

> If a script reaches the device but fails on command execution, do not assume authentication is the root cause. Check the prompt, timing, and command method first.

**Authentication failures**

Authentication problems usually appear when the script cannot establish a valid session. With agentless automation over SSH, the library must authenticate with the device using the credentials or keys you provide.

Paramiko is used for SSH-based automation, while Netmiko builds on SSH automation for network devices. That makes authentication a key checkpoint in both tools.

- Confirm that the username and password are correct.
- Verify that any SSH key or credential file is available and readable.
- Make sure the target device allows the authentication method you are using.
- Check whether the device expects a different login flow than your script assumes.

Authentication can look similar to transport failure at first, but the difference is simple: if the session never opens, start with credentials and access settings; if the session opens and then stalls, move to transport and prompt handling.

**Transport failures**

Transport failures happen after the script starts communicating with the device but cannot complete the exchange cleanly. In SSH-based automation, this often means the session opens, but command delivery or response detection does not behave as expected.

Paramiko supports both an EXEC channel for simple commands and a SHELL channel for interactive applications. That distinction matters because the wrong channel choice can make a script appear broken even when the network path is fine.

- Use the EXEC channel for simple command execution.
- Use the SHELL channel when the workflow needs interactive behavior.
- Expect timing differences between devices and adjust accordingly.
- Watch for prompt detection problems when the device response is delayed or unusual.

Netmiko exists to reduce some of these device interaction problems in network automation. It helps handle common network-device behaviors that raw SSH automation may not manage as smoothly.

**How to isolate the failure**

Use a simple, layered approach so you can identify where the problem starts.

1. Test the Python script for syntax errors before connecting to a device.
2. Verify that authentication succeeds with the expected username, password, or key.
3. Confirm that the SSH session opens and the device prompt appears.
4. Check whether the command method matches the device interaction style.
5. Adjust timing or delay settings if the device responds slowly.
6. Re-run the script after each change so you know which fix resolved the issue.

This approach keeps you from changing multiple variables at once. It also helps you distinguish between a bad script, a bad login, and a bad session exchange.

**Library selection and practical behavior**

Paramiko gives you direct SSH control, which is useful when you need low-level access. Netmiko adds network-device handling on top of SSH, which can make common automation tasks easier. NAPALM also supports device automation, but the evidence shows that even successful connections can still run into command or prompt-related issues.

- **Paramiko** is useful for direct SSH automation.
- **Netmiko** helps with network-device command handling.
- **NAPALM** can connect and retrieve facts, but command workflows may still need tuning.
- **Prompt detection** is a common source of transport-like failures in device automation.

When you troubleshoot, keep the device behavior in view. A script may be syntactically valid, authenticated correctly, and still fail because the device prompt, response timing, or command mode does not match the library’s expectations.

**Key takeaways**

Automation troubleshooting works best when you classify the failure first. Syntax problems stop the script early, authentication problems block the session, and transport problems interrupt communication after the session begins.

- Start with the simplest failure point.
- Match the library to the interaction style you need.
- Treat prompt detection and timing as first-class troubleshooting concerns.
- Use conservative changes, then retest.
