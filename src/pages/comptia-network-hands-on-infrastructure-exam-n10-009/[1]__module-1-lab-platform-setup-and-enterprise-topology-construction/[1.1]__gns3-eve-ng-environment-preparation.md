## **Course:** CompTIA Network+ Hands-On Infrastructure (Exam N10-009)

**Lab environment purpose**

GNS3 and EVE-NG are used to build virtual network labs for emulation, configuration, testing, and troubleshooting. They let you practice with small topologies on a laptop or scale up to larger designs hosted on multiple servers or in the cloud.

For exam prep, the main value is repeatable hands-on practice. You can validate network behavior without needing dedicated physical hardware, which makes it easier to rehearse routing, switching, and service integration tasks.

- **GNS3** is open source and free to download.
- **EVE-NG** is positioned as an emulated virtual environment for network, security, and DevOps use.
- Both platforms support practical lab work for network engineers and students.

**GNS3 preparation basics**

GNS3 works best when you start with the correct base components already installed. The documentation for appliance import assumes that you already have the GNS3 Graphical User Interface installed and the GNS3 VM integrated with GNS3.

A common workflow is to install the appliance from the GNS3 Marketplace or import a downloaded appliance file. Marketplace appliances use recommended settings and are thoroughly tested, which reduces setup errors compared with manual configuration.

> If you want to use IOSvL2, IOU, IOSv, IOS-XRv, or ASAv images with GNS3 on Windows or Mac, the GNS3 VM is mandatory.

- Use the GNS3 VM when the appliance requires QEMU-based virtualization support.
- Select the appliance file from your Downloads folder or local filesystem during import.
- Choose the VM version you want to install, then import the required file.
- If an image is not already present, the import flow may offer a download option, but some vendor images are not free.

**Typical GNS3 import flow**

1. Open GNS3 and choose `File` > `Import appliance`.
2. Select the appliance file from your local storage.
3. Review the server and QEMU settings shown by the wizard.
4. Select the required VM file version.
5. Click `Import` to complete the appliance installation.

```text
File > Import appliance
```

The import wizard may show different server choices depending on where the appliance runs. In the documented IOSvL2 example, the GNS3 VM is used, and the 64-bit QEMU binary is required.

**EVE-NG preparation basics**

EVE-NG is designed for virtual network and security labs, including enterprise-style topologies. The platform supports multi-vendor emulation and is used for proof-of-concept, training, and collaborative lab environments.

The public site presents EVE-NG as a downloadable platform with a professional edition available. For lab planning, the key idea is to prepare an environment that can host the images and topologies you intend to test.

- Plan for the topology size you want to run.
- Make sure your host resources are sufficient for the lab workload.
- Use EVE-NG when you want a dedicated emulation environment for network and security practice.

**Host system considerations**

Performance improves when the host has more CPU and RAM. That guidance applies to both GNS3 and similar lab platforms because virtual devices consume real system resources.

When preparing a lab machine, focus on the platform dependencies first, then the virtualization layer, then the images or appliances you want to run.

- Install the required virtualization software before importing appliances.
- Verify that the host platform supports the lab software you choose.
- Keep in mind that some optional add-ons may have licensing limits.

**Image and licensing considerations**

Not all network images are freely available. Some vendors provide images at no cost, while others require you to supply your own authorized images or licenses.

For Cisco-based labs in GNS3, you need authorized access to the images. The documentation states that GNS3 cannot provide Cisco IOS images.

- Use vendor-provided images only when you are authorized to do so.
- Expect some appliances to require downloads from the vendor site.
- Treat image availability as part of lab planning, not as an afterthought.

**Practical preparation checklist**

Before you start building topologies, confirm that the environment is ready for the images and appliances you plan to use.

1. Install the base lab platform.
2. Install or integrate the required virtualization layer.
3. Import a tested appliance or marketplace image.
4. Confirm that the correct VM or QEMU support is available.
5. Verify that you have legal access to the images you need.
6. Build a small test topology before moving to larger designs.

**Why this matters for Network+ study**

Hands-on labs help you move from theory to practice. GNS3 and EVE-NG let you rehearse the kinds of network behaviors that matter in real deployments, including topology design, device interaction, and troubleshooting.

A prepared environment reduces friction during study sessions. Instead of spending time fixing the lab itself, you can focus on learning how devices behave and how network services fit together.
