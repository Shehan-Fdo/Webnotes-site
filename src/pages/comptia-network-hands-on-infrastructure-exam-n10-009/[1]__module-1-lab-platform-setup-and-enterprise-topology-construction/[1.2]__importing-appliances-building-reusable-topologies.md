## **Course:** CompTIA Network+ Hands-On Infrastructure (Exam N10-009)

**Appliance images and lab planning**

Virtual lab platforms such as EVE-NG and GNS3 let you build practice environments with routers, switches, firewalls, and other appliances. The evidence shows that image libraries commonly include Cisco, Fortigate, Palo Alto, Sophos, and similar vendor devices, which makes it easier to assemble realistic network scenarios for study and testing.

When you work with these labs, the main goal is not just to import a device once. You want a repeatable topology that you can reuse, modify, and troubleshoot without rebuilding the entire environment each time.

- **Appliance images** are the virtual device files you add to the lab platform.
- Image repositories may include multiple vendor families, so you can mix device types in one topology.
- Reusable topologies help you save time when practicing the same network design repeatedly.
- A stable lab layout supports consistent troubleshooting and comparison across sessions.

**Choosing compatible images**

The evidence highlights that image version matters. For Cisco IOL (IOS on Linux), EVE-NG documentation recommends specific image versions and warns that older versions may not work well. It also notes that one older L3 image has a serious console freeze issue after some time running.

That means you should treat image selection as part of lab reliability, not just a download step. If a device behaves strangely, the image version may be the first thing to verify.

> Use only images that are appropriate for your platform and version. Unsupported or outdated images can cause boot failures, console problems, or unstable lab behavior.

- **IOL** stands for IOS on Linux.
- Recommended images include both L2 (Layer 2) and L3 (Layer 3) device types.
- Some newer IOL XE images are also listed for router and switch roles.
- Version notes matter because compatibility changes over time.

**Building a reusable topology**

A reusable topology is a lab design you can load again and again for different practice goals. The evidence from EVE-NG shows topology mapping features that let you assign active clickable areas to a custom logical topology, which supports a more organized lab experience.

You can think of the topology as a template:
- Place the devices you need.
- Connect them in the intended pattern.
- Save the layout so you can return to it later.
- Adjust the same base design for different exercises.

This approach is useful when you want to practice enterprise-style scenarios, such as cloud connections or multi-device labs, without recreating the full network every time.

**Topology management and lab usability**

EVE-NG screenshots and feature descriptions show that lab management includes monitoring node state, CPU and RAM utilization, and running labs. Admin users can connect in parallel sessions, stop or wipe nodes, and filter lab lists by resource usage.

These management features matter because reusable topologies are only helpful if the lab remains easy to operate. A well-structured topology should be simple to inspect, restart, and modify when you need to test a new idea.

- Monitor node state so you know which devices are running.
- Watch resource usage to avoid overloading the host.
- Use stop or wipe actions when a lab needs a clean reset.
- Keep the topology layout clear so troubleshooting stays fast.

**Practical workflow for importing and reusing labs**

1. Choose the appliance image that matches the device role you need.
2. Verify that the image version is supported or recommended for your platform.
3. Import the image into the lab environment.
4. Add the device to your topology and connect it to the other nodes.
5. Save the topology as a reusable layout or template.
6. Reopen the same design later and adjust it for the next practice task.

**What to remember for exam prep**

The Network+ exam emphasizes practical networking tools and technologies used to build secure, available networks. In hands-on lab work, that translates into knowing how to assemble devices, keep the environment stable, and reuse a topology efficiently.

- **Compatibility** is a major concern when importing appliances.
- **Repeatability** helps you practice the same design under different conditions.
- **Management tools** make it easier to monitor and maintain the lab.
- **Reusable layouts** reduce setup time and support consistent learning.

**Quick lab checklist**

- Confirm the image source and version before import.
- Match the appliance type to the intended role in the topology.
- Save the lab after the devices and links are in place.
- Reuse the topology as a base for future scenarios.
- Reset or wipe nodes when you need a clean test run.
