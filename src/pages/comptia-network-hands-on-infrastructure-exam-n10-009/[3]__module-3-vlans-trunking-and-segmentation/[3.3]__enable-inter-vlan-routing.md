## **Course:** CompTIA Network+ Hands-On Infrastructure (Exam N10-009)

**Inter-VLAN routing overview**

VLANs (Virtual Local Area Networks) separate devices into different broadcast domains on the same switching infrastructure. That improves scalability, security, and network management, but it also means devices in different VLANs cannot communicate directly without routing.

Inter-VLAN routing provides that path. In a router-on-a-stick design, one router interface carries traffic for multiple VLANs over an 802.1Q trunk, and the router uses subinterfaces to route between the VLANs.

- VLANs reduce broadcast scope and help organize network resources logically.
- Inter-VLAN routing restores communication between isolated VLANs.
- Router-on-a-stick uses one physical router link with multiple logical subinterfaces.
- 802.1Q tagging identifies which VLAN each frame belongs to on the trunk.

**Topology and addressing pattern**

A common lab layout uses two VLANs for user traffic and one native VLAN on the trunk. The router subinterfaces map to the VLANs, while switch management interfaces use an IP address in a management VLAN.

| Device | Interface | IP address | Subnet mask | Default gateway |
|---|---:|---:|---:|---:|
| R1 | `G0/0/1.3` | `192.168.3.1` | `255.255.255.0` | N/A |
| R1 | `G0/0/1.4` | `192.168.4.1` | `255.255.255.0` | N/A |
| R1 | `G0/0/1.8` | N/A | N/A | N/A |
| S1 | VLAN 3 | `192.168.3.11` | `255.255.255.0` | `192.168.3.1` |
| S2 | VLAN 3 | `192.168.3.12` | `255.255.255.0` | `192.168.3.1` |
| PC-A | NIC | `192.168.3.3` | `255.255.255.0` | `192.168.3.1` |
| PC-B | NIC | `192.168.4.3` | `255.255.255.0` | `192.168.4.1` |

The addressing pattern shows the key relationship: each host uses the router subinterface in its VLAN as the default gateway. The router then forwards traffic between VLAN 3 and VLAN 4.

**VLAN and port assignment**

The lab uses a small VLAN table to separate management, operations, parking lot, and native traffic.

- VLAN 3 is the management VLAN on both switches.
- VLAN 4 is the operations VLAN.
- VLAN 7 is the parking lot VLAN for unused access ports.
- VLAN 8 is the native VLAN on the trunk.

Port placement matters because access ports belong to one VLAN, while the trunk carries multiple VLANs between devices.

- `S1: F0/6` is assigned to VLAN 3.
- `S2: F0/18` is assigned to VLAN 4.
- `S1: F0/2-4, F0/7-24, G0/1-2` are in VLAN 7.
- `S2: F0/2-17, F0/19-24, G0/1-2` are in VLAN 7.

**Router-on-a-stick configuration**

You configure the router with subinterfaces on the physical interface connected to the trunk. Each subinterface matches a VLAN and carries the IP address used as that VLAN’s default gateway.

```bash
interface g0/0/1
 no shutdown

interface g0/0/1.3
 encapsulation dot1Q 3
 ip address 192.168.3.1 255.255.255.0

interface g0/0/1.4
 encapsulation dot1Q 4
 ip address 192.168.4.1 255.255.255.0

interface g0/0/1.8
 encapsulation dot1Q 8 native
```

- Bring up the physical router interface before relying on the subinterfaces.
- Match each subinterface to the correct VLAN ID.
- Use the native VLAN consistently on both sides of the trunk.
- Assign the gateway IP address to the subinterface, not the physical interface.

> If the native VLAN does not match on both ends of the trunk, traffic can fail or behave unexpectedly.

**802.1Q trunk between switches**

The trunk link carries tagged frames for the VLANs that need to cross the link. In this lab context, the trunk connects the switches and supports the VLANs used by the hosts and management interfaces.

- 802.1Q is the trunking method used for VLAN tagging.
- The trunk must allow the VLANs that need to traverse between switches.
- The native VLAN is untagged on the trunk.
- Trunk misconfiguration is a common cause of failed inter-VLAN communication.

**Verification and troubleshooting**

After configuration, verify that hosts can reach their default gateway and then reach devices in other VLANs. The lab objective explicitly includes verifying that inter-VLAN routing works.

1. Check that each switch port is in the intended VLAN.
2. Confirm the trunk is up and carrying the expected VLANs.
3. Verify the router subinterfaces are configured with the correct `encapsulation dot1Q` values.
4. Test connectivity from a host to its gateway.
5. Test connectivity across VLANs, such as from `PC-A` to `PC-B`.

Successful verification shows that segmentation still exists at Layer 2, but Layer 3 routing now allows controlled communication between VLANs.

**Key takeaways**

Inter-VLAN routing solves the communication gap created by VLAN segmentation. In a router-on-a-stick design, one router interface plus subinterfaces can route traffic for multiple VLANs over a trunk. The most important checks are VLAN membership, trunk tagging, native VLAN consistency, and correct default gateways.
