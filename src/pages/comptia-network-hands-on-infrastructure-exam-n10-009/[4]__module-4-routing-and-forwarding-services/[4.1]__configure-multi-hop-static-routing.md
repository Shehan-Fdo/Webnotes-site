## **Course:** CompTIA Network+ Hands-On Infrastructure (Exam N10-009)

**Multi-hop static routing overview**

Static routing uses manually defined paths instead of dynamic route discovery. In a multi-hop design, traffic must cross more than one router before it reaches the destination network, so you need routes that point packets toward the correct next router along the path.

A router makes forwarding decisions by consulting its **routing table**. When you configure a static route, you tell the router where to send traffic for a destination network, often by specifying a next-hop IP address. This is the practical foundation for routing traffic across multiple routers.

- Static routes are manually configured.
- A **next hop** is the IP address of the adjacent router used to reach a destination.
- Multi-hop paths require each router to know how to forward traffic toward the next segment.
- Routing verification starts with the routing table and connectivity tests.

**Choosing the next hop**

Cisco documentation describes static routes that use a next-hop IP address. This approach is useful when you want the router to forward traffic to the neighboring router that can continue delivery toward the destination. In a multi-hop path, each router only needs to know the next step, not the entire route.

When you design the route, keep the path simple and predictable:

- Point the route at the next router on the path.
- Confirm that the next-hop address is reachable from the local router.
- Use the routing table to confirm the route is installed as expected.
- Test end-to-end reachability after configuration.

> If the next hop is wrong or unreachable, the route will not move traffic toward the destination as intended.

**Three-router static route lab pattern**

A common lab pattern uses three routers and multiple networks that must all communicate. The goal is to configure static routes so that routers and hosts can reach one another across the full path. This reinforces how multi-hop forwarding works in practice.

The basic workflow is straightforward:

1. Identify the destination networks that are not directly connected.
2. Configure static routes on each router so traffic can move toward the next hop.
3. Verify that each router can forward packets across the chain.
4. Test connectivity with ping or similar reachability checks.

This type of lab helps you see that static routing is not just about one router knowing one destination. It is about building a complete forwarding path across several devices.

**Verification and troubleshooting**

Routing troubleshooting begins with the routing table because that is where the router decides how to forward traffic. If packets do not reach the destination, check whether the route exists, whether the next hop is correct, and whether the path is consistent across all routers involved.

Useful checks include:

- Confirm the destination network appears in the routing table.
- Confirm the next-hop IP address is valid and reachable.
- Check each router along the path for a matching static route.
- Test connectivity from both routers and hosts to isolate the break point.

**Key takeaways**

Multi-hop static routing depends on careful next-hop selection and consistent route configuration across routers. You do not need dynamic route exchange to move traffic across several hops, but you do need every router in the path to know where to send the packet next.

- Static routes are best when the path is known and stable.
- The next hop is the critical forwarding decision.
- Verification is essential after configuration.
- End-to-end testing confirms that the route chain works.
