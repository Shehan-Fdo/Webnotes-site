## **Course:** CompTIA Network+ Hands-On Infrastructure (Exam N10-009)

**NAT and PAT overview**

<abbr title="Network Address Translation">NAT</abbr> changes address information as traffic crosses a boundary between internal and external networks. <abbr title="Port Address Translation">PAT</abbr>, also called NAT overload, extends that idea by using ports so many internal IPv4 hosts can share a single public IP address.

PAT is the common choice when you need external connectivity for multiple private hosts but have limited public IPv4 space. The translation table tracks each unique private address and port combination so return traffic can be mapped back correctly.

- **NAT** translates addresses between networks.
- **PAT** translates addresses and ports.
- **Overload** means multiple internal hosts share one public address.
- A NAT table stores the active mappings.

**How PAT works**

PAT differentiates sessions by port number while using one global address. That lets a router or firewall support many internal clients with only a few public IP addresses.

A practical way to think about it is that the device rewrites the source address and source port on outbound traffic, then reverses the mapping for inbound replies. This is why PAT is often described as source NAT in outbound scenarios.

- One public IP can serve many private hosts.
- Each connection gets a unique translated port.
- The device keeps state for each active mapping.
- Return traffic depends on the translation entry.

**Basic configuration pattern**

On Cisco IOS-style devices, the usual pattern is to define the inside addresses to translate, mark the WAN-facing interface as outside, and mark LAN-facing interfaces as inside. When using overload, the translation can point to an interface address or a public pool.

```bash
ip access-list standard NAT-INSIDE
 permit 10.1.0.0 0.0.255.255

interface GigabitEthernet0/0
 description WAN-Internet
 ip address 203.0.113.2 255.255.255.252
 ip nat outside

interface GigabitEthernet0/1
 description LAN
 ip address 10.1.0.1 255.255.0.0
 ip nat inside

ip nat inside source list NAT-INSIDE interface GigabitEthernet0/0 overload
```

- Use an access list to define which inside addresses qualify for translation.
- Apply `ip nat outside` on the external interface.
- Apply `ip nat inside` on internal interfaces.
- Add `overload` when many hosts share one public address.

**Outbound NAT on pfSense**

Outbound NAT controls how source addresses and ports are translated as traffic leaves an interface. The documented modes are automatic, hybrid, and manual.

Automatic mode performs NAT from internal interfaces to external interfaces without requiring you to build the full rule set. Hybrid mode combines automatic rules with manual rules for traffic that needs special handling. Manual mode uses only the rules you enter, which gives the most control but also requires you to maintain every relevant change yourself.

- **Automatic** is the default and easiest to manage.
- **Hybrid** mixes manual rules with automatic coverage.
- **Manual** gives the most control and the most maintenance.
- Outbound NAT is configured under `Firewall > NAT` on the `Outbound` tab.

**Verification and troubleshooting**

After configuration, verify that the inside and outside roles are correct and that the translation rule matches the intended source network. If traffic does not translate, check the interface assignments, the address list, and whether the rule points to the correct WAN interface or pool.

For outbound NAT on pfSense, confirm the selected mode first. Automatic mode may already cover the traffic you expect, while manual mode requires an explicit rule for each case you want translated.

> If the inside/outside roles are reversed, translation will not behave as expected.

**Key takeaways**

NAT and PAT solve external connectivity problems by rewriting private addressing into routable public addressing. PAT is the most space-efficient form because it uses ports to multiplex many sessions through one public IP.

- NAT handles address translation.
- PAT adds port translation for session uniqueness.
- Overload is the common PAT behavior for shared public access.
- Verification starts with interface roles and translation scope.
