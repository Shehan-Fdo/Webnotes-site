## **Course:** CompTIA A+ Practical Labs & Troubleshooting (Exams 220-1201 & 220-1202)

**Static IP basics**

A static IP address stays the same instead of being assigned dynamically by a DHCP server. You use it when a device needs a predictable address for services, port forwarding, or stable network access.

On Ubuntu Server, Netplan is the default network configuration tool on 18.04 and later. It uses YAML-based configuration and can work with the underlying network daemon in use on the system.

- **DHCP** assigns addresses automatically.
- **Static addressing** keeps the address fixed.
- **Netplan** is the configuration layer used on modern Ubuntu Server systems.
- You need root or `sudo` access to change network settings.

> Use caution when changing remote network settings. A bad configuration can disconnect you from the system.

**Gather the current network details**

Before you change anything, identify the active adapter and the current network values you need to preserve or replace. This helps you avoid guessing the interface name or subnet information.

Typical details to collect include:

- Current IP address
- Subnet mask or prefix
- Default gateway
- Adapter name

A command-line workflow is common for this step, and network troubleshooting often starts with basic interface and connectivity checks.

```bash
ip addr
ip route
```

**Configure the static address**

Ubuntu Server uses Netplan configuration files for network setup. You edit the YAML file that defines the interface and its address settings, then apply the changes.

A practical static configuration usually includes:

- The interface name
- A fixed IP address
- Gateway information
- DNS settings if required

Keep the configuration consistent with the existing network so the host remains reachable on the correct subnet.

```yaml
network:
  version: 2
  ethernets:
    enp0s3:
      dhcp4: false
      addresses:
        - 192.168.1.50/24
      routes:
        - to: default
          via: 192.168.1.1
      nameservers:
        addresses:
          - 8.8.8.8
          - 1.1.1.1
```

1. Open the Netplan configuration file for the target interface.
2. Set `dhcp4: false` to disable automatic address assignment.
3. Add the static `addresses` entry with the correct prefix.
4. Define the default route with the gateway address.
5. Add DNS servers if the environment requires name resolution.
6. Apply the configuration and confirm the interface comes back up.

**Verify connectivity**

After applying the configuration, verify that the host can reach the local network and beyond. Connectivity checks help confirm that the address, gateway, and DNS settings work together.

Use these checks in order:

- Confirm the interface has the expected IP address.
- Test reachability to the default gateway.
- Test reachability to a known host on the network.
- Verify name resolution if DNS is configured.

```bash
ip addr show enp0s3
ping -c 4 192.168.1.1
ping -c 4 8.8.8.8
```

If the gateway responds but external hosts do not, the issue is often routing or upstream connectivity. If IP-based ping works but hostnames fail, review DNS settings.

**Troubleshooting mindset**

Command-line tools help you isolate where the failure occurs. `ping` checks reachability and round-trip response time using Internet Control Message Protocol (ICMP). `ip addr` and `ip route` show whether the interface and route table match the intended configuration.

- Use `ping` to test basic reachability.
- Use `ip addr` to confirm the assigned address.
- Use `ip route` to confirm the default gateway.
- Recheck the YAML file if the interface does not come up after applying changes.

A static IP is most useful when the device must remain predictable for services or network rules. The key is to verify each layer: address, route, and connectivity.
