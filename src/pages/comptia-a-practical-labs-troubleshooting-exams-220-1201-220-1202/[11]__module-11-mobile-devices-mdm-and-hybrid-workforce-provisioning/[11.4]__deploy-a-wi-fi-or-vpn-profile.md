## **Course:** CompTIA A+ Practical Labs & Troubleshooting (Exams 220-1201 & 220-1202)

**Profile deployment overview**

Device configuration profiles let you push baseline security and access settings to managed devices. In Microsoft Intune, these profiles can include Wi-Fi and VPN settings, along with other device features and security controls.

- **Device configuration profiles** help standardize settings across Android, iOS/iPadOS, macOS, and Windows.
- Microsoft describes these profiles as part of a minimum or baseline set of settings for device security and access.
- Wi-Fi and VPN profiles are commonly used to connect devices to corporate resources without manual setup on each endpoint.

**Wi-Fi profile basics**

A Wi-Fi profile defines how a device joins a wireless network. In managed environments, this is useful when you want users to connect to a corporate SSID with consistent settings.

- Use a Wi-Fi profile when you want devices to connect to a known wireless network automatically.
- For enterprise Wi-Fi, certificate-based authentication may be used.
- If the Wi-Fi profile references a trusted certificate for server validation, that certificate is tied to a trusted certificate profile.

> If the trusted certificate profile is not already deployed elsewhere, the Wi-Fi profile can reference it for validation, but you should test the deployment path before relying on it in production.

**VPN profile basics**

A VPN profile defines how a device reaches private resources through a virtual private network. It is part of the same device configuration workflow used for other security and access settings.

- Use a VPN profile when users need secure remote access to internal services.
- VPN settings are typically deployed as part of a managed configuration profile.
- On macOS, profile updates should be tested carefully before broad rollout, especially when the VPN payload is in active use.

**Certificate and validation considerations**

Certificate handling matters when Wi-Fi authentication depends on server validation. The evidence shows a Wi-Fi profile can point to a trusted certificate profile, which is used for validation of the network connection.

- A **trusted certificate profile** supplies the certificate used for validation.
- A Wi-Fi profile may reference that certificate for server validation.
- Keep certificate deployment and Wi-Fi deployment aligned so validation succeeds when the profile applies.

**Practical deployment workflow**

1. Create the device configuration profile in Intune.
2. Choose the profile type for Wi-Fi or VPN.
3. Add the required connection settings.
4. If needed, reference a trusted certificate profile for validation.
5. Assign the profile to the target device group.
6. Test on a small set of devices before wider rollout.

**What to remember for A+ troubleshooting**

When a device cannot join Wi-Fi or reach a VPN, the issue may be in the profile, the certificate, or the assignment path. Start with the managed configuration and verify that the correct profile is deployed to the correct devices.

- Check whether the profile is assigned to the intended group.
- Confirm that any required certificate profile is present.
- Verify that the device receives the updated configuration.
- Test changes before production deployment, especially for VPN updates.
