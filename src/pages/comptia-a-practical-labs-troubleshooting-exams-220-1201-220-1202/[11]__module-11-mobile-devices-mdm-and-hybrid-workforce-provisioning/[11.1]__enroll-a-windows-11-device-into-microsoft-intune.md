## **Course:** CompTIA A+ Practical Labs & Troubleshooting (Exams 220-1201 & 220-1202)

**Windows 11 enrollment overview**

Microsoft Intune enrolls personal and organization-owned Windows devices so they can receive policies and profiles. After enrollment, the device can access managed resources such as work email, files, VPN, and Wi-Fi.

Intune supports several Windows enrollment paths. The right choice depends on whether you are setting up a single device, using Windows Autopilot, or managing devices already connected to Configuration Manager.

- **Windows automatic enrollment** is used when you want devices to enroll without manual user setup.
- **Windows Autopilot** supports new-device deployment and pre-provisioned scenarios.
- **BYOD** means bring your own device, and user enrollment is one option for that model.
- **Co-management** lets Configuration Manager and Intune manage the same device.

**Enrollment options and when they fit**

The Intune enrollment guide groups Windows client enrollment into a few practical paths. Use that structure to decide how the device should join management and what the administrator and user must do.

| Option | Typical use | Notes |
|---|---|---|
| Windows automatic enrollment | Managed Windows deployment | Helps devices enroll with minimal manual steps |
| Windows Autopilot | New device setup | Supports streamlined provisioning and registration |
| BYOD user enrollment | Personal device access | Lets users enroll their own devices |
| Co-management with Configuration Manager | Mixed management | Uses both Configuration Manager and Intune |

When you are planning a deployment, keep the enrollment method aligned with the device lifecycle. A device that is already in use may need a different path than a device that is still being prepared for first use.

**Windows Autopilot and pre-provisioned deployment**

Windows Autopilot includes a pre-provisioned deployment flow for Microsoft Entra hybrid join. In that workflow, automatic Intune enrollment is the first step, followed by connector setup, device registration, group creation, profile assignment, and technician and user flows.

The documented sequence shows that enrollment is not isolated from the rest of deployment. It is part of a larger chain that prepares the device for hybrid join and later user use.

- Step 1: Set up Windows automatic Intune enrollment.
- Step 2: Install the Intune Connector for Active Directory.
- Step 3: Increase the computer account limit in the organizational unit.
- Step 4: Register devices as Windows Autopilot devices.
- Step 5: Create a device group.
- Step 6: Configure and assign the Enrollment Status Page.
- Step 7: Create and assign the Microsoft Entra hybrid join Autopilot profile.
- Step 8: Configure and assign the domain join profile.
- Step 9: Assign the device to a user, if needed.
- Step 10: Follow the technician flow.
- Step 11: Follow the user flow.

**Manual Autopilot registration**

If you need to register devices manually, Intune supports uploading hardware identity data, also called a hardware hash, through a CSV file. This process requires booting the device into Windows to capture the hash, so it is mainly used for testing and evaluation.

A practical limit applies to this method: you can manually register up to 500 devices through a CSV upload in the portal. Before adding more devices, verify that the previous batch registered successfully.

> Manual registration is best for controlled scenarios. For larger or production deployments, use the appropriate automated enrollment path instead of relying on repeated CSV uploads.

**Basic enrollment workflow**

A Windows 11 device enrollment flow usually starts with confirming that Intune is configured correctly, then moving to the user or administrator enrollment action, and finally verifying that the device appears in the admin center.

1. Confirm that Intune enrollment is configured correctly.
2. Choose the enrollment path that matches the device scenario.
3. Complete the enrollment on the device.
4. Verify the device in the Microsoft Intune admin center.
5. Check policies and profiles after enrollment.

**Troubleshooting enrollment failures**

If enrollment fails, start with the simplest checks first. Microsoft recommends confirming that Intune is configured properly before deeper troubleshooting. It also provides self-help diagnostics for common support scenarios.

One documented Windows enrollment error is `hr 0x8007064c`, which appears with the message “The machine is already enrolled.” This can happen if:

- The computer was previously enrolled.
- The device has a cloned image from a computer that was already enrolled.
- The account certificate from the previous account is still present on the computer.

In that case, the issue is usually not the current enrollment attempt itself, but leftover state from an earlier enrollment or imaging process.

**Practical verification points**

After enrollment, check that the device is actually managed and not just joined to the directory. A device can appear in identity-related views while still showing incomplete management status.

- Confirm the device is enrolled in Intune.
- Verify that policies and profiles apply.
- Check for signs of duplicate or cloned enrollment state if the device reports as already enrolled.
- Use the enrollment troubleshooting guidance when the device does not appear as expected.

**Key takeaways**

Windows 11 enrollment in Intune is about matching the right method to the right device scenario. Automatic enrollment, Autopilot, user enrollment, and co-management each serve different deployment needs.

For exam and lab work, focus on these points:

- Intune enrollment gives devices access to managed resources and policy enforcement.
- Autopilot can be part of a larger pre-provisioned deployment flow.
- Manual Autopilot registration uses hardware hashes in a CSV file and is limited to smaller-scale scenarios.
- Enrollment errors often point to prior enrollment state, cloned images, or leftover certificates.
