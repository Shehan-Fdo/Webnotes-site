## **Course:** CompTIA A+ Practical Labs & Troubleshooting (Exams 220-1201 & 220-1202)

**Drive setup basics**

Partitioning divides a physical disk into logical sections, and formatting prepares those sections with a file system so an operating system can store and retrieve data. In practice, you use these tasks when a drive is new, when you need to reorganize storage, or when you want to make a disk usable by a specific operating system.

Formatting does more than rename a drive. It creates the file system structure and may also involve a new partition table. Common file systems mentioned in the evidence include `FAT32`, `exFAT`, `NTFS`, `HFS+`, `APFS`, and `ext4`.

- **Partition table** defines how the disk is organized.
- **File system** defines how data is stored and accessed.
- **Quick format** rebuilds the file system without scanning for bad sectors.
- **Non-quick format** performs a fuller scan and marks unusable areas.

> Formatting removes existing data from the operating system’s point of view. Treat it as a destructive task unless you have a verified backup.

**Windows tools and workflow**

Windows includes **Disk Management**, a built-in tool for managing disks and volumes. You can use it to initialize new drives, create and format volumes, change drive letters, and extend or shrink existing volumes. This makes it the main graphical option for common storage tasks.

You can also use `diskpart`, the command-line utility for disk and partition management. The evidence shows it is used to format drives and work with partition layouts, including `FAT32`, `NTFS`, `exFAT`, and GPT-related tasks.

- Use **Disk Management** for visual, guided administration.
- Use **diskpart** when you need command-line control or scripted steps.
- Expect `diskpart` to be available across multiple Windows versions.
- Be careful with commands that modify the selected disk, because they act immediately.

A typical Windows flow is to identify the disk, initialize it if needed, create a volume or partition, and then format it with the file system you need.

```text
Disk Management can:
- initialize new drives
- create and format volumes
- change drive letters
- extend or shrink volumes
```

**macOS partitioning and formatting**

On macOS, **Disk Utility** is the standard graphical tool for formatting drives. The evidence shows it supports formats such as `APFS` and `Mac OS Extended`. For command-line work, `diskutil` can list disks and partition them.

`diskutil list` is especially useful because it shows attached disks and their identifiers. You use the identifier to target the correct device before making changes.

- Check the disk list before you act.
- Use the identifier column to confirm the target disk.
- Double-check the device name before partitioning or formatting.
- Treat command-line disk tools as high-risk because they do not always provide confirmation prompts.

> Be especially careful on macOS when selecting a disk. The evidence warns that `diskutil` can modify disks without “are you sure?” prompts.

A practical macOS workflow is to list disks, confirm the external or target drive, then partition or format it with the appropriate scheme and file system.

```bash
diskutil list
```

**Ubuntu and Linux tools**

On Ubuntu, the evidence points to tools such as `GParted`, `fdisk`, `parted`, and `mkfs`. These tools cover the same general workflow: create or inspect partitions, write a partition table, and format the resulting partition with a file system.

For large disks, `parted` is highlighted as a practical option. The example workflow shows listing drives, creating a GPT partition table, making a primary partition, and verifying the result.

- Use **parted** when you need to create or inspect partition tables.
- Use **mkfs** to create the file system after partitioning.
- Use **fdisk** or `parted` to verify the disk layout.
- Use **GParted** when you prefer a graphical interface.

A common Linux sequence is to identify the disk, create the partition table, create the partition, format it, and then verify the result.

```bash
sudo parted -l
sudo parted /dev/sda
(parted) mklabel gpt
(parted) print
(parted) mkpart primary 0% 100%
(parted) quit
sudo fdisk -l /dev/sda
```

**Choosing the right file system**

The file system you choose depends on the operating system and the intended use of the drive. The evidence names several common options, but it does not prescribe a single universal choice. Use the format that matches your platform and compatibility needs.

- `NTFS` is common in Windows environments.
- `APFS` and `Mac OS Extended` are used in macOS workflows.
- `ext4` is common in Linux environments.
- `FAT32` and `exFAT` are often used when compatibility matters.

A practical exam takeaway is that partitioning and formatting are separate steps. You first define the disk layout, then apply the file system that makes the partition usable.

**Safe lab habits**

Storage tools can erase data quickly, so verification matters more than speed. Before you format or repartition a drive, confirm the disk identifier, confirm whether the drive is internal or external, and confirm that you have selected the correct target.

1. Identify the disk with the appropriate tool.
2. Verify the disk identifier and capacity.
3. Confirm the target is the correct drive.
4. Partition or format only after you are certain.
5. Recheck the result after the operation completes.

These habits reduce the chance of wiping the wrong disk and help you troubleshoot storage tasks more confidently in Windows, macOS, and Ubuntu.
