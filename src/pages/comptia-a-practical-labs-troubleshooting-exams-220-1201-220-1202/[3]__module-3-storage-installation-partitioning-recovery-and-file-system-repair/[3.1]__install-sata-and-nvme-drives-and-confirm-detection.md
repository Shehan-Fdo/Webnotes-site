## **Course:** CompTIA A+ Practical Labs & Troubleshooting (Exams 220-1201 & 220-1202)

**Drive installation overview**

SATA and NVMe storage devices should be confirmed at two levels: physical installation and firmware detection. If a drive is not visible in BIOS/UEFI or the operating system, start with the simplest hardware checks before moving to firmware and driver-related causes.

- **SATA** drives depend on secure power and data connections.
- **NVMe** drives use the M.2 interface and may require BIOS/UEFI support or slot enablement.
- Detection problems often come from cabling, BIOS settings, outdated firmware, or missing drivers.

> If the drive is not detected in BIOS/UEFI, do not assume the operating system is the problem first. Check the hardware path and firmware settings before deeper software troubleshooting.

**Installing a SATA drive**

A SATA drive must be seated and connected correctly before you can expect it to appear in the system. In practice, the first troubleshooting step is to inspect the drive’s power and data cable connections.

1. Power off the system and open the case.
2. Mount the SATA drive in the bay or bracket.
3. Connect the SATA data cable to the drive and motherboard.
4. Connect the SATA power cable from the power supply.
5. Boot into BIOS/UEFI and confirm the drive appears.

- A loose or missing cable can prevent detection entirely.
- If the drive is absent in BIOS/UEFI, recheck the physical connections before formatting or initializing.
- If the drive appears in firmware but not in the OS, continue with OS-level storage checks.

**Installing an NVMe drive**

NVMe drives install into an M.2 slot and can fail to appear if the slot is disabled or the firmware does not support the device properly. Evidence from troubleshooting cases shows that a drive may appear on one kernel or platform and fail on another, which reinforces the need to verify firmware and platform compatibility.

1. Power off the system and disconnect power.
2. Insert the NVMe drive into the M.2 slot at the correct angle.
3. Secure the drive with the retaining screw.
4. Confirm the M.2 slot is enabled in BIOS/UEFI.
5. Update motherboard firmware if the drive still does not appear.
6. Boot and verify that the drive is listed in firmware and the operating system.

- Some systems require a BIOS update before they can detect newer NVMe devices.
- If the drive is visible in low-level hardware listings but not initialized, the issue may be firmware or platform related.
- A working drive on one system does not guarantee detection on another system with different firmware or kernel support.

**Confirming detection in BIOS/UEFI**

BIOS/UEFI is the first place to verify whether the system recognizes the storage device. Official motherboard guidance recommends checking BIOS settings and updating to the latest firmware when a hard disk or SSD is not found after power-on.

- Check that the drive is listed in storage or boot-related menus.
- Confirm the correct M.2 slot or SATA port is enabled.
- Update the motherboard BIOS/UEFI from the manufacturer’s support site when detection fails.
- Review boot device priorities if the drive is present but the system does not start from it.

**Troubleshooting no-detect conditions**

When a drive does not appear, use a practical sequence that isolates the most likely cause first. Exam-style questions and support guidance both point to connection checks and boot/firmware verification before more advanced actions.

- Inspect SATA power and data cables first.
- Check BIOS/UEFI storage settings and boot options.
- Verify the M.2 slot is enabled for NVMe devices.
- Update firmware if the motherboard is outdated.
- Load the correct storage driver if the operating system installer cannot see the drive.
- Consider controller mode settings such as AHCI when the platform requires it.

**Common detection clues**

Different symptoms point to different layers of the storage stack. Use the symptom to decide where to look next.

| Symptom | Likely area to check |
|---|---|
| Drive missing in BIOS/UEFI | Cable, slot, port, or firmware issue |
| Drive visible in hardware listings but not initialized | Firmware or platform compatibility |
| Drive visible in BIOS/UEFI but not in the OS | Driver, partition, or OS storage configuration |
| Installer shows no available drives | Storage mode, driver loading, or BIOS settings |

**Practical exam focus**

For A+ troubleshooting, the safest first response is usually to verify the physical connection and then confirm firmware detection. If the drive is still missing, move to BIOS/UEFI settings, firmware updates, and driver checks rather than jumping straight to formatting or recovery tools.

- First check the drive’s power and data path.
- Then confirm BIOS/UEFI detection.
- Next review boot options and storage settings.
- Finally, address drivers or firmware compatibility if the drive still does not appear.
