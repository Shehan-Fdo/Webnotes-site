## **Course:** CompTIA A+ Practical Labs & Troubleshooting (Exams 220-1201 & 220-1202)

**Windows recovery workflow**

When Windows fails to start, the first recovery path is the **Windows Recovery Environment** (Windows RE). Microsoft recommends trying **Startup Repair** first from the System Recovery Options menu. If that does not resolve the issue, you can move to manual repair with `Bootrec.exe` from the recovery command prompt.

Windows RE is the built-in recovery layer used for startup troubleshooting, reset and restore options, and repair tasks. In practice, you use it when the operating system will not boot normally or when automatic repair cannot fix the startup chain.

- Start with **Startup Repair** before manual commands.
- Use **Bootrec.exe** when you need deeper startup troubleshooting.
- Keep the focus on boot components, not general application repair.

**What Bootrec.exe is used for**

`Bootrec.exe` is a recovery tool for repairing startup-related structures. Microsoft documents it for repairing items such as the **master boot record** (MBR) and the boot configuration data used during startup.

This makes it useful when the system shows boot errors, startup loops, or missing boot entries. If the boot process is damaged, the tool helps rebuild or correct the parts Windows needs to locate and start the operating system.

- **MBR** repair targets the disk’s startup code.
- **Boot configuration** repair targets the data Windows uses to find the installation.
- Use it from Windows RE, not from a normal desktop session.

**Common recovery path**

1. Boot into **Windows RE** using recovery media or the built-in recovery options.
2. Open **Startup Repair** and let it attempt an automatic fix.
3. If startup repair fails, open the **Command Prompt** in Windows RE.
4. Run the appropriate `Bootrec.exe` repair command for the startup problem.
5. Restart and verify whether Windows loads normally.

> If the system still cannot boot after automatic repair, the issue may involve corrupted boot entries or inaccessible startup devices, so manual recovery becomes necessary.

**Boot-related failure patterns**

A broken boot can appear in several ways. One example from Microsoft Q&A describes a corrupted boot configuration database (BCD) after a storage mode change, where startup repair and manual editing were needed to restore boot entries. Another example shows Windows 11 stuck in automatic repair with `SFC /Scannow` not resolving the issue from Windows RE.

These cases show an important troubleshooting rule: file-system repair alone does not always fix startup problems. If the boot chain itself is damaged, you need recovery tools that target boot records and boot entries.

- A **BCD** issue can prevent Windows from finding the correct installation.
- A device mode change can make boot entries point to the wrong partition.
- Automatic repair may fail when the startup path is too damaged to resolve on its own.

**Recovery Environment command-line use**

The Windows RE command prompt is the place to run boot repair commands when the GUI repair path is not enough. You use it for manual troubleshooting when Startup Repair does not fix the issue or when you need to correct startup data directly.

A practical recovery session often includes checking the boot configuration, repairing startup records, and then testing the result with a reboot.

```text
Windows RE
  -> Startup Repair
  -> Command Prompt
  -> Bootrec.exe
  -> Restart
```

**Troubleshooting mindset**

Boot repair is about isolating the startup layer from the rest of Windows. If the machine will not reach the desktop, focus on recovery tools that operate before the operating system fully loads. If the system reaches recovery but not normal boot, the problem is often in the boot path, not in user applications.

- Use recovery tools when Windows cannot complete startup.
- Treat `SFC /Scannow` as a separate repair step, not a guaranteed boot fix.
- Verify the result by confirming that Windows starts normally after the repair.

**Exam focus**

For A+ troubleshooting, remember the sequence and purpose of the tools:

- **Windows RE** provides the recovery environment.
- **Startup Repair** is the first automatic option.
- **Bootrec.exe** is the manual tool for deeper boot repair.
- The goal is to restore the system’s ability to locate and start Windows.
