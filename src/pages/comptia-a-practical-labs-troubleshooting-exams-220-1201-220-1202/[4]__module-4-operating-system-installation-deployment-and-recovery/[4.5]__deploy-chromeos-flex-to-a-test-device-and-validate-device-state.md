## **Course:** CompTIA A+ Practical Labs & Troubleshooting (Exams 220-1201 & 220-1202)

**Overview**

ChromeOS Flex is a lightweight operating system you can install on existing PCs and Macs. For test deployments, you usually start with a USB installer, boot the target device from that media, and then validate whether the hardware behaves as expected.

Google recommends full installation for the best experience, but live booting from the USB installer is useful when you want to test compatibility before committing to a permanent install. That temporary boot helps you verify basic device state such as networking and input tools.

**Prepare for installation**

Before you create media, check whether the target model is certified or at least likely to work well. Google maintains a certified models list with status labels that help you judge risk:

- **Certified** models are expected to work with ChromeOS Flex.
- **Minor issues expected** models should support basic functionality, but may still have small problems.
- **Major issues expected** models are not currently recommended because of known problems such as boot issues.
- **Decertified** models reached end of support on the 31st of December in the listed year.

If you plan to test a non-certified device, use conservative expectations. ChromeOS Flex may still run, but Google does not guarantee stability, functionality, or performance across updates.

> Full installation erases all existing data, applications, settings, and the current operating system on the target device. Lost data is permanently unrecoverable, even if installation fails.

**Create the USB installer**

You can create the installer on a different device than the one you plan to test. The method depends on the source device you use:

- **Chromebook Recovery Utility** is recommended for Windows or Mac devices with Chrome browser installed, or for a ChromeOS device.
- **Download from Google** works on ChromeOS, Windows, Mac, or Linux devices and does not require Chrome browser.

When using the Chromebook Recovery Utility, you install the extension in Chrome, enable it in the Extensions menu, and then use it to create the USB installer. The USB drive contents are erased during this process.

**Boot the test device**

After the USB installer is ready, boot the target device from the USB media. This step lets you start ChromeOS Flex without immediately replacing the existing operating system.

A temporary live boot is useful when you want to confirm that the device can start correctly and that core features respond as expected. If the device passes your checks, you can then decide whether to proceed with a full installation.

**Validate device state**

Use the test session to confirm that the device behaves normally in the areas most likely to affect usability. Focus on practical checks rather than exhaustive benchmarking.

- **Networking**: verify that the device connects and can reach the network.
- **Input tools**: test keyboard, touchpad, mouse, or other input devices.
- **Boot behavior**: confirm the device starts from the USB installer without obvious errors.
- **General responsiveness**: make sure the system is usable for basic exploration.

If the device is not certified, treat successful booting as a positive sign but not a guarantee of long-term stability. Even devices that work initially can develop issues later, especially after updates.

**Installation decision points**

Use the test results to decide whether to continue:

1. Confirm the model status in the certified models list.
2. Create the USB installer using the appropriate method.
3. Boot the target device from the USB installer.
4. Validate networking and input functionality.
5. Decide whether to keep testing live or perform a full installation.

If you are not ready to erase the device, stay in the live environment. If the test device meets your requirements and you are prepared for data loss, proceed with full installation.

**Practical exam takeaways**

For A+ troubleshooting scenarios, the key idea is to match the deployment method to the goal:

- Use a USB installer for hands-on testing and installation.
- Use live booting only for temporary exploration or verification.
- Check certification status before you commit to a device.
- Treat non-certified hardware as a risk-managed test case, not a guaranteed deployment.

ChromeOS Flex can be a fast way to repurpose older hardware, but the safest workflow is still: verify compatibility, test basic functionality, and only then install permanently.
