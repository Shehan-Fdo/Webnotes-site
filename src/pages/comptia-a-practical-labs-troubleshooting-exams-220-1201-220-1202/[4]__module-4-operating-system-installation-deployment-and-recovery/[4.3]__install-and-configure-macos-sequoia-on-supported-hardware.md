## **Course:** CompTIA A+ Practical Labs & Troubleshooting (Exams 220-1201 & 220-1202)

**macOS Sequoia installation context**

A clean installation of macOS Sequoia is used when you want a fresh system state rather than an in-place upgrade. In the evidence, the goal is to back up data first, erase or reformat the target drive during installation, and then restore personal files afterward. This approach helps reduce conflicts from older software versions and leftover references.

- **Clean installation** removes existing data on the selected target partition.
- Back up important files before you begin.
- Restore personal files after the operating system is installed.
- Use this approach when you want a blank system for new software or troubleshooting stability issues.

> A clean install is not the same as an upgrade. It starts from a fresh target and is intended to reset the system state.

**Preparing for installation**

Before you install, make sure you have a backup plan. The evidence specifically mentions Time Machine and jump drives for key files, which gives you both a full-system recovery path and a separate copy of critical data. For exam purposes, connect this to the broader A+ idea of backup and recovery methods.

1. Back up the Mac with Time Machine.
2. Copy key files to external media such as jump drives.
3. Confirm the target Mac is supported for macOS Sequoia.
4. Start the installation process from the appropriate boot or recovery path.
5. Reformat the drive if you are performing a clean install.
6. Reinstall macOS Sequoia and then restore user data.

**Installation methods and boot options**

Operating system installation often begins by booting to an installer environment rather than launching setup from inside the existing OS. A common method is a bootable USB drive, which contains the installation files and starts the system in a minimal environment. This is a general installation concept that applies to macOS and other operating systems.

- **Bootable USB** media is a common way to start installation.
- The installer runs from a bare-bones environment before the full OS is present.
- Zero-touch deployment and other remote methods may exist in managed environments, but the evidence here emphasizes local installation concepts.

**macOS setup and core tools**

After installation, macOS uses **System Settings** rather than the older System Preferences label. For A+ Core 2, focus on the settings areas that help you manage and secure the system. The evidence highlights Displays, Network, Print & Scanners, Privacy & Security, and Accessibility.

- Open System Settings from the Apple menu.
- Review display and network configuration first if the Mac needs basic connectivity.
- Check privacy and security settings before adding software or user accounts.
- Use accessibility settings when the user needs interface adjustments.

**Practical exam takeaways**

The exam-relevant idea is not just that macOS Sequoia can be installed, but that you choose the right installation method for the situation. A clean install is appropriate when you want to remove old data and start over. A backup-first workflow protects user files and makes restoration possible after the reinstall.

- **Clean install** = erase or reformat the target and install fresh.
- **Backup first** = protect data before any destructive step.
- **System Settings** = current macOS control center for common configuration tasks.
- **Supported hardware** = verify compatibility before you begin.
