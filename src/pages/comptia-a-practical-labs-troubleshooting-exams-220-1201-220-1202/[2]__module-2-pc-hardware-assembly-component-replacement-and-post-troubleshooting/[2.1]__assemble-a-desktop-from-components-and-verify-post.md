## **Course:** CompTIA A+ Practical Labs & Troubleshooting (Exams 220-1201 & 220-1202)

**Desktop assembly overview**

Building a desktop from components means installing the core hardware, connecting power and data cables, and then confirming the system can complete <abbr title="Power-On Self-Test">POST</abbr>. POST is the first hardware check the system performs at startup, so it is your quickest signal that the board, CPU, memory, storage, and power delivery are working together.

For A+ lab practice, focus on a clean, methodical build. You want each part seated correctly, each cable connected to the right header or port, and each startup result interpreted before you move on to the next step.

- Verify that the motherboard, processor, memory, storage, and power supply are present before assembly.
- Treat POST as a validation step, not just a boot step.
- Use a practical, repeatable order so you can isolate problems quickly.

**Core components and what they do**

A desktop build centers on a few essential parts. The motherboard provides the main connection points, the CPU handles processing, memory supports active tasks, storage holds the operating system and files, and the power supply delivers electrical power to the system.

When you assemble the system, your goal is not only to install parts but also to make sure they are compatible and properly connected. A system that powers on but does not POST usually points to a seating, power, or component issue that you can narrow down by checking one part at a time.

- **Motherboard**: main circuit board and connection hub.
- **CPU**: processor that executes instructions.
- **RAM**: memory used for active workloads.
- **Storage**: drive that holds data and the operating system.
- **Power supply**: provides power to all components.

**Assembly workflow**

Use a consistent build sequence so you reduce rework and avoid missing a connection. A careful order also makes troubleshooting easier if the system does not start correctly.

1. Install the CPU into the motherboard socket.
2. Install the memory modules in the recommended slots.
3. Mount the motherboard in the case.
4. Install the storage device.
5. Connect the power supply cables to the motherboard and components.
6. Attach front-panel and case connections.
7. Connect display output and any required peripherals.
8. Power on the system and observe startup behavior.

> If the system does not POST, do not assume the motherboard is bad. Check power, seating, and connections first.

**Power and connection checks**

A desktop can appear assembled but still fail to start if a cable is loose or a component is not fully seated. Standby power indicators can help you confirm that the board is receiving power before you attempt a full startup.

Check the obvious physical connections before you move into deeper troubleshooting. This includes the main power lead, CPU power, memory seating, and any storage or front-panel connections that affect startup behavior.

- Confirm the power supply switch is on.
- Check that the motherboard receives standby power.
- Reseat memory if the system powers but does not progress.
- Verify storage and front-panel connections are secure.
- Make sure display output is connected to the correct port.

**Verifying POST**

POST is your first proof that the system can initialize hardware successfully. A successful POST usually means the board has recognized the essential components and can continue toward firmware setup or operating system loading.

If the system fails to POST, use the result as a diagnostic clue. No lights, no fans, or no standby power suggest a power delivery issue. Power with no display or no progress suggests you should recheck component seating and connections.

- Successful POST indicates the core hardware is functioning well enough to continue startup.
- Failed POST means you should inspect power, seating, and compatibility before replacing parts.
- Partial startup behavior is useful evidence, not just a symptom.

**Troubleshooting approach**

When a new build does not start correctly, work from the simplest checks to the more involved ones. This keeps you from changing multiple variables at once and makes it easier to identify the fault.

1. Confirm external power and the PSU switch.
2. Check motherboard standby indicators, if present.
3. Reseat the RAM and verify the modules are in the correct slots.
4. Confirm CPU installation and power connections.
5. Verify storage and front-panel wiring.
6. Test again and note the exact startup behavior.

A careful troubleshooting process matters because a system can show signs of life without completing POST. That distinction helps you separate power delivery problems from component initialization problems.

**Exam focus**

For A+ lab tasks, you should be able to assemble a desktop, identify the major internal components, and verify whether the system reaches POST. You also need to interpret startup behavior in a practical way so you can decide what to check next.

- Know the role of each major component in the build.
- Use a logical assembly order.
- Recognize POST as a key validation point.
- Troubleshoot from power and seating checks before assuming hardware failure.
