## **Course:** CompTIA A+ Practical Labs & Troubleshooting (Exams 220-1201 & 220-1202)

**Scenario focus**

A simulated fault scenario often presents as a system that powers on but does not display video, does not reach <abbr title="Basic Input/Output System">BIOS</abbr>, or loops during startup. In this kind of lab, you practice replacing core components and verifying whether the fault follows the part or remains with the system.

The evidence points to common failure patterns after hardware changes:
- A new <abbr title="Graphics Processing Unit">GPU</abbr> may spin up but still produce a black screen.
- Added <abbr title="Random Access Memory">RAM</abbr> can prevent <abbr title="Power-On Self-Test">POST</abbr> if it is not seated correctly or is incompatible.
- A storage swap can leave the system unable to boot if the drive or connection is not recognized.
- A power supply issue can allow fans and lights to run while still failing to deliver stable power.

**Component replacement priorities**

When you troubleshoot in a simulated lab, replace one part at a time and test after each change. That approach helps you isolate the fault instead of introducing multiple variables at once.

- **RAM**: Check seating and slot placement first, then test with fewer modules if needed.
- **SSD**: Confirm the drive is connected and recognized before assuming the operating system is damaged.
- **PSU**: Verify the power supply is installed correctly and provides enough power for the system.
- **GPU**: Confirm the card is fully seated and that display output is connected to the correct port.

> If the system powers on but shows no display, do not assume the GPU is the only cause. RAM and power delivery can produce the same symptom.

**Safe replacement workflow**

1. Power down the system and disconnect external power.
2. Open the case and identify the target component.
3. Remove the old part carefully and keep track of screws, cables, and slot orientation.
4. Install the replacement part and ensure it is fully seated.
5. Reconnect required power and data cables.
6. Boot the system and observe whether it reaches POST or displays video.
7. If the fault remains, revert to the last known-good configuration and test the next component.

**RAM replacement notes**

RAM problems often show up as boot failure, repeated restarts, or a blank screen. In the evidence, a user added memory alongside a GPU and PSU upgrade and still saw a black screen, which is a good reminder that memory changes can be part of a broader startup issue.

- Match the module type to the motherboard requirements.
- Seat each stick firmly until the retention clips lock.
- Test with one module at a time if the system does not POST.
- Use known-good slots when isolating a suspected bad stick or slot.

**SSD replacement notes**

An SSD swap can affect boot behavior even when the machine powers on normally. If the system no longer boots after the swap, check the drive connection and confirm the system can detect the device.

- Verify the data and power connections.
- Confirm the drive is installed in the correct bay or slot.
- Check whether the system reaches firmware setup.
- Distinguish between a drive detection issue and an operating system boot issue.

**PSU replacement notes**

A failing or mismatched power supply can create misleading symptoms. Fans may spin and LEDs may light, but the system still fails to initialize correctly if power delivery is unstable or insufficient.

- Confirm the PSU is installed correctly.
- Verify all required motherboard and device power connectors are attached.
- Make sure the replacement unit is appropriate for the system load.
- Watch for repeated boot loops, which can indicate power instability.

**GPU replacement notes**

A GPU can appear active because its fan spins, yet the system may still show no image. That makes video output checks essential during troubleshooting.

- Reseat the card if the display stays black.
- Confirm the monitor cable is connected to the GPU output, not the motherboard, when required.
- Check that auxiliary power connectors are attached.
- Test with a known-good display path if available.

**Troubleshooting mindset**

The most useful lab habit is controlled isolation. Replace one component, test, and record the result before moving on. That method helps you identify whether the fault is caused by the part itself, the installation, or a separate compatibility or power issue.

- Start with the symptom, not the assumption.
- Change one variable at a time.
- Recheck seating, cabling, and power before declaring a part bad.
- Use the simplest known-good configuration to restore POST first.

**Key takeaways**

A no-display or no-POST scenario after replacing RAM, SSD, PSU, or GPU usually requires careful isolation rather than guesswork. In practice, you verify installation, confirm power and data connections, and test each component independently until the system returns to a stable boot state.
