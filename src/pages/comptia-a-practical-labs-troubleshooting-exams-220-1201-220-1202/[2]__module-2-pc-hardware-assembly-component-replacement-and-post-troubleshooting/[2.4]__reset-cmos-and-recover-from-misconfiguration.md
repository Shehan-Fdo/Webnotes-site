## **Course:** CompTIA A+ Practical Labs & Troubleshooting (Exams 220-1201 & 220-1202)

**Why CMOS resets matter**

<abbr title="Complementary Metal-Oxide-Semiconductor">CMOS</abbr> settings store firmware data that the system uses during startup. Clearing CMOS resets the <abbr title="Real-Time Clock">RTC</abbr> random access memory and removes stored BIOS setup parameters, including the date, time, BIOS password, and other configuration values.

A CMOS reset is a practical recovery step when a system becomes unstable after a firmware change, overclock, or other misconfiguration. It helps you return the motherboard to a known-good baseline before you try more advanced troubleshooting.

- Clear CMOS when the system crashes, freezes, or behaves unpredictably after a BIOS change.
- Expect the reset to remove stored setup values, not just one isolated option.
- Treat the procedure as a recovery step, not a substitute for diagnosing the original cause.

**Ways to clear CMOS**

Motherboards may support different reset methods. The evidence identifies three common approaches: a Clear CMOS button, a CLRTC pin header, or removing the CMOS battery. You should confirm which method your motherboard supports before you begin.

| Method | Typical use |
|---|---|
| Clear CMOS button | Quick reset on supported boards |
| CLRTC pin | Jumper-style reset on supported boards |
| Remove CMOS battery | Manual reset when no button or pin is available |

> Always power off the PC and remove the power cord before clearing CMOS.

**Using a Clear CMOS button**

Some motherboards include a dedicated Clear CMOS or `CLR_CMOS` button. This method is straightforward and does not require opening the battery compartment.

1. Power off the PC.
2. Remove the power cord.
3. Press the motherboard Clear CMOS or `CLR_CMOS` button for about 5 to 10 seconds.
4. Reconnect power and boot the system.
5. Re-enter BIOS setup and restore the needed settings.

This method clears the RTC memory and is useful when you need a fast recovery from a bad firmware setting.

**Recovering from BIOS and security misconfiguration**

A misconfigured BIOS or UEFI setup can prevent normal booting even when the hardware is healthy. Evidence shows that Secure Boot and TPM (Trusted Platform Module) changes can cause startup repair loops, missing boot options, or a system that powers on but does not load the operating system.

When you troubleshoot this type of issue, verify the firmware settings that affect boot behavior:

- Check whether Secure Boot is enabled or disabled as required.
- Confirm TPM status has not been reset or disabled unexpectedly.
- Make sure the boot mode matches the security configuration, since Secure Boot requires UEFI.
- Review recent BIOS updates or hardware changes that may have reverted settings.

If the system no longer boots after a security change, restore the correct Secure Boot and TPM configuration, save the changes, and reboot. In some cases, temporarily disabling Secure Boot can help you see whether USB boot media appears as a boot option.

**Practical recovery workflow**

Use a structured approach so you do not change multiple variables at once.

1. Power down the system and disconnect AC power.
2. Clear CMOS using the supported motherboard method.
3. Boot into BIOS or UEFI and confirm the settings were reset.
4. Restore only the settings you need, such as boot mode, Secure Boot, and TPM.
5. Save changes and test whether the operating system loads normally.
6. If the system still fails to boot, reassess recent firmware changes and boot-device visibility.

**What to remember for troubleshooting**

- CMOS clearing resets stored firmware configuration data.
- BIOS misconfiguration can block startup even when the hardware is fine.
- Secure Boot and TPM settings must align with the system’s boot mode.
- A careful reset-and-verify process is safer than changing several settings at once.
