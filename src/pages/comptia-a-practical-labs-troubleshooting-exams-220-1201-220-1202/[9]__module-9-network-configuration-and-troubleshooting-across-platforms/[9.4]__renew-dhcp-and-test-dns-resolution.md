## **Course:** CompTIA A+ Practical Labs & Troubleshooting (Exams 220-1201 & 220-1202)

**Overview**

`ipconfig` is a Windows command-line tool that shows current TCP/IP configuration and can refresh DHCP (Dynamic Host Configuration Protocol) and DNS (Domain Name System) settings. In troubleshooting, you use it to force a new lease from the DHCP server and to clear or inspect name-resolution data when a host cannot reach a site by name.

- `ipconfig` displays IPv4 and IPv6 addresses, subnet mask, and default gateway when run without parameters.
- `ipconfig /renew` requests a new DHCP lease for the adapter.
- `ipconfig /flushdns` clears the local DNS resolver cache.
- `ipconfig /displaydns` shows cached DNS entries.
- `ipconfig /registerdns` asks the client to refresh its DNS registration.

**Renewing a DHCP Lease**

A DHCP lease is the temporary IP address assignment a client receives from a DHCP server. If connectivity is unstable or the address assignment seems stale, renewing the lease can restore normal communication. On Windows, `ipconfig /renew` tells the DHCP client to renegotiate an address lease with the server.

> Use `ipconfig /release` before `ipconfig /renew` when you need to drop the current lease first, especially during deeper troubleshooting or when the adapter appears stuck.

1. Open a Command Prompt with appropriate permissions.
2. Run `ipconfig /release` if you need to clear the current lease.
3. Run `ipconfig /renew` to request a fresh address.
4. Review the returned IP address, subnet mask, and default gateway.
5. Test connectivity again after the lease changes.

```cmd
ipconfig /release
ipconfig /renew
ipconfig /all
```

**Testing DNS Resolution**

DNS resolution turns a host name into an IP address. If a site fails by name but works by IP, the local DNS cache or registration may be part of the problem. `ipconfig /flushdns` removes cached DNS entries so the client must query DNS again.

- `ipconfig /flushdns` is useful when stale or incorrect cached records may be causing lookup failures.
- `ipconfig /displaydns` helps you inspect what the client currently has cached.
- `ipconfig /registerdns` can help refresh the client’s DNS records when registration is needed.
- Dynamic update allows DNS clients to register and update resource records when changes occur, reducing manual zone maintenance.

```cmd
ipconfig /flushdns
ipconfig /displaydns
ipconfig /registerdns
```

**Practical Troubleshooting Flow**

Use DHCP renewal and DNS testing together when a Windows device has network symptoms that look like address or name-resolution problems. Start by confirming the adapter has a valid configuration, then clear the DNS cache if name lookups still fail.

- Check the current configuration with `ipconfig /all`.
- Renew the lease if the address looks wrong or missing.
- Flush the DNS cache if name resolution still fails.
- Re-test the hostname after each change.
- Use the results to separate IP assignment issues from DNS issues.

**Key Takeaways**

`ipconfig` supports both address renewal and DNS troubleshooting in one workflow. DHCP renewal fixes lease-related problems, while DNS cache flushing and registration help when name resolution is stale or incomplete. Together, these commands give you a fast first response for common Windows connectivity issues.
