## **Course:** CompTIA A+ Practical Labs & Troubleshooting (Exams 220-1201 & 220-1202)

**Purpose of the technician workstation**

A technician workstation gives you a consistent place to build, test, and document lab work. In A+ practice, that consistency matters because you often repeat the same setup steps, compare results, and need a clean way to store evidence from each task.

The course context emphasizes hands-on labs and practical troubleshooting, so your workstation should support repeatable work rather than one-off experimentation. A good setup helps you move quickly between tasks while keeping your notes, screenshots, and lab artifacts organized.

- Keep the workstation dedicated to lab activity when possible.
- Use a predictable naming pattern for files and folders.
- Store evidence so you can review it later without searching through unrelated files.
- Treat the workstation as a staging area for troubleshooting and validation.

**Why evidence folders matter**

Evidence folders help you separate what you did from what you observed. That separation is useful when you need to prove a configuration change, compare before-and-after states, or revisit a troubleshooting path.

A practical folder structure also reduces confusion when you work through multiple labs. The evidence should be easy to identify by topic, date, and task so you can find it again during review or exam prep.

> Keep evidence organized as you create it. Waiting until the end usually leads to missing screenshots, unclear filenames, and mixed lab artifacts.

**Recommended folder structure**

Use a top-level lab folder, then divide it into clear subfolders for each activity. A simple structure is easier to maintain than a deeply nested one.

```text
Aplus-Labs/
├── 01-Workstation-Setup/
├── 02-Lab-Notes/
├── 03-Screenshots/
├── 04-Logs/
└── 05-Exports/
```

This layout keeps the workstation setup separate from the evidence you collect during labs. It also gives you a place for notes, screenshots, logs, and exported files without mixing them together.

- `01-Workstation-Setup/` for baseline configuration files and setup references.
- `02-Lab-Notes/` for written observations and task summaries.
- `03-Screenshots/` for visual proof of settings, errors, and results.
- `04-Logs/` for text-based output and troubleshooting records.
- `05-Exports/` for saved reports or exported configuration data.

**Naming conventions**

Use filenames that tell you what the file contains without opening it. Include the lab topic, a short descriptor, and a date or sequence number if needed.

Examples:

```text
2026-04-21-networking-ipconfig-before.png
2026-04-21-networking-ipconfig-after.png
2026-04-21-lab-notes.txt
2026-04-21-event-log-export.evtx
```

Good names make it easier to compare evidence across steps. They also help when you revisit a lab later and need to identify the exact point where a change occurred.

- Use consistent separators such as hyphens or underscores.
- Keep names short but descriptive.
- Avoid vague labels like `image1.png` or `newfile.docx`.
- Add dates when order matters.

**Workstation setup habits**

A technician workstation should support efficient lab work and reduce friction. The goal is to make common tasks easy to repeat and easy to document.

- Keep tools and folders in the same place each time.
- Use a clean desktop or a dedicated working directory.
- Save evidence immediately after you capture it.
- Close unrelated applications when you need focused troubleshooting.

If you are working with virtual machines or lab templates, a stable workstation layout helps you track which system you are configuring and which evidence belongs to which test. That is especially useful when you are building or preconfiguring lab systems for repeatable use.

**Practical workflow for evidence collection**

1. Create the lab folder before you begin the task.
2. Add subfolders for notes, screenshots, logs, and exports.
3. Capture evidence as you complete each step.
4. Save files with descriptive names right away.
5. Review the folder at the end to confirm that the evidence is complete.

This workflow keeps your documentation aligned with the work you performed. It also makes it easier to retrace your steps if a configuration does not behave as expected.

**What to include in evidence**

Evidence should show both the action and the result. In practice, that often means pairing a note with a screenshot or log entry.

- Before-and-after screenshots for configuration changes.
- Text notes that explain what you changed.
- Log exports that support troubleshooting.
- Saved outputs that confirm the system state.

When you collect evidence this way, you build a clear record of the lab process. That record is useful for study review and for checking whether a troubleshooting step actually resolved the issue.

**Common organization mistakes to avoid**

- Mixing unrelated labs in the same folder.
- Saving screenshots without meaningful names.
- Storing notes in multiple places.
- Waiting until the end to sort evidence.
- Overcomplicating the folder tree.

A simple, repeatable structure is usually the best choice. It is easier to maintain during a lab session and easier to understand later when you return to study the material.

**Key takeaway**

A technician workstation is most effective when it supports repeatable lab work, and an evidence folder structure is most effective when it makes your results easy to find and verify. Keep the setup simple, name files clearly, and capture evidence as you go.
