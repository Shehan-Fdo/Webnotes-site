## **Course:** CompTIA A+ Practical Labs & Troubleshooting (Exams 220-1201 & 220-1202)

**Restore points and rollback basics**

A restore point captures a system state so you can return the operating system and related settings to an earlier point in time if a change causes problems. On Windows, `Checkpoint-Computer` creates a system restore point on the local computer.

Restore points are useful for undoing software installs or settings changes, but they are not a full backup strategy. They do not preserve personal files, and they do not replace image backups when you need broader recovery coverage.

- A restore point records system files, settings, installed programs, and registry state.
- It helps you roll back the OS after a bad change.
- It does not include personal documents or full-disk partition recovery.
- It is best treated as a recovery aid, not a backup replacement.

> `Checkpoint-Computer` is supported only on client Windows operating systems such as Windows 10 and Windows 11.

**Creating Windows restore points**

Use `Checkpoint-Computer` when you need a restore point on a supported Windows client system. The cmdlet accepts a description and can use a restore point type.

```powershell
Checkpoint-Computer -Description "Install MyApp"
```

This example creates a restore point named `Install MyApp` and uses the default `APPLICATION_INSTALL` restore point type. You can also create a restore point for settings changes with a different restore point type when needed.

- `-Description` gives the restore point a clear label.
- `-RestorePointType` lets you describe the kind of change being protected.
- The cmdlet creates the restore point on the local computer.
- Windows limits you to one checkpoint per day.

**Snapshots on macOS**

On APFS (Apple File System) startup drives, snapshots capture the state of the volume at a specific point in time. Time Machine can use these snapshots to restore files, folders, or the whole startup volume.

Snapshots are stored on the startup volume. They are not the same as the older HFS+ local backups stored in `.MobileBackups`.

- APFS snapshots preserve the volume state at a point in time.
- Time Machine can use them for file, folder, or volume recovery.
- They live on the startup volume itself.
- They differ from HFS+ local backups.

**Rollback and recovery planning**

Rollback procedures depend on the platform and the type of recovery you need. For Windows, a restore point can return the system to a prior state after a bad install or configuration change. For macOS APFS systems, snapshots support recovery of the startup volume state, but volume-level rollback is typically handled in recovery-oriented workflows.

Use the least disruptive recovery method that matches the problem:

1. Identify whether the issue is limited to settings, installed software, or broader system state.
2. Choose a restore point or snapshot if the problem matches that scope.
3. Confirm that the recovery method fits the platform and storage format.
4. Restore or roll back, then verify the system starts and behaves normally.

**Operational limits and cautions**

Restore points and snapshots are practical, but they have limits that affect planning. Windows restore points are constrained by the one-per-day rule, which can interfere with automation attempts that try to create multiple checkpoints in a short period. On macOS, snapshots can consume disk space and may require careful management when storage is tight.

- Scheduled restore-point creation can fail if another checkpoint already occurred that day.
- Snapshot storage can reduce available disk space.
- Recovery behavior may differ between file-level restore and full-volume rollback.
- Always verify the result after rollback before returning the system to service.

**Exam focus**

For A+ troubleshooting, focus on what each recovery tool protects and what it does not. A restore point is for Windows system state. A snapshot is a point-in-time volume state, especially relevant on APFS systems. Neither should be confused with a complete backup.

- Restore point: Windows system state recovery.
- Snapshot: point-in-time volume recovery.
- Backup: broader protection, including data not covered by restore points or snapshots.
- Rollback: return to a known-good state after a failed change.
