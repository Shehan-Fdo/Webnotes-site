## **Course:** CompTIA A+ Practical Labs & Troubleshooting (Exams 220-1201 & 220-1202)

**Storage and startup troubleshooting overview**

macOS includes built-in tools that help you diagnose drive problems, repair file system issues, and restore data when a system will not start normally. For A+ troubleshooting, the key idea is to match the tool to the symptom: use recovery options when startup is blocked, and use Disk Utility when the drive needs inspection or repair.

- **Disk Utility** is the main macOS tool for disk maintenance.
- **First Aid** is the repair function you use when a drive shows errors or instability.
- **Time Machine** supports file recovery and rollback to earlier versions.
- **Recovery mode** helps when the operating system cannot boot normally.

> Use conservative troubleshooting: verify the symptom first, then choose the least disruptive repair option before attempting broader recovery steps.

**Disk Utility and First Aid**

Disk Utility is used as needed rather than on a fixed schedule. It is the built-in utility for storage maintenance, and its First Aid feature checks a drive for problems and attempts repair. This is the macOS equivalent of a basic disk repair workflow.

When you suspect storage corruption, focus on symptoms such as slow access, mounting problems, or startup failures tied to the drive. Disk Utility helps you confirm whether the issue is with the volume itself rather than with an application or user profile.

- Open Disk Utility when the drive is accessible but behaving abnormally.
- Run **First Aid** to check and repair the file system.
- Treat repair results as evidence, not a guarantee; if the drive continues to fail, escalate to backup and replacement planning.
- Use the tool as needed, not as routine maintenance.

**Recovery mode for startup problems**

If the Mac will not boot normally, recovery tools become the priority. Startup problems often require you to work outside the installed operating system so you can inspect the disk or restore the system.

Recovery mode gives you a path to repair or restore when the normal desktop is unavailable. In practical troubleshooting, this is where you begin when the machine is stuck in a boot loop, hangs during startup, or fails before you can reach standard utilities.

1. Start the system in recovery mode.
2. Open Disk Utility from the recovery environment.
3. Run First Aid on the affected volume.
4. If repair does not resolve the issue, move to backup-based recovery or further escalation.

**Time Machine for file recovery**

Time Machine is macOS’s built-in backup utility. It creates automatic backups and lets you restore files from earlier points in time. It also supports snapshots when you are away from the backup drive, then updates the backup later when the system reconnects.

This makes Time Machine useful when the problem is not a failed boot but lost or changed data. You can restore a file from a familiar Finder-style interface and choose the date you want to recover from.

- Use **Time Machine** when you need to recover deleted or changed files.
- Restore from the date that matches the last known good version.
- Expect hourly backups for the last 24 hours, daily backups for the last month, and weekly backups after that.
- Remember that older backups are removed when the backup drive fills up.

**File and disk image basics**

macOS uses file extensions to identify file types, and that matters during troubleshooting. A `.dmg` file is a disk image that can be mounted as a volume, similar in purpose to an ISO image. A `.pkg` file is an installer package.

These file types matter because startup and installation issues often involve mounted images, installers, or corrupted packages rather than the operating system itself.

- `.dmg` files mount like a drive in Finder.
- `.pkg` files are installer packages.
- Recognizing the file type helps you decide whether the issue is with installation media, a mounted image, or the system volume.

**Practical troubleshooting approach**

A good workflow is to separate startup failure from storage failure, then choose the right tool. If the system cannot boot, use recovery mode first. If the system boots but the drive behaves badly, use Disk Utility and First Aid. If the issue is missing data, use Time Machine.

- Start with the least invasive option.
- Confirm whether the problem is boot-related or storage-related.
- Repair the disk before replacing the system or reinstalling software.
- Restore from backup when the data issue is the main concern.

**Key takeaways**

macOS gives you three core tools for this troubleshooting area: Recovery mode for startup access, Disk Utility with First Aid for drive repair, and Time Machine for restoring files. For exam and lab work, the important skill is knowing which tool fits the symptom and using it in the correct order.
