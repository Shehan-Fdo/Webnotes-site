## **Course:** CompTIA A+ Practical Labs & Troubleshooting (Exams 220-1201 & 220-1202)

**ChromeOS Flex installation overview**

ChromeOS Flex is a cloud-first operating system for existing PCs and Macs. You can install it from USB media or deploy it remotely via the cloud, but this topic focuses on creating boot media and using it on a test device.

Before you begin, verify that the target model is certified when possible. Google tests and maintains certified models, and it does not guarantee performance, functionality, or stability on non-certified devices. If you test on unsupported hardware, use conservative expectations and be ready for issues.

- ChromeOS Flex can be installed on Windows, Mac, or Linux devices.
- The device used to create the USB installer can be different from the device you plan to install on.
- Certified models are expected to work; non-certified models may work but are not guaranteed.
- Devices not on the certified list are not eligible for Chrome Education or Chrome Enterprise support.

**What you need**

The installer creation method depends on the device you use. Google documents two main paths: the recommended Chromebook Recovery Utility method and a direct download method from Google.

- **Chromebook Recovery Utility** requires a Chrome browser on ChromeOS, Windows, or Mac.
- **Direct download** works on ChromeOS, Windows, Mac, or Linux and does not require Chrome browser.
- The USB drive is erased during installer creation.
- Use a test device when you want to validate compatibility before broader rollout.

> All contents on the USB drive will be erased. Back up anything important before you start.

**Certified and non-certified device planning**

Google maintains a certified models list with status categories that help you judge risk before installation. Use that list to compare the intended device with known-supported hardware.

- **Certified** models are expected to work with ChromeOS Flex.
- **Minor issues expected** models should support at least basic functionality, but minor issues may appear.
- **Major issues expected** models are known to have major issues, such as boot problems, and are not currently recommended.
- **Decertified** models have reached end of support.

If you are working with a non-certified model, follow the same installation steps as you would for a certified model, but plan for possible instability across updates.

**Create the USB installer with Chromebook Recovery Utility**

The Chromebook Recovery Utility is the recommended method when you have a Chrome browser on a supported desktop OS or a ChromeOS device. The utility installs as a Chrome extension.

1. Open the Chrome browser on your ChromeOS, Windows, or Mac device.
2. Go to the Chrome Web Store.
3. Click `Add to Chrome`.
4. When prompted, click `Add extension`.
5. Confirm that the Chromebook Recovery Utility extension is enabled.
6. Open the extension from the browser and follow the installer prompts.
7. Write the ChromeOS Flex image to the USB drive.

This method is useful when you want a guided workflow and are already working inside Chrome.

**Create the USB installer by direct download**

If you are not using the Chromebook Recovery Utility, Google also provides a download-based path for ChromeOS Flex installer creation.

- This method supports ChromeOS, Windows, Mac, and Linux devices.
- Chrome browser is not required.
- The same USB erase warning still applies.
- Use this option when you want to prepare media from a system that does not fit the extension-based workflow.

**Boot the test device from USB**

After the installer is ready, use it to start the target device and begin the ChromeOS Flex installation process.

1. Insert the USB installer into the test device.
2. Power on the device and open the boot menu if needed.
3. Select the USB device as the boot source.
4. Start the ChromeOS Flex installer.
5. Follow the on-screen installation prompts.
6. Complete the install and verify that the device boots into ChromeOS Flex.

If the device does not boot from USB, review firmware settings and boot support for the model. Google specifically notes that administrators familiar with BIOS and UEFI settings are the best fit for non-certified installations.

**Troubleshooting and practical exam points**

ChromeOS Flex installation questions often focus on compatibility, media creation, and boot order rather than deep OS configuration. Keep the workflow simple and methodical.

- Check the certified models list before you install.
- Use a test device first when the hardware is unfamiliar.
- Remember that the USB installer is not the same as the target device.
- Expect the USB drive to be wiped during creation.
- Treat non-certified hardware as a risk-managed test, not a guaranteed deployment.

**Key takeaways**

ChromeOS Flex gives older PCs and Macs a second life with a fast, secure, cloud-first operating system. For lab work, the important steps are choosing a compatible device, creating the USB installer correctly, and booting the test device from that media.
