## **Course:** CompTIA A+ Practical Labs & Troubleshooting (Exams 220-1201 & 220-1202)

**Troubleshooting approach**

Start with the simplest physical and configuration checks before you move to deeper software or hardware diagnosis. For wireless, input, and display problems, that usually means confirming power, connections, device state, and whether the issue is isolated to one device or affects multiple devices.

- **Wi-Fi** issues often begin with basic connectivity checks, not advanced repair.
- **Keyboard** and **touchpad** problems can come from dirt, device state, or unsupported hardware behavior.
- **Display** problems commonly trace back to cabling, input selection, or adapter/driver-related issues.
- In ChromeOS Flex environments, hardware support can differ from standard ChromeOS, so verify compatibility before assuming a device fault.

**Wi-Fi and installer-related checks**

ChromeOS Flex documentation highlights that USB installer problems can look like network or boot failures during setup. If a USB installer does not write correctly, stalls, or fails to boot, recreate the installer and try a different USB drive manufacturer. This is a practical reminder to separate media issues from actual wireless problems.

> If the installer halts at `0%`, fails with error code `1`, or the EFI file is missing, treat the USB media as suspect first.

- Reformat the USB drive and recreate the installer.
- Try a different USB drive brand if the first one fails.
- Confirm the device can boot from the installer before you troubleshoot wireless behavior.

**Keyboard and touchpad issues**

For touchpad problems, start with cleaning and simple physical interaction checks. A dirty surface or debris around the edges can prevent proper response, so a quick cleaning is a reasonable first step. If the touchpad still does not respond, a brief repeated tap can sometimes help dislodge debris.

1. Turn the device off.
2. Clean the touchpad surface with a microfiber cloth and a 50-50 mix of isopropyl alcohol and distilled water.
3. Use compressed air around the edges if dust is present.
4. Test the touchpad again after cleaning.
5. If needed, gently tap your fingers on the touchpad for about 10 seconds.

- **Touchpad** failures may be mechanical or contamination-related.
- A nonresponsive touchpad does not always mean the device is broken.
- Keep the troubleshooting sequence simple before replacing parts.

**Display issues**

A display that shows `No Signal` while the device is powered on often points to a connection or input problem rather than a failed display panel. In practical troubleshooting, check the cable path and confirm the monitor is set to the correct input source.

- **No Signal** commonly means the display is not receiving the expected video input.
- Loose power or video connections are a common first check.
- Incorrect input selection can mimic a hardware failure.
- If the issue persists, continue with adapter, driver, or GPU-related checks as appropriate.

**What to verify first**

Use a consistent order so you do not skip obvious causes.

- Confirm the device is powered on and awake.
- Check cables, ports, and adapters.
- Verify the correct input source on the monitor.
- Test with known-good peripherals or media when possible.
- Recreate or replace suspect USB installer media if boot/setup fails.

**Exam focus reminders**

The evidence supports a practical, first-step mindset:

- For display problems, do not jump straight to advanced component replacement.
- For touchpad issues, clean and retest before assuming failure.
- For ChromeOS Flex setup problems, validate the installer USB before blaming the target device.
- For wireless or input issues, isolate whether the fault is environmental, configuration-based, or hardware-related.
