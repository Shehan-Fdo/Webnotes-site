## **Course:** Comptia A+ 220-1201

**Diagnostic Tools in Troubleshooting**

Diagnostic tools help you identify whether a problem starts at boot, during hardware checks, or inside the operating system.

- **Power-On Self-Test (POST)** runs when you start the computer.
- POST checks core components such as the **CPU**, video, keyboard, and memory.
- POST can show error messages on screen when video works.
- POST can also use beep codes when the screen is unavailable.
- Beep codes and messages vary by manufacturer.
- Use vendor documentation to interpret system-specific codes.

**Startup and Hardware Symptoms**

Common hardware symptoms often point to the stage where the failure occurs.

- A blank screen can indicate a display or startup problem.
- No power can indicate a power delivery issue.
- Sluggish performance can indicate a resource or hardware issue.
- Overheating can cause instability and shutdowns.
- Blue Screen of Death (**BSoD**) indicates a Windows stop error.
- Proprietary error messages may appear before the operating system loads.

**Using Diagnostic Output**

You use diagnostic output to narrow the fault before replacing parts.

1. Observe the symptom during startup.
2. Note any beep codes or on-screen messages.
3. Compare the result with vendor guidance.
4. Test the likely failing component.
5. Confirm the fix by repeating the startup check.

**Real-Time Monitoring Tools**

Some tools help you troubleshoot after the system loads.

- **Task Manager** provides a real-time view of system activity.
- Task Manager helps you review processes, services, startup items, performance, networking, and users.
- Use it to spot abnormal resource use during troubleshooting.
- Open Task Manager with `Ctrl+Shift+Esc`.
- You can also open it from `Ctrl+Alt+Delete`.

**Practical Troubleshooting Approach**

Use diagnostic tools in a simple order.

1. Check the symptom.
2. Identify the system stage where the failure appears.
3. Read the available diagnostic output.
4. Use the appropriate tool for the stage of failure.
5. Verify normal operation after the repair.
