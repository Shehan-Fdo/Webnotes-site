## **Course:** Comptia A+ 220-1201

**Cloud Deployment Models**

Cloud deployment models describe how cloud resources are shared, controlled, and delivered to users.

**Public Cloud**

- A **public cloud** is the most common deployment model.
- Multiple clients share the same provider infrastructure.
- The cloud vendor owns and operates the underlying resources.
- You use it when you want broad access and flexible scaling.

**Private Cloud**

- A **private cloud** is used by one organization only.
- The organization often owns the hardware and data center.
- You use it when you need tighter control over access and resources.
- It fits environments with stronger internal security or governance needs.

**Hybrid Cloud**

- A **hybrid cloud** combines public and private clouds.
- It uses automation and orchestration to connect the environments.
- You use it when you want to balance control with flexibility.
- It supports workloads that need different placement requirements.

**Community Cloud**

- A **community cloud** is shared by organizations with common needs.
- It supports shared security, compliance, or jurisdictional requirements.
- Governments and similar groups may use this model.
- You use it when multiple organizations need the same policy alignment.

**Choosing a Deployment Model**

- Match the model to business requirements.
- Consider access control, compliance, and ownership.
- Consider whether you need shared infrastructure or dedicated infrastructure.
- Consider whether workloads need to move between environments.

**Exam Focus**

- Know the difference between shared and dedicated cloud infrastructure.
- Know that public cloud serves multiple clients.
- Know that private cloud serves one organization.
- Know that hybrid cloud combines public and private environments.
- Know that community cloud supports organizations with shared requirements.
