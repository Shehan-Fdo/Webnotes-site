## **Course:** Comptia A+ 220-1201

**Virtualization Basics**

Virtualization lets one physical system run a **virtual machine (VM)** that behaves like a separate computer.

- A VM can have its own **CPU**, memory, storage, and devices.
- Virtualization supports multiple operating systems on one host.
- A VM still depends on the host system for real hardware resources.

**Cloud and Virtualization Scope**

CompTIA A+ Core 1 includes virtualization and cloud computing concepts as part of the exam objectives.

- You should understand how virtual systems support IT support tasks.
- You should recognize that cloud services often rely on virtualized infrastructure.
- You should connect virtualization concepts to troubleshooting and support workflows.

**Troubleshooting Virtual Machines**

When a VM does not work as expected, start with the host and then check the guest configuration.

1. Confirm the host has enough available resources.
2. Verify the VM settings for CPU, RAM, storage, and device access.
3. Check whether the guest operating system starts and responds normally.
4. Compare the issue against a known-good VM or host configuration if available.

- Resource limits on the host can affect VM performance.
- Incorrect device assignment can prevent expected hardware access.
- Boot or startup issues may come from the guest configuration rather than the host.

**Common Support Checks**

Use practical checks to isolate whether the issue is in the host, the VM, or the cloud service.

- Verify the virtualization feature is enabled where required.
- Confirm the VM has the expected virtual hardware assigned.
- Check network connectivity inside the VM and on the host.
- Review whether the cloud service is available and properly configured.

**Exam Focus**

The exam expects you to explain virtualization and cloud computing concepts and apply basic troubleshooting logic.

- Know the difference between a host and a VM.
- Know that a VM can simulate hardware resources.
- Know that cloud services may depend on virtualized systems.
- Use a step-by-step approach when a virtual or cloud service fails.
