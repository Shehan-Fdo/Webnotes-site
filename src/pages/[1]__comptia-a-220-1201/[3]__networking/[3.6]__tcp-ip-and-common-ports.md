## **Course:** Comptia A+ 220-1201

**TCP/IP Overview**

TCP/IP (Transmission Control Protocol/Internet Protocol) is the core communication model for network traffic.

- **IP** addresses help route information to the correct device.
- **TCP** and **UDP** (User Datagram Protocol) carry data for services and applications.
- You use port numbers to identify the destination service on a host.

**TCP vs UDP**

TCP and UDP serve different transport needs.

- **TCP** is connection-oriented.
- TCP provides reliable delivery with acknowledgements.
- TCP can handle out-of-order packets and retransmission.
- **UDP** is used when lower overhead matters more than delivery confirmation.

**Common Port Concepts**

Applications often use well-known port numbers.

- Port numbers help you recognize services during troubleshooting.
- Firewalls often use TCP and UDP port numbers to allow or block traffic.
- Knowing common ports helps you diagnose communication issues faster.

**Common TCP and UDP Ports**

| Port | Protocol | Common Use |
|---|---|---|
| `20` / `21` | FTP | File transfer |
| `22` | SSH | Secure remote access |
| `23` | Telnet | Remote text-based access |
| `25` | SMTP | Email sending |
| `53` | DNS | Name resolution |
| `67` / `68` | DHCP | Address assignment |
| `80` | HTTP | Web traffic |
| `110` | POP3 | Email retrieval |
| `143` | IMAP | Email retrieval and sync |
| `389` | LDAP | Directory services |
| `443` | HTTPS | Secure web traffic |
| `445` | SMB | File and printer sharing |
| `636` | LDAPS | Secure LDAP |
| `3389` | RDP | Remote desktop access |

**Practical Troubleshooting Use**

You use port knowledge to narrow down service failures.

- Check whether the expected service is listening on the correct port.
- Verify firewall rules for the required port and protocol.
- Confirm the application matches the service you expect to reach.
- Use secure alternatives when available, such as `SSH` instead of `Telnet` and `HTTPS` instead of `HTTP`.

**Study Focus**

Memorize the most common ports for exam and support work.

- Match each port to its service.
- Distinguish `TCP` services from `UDP` services when needed.
- Focus on ports used for web, email, name resolution, remote access, and file sharing.
