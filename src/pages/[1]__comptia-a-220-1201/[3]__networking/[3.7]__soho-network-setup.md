## **Course:** Comptia A+ 220-1201

**SOHO Network Overview**

A small office/home office (SOHO) network often combines multiple functions into one device.

- A **SOHO router** can include routing, switching, wireless access point, and firewall features.
- This design reduces the number of separate devices you need.
- You still configure the device carefully; it is not just plug-and-play.

**Core Network Functions**

SOHO devices handle common network tasks for a small environment.

- **Router** functions move traffic between IP subnets.
- **Switch** functions connect wired devices on the local network.
- **Wireless access point** functions provide Wi-Fi access.
- **Firewall** functions help block unwanted inbound access from the internet.

**Initial Setup**

You should verify the basic network path before you add advanced settings.

1. Connect the SOHO device to the internet source, such as a cable modem.
2. Connect wired devices to Ethernet ports as needed.
3. Confirm that the device provides local network access.
4. Test internet connectivity from a client device.

**Wireless Configuration**

Wireless settings control how users find and join the network.

- Set a clear **SSID** (Service Set ID), which is the wireless network name.
- Consider changing default SSIDs such as `Linksys`, `Default`, or `Netgear`.
- You can disable SSID broadcast, but users who know the SSID can still connect.
- Use strong wireless encryption.
- Prefer `WPA2` over `WPA`.
- Avoid `WEP` because of known weaknesses.

**Access Control Options**

SOHO routers often include controls that limit who can connect.

- **MAC address filtering** limits access by device hardware address.
- This can reduce casual access attempts.
- It does not replace strong encryption.

**Firewall and Exposure Settings**

Firewall features control what outside systems can reach on your network.

- **Network Address Translation (NAT)** helps hide internal addresses behind the router.
- **Port forwarding** sends specific inbound traffic to an internal device.
- **Port triggering** opens ports only when a client starts a matching connection.
- A **DMZ** (demilitarized zone) can expose one device more directly to the internet.
- Use DMZ settings carefully because they increase exposure.

**Security Hardening**

You should secure the device immediately after first login.

- Change the default username and password.
- Default credentials are easy to find online.
- Update firmware when the vendor provides a fix.
- Review content filtering and other built-in security features.

**Practical Configuration Priorities**

Focus on the settings that most affect access and safety.

1. Change default administrative credentials.
2. Set the wireless SSID and encryption.
3. Confirm NAT and firewall behavior.
4. Add port forwarding only when a service needs it.
5. Review any DMZ or filtering settings before deployment.
