## **Course:** Cisco CCNA Complete Course (Exam 200-301 v1.1)

**Static routing fundamentals**

Static routes are manually configured paths to remote networks. You use them when you want predictable forwarding, minimal overhead, or a simple edge design. They do not rely on a routing protocol, so the router only knows what you tell it.

Static routing is common in small networks, stub networks, lab topologies, and as a backup path. It is also useful for default routing, route summarization, and blackholing traffic with a null interface.

- **Predictability** makes static routes useful when topology changes are rare.
- **Low overhead** means no routing protocol updates, timers, or neighbor relationships.
- **Manual maintenance** becomes a drawback as the network grows.

A static route is only useful if the router can forward the packet to a valid next hop or exit interface. If the next hop is unreachable, the route may still appear in the table depending on the configuration, but forwarding fails.

**IPv4 static route syntax**

The basic IPv4 static route command is:

```bash
ip route <destination-network> <subnet-mask> {<next-hop-ip> | <exit-interface> | <exit-interface> <next-hop-ip>} [administrative-distance]
```

Common examples:

```bash
ip route 192.168.20.0 255.255.255.0 10.1.1.2
ip route 192.168.30.0 255.255.255.0 GigabitEthernet0/0/1
ip route 192.168.40.0 255.255.255.0 GigabitEthernet0/0/1 10.1.1.2
```

- **Next-hop IP** is the most common choice on Ethernet links because it avoids some recursive lookup issues.
- **Exit interface** is often used on point-to-point links.
- **Administrative distance** can be set to influence route preference when multiple routes exist.

The router installs the route only when it can resolve the path. On multiaccess networks, using only an exit interface can cause the router to ARP for the destination next hop more broadly than necessary, so next-hop IP is usually cleaner.

**IPv4 host routes and default routes**

A host route is a route to a single IP address. In IPv4, the mask is `/32`, which is `255.255.255.255`. CCNA scenarios often use host routes for management addresses, loopbacks, or very specific reachability.

```bash
ip route 192.168.10.50 255.255.255.255 10.1.1.2
```

A default route matches all destinations not otherwise known. In IPv4, the destination is `0.0.0.0 0.0.0.0`.

```bash
ip route 0.0.0.0 0.0.0.0 10.1.1.2
```

- **Host route** is the most specific route possible for one destination.
- **Default route** is the least specific route and acts as a gateway of last resort.
- **Specific routes beat default routes** because longest-prefix match always wins.

**Summary static routes and null0**

A summary static route covers multiple more-specific networks with one route. You use it to reduce routing table size and simplify design. In exam scenarios, summary routes often point to `Null0` to prevent loops if traffic matches the summary but not any real subnet.

```bash
ip route 10.10.0.0 255.255.252.0 Null0
```

This route summarizes a block of addresses and discards packets that do not match a more specific route. The `Null0` interface is a logical discard interface, not a physical port.

- **Summary route** aggregates several networks into one prefix.
- **Default route** matches everything and is not a summary of a specific block.
- **Null0** is commonly used with summaries to avoid routing loops and misdelivery.

> If you summarize without a discard route, packets for unused addresses inside the summary may be forwarded incorrectly or loop if the topology has alternate paths.

**IPv6 static route syntax**

IPv6 static routes use `ipv6 route` and prefix length notation.

```bash
ipv6 route <prefix>/<length> {<next-hop-addr> | <exit-interface> | <exit-interface> <next-hop-addr>} [administrative-distance]
```

Examples:

```bash
ipv6 route 2001:db8:10:20::/64 2001:db8:1:2::2
ipv6 route 2001:db8:30:40::/64 GigabitEthernet0/0/1
ipv6 route 2001:db8:50:60::/64 GigabitEthernet0/0/1 2001:db8:1:2::2
```

IPv6 static routes behave similarly to IPv4 routes, but IPv6 commonly uses link-local next hops on directly connected links.

**IPv6 link-local next hop**

A common IOS syntax case is using a link-local address as the next hop. Because link-local addresses are only unique on the local link, you must specify the exit interface as well.

```bash
ipv6 route 2001:db8:10:20::/64 GigabitEthernet0/0/1 fe80::2
```

- **Link-local next hop** is valid only on the local segment.
- **Exit interface** is required with a link-local next hop so IOS knows which link to use.
- **Global unicast next hop** can be used without the interface in many cases, but the interface is still often clearer in labs.

**IPv6 host routes, default routes, and summaries**

An IPv6 host route uses `/128`, which matches one address exactly.

```bash
ipv6 route 2001:db8:10:20::50/128 2001:db8:1:2::2
```

An IPv6 default route uses `::/0`.

```bash
ipv6 route ::/0 2001:db8:1:2::2
ipv6 route ::/0 GigabitEthernet0/0/1 fe80::2
```

Summary routes in IPv6 work the same way conceptually as IPv4: you advertise or configure a shorter prefix that covers multiple more-specific networks. A summary route to `Null0` is also common for loop prevention.

```bash
ipv6 route 2001:db8:10::/48 Null0
```

- **/128** is the IPv6 host route equivalent.
- **::/0** is the IPv6 default route.
- **Summary prefixes** should be chosen on proper bit boundaries to avoid covering unrelated networks.

**Administrative distance and route preference**

Administrative distance (AD) is the trust value of a route source. Static routes default to AD 1, which makes them preferred over most dynamic routing protocol routes.

```bash
ip route 192.168.20.0 255.255.255.0 10.1.1.2 5
ipv6 route 2001:db8:10:20::/64 2001:db8:1:2::2 5
```

- **Lower AD** is preferred.
- **Floating static route** uses a higher AD so it acts as a backup.
- **Primary static route** usually keeps the default AD of 1.

A floating static route is useful when you want a backup path if a dynamic route disappears.

**Configuration workflow**

1. Configure interface addressing on each router.
2. Verify local interfaces are up and reachable.
3. Add static routes for remote networks.
4. Verify the routing table and end-to-end reachability.
5. Adjust or remove routes if the path changes.

Example IPv4 topology:

```bash
R1 G0/0: 10.1.1.1/30
R2 G0/0: 10.1.1.2/30
R2 G0/1: 192.168.20.1/24
```

Static route on R1:

```bash
ip route 192.168.20.0 255.255.255.0 10.1.1.2
```

Example IPv6 topology:

```bash
R1 G0/0: 2001:db8:1:2::1/64
R2 G0/0: 2001:db8:1:2::2/64
R2 G0/1: 2001:db8:20:1::1/64
```

Static route on R1:

```bash
ipv6 route 2001:db8:20:1::/64 2001:db8:1:2::2
```

**Verification commands**

Use the routing table first, then confirm interface status and end-to-end reachability.

```bash
show ip route
show ipv6 route
show ip interface brief
show ipv6 interface brief
ping 192.168.20.1
ping ipv6 2001:db8:20:1::1
traceroute 192.168.20.1
traceroute ipv6 2001:db8:20:1::1
```

- **`show ip route`** confirms the route is installed and shows the code `S` for static.
- **`show ipv6 route`** confirms IPv6 route installation and next-hop resolution.
- **Interface brief commands** help you verify that the local and remote interfaces are up/up and correctly addressed.
- **Ping and traceroute** confirm both reachability and path selection.

A static route may fail to work even if it is configured correctly if the outgoing interface is down, the next hop is wrong, or the connected interface addressing is incorrect.

**Troubleshooting static routes**

When a static route does not work, check the path from the source to the next hop and then from the next hop to the destination. Static routing problems are often caused by one missing return route.

- **Wrong next hop** prevents recursive resolution.
- **Wrong mask or prefix length** sends traffic to the wrong destination set.
- **Down interface** removes the connected path needed to reach the next hop.
- **Missing return route** causes one-way connectivity.
- **Overlapping summary** can hide more-specific routes if the topology is poorly designed.

Useful checks:

```bash
show running-config | include ^ip route|^ipv6 route
show ip route static
show ipv6 route static
show cdp neighbors
show lldp neighbors
```

If you need to remove or change a static route, negate the exact command and then add the corrected one.

```bash
no ip route 192.168.20.0 255.255.255.0 10.1.1.2
ip route 192.168.20.0 255.255.255.0 10.1.1.3

no ipv6 route 2001:db8:20:1::/64 2001:db8:1:2::2
ipv6 route 2001:db8:20:1::/64 2001:db8:1:2::3
```

- **`no ip route`** removes the exact IPv4 static route entry.
- **`no ipv6 route`** removes the exact IPv6 static route entry.
- **Re-entering the route** with the corrected next hop or interface updates the configuration cleanly.

**Exam-focused distinctions**

Static route questions often test whether you can distinguish among route types and choose the correct syntax.

| Route type | IPv4 example | IPv6 example | Typical use |
|---|---|---|---|
| Host route | `ip route 192.168.10.50 255.255.255.255 10.1.1.2` | `ipv6 route 2001:db8:10:20::50/128 2001:db8:1:2::2` | Single host reachability |
| Default route | `ip route 0.0.0.0 0.0.0.0 10.1.1.2` | `ipv6 route ::/0 2001:db8:1:2::2` | Gateway of last resort |
| Summary route | `ip route 10.10.0.0 255.255.252.0 Null0` | `ipv6 route 2001:db8:10::/48 Null0` | Aggregate multiple subnets |
| Floating static | `ip route 192.168.20.0 255.255.255.0 10.1.1.2 200` | `ipv6 route 2001:db8:20:1::/64 2001:db8:1:2::2 200` | Backup path |

**Practical lab example**

A simple two-router lab helps you validate both IPv4 and IPv6 static routing.

```bash
R1(config)# interface g0/0
R1(config-if)# ip address 10.1.1.1 255.255.255.252
R1(config-if)# ipv6 address 2001:db8:1:2::1/64
R1(config-if)# no shutdown

R2(config)# interface g0/0
R2(config-if)# ip address 10.1.1.2 255.255.255.252
R2(config-if)# ipv6 address 2001:db8:1:2::2/64
R2(config-if)# no shutdown

R2(config)# interface g0/1
R2(config-if)# ip address 192.168.20.1 255.255.255.0
R2(config-if)# ipv6 address 2001:db8:20:1::1/64
R2(config-if)# no shutdown

R1(config)# ip route 192.168.20.0 255.255.255.0 10.1.1.2
R1(config)# ipv6 route 2001:db8:20:1::/64 2001:db8:1:2::2
```

After configuration, verify:

```bash
show ip route static
show ipv6 route static
show ip interface brief
show ipv6 interface brief
ping 192.168.20.1
ping ipv6 2001:db8:20:1::1
```

**Key takeaways**

Static routes are simple, deterministic, and highly testable in CCNA labs. You should know how to configure IPv4 and IPv6 routes, host routes, default routes, summary routes, and `Null0` discard routes. You should also know how to verify routes, remove them with `no ip route` or `no ipv6 route`, and troubleshoot using interface and routing-table checks.
