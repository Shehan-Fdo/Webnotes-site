## **Course:** Cisco CCNA Complete Course (Exam 200-301 v1.1)

**IPv4 fundamentals**

<abbr title="Internet Protocol version 4">IPv4</abbr> is the most common Layer 3 addressing scheme in traditional enterprise networks. It uses a 32-bit address written as four decimal octets separated by dots, such as `192.168.1.10`. Each octet represents 8 bits, so the full address space contains about 4.3 billion unique addresses.

IPv4 addresses identify both the network and the host. The subnet mask tells you which bits belong to the network portion and which bits belong to the host portion. This separation is essential for routing, subnetting, and determining whether a destination is local or remote.

- IPv4 is a **32-bit** logical address.
- It is written in **dotted-decimal** notation.
- Each octet ranges from `0` to `255`.
- The subnet mask defines the **network boundary**.
- Routers use IPv4 addresses to forward packets between networks.

**Binary structure and decimal conversion**

IPv4 is easier to understand when you think in binary. Each octet contains 8 bits, and each bit has a value. The common bit values in one octet are:

- `128`
- `64`
- `32`
- `16`
- `8`
- `4`
- `2`
- `1`

For example, the decimal value `192` equals `128 + 64`, so the binary form is `11000000`. Understanding binary helps you calculate subnet sizes, network IDs, broadcast addresses, and usable host ranges.

```text
Decimal:  192.168.1.10
Binary:   11000000.10101000.00000001.00001010
Mask:     255.255.255.0
Binary:   11111111.11111111.11111111.00000000
```

A subnet mask uses contiguous `1` bits for the network portion and contiguous `0` bits for the host portion. In the example above, the first 24 bits are network bits, so the prefix length is `/24`.

**Classful IPv4 addressing**

Classful addressing is the original IPv4 model. It divides addresses into fixed classes based on the first octet. Modern networks use classless addressing and CIDR (<abbr title="Classless Inter-Domain Routing">CIDR</abbr>), but classful ranges still matter for CCNA because they help you recognize address types quickly.

| Class | First Octet Range | Default Mask | Typical Use |
|---|---:|---:|---|
| A | 1-126 | `255.0.0.0` (`/8`) | Very large networks |
| B | 128-191 | `255.255.0.0` (`/16`) | Medium networks |
| C | 192-223 | `255.255.255.0` (`/24`) | Small networks |
| D | 224-239 | N/A | Multicast |
| E | 240-255 | N/A | Experimental/reserved |

Important classful notes:

- `127.x.x.x` is reserved for **loopback** testing, not Class A host assignment.
- Class D addresses are used for **multicast**, not unicast hosts.
- Class E addresses are reserved and not used for normal host addressing.
- Classful defaults are historical; real networks usually apply custom masks.

**Private and special-use IPv4 ranges**

Some IPv4 ranges are reserved for internal use or special functions. You should recognize them immediately because they are common in labs, home networks, and enterprise designs.

- `10.0.0.0/8` is private.
- `172.16.0.0/12` is private.
- `192.168.0.0/16` is private.
- `127.0.0.0/8` is loopback.
- `169.254.0.0/16` is link-local <abbr title="Automatic Private IP Addressing">APIPA</abbr> addressing.
- `224.0.0.0/4` is multicast.
- `255.255.255.255` is the limited broadcast address.

> `169.254.0.0/16` usually appears when a host cannot obtain an address from <abbr title="Dynamic Host Configuration Protocol">DHCP</abbr>. It is not a valid routed enterprise address.

**Network, host, and broadcast portions**

The subnet mask separates the address into three practical ideas:

- **Network address**: identifies the subnet itself.
- **Host address**: identifies a device on that subnet.
- **Broadcast address**: reaches all hosts in the subnet.

For a `/24` network like `192.168.10.0/24`:

- Network address: `192.168.10.0`
- Usable hosts: `192.168.10.1` through `192.168.10.254`
- Broadcast address: `192.168.10.255`

The first address in a subnet is the network ID, and the last address is the broadcast address. These two addresses are not assignable to hosts.

**Subnet masks and prefix lengths**

A subnet mask can be written in dotted-decimal form or as a prefix length. The prefix length counts the number of network bits.

Examples:

- `255.0.0.0` = `/8`
- `255.255.0.0` = `/16`
- `255.255.255.0` = `/24`
- `255.255.255.128` = `/25`
- `255.255.255.192` = `/26`

A longer prefix means fewer host bits and fewer usable hosts. The number of usable hosts is generally:

`2^host bits - 2`

The subtraction of 2 accounts for the network and broadcast addresses. For example:

- `/24` leaves 8 host bits: `2^8 - 2 = 254`
- `/26` leaves 6 host bits: `2^6 - 2 = 62`
- `/30` leaves 2 host bits: `2^2 - 2 = 2`

**Practical subnetting examples**

A `/26` subnet divides a `/24` into four equal subnets. The block size is `64` because `256 - 192 = 64`.

```text
192.168.1.0/26    hosts 1-62     broadcast 63
192.168.1.64/26   hosts 65-126   broadcast 127
192.168.1.128/26  hosts 129-190  broadcast 191
192.168.1.192/26  hosts 193-254  broadcast 255
```

A `/30` subnet is often used for point-to-point links because it provides exactly two usable host addresses.

```text
10.0.0.0/30
Network:   10.0.0.0
Hosts:     10.0.0.1, 10.0.0.2
Broadcast: 10.0.0.3
```

This is efficient for router-to-router links in smaller designs, though many modern networks also use `/31` on point-to-point links when supported.

**How to identify the class and subnet quickly**

When you see an IPv4 address, use the first octet to infer the classful range, then use the mask to determine the actual subnet.

1. Look at the first octet.
2. Identify whether the address is private, public, or special-use.
3. Read the prefix length or mask.
4. Calculate the network, host range, and broadcast address.
5. Confirm whether the address is valid for a host.

Example:

- Address: `172.20.14.9/20`
- First octet `172` suggests Class B range.
- `172.20.14.9` is inside the private `172.16.0.0/12` block.
- `/20` means the subnet mask is `255.255.240.0`.
- The subnet size in the third octet is `16` because `256 - 240 = 16`.

**Cisco IOS verification commands**

On Cisco devices, you often verify IPv4 configuration with `show` commands. These commands help you confirm the assigned address, mask, and interface state.

```bash
show ip interface brief
show running-config interface g0/0
show ip route
show interfaces g0/0
```

Typical output clues:

- `show ip interface brief` shows the interface IP address and status.
- `show running-config interface g0/0` shows the configured address and mask.
- `show ip route` confirms connected networks.
- `show interfaces g0/0` provides detailed Layer 1/2 and Layer 3 information.

Example interface configuration:

```cisco
conf t
interface g0/0
 ip address 192.168.10.1 255.255.255.0
 no shutdown
end
```

You can also use prefix notation in some contexts, but Cisco IOS interface configuration typically uses dotted-decimal masks.

**Host configuration examples**

A workstation needs a valid IP address, subnet mask, default gateway, and often DNS server information. The default gateway is the router interface in the same subnet.

Example host settings for `192.168.10.0/24`:

- IP address: `192.168.10.25`
- Subnet mask: `255.255.255.0`
- Default gateway: `192.168.10.1`
- DNS server: `8.8.8.8` or an internal DNS server

If the gateway is outside the local subnet, the host cannot reach it directly at Layer 2. The host must ARP for the gateway’s MAC address and send traffic to that next hop.

**Address planning considerations**

Good IPv4 planning reduces waste and simplifies troubleshooting. You should choose subnet sizes based on host needs, growth, and topology.

- Use larger subnets for user VLANs with many endpoints.
- Use smaller subnets for infrastructure links and management networks.
- Keep point-to-point links efficient with `/30` or `/31` where appropriate.
- Reserve address space for future expansion.
- Document subnet purpose, gateway, and DHCP scope boundaries.

A practical enterprise plan often separates:

- User VLANs
- Voice VLANs
- Server networks
- Management networks
- Transit links
- Guest networks

This separation improves security, broadcast containment, and routing clarity.

**Common mistakes to avoid**

- Confusing the **network address** with a usable host address.
- Assigning the **broadcast address** to a device.
- Using the wrong subnet mask on a host or router interface.
- Forgetting that private addresses are not routable on the public Internet without translation.
- Assuming classful defaults instead of checking the actual prefix length.
- Misreading `169.254.x.x` as a valid static address.

**Quick reference**

- IPv4 uses 32 bits and dotted-decimal notation.
- The subnet mask defines network and host portions.
- Class A, B, and C are the main unicast classful ranges.
- Class D is multicast.
- Class E is reserved.
- Private ranges are `10/8`, `172.16/12`, and `192.168/16`.
- Network and broadcast addresses are not assignable to hosts.
- CIDR and prefix lengths are the modern way to describe subnets.

**Exam focus**

For CCNA, you should be able to:

- Identify whether an address is private, public, loopback, link-local, or multicast.
- Match a subnet mask to its prefix length.
- Determine the network and broadcast address for a subnet.
- Recognize classful ranges and default masks.
- Verify IPv4 settings on Cisco devices with `show` commands.
- Explain why subnetting is used for efficient address allocation and segmentation.
