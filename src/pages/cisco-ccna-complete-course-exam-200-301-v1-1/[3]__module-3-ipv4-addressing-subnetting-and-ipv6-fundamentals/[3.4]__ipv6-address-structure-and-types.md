**IPv6 address fundamentals**

IPv6 (Internet Protocol version 6) uses 128-bit addresses to solve IPv4 exhaustion and improve address planning. You read IPv6 in hexadecimal, split into eight 16-bit groups called hextets, separated by colons.

A full IPv6 address looks like this:

```text
2001:0db8:0000:0000:0000:ff00:0042:8329
```

Key ideas:

- **128 bits** provide a vastly larger address space than IPv4.
- **Hexadecimal** notation uses digits `0-9` and letters `a-f`.
- **Hextets** are 16-bit chunks, each written as four hex digits in full form.
- **Prefix length** replaces the IPv4-style subnet mask, such as `/64`.

IPv6 addressing is designed for hierarchical routing and simpler host configuration. In most enterprise LANs, you commonly see `/64` on user and server subnets because Stateless Address Autoconfiguration (SLAAC) expects that size.

**IPv6 notation and compression**

IPv6 addresses often appear shortened. You should know how to expand and compress them correctly.

Rules:

- Remove leading zeros in a hextet.
- Replace one contiguous run of all-zero hextets with `::` once per address.
- Do not use `::` more than once in the same address.

Examples:

```text
Full:      2001:0db8:0000:0000:0000:ff00:0042:8329
Compressed 2001:db8::ff00:42:8329
```

Another example:

```text
Full:      fe80:0000:0000:0000:0202:b3ff:fe1e:8329
Compressed fe80::202:b3ff:fe1e:8329
```

> `::` is only valid when it replaces one or more consecutive zero hextets. If the address already contains two `::` sequences, it is invalid.

When you troubleshoot, expand the address mentally or on paper so you can compare prefixes, interface IDs, and network boundaries accurately.

**Prefix length and subnetting**

IPv6 uses prefix length notation to identify the network portion of the address. For example, `/64` means the first 64 bits are the network prefix and the remaining 64 bits are the interface identifier.

Common prefix ideas:

- `/64` is the standard LAN prefix in most designs.
- `/128` identifies a single host address.
- `/48` is often assigned to an organization for internal subnetting.
- `/56` and `/60` are common delegated sizes from ISPs to customers.

A `/64` subnet gives you `2^64` interface identifiers, which is why IPv6 subnet exhaustion is not a practical concern in normal LAN design. The challenge is planning, not conserving addresses.

Example subnet:

```text
2001:db8:acad:10::/64
```

In that subnet:

- `2001:db8:acad:10` is the network prefix.
- The remaining 64 bits identify the interface.
- Hosts can use SLAAC, DHCPv6, or manual configuration depending on policy.

**IPv6 address types**

IPv6 uses several address types, and you should recognize them by prefix and behavior.

**Unicast addresses**

Unicast identifies one interface. A packet sent to a unicast address is delivered to a single destination.

Common unicast categories:

- **Global unicast**: Routable on the public Internet, similar in purpose to public IPv4 addresses.
- **Unique local address (ULA)**: Private addressing for internal use, not routed on the public Internet.
- **Link-local**: Automatically created on every IPv6-enabled interface and used only on the local link.

Global unicast addresses typically fall within `2000::/3`, which includes most globally routable IPv6 space.

Example:

```text
2001:db8:acad:10::10/64
```

ULA uses `fc00::/7`, with `fd00::/8` commonly used in practice for locally generated private space.

Example:

```text
fd12:3456:789a:1::1/64
```

Link-local addresses use `fe80::/10` and are essential for local neighbor discovery and routing protocol adjacencies.

Example:

```text
fe80::1
```

**Multicast addresses**

Multicast delivers a packet to multiple interested receivers. IPv6 does not use broadcast; multicast replaces that function.

Important multicast prefixes and examples:

- `ff00::/8` identifies multicast addresses.
- `ff02::1` means all IPv6 nodes on the local link.
- `ff02::2` means all IPv6 routers on the local link.

Multicast is used heavily by IPv6 control functions, including Neighbor Discovery Protocol (NDP), Router Advertisements, and routing protocol communication.

**Anycast addresses**

Anycast is a unicast address assigned to multiple interfaces, usually on different devices. Traffic goes to the nearest or best destination based on routing.

Use cases:

- Redundant services
- Distributed DNS or gateway services
- Load distribution across multiple sites

Anycast is conceptually important, but on CCNA you usually focus on recognizing that the address looks like unicast while routing delivers it to one of several possible nodes.

**Special IPv6 addresses**

Some addresses have special meaning and appear often in labs and troubleshooting.

- `::` is the unspecified address, equivalent to `0.0.0.0` in IPv4.
- `::1` is the loopback address, equivalent to `127.0.0.1`.
- `fe80::/10` is link-local space.
- `ff02::1` and `ff02::2` are common local multicast groups.

The unspecified address is used when a host does not yet know its own IPv6 address, such as during initial configuration. You do not assign `::` to an interface.

**Interface identifiers and EUI-64**

In IPv6, the lower 64 bits often identify the interface. Historically, devices could derive this identifier from the MAC address using Modified EUI-64.

Basic idea:

- Take the 48-bit MAC address.
- Insert `fffe` in the middle.
- Flip the seventh bit of the first byte, called the universal/local bit.

Example:

```text
MAC:  00:1A:2B:3C:4D:5E
EUI-64 result:
021a:2bff:fe3c:4d5e
```

This method is less common in modern security-conscious deployments because many systems use privacy extensions or stable random interface identifiers. Still, you should understand the concept for exam questions and legacy configurations.

**SLAAC, DHCPv6, and manual addressing**

IPv6 hosts can obtain addresses in several ways.

- **SLAAC**: The host learns the prefix from Router Advertisements and builds its own interface identifier.
- **DHCPv6**: A server assigns addressing information, similar in purpose to DHCP in IPv4.
- **Static configuration**: An administrator manually sets the address, prefix, and gateway.

SLAAC is common for simple networks because it reduces administrative overhead. DHCPv6 is useful when you need centralized control, logging, or additional options. Static addressing is common for infrastructure devices, servers, and lab environments.

Example Cisco IOS interface configuration:

```cisco
interface GigabitEthernet0/0
 ipv6 address 2001:db8:acad:10::1/64
 ipv6 address fe80::1 link-local
 no shutdown
```

This configuration assigns a global unicast address and a manually chosen link-local address. The link-local address is especially useful for routing protocols and next-hop references.

**Cisco IOS IPv6 verification commands**

Use verification commands to confirm addressing, neighbor discovery, and routing behavior.

```cisco
show ipv6 interface brief
show ipv6 interface GigabitEthernet0/0
show ipv6 route
show ipv6 neighbors
```

What to look for:

- Interface status and assigned IPv6 addresses
- Link-local address presence on enabled interfaces
- Connected and local routes in the IPv6 routing table
- Neighbor cache entries learned through Neighbor Discovery

If you need to test reachability, use `ping ipv6` or the shorter IPv6-aware ping syntax on Cisco devices:

```cisco
ping ipv6 2001:db8:acad:10::2
ping fe80::2 source fe80::1
```

When pinging a link-local address, you often need to specify the outgoing interface or source address because link-local scope is limited to the local link.

**Practical topology example**

A simple IPv6 LAN might look like this:

- Router interface `G0/0`: `2001:db8:acad:10::1/64`
- Switch management SVI: `2001:db8:acad:10::2/64`
- PC1: `2001:db8:acad:10::10/64`
- PC2: `2001:db8:acad:10::11/64`

The router sends Router Advertisements so hosts can learn the prefix and default gateway. Hosts then form their own addresses using SLAAC or DHCPv6, depending on the network design.

**How to identify IPv6 address types quickly**

Use the prefix and context to classify an address:

- `2001:`, `2002:`, or generally `2000::/3` indicates **global unicast**
- `fc00:` or `fd00:` indicates **unique local**
- `fe80:` indicates **link-local**
- `ff00:` indicates **multicast**
- `::` indicates **unspecified**
- `::1` indicates **loopback**

A fast exam strategy is to inspect the first hextet and the scope implied by the prefix before reading the rest of the address.

**Common exam traps**

- Confusing `::` compression with a valid address pattern; only one zero-run can be compressed.
- Assuming IPv6 uses broadcast; it does not.
- Forgetting that link-local addresses are automatically present on enabled interfaces.
- Treating `/64` as optional on normal LANs; it is the standard design choice.
- Using a link-local address without specifying scope when the destination is not on the same interface.

**Key takeaways**

IPv6 addressing is simpler than it first appears once you focus on structure, prefix, and type. You should be able to:

- Read and compress IPv6 addresses correctly
- Distinguish global unicast, ULA, link-local, multicast, anycast, loopback, and unspecified addresses
- Understand why `/64` is the standard LAN prefix
- Recognize how SLAAC, DHCPv6, and static configuration fit into host addressing
- Verify IPv6 settings with Cisco IOS show commands
