## **Course:** Cisco CCNA Complete Course (Exam 200-301 v1.1)

**IPv4 subnetting fundamentals**

IPv4 subnetting divides a network into smaller logical networks by borrowing bits from the host portion. You use it to reduce broadcast scope, improve address efficiency, and support structured network design. For CCNA, you should be able to move quickly between dotted-decimal masks, prefix lengths, binary boundaries, and usable host ranges.

A subnetting problem usually asks you to determine one or more of these items:

- Network address
- Broadcast address
- Usable host range
- Number of subnets
- Number of hosts per subnet
- Prefix length or subnet mask
- Interesting octet, also called the **subnetting octet**

**Classful vs. classless context**

Classful addressing is the older idea that networks belong to fixed classes:

- Class A: default mask `255.0.0.0` or `/8`
- Class B: default mask `255.255.0.0` or `/16`
- Class C: default mask `255.255.255.0` or `/24`

Modern routing is **classless**, which means the prefix length can be anything valid, such as `/19`, `/27`, or `/30`. You still use the classful default as a reference point when calculating borrowed bits, but you do not assume the mask must match the class boundary.

A quick way to identify the interesting octet is to find the first mask octet that is neither `255` nor `0`.

- `255.255.255.224` → interesting octet is the 4th octet
- `255.255.240.0` → interesting octet is the 3rd octet
- `255.192.0.0` → interesting octet is the 2nd octet

**Binary review for subnetting**

Subnetting becomes easier when you can convert the interesting octet into binary and recognize block sizes.

- `255` = `11111111`
- `254` = `11111110`
- `252` = `11111100`
- `248` = `11111000`
- `240` = `11110000`
- `224` = `11100000`
- `192` = `11000000`
- `128` = `10000000`

The block size is `256 - mask_octet` in the interesting octet.

- `255.255.255.224` → block size `32`
- `255.255.255.240` → block size `16`
- `255.255.240.0` → block size `16` in the 3rd octet
- `255.192.0.0` → block size `64` in the 2nd octet

**Binary conversion practice**

Use binary to verify your answers, especially when the subnet boundary is not obvious.

1. Convert the octet to binary.
2. Count the borrowed bits.
3. Determine the block size.
4. List subnet boundaries.
5. Find the network, broadcast, and usable range.

Example: `255.255.255.224`

```text
224 = 128 + 64 + 32
224 = 11100000
Borrowed bits = 3
Host bits remaining in the octet = 5
Block size = 256 - 224 = 32
```

Example: `255.255.240.0`

```text
240 = 128 + 64 + 32 + 16
240 = 11110000
Borrowed bits in the 3rd octet = 4
Host bits remaining in the 3rd octet = 4
Block size = 256 - 240 = 16
```

**Prefix-to-mask and mask-to-prefix conversion**

You should be able to convert in both directions without hesitation.

| Prefix | Mask | Usable hosts per subnet |
|---|---|---:|
| `/24` | `255.255.255.0` | 254 |
| `/25` | `255.255.255.128` | 126 |
| `/26` | `255.255.255.192` | 62 |
| `/27` | `255.255.255.224` | 30 |
| `/28` | `255.255.255.240` | 14 |
| `/29` | `255.255.255.248` | 6 |
| `/30` | `255.255.255.252` | 2 |

For non-`/24` networks, count all network bits from left to right across octets.

- `255.255.240.0` = `/20`
- `255.255.252.0` = `/22`
- `255.192.0.0` = `/10`
- `255.255.255.248` = `/29`

**Borrowed bits and number of subnets**

Borrowed bits are the bits taken from the host portion to create subnets. The number of subnets is `2^borrowed_bits` in the classic CCNA model.

Examples:

- Starting from `/24` to `/27`
  - Borrowed bits: `3`
  - Subnets: `2^3 = 8`
  - Hosts per subnet: `2^5 - 2 = 30`

- Starting from `/16` to `/20`
  - Borrowed bits: `4`
  - Subnets: `2^4 = 16`
  - Hosts per subnet: `2^12 - 2 = 4094`

- Starting from `/8` to `/12`
  - Borrowed bits: `4`
  - Subnets: `16`
  - Hosts per subnet: `2^20 - 2 = 1,048,574`

> In modern practice, some environments use variable-length subnet masks and may not care about the old classful starting point. For CCNA-style exercises, the borrowed-bit method is still the fastest way to reason about subnetting.

**Subnet boundary table method**

A subnet boundary table helps you see ranges quickly. For `192.168.10.0/27`, the block size is `32`, so the subnets are:

| Subnet # | Network | First Host | Last Host | Broadcast |
|---|---|---|---|---|
| 1 | 192.168.10.0 | 192.168.10.1 | 192.168.10.30 | 192.168.10.31 |
| 2 | 192.168.10.32 | 192.168.10.33 | 192.168.10.62 | 192.168.10.63 |
| 3 | 192.168.10.64 | 192.168.10.65 | 192.168.10.94 | 192.168.10.95 |
| 4 | 192.168.10.96 | 192.168.10.97 | 192.168.10.126 | 192.168.10.127 |

This method is especially useful when the host address is not near a boundary and you need to compare multiple subnets quickly.

**Worked examples with different octets**

Example 1: `172.16.0.0/20`

- Default classful mask for Class B is `/16`
- Borrowed bits: `4`
- Block size in the 3rd octet: `16`
- Subnets begin at:
  - `172.16.0.0`
  - `172.16.16.0`
  - `172.16.32.0`
  - `172.16.48.0`

If the host is `172.16.33.77/20`:

- Network: `172.16.32.0`
- Broadcast: `172.16.47.255`
- Usable range: `172.16.32.1` to `172.16.47.254`

Example 2: `10.0.0.0/12`

- Default classful mask for Class A is `/8`
- Borrowed bits: `4`
- Block size in the 2nd octet: `16`
- Subnets begin at:
  - `10.0.0.0`
  - `10.16.0.0`
  - `10.32.0.0`
  - `10.48.0.0`

If the host is `10.19.7.200/12`:

- Network: `10.16.0.0`
- Broadcast: `10.31.255.255`
- Usable range: `10.16.0.1` to `10.31.255.254`

**Lab topology for practice**

Use a simple three-router, three-LAN topology to practice addressing and verification.

```text
PC-A --- SW-A --- R1 --- R2 --- R3 --- SW-C --- PC-C
              \           /
               \-- SW-B --/
```

Suggested addressing plan:

- LAN A: `192.168.10.0/27`
- LAN B: `192.168.10.32/27`
- LAN C: `192.168.10.64/27`
- R1-R2 link: `10.0.0.0/30`
- R2-R3 link: `10.0.0.4/30`

This design gives you practice with both LAN subnetting and point-to-point links.

**Interface configuration example**

```cisco
! R1
conf t
interface g0/0
 description LAN_A
 ip address 192.168.10.1 255.255.255.224
 no shutdown
exit

interface g0/1

end
```


**Routing and verification**

After addressing, you need routing so the subnets can communicate. Static routes are a good lab choice because they keep the focus on subnetting.

```cisco
! R1
ip route 192.168.10.64 255.255.255.224 10.0.0.2

! R2
ip route 192.168.10.0 255.255.255.224 10.0.0.1
ip route 192.168.10.64 255.255.255.224 10.0.0.6

! R3
ip route 192.168.10.0 255.255.255.224 10.0.0.5
```

Verify with common commands:

```cisco
show ip interface brief
show ip route
show running-config interface g0/0
ping 192.168.10.65
traceroute 192.168.10.65
```

A healthy interface should show `up/up`. If an interface is `administratively down`, you likely forgot `no shutdown`. If the IP address is wrong, the interface may still be up but traffic will fail.

**Common misconfigurations to diagnose**

- Wrong mask on one side of a link
- Overlapping subnets on different interfaces
- Incorrect default gateway on a PC
- Missing static route or dynamic routing advertisement
- Interface left in shutdown state
- Host placed in the wrong subnet boundary

Example of a mask mismatch:

```text
R1 g0/0: 192.168.10.1 255.255.255.224
PC-A:    192.168.10.10 255.255.255.240
```

This can break local communication because the PC and router disagree on the subnet boundary.

**Mixed-difficulty subnetting exercises**

Work these before checking the solutions. Try to solve them with binary, block size, and boundary tables.

1. Convert `255.255.255.240` to a prefix length.
2. Convert `/26` to a dotted-decimal mask.
3. For `192.168.50.77/28`, find:
   - Network address

   - Network address

5. Starting with `192.168.1.0/24`, subnet it into `/27` networks:
   - How many subnets do you get?
   - How many usable hosts per subnet?
   - What is the third subnet?
6. Starting with `10.0.0.0/8`, create `/12` subnets:
   - How many bits are borrowed?
   - How many subnets are created?
   - What is the network range for `10.48.0.0/12`?
7. Given the mask `255.255.252.0`, determine:
   - Prefix length
   - Number of borrowed bits from a `/16`
   - Number of subnets created from a `/16`
8. Given the prefix `/29`, determine:
   - Mask
   - Usable hosts
   - Block size
9. Host `172.20.14.200` belongs to subnet `172.20.12.0/22`:
   - Is the host inside the subnet?

10. Compare these subnets and identify which one contains `192.168.100.130`:

    - `192.168.100.192/26`

1. `255.255.255.240` = `/28`
2. `/26` = `255.255.255.192`
3. `192.168.50.77/28`
   - Network: `192.168.50.64`
   - Broadcast: `192.168.50.79`
   - Usable: `192.168.50.65` to `192.168.50.78`
4. `172.16.200.10/20`
   - Interesting octet: 3rd
   - Block size: `16`
   - Network: `172.16.192.0`
   - Broadcast: `172.16.207.255`
5. `192.168.1.0/24` to `/27`
   - Subnets: `8`
   - Usable hosts per subnet: `30`
   - Third subnet: `192.168.1.64/27`
6. `10.0.0.0/8` to `/12`
   - Borrowed bits: `4`
   - Subnets: `16`
   - `10.48.0.0/12` range:
     - Network: `10.48.0.0`
     - Broadcast: `10.63.255.255`
7. `255.255.252.0`
   - Prefix: `/22`
   - Borrowed bits from `/16`: `6`
   - Subnets from `/16`: `64`
8. `/29`
   - Mask: `255.255.255.248`
   - Usable hosts: `6`
   - Block size: `8`
9. Yes, `172.20.14.200` is inside `172.20.12.0/22`
   - Broadcast: `172.20.15.255`
10. `192.168.100.128/26` contains `192.168.100.130`

**Binary practice problems**

Use these to build speed with octet conversion.

1. Convert `224` to binary.
2. Convert `240` to binary.
3. Convert `192` to binary.
4. Convert `255.255.255.224` to binary in the last octet.
5. Convert `255.255.240.0` to binary in the third octet.
6. Determine the block size for `255.255.255.248`.
7. Determine the block size for `255.255.252.0`.

**Binary answers**

1. `224 = 11100000`
2. `240 = 11110000`
3. `192 = 11000000`
4. `224 = 11100000`
5. `240 = 11110000`
6. Block size = `8`
7. Block size = `4` in the 3rd octet

**Troubleshooting mindset**

When a subnetting lab fails, do not guess. Check the address, mask, and route in that order.

- Confirm the host IP and mask on each device
- Confirm the router interface IPs
- Confirm the default gateway on each PC
- Confirm the routing table contains the remote subnet
- Confirm the subnet boundaries do not overlap

A common symptom is that local pings work but remote pings fail. That usually means the local subnet is correct, but routing is missing or the return path is broken.

**Quick reference workflow**

1. Identify the mask or prefix.
2. Find the interesting octet.
3. Convert the octet to binary if needed.
4. Calculate block size.
5. List subnet boundaries.
6. Find the network containing the host.
7. Derive broadcast and usable range.
8. Verify with CLI commands.

**Mini design scenario**

You are given `172.31.0.0/16` and need at least 20 subnets with at least 500 hosts each.

- `/24` gives 256 subnets but only 254 hosts, so it fails host count.
- `/23` gives 126 hosts, still too small.
- `/22` gives 1022 hosts, which works.
- Borrowed bits from `/16` to `/22`: `6`
- Subnets created: `64`

This kind of reasoning is common on the exam because you must satisfy both subnet count and host count requirements.

**Key takeaways**

- Use the interesting octet to focus your calculations.
- Use block size to find subnet boundaries quickly.
- Convert between prefix and mask in both directions.
- Count borrowed bits to determine the number of subnets.
- Verify your work with CLI commands and real interface configuration.
- Practice both `/24` and non-`/24` networks so you can handle different octets confidently.
