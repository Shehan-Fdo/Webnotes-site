## **Course:** Cisco CCNA Complete Course (Exam 200-301 v1.1)

**Wi-Fi 6 overview**

Wi-Fi 6 is the Wi-Fi Alliance certification for IEEE 802.11ax. It improves efficiency more than raw peak speed, especially in dense environments with many clients competing for airtime. For CCNA-level study, focus on why it matters: better capacity, lower latency under load, and improved battery life for clients.

Key Wi-Fi 6 features include:

- **OFDMA**: Orthogonal Frequency Division Multiple Access lets an access point (AP) divide a channel into smaller resource units so multiple clients can transmit in the same time slot.
- **MU-MIMO**: Multi-User, Multiple-Input, Multiple-Output improves simultaneous transmission to multiple clients.
- **BSS coloring**: Basic Service Set coloring helps clients distinguish overlapping networks and reduce unnecessary deferral.
- **Target Wake Time**: TWT schedules client wake/sleep cycles to save battery on supported devices.
- **1024-QAM**: Higher-order modulation can increase throughput at short range with good signal quality.

Wi-Fi 6 is not just about faster downloads. In a classroom, office, or stadium, the main gain is efficiency when many devices are active at once.

**Wi-Fi 6, Wi-Fi 6E, and channel implications**

Wi-Fi 6 uses the 2.4 GHz and 5 GHz bands. Wi-Fi 6E extends 802.11ax into the 6 GHz band, which is important because 6 GHz offers more clean spectrum and fewer legacy devices. That reduces contention and interference, but it also changes client compatibility and regulatory behavior.

Important exam and design points:

- **6 GHz requires compatible clients**: Only Wi-Fi 6E or Wi-Fi 7-capable clients can use 6 GHz.
- **No legacy 2.4/5 GHz fallback on 6 GHz radios**: A 6 GHz SSID is not for older Wi-Fi clients.
- **More contiguous spectrum**: 6 GHz supports wider channels and more non-overlapping capacity.
- **Regulatory domain matters**: Available channels and power limits vary by country and region.
- **AFC in some environments**: Automated Frequency Coordination may be required for certain standard-power 6 GHz deployments, depending on region and use case.

For CCNA scenarios, if a user has an older laptop or phone, it may connect to 2.4 GHz or 5 GHz but not 6 GHz. That is a compatibility issue, not necessarily a configuration fault.

**Wireless security model**

Wireless security protects confidentiality, integrity, and access control over an open radio medium. The main standards you should know are WPA2 and WPA3, with enterprise variants that use 802.1X authentication.

At a high level:

- **WPA2-Personal** uses a pre-shared key (PSK) or passphrase.
- **WPA2-Enterprise** uses 802.1X with a RADIUS server and user/device credentials.
- **WPA3-Personal** uses SAE, the Simultaneous Authentication of Equals handshake.
- **WPA3-Enterprise** uses 802.1X and can support stronger cryptographic requirements than WPA2-Enterprise.

The security choice depends on scale and control:

- Use **Personal** for small environments where one shared password is acceptable.
- Use **Enterprise** when you need per-user authentication, revocation, auditing, or dynamic policy.

**WPA2 details**

WPA2 is still common and is based on IEEE 802.11i security. Its typical data protection mechanism is AES-CCMP, where CCMP stands for Advanced Encryption Standard in Counter Mode with Cipher Block Chaining Message Authentication Code Protocol.

Key WPA2 points:

- **AES-CCMP** is the standard encryption and integrity mechanism for WPA2.
- **PSK** mode uses one shared passphrase for all clients.
- **Enterprise** mode uses 802.1X and RADIUS for authentication.
- **PMF** is optional in WPA2, though it can be enabled in some deployments.

WPA2-Personal is simple to deploy but has weak operational control because every user shares the same secret. If one person leaves, you must change the passphrase for everyone.

**WPA3 details**

WPA3 improves security and addresses weaknesses in WPA2-Personal. The most visible change is the move from PSK to SAE in Personal mode.

WPA3-Personal:

- Uses **SAE** instead of PSK.
- Resists offline dictionary attacks better than WPA2-Personal.
- Improves protection when users choose weak passwords, though strong passphrases are still important.

WPA3-Enterprise:

- Uses **802.1X/EAP** with a RADIUS server, like WPA2-Enterprise.
- Can support stronger security profiles and better cryptographic requirements.
- Is the better choice for managed networks that need identity-based access.

Protected Management Frames, or **PMF** (802.11w), protect management traffic such as deauthentication and disassociation frames. WPA3 requires PMF in WPA3-Personal and WPA3-Enterprise. In mixed or transition deployments, PMF behavior depends on the exact mode and client support, so do not assume every WPA3-related SSID behaves identically.

> WPA3 transition mode is a compatibility feature. It can allow WPA2 and WPA3 clients on the same SSID, but the security posture is weaker than a pure WPA3-only design because legacy compatibility remains part of the setup.

**WPA2 vs WPA3 comparison**

| Feature | WPA2-Personal | WPA2-Enterprise | WPA3-Personal | WPA3-Enterprise |
|---|---|---|---|---|
| Authentication | Shared passphrase | 802.1X/EAP | SAE | 802.1X/EAP |
| Backend server | No | Yes, RADIUS | No | Yes, RADIUS |
| PMF | Optional | Optional | Required | Required |
| Main benefit | Simple deployment | Per-user control | Better password attack resistance | Stronger enterprise security |
| Typical use | Home/small office | Campus/enterprise | Newer home/small office | Managed enterprise |

This table is useful for exam questions that ask you to match a security mode to a deployment scenario.

**802.1X and EAP in enterprise WLANs**

Enterprise wireless security uses 802.1X port-based access control with Extensible Authentication Protocol, or **EAP**. The AP acts as the authenticator, the client is the supplicant, and the RADIUS server performs authentication decisions.

Typical flow:

1. The client associates to the SSID.
2. The AP blocks normal data traffic until authentication completes.
3. The client and RADIUS server exchange EAP messages through the AP.
4. The RADIUS server validates credentials and returns an accept or reject.
5. If accepted, the AP and client derive session keys and allow access.

Common EAP methods:

- **PEAP**: Protected EAP, often used with username/password inside a TLS tunnel.
- **EAP-TLS**: Uses client certificates and server certificates for mutual authentication.
- **EAP-TTLS**: Similar tunnel-based approach, less common in many Cisco exam scenarios.

Practical distinction:

- **PEAP** is easier to deploy because users authenticate with credentials, but password hygiene still matters.
- **EAP-TLS** is stronger because it uses certificates, but it requires certificate lifecycle management.

A certificate-based flow usually looks like this:

- The client trusts the RADIUS server certificate.
- The client presents its own certificate if EAP-TLS is used.
- The server validates the certificate chain and identity.
- The session is authorized based on policy.

For CCNA, remember that enterprise WLANs often integrate with identity services, VLAN assignment, and role-based access after authentication.

**Transition mode and mixed compatibility**

Transition mode is a compatibility strategy that lets newer and older clients coexist during migration. You may see this when moving from WPA2 to WPA3.

Use transition mode when:

- You have older clients that cannot support WPA3.
- You need a staged migration instead of a cutover.
- You want to reduce help-desk impact during rollout.

Tradeoffs:

- **Compatibility improves**, but security is not as strong as a pure WPA3-only SSID.
- Some clients may prefer WPA2 even when WPA3 is available, depending on device behavior and policy.
- PMF and cipher requirements can vary by platform and configuration.

For exam questions, the key idea is that transition mode is a bridge, not the final target.

**Practical WLAN design examples**

A small office might use WPA2-Personal or WPA3-Personal if it has no identity infrastructure. A campus or enterprise should usually use WPA2-Enterprise or WPA3-Enterprise with RADIUS and 802.1X.

Example design choices:

- **Home office**: WPA3-Personal if all clients support it; otherwise WPA2/WPA3 transition mode.
- **Branch office**: WPA2-Enterprise or WPA3-Enterprise with centralized policy.
- **Guest network**: Separate SSID, captive portal if required, and isolated VLAN.
- **IoT network**: Often needs special handling because many devices support only WPA2 or older features.

A simple logical topology:

```text
Client -> AP -> Switch -> WLC/Controller -> RADIUS Server -> Directory/Identity Store
```

In a controller-based design, the AP forwards authentication traffic to the controller or directly to the RADIUS server depending on architecture. The important concept is that the AP enforces access while the RADIUS server makes the authentication decision.

**Configuration concepts and examples**

Cisco wireless syntax varies by platform, controller type, and software release. Treat the following as conceptual examples, not universal commands.

Example enterprise WLAN intent:

```text
SSID: CorpWiFi
Security: WPA2-Enterprise or WPA3-Enterprise
Authentication: 802.1X
RADIUS server: 10.10.10.20
VLAN: 20
```

Example RADIUS server definition on a Cisco IOS XE-style platform:

```bash
radius server RAD1
 address ipv4 10.10.10.20 auth-port 1812 acct-port 1813
 key MySharedSecret
```

Example AAA configuration:

```bash
aaa new-model
aaa authentication dot1x default group radius
aaa authorization network default group radius
dot1x system-auth-control
```

Example wireless policy intent on a controller or AP platform:

```text
- Create SSID CorpWiFi
- Set security to WPA2-Enterprise or WPA3-Enterprise
- Enable 802.1X authentication
- Bind the SSID to VLAN 20
- Apply the RADIUS server group
```

If you are studying for CCNA, focus on the relationship between components rather than memorizing one platform’s exact WLAN CLI.

**Verification and troubleshooting**

Wireless verification depends on the platform, but you should know the kinds of outputs and checks that matter.

Useful checks include:

- Confirm the SSID is broadcasting.
- Verify the client sees the correct security type.
- Check that the client negotiates the expected band and standard.
- Confirm RADIUS authentication succeeds.
- Validate VLAN assignment and IP addressing after authentication.

Common troubleshooting commands on Cisco IOS/IOS XE-style devices include:

```bash
show wlan summary
show wireless client summary
show wireless client mac <client-mac>
show radius statistics
show aaa servers
show dot1x all
show logging | include DOT1X|RADIUS|WPA|PMF
```

On a switch carrying wired 802.1X traffic to a wireless AP or controller, `show authentication sessions` can help verify the wired port state, but it is not the primary wireless association command for AP/client troubleshooting.

Example troubleshooting output pattern:

```text
Client MAC: 00:11:22:33:44:55
SSID: CorpWiFi
Security: WPA2-Enterprise
State: RUN
Auth Method: 802.1X
RADIUS: Access-Accept
VLAN: 20
IP Address: 10.20.20.45
```

If a client fails to connect, isolate the problem in this order:

1. Check RF and SSID visibility.
2. Confirm the client supports the band and security mode.
3. Verify credentials, certificates, or passphrase.
4. Check RADIUS reachability and shared secret.
5. Review PMF, cipher, and transition-mode compatibility.

**Common exam traps**

- **WPA2 and WPA3 are not the same**: WPA3-Personal uses SAE, not PSK.
- **PMF is not universally mandatory in every mixed deployment**: check the mode and compatibility requirements.
- **Wi-Fi 6 is not the same as Wi-Fi 6E**: 6E adds 6 GHz support.
- **6 GHz is not backward compatible with legacy clients**: older devices need 2.4 GHz or 5 GHz.
- **Enterprise security is not just encryption**: it includes authentication, authorization, and accounting, often abbreviated as **AAA**.
- **RADIUS is for authentication and policy**: it is not the same as the wireless encryption protocol.

**Quick memory aid**

- **Wi-Fi 6**: efficiency and density.
- **Wi-Fi 6E**: Wi-Fi 6 plus 6 GHz.
- **WPA2-Personal**: shared password.
- **WPA2-Enterprise**: 802.1X with RADIUS.
- **WPA3-Personal**: SAE.
- **WPA3-Enterprise**: 802.1X with stronger security expectations.
- **PMF**: protects management frames.

**Study takeaway**

For CCNA, connect the wireless standard to the deployment need. Wi-Fi 6 improves performance in crowded environments, Wi-Fi 6E expands usable spectrum, and WPA3 strengthens authentication and management-frame protection. When you see a scenario, ask whether the environment needs compatibility, centralized identity, or stronger security, then choose the matching WLAN design.
