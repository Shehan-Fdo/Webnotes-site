## **Course:** Cisco CCNA Complete Course (Exam 200-301 v1.1)

**ARP and MAC address basics**

A <abbr title="Address Resolution Protocol">ARP</abbr> table and a switch MAC address table solve different problems. ARP maps an IP address to a MAC address so a device can send frames to the correct next-hop host on the local network. A switch learns which MAC address is reachable on which port so it can make forwarding and filtering decisions at Layer 2.

- A MAC address is a Layer 2 identifier used on a local network segment.
- A switch learns MAC addresses by examining the **source MAC address** of received frames.
- The learned association is stored in a MAC address table, also called a CAM table.
- The table records the MAC address, the interface, and the VLAN.
- Switches use the table to decide whether to forward or filter a frame.
- ARP keeps IP-to-MAC mappings for local delivery.

**How a switch learns MAC addresses**

When a frame enters a switch port, the switch checks the source MAC address and associates it with the ingress port. If the entry does not exist, the switch creates it. If the entry already exists, the switch refreshes the aging timer for that entry. This learning process happens dynamically as traffic flows through the switch.

> A switch learns from the source MAC address, not the destination MAC address.

- The first frame from a device can create a new table entry.
- Later frames from the same device refresh the existing entry.
- The table is stored in volatile memory, so entries are lost after a reboot.
- The switch uses the table to reduce unnecessary flooding and improve forwarding accuracy.

**ARP table behavior on a router**

A router also needs MAC addresses before it can exchange frames with hosts on a connected subnet. The router stores IP-to-MAC mappings in its ARP table. In the example evidence, the ARP table includes entries for the router’s own interfaces and for connected hosts. The age column shows how long an entry has existed, and a dash indicates an entry that does not time out.

- The ARP table maps an IP address to a hardware address.
- The table can include the router’s own interface addresses.
- The interface column shows which router interface uses the mapping.
- The age value helps you see whether an entry is dynamic or non-expiring.

**Switch MAC table vs. router ARP table**

| Feature | Switch MAC table | Router ARP table |
|---|---|---|
| Purpose | Tracks where MAC addresses are learned | Tracks IP-to-MAC mappings |
| Layer | Layer 2 | Layer 3 to Layer 2 resolution |
| Learned from | Source MAC address in frames | ARP resolution for local delivery |
| Used for | Forwarding and filtering decisions | Sending frames to the correct host |

**Practical exam focus**

For CCNA, focus on the relationship between Layer 2 switching and ARP. A switch does not need IP addresses to learn where devices are connected. It learns from Ethernet frames. A router, by contrast, needs ARP to resolve a local IP address to a MAC address before it can send a frame on the subnet.

1. Identify whether the device is a switch or a router.
2. Determine whether the question is about MAC learning or IP-to-MAC resolution.
3. Use the correct table: MAC address table for switches, ARP table for routers.
4. Remember that switches learn from source MAC addresses only.

**Key takeaways**

- MAC learning is a switch function based on source MAC addresses.
- ARP resolves IP addresses to MAC addresses for local communication.
- Switch MAC tables are volatile and support forwarding decisions.
- Router ARP tables store IP-to-MAC mappings for connected networks.
