## **Course:** Cisco CCNA Complete Course (Exam 200-301 v1.1)

**Ethernet overview**

Ethernet is the foundation of modern local area networks (LANs). It operates at the physical and data link layers and uses frames to move encapsulated data across a network.

Ethernet framing follows the IEEE 802.3 standard. A frame carries addressing, protocol identification, payload data, and error-detection information so devices can communicate consistently on shared or switched media.

- **MAC address** means Media Access Control address.
- Ethernet uses MAC addresses to uniquely identify hosts on a LAN.
- A MAC address is 48 bits, or 6 bytes, long.
- The first 3 bytes are the Organizationally Unique Identifier (OUI).
- The last 3 bytes are assigned by the manufacturer to keep the address unique.
- MAC addresses are often called hardware addresses because they are tied to a network interface card (NIC).

**Ethernet frame structure**

An Ethernet frame begins with synchronization fields, includes source and destination addressing, and ends with error detection. The destination address helps the receiving host decide whether the frame is meant for it.

<dl>
<dt>Preamble</dt>
<dd>Alerts the receiver that a frame is starting and helps with synchronization.</dd>
<dt>SFD</dt>
<dd>The Start Frame Delimiter marks the start of the destination MAC address field.</dd>
<dt>Destination MAC</dt>
<dd>Identifies the receiving system.</dd>
<dt>Source MAC</dt>
<dd>Identifies the sending system.</dd>
<dt>Type/Length</dt>
<dd>Identifies the encapsulated higher-layer protocol, such as IPv4 or IPv6.</dd>
<dt>Data and Pad</dt>
<dd>Contains the payload. Padding is added when needed to meet the minimum field length of 46 bytes.</dd>
<dt>FCS</dt>
<dd>The Frame Check Sequence uses a 32-bit CRC (Cyclic Redundancy Check) to detect corrupted data.</dd>
</dl>

> The FCS is the field used to detect errors in a received frame.

**Field purpose and exam focus**

When you study Ethernet frames, focus on what each field does rather than memorizing the diagram alone. Several questions on this topic ask you to match a field with its function.

- The **destination address** tells the host whether the frame is addressed to it.
- The **preamble** prepares the destination for a new frame.
- The **Type/Length** field identifies the encapsulated protocol.
- The **FCS** helps detect frame errors after reception.

**Practical study points**

Ethernet framing is not just a naming exercise. It explains how devices recognize traffic, identify the sender and receiver, and verify whether the frame arrived intact.

- Use the destination MAC to identify the intended receiver.
- Use the source MAC to identify the sender.
- Use the Type/Length field to infer the upper-layer protocol.
- Use the FCS to check for corruption.
- Remember that padding exists to satisfy the minimum data size requirement.

**Quick review**

1. Ethernet frames carry encapsulated data across LANs.
2. MAC addresses uniquely identify devices on Ethernet networks.
3. The frame structure includes synchronization, addressing, payload, and error detection.
4. The FCS provides corruption detection using CRC.
