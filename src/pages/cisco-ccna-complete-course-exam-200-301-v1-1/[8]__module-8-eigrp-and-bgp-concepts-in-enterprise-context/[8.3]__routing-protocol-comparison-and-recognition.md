## **Course:** Cisco CCNA Complete Course (Exam 200-301 v1.1)

**Routing protocol recognition at a glance**

CCNA questions often ask you to identify a routing protocol from a route table, a neighbor table, or a short configuration snippet. You need to recognize the protocol by its route code, update behavior, metric style, and common output clues.

A practical way to study this topic is to compare the protocols by:
- **How they share routes**
- **How they choose the best path**
- **What their output looks like**
- **How quickly they converge**
- **Where they are typically used**

**Core comparison of routing protocols**

| Protocol | Type | Update behavior | Metric | Route code in `show ip route` | Typical CCNA clue |
|---|---|---|---|---|---|
| RIP | Distance vector | Periodic full-table updates every 30 seconds | Hop count | `R` | Simple, old, slow convergence |
| OSPF | Link-state | Event-driven LSAs, then SPF recalculation | Cost | `O` | Neighbor states, areas, DR/BDR |
| EIGRP | Advanced distance vector | Partial, bounded, triggered updates | Composite metric | `D` | `D` and `D EX`, feasible distance |
| BGP | Path vector | Incremental path advertisements | Path attributes, especially AS-path | `B` | Internet routing, AS numbers, path-vector behavior |

> <mark>RIP v1 is largely obsolete</mark>, and RIP is rarely used in modern enterprise networks. It still matters for exam recognition and legacy troubleshooting.

**How to recognize the protocol from output**

When you see a route table, start with the route code:
- `R` usually means RIP
- `O` usually means OSPF
- `D` usually means EIGRP internal routes
- `D EX` usually means EIGRP external routes
- `B` usually means BGP

Then check the supporting clues:
- RIP often shows hop-count-based paths and simple next-hop selection.
- OSPF often shows area information, neighbor states, and equal-cost paths.
- EIGRP often shows feasible distance, reported distance, and successor/feasible successor logic.
- BGP often shows AS-path information and is used between autonomous systems.

**RIP: simple but outdated**

RIP (Routing Information Protocol) is a distance-vector protocol that uses hop count as its metric. It is easy to understand, but it scales poorly because the maximum usable hop count is 15. A hop count of 16 means unreachable.

RIP sends periodic updates, which means routers advertise their routing table on a timer even if nothing changes. That makes RIP easy to implement but inefficient compared to newer protocols.

Key recognition points:
- **Metric:** hop count
- **Maximum hops:** 15
- **Update style:** periodic
- **Route code:** `R`
- **Modern use:** rare, mostly legacy or lab environments

Example route table output:
```bash
Router# show ip route
R    192.168.10.0/24 [120/1] via 10.1.1.2, 00:00:12, GigabitEthernet0/0
```

Interpretation:
- `R` identifies RIP
- `120` is the administrative distance
- `1` is the hop count metric
- The route is learned from a RIP neighbor

RIP is usually not the best answer unless the question clearly points to a simple, legacy, periodic-update protocol.

**OSPF: link-state and fast convergence**

OSPF (Open Shortest Path First) is a link-state routing protocol. Routers build a map of the network by exchanging link-state advertisements (LSAs), then run the Shortest Path First (SPF) algorithm to calculate best paths.

OSPF converges faster than RIP because it does not rely on periodic full-table updates. Instead, it sends updates when the topology changes. That makes it more efficient and more scalable.

Key recognition points:
- **Metric:** cost
- **Update style:** LSAs, event-driven
- **Route code:** `O`
- **Neighbor discovery:** hello packets
- **Topology behavior:** area-based design

OSPF neighbor states are important for exam recognition:
- **Down:** no hello received yet
- **Init:** hello received, but not bidirectional
- **2-Way:** bidirectional communication established
- **ExStart:** master/slave negotiation begins
- **Exchange:** database description packets are exchanged
- **Loading:** routers request missing LSAs
- **Full:** adjacency is fully formed and databases are synchronized

On broadcast networks, OSPF also elects a Designated Router (DR) and Backup Designated Router (BDR) to reduce adjacency overhead:
- **DR:** central exchange point for LSAs on the segment
- **BDR:** standby router that takes over if the DR fails
- **DROTHER:** routers that are neither DR nor BDR

A common exam clue is that routers on a multiaccess network may reach **2-Way** with each other, but only form **Full** adjacency with the DR and BDR.

Example output:
```bash
Router# show ip ospf neighbor

Neighbor ID     Pri   State           Dead Time   Address         Interface
10.1.1.2          1   FULL/DR         00:00:33    10.0.0.2        GigabitEthernet0/0
10.1.1.3          1   FULL/BDR        00:00:31    10.0.0.3        GigabitEthernet0/0
10.1.1.4          1   2WAY/DROTHER   00:00:35    10.0.0.4        GigabitEthernet0/0
```

Interpretation:
- `FULL/DR` means this neighbor is fully adjacent and is the DR
- `FULL/BDR` means this neighbor is fully adjacent and is the BDR
- `2WAY/DROTHER` means the neighbor is seen bidirectionally, but not fully adjacent on that segment

OSPF route table example:
```bash
Router# show ip route ospf
O    10.20.30.0/24 [110/20] via 10.1.1.2, 00:02:11, GigabitEthernet0/0
```

Interpretation:
- `O` identifies OSPF
- `110` is the administrative distance
- `20` is the OSPF cost

**EIGRP: fast, efficient, and metric-rich**

EIGRP (Enhanced Interior Gateway Routing Protocol) is often described as an advanced distance-vector protocol. It uses the Diffusing Update Algorithm (DUAL) to calculate loop-free paths and converge quickly.

EIGRP sends partial, bounded, triggered updates:
- **Partial:** only changed routes are sent
- **Bounded:** updates go only to affected neighbors
- **Triggered:** updates are sent when a change occurs

This makes EIGRP more efficient than RIP and often faster to converge than older distance-vector protocols.

Key recognition points:
- **Metric:** composite metric, usually based on bandwidth and delay
- **Route code:** `D`
- **External route code:** `D EX`
- **Algorithm:** DUAL
- **Important terms:** successor, feasible successor, feasible distance, reported distance

EIGRP route table clues:
- `D` means an internal EIGRP route
- `D EX` means an external EIGRP route redistributed into EIGRP
- The bracketed values often show administrative distance and metric

Example route table output:
```bash
Router# show ip route eigrp
D     172.16.10.0/24 [90/30720] via 10.1.1.2, 00:00:18, GigabitEthernet0/0
D EX  172.16.20.0/24 [170/28160] via 10.1.1.3, 00:00:22, GigabitEthernet0/1
```

Interpretation:
- `D` is an internal EIGRP route
- `D EX` is an external EIGRP route
- `90` is the administrative distance for internal EIGRP
- `170` is the administrative distance for external EIGRP

Feasible distance and reported distance matter when interpreting EIGRP output:
- **Feasible distance (FD):** the best total metric from the local router to the destination
- **Reported distance (RD):** the metric a neighbor advertises to reach the destination
- A **feasible successor** is a backup route that satisfies the feasibility condition and can be used quickly if the successor fails

Example EIGRP topology output:
```bash
Router# show ip eigrp topology 172.16.10.0/24

P 172.16.10.0/24, 1 successors, FD is 30720
        via 10.1.1.2 (30720/28160), GigabitEthernet0/0
        via 10.1.1.3 (40960/30720), GigabitEthernet0/1
```

Interpretation:
- `P` means passive, which is normal and stable
- `1 successors` means one best path is installed
- `FD is 30720` is the feasible distance
- `30720/28160` shows local metric and reported distance for the first path
- The second path may be a feasible successor if it meets the feasibility condition

EIGRP is a strong exam candidate when you see:
- `D` or `D EX`
- DUAL terminology
- Feasible distance and reported distance
- Partial updates and fast convergence

**BGP: path-vector and interdomain routing**

BGP (Border Gateway Protocol) is a path-vector protocol used to exchange routes between autonomous systems (ASes). It is the routing protocol of the Internet.

For CCNA, BGP is usually recognized at a high level rather than configured in depth. You should know its role and output clues, even if the exam does not expect advanced policy design.

Key recognition points:
- **Type:** path-vector
- **Route code:** `B`
- **Main path attribute:** AS-path
- **Use case:** interdomain routing between ASes
- **Convergence:** generally slower and policy-driven compared to IGPs

BGP does not choose paths the same way as RIP, OSPF, or EIGRP. It uses attributes such as:
- **AS-path**
- **Next hop**
- **Local preference**
- **MED (Multi-Exit Discriminator)**

For CCNA, the most important concept is that BGP is policy-based and path-vector oriented, not a typical interior gateway protocol.

Example route table output:
```bash
Router# show ip route bgp
B    203.0.113.0/24 [20/0] via 192.0.2.2, 00:10:44
```

Example neighbor output:
```bash
Router# show ip bgp summary

BGP router identifier 192.0.2.1, local AS number 65001
Neighbor        V    AS MsgRcvd MsgSent   TblVer  InQ OutQ Up/Down  State/PfxRcd
192.0.2.2       4 65002     120     118        7    0    0 00:12:31        5
```

Interpretation:
- `local AS number` identifies the local autonomous system
- The neighbor AS number shows the remote AS
- `State/PfxRcd` indicates session status or number of prefixes received
- `B` in the route table identifies BGP-learned routes

BGP is typically not configured in depth in CCNA, but you should recognize it when a question mentions:
- Autonomous systems
- Internet edge routing
- AS-path
- `show ip bgp summary`

**Administrative distance comparison**

Administrative distance (AD) is the trust value a router uses when multiple routing sources advertise the same destination. Lower is better.

| Source | Administrative distance | Notes |
|---|---:|---|
| Connected | 0 | Directly attached network |
| Static | 1 | Manually configured route |
| EIGRP internal | 90 | Trusted more than OSPF |
| OSPF | 110 | Common enterprise IGP |
| RIP | 120 | Least preferred among common IGPs |
| EIGRP external | 170 | Redistributed into EIGRP |
| BGP eBGP | 20 | Learned from external AS |
| BGP iBGP | 200 | Learned from internal BGP peer |

Administrative distance helps you answer questions like:
- Which route is installed when OSPF and RIP both advertise the same prefix?
- Why does a router prefer a static route over a dynamic route?
- Why does EIGRP internal beat OSPF, but EIGRP external does not?

> If two protocols advertise the same destination, the router first compares **administrative distance**, then the protocol metric.

**Update behavior and convergence comparison**

| Protocol | Update style | Convergence | Why it matters |
|---|---|---|---|
| RIP | Periodic full-table updates | Slow | More traffic, slower reaction to change |
| OSPF | LSA-based, event-driven | Fast | Efficient recalculation after topology change |
| EIGRP | Partial, bounded, triggered | Very fast | Sends only what changed and can use backups |
| BGP | Path-vector advertisements | Usually slower | Policy-driven, not optimized for rapid IGP convergence |

This comparison is useful in scenario questions:
- If a network needs simple legacy routing, RIP may appear, but it is rarely the best modern choice.
- If a network needs scalable internal routing with areas, OSPF is a common answer.
- If a network needs fast convergence with efficient updates, EIGRP is a strong candidate where supported.
- If a network connects different organizations or autonomous systems, BGP is the protocol to recognize.

**Decision tree for protocol recognition**

1. Look at the route code in `show ip route`.
   - `R` points to RIP
   - `O` points to OSPF
   - `D` or `D EX` points to EIGRP
   - `B` points to BGP

2. Check the metric style.
   - Hop count suggests RIP
   - Cost suggests OSPF
   - Composite metric suggests EIGRP
   - AS-path and policy clues suggest BGP

3. Check the update behavior clue.
   - Periodic updates suggest RIP
   - LSAs and neighbor states suggest OSPF
   - Partial and triggered updates suggest EIGRP
   - Path-vector and AS numbers suggest BGP

4. Check the administrative distance.
   - `120` often confirms RIP
   - `110` often confirms OSPF
   - `90` or `170` often confirms EIGRP
   - `20` or `200` often confirms BGP

**Practical recognition scenarios**

A router shows this route:
```bash
O    10.10.0.0/16 [110/20] via 192.168.1.2, 00:01:12, GigabitEthernet0/0
```
You should think:
- OSPF
- AD 110
- Cost 20
- Internal enterprise routing

A router shows this route:
```bash
D EX  198.51.100.0/24 [170/30720] via 10.1.1.3, 00:00:44, GigabitEthernet0/1
```
You should think:
- EIGRP external route
- Redistributed from another source
- AD 170

A router shows this route:
```bash
R    172.16.5.0/24 [120/2] via 10.1.1.4, 00:00:20, GigabitEthernet0/2
```
You should think:
- RIP
- Two hops away
- Legacy distance-vector behavior

A router shows this route:
```bash
B    203.0.113.0/24 [20/0] via 192.0.2.2, 00:10:44
```
You should think:
- BGP
- Inter-AS route
- Path-vector protocol

```bash
 no auto-summary

```

```bash
router ospf 1
 router-id 1.1.1.1
 network 10.0.0.0 0.255.255.255 area 0
 network 192.168.1.0 0.0.0.255 area 0
```

EIGRP example:
```bash
router eigrp 100
 network 10.0.0.0 0.255.255.255

```

```bash
router bgp 65001
 neighbor 192.0.2.2 remote-as 65002

```

These snippets are not full design templates, but they help you connect the protocol name to the kind of configuration you may see in a question.

**Troubleshooting mindset**

When a route is missing, ask:
- Is the protocol enabled on the correct interfaces?
- Are neighbors formed?
- Is the route being filtered or redistributed?
- Is the administrative distance causing a different route to win?
- Is the metric valid for the protocol?

For OSPF, check neighbor state first if routes are missing.
For EIGRP, check neighbor formation and topology table entries.
For RIP, check version, network statements, and whether the route is too far away.
For BGP, check AS numbers, neighbor reachability, and session state.

**Exam tips**

- Memorize the route codes: `R`, `O`, `D`, `D EX`, and `B`.
- Memorize the common administrative distances.
- Know that RIP uses hop count and is slow.
- Know that OSPF uses cost, LSAs, and neighbor states.
- Know that EIGRP uses DUAL, feasible distance, and reported distance.
- Know that BGP uses AS-path and is used between autonomous systems.
- Use the route table first, then confirm with neighbor or protocol output.

**Quick memory aid**

- **RIP** = periodic, hop count, legacy
- **OSPF** = link-state, cost, areas, DR/BDR
- **EIGRP** = partial updates, DUAL, FD/RD
- **BGP** = AS-path, path-vector, interdomain
