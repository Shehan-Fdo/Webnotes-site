## **Course:** Cisco CCNA Complete Course (Exam 200-301 v1.1)

**VLAN fundamentals**

A <abbr title="Virtual Local Area Network">VLAN</abbr> is a logical segmentation of a Layer 2 network. It lets you group devices by function, department, or policy instead of by physical switch location. This is useful because a single physical switch can carry traffic for multiple isolated logical networks.

VLANs matter because they control where <abbr title="broadcast">broadcast</abbr> and unknown unicast traffic can travel. Without VLANs, every device on the same switch or bridged network shares one broadcast domain. With VLANs, each VLAN becomes its own broadcast domain, which reduces unnecessary traffic and improves design flexibility.

- **Layer 2 separation** keeps devices in different VLANs from directly exchanging frames without routing.
- **Broadcast containment** limits ARP and other broadcast traffic to the local VLAN.
- **Policy grouping** lets you place users and services together logically, even across multiple switches.
- **Scalability** improves because you can expand a network without redesigning physical cabling.

**Why VLANs are used**

VLANs solve common campus network problems. In a flat network, every broadcast reaches every host, which wastes bandwidth and can create security and management issues. VLANs let you divide a large Layer 2 domain into smaller, more manageable segments.

Typical reasons to deploy VLANs include:

- Separating departments such as HR, Finance, and Engineering.
- Isolating voice, data, guest, and management traffic.
- Reducing the size of failure and broadcast domains.
- Applying different security or quality-of-service policies per group.
- Supporting logical moves, adds, and changes without rewiring.

A VLAN does not encrypt traffic or provide full security by itself. It is a segmentation tool, not a complete access-control solution. You still need ACLs, authentication, and other controls where appropriate.

**Broadcast domains and segmentation**

A <abbr title="broadcast domain">broadcast domain</abbr> is the set of devices that receive a Layer 2 broadcast frame. In a traditional switch network, all ports in the same VLAN belong to the same broadcast domain. When a host sends an ARP request, every device in that VLAN receives it.

VLANs segment broadcast domains by assigning switch ports or trunk-tagged traffic to specific VLAN IDs. Devices in VLAN 10 do not receive broadcasts from VLAN 20 unless a Layer 3 device routes between them.

- **One VLAN = one broadcast domain** in normal switched designs.
- **Broadcasts stay local** to the VLAN unless routed.
- **Unknown unicast flooding** is also limited to the VLAN.
- **Multicast behavior** depends on the protocol and switch features, but VLAN boundaries still define the Layer 2 scope.

This segmentation is especially important in larger networks where broadcast traffic can become noisy. Smaller broadcast domains generally mean less unnecessary traffic and easier troubleshooting.

**VLAN membership and port types**

Switch ports can participate in VLANs in different ways. The most common access port belongs to one VLAN and sends untagged frames to end devices. Trunk ports carry traffic for multiple VLANs between switches, routers, or Layer 3 switches using VLAN tags.

- **Access port**: associates a port with a single VLAN for an end device.
- **Trunk port**: carries multiple VLANs across one link.
- **Native VLAN**: on 802.1Q trunks, untagged frames are associated with the native VLAN.
- **802.1Q tag**: identifies the VLAN for a frame on a trunk.

A simple example topology:

```text
PC1 --- access VLAN 10 --- SW1 === trunk === SW2 --- access VLAN 10 --- PC2
PC3 --- access VLAN 20 --- SW1 === trunk === SW2 --- access VLAN 20 --- PC4
```

In this design, PC1 and PC2 can communicate at Layer 2 if they are in the same VLAN and the same subnet. PC1 and PC3 cannot communicate directly at Layer 2 because they are in different VLANs.

**Practical Cisco IOS configuration**

You create VLANs on Cisco switches, then assign ports to them. The exact commands vary by platform, but the common CCNA workflow is consistent.

```cisco
conf t
vlan 10
 name SALES
vlan 20
 name ENGINEERING
exit

interface range gigabitEthernet0/1 - 12
 switchport mode access
 switchport access vlan 10

interface range gigabitEthernet0/13 - 24
 switchport mode access
 switchport access vlan 20
end
```

For a trunk between switches:

```cisco
conf t
interface gigabitEthernet0/48
 switchport mode trunk
 switchport trunk allowed vlan 10,20
 switchport trunk native vlan 99
end
```

Useful verification commands:

```cisco
show vlan brief
show interfaces trunk
show interfaces gigabitEthernet0/48 switchport
show mac address-table vlan 10
```

These commands help you confirm VLAN creation, port membership, trunk status, and learned MAC addresses per VLAN.

**Inter-VLAN communication**

Devices in different VLANs cannot communicate at Layer 2. If communication is required, you need Layer 3 routing. This is called <abbr title="inter-VLAN routing">inter-VLAN routing</abbr>.

Common methods include:

- **Router-on-a-stick**: one router interface uses subinterfaces with 802.1Q tags.
- **Layer 3 switch SVIs**: switched virtual interfaces route between VLANs on the switch.
- **Legacy router interfaces**: separate physical interfaces per VLAN, used less often.

Router-on-a-stick example:

```cisco
conf t
interface gigabitEthernet0/0
 no shutdown

interface gigabitEthernet0/0.10
 encapsulation dot1Q 10
 ip address 192.168.10.1 255.255.255.0

interface gigabitEthernet0/0.20
 encapsulation dot1Q 20
 ip address 192.168.20.1 255.255.255.0
end
```

In this design, the router receives tagged frames for each VLAN and routes traffic between the subnets. The default gateway for hosts in VLAN 10 is `192.168.10.1`, and the default gateway for hosts in VLAN 20 is `192.168.20.1`.

**Design benefits**

VLANs improve network design in several practical ways:

- **Reduced broadcast scope** lowers unnecessary traffic.
- **Logical grouping** supports organizational structure and service separation.
- **Operational flexibility** makes moves and changes easier.
- **Fault isolation** helps contain some Layer 2 issues within a VLAN.
- **Policy alignment** allows different security or QoS rules for different groups.

A well-designed VLAN plan usually maps VLANs to business roles or traffic types, not to individual switch ports. That makes the network easier to document and scale.

**Common VLAN-related terms**

<dl>
<dt>Broadcast domain</dt>
<dd>A set of devices that receive Layer 2 broadcast frames.</dd>
<dt>Collision domain</dt>
<dd>A legacy Ethernet concept mostly eliminated by switches; each switch port is typically its own collision domain.</dd>
<dt>Tagging</dt>
<dd>Adding VLAN information to a frame so it can traverse a trunk.</dd>
<dt>Untagged frame</dt>
<dd>A frame without VLAN information, typically sent on an access port.</dd>
</dl>

**Troubleshooting checklist**

When VLAN traffic does not work as expected, verify the basics first:

1. Confirm the VLAN exists on the switch.
2. Check that the access port is assigned to the correct VLAN.
3. Verify the trunk is up and carrying the required VLANs.
4. Confirm the native VLAN matches on both ends of the trunk if used.
5. Check host IP addressing and default gateway settings.
6. Verify inter-VLAN routing if communication crosses VLAN boundaries.

Example host-side checks:

```bash
ip addr show
ip route
ping 192.168.10.1
arp -a
```

If two hosts in the same VLAN cannot communicate, the issue is often one of these:
- Wrong access VLAN assignment
- Trunk not allowing the VLAN
- Mismatched subnet or mask
- Incorrect default gateway for routed traffic
- Port security or STP state blocking forwarding

> A VLAN only segments Layer 2 traffic. If you need communication between VLANs, you must provide Layer 3 routing explicitly.

**Exam focus points**

For CCNA-level understanding, make sure you can explain:

- Why VLANs reduce broadcast domain size.
- The difference between access and trunk ports.
- Why hosts in different VLANs need routing to communicate.
- How VLANs support logical segmentation without changing physical cabling.
- How to verify VLAN and trunk configuration on Cisco switches.

**Quick mental model**

Think of a switch as a building and VLANs as separate rooms inside it. Devices in the same room can hear each other’s broadcasts. Devices in different rooms cannot, unless a router acts like a hallway between them.
