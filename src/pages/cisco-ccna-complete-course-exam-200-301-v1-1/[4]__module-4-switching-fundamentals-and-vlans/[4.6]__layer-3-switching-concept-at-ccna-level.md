## **Course:** Cisco CCNA Complete Course (Exam 200-301 v1.1)

**Layer 3 switching overview**

A Layer 3 switch, also called a **multilayer switch**, combines Layer 2 switching and Layer 3 routing in one device. At the CCNA level, the key idea is that it can forward frames within a VLAN like a normal switch and route packets between VLANs without sending traffic to an external router for every hop.

This matters because it gives you:
- **Fast inter-VLAN routing** using hardware forwarding
- **Simpler campus designs** with fewer devices
- **Better scalability** than router-on-a-stick in many access and distribution roles

A Layer 3 switch still has switch ports, but it can also create and use routed interfaces. In practice, you use it when you need high-speed routing between VLANs, static routes, or dynamic routing in a campus network.

**How Layer 3 switching works**

A traditional Layer 2 switch forwards traffic based on MAC addresses. A Layer 3 switch does that too, but it can also make forwarding decisions using IP addresses when routing is required.

The basic flow is:
- A host sends traffic to its default gateway.
- The Layer 3 switch receives the frame on an access port or trunk.
- The switch removes the Layer 2 header, examines the IP packet, and consults the routing table.
- The switch forwards the packet out the correct routed interface or VLAN interface.
- The switch rewrites the Layer 2 header for the next hop.

This is why a Layer 3 switch is often used as the default gateway for multiple VLANs. Each VLAN gets an interface on the switch, and the switch routes between them.

**Common CCNA use case: inter-VLAN routing**

The most common use of Layer 3 switching at CCNA level is **inter-VLAN routing**. Instead of sending VLAN traffic to a separate router, the switch routes locally.

Typical topology:
- VLAN 10 for users
- VLAN 20 for voice or another user group
- VLAN 30 for servers
- A Layer 3 switch in the distribution layer
- Access switches connecting end devices

Example logical design:
- `192.168.10.0/24` on VLAN 10
- `192.168.20.0/24` on VLAN 20
- `192.168.30.0/24` on VLAN 30

The switch provides the default gateway for each subnet, often using switched virtual interfaces, or **SVIs**.

**Switched Virtual Interfaces (SVIs)**

An SVI is a logical Layer 3 interface tied to a VLAN. It is not a physical port. You configure one SVI per VLAN that needs Layer 3 connectivity.

Why SVIs matter:
- They provide the default gateway for hosts in a VLAN
- They allow the switch to route between VLANs
- They are the standard CCNA method for Layer 3 switching on Cisco campus switches

Example:
```cisco
conf t
vlan 10
vlan 20
vlan 30

interface vlan 10
 ip address 192.168.10.1 255.255.255.0
 no shutdown

interface vlan 20
 ip address 192.168.20.1 255.255.255.0
 no shutdown

interface vlan 30
 ip address 192.168.30.1 255.255.255.0
 no shutdown
end
```

Hosts in VLAN 10 use `192.168.10.1` as their default gateway, and so on.

> If an SVI stays down, check whether the VLAN exists and whether at least one active port belongs to that VLAN. On many Cisco switches, an SVI comes up only when the VLAN is active in the switching table and has an up/up condition.

**Routed ports versus switchports**

A Layer 3 switch can use both Layer 2 switchports and Layer 3 routed ports.

- **Switchport**
  - Carries frames in a VLAN
  - Can be access or trunk
  - Used for end devices and VLAN transport
- **Routed port**
  - Operates like a router interface
  - Has an IP address directly on the physical interface
  - Does not belong to a VLAN

To convert a switchport into a routed port:
```cisco
conf t
interface gigabitEthernet1/0/1
 no switchport
 ip address 10.1.1.1 255.255.255.252
 no shutdown
end
```

Use routed ports when you want point-to-point Layer 3 links, such as:
- Switch-to-router uplinks
- Switch-to-switch routed links
- Connections to firewalls or WAN edge devices

**Layer 3 switching versus router-on-a-stick**

Router-on-a-stick uses one physical router interface with subinterfaces and an 802.1Q trunk to route between VLANs. Layer 3 switching performs the same logical job, but usually faster and with less bottleneck risk.

| Feature | Layer 3 switch | Router-on-a-stick |
|---|---|---|
| Forwarding method | Hardware-based, often wire speed | Router CPU and interface dependent |
| Cabling | No single trunk bottleneck for all inter-VLAN traffic | One trunk carries all VLAN traffic |
| Scalability | Better for campus access/distribution | Better for small networks or labs |
| Typical use | Enterprise switching | Small branch or learning environments |

Use Layer 3 switching when:
- You need high throughput between VLANs
- You want the default gateway close to the access layer
- You are designing a campus distribution layer

Use router-on-a-stick when:
- You have a small network
- You only have a Layer 2 switch
- You need a simple lab or low-cost design

**Routing table and forwarding behavior**

A Layer 3 switch uses a routing table just like a router. It learns routes from:
- Connected networks
- Static routes
- Dynamic routing protocols, if supported and configured

Example static route:
```cisco
conf t
ip route 0.0.0.0 0.0.0.0 10.1.1.2
end
```

Useful verification commands:
```cisco
show ip interface brief
show ip route
show vlan brief
show interfaces trunk
show running-config interface vlan 10
show mac address-table
```

What to look for:
- `show ip interface brief` confirms SVI and routed-port status
- `show ip route` confirms connected and learned routes
- `show vlan brief` confirms VLAN existence
- `show interfaces trunk` confirms trunk transport for VLANs
- `show mac address-table` confirms Layer 2 learning

**Hardware forwarding and why it is fast**

Layer 3 switches are designed to move traffic in hardware using specialized forwarding logic. At a CCNA level, the important takeaway is that the switch does not need to process every packet in software the way a general-purpose router might in some cases.

This gives you:
- Lower latency
- Higher throughput
- Better performance for many VLANs and users

The exact internal architecture varies by platform, so avoid assuming every switch behaves identically. The exam focus is the concept: Layer 3 switching is optimized for fast inter-VLAN routing.

**Configuration example for inter-VLAN routing**

A common CCNA configuration uses SVIs and access ports.

```cisco
conf t
vlan 10
 name USERS
vlan 20
 name VOICE

interface range gigabitEthernet1/0/2-10
 switchport mode access
 switchport access vlan 10

interface gigabitEthernet1/0/11
 switchport mode access
 switchport access vlan 20

interface vlan 10
 ip address 192.168.10.1 255.255.255.0
 no shutdown

interface vlan 20
 ip address 192.168.20.1 255.255.255.0
 no shutdown

ip routing
end
```

The `ip routing` command is critical on many Cisco multilayer switches. Without it, the switch may have SVIs but not route between them.

**Verification and troubleshooting workflow**

When inter-VLAN routing does not work, check the problem in layers.

1. Confirm the VLAN exists.
   - Use `show vlan brief`
   - Make sure access ports are assigned correctly

2. Confirm the SVI is up.
   - Use `show ip interface brief`
   - Check for `up/up`

3. Confirm routing is enabled.
   - Use `show running-config | include ip routing`
   - Ensure `ip routing` is configured

4. Confirm host addressing.
   - Verify IP address, subnet mask, and default gateway
   - The gateway must match the SVI IP

5. Confirm trunking if the VLAN crosses switches.
   - Use `show interfaces trunk`
   - Ensure the VLAN is allowed on the trunk

6. Confirm the routing table.
   - Use `show ip route`
   - Look for connected routes for each VLAN

Common problems:
- The SVI is administratively down
- No active ports exist in the VLAN
- The host uses the wrong default gateway
- The VLAN is missing from the trunk allowed list
- `ip routing` is not enabled

**Static routing and dynamic routing on a Layer 3 switch**

A Layer 3 switch can participate in routing beyond local VLANs. In many campus designs, it connects to an upstream router or firewall and uses a default route.

Example:
```cisco
conf t
interface gigabitEthernet1/0/48
 no switchport
 ip address 10.10.10.2 255.255.255.252
 no shutdown

ip route 0.0.0.0 0.0.0.0 10.10.10.1
end
```

Some Cisco multilayer switches also support dynamic routing protocols such as OSPF (Open Shortest Path First) and EIGRP (Enhanced Interior Gateway Routing Protocol), depending on platform and licensing. For CCNA, focus on the concept that the switch can route like a router, not just switch like a Layer 2 device.

**Design considerations**

Layer 3 switching is usually preferred in larger LANs because it reduces unnecessary traffic across the network core.

Benefits:
- Limits broadcast domain size by VLAN
- Routes locally at distribution or access layers
- Reduces dependence on a central router
- Improves resilience and performance

Tradeoffs:
- More complex than pure Layer 2 switching
- Requires correct SVI and routing configuration
- May require careful planning for redundancy and gateway design

In redundant designs, you may see first-hop redundancy protocols such as HSRP (Hot Standby Router Protocol), VRRP (Virtual Router Redundancy Protocol), or GLBP (Gateway Load Balancing Protocol). These are beyond the core CCNA switching concept, but they are often associated with multilayer switch deployments.

**Exam focus points**

For CCNA, remember these core ideas:
- A Layer 3 switch performs both switching and routing
- SVIs provide Layer 3 gateways for VLANs
- `ip routing` enables routing between SVIs on many Cisco switches
- Routed ports are physical Layer 3 interfaces
- Layer 3 switching is commonly used for inter-VLAN routing
- It is generally faster and more scalable than router-on-a-stick

**Quick comparison summary**

- **Layer 2 switch**
  - Forwards frames by MAC address
  - Does not route between VLANs
- **Layer 3 switch**
  - Forwards frames and routes packets
  - Supports SVIs and routed ports
- **Router-on-a-stick**
  - Routes between VLANs using subinterfaces on one trunk
  - Useful in smaller or simpler environments

**Practical lab checklist**

Use this checklist when building a CCNA lab:
- Create VLANs
- Assign access ports to the correct VLANs
- Configure SVIs with gateway IP addresses
- Enable `ip routing`
- Add routed uplinks if needed
- Verify host default gateways
- Test connectivity with `ping` and `traceroute`

Example test commands:
```cisco
ping 192.168.20.10
traceroute 192.168.30.15
show ip route
show ip interface brief
```

A successful Layer 3 switching lab should let hosts in different VLANs communicate through the switch without requiring an external router for local inter-VLAN traffic.
