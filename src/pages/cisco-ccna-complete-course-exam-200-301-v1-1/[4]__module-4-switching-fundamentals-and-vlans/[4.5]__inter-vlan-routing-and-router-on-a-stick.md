## **Course:** Cisco CCNA Complete Course (Exam 200-301 v1.1)

**Inter-VLAN routing overview**

VLANs (Virtual Local Area Networks) separate broadcast domains at Layer 2, but hosts in different VLANs cannot communicate without Layer 3 routing. Inter-VLAN routing provides that Layer 3 path while preserving VLAN segmentation.

- A **VLAN** isolates traffic logically even when devices share the same physical switch.
- A **default gateway** is the Layer 3 address hosts use to reach other networks.
- Inter-VLAN routing is required when users in different VLANs must access shared services, printers, or the internet through a router or Layer 3 switch.
- In CCNA labs, the most common router-based design is **router-on-a-stick**.

**Why inter-VLAN routing is needed**

A switch forwards frames based on MAC addresses within the same VLAN. If a host in VLAN 10 needs to reach a host in VLAN 20, the frame must be sent to a Layer 3 device that can route between those networks. Without routing, the traffic stays confined to the source VLAN.

Typical reasons to route between VLANs include:

- User access to shared servers in another VLAN
- Voice and data separation with controlled communication
- Guest VLAN isolation with selective access
- Management access to network devices from an admin VLAN

**Router-on-a-stick concept**

Router-on-a-stick uses one physical router interface connected to a switch trunk. The router creates multiple logical interfaces called **subinterfaces**, and each subinterface handles one VLAN.

The switch sends frames for multiple VLANs across a single trunk link using 802.1Q tagging. The router receives tagged frames on the trunk, matches the VLAN tag to the correct subinterface, routes the packet, and sends the return traffic back out the same physical interface with the appropriate tag.

> On 802.1Q trunks, the sending device tags frames on egress and the receiving device interprets the tag on ingress. The switch does not continuously rewrite every frame in transit.

Key characteristics:

- One physical router interface supports multiple VLANs.
- Each subinterface maps to one VLAN ID.
- Each VLAN gets its own IP address on the router, which becomes the hosts’ default gateway.
- The switch port facing the router must be a trunk.

**Topology example**

A simple design might look like this:

- VLAN 10: `192.168.10.0/24`
- VLAN 20: `192.168.20.0/24`
- Router interface: `G0/0`
- Switch uplink: `G0/1`

Traffic flow:

1. Host in VLAN 10 sends traffic to its default gateway `192.168.10.1`.
2. The switch forwards the frame across the trunk to the router with VLAN 10 tagging.
3. The router routes the packet to VLAN 20.
4. The router sends the return traffic back out the trunk tagged for VLAN 20.
5. The switch delivers the frame to the destination host in VLAN 20.

**802.1Q trunking essentials**

802.1Q is the standard VLAN tagging method used on trunks. It inserts a VLAN tag into Ethernet frames so multiple VLANs can share one physical link.

Important points:

- A trunk carries traffic for multiple VLANs.
- An access port carries traffic for one VLAN only.
- Native VLAN traffic is sent untagged on an 802.1Q trunk unless the platform or configuration changes that behavior.
- Both ends of the trunk should agree on trunking parameters, allowed VLANs, and native VLAN.

Common trunk verification commands:

```bash
show interfaces trunk
show interfaces g0/1 switchport
show vlan brief
```

**Router-on-a-stick configuration**

You configure the switch port as a trunk and create router subinterfaces. Each subinterface gets an 802.1Q encapsulation statement and an IP address.

Example switch configuration:

```cisco
conf t
vlan 10
 name SALES
vlan 20
 name ENGINEERING

interface g0/1
 description Trunk to Router
 switchport mode trunk
 switchport trunk allowed vlan 10,20
 no shutdown
end
```

On many modern Cisco switches, `switchport trunk encapsulation dot1q` is not available because 802.1Q is the only supported trunk encapsulation. Use it only if the platform supports the command.

Example router configuration:

```cisco
conf t
interface g0/0
 no shutdown

interface g0/0.10
 encapsulation dot1Q 10
 ip address 192.168.10.1 255.255.255.0

interface g0/0.20
 encapsulation dot1Q 20
 ip address 192.168.20.1 255.255.255.0
end
```

If you need a native VLAN on the router, you can mark one subinterface as native:

```cisco
interface g0/0.99
 encapsulation dot1Q 99 native
 ip address 192.168.99.1 255.255.255.0
```

**How subinterfaces work**

A subinterface is a logical division of a physical interface. The router treats each subinterface as a separate Layer 3 interface.

- `G0/0.10` is not a separate physical port.
- The encapsulation command binds the subinterface to a VLAN tag.
- The IP address on the subinterface is the gateway for that VLAN.
- The physical interface itself usually has no IP address when using router-on-a-stick.

Practical mapping example:

| VLAN | Subinterface | Gateway IP |
|---|---|---|
| 10 | `G0/0.10` | `192.168.10.1/24` |
| 20 | `G0/0.20` | `192.168.20.1/24` |
| 99 | `G0/0.99` | `192.168.99.1/24` |

**Host configuration**

Each host must use the correct default gateway for its VLAN.

Example host settings:

- VLAN 10 host:
  - IP: `192.168.10.10`
  - Mask: `255.255.255.0`
  - Gateway: `192.168.10.1`
- VLAN 20 host:
  - IP: `192.168.20.10`
  - Mask: `255.255.255.0`
  - Gateway: `192.168.20.1`

If the gateway is wrong, the host may still communicate inside its own VLAN but fail to reach other VLANs.

**Verification commands**

Use verification to confirm the trunk, VLAN membership, and router subinterfaces.

Switch-side checks:

```bash
show vlan brief
show interfaces trunk
show interfaces g0/1 switchport
show running-config interface g0/1
```

Router-side checks:

```bash
show ip interface brief
show running-config interface g0/0
show running-config interface g0/0.10
show running-config interface g0/0.20
show ip route
show ip arp
```

What to look for:

- The trunk port is up and in trunking mode.
- The correct VLANs are allowed on the trunk.
- Each subinterface is up/up.
- Each subinterface has the correct `encapsulation dot1Q` value.
- The router has connected routes for each VLAN subnet.
- The ARP table contains host IP-to-MAC mappings after traffic is generated.

**Troubleshooting workflow**

When inter-VLAN routing fails, isolate the problem layer by layer.

1. Verify the host IP address, mask, and default gateway.
2. Confirm the access port is in the correct VLAN.
3. Confirm the switch uplink is trunking.
4. Confirm the trunk allows the required VLANs.
5. Confirm the router subinterface exists and uses the correct VLAN tag.
6. Confirm the router interface and subinterfaces are not administratively down.
7. Test connectivity with `ping` and inspect `show ip arp`.

Common problems:

- The access port is assigned to the wrong VLAN.
- The trunk does not allow the VLAN.
- The router subinterface uses the wrong VLAN ID.
- The host gateway points to the wrong IP address.
- The physical router interface is shut down.
- The native VLAN differs between switch and router expectations.

**Useful ping tests**

Start with the local gateway, then test remote VLAN reachability.

```bash
ping 192.168.10.1
ping 192.168.20.1
ping 192.168.20.10
traceroute 192.168.20.10
```

If the host can ping its gateway but not another VLAN, the issue is usually on the router, trunk, or remote host configuration.

**Router-on-a-stick advantages**

Router-on-a-stick is simple and useful in small networks and labs.

- It requires only one router interface.
- It is easy to understand for CCNA study.
- It supports multiple VLANs without needing a Layer 3 switch.
- It is cost-effective for small deployments.

**Router-on-a-stick limitations**

This design has scaling and performance limits.

- All inter-VLAN traffic shares one physical link.
- The router can become a bottleneck under heavy traffic.
- Troubleshooting can be more involved than with a Layer 3 switch.
- It is less suitable for large campus networks.

**Layer 3 switch comparison**

A Layer 3 switch performs inter-VLAN routing using switched virtual interfaces (SVIs), which often provides better performance than router-on-a-stick.

| Feature | Router-on-a-stick | Layer 3 switch |
|---|---|---|
| Routing device | Router | Switch |
| Interface model | Subinterfaces | SVIs |
| Link usage | One trunk link | Routed or switched design dependent |
| Scalability | Lower | Higher |
| CCNA lab relevance | Very high | High |

**Example end-to-end configuration**

A compact lab example:

```cisco
! Switch
conf t
vlan 10
 name SALES
vlan 20
 name ENGINEERING

interface range f0/1-10
 switchport mode access
 switchport access vlan 10

interface range f0/11-20
 switchport mode access
 switchport access vlan 20

interface g0/1
 description To Router
 switchport mode trunk
 switchport trunk allowed vlan 10,20
 no shutdown
end
```

```cisco
! Router
conf t
interface g0/0
 no shutdown

interface g0/0.10
 encapsulation dot1Q 10
 ip address 192.168.10.1 255.255.255.0

interface g0/0.20
 encapsulation dot1Q 20
 ip address 192.168.20.1 255.255.255.0
end
```

This design lets VLAN 10 and VLAN 20 communicate through the router while keeping their broadcast domains separate.

**Exam focus**

For CCNA, know how to:

- Explain why inter-VLAN routing is needed.
- Configure a trunk between a switch and router.
- Create router subinterfaces with `encapsulation dot1Q`.
- Assign the correct gateway IP to each VLAN.
- Verify the configuration with `show` commands.
- Troubleshoot common trunk, VLAN, and gateway mistakes.

**Key takeaways**

Inter-VLAN routing connects separate VLANs at Layer 3 while preserving segmentation at Layer 2. Router-on-a-stick is the standard CCNA router-based method: one trunk, multiple router subinterfaces, one gateway per VLAN, and careful matching of VLAN IDs, IP subnets, and trunk settings.
