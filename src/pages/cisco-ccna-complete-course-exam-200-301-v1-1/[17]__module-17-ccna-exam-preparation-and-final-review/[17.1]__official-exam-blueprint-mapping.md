## **Course:** Cisco CCNA Complete Course (Exam 200-301 v1.1)

**What the CCNA blueprint is**

The Cisco CCNA 200-301 v1.1 exam blueprint is the official outline of what Cisco expects you to know. It is organized into six domains, each with published topic weights and subtopics. That means you should study by domain and subtopic, not treat every item as equally important.

The blueprint is best used as a coverage map:
- It tells you what to learn.
- It helps you identify weak areas.
- It keeps your study plan aligned with the exam instead of drifting into unrelated networking topics.

> The blueprint is a study guide, not a promise that every question will match the same level of detail across all subtopics.

**How to use the blueprint effectively**

Start by reading the official domain names and their weight ranges. Then break each domain into smaller objectives and verify that you can explain, configure, and troubleshoot each one. For CCNA, you need both conceptual understanding and hands-on familiarity.

A practical study workflow looks like this:
1. Read the domain objective.
2. Learn the theory and key terms.
3. Configure the feature in a lab.
4. Verify it with show commands.
5. Troubleshoot common failures.
6. Revisit weak subtopics until you can explain them without notes.

**CCNA 200-301 v1.1 domain map**

| Domain | Official focus | What you should be able to do |
|---|---|---|
| Network Fundamentals | Core networking concepts, Ethernet, IPv4/IPv6, wireless, virtualization, and architecture | Explain how networks move traffic and how devices identify, address, and reach each other |
| Network Access | Switching concepts, VLANs, trunking, inter-VLAN connectivity, STP, EtherChannel, and wireless access | Configure and verify Layer 2 access features |
| IP Connectivity | Routing concepts, static routing, OSPF, and route selection | Build and verify Layer 3 reachability |
| IP Services | DHCP, NAT, NTP, DNS, SNMP, syslog, QoS, and related services | Support and troubleshoot common network services |
| Security Fundamentals | Device hardening, access control, AAA, VPN concepts, and security best practices | Reduce risk and control access to devices and traffic |
| Automation and Programmability | APIs, controllers, data formats, and network automation concepts | Recognize how modern networks are managed and automated |

**Network Fundamentals**

This domain covers the building blocks of networking. You need to understand how data moves, how devices identify each other, and how common protocols support communication.

Key subtopics include:
- Network components such as routers, switches, access points, firewalls, and endpoints
- Network topologies and architectures, including LAN, WAN, and cloud connectivity
- IPv4 and IPv6 addressing and subnetting
- Ethernet concepts, MAC address learning, and ARP
- Wireless fundamentals and basic deployment concepts
- Virtualization and cloud concepts
- Basic troubleshooting tools and commands

**Ethernet and Layer 2 basics**

Ethernet is the dominant LAN technology in CCNA-level networks. Switches forward frames using MAC addresses, and they learn source MAC addresses dynamically by observing incoming traffic.

Important concepts:
- A switch builds a MAC address table from source addresses.
- Unknown unicast frames are flooded out all ports in the VLAN except the incoming port.
- Broadcast frames are flooded within the VLAN.
- Multicast frames are forwarded based on multicast handling and switch capability.

Address Resolution Protocol (<abbr title="Address Resolution Protocol">ARP</abbr>) maps IPv4 addresses to MAC addresses on the local network. When a host wants to send to a local IPv4 destination, it first checks its ARP cache. If the mapping is missing, it sends an ARP request.

Example verification:
```bash
show mac address-table
show arp
show ip interface brief
```

**IPv4 and IPv6 addressing**

You should be able to subnet IPv4 networks quickly and understand how IPv6 addressing differs from IPv4. IPv6 uses a much larger address space, supports global unicast, link-local, unique local, and multicast addressing, and does not use broadcast.

Useful IPv6 points:
- Link-local addresses begin with `fe80::/10`.
- Hosts use link-local addresses for neighbor discovery and next-hop communication on the local link.
- Static IPv6 routes can use either a global next hop or an exit interface plus link-local next hop.

Examples:
```bash
ipv6 route 2001:db8:10:20::/64 2001:db8:1::2
ipv6 route 2001:db8:10:20::/64 GigabitEthernet0/0 fe80::1
```

**Wireless fundamentals**

CCNA expects you to understand basic wireless architecture and terminology, not deep RF engineering. Know the difference between access points, wireless LAN controllers (<abbr title="Wireless LAN Controller">WLC</abbr>), SSIDs, and common security modes.

Study points:
- An AP can operate in autonomous or controller-based designs.
- A WLC centralizes configuration and policy for multiple APs.
- SSID design affects user experience and segmentation.
- Channel selection and interference affect performance.

**Virtualization and cloud concepts**

Virtualization lets multiple virtual machines share physical hardware. Cloud models extend this idea into on-demand services.

Know the basics:
- Hypervisors can be Type 1 or Type 2.
- Virtual machines isolate workloads.
- Public, private, and hybrid cloud models solve different operational needs.
- Network functions may be virtualized as software appliances.

**Network Access**

This domain focuses on how devices connect at Layer 2 and how switching features control traffic flow. It is one of the most practical parts of CCNA because you can lab most of it directly.

Key subtopics include:
- VLANs and trunking
- Inter-VLAN routing
- Spanning Tree Protocol (<abbr title="Spanning Tree Protocol">STP</abbr>)
- EtherChannel
- Switch security features
- Wireless access and basic WLAN concepts

**VLANs and trunks**

A VLAN separates broadcast domains on the same physical switch infrastructure. Trunks carry traffic for multiple VLANs between switches or between a switch and a router or firewall.

Example configuration:
```bash
vlan 10
 name USERS
vlan 20
 name VOICE

interface GigabitEthernet0/1
 switchport mode access
 switchport access vlan 10

interface GigabitEthernet0/24
 switchport mode trunk
 switchport trunk allowed vlan 10,20
```

Verification:
```bash
show vlan brief
show interfaces trunk
```

**Inter-VLAN routing**

You need to know the common methods for routing between VLANs:
- Router-on-a-stick uses one physical router interface with subinterfaces.
- Layer 3 switches can route between VLANs using switched virtual interfaces (<abbr title="Switched Virtual Interface">SVI</abbr>s).

Router-on-a-stick example:
```bash
interface GigabitEthernet0/0.10
 encapsulation dot1Q 10
 ip address 192.168.10.1 255.255.255.0

interface GigabitEthernet0/0.20
 encapsulation dot1Q 20
 ip address 192.168.20.1 255.255.255.0
```

**Spanning Tree and loop prevention**

STP prevents Layer 2 loops by blocking redundant paths. You should understand why loops are dangerous: they create broadcast storms, MAC table instability, and duplicate frames.

Important ideas:
- STP elects a root bridge.
- Switch ports can be designated, root, or blocked depending on topology.
- PortFast speeds up edge-port forwarding.
- BPDU Guard protects access ports from accidental switch connections.

Verification:
```bash
show spanning-tree
show spanning-tree vlan 10
```

**EtherChannel**

EtherChannel bundles multiple physical links into one logical link for redundancy and load sharing. CCNA commonly expects you to recognize LACP (<abbr title="Link Aggregation Control Protocol">LACP</abbr>) and basic configuration.

Example:
```bash
interface range GigabitEthernet0/1 - 2
 channel-group 1 mode active

interface Port-channel1
 switchport mode trunk
```

Verification:
```bash
show etherchannel summary
show interfaces port-channel 1
```

**Switch security**

Switch security features reduce unauthorized access and limit damage from misconfiguration or attacks.

Common features:
- Port security limits learned MAC addresses.
- DHCP snooping helps block rogue DHCP servers.
- Dynamic ARP Inspection (<abbr title="Dynamic ARP Inspection">DAI</abbr>) helps protect against ARP spoofing.
- Storm control limits broadcast, multicast, or unknown unicast traffic.

Example port security:
```bash
interface GigabitEthernet0/10
 switchport mode access
 switchport port-security
 switchport port-security maximum 2
 switchport port-security mac-address sticky
 switchport port-security violation restrict
```

**IP Connectivity**

This domain covers routing behavior, route selection, and OSPF. You need to understand both how routers make forwarding decisions and how to verify those decisions.

Key subtopics include:
- Routing table structure
- Static routes and default routes
- OSPFv2 and OSPFv3 basics
- Neighbor relationships and verification
- Route preference and administrative distance

**Routing table and route selection**

Routers choose the best route based on longest prefix match first, then administrative distance, then metric. This is a core exam concept because it explains why one route wins over another.

Useful verification:
```bash
show ip route
show ipv6 route
show ip protocols
```

**Static routing**

Static routes are manually configured and useful for small networks, stub networks, and default paths. They are predictable but do not adapt automatically to topology changes.

IPv4 example:
```bash
ip route 192.168.30.0 255.255.255.0 10.1.1.2
ip route 0.0.0.0 0.0.0.0 10.1.1.1
```

IPv6 example:
```bash
ipv6 route 2001:db8:30::/64 2001:db8:1::2
ipv6 route 2001:db8:40::/64 GigabitEthernet0/0 fe80::1
```

**OSPF basics**

Open Shortest Path First (<abbr title="Open Shortest Path First">OSPF</abbr>) is a link-state routing protocol. CCNA focuses on the fundamentals: neighbor formation, area 0, router IDs, passive interfaces, and verification.

You should know:
- OSPF uses hello packets to discover neighbors.
- Routers must agree on key parameters such as area ID, timers, and authentication settings if used.
- The router ID is a 32-bit identifier chosen from configured values or interface addresses.
- Passive interfaces stop OSPF hello packets on selected interfaces while still advertising the network.

Classic configuration example:
```bash
router ospf 1
 router-id 1.1.1.1
 network 10.1.1.0 0.0.0.255 area 0
 passive-interface GigabitEthernet0/2
```

Interface-based OSPF is also common on modern IOS:
```bash
interface GigabitEthernet0/0
 ip ospf 1 area 0
```

Verification:
```bash
show ip ospf neighbor
show ip ospf interface brief
show ip route ospf
```

**OSPF neighbor states**

Neighbor states help you troubleshoot adjacency problems.

Common states:
- Down
- Init
- 2-Way
- ExStart
- Exchange
- Loading
- Full

If neighbors stop at 2-Way or never reach Full, check area mismatch, hello/dead timers, network type, authentication, or interface issues.

**IP Services**

This domain covers services that support network operation and user connectivity. Many exam questions are scenario-based, so you should know what each service does and how to verify it.

Key subtopics include:
- DHCP
- NAT
- DNS
- NTP
- SNMP
- Syslog
- QoS

**DHCP**

Dynamic Host Configuration Protocol (<abbr title="Dynamic Host Configuration Protocol">DHCP</abbr>) automatically assigns IP settings to clients. You should understand the DHCP process and how relay works when the server is on another network.

Basic server example:
```bash
ip dhcp excluded-address 192.168.10.1 192.168.10.20

ip dhcp pool USERS
 network 192.168.10.0 255.255.255.0
 default-router 192.168.10.1
 dns-server 8.8.8.8
```

Relay example:
```bash
interface GigabitEthernet0/0
 ip helper-address 10.1.1.10
```

**NAT**

Network Address Translation (<abbr title="Network Address Translation">NAT</abbr>) translates private addresses to public addresses or between address spaces. CCNA commonly emphasizes inside local, inside global, outside local, and outside global terminology.

PAT example:
```bash
access-list 1 permit 192.168.10.0 0.0.0.255

interface GigabitEthernet0/0
 ip nat inside

interface GigabitEthernet0/1
 ip nat outside

ip nat inside source list 1 interface GigabitEthernet0/1 overload
```

Verification:
```bash
show ip nat translations
show ip nat statistics
```

**DNS**

Domain Name System (<abbr title="Domain Name System">DNS</abbr>) resolves names to IP addresses. In CCNA, you usually need to understand the role of DNS more than build a full DNS server, but you should still know how clients use it and how to test resolution.

Useful checks:
```bash
ping www.example.com
nslookup www.example.com
show hosts
```

**NTP and syslog**

Network Time Protocol (<abbr title="Network Time Protocol">NTP</abbr>) keeps device clocks synchronized, which matters for logs, certificates, and troubleshooting timelines. Syslog centralizes event messages so you can correlate incidents across devices.

NTP example:
```bash
ntp server 10.1.1.50
show ntp status
show ntp associations
```

Syslog example:
```bash
logging host 10.1.1.60
logging trap informational
show logging
```

**SNMP**

Simple Network Management Protocol (<abbr title="Simple Network Management Protocol">SNMP</abbr>) is used for monitoring and management. CCNA commonly expects you to distinguish SNMP versions:
- SNMPv1 and SNMPv2c use community strings.
- SNMPv3 adds authentication and encryption.

Example:
```bash
snmp-server community public RO
snmp-server community private RW
snmp-server group NMS v3 priv
```

**QoS**

Quality of Service (<abbr title="Quality of Service">QoS</abbr>) helps networks handle traffic differently based on business need. At CCNA level, focus on the major functions:
- Classification identifies traffic.
- Marking tags traffic for priority handling.
- Queuing manages congestion.
- Policing drops or remarks traffic that exceeds a rate.
- Shaping buffers excess traffic and sends it later.

A simple mental model:
- Use policing when you want to enforce a hard limit.
- Use shaping when you want to smooth bursts.
- Use marking when downstream devices need to recognize traffic priority.

**Security Fundamentals**

This domain covers protecting devices, users, and traffic. The exam expects you to recognize common security controls and apply basic hardening.

Key subtopics include:
- Device access control
- Password and SSH hardening
- AAA
- ACLs
- VPN concepts
- Wireless security basics

**Device hardening**

Secure management access by disabling unused services, using strong credentials, and preferring encrypted management protocols.

Example hardening:
```bash
hostname SW1
no ip domain-lookup
service password-encryption
enable secret Str0ngEnableSecret
banner motd ^CUnauthorized access prohibited^C
```

**SSH configuration**

SSH is the preferred remote management protocol because it encrypts the session. A working local-authentication setup usually includes a username, domain name, RSA keys, and VTY configuration.

Example:
```bash
ip domain-name lab.local
username admin privilege 15 secret AdminP@ss123
crypto key generate rsa modulus 2048
ip ssh version 2

line vty 0 4
 login local
 transport input ssh
```

**AAA**

Authentication, Authorization, and Accounting (<abbr title="Authentication, Authorization, and Accounting">AAA</abbr>) separates three security functions:
- Authentication verifies identity.
- Authorization determines what the user can do.
- Accounting records what the user did.

At CCNA level, you should understand the purpose of AAA and recognize that it can be implemented locally or through centralized servers such as RADIUS or TACACS+.

**ACLs**

Access Control Lists (<abbr title="Access Control List">ACL</abbr>s) filter traffic based on source, destination, protocol, and ports depending on ACL type. Standard ACLs match source IP only, while extended ACLs can match more fields.

Standard ACL example:
```bash
access-list 10 permit 192.168.10.0 0.0.0.255

interface GigabitEthernet0/1
 ip access-group 10 in
```

Placement matters:
- Standard ACLs are usually placed close to the destination.
- Extended ACLs are usually placed close to the source.

That is a guideline, not a rule. You still need to match the ACL direction to the traffic flow you want to control.

**VPN concepts**

Virtual Private Networks (<abbr title="Virtual Private Network">VPN</abbr>s) create secure tunnels over untrusted networks. For CCNA, focus on the purpose and basic types rather than deep cryptographic details.

Know the difference between:
- Site-to-site VPNs for network-to-network connectivity
- Remote-access VPNs for individual users
- Encryption, integrity, and authentication as core goals

**Wireless security**

Wireless security protects the radio access layer and client authentication. Know the common security modes and why stronger authentication matters.

Study points:
- Open networks provide no encryption.
- WPA2 and WPA3 improve confidentiality and access control.
- Enterprise wireless often uses 802.1X with centralized authentication.
- Guest networks should be isolated from internal resources.

**Automation and Programmability**

This domain introduces how modern networks are controlled and monitored through software. The exam focuses on concepts, data formats, and common interfaces rather than advanced coding.

Key subtopics include:
- Controller-based networking
- APIs
- JSON and XML
- NETCONF and RESTCONF
- YANG
- Python and automation tools

**Controller-based networking**

In controller-based architectures, a central controller manages policy, visibility, and automation across the network. This reduces manual configuration and helps enforce consistency.

Important terms:
- A controller provides centralized control.
- Devices may still forward traffic locally.
- Policy can be pushed from software rather than configured device by device.

**APIs and data formats**

An API (<abbr title="Application Programming Interface">API</abbr>) lets software communicate with devices or controllers. CCNA commonly expects you to recognize REST-style interactions and common data formats.

Know these formats:
- JSON is lightweight and common in APIs.
- XML is structured and still used in some network systems.

Example JSON:
```json
{
  "hostname": "SW1",
  "interfaces": [
    {
      "name": "GigabitEthernet0/1",
      "description": "Uplink",
      "enabled": true
    }
  ]
}
```

**NETCONF, RESTCONF, and YANG**

NETCONF and RESTCONF are network management protocols used for programmatic configuration and retrieval. YANG is the data modeling language that describes the structure of network configuration and operational data.

How they relate:
- YANG defines the model.
- NETCONF and RESTCONF use the model to exchange data.
- Controllers and automation tools often consume these interfaces.

Recognition clues:
- NETCONF often uses XML and SSH.
- RESTCONF often uses HTTP methods and JSON or XML payloads.

**Python and automation tools**

Python is a common automation language because it is readable and has strong library support. In CCNA, you usually need to understand why it is useful, not write complex programs.

Simple example:
```python
import requests

url = "https://controller.example/api/interfaces"
response = requests.get(url, verify=False)
print(response.status_code)
print(response.json())
```

**How to study the blueprint by domain**

Use the blueprint as a checklist, but study each item in context. A topic is not fully learned until you can explain it, configure it, verify it, and troubleshoot it.

A strong study pattern is:
1. Learn the concept.
2. Build a small lab.
3. Break the configuration on purpose.
4. Use show commands to identify the fault.
5. Fix it without looking at the answer.

**High-value verification commands**

These commands appear often in CCNA labs and troubleshooting scenarios:
```bash
show ip interface brief
show running-config
show vlan brief
show interfaces trunk
show mac address-table
show spanning-tree
show etherchannel summary
show ip route
show ip ospf neighbor
show ip nat translations
show logging
show access-lists
```

**Common exam traps**

- Confusing a concept with a configuration command.
- Memorizing a feature without knowing where it fits in the blueprint.
- Forgetting that some topics are conceptual while others require CLI verification.
- Assuming one configuration style is the only valid one, such as classic OSPF network statements instead of interface-based OSPF.
- Ignoring direction and placement when applying ACLs.
- Treating IPv6 static routing as identical to IPv4 without understanding link-local next-hop syntax.

**Final blueprint strategy**

The best CCNA preparation aligns every lab and review session to a blueprint subtopic. If you can explain the domain, configure the feature, and verify it with the right command, you are studying in the same style the exam expects.

Focus on coverage first, then speed, then troubleshooting. That sequence gives you the best chance of turning blueprint knowledge into exam-ready skill.
