## **Course:** Cisco CCNA Complete Course (Exam 200-301 v1.1)

**How to use these notes**

These notes are organized by CCNA domain so you can review concepts in the same way the exam blueprint is structured. Focus on understanding how features work together in a small enterprise network: addressing, switching, routing, services, security, and automation.

- Treat each section as both a concept review and a troubleshooting checklist.
- Memorize common commands, but also know what the output means.
- Practice with small topologies so you can connect theory to CLI behavior.

**Network Fundamentals**

The exam expects you to understand how data moves across a network and how addresses are represented. You should be comfortable with binary, decimal, and hexadecimal because subnetting and IPv6 addressing depend on fast conversion.

- **Binary to decimal** conversion uses place values: `128 64 32 16 8 4 2 1`.
- **Subnetting** is about borrowing host bits to create smaller networks with predictable host counts.
- **VLSM** (Variable Length Subnet Mask) lets you allocate different subnet sizes efficiently.
- **IPv4 planning** should account for network ID, broadcast address, usable host range, and growth.
- **IPv6 planning** should account for prefix length, subnet boundaries, and address type.

A quick binary example:

```text
11000000.10101000.00000001.00001010
= 192.168.1.10
```

A common subnetting workflow is:

1. Identify the required number of hosts per subnet.
2. Choose the smallest subnet mask that fits.
3. Allocate the largest subnets first when using VLSM.
4. Verify network, broadcast, and usable host ranges.

For example, a `/26` subnet provides 64 addresses total and 62 usable hosts:

```text
192.168.10.0/26
192.168.10.0   network
192.168.10.1   first host
192.168.10.62  last host
192.168.10.63  broadcast
```

IPv6 has different operational behavior than IPv4. You should know the major address types and how hosts learn configuration.

- **Global unicast** addresses are routable on the internet and identify hosts.
- **Link-local** addresses (`fe80::/10`) are used on the local link and commonly serve as the next-hop address in routing protocols.
- **Unique local** addresses (`fc00::/7`) are private-like internal addresses.
- **Multicast** replaces broadcast in IPv6.
- **EUI-64** can derive the interface ID from a MAC address, though many systems now use privacy or stable random addresses instead.

Router Advertisements (RA) carry important flags:

- **M flag** indicates managed addressing, meaning use DHCPv6 for address assignment.
- **O flag** indicates other configuration, meaning use DHCPv6 for additional information such as DNS.
- SLAAC (Stateless Address Autoconfiguration) uses RA messages to build an address without DHCPv6 for the address itself.

Practical IPv6 examples:

```bash
show ipv6 interface brief
show ipv6 route
ping ipv6 fe80::1 source g0/0
```

**Network Access**

Switching fundamentals are heavily tested because they affect segmentation, loop prevention, and Layer 2 forwarding. You should know how VLANs, trunks, and spanning tree interact.

- **VLANs** separate broadcast domains on the same physical switch infrastructure.
- **Access ports** carry traffic for one VLAN.
- **Trunks** carry multiple VLANs using 802.1Q tagging.
- **Native VLAN** traffic is untagged on an 802.1Q trunk.
- **Allowed VLAN lists** restrict which VLANs may traverse a trunk.

Basic VLAN configuration:

```bash
conf t
vlan 10
 name USERS
vlan 20
 name VOICE
interface g0/1
 switchport mode access
 switchport access vlan 10
interface g0/2
 switchport mode trunk
 switchport trunk allowed vlan 10,20
end
```

Verification commands matter as much as configuration:

```bash
show vlan brief
show interfaces trunk
show interfaces switchport
```

Spanning Tree Protocol (STP) prevents Layer 2 loops by blocking redundant paths. In Rapid Spanning Tree Protocol (RSTP), port roles and states are simplified.

- **Root port** is the best path toward the root bridge.
- **Designated port** forwards traffic for a segment.
- **Alternate port** is a backup path in RSTP.
- **Backup port** can exist on shared segments in RSTP.
- RSTP states are typically **discarding**, **learning**, and **forwarding**.

You should also know common STP protections:

- **BPDU Guard** disables an edge port if it receives a Bridge Protocol Data Unit (BPDU).
- **Root Guard** prevents a downstream switch from becoming root.
- **Loop Guard** helps protect against unidirectional link failures that could create loops.

Example STP protection configuration:

```bash
conf t
interface range g0/1 - 2
 spanning-tree portfast
 spanning-tree bpduguard enable
interface g0/24
 spanning-tree guard root
end
```

Useful verification:

```bash
show spanning-tree
show spanning-tree vlan 10
show spanning-tree interface g0/1 detail
```

Wireless networking is another high-yield area. Know how access points (APs) are discovered, how clients associate, and how radio bands differ.

- An AP advertises an **SSID** (service set identifier), which is the network name.
- A **BSSID** is the radio MAC address for a specific AP radio.
- Clients discover APs through beacon frames and probe responses.
- APs register to controllers or cloud systems using discovery methods such as DHCP, DNS, or broadcast depending on the design.

Band selection matters:

- **2.4 GHz** has longer range and better wall penetration but more interference and fewer non-overlapping channels.
- **5 GHz** offers more channels and generally better performance with shorter range.
- **6 GHz** provides additional clean spectrum and lower congestion, but client support and range considerations still matter.

Channel planning should reduce overlap and co-channel interference. In dense environments, use narrower channel reuse and careful power tuning rather than simply increasing transmit power.

**IP Connectivity**

Routing concepts build on addressing and determine how packets move between networks. You should know route sources, administrative distance, and how routers choose the best path.

- **Connected routes** appear when an interface is up and has an IP address.
- **Local routes** represent the exact interface address in the routing table.
- **Static routes** are manually configured.
- **Dynamic routes** are learned through routing protocols.
- **Administrative distance** is the trust value used when multiple sources advertise the same destination.

Common administrative distance values:

- Connected: `0`
- Static: `1`
- EIGRP internal: `90`
- OSPF: `110`
- RIP: `120`

A **floating static route** uses a higher administrative distance than the primary route so it acts as backup.

```bash
ip route 10.10.10.0 255.255.255.0 192.0.2.2 200
```

OSPF is a major CCNA routing topic. Know how to configure it, how neighbors form, and how to verify the database.

- **Wildcard masks** are inverse masks used in OSPF network statements.
- **Router ID** is a 32-bit identifier chosen from the highest loopback IP, then highest active interface IP if no loopback exists.
- **Area 0** is the backbone area.
- All non-backbone areas must connect logically to area 0 in a standard design.

Basic OSPF configuration:

```bash
conf t
router ospf 1
 router-id 1.1.1.1
 passive-interface default
 no passive-interface g0/0
 network 10.1.1.0 0.0.0.255 area 0
 network 10.1.2.0 0.0.0.255 area 0
end
```

Verification commands:

```bash
show ip ospf neighbor
show ip ospf interface
show ip ospf database
show ip route ospf
```

Troubleshooting OSPF often comes down to a few checks:

- Interfaces must be up/up and in the same subnet.
- Hello and dead timers must match on neighbors.
- Area IDs must match on the shared link.
- Authentication, MTU, and network type mismatches can prevent adjacency.
- Passive interfaces stop OSPF hellos on that interface.

**IP Services**

Services support address assignment, name resolution, time synchronization, and logging. These are common exam topics because they affect both user experience and troubleshooting.

DHCP (Dynamic Host Configuration Protocol) can be centralized or relayed.

- A DHCP server assigns IP address, mask, gateway, DNS, and lease time.
- A DHCP relay forwards client broadcasts to a remote server using `ip helper-address`.
- If clients fail to get addresses, check the relay interface, pool exhaustion, exclusions, and server reachability.

Example DHCP server configuration:

```bash
conf t
ip dhcp excluded-address 192.168.10.1 192.168.10.20
ip dhcp pool USERS
 network 192.168.10.0 255.255.255.0
 default-router 192.168.10.1
 dns-server 8.8.8.8
 domain-name example.local
end
```

Example relay configuration:

```bash
interface g0/0
 ip helper-address 192.0.2.10
```

Verification commands:

```bash
show ip dhcp binding
show ip dhcp pool
show ip interface g0/0
```

DNS (Domain Name System) resolves names to IP addresses. On Cisco devices, `ip domain-lookup` enables lookup behavior, and `ip name-server` defines resolvers.

```bash
conf t
ip domain-lookup
ip name-server 8.8.8.8
ip domain-name example.local
end
```

NTP (Network Time Protocol) keeps device clocks aligned, which matters for logs and certificate validation.

- A device can act as an **NTP client** or **NTP server**.
- Use `ntp server` to point a client at a time source.
- Use `ntp master` to make a device a fallback time source.

```bash
conf t
ntp server 192.0.2.50
ntp master 3
end
```

Verification:

```bash
show ntp associations
show ntp status
show clock detail
```

Syslog centralizes event messages.

- Lower severity numbers are more critical.
- Severity `0` is emergency and `7` is debugging.
- Use timestamps so logs are useful during incident analysis.

```bash
conf t
service timestamps log datetime msec
logging host 192.0.2.60
logging trap informational
end
```

Verification:

```bash
show logging
```

SNMP (Simple Network Management Protocol) supports monitoring and management.

- **SNMPv2c** uses community strings and is less secure.
- **SNMPv3** adds authentication and encryption.
- Use SNMP for monitoring, not for general remote administration.

**Security Fundamentals**

Security on the CCNA focuses on protecting management access, limiting lateral movement, and understanding common access control methods.

- **AAA** stands for Authentication, Authorization, and Accounting.
- Authentication verifies identity.
- Authorization determines what the user can do.
- Accounting records what the user did.

Local AAA example:

```bash
conf t
username admin secret Str0ngP@ss
aaa new-model
aaa authentication login default local
line vty 0 4
 login authentication default
 transport input ssh
end
```

Device hardening includes more than SSH.

- Disable unused services where appropriate.
- Use strong passwords and encrypted secrets.
- Restrict management access with ACLs.
- Use banners to warn unauthorized users.
- Prefer SSH over Telnet.

Management-plane access control with an ACL:

```bash
conf t
ip access-list standard MGMT-ONLY
 permit 192.168.10.0 0.0.0.255
 deny any
line vty 0 4
 access-class MGMT-ONLY in
 transport input ssh
end
```

ACL fundamentals are essential.

- **Standard ACLs** match source IP only.
- **Extended ACLs** match source, destination, protocol, and ports.
- ACLs are processed top-down, first match wins.
- An implicit deny exists at the end.

Example extended ACL:

```bash
conf t
ip access-list extended WEB-FILTER
 permit tcp 192.168.10.0 0.0.0.255 host 203.0.113.10 eq 443
 deny ip any any
interface g0/0
 ip access-group WEB-FILTER in
end
```

Port security protects access ports from unauthorized devices.

- Limit the number of MAC addresses on a port.
- Use sticky MAC learning when appropriate.
- Choose a violation mode: `protect`, `restrict`, or `shutdown`.

```bash
conf t
interface g0/1
 switchport mode access
 switchport port-security
 switchport port-security maximum 2
 switchport port-security mac-address sticky
 switchport port-security violation restrict
end
```

Verification:

```bash
show port-security interface g0/1
show port-security address
show mac address-table interface g0/1
```

802.1X provides port-based network access control.

- The supplicant is the client.
- The authenticator is the switch or AP.
- The authentication server is usually RADIUS.
- 802.1X is often paired with AAA and centralized policy.

Wireless security distinctions matter.

- **WPA2-Personal** uses a pre-shared key and suits small environments.
- **WPA2-Enterprise** uses 802.1X and RADIUS for per-user authentication.
- Enterprise security scales better and supports user-based control.

**Automation and Programmability**

Automation topics focus on how devices expose data and how software interacts with network systems. You do not need to be a developer, but you should know the vocabulary.

- **API** (Application Programming Interface) lets software communicate with a device or service.
- **REST** (Representational State Transfer) commonly uses HTTP methods such as `GET`, `POST`, `PUT`, `PATCH`, and `DELETE`.
- **JSON** (JavaScript Object Notation) is a common data format.
- **YANG** models define structured network data.
- **Configuration data** is what you set on the device.
- **Operational data** is what the device reports about current state.

REST method basics:

- `GET` retrieves data.
- `POST` creates a resource or triggers an action.
- `PUT` replaces a resource.
- `PATCH` updates part of a resource.
- `DELETE` removes a resource.

Example REST-style request concept:

```bash
curl -X GET https://device.example.com/restconf/data/ietf-interfaces:interfaces
```

YANG is important because it describes the schema for network configuration and state. RESTCONF and NETCONF often use YANG models to structure requests and responses.

- NETCONF commonly uses XML.
- RESTCONF commonly uses HTTP and JSON or XML.
- Both are used to programmatically manage network devices.

A simple mental model:

- Configuration data answers: “What should the device do?”
- Operational data answers: “What is the device doing right now?”

**Troubleshooting Methodology**

Troubleshooting is a process, not a guess. Start with the symptom, isolate the layer, and verify the most likely failure points first.

- Check physical status before logical configuration.
- Compare expected and actual behavior.
- Change one variable at a time.
- Verify after every fix.

A practical workflow:

1. Identify the scope of the problem.
2. Check interface status and error counters.
3. Verify Layer 2 settings such as VLANs and trunks.
4. Verify Layer 3 addressing and routing.
5. Test name resolution, DHCP, and service reachability.
6. Confirm security controls are not blocking traffic.

Common command interpretations:

- `show interfaces` with increasing input errors, CRC errors, or late collisions can indicate cabling, duplex, or speed issues.
- A trunk mismatch often shows one side as trunk and the other as access, or VLANs missing from the allowed list.
- DHCP failure often appears as an APIPA address on the client or no lease in the binding table.
- OSPF adjacency failure often shows neighbors stuck in `INIT`, `EXSTART`, or `EXCHANGE`.

Examples:

```bash
show interfaces status
show interfaces counters errors
show cdp neighbors detail
show ip interface brief
show ip route
```

For duplex mismatch, look for symptoms such as:

- High CRC errors
- Late collisions
- Slow or intermittent connectivity
- One side forced and the other side auto-negotiated incorrectly

For trunk mismatch, verify:

```bash
show interfaces trunk
show vlan brief
show interfaces switchport
```

For DHCP failure, verify:

```bash
show ip dhcp binding
show ip dhcp pool
show ip interface
```

For OSPF adjacency failure, verify:

```bash
show ip ospf neighbor
show ip ospf interface
show ip protocols
```

**High-Yield Exam Reminders**

These are the details that often separate a correct answer from a close distractor.

- IPv4 uses broadcast; IPv6 uses multicast and does not use broadcast.
- Link-local IPv6 addresses are valid only on the local segment, but they are commonly used as next-hop addresses in routing tables and routing protocol adjacencies.
- A router can have multiple IPv6 addresses on one interface, including link-local and global unicast.
- STP root bridge election depends on bridge ID, which includes priority and MAC address.
- A lower administrative distance wins when routes to the same destination exist from different sources.
- Static routes can be used as backups when you increase their administrative distance.
- ACL placement matters: standard ACLs are usually placed near the destination, extended ACLs near the source.
- Wireless channel planning should reduce overlap rather than maximize power.
