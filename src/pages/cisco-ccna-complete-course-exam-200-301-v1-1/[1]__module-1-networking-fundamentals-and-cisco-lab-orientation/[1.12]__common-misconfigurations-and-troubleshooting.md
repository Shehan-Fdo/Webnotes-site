## **Course:** Cisco CCNA Complete Course (Exam 200-301 v1.1)

**Troubleshooting mindset**

Cisco IOS is the operating system that runs on most Cisco routers and switches, so many issues come from configuration mistakes rather than hardware failure. Good troubleshooting starts by checking the active configuration, interface state, and whether recent changes were saved.

- Use `show running-config` to inspect the active configuration for misconfigurations.
- Use `show ip interface brief` for a fast view of interface status and IP addressing.
- Use `show interfaces` when you need error counters, duplex clues, or physical-layer detail.
- Use `show ip route` to verify whether the device has the expected paths.
- Use `ping <IP>` and `traceroute <IP>` to test reachability and path behavior.

**Common configuration mistakes**

A frequent issue is assuming a change is permanent when it only exists in the running configuration. The running configuration is stored in volatile memory, so it is lost after a reload unless you copy it to the startup configuration.

> If you make a change and do not save it, a reboot can erase the fix and make the problem appear to “come back.”

- `running-config` holds the active settings in memory.
- `startup-config` preserves settings for the next boot.
- Copy the running configuration to startup configuration after verified changes.
- Review interface and routing settings after any maintenance window or VLAN change.

**Saving configuration changes**

When you finish a valid change, save it so the device keeps the configuration after restart. This is a core operational habit for routers and switches.

```bash
copy running-config startup-config
```

- Confirm the change before saving if you are still testing.
- Save after you verify the device behaves as expected.
- Treat unsaved changes as temporary until they are written to startup configuration.

**Access and login problems**

Cisco IOS includes security features for passwords, privileges, and logins, so misconfiguration here can block access to the device. Problems may involve user EXEC mode, privileged EXEC mode, local sessions, or remote CLI sessions.

- Check whether the correct password is configured for the intended access level.
- Verify privilege levels if a user can log in but cannot enter the expected mode.
- Review local and remote CLI session settings when access behaves differently by connection type.
- Be careful with password recovery-related settings, since they can affect how you regain access.

**Practical verification flow**

Use a simple sequence so you do not chase symptoms before confirming the basics. Start with device health, then interfaces, then routing, then end-to-end reachability.

1. Check `show version` for IOS version, uptime, and hardware context.
2. Check `show ip interface brief` for interface state and addressing.
3. Check `show interfaces` for errors, collisions, or duplex mismatch clues.
4. Check `show running-config` for incorrect or missing settings.
5. Check `show ip route` to confirm the routing table matches expectations.
6. Test with `ping <IP>` and `traceroute <IP>` to isolate where traffic stops.

**What to look for in show output**

Show commands are most useful when you know what abnormal output looks like. A down interface, missing IP address, unexpected route, or interface errors can point directly to the fault domain.

- Interface state can reveal cabling, shutdown, or negotiation issues.
- Error counters can suggest physical problems or mismatched duplex settings.
- Routing output can show missing networks or incorrect next hops.
- Configuration output can expose typos, incomplete commands, or forgotten settings.

**Troubleshooting habits that reduce repeat issues**

Good network management is not only about fixing the current fault. It is also about making the configuration predictable, secure, and easy to verify later.

- Keep changes small and verify each one.
- Save only after the configuration is confirmed.
- Recheck the device after reloads or maintenance windows.
- Use consistent verification commands so you can compare results quickly.
