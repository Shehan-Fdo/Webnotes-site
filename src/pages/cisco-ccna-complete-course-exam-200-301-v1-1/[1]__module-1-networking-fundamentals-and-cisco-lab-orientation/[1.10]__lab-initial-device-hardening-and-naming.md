## **Course:** Cisco CCNA Complete Course (Exam 200-301 v1.1)

**Lab focus**

This lab covers the first-pass configuration tasks you use to make a Cisco device easier to identify and safer to manage. The evidence emphasizes basic device configuration, device hardening, and secure administrative access. In practice, that means you set a clear hostname, protect access with passwords, and prepare the device for controlled management.

- **Hostname** changes help you identify the device quickly in the CLI and in network documentation.
- **Device hardening** reduces exposure by securing administrative access and limiting unnecessary services.
- **Secure management** includes using stronger login controls and encrypted remote access when possible.

**Core configuration goals**

The lab materials point to a standard early workflow: configure global settings, secure password access, configure interfaces, and save the configuration. These are foundational tasks before you move on to more advanced routing, switching, or security features.

1. Enter global configuration mode.
2. Set the device name with a meaningful hostname.
3. Configure password protection for administrative access.
4. Bring up and verify interfaces as needed.
5. Save the running configuration so the settings persist after reload.

**Hardening concepts**

Cisco’s IOS XE hardening guidance highlights several management-plane protections that are relevant to initial device setup. The goal is to reduce risk on the device’s administrative surface, especially where users connect through the CLI or remote management sessions.

- **Authentication, authorization, and accounting** help control who can access the device and what they can do.
- **Password management** should be deliberate, with stronger password handling and retry controls where supported.
- **Unused services** should be disabled to reduce attack surface.
- **EXEC timeout** and keepalives help limit idle management sessions.
- **SSHv2** is preferred for encrypted management sessions when remote access is required.

> Use conservative hardening choices when you are unsure which services are required. Keep only the management features you need for the lab or production role.

**Naming and identification**

A hostname is more than a label. It gives you a consistent way to recognize the device in prompts, logs, and troubleshooting output. In a lab, a clear name also helps you avoid confusion when multiple routers or switches are present.

- Choose a hostname that reflects the device role or location.
- Keep the naming pattern consistent across the topology.
- Verify the prompt changes after the hostname is applied.

**Access protection**

The evidence supports securing administrative access as part of the initial setup. That includes protecting console and virtual terminal access, and using stronger methods for remote management where available.

- Set passwords for privileged and line access.
- Control `vty` and `tty` line access as part of management hardening.
- Prefer encrypted management sessions over unencrypted alternatives.
- Add warning banners if your lab or policy requires them.

**Interface and save workflow**

Basic device configuration labs commonly include interface setup and saving the configuration. This ensures the device is not only named and secured, but also operational and persistent across reloads.

```bash
enable
configure terminal
hostname R1
interface fa0/0
no shutdown
end
copy running-config startup-config
```

- Use `no shutdown` to enable an interface after configuration.
- Verify interface status before moving on to the next device.
- Save the configuration once the initial setup is complete.

**Practical lab mindset**

The evidence shows that this lab is meant to build the habits you need for later CCNA work. You are not just typing commands; you are learning to establish a clean baseline for device administration. That baseline makes troubleshooting easier and reduces the chance of leaving a device exposed or inconsistently named.

- Start with identification.
- Secure access early.
- Enable only the services you need.
- Save and verify every change.
