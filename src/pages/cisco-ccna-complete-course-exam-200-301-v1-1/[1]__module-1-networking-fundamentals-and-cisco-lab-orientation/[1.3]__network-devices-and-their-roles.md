**Network device fundamentals**

A **network device** is hardware or software that helps communication between endpoints and network services. These devices move data, connect segments, and can also restrict traffic for security or policy reasons.

Common network devices include:

- **Routers** for connecting different networks and directing packets toward their destination.
- **Switches** for forwarding frames within a local area network (LAN).
- **Hubs** for basic device interconnection, though they are less efficient than switches.
- **Modems** for connecting a local network to an Internet service.
- **Firewalls** for controlling and filtering traffic.
- **Access points** for providing wireless connectivity.

Network devices are the building blocks that let services and endpoints communicate. In practice, a single device may perform more than one role, so the category can be broader than a single function.

**Core roles in a network**

Network devices support three broad functions:

- **Traffic management** by directing data flow and controlling which packets are allowed through.
- **Connectivity** by linking devices, segments, or separate networks.
- **Security** by blocking unauthorized connection attempts and helping maintain a protected environment.

These roles often overlap. For example, a router can connect networks while also influencing traffic paths, and a firewall can both restrict communication and support policy enforcement.

**Common device roles**

| Device | Primary role | Practical note |
|---|---|---|
| Router | Connects networks | Directs traffic between different networks |
| Switch | Connects devices in a LAN | Forwards frames between local devices |
| Hub | Basic connectivity | Simple interconnection device |
| Modem | Internet access link | Connects a local network to an ISP service |
| Firewall | Traffic control | Filters and blocks unauthorized traffic |
| Access point | Wireless access | Extends network access over Wi-Fi |

**How to think about device placement**

When you identify a device in a network diagram, ask what problem it solves:

1. Does it connect separate networks?
2. Does it move traffic inside a LAN?
3. Does it provide Internet access?
4. Does it filter or secure traffic?
5. Does it provide wireless access?

This approach helps you distinguish devices by function instead of by name alone.

**Important exam takeaway**

The same device category can appear in multiple roles, and the lines between categories can be fuzzy. Focus on the main job the device performs in the network path, especially whether it enables communication, manages traffic, or restricts communication.
