## **Course:** Cisco CCNA Complete Course (Exam 200-301 v1.1)

**Cisco IOS CLI hierarchy**

Cisco IOS software organizes commands into a hierarchy. Your current mode determines which commands are available, so navigation matters as much as the command itself. The main security split is between <abbr title="Command-Line Interface">CLI</abbr> sessions at the user EXEC level and the privileged EXEC level.

- **User EXEC** gives you basic monitoring access.
- **Privileged EXEC** gives you access to all router commands.
- Privileged EXEC can be password protected to restrict configuration and management access.
- Command options change as you move deeper into the hierarchy.

> Configuration command options are not available until you navigate to the correct mode.

**EXEC modes**

When you first log in, you land in user EXEC mode. This is the least powerful mode and is mainly used for viewing status and doing basic troubleshooting. The prompt typically shows the device hostname followed by `>`.

You can use `?` to see available commands in the current mode. In user EXEC, common actions include showing system information, testing connectivity, and opening or closing remote sessions.

- `show` displays running system information.
- `ping` sends echo messages.
- `traceroute` traces a route to a destination.
- `enable` turns on privileged commands.
- `disable` turns off privileged commands.

**Privileged EXEC and access control**

Privileged EXEC is the gateway to the rest of IOS command modes. From here, you can reach all command modes used for configuration. Because it exposes full router control, it is commonly protected with a password.

- Use privileged EXEC when you need to manage or configure the device.
- Use user EXEC when you only need limited monitoring.
- Treat privileged access as sensitive because it unlocks all router commands.

**Configuration modes**

Cisco IOS includes several configuration modes, and each one scopes commands to a specific part of the device. The evidence identifies five command modes beyond EXEC access:

- **Global configuration mode**
- **Interface configuration mode**
- **Subinterface configuration mode**
- **Router configuration mode**
- **Line configuration mode**

These modes are hierarchical. You must enter the correct mode before the related configuration commands become available.

**Practical navigation mindset**

Think of IOS as a layered command tree. You start with limited visibility, then move upward only when you need broader control. This helps you avoid entering commands in the wrong context and makes troubleshooting more predictable.

1. Log in and observe the current prompt.
2. Use `?` to inspect the commands available in that mode.
3. Enter `enable` when you need privileged access.
4. Move into the required configuration mode before applying changes.
5. Return to a less privileged mode when you only need monitoring.

**What to remember for exam prep**

- User EXEC is limited to basic monitoring.
- Privileged EXEC provides full router access.
- IOS commands are hierarchical, so mode selection affects command availability.
- Configuration commands are mode-specific, not universal.
- The prompt helps you identify where you are in the CLI.

**Quick prompt cues**

| Mode | Typical prompt cue | Main purpose |
|---|---|---|
| User EXEC | `>` | Basic monitoring and simple troubleshooting |
| Privileged EXEC | `#` | Full access to router commands |
| Configuration modes | varies | Device-specific configuration |
