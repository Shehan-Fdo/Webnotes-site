## **Course:** Cisco CCNA Complete Course (Exam 200-301 v1.1)

**OSI and TCP/IP overview**

The OSI (Open Systems Interconnection) model and the TCP/IP model are layered networking models that help you understand how data moves across a network. They simplify a complex process by breaking communication into functions with distinct responsibilities.

The OSI model is a conceptual, protocol-independent framework. The TCP/IP model is more practical and tied to standardized protocols used for real communication.

- **OSI** focuses on functions and standardization across network communication methods.
- **TCP/IP** focuses on solving communication problems with standardized protocols.
- Both models describe how information is transmitted between devices across a network.

**OSI model layers**

The OSI model has seven layers. From top to bottom, they are:

1. Application
2. Presentation
3. Session
4. Transport
5. Network
6. Data Link
7. Physical

Each layer has a distinct role, and information is exchanged between layers using protocol data units (PDUs). The model is useful for understanding and designing network communication, even though it is not a direct implementation of a protocol stack.

- **Application** connects directly to end-user applications and services.
- **Presentation** translates data between formats.
- **Session** manages communication sessions.
- **Transport** handles end-to-end transport functions.
- **Network** supports internetworking and logical addressing.
- **Data Link** handles local network delivery.
- **Physical** sends raw bits over the medium.

**TCP/IP model layers**

The TCP/IP model is commonly described with four layers in modern networking references:

1. Application
2. Transport
3. Internet
4. Network Interface

Some sources describe additional sublayers or variations, but the practical CCNA view centers on these four layers. TCP/IP is the Internet Protocol Suite and is used as a communications model for real networks.

- **Application** covers user-facing services and higher-level communication.
- **Transport** manages data delivery between hosts.
- **Internet** handles internetworking and routing.
- **Network Interface** covers link-level delivery on the local network.

> The TCP/IP model may appear simpler than OSI because several OSI functions are combined into fewer layers.

**Key differences**

The two models serve different purposes. OSI is a reference model for understanding and standardizing network functions, while TCP/IP is a practical model built around working protocols.

| Aspect | OSI | TCP/IP |
|---|---|---|
| Purpose | Conceptual reference model | Practical protocol model |
| Layer count | 7 | 4 commonly used, sometimes described differently in sources |
| Focus | Functions of communication | Standardized protocol communication |
| Use | Learning, design, troubleshooting | Real-world networking |

A useful exam takeaway is that OSI gives you a detailed vocabulary for network functions, while TCP/IP reflects how networks are commonly implemented.

**Layer mapping**

You should be able to relate the models conceptually, even when the exact layer names differ across references.

- OSI `Application`, `Presentation`, and `Session` map to TCP/IP `Application`.
- OSI `Transport` maps to TCP/IP `Transport`.
- OSI `Network` maps to TCP/IP `Internet`.
- OSI `Data Link` and `Physical` map to TCP/IP `Network Interface`.

This mapping helps you reason about where a protocol or function belongs when studying troubleshooting and packet flow.

**Why layered models matter**

Layered models make it easier to understand how protocols interact. They show how data moves from high-level applications down to signals on a wire and back again.

- They separate complex communication into manageable parts.
- They help you identify where a function belongs.
- They support troubleshooting by narrowing problems to a specific layer.
- They provide a shared language for networking concepts.

**CCNA study focus**

For CCNA, you should know the layer names, the general purpose of each layer, and the relationship between OSI and TCP/IP. You should also recognize that OSI is a conceptual framework and TCP/IP is the practical model used in modern networking.

- Memorize the OSI layer order from `Application` to `Physical`.
- Know that TCP/IP is usually shown with four layers.
- Understand that OSI is protocol-independent.
- Use the model mapping to place networking functions correctly.
