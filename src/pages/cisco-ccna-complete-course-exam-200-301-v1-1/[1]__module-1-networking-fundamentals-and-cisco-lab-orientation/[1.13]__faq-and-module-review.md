## **Course:** Cisco CCNA Complete Course (Exam 200-301 v1.1)

**Core model review**

The OSI (Open Systems Interconnection) model and the TCP/IP model are foundational networking references. OSI was created by the International Organization for Standardization (ISO) as a reference model for describing communication functions, while TCP/IP is the practical model used in modern networks.

The OSI model has seven layers. It helps you describe how applications communicate across network media and is a common exam topic. A useful mnemonic for the layers is `Please Do Not Throw Sausage Pizza Away`.

- **OSI** gives you a layered framework for understanding network communication.
- **TCP/IP** is the model you use to think about real-world protocol interaction.
- The OSI layers are commonly numbered from the bottom up, with Physical as layer 1.
- Memorizing the layer order helps you answer comparison and encapsulation questions quickly.

**Encapsulation and layer behavior**

Encapsulation is the process of adding control information as data moves through the layers. The transport layer segments data into smaller pieces and may add reliability and flow control information. In TCP, acknowledgment-based flow control helps the sender pace transmission.

The network layer adds Layer 3 addresses and control information to a segment, creating a packet. The data link layer adds physical source and destination addresses plus a frame check sequence (FCS) to the packet, creating a frame.

- **Transport layer** handles segmentation and can support reliability and flow control.
- **Network layer** adds Layer 3 addressing.
- **Data link layer** adds Layer 2 addressing and FCS.
- **Physical layer** is responsible for transmission across the medium, not addressing or segmentation.

> Be careful with layer questions: the same data unit changes names as it moves through the stack, and exam items often test the exact layer action.

**Running configuration vs startup configuration**

Cisco devices store configuration in two places: running configuration and startup configuration. The running configuration is stored in RAM, so changes are lost if the device loses power. The startup configuration is stored in nonvolatile memory, so it remains after a reboot.

When you enter commands in global configuration mode, they are saved to the running configuration immediately. To preserve them across restarts, copy the running configuration to the startup configuration.

```bash
copy running-config startup-config
```

- **Running configuration** is active and stored in RAM.
- **Startup configuration** is saved in nonvolatile memory.
- Copying the running config to startup config preserves changes after power loss.
- If you forget to save, the device can reboot without your latest changes.

**Quick review checklist**

Use this section to test whether you can explain the topic without looking at notes.

1. State the difference between the OSI model and the TCP/IP model.
2. Identify what the transport, network, and data link layers add during encapsulation.
3. Explain where running configuration and startup configuration are stored.
4. Recall the command used to save the current configuration.

**Common exam reminders**

- Know the seven OSI layers and their order.
- Match each layer with its main function before answering multiple-choice questions.
- Remember that `copy running-config startup-config` saves the current configuration.
- Use conservative wording when unsure: focus on the layer action that is explicitly described.
