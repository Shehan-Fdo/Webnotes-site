## **Course:** CompTIA Security+ Applied Defense (Exam SY0-701)

**Public intelligence collection with theHarvester**

theHarvester is an open-source intelligence (<abbr title="Open Source Intelligence">OSINT</abbr>) tool used during the reconnaissance stage. It gathers information about a target domain from public sources and helps you map exposed assets before deeper testing begins.

The tool is designed to be simple to use while still being effective for collecting data that can support attack surface discovery and threat analysis.

- Collects **email addresses** associated with a domain.
- Discovers **subdomains** and **hosts** from public sources.
- Can identify **names** tied to an organization.
- Supports reconnaissance with sources such as search engines, PGP key servers, and Shodan-style public data sources.

**What you use it for**

Use theHarvester when you need a quick view of what a target organization exposes publicly. In practice, it helps you find likely entry points, understand naming patterns, and identify assets that may not be obvious from a simple domain lookup.

- **Email enumeration** helps you identify possible user accounts and social engineering targets.
- **Subdomain discovery** helps you expand the visible attack surface.
- **Host discovery** helps you spot systems that may deserve follow-up validation.
- **Public-source aggregation** reduces the need to query each source manually.

> Use theHarvester for reconnaissance only and stay within authorized testing scope.

**Common usage pattern**

A typical workflow is to choose a target domain, select a source, and limit the result set so you can review the output efficiently. Kali Linux includes the tool and shows a usage example that searches DuckDuckGo for `kali.org` with a result limit.

```bash
theHarvester -d kali.org -l 500 -b duckduckgo
```

- `-d` sets the target domain.
- `-l` limits the number of results.
- `-b` selects the data source or search engine.

The output can include counts or lists for emails, IPs, and hosts. In the Kali example, the search returns no IPs or emails but does find hosts, which shows that results vary by source and target.

**Operational notes**

theHarvester continues to evolve, and release notes show support changes such as socks proxy support and API key handling for some data sources. That means you should expect source availability and output to vary over time.

- Newer releases may add or adjust data-source integrations.
- Some features depend on external APIs or keys.
- Proxy support can matter in controlled lab or routed environments.

**Exam focus**

For Security+ purposes, focus on the role of theHarvester in the reconnaissance phase rather than memorizing every source plugin. You should be able to explain what it collects, why it is useful, and how it supports public intelligence gathering.

- It is an **OSINT** tool.
- It supports **reconnaissance** before exploitation.
- It helps collect publicly available contact and infrastructure data.
- It is useful for mapping a target’s visible footprint.
