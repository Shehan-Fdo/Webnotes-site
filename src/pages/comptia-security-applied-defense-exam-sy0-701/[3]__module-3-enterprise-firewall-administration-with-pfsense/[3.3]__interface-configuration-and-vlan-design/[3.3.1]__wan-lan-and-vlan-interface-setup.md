## **Course:** CompTIA Security+ Applied Defense (Exam SY0-701)

**Interface roles and segmentation**

A firewall or router often sits at the boundary between outside and inside networks, and it can also act as a router or Layer 3 device. In pfSense, interface setup is part of how you separate traffic into WAN, LAN, and VLAN segments so each network path is handled intentionally.

A **VLAN** (Virtual Local Area Network) is a logical grouping of hosts and resources that behaves like a separate subnet or broadcast domain. That separation helps you segment traffic without requiring a separate physical network for every group.

- **WAN** is the external-facing path where traffic enters or leaves the network boundary.
- **LAN** is the internal network segment for trusted local systems.
- **VLANs** let you divide a physical network into multiple logical networks.
- Traffic within a VLAN is switched only among ports in that VLAN.

**VLAN configuration basics in pfSense**

When you create or edit a VLAN in the pfSense GUI, you define how tagged traffic maps to a physical interface. The physical interface carries the VLAN traffic, and the tag settings determine how frames are identified on the wire.

- **Parent Interface**: the physical interface that carries the VLAN traffic.
- **VLAN Tag Type**: the tag protocol identifier used in the packet.
- **VLAN Tag**: the VLAN identifier, or VID, in the range `1-4094`.
- **VLAN Priority**: the 802.1p Priority Code Point (PCP) value, from `0-7`.

The tag type matters when you work with standard VLANs or double-tagged traffic. In typical 802.1Q setups, the customer tag is the common choice.

- **C-Tag** uses value `0x8100` and is the default for typical 802.1Q VLAN interfaces.
- **S-Tag** uses value `0x88a8` and is used as the outer tag in double-tagging scenarios such as 802.1ad QinQ.
- If there is only one layer of VLAN tagging, `C-Tag` is usually the correct choice.

**Practical setup considerations**

You should treat interface setup as part of network hardening and segmentation. A newly deployed system is typically secured by reducing unnecessary exposure first, then applying network controls such as firewall rules and VLAN placement where needed.

- Close unnecessary service ports before exposing a system broadly.
- Use firewall rules to control traffic between segments.
- Place systems into the correct VLAN when you need logical separation.
- Use host-based controls alongside network segmentation for layered defense.

> If you use double tagging, be careful to match the tag type to the role of the frame. A mismatch can cause traffic to be classified incorrectly.

**Quick reference**

| Item | Purpose | Value or range |
|---|---|---|
| Parent Interface | Physical carrier for the VLAN | Physical interface |
| VLAN Tag Type | Identifies tag protocol | `C-Tag` or `S-Tag` |
| VLAN Tag | VLAN identifier | `1-4094` |
| VLAN Priority | Packet priority marking | `0-7` |

**Review points**

- A firewall can sit inline and make allow or deny decisions for traffic.
- VLANs create logical separation even when devices share the same physical infrastructure.
- pfSense VLAN setup centers on the parent interface, tag type, tag value, and priority.
- `C-Tag` is the common single-tag option; `S-Tag` supports outer tagging in QinQ-style designs.
