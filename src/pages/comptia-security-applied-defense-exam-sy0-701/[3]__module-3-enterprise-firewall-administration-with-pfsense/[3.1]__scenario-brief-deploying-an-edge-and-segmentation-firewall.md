## **Course:** CompTIA Security+ Applied Defense (Exam SY0-701)

**Scenario focus**

An edge firewall sits at the ingress and egress point where the outside network meets the inside network. In Security+ terms, you use it to make allow-or-deny decisions for traffic and to reduce exposure before traffic reaches internal systems.

A segmentation firewall adds another control layer inside the network. The practical goal is to separate systems so a compromise in one area does not automatically spread to others. This is especially important when a new system is added and ports are opened for it.

- **Edge firewall** placement matters because it protects the boundary between external and internal networks.
- **Segmentation** limits lateral movement by keeping traffic paths narrower.
- **Firewall rules** should reflect the system’s role and the traffic it actually needs.
- **Non-segmented network** design increases the risk that one exposed system can affect others.

**What firewalls do in this scenario**

A network-based firewall is an appliance that sits inline and evaluates traffic. It can make decisions based on ports, applications, or both, depending on the firewall type. Traditional firewalls focus on ports, while next-generation firewalls can inspect applications as well.

Firewalls can also provide more than filtering. In many deployments, they act as routers or layer 3 devices, and they may serve as VPN endpoints or VPN concentrators. That makes them useful at the network boundary and between internal zones.

- **Traditional firewall** decisions often center on port numbers.
- **Next-generation firewall** decisions can include application awareness.
- **Router** functionality may be part of the same device.
- **VPN** support can extend secure connectivity for remote access or site-to-site links.

**Deployment and lab considerations**

Training and lab environments commonly use pfSense as a firewall platform because it supports practical exercises around deployment, interface assignment, NAT, and rule creation. In a lab, you may work with a WAN side, a LAN side, and one or more internal subnets to model segmentation.

A realistic setup tracks addresses carefully so each zone has a clear role. The firewall interface on the outside network and the internal client or server networks should be documented before you test connectivity.

- **WAN** identifies the external-facing interface.
- **LAN** identifies the protected internal side.
- **Subnet planning** helps you keep zones separate and easier to manage.
- **Interface assignment** is a first step before rule testing.

**Common configuration tasks**

When you deploy a firewall in a scenario like this, you usually start by assigning interfaces, then define NAT and firewall rules, and finally verify traffic flow. If the firewall is part of a lab scenario, you may also need to start the virtual environment and wait for the firewall to finish booting before testing.

1. Assign the firewall interfaces for the external and internal networks.
2. Define the required NAT and firewall rules for the intended traffic.
3. Verify that only approved traffic passes between zones.
4. Test that blocked traffic does not cross the segmentation boundary.

> Opening ports for a new system can expose a **non-segmented network** risk if the system is not isolated from other internal assets.

**How to think about the risk**

If you open firewall ports for a new SaaS-supported system, the main concern is not just the port itself. The larger issue is whether that system sits in a segmented network or blends into the rest of the environment. Without segmentation, the new system can become a path for broader access.

This is why firewall design and network design work together. The firewall enforces policy, but segmentation defines the blast radius if something goes wrong.

- **Attack surface** grows when unnecessary ports are opened.
- **Lateral movement** becomes easier when internal zones are not separated.
- **Policy enforcement** is stronger when each zone has a clear purpose.
- **Containment** improves when the firewall supports distinct network segments.

**Key takeaways**

An edge firewall protects the boundary, while a segmentation firewall protects the interior. Together, they help you control traffic, support routing and VPN use cases, and reduce the chance that one compromised system affects the rest of the network.

- Place the firewall where it can inspect traffic entering and leaving the environment.
- Separate internal systems into zones when you need stronger containment.
- Open only the ports required for the service.
- Validate that the network design supports the firewall policy.
