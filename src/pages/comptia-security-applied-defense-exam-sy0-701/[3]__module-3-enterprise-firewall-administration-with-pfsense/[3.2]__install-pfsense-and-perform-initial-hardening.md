## **Course:** CompTIA Security+ Applied Defense (Exam SY0-701)

**pfSense installation overview**

pfSense is a firewall and router platform based on FreeBSD. In practice, you install it from Netgate installation media, then complete the installer prompts to place the software on a target drive such as an SSD or HDD. The installer is designed to guide you through the process with a small set of clear choices.

During installation, you should expect the following flow:

- Accept the license terms before continuing.
- Use the welcome menu to choose whether to install, open a rescue shell, or restore a configuration.
- Use the rescue shell only when you need advanced preparation, diagnostics, or repair tasks.
- Use configuration restore when you already have a valid saved configuration from a prior installation or other media.

> If the installer reports boot or media errors, treat the installation media and target hardware as the first items to verify.

**Installer interaction and recovery options**

The installer uses keyboard navigation rather than mouse input. You move through the license text and menus with standard keys, then confirm selections to proceed. This matters in lab and recovery scenarios because you may need to work without a fully configured web interface.

Key installer behaviors include:

- Read the license text carefully and page through it as needed.
- Press `Enter` to accept the license and continue.
- Use the arrow keys to select menu options.
- Choose the rescue shell when you need a basic command prompt for preparation or troubleshooting.
- Choose configuration restore when you want to recover a prior configuration during installation.

**Initial hardening goals**

Initial hardening starts as soon as pfSense is installed and reachable. The goal is to reduce unnecessary exposure and make administration safer. A firewall is a control point for traffic, so hardening focuses on limiting access to the management plane and removing avoidable risk.

Practical hardening priorities from the evidence include:

- Keep the firewall software updated.
- Change default passwords.
- Enable two-factor authentication where available.
- Disable unused services.
- Limit administrative access.
- Protect backup files.
- Disable SSH if you do not need it.
- Use regular backups for recovery planning.

**Secure administrative access**

The administrative interface is controlled from `System > Advanced > Admin Access`. This area governs access through the web interface, SSH, serial, and the physical console. The most important choice for the web GUI is the protocol.

Use these principles:

- Prefer **HTTPS** for the webConfigurator so traffic is encrypted.
- Avoid plain **HTTP** except in limited compatibility scenarios.
- Remember that HTTPS requires an SSL/TLS certificate.
- Expect browser warnings if you use a self-signed certificate.
- Treat insecure networks as a reason to avoid exposing the GUI over HTTP.

**Hardening checklist**

1. Complete the installation and confirm the firewall boots successfully.
2. Open the administrative settings and verify the GUI uses HTTPS.
3. Review which access methods are enabled for administration.
4. Remove or disable services you do not need.
5. Change any default credentials immediately.
6. Enable stronger authentication controls such as two-factor authentication if supported.
7. Create a backup and store it securely.
8. Recheck the system after updates or configuration changes.

**Why this matters for Security+**

For exam purposes, pfSense is a concrete example of firewall hardening in action. The same defensive ideas apply broadly: reduce attack surface, secure management access, and maintain recoverability. A hardened firewall is more resilient because it exposes fewer unnecessary paths for misuse.
