## **Course:** CompTIA Security+ Applied Defense (Exam SY0-701)

**Credential theft overview**

Credential theft is a high-impact attack path because it targets the identity layer instead of the device layer. Attackers use stolen usernames, passwords, tokens, or app-based grants to sign in as legitimate users and blend into normal activity.

A key shift is the move from password theft to **token theft**. Tokens can bypass MFA (multi-factor authentication) because they act as bearer credentials: whoever has the token can use it. That means a valid token can grant access without triggering a new login prompt or a failed password event.

- Credential theft is a top breach vector and often starts with stolen access data.
- Token theft can abuse OAuth, session, and API tokens.
- Valid-looking sign-ins can evade basic defenses because activity appears legitimate.
- Identity compromise can affect email, VPN, SSO (single sign-on), cloud apps, and service accounts.

**Common attack patterns**

Attackers often combine phishing, infostealers, and session hijacking to obtain usable credentials. The goal is not always to crack a password; it is to capture something that already authenticates.

Typical patterns include:

- **Phishing** to capture passwords or session data.
- **Infostealers** that harvest browser-stored credentials and tokens.
- **Session hijacking** that reuses an active authenticated session.
- **API token abuse** that grants programmatic access to services.
- **SSO abuse** that leverages trusted identity flows to move laterally.

> Traditional MFA alone is not enough when the attacker steals a live token or bypasses push-based approval in real time. Phishing-resistant MFA such as FIDO2/passkeys provides stronger protection.

**Detection signals**

Identity compromise detection depends on behavior, not just static indicators. Security teams look for unusual login patterns, abnormal access requests, and suspicious data usage.

Useful signals include:

- Logins from unusual geographic locations.
- Access attempts outside normal working hours.
- Sudden changes in account behavior or access scope.
- Unexpected use of sensitive files or cloud resources.
- Repeated authentication activity that does not match the user’s baseline.

Detection commonly uses:

- **SIEM** (Security Information and Event Management) for log correlation.
- **UEBA** (User and Entity Behavior Analytics) for behavior anomalies.
- **IGA** (Identity Governance and Administration) for access governance.
- Identity threat detection and response tools for continuous monitoring.

**First response priorities**

When you suspect credential compromise, act in the right order. Revoke active access first, then rotate secrets. This prevents the attacker from keeping a live session while you change passwords.

1. Assign roles before you start.
   - Incident Commander owns decisions and timeline.
   - IAM (Identity and Access Management) Lead handles IdP, SSO, MFA, password resets, and token revocation.
   - Forensics Lead preserves evidence and scopes the incident.
   - Threat Hunting and Detection tune queries and hunt for related activity.
   - Comms Lead coordinates executive, legal, privacy, and customer notifications.
2. Revoke sessions and tokens first.
   - Remove active bearer access before changing credentials.
3. Rotate passwords and keys.
   - Reset affected accounts and service credentials after session revocation.
4. Contain endpoints and accounts.
   - Isolate systems if the compromise includes endpoint activity.
5. Preserve evidence.
   - Keep logs, timestamps, and relevant artifacts for investigation.
6. Scope the blast radius.
   - Check for lateral movement, privilege escalation, and additional compromised identities.

**Why speed matters**

Credential-based breaches can be expensive and slow to detect. The longer an attacker remains authenticated, the more time they have to access data, escalate privileges, and blend into normal operations.

A practical response plan focuses on:

- Early containment.
- Clear reset sequencing.
- Evidence preservation.
- Targeted communications.
- Continuous monitoring after remediation.

**Defensive controls**

Strong defense combines identity controls, monitoring, and architecture that assumes credentials may be stolen.

Prioritize:

- **Phishing-resistant MFA** such as FIDO2/passkeys.
- Continuous credential monitoring across network, identity, and cloud.
- Behavioral analytics for abnormal access patterns.
- Zero trust principles that limit implicit trust.
- Identity-centric response playbooks for rapid containment.

**Exam focus reminders**

For exam prep, remember the relationship between credential theft and identity trust:

- Stolen credentials can authenticate as a legitimate user.
- Tokens can bypass password-based defenses.
- Detection relies on behavior and context, not only on static indicators.
- Response should revoke sessions before rotating passwords and keys.
- Modern defense depends on layered identity monitoring and phishing-resistant authentication.

**Quick comparison**

| Area | What to watch | Why it matters |
|---|---|---|
| Password theft | Stolen login secrets | Enables direct account access |
| Token theft | OAuth, session, API tokens | Can bypass MFA and reuse trust |
| Detection | Behavior anomalies | Finds misuse that looks legitimate |
| Response | Revoke then rotate | Stops active attacker sessions first |
