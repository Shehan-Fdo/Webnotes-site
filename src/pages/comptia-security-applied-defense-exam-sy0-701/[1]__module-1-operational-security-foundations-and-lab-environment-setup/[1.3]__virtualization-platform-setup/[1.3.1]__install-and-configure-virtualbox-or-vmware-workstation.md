## **Course:** CompTIA Security+ Applied Defense (Exam SY0-701)

**Virtualization setup for a Security+ lab**

Virtual machines let you build a safe practice environment for Security+ objectives without relying on physical hardware. VirtualBox and VMware Workstation both support lab workflows where you can isolate systems, test network behavior, and restore a known state after changes.

For exam prep, focus on two practical ideas:

- **Virtual networking** controls how a VM reaches the host, other VMs, and external resources.
- **Snapshots** let you preserve a VM state and return to it later.

**VirtualBox installation and network configuration**

VirtualBox supports up to eight virtual PCI Ethernet cards per virtual machine. You can configure four of them in the graphical settings and all eight from the command line with `VBoxManage modifyvm`. The key exam takeaway is that each adapter can use a different networking mode depending on the lab goal.

VirtualBox networking modes called out in the evidence include:

- **NAT**: the default mode, suitable for most communication between VMs and external networks.
- **Host-only**: communication stays on the host machine and does not reach external networks.
- **Bridged**: connects the VM more directly to the physical network.
- **Internal networking**: keeps traffic between VMs on the same internal network.
- **Cloud**: connects to Oracle Cloud Infrastructure.

When you configure VirtualBox, use the Network tool in VirtualBox Manager to create and manage the networks your VMs use. You also need to enable the virtual networks for each VM you create.

- Use `Settings` for per-VM adapter configuration.
- Use `VBoxManage modifyvm` when you need command-line control over all eight adapters.
- Match the network mode to the lab purpose before you start testing.

> VirtualBox notes in the evidence emphasize that a VM can have multiple adapters, but each one still needs to be enabled and assigned intentionally.

**VMware Workstation snapshots and lab recovery**

A snapshot preserves the state of a virtual machine at a specific moment in time while the VM continues to run. You can take a snapshot while the VM is powered on, powered off, or suspended. This is useful when you want to test a change and then return to a known-good state.

Snapshot behavior to remember:

- A revert returns the VM’s **memory**, **settings**, and **virtual disks** to the saved state.
- You cannot take a snapshot of a VM that uses a physical disk.
- If the VM has multiple disks in different disk modes, power it off before taking the snapshot.
- For better performance, defragment the guest operating system drives before snapshot-heavy work.

A practical workflow is to snapshot before risky changes, test the change, and revert if the result is not what you want.

1. Prepare the VM in the state you want to preserve.
2. Verify it is not using a physical disk.
3. Take the snapshot.
4. Make your change.
5. Revert to the snapshot if needed.

**Choosing the right network mode**

Use the network mode that matches the isolation level you need. This is especially important in a security lab where you may want to separate internal test systems from external traffic.

| Mode | Practical use |
|---|---|
| `NAT` | Default choice for general external connectivity |
| `Host-only` | Isolated communication with the host machine |
| `Bridged` | VM appears on the physical network |
| `Internal networking` | VM-to-VM communication only |
| `Cloud` | Connection to Oracle Cloud Infrastructure |

**Lab setup habits that reduce mistakes**

A stable lab starts with deliberate configuration. Keep the VM’s adapter count, network mode, and snapshot state aligned with the exercise you are running. If you change the lab purpose, update the network settings before you begin.

- Keep one VM state for baseline testing.
- Use snapshots before configuration changes.
- Prefer isolated modes when you do not need external access.
- Confirm the VM’s network path before troubleshooting application behavior.

**Key exam reminders**

- VirtualBox supports up to eight virtual network adapters per VM.
- `NAT` is the default VirtualBox networking mode.
- `Host-only` keeps traffic on the host and away from external networks.
- VMware snapshots preserve VM state and support rollback.
- A VM using a physical disk cannot be snapshotted.
