## **Course:** Red Hat Certified System Administrator Complete Course (Exam EX200 – RHCSA)

**Overview**

These commands are the backbone of shell-based text manipulation on Linux. You use them to sort records, remove duplicates, extract fields, translate characters, merge columns, and compare files. In RHCSA work, they are especially useful when you need to inspect logs, validate configuration changes, or transform command output into something easier to read.

Most of these tools operate on standard input and standard output, which makes them easy to combine with pipes. That design is important: you can build small, reliable command chains instead of writing custom scripts for simple data tasks.

- **`sort`** orders lines of text.
- **`uniq`** collapses adjacent duplicate lines.
- **`cut`** extracts selected columns or character ranges.
- **`tr`** translates or deletes characters.
- **`paste`** combines lines from multiple files side by side.
- **`diff`** shows line-by-line differences between files.
- **`cmp`** compares files byte by byte.

**`sort` for ordering and grouping**

`sort` arranges input lines in a defined order. By default, it sorts lexicographically using the current locale. This matters because plain alphabetical order is not always the same as numeric order, and locale settings can affect collation.

Use `sort` when you need to prepare data for `uniq`, generate predictable output, or inspect records in a stable order.

```bash
sort /etc/passwd
sort -r /etc/passwd
sort -n numbers.txt
sort -u names.txt
sort -k 3,3 -t: /etc/passwd
```

Common options:

- **`-n`** sorts numerically instead of lexicographically.
- **`-r`** reverses the sort order.
- **`-u`** removes duplicate lines after sorting.
- **`-k`** selects a key field for sorting.
- **`-t`** sets the field delimiter.
- **`-h`** sorts human-readable numbers such as `1K`, `2M`, and `3G`.
- **`-M`** sorts by month name.
- **`-V`** sorts version-like strings naturally.

Practical examples:

```bash
# Sort usernames from /etc/passwd by login name
sort -t: -k1,1 /etc/passwd

# Sort passwd entries by UID numerically
sort -t: -k3,3n /etc/passwd

# Sort file sizes in human-readable form
ls -lh | sort -k5,5h
```

A useful pattern is to sort before deduplicating:

```bash
sort access.log | uniq
```

That works because `uniq` only removes adjacent duplicates.

**`uniq` for adjacent duplicates**

`uniq` filters repeated adjacent lines. It does not search the whole file for duplicates; it only compares each line to the one before it. That is why `sort | uniq` is such a common combination.

Use `uniq` when you want to collapse repeated output, count repeated records, or show only unique or duplicate lines after sorting.

```bash
uniq file.txt
sort file.txt | uniq
sort file.txt | uniq -c
sort file.txt | uniq -d
sort file.txt | uniq -u
```

Common options:

- **`-c`** prefixes each line with the number of occurrences.
- **`-d`** prints only duplicate lines.
- **`-u`** prints only lines that appear once.
- **`-i`** ignores case when comparing.
- **`-f N`** skips the first `N` fields.
- **`-s N`** skips the first `N` characters.

Example workflow:

```bash
# Count repeated IP addresses in a log extract
awk '{print $1}' access.log | sort | uniq -c | sort -nr
```

Important behavior:

> `uniq` is not a general duplicate finder. If identical lines are separated by other lines, `uniq` will not collapse them unless you sort first.

**`cut` for extracting fields and character ranges**

`cut` extracts portions of each line. You use it most often with delimited text such as colon-separated files, CSV-like output, or tab-separated data. It can also extract fixed character positions.

Use `cut` when you need one or more fields from structured text and you know the delimiter.

```bash
cut -d: -f1 /etc/passwd
cut -d: -f1,3 /etc/passwd
cut -c1-10 file.txt
```

Common options:

- **`-d`** sets the delimiter character.
- **`-f`** selects fields by number.
- **`-c`** selects character positions.
- **`--complement`** selects everything except the chosen fields or characters.
- **`-s`** suppresses lines without the delimiter.

Examples:

```bash
# Show usernames and shells from /etc/passwd
cut -d: -f1,7 /etc/passwd

# Show only the first 8 characters of each line
cut -c1-8 report.txt

# Remove the first field from colon-delimited data
cut -d: --complement -f1 /etc/passwd
```

Notes on limitations:

- `cut` is simple and fast, but it is not a full parser.
- It does not understand quoted CSV fields reliably.
- For complex column extraction, `awk` is often more flexible.

**`tr` for translating, squeezing, and deleting characters**

`tr` works on characters, not lines or fields. It is useful for case conversion, character replacement, and removing unwanted characters from streams. Because it reads from standard input, you usually combine it with pipes or redirection.

```bash
echo "hello" | tr 'a-z' 'A-Z'
echo "a,b,c" | tr ',' '\n'
echo "abc123" | tr -d '0-9'
```

Common uses:

- **Translation** maps one set of characters to another.
- **Deletion** removes characters.
- **Squeezing** collapses repeated characters into one.

Examples:

```bash
# Convert lowercase to uppercase
tr '[:lower:]' '[:upper:]' < input.txt

# Replace spaces with tabs
tr ' ' '\t' < file.txt

# Delete digits
tr -d '0-9' < file.txt

# Collapse repeated spaces
tr -s ' ' < file.txt
```

Character classes are often safer than literal ranges:

```bash
tr '[:lower:]' '[:upper:]'
tr -d '[:cntrl:]'
```

Practical use cases:

- Normalize text before comparison.
- Clean noisy output from scripts.
- Convert delimiters for simple pipelines.

> `tr` operates on bytes/characters, not on words or fields. It cannot replace a substring like `foo` with `bar`; it only maps individual characters.

**`paste` for combining lines from files**

`paste` merges lines from one or more files horizontally. It is useful when you want to align related data from separate sources into columns. By default, it uses tabs as separators.

```bash
paste file1 file2
paste -d, file1 file2
paste -s file1
```

Common options:

- **`-d`** sets the delimiter used between pasted columns.
- **`-s`** serializes lines from each file into one line at a time.

Examples:

```bash
# Combine usernames and shells from two command outputs
cut -d: -f1 /etc/passwd > users.txt
cut -d: -f7 /etc/passwd > shells.txt
paste users.txt shells.txt

# Use commas instead of tabs
paste -d, users.txt shells.txt

# Turn each file into a single line
paste -s -d' ' file.txt
```

A common pattern is to pair `paste` with process substitution or command substitution when you need quick side-by-side output.

```bash
paste <(cut -d: -f1 /etc/passwd) <(cut -d: -f7 /etc/passwd)
```

**`diff` for line-by-line comparison**

`diff` compares files and reports textual differences. It is one of the most important tools for validating configuration changes because it shows exactly what changed between two versions.

Use `diff` when you want to compare text files, review edits, or confirm that a configuration file matches a known-good version.

```bash
diff file1 file2
diff -u file1 file2
diff -y file1 file2
```

Common output styles:

- **Default diff** shows edit commands and line numbers.
- **Unified diff** with `-u` is easier to read and is the format commonly used in patches.
- **Side-by-side** with `-y` helps visually compare two files.

Examples:

```bash
# Compare two configuration files
diff /etc/ssh/sshd_config /root/sshd_config.new

# Unified format for review
diff -u /etc/fstab /root/fstab.new

# Ignore whitespace differences
diff -w file1 file2
```

Useful options:

- **`-u`** produces unified output.
- **`-c`** produces context output.
- **`-y`** shows side-by-side comparison.
- **`-w`** ignores all whitespace.
- **`-B`** ignores blank-line changes.
- **`-q`** reports only whether files differ.
- **`-r`** compares directories recursively.

Example patch workflow:

```bash
diff -u /etc/ssh/sshd_config /root/sshd_config.new > /root/sshd_config.patch
```

You can then inspect the patch before applying it.

> `diff` is for text files. It is not ideal for binary files or for proving that two files are identical.

**`cmp` for byte-level comparison**

`cmp` compares files byte by byte. It is faster and simpler than `diff` when you only need to know whether two files are identical. It is especially useful for binary files, images, archives, and exact file validation.

```bash
cmp file1 file2
cmp -s file1 file2
cmp -l file1 file2
```

Common options:

- **`-s`** suppresses output and returns only the exit status.
- **`-l`** lists differing byte positions and values.

Examples:

```bash
# Check whether two files are identical
cmp /usr/bin/ls /tmp/ls.copy

# Use in scripts
if cmp -s config.old config.new; then
  echo "No changes"
else
  echo "Files differ"
fi
```

Behavior to remember:

- `cmp` stops at the first difference unless you use `-l`.
- It is exact, so even a single byte difference matters.
- It does not interpret lines or whitespace.

**Choosing the right command**

Different tasks call for different tools. The key is to match the command to the data shape and the question you are asking.

| Command | Best for | Key limitation |
|---|---|---|
| `sort` | Ordering lines and preparing data for `uniq` | Does not remove duplicates unless asked |
| `uniq` | Collapsing adjacent duplicates | Requires sorted input for global deduplication |
| `cut` | Extracting fields or fixed columns | Weak for quoted or complex formats |
| `tr` | Character translation or deletion | Cannot handle substrings or fields |
| `paste` | Combining parallel lines into columns | Assumes line counts align reasonably |
| `diff` | Human-readable text comparison | Not ideal for binary files |
| `cmp` | Exact byte comparison | Does not show semantic line differences |

**Common pipelines**

These commands become more powerful when chained together. In RHCSA-style tasks, you often combine them with `grep`, `awk`, `sed`, `wc`, or shell redirection.

```bash
# Count unique shell types in /etc/passwd
cut -d: -f7 /etc/passwd | sort | uniq -c

# List duplicate usernames from a file
sort users.txt | uniq -d

# Normalize case before comparison
tr '[:upper:]' '[:lower:]' < names.txt | sort | uniq

# Compare two command outputs after sorting
sort file1.txt > /tmp/file1.sorted
sort file2.txt > /tmp/file2.sorted
diff -u /tmp/file1.sorted /tmp/file2.sorted
```

Another practical example is log analysis:

```bash
# Show the top 10 source IPs from an access log
awk '{print $1}' access.log | sort | uniq -c | sort -nr | head
```

**Exam-focused usage tips**

For RHCSA tasks, you should be comfortable reading command output and choosing the simplest tool that solves the problem.

- Use **`sort`** before `uniq` when duplicates are not adjacent.
- Use **`cut`** for predictable, delimiter-based extraction.
- Use **`tr`** for quick character cleanup or case conversion.
- Use **`paste`** when you need to join parallel text streams.
- Use **`diff -u`** when reviewing configuration changes.
- Use **`cmp -s`** when you only need an equality check.

A few practical reminders:

- Always verify the delimiter before using `cut`.
- Remember that `uniq` works on adjacent lines only.
- Prefer `diff -u` for readable text comparisons.
- Prefer `cmp` for exact file identity checks.
- Use quotes around character sets and delimiters when needed to avoid shell interpretation issues.

**Quick reference**

```bash
sort -n file.txt
uniq -c file.txt
cut -d: -f1,3 /etc/passwd
tr '[:lower:]' '[:upper:]' < file.txt
paste file1 file2
diff -u old.conf new.conf
cmp -s file1 file2
```

**Summary**

These utilities are small, but together they cover a large portion of everyday Linux text processing. `sort` and `uniq` help you organize and deduplicate data, `cut` and `tr` reshape streams, `paste` recombines columns, and `diff` and `cmp` let you validate changes with confidence. If you can choose the right tool quickly, you can solve many RHCSA tasks with short, reliable command pipelines.
