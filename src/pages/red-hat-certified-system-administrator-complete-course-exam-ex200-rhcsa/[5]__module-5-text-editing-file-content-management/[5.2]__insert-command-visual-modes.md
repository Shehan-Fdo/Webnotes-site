## **Course:** Red Hat Certified System Administrator Complete Course (Exam EX200 ÔÇô RHCSA)

**Why Vim modes matter in RHCSA work**

On the RHCSA exam, you often edit configuration files on a remote system where speed and accuracy matter more than memorizing a full editor feature set. Vim is common because it is installed by default on many Red Hat systems and works well over SSH. The key skill is not typing text; it is knowing which mode you are in and how to move between modes without losing work.

Vim is modal, which means the same keys do different things depending on the current mode. That is useful for exam work because you can switch between reading, editing, and command execution without leaving the editor. The main modes you need here are:

- **Normal mode** for navigation, deletion, copying, pasting, and entering commands.
- **Insert mode** for typing and editing text.
- **Visual mode** for selecting text blocks before applying an action.
- **Command-line mode** for saving, quitting, searching, and running editor commands.

**Core mode transitions**

The most important habit is to recognize how to enter and exit each mode quickly.

- Press `Esc` to return to **Normal mode** from Insert, Visual, or Command-line mode.
- Press `i` to enter **Insert mode** before the cursor.
- Press `a` to enter Insert mode after the cursor.
- Press `o` to open a new line below and enter Insert mode.
- Press `O` to open a new line above and enter Insert mode.
- Press `:` in Normal mode to enter **Command-line mode**.
- Press `v` to enter **Visual mode** for character selection.
- Press `V` to enter **Visual Line mode** for whole-line selection.
- Press `Ctrl` + `v` to enter **Visual Block mode** for rectangular selections.

> If you feel ÔÇťstuck,ÔÇŁ press `Esc` once or twice. In Vim, returning to Normal mode is the safest recovery step.

**Insert mode: when you need to type content**

Use Insert mode when you are adding or changing file contents directly. This is the mode you use most often for editing configuration values, comments, hostnames, IP addresses, and service settings.

Common ways to enter Insert mode:

- `i` inserts text at the cursor position.
- `I` inserts at the beginning of the current line.
- `a` appends after the cursor.
- `A` appends at the end of the current line.
- `o` creates a new line below the current line.
- `O` creates a new line above the current line.

Practical example: editing a network or service configuration file.

```vim
# Normal mode
i
# type or replace text
Esc
```

A typical RHCSA workflow is to open a file, move to the line you need, enter Insert mode, make the change, then return to Normal mode to save.

```bash
sudo vim /etc/ssh/sshd_config
```

Inside the file, you might change a directive such as:

```text
PermitRootLogin no
PasswordAuthentication yes
```

After editing, press `Esc`, then save and quit with `:wq` or save without quitting with `:w`.

**Command-line mode: saving, quitting, and running editor commands**

Command-line mode starts with `:` from Normal mode. It is where you issue editor commands such as save, quit, search, and substitution.

Common commands:

- `:w` writes the file.
- `:q` quits if no changes were made.
- `:q!` quits without saving changes.
- `:wq` writes and quits.
- `:x` writes only if needed, then quits.
- `:set number` shows line numbers.
- `:set nonumber` hides line numbers.
- `:help` opens help.

Use `:x` carefully. It is not just a perfect synonym for `:wq`; it may avoid rewriting an unchanged file and can behave differently in some permission or read-only situations. For exam work, `:wq` is the clearest ÔÇťsave and quitÔÇŁ command when you know you want to write changes.

A practical save-and-exit sequence looks like this:

```vim
Esc
:wq
Enter
```

If you need to abandon a mistaken edit:

```vim
Esc
:q!
Enter
```

Command-line mode also supports search and substitution. For example, to replace a setting across the file, use a pattern that matches the actual text in the file rather than assuming exact spacing.

```vim
:%s/^PasswordAuthentication\s\+no$/PasswordAuthentication yes/
```

That example is safer than a literal text swap because configuration files often vary in spacing, comments, or formatting.

**Visual mode: selecting text before acting on it**

Visual mode is useful when you want to operate on a range of text without typing a line range manually. This is especially helpful when editing blocks of configuration, moving lines, or replacing repeated content.

Visual mode types:

- `v` selects characters.
- `V` selects whole lines.
- `Ctrl` + `v` selects a rectangular block.

Exit Visual mode with `Esc` to return to Normal mode. That is the standard way to recover after a selection or if you enter the wrong visual type.

Common actions after selecting text:

- `d` deletes the selection.
- `y` yanks, or copies, the selection.
- `p` pastes over or after the cursor depending on context.
- `>` indents the selection.
- `<` unindents the selection.
- `c` changes the selection and enters Insert mode.

A practical RHCSA example is editing a block of lines in a config file.

Suppose you need to adjust several SSH settings in `/etc/ssh/sshd_config`:

```text
PermitRootLogin no
PasswordAuthentication yes
PubkeyAuthentication yes
```

You can:

1. Open the file in Vim.
2. Move to the first line of the block.
3. Press `V` to select the full line.
4. Use `j` to extend the selection downward.
5. Press `d` to delete the block or `>` to indent it if needed.
6. Press `p` to paste a corrected block copied from elsewhere.
7. Press `Esc` to return to Normal mode.
8. Save with `:wq`.

This is faster than editing each line individually when you need to replace a multi-line section.

**Practical selection workflows for RHCSA tasks**

Visual mode becomes especially useful when you need to transform a block rather than a single line.

- Use **line visual mode** to reorder or replace groups of configuration lines.
- Use **block visual mode** to edit columns of text, such as aligned lists or repeated prefixes.
- Use **character visual mode** to change part of a line, such as a hostname, path, or option value.

Example: replacing a repeated address block in a hosts file.

```text
192.0.2.10   server1
192.0.2.11   server2
192.0.2.12   server3
```

If you need to change only the IP column, `Ctrl` + `v` lets you select the rectangular region and replace it consistently.

Another practical use is copying a known-good block from one file section to another:

1. Select the block with `V`.
2. Press `y` to copy it.
3. Move to the destination.
4. Press `p` to paste.
5. Enter Insert mode only for the values that differ.

This is often safer than retyping a working configuration from memory.

**Mode recovery: how to get back on track quickly**

When you are editing under time pressure, the biggest risk is not syntax; it is being in the wrong mode. A simple recovery routine helps.

- If keys seem to insert text unexpectedly, press `Esc` to return to Normal mode.
- If you see `-- INSERT --`, you are in Insert mode.
- If text is highlighted, you are likely in Visual mode; press `Esc` to cancel the selection.
- If the bottom line shows `:`, you are in Command-line mode; press `Esc` to cancel or `Enter` to execute the command.
- If nothing seems to work, press `Esc` twice, then check your mode before continuing.

A useful mental model is:

- **Normal mode** = control and navigation
- **Insert mode** = typing
- **Visual mode** = selecting
- **Command-line mode** = editor commands

**When to choose each mode**

Choosing the right mode saves time and reduces mistakes.

| Mode | Best use | RHCSA example |
|---|---|---|
| Normal | Navigate, delete, copy, paste, save | Jump to a line and delete a bad entry |
| Insert | Type or modify text directly | Change `yes` to `no` in a config file |
| Visual | Select a range before acting | Copy a block of SSH settings |
| Command-line | Save, quit, search, substitute | `:wq`, `:q!`, or `:%s/.../.../` |

Use **Normal mode** whenever you are moving around or applying an action to text. Use **Insert mode** only when you are actively typing. Use **Visual mode** when the change applies to a range. Use **Command-line mode** when the task is about file operations or editor commands rather than text entry.

**Real-world editing examples**

A common exam task is to adjust a service configuration and then verify the result.

```bash
sudo vim /etc/ssh/sshd_config
```

Inside Vim:

1. Search for the directive with `/PasswordAuthentication`.
2. Press `Enter` to jump to the match.
3. Press `0` to move to the start of the line if needed.
4. Press `cw` to change the word or `i` to insert at the cursor.
5. Type the new value.
6. Press `Esc`.
7. Save with `:wq`.

Another example is editing a file that contains multiple similar lines and using Visual mode to remove a bad block.

```text
AllowUsers alice
AllowUsers bob
AllowUsers carol
```

If `bob` should be removed:

1. Move to the `AllowUsers bob` line.
2. Press `V`.
3. Press `d`.
4. Press `Esc`.
5. Save with `:wq`.

This is often faster and less error-prone than line-by-line deletion with repeated cursor movement.

**Useful recovery and safety habits**

- Save early when you make a valid change, especially before trying a risky substitution.
- Use `:q!` only when you are sure you want to discard changes.
- Prefer `:wq` when you want a clear save-and-exit action.
- Use `:set number` when you need to reference line numbers during troubleshooting.
- Use search before editing so you change the correct occurrence.

> Be careful with substitutions in system files. A command that looks correct may not match the fileÔÇÖs exact formatting, and a text replacement does not always change the running service state. For example, editing a configuration file may require a service reload or restart before the change takes effect.

**Quick mode reference**

- `Esc` returns to Normal mode.
- `i`, `a`, `o` enter Insert mode.
- `:` enters Command-line mode.
- `v`, `V`, `Ctrl` + `v` enter Visual modes.
- `y` copies, `d` deletes, `p` pastes, `c` changes.
- `:w`, `:q`, `:wq`, `:q!` handle save and exit workflows.

**Exam strategy**

For RHCSA, the goal is not to master every Vim feature. You need enough control to edit configuration files confidently under time pressure.

- Use **Insert mode** for direct text changes.
- Use **Visual mode** for block edits and copy/paste workflows.
- Use **Command-line mode** for saving, quitting, and substitutions.
- Return to **Normal mode** often so you can recover quickly and avoid accidental edits.

If you can move between these modes without hesitation, you can handle most exam editing tasks efficiently.
