## **Course:** Red Hat Certified System Administrator Complete Course (Exam EX200 – RHCSA)

**Overview**

`find` and `locate` solve different file-discovery problems. `find` searches the live filesystem in real time and can filter by name, type, size, time, ownership, permissions, and more. `locate` searches a prebuilt database, so it is usually much faster, but it can return stale results until the database is updated.

Use `find` when you need accuracy, complex filtering, or immediate results. Use `locate` when you need a quick path lookup and can tolerate database lag.

**`find` fundamentals**

`find` evaluates a boolean expression against each path under one or more starting points. The expression is not just a list of tests; it is a logical expression with operators such as `-a` (AND), `-o` (OR), `!` (NOT), and actions such as `-print`, `-delete`, and `-exec`. Evaluation order matters, and parentheses are often needed to make intent explicit.

Common syntax:

```bash
find [starting-point...] [expression]
```

Examples:

```bash
find /etc -name "*.conf"
find /var/log -type f -mtime -7
find . -user alice -perm -u+x
```

Important behavior:

- `find` starts at one or more paths you provide.
- It walks the directory tree recursively unless you restrict it.
- Tests and actions are evaluated as part of an expression.
- `-print` is the default action on many systems when no explicit action is given, but relying on implicit behavior is less clear than writing it explicitly.

**Name and path matching**

`find` can match by basename with `-name` or `-iname`, and by full path with `-path` or `-ipath`.

- `-name` matches the final component of the path.
- `-path` matches the entire path string as `find` sees it.
- `-iname` and `-ipath` are case-insensitive variants.

Examples:

```bash
find /var/log -name "*.log"
find /var/log -path "*/nginx/*.log"
find /home -iname "report*.txt"
```

Use `-name` when you care about the filename only. Use `-path` when directory structure is part of the match.

**File type and metadata filters**

`find` is especially useful for narrowing results by file metadata.

- `-type f` finds regular files.
- `-type d` finds directories.
- `-type l` finds symbolic links.
- `-size` filters by file size.
- `-mtime`, `-atime`, and `-ctime` filter by modification, access, and metadata change time.
- `-user` and `-group` filter by ownership.
- `-perm` filters by permission bits.

Examples:

```bash
find /tmp -type f -size +100M
find /home -type d -mtime -2
find /srv -user apache -group apache
find /var/www -type f -perm -u=x
```

Time filters use day-based units by default:

- `-mtime 0` means modified within the last 24 hours.
- `-mtime -7` means modified less than 7 days ago.
- `-mtime +7` means modified more than 7 days ago.

For finer granularity, `find` also supports minute-based variants such as `-mmin`, `-amin`, and `-cmin`.

**Combining conditions safely**

Because `find` expressions are boolean, you should use parentheses when combining `-o` with other tests. Parentheses must be escaped or quoted in the shell.

```bash
find /var \( -name "*.log" -o -name "*.txt" \) -type f
```

This means:

- Search under `/var`
- Match files named `*.log` or `*.txt`
- Restrict the result to regular files

Without parentheses, the expression may not mean what you expect.

> Be careful with `-o` and `-a`. `find` expression precedence can produce surprising results if you do not group conditions explicitly.

**Actions and command execution**

`find` can perform actions on matched files. The most common are `-print`, `-exec`, `-ok`, and `-delete`.

- `-print` outputs matching paths.
- `-exec` runs a command on each match.
- `-ok` is like `-exec` but prompts before each execution.
- `-delete` removes matched files or directories.

Safe inspection example:

```bash
find /var/log -type f -name "*.log" -exec ls -lh {} \;
```

Batching example:

```bash
find /var/log -type f -name "*.log" -exec stat {} +
```

Notes on `-exec`:

- `{}` is replaced by the current match.
- `\;` ends the command for one match at a time.
- `+` ends the command and passes many matches at once when supported.
- `-exec ... +` is usually more efficient than `-exec ... \;`.

Use caution with destructive actions. For example, `-delete` removes files immediately, and `-exec` can run commands that modify data.

> Avoid using examples that change files unless you intend to demonstrate a real administrative task and understand the impact.

**Practical `find` patterns**

Find empty files:

```bash
find /tmp -type f -empty
```

Find files larger than 1 GiB:

```bash
find /data -type f -size +1G
```

Find files modified in the last 30 minutes:

```bash
find /var/log -type f -mmin -30
```

Find world-writable files:

```bash
find / -type f -perm -0002
```

Find files owned by a specific user and group:

```bash
find /home -user alice -group developers
```

Find directories with a specific name:

```bash
find / -type d -name "backup"
```

Find and inspect permissions:

```bash
find /srv -type f -perm -u=x -exec ls -l {} +
```

**Pruning directories**

`-prune` prevents `find` from descending into a directory. This is useful when you want to skip large or irrelevant trees such as mounted filesystems, caches, or backup directories.

Example:

```bash
find / \( -path /proc -o -path /sys -o -path /dev \) -prune -o -type f -name "*.conf" -print
```

How it works:

- The first part matches directories to skip.
- `-prune` stops recursion into those directories.
- The `-o` branch handles the remaining search.
- `-print` outputs only the desired matches.

This pattern is common in system administration because it avoids noisy or expensive searches.

**`locate` fundamentals**

`locate` searches a database of file paths instead of walking the filesystem. That makes it fast, especially on large systems.

Basic usage:

```bash
locate sshd_config
locate /var/log/messages
```

Important behavior:

- It matches against entries stored in the database.
- Results can include paths that no longer exist if the database is outdated.
- It may match the full stored path string, not just the filename.
- Matching behavior can vary by implementation and options, so do not assume one universal wildcard rule.

Use `locate` for quick discovery, then verify with `ls`, `stat`, or `find` if you need current filesystem state.

**Updating the `locate` database**

On many Red Hat systems, the database is updated by `updatedb`, often through a scheduled job or timer.

Common commands:

```bash
sudo updatedb
locate bashrc
```

If `locate` returns stale or missing results, the database may need to be refreshed. Database location and update scheduling can vary by distribution and configuration.

**`locate` filtering and options**

Some implementations support options such as:

- `-i` for case-insensitive matching
- `-n` to limit the number of results
- `-r` for regular expression matching

Examples:

```bash
locate -i sshd_config
locate -n 10 kernel
```

Because `locate` behavior depends on the installed implementation, check `man locate` on the target system before relying on advanced options.

**`find` vs `locate`**

| Feature | `find` | `locate` |
|---|---|---|
| Search source | Live filesystem | Prebuilt database |
| Speed | Slower | Faster |
| Freshness | Current | Can be stale |
| Filtering | Very rich | Limited |
| Actions | Yes | No |
| Best use | Precise admin tasks | Fast path discovery |

Choose `find` when you need:

- Current results
- Metadata filtering
- Recursive actions
- Safe verification before modification

Choose `locate` when you need:

- Fast path lookup
- A rough starting point for investigation
- A quick way to remember where something lives

**Real-world administration examples**

Find configuration files changed recently:

```bash
find /etc -type f -name "*.conf" -mtime -1
```

Find large log files before rotation:

```bash
find /var/log -type f -name "*.log" -size +50M
```

Find files that may need permission review:

```bash
find / -type f -perm -4000
```

Find a package-related file quickly:

```bash
locate httpd.conf
```

Then verify the result:

```bash
ls -l /etc/httpd/conf/httpd.conf
```

**Troubleshooting tips**

- If `find` returns too many results, narrow the starting path first.
- If `find` seems to miss files, check quoting, glob expansion, and parentheses.
- If `locate` misses a file you know exists, refresh the database with `updatedb`.
- If `locate` returns unexpected paths, remember that it searches database entries, not the live tree.
- If you need to act on results, test with `-print` first before using `-exec` or `-delete`.

**Exam-focused reminders**

- Know the difference between basename matching and full-path matching.
- Use `find` for time, ownership, permission, and type filters.
- Use parentheses with `-o` to avoid logic errors.
- Understand `-exec ... \;` versus `-exec ... +`.
- Remember that `locate` is fast but may be stale.

**Quick reference**

```bash
find /path -name "pattern"
find /path -type f -mtime -7
find /path \( -name "*.log" -o -name "*.txt" \) -print
find /path -type f -exec command {} +
locate filename
sudo updatedb
```
