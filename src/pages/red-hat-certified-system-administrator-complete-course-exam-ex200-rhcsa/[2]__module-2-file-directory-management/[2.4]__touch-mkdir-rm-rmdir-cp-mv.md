## **Course:** Red Hat Certified System Administrator Complete Course (Exam EX200 – RHCSA)

**File and directory management overview**

These commands are the core tools for creating, copying, moving, and removing files and directories on Linux. For RHCSA, you need to know both the basic syntax and the behavior differences that matter in real administration tasks.

- `touch` creates empty files or updates timestamps.
- `mkdir` creates directories, including parent directories when needed.
- `rm` removes files and directories recursively.
- `rmdir` removes only empty directories.
- `cp` copies files and directories.
- `mv` renames or moves files and directories.

A strong exam approach is to understand default behavior first, then add options only when the task requires them. Many mistakes come from assuming a command is “safe” by default when it is not.

**`touch`**

`touch` is mainly used to create an empty file quickly or update file timestamps. If the file does not exist, `touch` creates it with the current time. If it already exists, `touch` updates the access and modification timestamps without changing file contents.

Common uses:

- Create placeholder files for later use.
- Refresh timestamps for build or automation workflows.
- Test whether you can write in a directory.

Examples:

```bash
touch notes.txt
touch /tmp/app.log
```

Useful options:

- `-a` updates only access time.
- `-m` updates only modification time.
- `-c` does not create the file if it does not exist.
- `-t [[CC]YY]MMDDhhmm[.ss]` sets a specific timestamp.
- `-r reference_file` copies timestamps from another file.

Examples:

```bash
touch -a report.txt
touch -m report.txt
touch -c missing.txt
touch -t 202604261530.00 archive.txt
touch -r source.txt copy.txt
```

`touch` is often used in scripts to ensure a file exists before writing to it. It does not add content, so it is not a text editor replacement.

**`mkdir`**

`mkdir` creates directories. On RHCSA, you should know how to create parent directories and set permissions at creation time.

Basic examples:

```bash
mkdir projects
mkdir /tmp/lab1
```

Useful options:

- `-p` creates parent directories as needed and does not fail if the path already exists.
- `-m MODE` sets the directory permissions at creation time.
- `-v` prints a message for each created directory.

Examples:

```bash
mkdir -p /srv/app/logs
mkdir -m 750 secure
mkdir -pv /opt/lab/data
```

Important behavior:

- `mkdir -p /a/b/c` creates the full path if missing.
- `mkdir -m 700 private` creates the directory with the requested mode.
- `mkdir -m MODE` sets the mode directly for the new directory; do not treat it like a post-creation `chmod` that is filtered the same way as default creation permissions.

Use `mkdir -p` when a script or exam task may require nested directories. Use `-m` when the required permissions must be explicit at creation time.

**`rm`**

`rm` removes files and directories. It is powerful and dangerous because deletion is immediate and usually irreversible without backups or snapshots.

Basic examples:

```bash
rm file.txt
rm old1.log old2.log
```

Useful options:

- `-i` prompts before each removal.
- `-f` forces removal and suppresses most errors.
- `-r` or `-R` removes directories recursively.
- `-v` shows what is being removed.

Examples:

```bash
rm -i important.txt
rm -f temp.txt
rm -r testdir
rm -rf build/
```

Key behavior:

- `rm` does not remove directories unless you use `-r` or `-R`.
- `rm -r` removes directory contents recursively, including subdirectories.
- `rm -rf` is the common “force recursive” form, but it is also the most dangerous.

> Be careful with `rm -rf`, especially when variables, wildcards, or relative paths are involved. Verify the target with `pwd`, `ls`, or `echo` before deleting.

Practical safety pattern:

```bash
pwd
ls -ld targetdir
rm -r targetdir
```

**`rmdir`**

`rmdir` removes empty directories only. It is safer than `rm -r` because it refuses to delete a directory that contains files or subdirectories.

Examples:

```bash
rmdir emptydir
rmdir /tmp/lab/old
```

Useful options:

- `-p` removes parent directories if they become empty.
- `-v` prints each directory removed.

Examples:

```bash
rmdir -p /tmp/demo/a/b
rmdir -v emptydir
```

Use `rmdir` when you want a strict check that the directory is empty. If the directory is not empty, the command fails instead of deleting contents.

**`cp`**

`cp` copies files and directories. It is one of the most important commands for exam tasks because it supports simple file duplication, recursive directory copies, and attribute preservation.

Basic examples:

```bash
cp file1 file2
cp file1 /tmp/
cp file1 file2 file3 /backup/
```

Useful options:

- `-r` or `-R` copies directories recursively.
- `-a` archives a copy by preserving as much as possible, including permissions, ownership, timestamps, and symlinks.
- `-p` preserves mode, ownership, and timestamps where possible.
- `-i` prompts before overwrite.
- `-f` forces overwrite.
- `-u` copies only when the source is newer than the destination or the destination is missing.
- `-v` shows copied files.
- `-n` prevents overwrite.
- `-t DIR` copies sources into a target directory.

Examples:

```bash
cp -i config.ini config.ini.bak
cp -p report.txt /backup/
cp -r project/ /tmp/project-copy/
cp -a /var/www/html/ /backup/html/
cp -t /tmp/archive file1 file2 file3
```

Directory copy examples:

```bash
cp -r srcdir destdir
cp -a srcdir/ destdir/
```

Trailing slash behavior matters:

- `cp -r srcdir destdir` creates `destdir` containing `srcdir` if `destdir` does not already exist.
- `cp -r srcdir/ destdir/` copies the contents of `srcdir` into `destdir` when `destdir` exists.

Symlink behavior:

- By default, `cp` follows symbolic links when copying a symlink source and copies the target file contents.
- Use `-P` or `--no-dereference` to copy the symlink itself.
- `-d` is equivalent to `--no-dereference --preserve=links` in GNU `cp`; it preserves symlink identity and link relationships in the copied tree when applicable.

Examples:

```bash
cp -P linkname link-copy
cp -d /src/tree /dst/tree
```

Preserving metadata:

- `cp -p` is useful when you need timestamps and permissions to remain close to the original.
- `cp -a` is the safest choice for full-tree copies when you want a near-clone of the source.
- `cp -a` is commonly used for backups and migrations.

A practical backup example:

```bash
cp -a /etc /backup/etc-$(date +%F)
```

**`mv`**

`mv` moves files and directories or renames them. On Linux, `mv` overwrites the destination by default if the destination exists and is writable, unless you use `-i`, `-n`, or an alias changes the behavior.

Basic examples:

```bash
mv oldname newname
mv file.txt /tmp/
mv dir1 dir2
```

Useful options:

- `-i` prompts before overwrite.
- `-f` forces overwrite.
- `-n` prevents overwrite.
- `-v` shows the move operation.
- `-u` moves only when the source is newer than the destination or the destination is missing.

Examples:

```bash
mv report.txt report.old
mv -i config.ini /backup/
mv -n draft.txt /tmp/
mv -v logs/ /var/tmp/
```

Important behavior:

- Renaming within the same filesystem is usually fast because `mv` changes directory entries rather than copying data.
- Moving across filesystems requires copying and then deleting the source.
- `mv` can move directories as well as files.

A safe rename workflow:

```bash
ls -l oldname
mv oldname newname
ls -l newname
```

**Comparing the commands**

| Command | Main purpose | Key limitation |
|---|---|---|
| `touch` | Create empty files or update timestamps | Does not write content |
| `mkdir` | Create directories | Cannot create files |
| `rm` | Remove files and directories | Dangerous with recursive deletion |
| `rmdir` | Remove empty directories | Fails if directory is not empty |
| `cp` | Copy files and directories | Does not move original data |
| `mv` | Move or rename files and directories | May overwrite unless controlled |

**Common RHCSA task patterns**

These commands often appear in tasks that require you to prepare a directory tree, copy configuration files, or clean up temporary data.

Typical workflows:

1. Create a directory structure.
2. Copy a file or tree into a working location.
3. Rename or move files into place.
4. Remove temporary files after verification.

Examples:

```bash
mkdir -p /srv/project/{bin,conf,logs}
cp -a /etc/ssh/sshd_config /srv/project/conf/
mv /srv/project/conf/sshd_config /srv/project/conf/sshd_config.bak
rm -f /srv/project/logs/*.old
```

Another common pattern is making a backup before editing:

```bash
cp -p /etc/fstab /etc/fstab.bak
```

**Permissions, ownership, and timestamps**

These commands interact with file metadata, so you should know what they preserve and what they change.

- `touch` updates timestamps.
- `mkdir` creates a new directory with permissions influenced by the requested mode and the system defaults.
- `cp -p` preserves timestamps and mode where possible.
- `cp -a` preserves metadata more completely than `cp -p`.
- `mv` usually preserves metadata because it changes the file name or location rather than recreating the file, unless it crosses filesystems.

If you need exact ownership or permissions after a copy, verify with:

```bash
ls -l
stat filename
```

**Practical examples**

Create a workspace, copy a file, rename it, and clean up:

```bash
mkdir -p /tmp/rhcsa/lab1
touch /tmp/rhcsa/lab1/input.txt
cp -p /tmp/rhcsa/lab1/input.txt /tmp/rhcsa/lab1/input.txt.bak
mv /tmp/rhcsa/lab1/input.txt /tmp/rhcsa/lab1/input.old
rm -f /tmp/rhcsa/lab1/input.txt.bak
rmdir /tmp/rhcsa/lab1
```

Copy a directory tree while preserving metadata:

```bash
cp -a /etc/ssh /backup/ssh
```

Remove an empty directory tree from the bottom up:

```bash
rmdir -p /tmp/demo/a/b/c
```

**Exam tips**

- Use `mkdir -p` when the parent path may not exist.
- Use `rmdir` when you must prove a directory is empty.
- Use `cp -a` for full directory-tree copies.
- Use `mv` for renaming and relocating files.
- Use `rm -r` only when recursive deletion is required and you have verified the target.
- Use `-i` when you want confirmation before overwrite or deletion.
- Remember that `cp` and `mv` can overwrite existing files unless you prevent it.

**Quick command reminders**

```bash
touch file
mkdir -p dir/subdir
rm -r dir
rmdir emptydir
cp -a source/ dest/
mv oldname newname
```
