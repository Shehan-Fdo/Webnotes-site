## **Course:** Red Hat Certified System Administrator Complete Course (Exam EX200 – RHCSA)

**Overview**

Linux administrators spend a lot of time inspecting text files: logs, configuration files, command output, and reports. The core tools in this topic each solve a different viewing problem:

- **`cat`** for quick, full-file output and simple concatenation.
- **`less`** for interactive paging through long files.
- **`head`** for the beginning of a file or stream.
- **`tail`** for the end of a file or stream.
- **`wc`** for counting lines, words, and bytes.

These commands are especially useful in RHCSA tasks because they help you verify configuration, inspect logs, and confirm file size or structure without opening an editor.

**`cat`: print and combine files**

`cat` stands for *concatenate*. In practice, you use it to display small files, join files together, or feed file content into another command. It is fast and simple, but it is not ideal for large files because it dumps everything to the terminal at once.

Common uses:

- Display a file:
  ```bash
  cat /etc/hosts
  ```
- Combine files in order:
  ```bash
  cat part1.txt part2.txt > full.txt
  ```
- Create a file from standard input:
  ```bash
  cat > notes.txt
  ```
  Type content, then press <kbd>Ctrl</kbd> + <kbd>D</kbd> to save and exit.
- Append one file to another:
  ```bash
  cat extra.txt >> report.txt
  ```

Practical notes:

- Use `cat` when the file is short and you want immediate output.
- Use it with redirection carefully; `>` overwrites the destination file.
- `cat` does not paginate, so it can flood your terminal with large logs.

> Avoid using `cat` on very large files when you only need to inspect part of the content. Use `less`, `head`, or `tail` instead.

**`less`: interactive paging**

`less` is the preferred tool for reading long text files because it lets you move forward and backward through content. It does not load the entire file into your terminal view at once, and it supports searching.

Basic usage:

```bash
less /var/log/messages
less /etc/fstab
```

Useful navigation keys:

- `Space` or `f` moves forward one screen.
- `b` moves backward one screen.
- `Enter` moves forward one line.
- `/pattern` searches forward for text.
- `n` repeats the last search forward.
- `N` repeats the last search backward.
- `g` jumps to the beginning.
- `G` jumps to the end.
- `q` quits.

Why `less` matters:

- It is safer than `cat` for large files.
- It supports searching without opening an editor.
- It is useful for reviewing logs, configuration files, and command output piped from other commands.

Examples with pipelines:

```bash
journalctl -xe | less
dmesg | less
ps aux | less
```

You can also open compressed or filtered output through other tools, then page through it with `less` if needed.

**`head`: inspect the beginning**

`head` shows the first lines of a file. This is useful when you want to confirm file format, headers, or the top of a log or report.

Default behavior:

```bash
head /etc/passwd
```

By default, `head` shows the first 10 lines. You can change that with `-n`:

```bash
head -n 5 /etc/passwd
head -n 20 access.log
```

Typical uses:

- Check the top of a configuration file.
- Verify the first lines of a CSV or text export.
- Inspect the start of a log file for metadata or headers.

`head` is also useful with pipelines:

```bash
ip addr show | head
rpm -qa | head -n 10
```

**`tail`: inspect the end and follow updates**

`tail` shows the last lines of a file. This is especially useful for logs, because new entries are usually appended to the end.

Default behavior:

```bash
tail /var/log/messages
```

By default, `tail` shows the last 10 lines. Use `-n` to change the number of lines:

```bash
tail -n 20 /var/log/secure
tail -n 50 /var/log/messages
```

A key feature is `-f`, which follows a file as it grows:

```bash
tail -f /var/log/messages
```

This is useful when you want to watch live log activity while testing a service or troubleshooting a problem.

Common workflow:

1. Open a terminal and start following the log.
2. Run the action you want to test in another terminal.
3. Watch new log entries appear in real time.
4. Press <kbd>Ctrl</kbd> + <kbd>C</kbd> to stop following.

Practical examples:

```bash
tail -f /var/log/secure
tail -n 100 /var/log/audit/audit.log
```

> `tail -f` is a live view, so it keeps running until you stop it.

**`wc`: count lines, words, and bytes**

`wc` means *word count*. It reports counts that help you measure file size and structure.

Common options:

- `-l` counts lines.
- `-w` counts words.
- `-c` counts bytes.
- `-m` counts characters, which can differ from bytes in multibyte encodings.

Examples:

```bash
wc /etc/passwd
wc -l /etc/passwd
wc -w /etc/passwd
wc -c /etc/passwd
```

Practical uses:

- Confirm how many lines are in a log or report.
- Check whether a file is empty or unexpectedly short.
- Compare output sizes in scripts or pipelines.

Examples with pipelines:

```bash
journalctl -u sshd | wc -l
grep -c "ERROR" application.log
find /etc -type f | wc -l
```

Remember that `grep -c` counts matching lines, while `wc -l` counts all lines in the input.

**Choosing the right command**

Use the tool that matches the task instead of defaulting to `cat`.

| Task | Best command | Why |
|---|---|---|
| View a short file quickly | `cat` | Simple full output |
| Read a long file interactively | `less` | Paging and search |
| Check the first lines | `head` | Fast top-of-file inspection |
| Check the last lines | `tail` | Best for logs and recent output |
| Watch a growing log | `tail -f` | Live updates |
| Count file size or structure | `wc` | Line, word, byte counts |

**Practical RHCSA examples**

Configuration verification often starts with reading files directly:

```bash
cat /etc/hostname
less /etc/fstab
head -n 20 /etc/ssh/sshd_config
tail -n 20 /var/log/secure
wc -l /etc/passwd
```

When troubleshooting a service, you often combine commands:

```bash
systemctl status sshd | less
journalctl -u sshd | tail -n 50
grep -n "PermitRootLogin" /etc/ssh/sshd_config
```

If you need to inspect a file without editing it, `less` is usually the safest choice. If you need only a small portion, `head` or `tail` is faster. If you need a count, `wc` gives immediate feedback.

**Command behavior and output details**

These commands read from files or standard input, which makes them easy to chain in pipelines.

- `cat` can accept multiple files and standard input.
- `less` is interactive and best used at a terminal.
- `head` and `tail` default to 10 lines unless you specify otherwise.
- `wc` prints counts in a fixed order unless you select specific options.

Examples:

```bash
printf "one\ntwo\nthree\n" | wc -l
printf "alpha\nbeta\n" | head -n 1
printf "alpha\nbeta\n" | tail -n 1
```

You can also use `cat` to merge streams into a pipeline, but for simple viewing, direct commands are usually clearer:

```bash
grep "Failed" /var/log/secure | head -n 5
grep "sshd" /var/log/secure | tail -n 20
```

**Common mistakes**

- Using `cat` on a huge log when you only need the start or end.
- Forgetting that `tail` shows the last 10 lines by default.
- Confusing `wc -w` with line counts.
- Leaving `tail -f` running and thinking the terminal is frozen.
- Overwriting files accidentally with `>` when you meant to append with `>>`.

**Quick reference**

- `cat file` displays the whole file.
- `less file` pages through the file interactively.
- `head -n 5 file` shows the first 5 lines.
- `tail -n 5 file` shows the last 5 lines.
- `tail -f file` follows new content.
- `wc -l file` counts lines.

**Exam tip**

For RHCSA tasks, use the simplest command that answers the question. If you need to verify a configuration value, `grep` plus `head`, `tail`, or `less` is often faster than opening an editor. If you need to inspect logs during troubleshooting, `tail -f` is usually the most effective starting point.
