## **Course:** Red Hat Certified System Administrator Complete Course (Exam EX200 – RHCSA)

**File and directory fundamentals**

Linux file management is a core RHCSA skill because nearly every administrative task depends on creating, organizing, and protecting data in the filesystem. You use these commands to build directory structures, stage configuration files, move content between locations, and remove obsolete data safely.

At the shell, file operations are usually performed with a small set of commands:

- `touch` creates empty files or updates timestamps.
- `mkdir` creates directories.
- `cp` copies files and directories.
- `mv` moves or renames files and directories.
- `rm` removes files and directories.
- `rmdir` removes empty directories only.

Understanding how these commands behave with paths, wildcards, permissions, and recursive options helps you avoid accidental data loss and work efficiently under exam pressure.

**Creating files and directories**

Use `touch` when you need a file to exist immediately, even if it is empty. This is common when preparing placeholder files, log files, or configuration files that will be edited later.

```bash
touch report.txt
touch /tmp/app.log
```

`touch` also updates access and modification timestamps on existing files. That makes it useful for testing timestamp-based workflows, but be careful not to change metadata unintentionally.

Use `mkdir` to create directories. The `-p` option is especially important because it creates parent directories as needed and does not fail if the path already exists.

```bash
mkdir projects
mkdir -p /srv/app/config
mkdir -p /home/student/labs/week1
```

Key points:

- **`mkdir`** creates one or more directories.
- **`-p`** creates missing parent directories and avoids errors for existing paths.
- **`touch`** is for files, not directories.
- A directory path must exist before you can create files inside it unless you create the path with `mkdir -p`.

A practical workflow is to create the directory tree first, then create files inside it:

```bash
mkdir -p ~/work/docs
touch ~/work/docs/notes.txt
```

**Copying files and directories**

Use `cp` to duplicate files or directories. Copying is different from moving because the original remains in place. This is useful when you need a backup, a template, or a second version of a file.

```bash
cp file1.txt file2.txt
cp /etc/hosts ~/hosts.backup
```

To copy directories, use `-r` or `-R` for recursive copy. Without recursion, `cp` refuses to copy directories.

```bash
cp -r project project-backup
cp -R /etc/skel ~/skel-copy
```

Common options:

- **`-i`** prompts before overwriting an existing file.
- **`-v`** shows verbose output.
- **`-u`** copies only when the source is newer than the destination or the destination is missing.
- **`-a`** preserves attributes and copies recursively; it is often the safest choice for backups.

```bash
cp -av /etc/ssh ~/backup-etc-ssh
```

When preserving ownership, permissions, timestamps, and symbolic links matters, use `-a`. It is effectively a preservation-oriented recursive copy and is often preferred for administrative backups.

> Be careful with `cp -r` and `cp -a` when the destination path already exists. The resulting directory layout depends on whether the destination is a directory or a new path name.

A useful pattern is to copy a file into a directory while keeping the original name:

```bash
cp /etc/fstab /tmp/
```

If the destination is a directory, `cp` places the file inside it. If the destination does not exist, `cp` may create a new file with that name.

**Moving and renaming files and directories**

Use `mv` to move files or directories to a new location. The same command also renames items when the destination is in the same directory.

```bash
mv oldname.txt newname.txt
mv report.txt /tmp/
mv project /srv/archive/
```

Important behavior:

- **Moving** changes the path.
- **Renaming** changes the name.
- `mv` can handle both files and directories.
- If the destination exists and is a directory, the source is moved into that directory.

Examples:

```bash
mv draft.txt final.txt
mv logs/ /var/tmp/
mv config.ini /etc/myapp/config.ini
```

Use `-i` to prompt before overwriting, especially when working with important files.

```bash
mv -i config.ini /etc/myapp/
```

Use `-v` to confirm what changed:

```bash
mv -v old.conf new.conf
```

A common exam task is to reorganize files by moving them into a target directory. If the target directory does not exist, create it first:

```bash
mkdir -p ~/archive/2026
mv *.log ~/archive/2026/
```

**Deleting files and directories**

Use `rm` to delete files. This action is immediate and usually irreversible unless you have backups or snapshots.

```bash
rm oldfile.txt
rm /tmp/test1 /tmp/test2
```

For directories, use recursive removal:

```bash
rm -r olddir
rm -rf tempdir
```

The difference matters:

- **`rm`** removes files.
- **`rm -r`** removes directories and their contents recursively.
- **`rm -f`** forces deletion and suppresses most prompts and missing-file errors.
- **`rm -rf`** is powerful and dangerous because it removes recursively without prompting.

Use `rmdir` only for empty directories:

```bash
rmdir emptydir
rmdir -p /tmp/a/b/c
```

The `-p` option removes parent directories if they become empty, which is useful when cleaning up temporary directory trees.

> Never use `rm -rf` casually. Verify the path first with `pwd`, `ls`, or `find` before deleting anything important.

A safer deletion workflow is:

1. Confirm the current directory with `pwd`.
2. List the target contents with `ls -l`.
3. Remove only the intended files or directories.
4. Recheck with `ls` or `find` to confirm the result.

```bash
pwd
ls -l /tmp/testdir
rm -r /tmp/testdir
```

**Wildcards and pattern matching**

Shell wildcards help you operate on multiple files at once. The shell expands these patterns before the command runs.

Common patterns:

- `*` matches any number of characters.
- `?` matches exactly one character.
- `[abc]` matches one character from a set.
- `[0-9]` matches one character in a range.

Examples:

```bash
cp *.conf /backup/configs/
mv report?.txt /tmp/reports/
rm old-[0-9].log
```

Use wildcards carefully because they can match more files than expected. Always preview the match with `ls` first:

```bash
ls *.conf
rm *.tmp
```

This habit reduces the risk of deleting or moving the wrong files.

**Relative and absolute paths**

File commands work with both relative and absolute paths. An absolute path starts at `/`, while a relative path is interpreted from the current working directory.

Examples:

```bash
cd /var/log
touch app.log
cp app.log /tmp/
mv app.log archived-app.log
```

Relative paths are shorter, but absolute paths are safer in scripts and when you need to avoid ambiguity. On the exam, use whichever is clearest for the task.

Useful navigation commands:

```bash
pwd
ls
cd /path/to/dir
```

**Practical command patterns**

A few common workflows appear often in administration tasks:

- Create a directory tree for a project:
  ```bash
  mkdir -p ~/projects/app/{bin,conf,logs}
  ```
- Copy a configuration file before editing:
  ```bash
  cp /etc/ssh/sshd_config /etc/ssh/sshd_config.bak
  ```
- Move log files into an archive directory:
  ```bash
  mkdir -p ~/archive/logs
  mv *.log ~/archive/logs/
  ```
- Rename a file for versioning:
  ```bash
  mv report.txt report-2026-04-26.txt
  ```
- Remove temporary files:
  ```bash
  rm -f /tmp/*.tmp
  ```

**Permissions and ownership considerations**

File operations can fail if you do not have the required permissions. Creating, copying, moving, and deleting depend on permissions on the source, destination, and parent directories.

General rules:

- You need write permission on a directory to create, rename, move, or delete entries inside it.
- You need read permission on a file to copy its contents.
- You need execute permission on a directory to access items within it.
- Root can bypass many restrictions, but you should still verify paths carefully.

If a command fails, check permissions with:

```bash
ls -ld /path/to/dir
ls -l /path/to/file
```

If needed, adjust ownership or permissions with other administrative tools, but only after confirming the task requires it.

**Troubleshooting and verification**

When a file operation does not behave as expected, inspect the path and the command syntax first. Many mistakes come from confusing a file with a directory or using the wrong destination form.

Useful checks:

```bash
pwd
ls -l
file name
stat name
```

Typical issues:

- The destination directory does not exist.
- A wildcard matches nothing or too many files.
- You used `rm` on a directory without `-r`.
- You used `cp` on a directory without recursion.
- You renamed a file but expected a copy.
- You deleted the wrong path because you were in the wrong directory.

A disciplined workflow is to inspect first, act second, and verify last.

**Exam-focused reminders**

For RHCSA tasks, speed and accuracy matter more than memorizing many options. Focus on the behavior of the core commands and the safest defaults.

- Use `mkdir -p` for nested directory creation.
- Use `cp -a` when preserving attributes matters.
- Use `mv` for both moving and renaming.
- Use `rm -r` only when you intend to remove directory contents.
- Use `rm -i` or `mv -i` when overwriting is possible.
- Verify with `ls`, `pwd`, and `find` before destructive actions.

**Quick reference**

| Task | Command | Common option |
|---|---|---|
| Create empty file | `touch file` | `touch` |
| Create directory | `mkdir dir` | `-p` |
| Copy file | `cp src dst` | `-i`, `-v`, `-u`, `-a` |
| Copy directory | `cp -r dir1 dir2` | `-a` |
| Move or rename | `mv src dst` | `-i`, `-v` |
| Remove file | `rm file` | `-i`, `-f` |
| Remove directory | `rm -r dir` | `-rf` |
| Remove empty directory | `rmdir dir` | `-p` |
