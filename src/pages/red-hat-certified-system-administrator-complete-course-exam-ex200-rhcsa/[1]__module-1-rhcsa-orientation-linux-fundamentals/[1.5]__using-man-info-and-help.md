## **Course:** Red Hat Certified System Administrator Complete Course (Exam EX200 – RHCSA)

**Why built-in documentation matters**

Linux systems ship with local documentation so you can answer questions without internet access. For RHCSA work, you should be able to identify a command, inspect its options, and confirm syntax quickly. The three most common entry points are `--help`, `man`, and `info`.

These tools complement each other:

- `--help` gives a fast summary of common options.
- `man` provides the standard reference for most commands and configuration files.
- `info` often contains more structured, tutorial-style documentation for GNU utilities.

**Using `--help` for a quick overview**

Many commands support `--help`, which prints a short usage summary and the most important flags. This is the fastest way to confirm syntax when you already know the command name.

```bash
ls --help
cp --help
useradd --help
```

Use `--help` when you need:

- A quick reminder of option names.
- The basic command form.
- A short list of common flags without leaving the terminal.

`--help` is not standardized across every program, so the output format varies. Some commands print a concise usage line, while others provide a longer summary. If a command does not support `--help`, try `-h` or consult the manual page.

**Reading manual pages with `man`**

The `man` command opens the system manual pages. This is the primary documentation source for most Linux commands, system calls, library functions, and configuration files.

```bash
man ls
man useradd
man passwd
```

A manual page usually includes:

- **NAME**: short description of the command.
- **SYNOPSIS**: command syntax and argument patterns.
- **DESCRIPTION**: detailed behavior and option explanations.
- **OPTIONS**: flag-by-flag reference.
- **EXAMPLES**: practical usage, when available.
- **FILES**: related paths and configuration files.
- **SEE ALSO**: related commands and documentation.

The `SYNOPSIS` section is especially useful because it shows required and optional arguments in a compact form. Brackets usually indicate optional items, and vertical bars indicate alternatives.

Example interpretation:

```text
cp [OPTION]... [-T] SOURCE DEST
cp [OPTION]... SOURCE... DIRECTORY
```

This tells you that `cp` supports multiple forms depending on whether the destination is a file or directory.

**Navigating `man` pages efficiently**

The `man` viewer is interactive. Common navigation keys help you search and move through long pages.

- `Space` moves forward one screen.
- `b` moves back one screen.
- `/pattern` searches forward for text.
- `n` repeats the last search.
- `q` quits the page.

Useful terminal examples:

```bash
man 5 passwd
man 8 useradd
man -k network
```

The section number matters because many names exist in multiple documentation categories. For example:

- `1` = user commands
- `5` = file formats and conventions
- `8` = system administration commands

So `man passwd` may show the user command, while `man 5 passwd` shows the password file format if that page exists.

**Searching documentation with `man -k` and `apropos`**

If you do not know the exact command name, search the manual page database by keyword.

```bash
man -k copy
apropos copy
```

`man -k` and `apropos` are effectively the same idea: they search the short descriptions from installed man pages. This is useful when you know the task but not the command.

Examples of use:

- Find commands related to file copying, compression, or permissions.
- Discover the correct utility for a task before trying random commands.
- Confirm that a command exists on the system.

If the search returns too many results, narrow the keyword or use a more specific term.

**Using `info` for structured GNU documentation**

The `info` system provides documentation in a hypertext-like format. It is common for GNU utilities and often contains more detail than the man page, including conceptual explanations and examples.

```bash
info ls
info coreutils
info grep
```

`info` is useful when you want:

- A more tutorial-style explanation.
- Navigation by topic and node.
- Additional examples and conceptual background.

Inside `info`, navigation is similar to a menu-driven document:

- `n` goes to the next node.
- `p` goes to the previous node.
- `u` goes up one level.
- `q` quits.

If you are studying GNU tools, `info` can clarify behavior that the man page summarizes more briefly.

**Comparing `man`, `info`, and `--help`**

| Tool | Best for | Strengths | Limitations |
|---|---|---|---|
| `--help` | Quick syntax check | Fast, always near the command | Often brief and inconsistent |
| `man` | Standard reference | Comprehensive, widely available | Can be dense and terse |
| `info` | GNU utility details | Structured, explanatory, examples | Not available for every command |

A practical workflow is:

1. Start with `--help` for a quick reminder.
2. Use `man` for authoritative syntax and options.
3. Use `info` when you need deeper explanation or examples.

**Finding the right manual section**

Some topics appear in multiple sections, so you may need to specify the section number explicitly.

```bash
man 1 printf
man 3 printf
man 5 crontab
man 8 mount
```

This matters because the same name can refer to:

- A user command.
- A library function.
- A file format.
- An administrative command.

When in doubt, search the section list or use `man -k` to identify the correct page.

**Practical examples for RHCSA tasks**

When you are configuring a system, documentation helps you verify exact syntax before making changes.

- To inspect user creation options:
  ```bash
  man useradd
  useradd --help
  ```
- To check file permission behavior:
  ```bash
  man chmod
  man chown
  ```
- To review service management commands:
  ```bash
  man systemctl
  systemctl --help
  ```
- To understand mount options:
  ```bash
  man mount
  man fstab
  ```

For configuration files, `man` pages in section `5` are especially valuable because they describe file format and valid directives.

**Reading command syntax carefully**

Manual pages use notation that you should learn to interpret quickly:

- `[]` means optional content.
- `...` means repetition.
- `|` means one of several alternatives.
- Uppercase words often represent placeholders such as filenames or arguments.

Example:

```text
useradd [options] LOGIN
```

This means you must provide a login name, and you may add options before it.

Another example:

```text
grep [OPTION]... PATTERNS [FILE]...
```

This means `grep` accepts one or more patterns and zero or more files.

**Common troubleshooting habits**

If a command behaves unexpectedly, documentation can help you verify whether the issue is syntax, permissions, or environment.

- Check the exact option spelling.
- Confirm whether the command expects a file path, a username, or a device name.
- Review examples for edge cases.
- Look at the `EXIT STATUS` or `NOTES` sections when available.

> Be careful not to assume every command supports the same flags. GNU-style options are common, but not universal.

**Useful command patterns**

```bash
# Quick summary
command --help

# Full manual
man command

# Search by keyword
man -k keyword

# Open GNU info documentation
info command
```

You can also combine these tools with shell search habits:

```bash
man -k mount
man 8 mount
man 5 fstab
info coreutils
```

This approach helps you move from a general task to the exact syntax you need.

**Study strategy for the exam**

During RHCSA preparation, practice looking up documentation without leaving the terminal. The exam rewards speed and accuracy, so you should be comfortable finding answers locally.

- Use `--help` first when you need a fast reminder.
- Use `man` when you need authoritative details.
- Use `info` when a GNU utility needs deeper explanation.
- Use `man -k` or `apropos` when you do not know the command name.

The goal is not memorization of every option. The goal is to know how to find the correct option safely and quickly.
