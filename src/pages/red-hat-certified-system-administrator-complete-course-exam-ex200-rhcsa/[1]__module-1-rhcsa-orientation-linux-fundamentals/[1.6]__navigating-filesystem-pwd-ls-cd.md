## **Course:** Red Hat Certified System Administrator Complete Course (Exam EX200 – RHCSA)

**Filesystem navigation fundamentals**

Linux organizes files in a single hierarchical tree rooted at `/`. You move through that tree by combining the current working directory, path names, and a small set of core commands. For RHCSA-level work, you should be able to identify where you are, inspect what is nearby, and move efficiently without relying on a graphical file manager.

The three essential commands are:

- `pwd` to print the current working directory
- `ls` to list directory contents
- `cd` to change the current directory

These commands are simple, but they are used constantly in administration, troubleshooting, scripting, and exam tasks. Good navigation habits reduce mistakes when editing configuration files, copying data, or running commands against the wrong path.

**Linux path concepts**

A path tells you where a file or directory lives in the filesystem tree. You should understand the difference between absolute and relative paths before using navigation commands effectively.

- **Absolute path** starts at `/` and names the full location.
  - Example: `/etc/ssh/sshd_config`
- **Relative path** starts from your current directory.
  - Example: `Documents/report.txt`
- `.` means the current directory.
- `..` means the parent directory.
- `~` expands to your home directory.
- `-` with `cd` switches to the previous directory.

A practical mental model is to think of the filesystem as a map. `pwd` tells you your current coordinates, `ls` shows nearby landmarks, and `cd` moves you to a new location.

**Using `pwd` to confirm location**

`pwd` stands for *print working directory*. It is the fastest way to verify where you are before making changes. This matters because many Linux commands act on the current directory if you give them relative paths.

```bash
pwd
```

Typical output:

```bash
/home/student
```

Useful behavior to remember:

- `pwd` helps prevent accidental edits in the wrong directory.
- It is especially useful after multiple `cd` operations.
- In scripts, it can help confirm execution context.
- `pwd -P` shows the physical directory path, resolving symbolic links.
- `pwd -L` shows the logical path as the shell sees it, which may include symlinks.

Example:

```bash
cd /var/log
pwd
pwd -P
```

If `/var/log` is reached through a symbolic link, the logical and physical paths may differ. For exam tasks, the default `pwd` output is usually enough, but knowing the distinction helps when troubleshooting path confusion.

**Using `ls` to inspect directory contents**

`ls` lists files and directories. It is the primary command for checking what is available before you move or act on it. You should know the most common options and how they affect output.

```bash
ls
```

Common options:

- `-l` shows long format with permissions, ownership, size, and timestamps.
- `-a` includes hidden entries whose names begin with `.`.
- `-h` makes sizes human-readable when used with `-l`.
- `-R` lists directories recursively.
- `-t` sorts by modification time.
- `-S` sorts by size.
- `-d` lists the directory itself, not its contents.
- `-1` prints one entry per line.

Examples:

```bash
ls -l
ls -la
ls -lh /var/log
ls -lt
```

A long listing is especially useful because it reveals:

- **File type** and permissions
- **Link count**
- **Owner** and group
- **Size**
- **Timestamp**
- **Name**

Example output pattern:

```bash
-rw-r--r--. 1 root root  1234 Apr 26 10:15 example.txt
drwxr-xr-x. 2 root root  4096 Apr 26 10:20 config
```

Interpretation:

- The first character indicates type:
  - `-` regular file
  - `d` directory
  - `l` symbolic link
- Permission bits follow the type character.
- The trailing `.` may indicate SELinux context is present on the system.
- Hidden files are not shown unless you use `-a`.

A few practical combinations are worth memorizing:

```bash
ls -lah
ls -l /etc
ls -ld /etc/ssh
```

Use `ls -ld` when you want details about a directory itself rather than its contents.

> `ls` does not show everything by default. Hidden files and directories are common in Linux, especially for user configuration and application settings.

**Using `cd` to move through the filesystem**

`cd` stands for *change directory*. It updates your shell’s current working directory, which affects how relative paths are interpreted.

```bash
cd /etc
```

Common forms:

- `cd /path/to/dir` moves to an absolute path.
- `cd relative/path` moves relative to the current directory.
- `cd` with no arguments returns you to your home directory.
- `cd ~` also returns you to your home directory.
- `cd ..` moves up one level.
- `cd -` returns to the previous directory.

Examples:

```bash
cd /var/log
cd ..
cd -
cd
```

Practical behavior:

- `cd` does not print output when successful.
- If the target does not exist or is not a directory, the shell prints an error.
- `cd` is a shell builtin, not usually an external executable.
- Because it changes the shell state, it must run in your current shell session to be useful.

A common workflow is:

1. Check where you are with `pwd`.
2. Inspect the destination with `ls`.
3. Move with `cd`.
4. Confirm the new location with `pwd`.

**Relative navigation patterns**

Relative paths are efficient once you understand the directory structure. They are common in administration tasks and exam scenarios.

Examples:

```bash
cd /etc
cd ssh
pwd
```

Here, `ssh` is relative to `/etc`, so the final location is `/etc/ssh`.

Useful path shortcuts:

- `.` refers to the current directory.
- `..` refers to the parent directory.
- `../..` moves up two levels.
- `./script.sh` runs a script in the current directory when the current directory is not in `PATH`.

Examples:

```bash
cd ..
cd ../..
ls ./configs
```

These shortcuts are especially helpful when you need to move quickly between related directories without typing full paths.

**Home directories and user context**

Each user typically has a home directory, such as `/home/student` or `/root`. The tilde shortcut expands to the current user’s home directory.

Examples:

```bash
cd ~
pwd
ls ~
```

Important points:

- `~` refers to the current user’s home directory.
- `~username` refers to another user’s home directory, if permitted.
- `cd` with no arguments is equivalent to `cd ~` in most shells.
- `root` usually has `/root` as the home directory, not `/home/root`.

This matters when you switch users with `su` or `sudo -i`, because your home directory and shell context may change.

**Practical navigation workflow**

A disciplined workflow helps avoid mistakes when working on a live system.

1. Verify your current location with `pwd`.
2. Inspect the target directory with `ls` or `ls -l`.
3. Move with `cd` using the most specific path you know.
4. Confirm the move with `pwd`.
5. Re-check contents before editing or copying files.

Example:

```bash
pwd
ls -l /etc/ssh
cd /etc/ssh
pwd
ls -l
```

This pattern is useful before editing configuration files such as `/etc/ssh/sshd_config`, because it reduces the chance of editing the wrong file or directory.

**Common exam-relevant examples**

You should be comfortable navigating to standard system locations quickly.

```bash
cd /etc
ls
cd /var/log
ls -lh
cd /home/student
pwd
```

Typical use cases:

- Inspecting service configuration in `/etc`
- Reviewing logs in `/var/log`
- Working in a user’s home directory under `/home`
- Checking system data under `/usr`, `/opt`, or `/srv`

If you need to return to the previous directory after checking another location, use:

```bash
cd -
```

This is useful when comparing files in two directories or moving back and forth during troubleshooting.

**Troubleshooting navigation issues**

If a navigation command fails, check the path carefully and verify that the target is a directory.

Common problems:

- Typing a file path instead of a directory path with `cd`
- Misspelling a directory name
- Using a relative path from the wrong location
- Forgetting that hidden files require `ls -a`
- Assuming a directory exists when it does not

Examples of errors:

```bash
cd /etc/sshd_config
```

This fails because `sshd_config` is a file, not a directory.

```bash
cd /var/logg
```

This fails because the path is misspelled.

Useful checks:

```bash
pwd
ls -ld /path/to/target
file /path/to/target
```

`ls -ld` is especially helpful because it shows whether the target is a directory without entering it.

**Command reference summary**

| Command | Purpose | Common use |
|---|---|---|
| `pwd` | Print current directory | Confirm where you are |
| `ls` | List contents | Inspect files and directories |
| `cd` | Change directory | Move through the filesystem |

Common options:

| Command | Option | Effect |
|---|---|---|
| `pwd` | `-P` | Show physical path |
| `pwd` | `-L` | Show logical path |
| `ls` | `-l` | Long listing |
| `ls` | `-a` | Include hidden files |
| `ls` | `-h` | Human-readable sizes |
| `ls` | `-d` | List directory entry itself |
| `cd` | `-` | Switch to previous directory |

**Quick practice examples**

```bash
pwd
ls -la
cd /etc
ls -ld ssh
cd ssh
pwd
cd -
```

These commands demonstrate the core workflow: confirm, inspect, move, verify, and return.

**Key takeaways**

- Use `pwd` to confirm your current location before making changes.
- Use `ls` to inspect directory contents and verify targets.
- Use `cd` to move through the filesystem using absolute or relative paths.
- Learn `.` `..` `~` and `-` because they speed up navigation.
- Combine `pwd`, `ls`, and `cd` in a repeatable workflow to reduce mistakes.
