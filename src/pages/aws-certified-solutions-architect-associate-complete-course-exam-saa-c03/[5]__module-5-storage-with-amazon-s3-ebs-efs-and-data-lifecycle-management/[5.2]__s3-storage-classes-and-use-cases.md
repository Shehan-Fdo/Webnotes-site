## **Course:** AWS Certified Solutions Architect – Associate Complete Course (Exam SAA-C03)

**S3 storage class fundamentals**

Amazon S3 storage classes let you trade off storage cost, retrieval speed, availability, and resilience based on how often you access data. For the exam, the key skill is matching the workload pattern to the class rather than memorizing every price point.

S3 classes fall into three broad groups:

- **Frequent access** classes for active data.
- **Infrequent access** classes for data you still need quickly, but not often.
- **Archive** classes for long-term retention where retrieval can be delayed.

A useful mental model is:

- If you need immediate access and high durability/availability, start with a general-purpose class.
- If access is unpredictable, use Intelligent-Tiering.
- If data is rarely read and can tolerate restore delays, use an archive class.

**Core storage classes at a glance**

| Storage class | Best for | Retrieval latency | Availability model | Exam cue |
|---|---|---:|---|---|
| S3 Standard | Frequently accessed data | Milliseconds | Multi-AZ | Default choice for active workloads |
| S3 Intelligent-Tiering | Unknown or changing access patterns | Milliseconds for active tiers | Multi-AZ | Let S3 optimize cost automatically |
| S3 Standard-IA | Infrequent access, but immediate retrieval needed | Milliseconds | Multi-AZ | Lower storage cost, retrieval fee applies |
| S3 One Zone-IA | Infrequent access, non-critical data | Milliseconds | Single AZ | Cheaper, but less resilient |
| S3 Glacier Instant Retrieval | Archive data needing instant access | Milliseconds | Multi-AZ | Archive economics with immediate access |
| S3 Glacier Flexible Retrieval | Archive data with minutes-to-hours restore tolerance | Minutes to hours | Multi-AZ | Choose for low-cost archive retrieval |
| S3 Glacier Deep Archive | Long-term retention, very rare access | Hours | Multi-AZ | Lowest-cost archive, slowest restore |
| S3 Express One Zone | Ultra-low latency, high request rate workloads | Microseconds to low milliseconds | Single AZ | Specialized high-performance class |

> <mark>Exam tip:</mark> S3 Standard is the highest-cost among the common multi-purpose classes, but not the most expensive S3 option overall. S3 Express One Zone is a specialized high-performance class and is not the default answer for general workloads.

**S3 Standard and Standard-IA**

S3 Standard is the default choice when you need frequent access, strong durability, and high availability across multiple Availability Zones (AZs). It is the safest answer for active application data, websites, analytics inputs, and content that users read often.

Use S3 Standard when:

- Data is read and written frequently.
- You need immediate access without retrieval charges.
- The workload is critical and should remain multi-AZ.

S3 Standard-IA is for data you access less often, but still need immediately when requested. It keeps the same multi-AZ durability and availability expectations as Standard, but lowers storage cost in exchange for retrieval fees and minimum storage duration considerations.

Use S3 Standard-IA when:

- Data is infrequently accessed, but latency still matters.
- You want lower storage cost than Standard.
- You can tolerate retrieval charges and minimum storage duration rules.

Common examples:

- Backups that are occasionally restored.
- Disaster recovery copies.
- Older application assets that are still served sometimes.

**Standard-IA vs Glacier Instant Retrieval**

This distinction shows up often on the exam.

- **Standard-IA** is for infrequent access with immediate retrieval and a more general-purpose feel.
- **Glacier Instant Retrieval** is for archive-oriented data that still needs millisecond access, often at a lower storage cost for very cold data patterns.

Choose Standard-IA when:

- The data is infrequently accessed, but still part of a normal operational workflow.
- You want a familiar general-purpose infrequent-access class.

Choose Glacier Instant Retrieval when:

- The data is archival in nature.
- You want instant access, but the data is colder and more archive-like than Standard-IA use cases.

A practical rule:

- If the question says “infrequent access, immediate retrieval, general-purpose,” think **Standard-IA**.
- If the question says “archive data, instant retrieval,” think **Glacier Instant Retrieval**.

**Intelligent-Tiering and access pattern uncertainty**

S3 Intelligent-Tiering is designed for data with unknown, changing, or unpredictable access patterns. It automatically moves objects between access tiers based on observed usage, which helps reduce cost without requiring you to predict the pattern up front.

It is especially useful when:

- You do not know whether data will stay hot or become cold.
- Access patterns change over time.
- You want to reduce manual lifecycle tuning.

Intelligent-Tiering includes multiple access tiers:

- **Frequent Access tier** for active objects.
- **Infrequent Access tier** for objects not accessed recently.
- Optional archive tiers:
  - **Archive Access tier**
  - **Deep Archive Access tier**

Important exam detail:

- Objects in the archive tiers are not immediately retrievable like the active tiers.
- Retrieval from archive tiers follows archive-style restore behavior and delay.

Use Intelligent-Tiering when:

- You want automatic optimization.
- You have large datasets with uncertain access patterns.
- You want to avoid overpaying for data that may become cold.

A common real-world example is a data lake where some partitions are queried heavily for a few weeks and then become dormant. Intelligent-Tiering can reduce cost without requiring you to constantly rewrite lifecycle rules.

**One Zone classes and resilience tradeoffs**

One Zone classes store data in a single Availability Zone. That makes them cheaper, but also less resilient than multi-AZ classes.

Use One Zone-IA when:

- Data can be recreated.
- You can tolerate AZ loss.
- Cost matters more than resilience.

Do not use One Zone classes for:

- Critical production data.
- Compliance-sensitive data that needs stronger fault tolerance.
- Workloads that must survive an AZ outage without data loss.

Use this rule of thumb:

- **Standard** and **Standard-IA** are the default multi-AZ choices for general durability and availability expectations.
- **One Zone** classes are for non-critical or easily reproducible data.

**Archive classes and restore behavior**

Archive classes reduce storage cost dramatically, but retrieval is delayed and may require a restore operation before you can read the object.

The archive family includes:

- **S3 Glacier Instant Retrieval**
- **S3 Glacier Flexible Retrieval**
- **S3 Glacier Deep Archive**

**S3 Glacier Flexible Retrieval**

This class is the classic archive option for data that is rarely accessed but still needs to be recoverable. It supports different restore speeds, which is a common exam topic.

Restore options:

- **Expedited**: fastest, for urgent retrieval.
- **Standard**: balanced cost and speed.
- **Bulk**: lowest cost, for non-urgent retrieval.

Use Glacier Flexible Retrieval when:

- You need low-cost archival storage.
- You can wait minutes to hours for restore completion.
- You want a choice between urgent and low-cost retrieval.

**S3 Glacier Deep Archive**

This is the lowest-cost S3 storage class for long-term retention. It is intended for data you almost never access, such as compliance archives or historical records.

Use Deep Archive when:

- Data is retained for years.
- Retrieval is extremely rare.
- Hours of restore time are acceptable.

**Restore semantics you should know**

When you restore an archived object:

- S3 creates a temporary restored copy that is readable for a limited time.
- The original object remains in the archive class.
- You may incur separate storage charges for the restored copy during the restore window.

This matters on the exam because a restored object is not permanently moved out of the archive class unless you explicitly copy or transition it.

> <mark>Warning:</mark> Archive restores are not the same as a permanent storage-class change. A restore makes the object temporarily available; it does not rewrite the object’s underlying class by itself.

**S3 Express One Zone**

S3 Express One Zone is a specialized class for very high request rates and very low latency. It is not a general replacement for Standard.

Use it when:

- You need extremely fast access.
- The workload is latency-sensitive and performance-critical.
- You can accept single-AZ design tradeoffs.

Do not choose it for:

- General-purpose storage.
- Long-term archive use.
- Workloads that require multi-AZ resilience by default.

For exam purposes, treat it as a specialized performance class rather than a default storage recommendation.

**Lifecycle policies and transitions**

S3 Lifecycle rules automate transitions between storage classes and can also expire objects. They are useful when data naturally becomes colder over time.

Typical lifecycle flow:

- New objects start in S3 Standard.
- After a period of inactivity, they transition to Standard-IA or Intelligent-Tiering.
- Much older data may move to Glacier classes.
- Eventually, objects can expire if retention is no longer needed.

Example lifecycle configuration:

```json
{
  "Rules": [
    {
      "ID": "MoveLogsToColdStorage",
      "Status": "Enabled",
      "Filter": {
        "Prefix": "logs/"
      },
      "Transitions": [
        {
          "Days": 30,
          "StorageClass": "STANDARD_IA"
        },
        {
          "Days": 90,
          "StorageClass": "GLACIER"
        }
      ],
      "Expiration": {
        "Days": 365
      }
    }
  ]
}
```

In current terminology, `GLACIER` is the legacy name used in configuration for the S3 Glacier Flexible Retrieval family.

Lifecycle rules are powerful, but they are not always immediate. Be aware of these practical constraints:

- Some transitions are subject to minimum object size or transition eligibility rules.
- Newly uploaded objects may not be eligible for certain transitions immediately.
- Frequent transitions can increase request overhead and complicate cost modeling.
- Not every class-to-class transition is equally useful for every workload.

A good exam habit is to think about lifecycle as a policy engine, not a magic cost reducer. It works best when the data lifecycle is predictable.

**Storage class analysis and monitoring**

When you are unsure whether Standard-IA or Intelligent-Tiering is the better choice, monitoring helps.

Useful tools and features:

- **S3 Storage Class Analysis** to observe access patterns.
- **S3 Inventory** to report object metadata at scale.
- **CloudWatch** metrics for request and storage trends.
- **AWS Cost Explorer** for cost visibility.

Use analysis when:

- You have a large dataset and uncertain access frequency.
- You want evidence before moving data to colder storage.
- You need to justify lifecycle policy changes.

A practical workflow is:

1. Start with Standard or Intelligent-Tiering.
2. Observe access patterns with analysis tools.
3. Transition colder data with lifecycle rules once the pattern is clear.

**Decision matrix for exam scenarios**

| Scenario | Best choice | Why |
|---|---|---|
| Frequently accessed application data | S3 Standard | Immediate access and multi-AZ durability |
| Unknown or changing access pattern | S3 Intelligent-Tiering | Automatic cost optimization |
| Infrequent access, immediate retrieval needed | S3 Standard-IA | Lower storage cost with fast access |
| Infrequent access, non-critical data | S3 One Zone-IA | Lower cost, single-AZ acceptable |
| Archive data, instant access required | S3 Glacier Instant Retrieval | Archive economics with millisecond access |
| Archive data, restore can take minutes to hours | S3 Glacier Flexible Retrieval | Low-cost archive with restore options |
| Long-term retention, very rare access | S3 Glacier Deep Archive | Lowest storage cost, slow restore |
| Ultra-low latency, specialized workload | S3 Express One Zone | Performance-focused, single-AZ design |

**Practical workload examples**

A media company stores newly uploaded video files that editors access constantly during production. Those files belong in S3 Standard. After the project ships, the files become less active and can transition to Standard-IA or Intelligent-Tiering.

A security team keeps audit logs for seven years. The logs are rarely read, but they must be retained. A lifecycle policy can move them from Standard to Glacier Flexible Retrieval, and eventually to Deep Archive if the organization wants the lowest long-term cost.

A data science team stores feature datasets with unpredictable access. Some datasets are queried heavily for a few days, then ignored for months. Intelligent-Tiering is a strong fit because it adapts without manual tuning.

A backup system stores nightly snapshots that are rarely restored, but when restored they must be available quickly. Standard-IA is often a better fit than archive classes because the restore should be immediate.

**CLI examples**

You can set a storage class at upload time with the AWS CLI (Command Line Interface):

```bash
aws s3 cp report.csv s3://my-bucket/archive/report.csv --storage-class STANDARD_IA
```

You can also copy an existing object to a new key with a different storage class:

```bash
aws s3api copy-object \
  --copy-source my-bucket/logs/app.log \
  --bucket my-bucket \
  --key archive/app.log \
  --storage-class GLACIER
```

For lifecycle management, you typically apply a bucket lifecycle configuration:

```bash
aws s3api put-bucket-lifecycle-configuration \
  --bucket my-bucket \
  --lifecycle-configuration file://lifecycle.json
```

A restore request for an archived object might look like this:

```bash
aws s3api restore-object \
  --bucket my-bucket \
  --key archive/app.log \
  --restore-request '{
    "Days": 7,
    "GlacierJobParameters": {
      "Tier": "Standard"
    }
  }'
```

**How to choose quickly on the exam**

1. Ask whether the data is accessed frequently.
2. If access is unpredictable, choose Intelligent-Tiering.
3. If access is infrequent but immediate, choose Standard-IA or Glacier Instant Retrieval.
4. If the data is archive-only and can wait, choose Glacier Flexible Retrieval or Deep Archive.
5. If the workload is non-critical and single-AZ is acceptable, consider One Zone-IA.
6. If the question emphasizes ultra-low latency and high request rates, consider S3 Express One Zone.

**Common exam traps**

- Choosing an archive class when the question requires immediate retrieval.
- Choosing One Zone-IA for critical data that must survive an AZ outage.
- Assuming archive restores permanently change the storage class.
- Using Standard when the workload is clearly cold and long-lived.
- Ignoring Intelligent-Tiering when the access pattern is unknown.
- Treating Glacier Flexible Retrieval and Deep Archive as interchangeable; they differ mainly in restore speed and cost.

**Summary**

S3 storage class selection is mostly about matching access frequency, retrieval latency, and resilience requirements. Standard and Standard-IA are the common multi-AZ choices for general workloads, Intelligent-Tiering is the safest option when access is uncertain, and the Glacier family is for archive use cases where restore delay is acceptable.

If you remember only a few rules, remember these:

- **Standard** for active data.
- **Standard-IA** for infrequent but immediate access.
- **Intelligent-Tiering** for unknown patterns.
- **Glacier Flexible Retrieval** for low-cost archive with restore options.
- **Glacier Deep Archive** for the coldest long-term retention.
- **One Zone** classes only when single-AZ risk is acceptable.
