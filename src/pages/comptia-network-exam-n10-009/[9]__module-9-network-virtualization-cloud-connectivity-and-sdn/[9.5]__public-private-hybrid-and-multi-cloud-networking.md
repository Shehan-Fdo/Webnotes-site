## **Course:** CompTIA Network+ (Exam N10-009)

**Cloud deployment model basics**

Cloud computing delivers resources such as servers, storage, networking, and software over the internet. For Network+ study, focus on how deployment models change where infrastructure lives, who manages it, and how workloads move between environments.

- **Public cloud** uses third-party infrastructure and is available over the internet.
- **Private cloud** is dedicated to one organization and offers greater control.
- **Hybrid cloud** combines on-premises, private cloud, public cloud, and edge environments.
- **Multicloud** uses multiple cloud providers instead of relying on a single vendor.

Public cloud is commonly associated with scalability, cost efficiency, and a multitenant model. It is a practical fit for dynamic workloads and rapid innovation because the provider handles much of the underlying administration and security. Public cloud services are often delivered as infrastructure as a service (IaaS), platform as a service (PaaS), and software as a service (SaaS).

**Public cloud characteristics**

Public cloud services are offered through hyperscale providers and accessed through the internet. Common examples include major cloud service providers such as AWS, Azure, GCP, and OCI.

- **Pay-per-use** pricing helps align cost with actual consumption.
- **Shared infrastructure** supports broad availability and elasticity.
- **Provider-managed operations** reduce the burden on internal IT teams.
- **Service variety** can include IaaS, PaaS, and SaaS.

Private cloud is built for a single organization and is typically chosen when control, security, or governance requirements are higher. It can support more customized administration than a public cloud, but it usually trades some of the simplicity and elasticity of a public offering for that control.

**Hybrid cloud networking**

Hybrid cloud networking securely connects public and private environments so you can place workloads where they fit best. This model is useful when you need to balance agility with data security or workload sensitivity.

- **Workload placement** can vary by application, data, and compliance needs.
- **Scalability** lets you expand resources when demand increases.
- **Flexibility** supports modernization of legacy applications.
- **Single managed infrastructure** is the goal, even though the environment spans multiple locations.

> Hybrid cloud is not a one-size-fits-all design. The mix of on-premises, private cloud, public cloud, and edge resources depends on business needs and use cases.

**Multicloud networking**

Multicloud means using several cloud providers in the same overall strategy. Instead of relying on one vendor for hosting, storage, and the full application stack, you distribute services across multiple providers.

- **Vendor diversity** reduces dependence on a single cloud platform.
- **Service selection** lets you choose the best provider for a given workload.
- **Connectivity planning** becomes more important as environments spread across providers.
- **Operational consistency** is harder to maintain than in a single-cloud design.

Multicloud is often discussed alongside hybrid cloud, but they are not the same thing. Hybrid describes the mix of environments; multicloud describes the use of multiple cloud providers. A design can be hybrid, multicloud, or both.

**How to compare the models**

| Model | Main idea | Common strength | Common trade-off |
|---|---|---|---|
| Public cloud | Third-party shared infrastructure | Fast scaling and lower upfront cost | Less direct control |
| Private cloud | Dedicated environment for one organization | Greater control and customization | More management responsibility |
| Hybrid cloud | Connected mix of on-premises and cloud environments | Flexible workload placement | More architectural complexity |
| Multicloud | Multiple cloud providers | Avoids single-vendor dependence | Harder integration and governance |

**Study focus for Network+**

When you see cloud questions on the exam, identify the deployment model first, then match it to the business need. Ask whether the scenario emphasizes cost, control, scalability, workload sensitivity, or provider diversity.

1. Identify whether the environment is public, private, hybrid, or multicloud.
2. Look for clues about workload placement, security, and management responsibility.
3. Match the model to the stated goal, such as agility, governance, or vendor flexibility.

**Key terms to remember**

- **Multitenant**: multiple customers share provider infrastructure.
- **IaaS**: virtualized computing resources delivered over the internet.
- **PaaS**: a managed platform for building and deploying applications.
- **SaaS**: software delivered as a hosted service.
- **Edge**: computing resources placed closer to where data is generated or used.
