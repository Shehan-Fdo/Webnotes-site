## **Course:** CompTIA Network+ (Exam N10-009)

**Virtualization in network design**

Virtualization moves networked services from many physical devices into software-based environments. From a network perspective, that changes how traffic is switched, segmented, and connected to the rest of the infrastructure. You still need the same core goals: connectivity, isolation, and predictable communication.

A **virtual network** is a logical network that lets devices and services communicate without being physically connected. In practice, virtualization can improve flexibility and resource use while still supporting separation between workloads.

- Virtualized environments collapse many servers into fewer physical hosts.
- Network design still depends on switching, routing, and segmentation.
- Virtual networks can support isolation and independent management.

**Virtual switches and virtual NICs**

A **virtual switch** lets virtual machines communicate with other computers and network resources. On Hyper-V, you can create and configure the switch with Hyper-V Manager or PowerShell. When you install the Hyper-V role on Windows Server, you can optionally create a virtual switch during setup.

A **virtual network interface card** (<abbr title="virtual network interface card">vNIC</abbr>) is the virtual adapter a machine uses to reach the network through the virtual switch. In exam terms, connect the idea of the vNIC to the virtual machine and the virtual switch to the host-side forwarding function.

- A virtual switch provides connectivity for virtual machines on a Hyper-V host.
- You can create it with management tools or PowerShell.
- The host may create the switch during Hyper-V installation.

**VLANs in virtualized networks**

A **virtual local area network** (<abbr title="virtual local area network">VLAN</abbr>) isolates network traffic. VLANs are configured on switches and routers that support `802.1Q` tagging. If you need communication between multiple VLANs, you must configure the network devices to allow it.

For Hyper-V VLAN use, the physical adapter, its driver, and the physical switch must support `802.1Q` VLAN tagging. On the host, configure the virtual switch so the physical switch port allows the VLAN IDs used by the virtual machines.

> <mark>Critical:</mark> VLAN support depends on both the physical network hardware and the virtual switch configuration. If either side does not support `802.1Q`, VLAN tagging will not work as expected.

- VLANs separate traffic into logical segments.
- Inter-VLAN communication requires explicit network device configuration.
- Hyper-V VLAN setups need compatible adapter, driver, and switch support.

**How virtualization changes the network**

Virtualization does not remove the need for network planning. It changes where the functions live. Instead of many physical servers each with their own direct network presence, you place multiple virtual servers on one physical system and connect them through virtual networking components.

This is why virtualization topics often include virtual switches, vNICs, and VLANs together. They describe how traffic enters, moves through, and exits the virtual environment.

- Virtualization concentrates workloads on fewer hosts.
- Network functions still need segmentation and connectivity.
- Virtual components mirror physical networking roles.

**Study focus**

When you review this topic, make sure you can explain the relationship between the physical network and the virtual layer. You should also be able to identify where VLAN tagging is required and why a virtual switch matters to VM connectivity.

1. Define a virtual network in simple terms.
2. Explain what a virtual switch does for virtual machines.
3. Describe why `802.1Q` support matters for VLANs.
4. Connect VLAN configuration to both the host and the physical network.

**Quick comparison**

| Concept | Role |
|---|---|
| Virtual network | Logical network for communication without physical attachment |
| Virtual switch | Connects virtual machines to other systems |
| vNIC | Virtual adapter used by a machine to access the network |
| VLAN | Segments traffic into isolated logical networks |

**Key takeaways**

Virtualization changes the network from a purely physical model to a software-defined one, but the same design goals remain. You still need switching, segmentation, and compatibility between host, adapter, and switch when you use VLANs in a virtual environment.
