## **Course:** CompTIA Network+ (Exam N10-009)

**Overview**

VMware NSX and Cisco Application Centric Infrastructure (ACI) are both data center network virtualization platforms, but they solve the problem from different layers. NSX is described as a software-based platform that virtualizes networking from the hypervisor up, while ACI is a hardware-integrated fabric that manages physical and virtual infrastructure through an application-centric policy model.

For exam prep, focus on the architectural distinction first. That difference explains most of the practical comparisons you may see in design, operations, and automation questions.

- **NSX** emphasizes software-defined networking on hypervisors such as ESXi or KVM.
- **ACI** emphasizes a fabric approach that includes physical and virtual infrastructure.
- Both platforms support centralized policy and automation.
- Both can appear in the same enterprise environment when requirements span multiple layers.

**Core architecture differences**

NSX uses an overlay model that virtualizes the network in software. Evidence describes GENEVE as the overlay for NSX-T, with physical hardware from any vendor providing underlay connectivity. It also includes distributed networking services such as routing, switching, distributed firewall, IDS, load balancing, VPN, NAT, DHCP, and DNS relay.

ACI uses a fabric model built around Cisco infrastructure. Evidence describes ACI as integrating with hardware-based switches such as the Nexus 9000 series and using VXLAN as an overlay, with IS-IS and MP-BGP handling underlay routing. ACI is also described as enabling centralized routing, switching, and stateless firewalling across the fabric.

| Area | VMware NSX | Cisco ACI |
|---|---|---|
| Primary model | Software-based overlay | Hardware-integrated fabric |
| Typical scope | Virtual side of the data center | Physical and virtual sides of the data center |
| Overlay mentioned in evidence | GENEVE | VXLAN |
| Underlay mentioned in evidence | Any vendor hardware | IS-IS and MP-BGP |
| Common services mentioned | Distributed firewall, routing, switching, IDS, load balancing, VPN, NAT, DHCP, DNS relay | Centralized routing, switching, stateless firewalling |

**How to think about the platforms**

NSX is useful when you want networking to follow the virtual machine or workload in software. ACI is useful when you want policy to extend across a fabric that includes both physical and virtual resources. The evidence consistently frames them as complementary in many environments rather than direct substitutes.

> Do not assume one platform replaces the other in every design. The practical choice depends on whether the requirement is centered on hypervisor-level virtualization, fabric-wide policy, or both.

- **Complementary use** is common in enterprise deployments.
- **Microsegmentation** is a shared goal, but the implementation model differs.
- **Automation** exists in both platforms, but the control plane and operational model are not the same.
- **Scalability** is a design goal in both, especially in data center environments.

**Cisco ACI terminology and deployment context**

ACI is often discussed in the context of leaf-spine data center fabrics. The evidence references a lab with spines and leaf switches, which reflects the common fabric-oriented way ACI is introduced. In practice, you should recognize that ACI is not just a virtual overlay product; it is tied to the physical fabric and its policy model.

Key terms to recognize:

- **Fabric**: the interconnected switching environment that carries policy and traffic.
- **Leaf switch**: the access layer in a leaf-spine design.
- **Spine switch**: the core interconnect layer in a leaf-spine design.
- **Policy model**: the application-centric rules that define how traffic is handled.

**Operational comparison**

ACI and NSX both simplify network operations, but they do so differently. NSX focuses on software-defined control over virtual networking, while ACI focuses on centralized policy across a fabric that includes physical and virtual endpoints. If you are asked which one is “more virtual,” NSX is the better fit. If you are asked which one is more fabric-integrated, ACI is the better fit.

- **NSX** aligns with hypervisor-centric environments.
- **ACI** aligns with fabric-centric environments.
- **Both** can support segmentation and policy enforcement.
- **Neither** should be treated as a universal replacement for the other.

**Exam focus**

For Network+ purposes, the most important takeaway is the relationship between overlay, underlay, and control model. You should be able to identify which platform is software-centric and which is fabric-centric, and you should know the overlay protocols named in the evidence.

1. Identify whether the question describes a software overlay or a hardware-integrated fabric.
2. Match **GENEVE** with NSX-T when the prompt mentions software-based overlay networking.
3. Match **VXLAN** with ACI when the prompt mentions Cisco fabric integration.
4. Look for underlay clues such as vendor-neutral hardware for NSX or IS-IS and MP-BGP for ACI.
5. Use the service list to distinguish NSX’s distributed services from ACI’s centralized fabric policy.

**Quick memory aid**

- **NSX**: software-first, hypervisor-based, overlay-driven.
- **ACI**: fabric-first, hardware-integrated, policy-driven.
- **Both**: enterprise data center networking, segmentation, and automation.
