## **Course:** CompTIA Network+ (Exam N10-009)

**SD-WAN overview**

<abbr title="Software-defined Wide Area Network">SD-WAN</abbr> applies software-defined networking ideas to WAN (Wide Area Network) traffic. It uses predefined policies to choose effective paths for application traffic across branch offices, headquarters, cloud services, and the Internet.

SD-WAN commonly abstracts multiple transport types into a virtual overlay. That lets you keep using existing WAN links while steering traffic based on application needs and network conditions.

- SD-WAN focuses on application-aware traffic handling.
- It can use broadband, LTE/5G, fiber, Internet, or MPLS as transport.
- A centralized controller can distribute policy to connected devices.
- Branch devices often need less manual router configuration.

**Centralized policy behavior**

Centralized policy gives the controller a way to push consistent rules across the SD-WAN fabric. In practical terms, this reduces per-site configuration and helps keep routing and forwarding decisions aligned with the intended design.

Without a centralized control policy, the default behavior is that no centralized policy is provisioned unless you configure and apply one. In that case, edge routers still exchange route information with the controller using Overlay Management Protocol (<abbr title="Overlay Management Protocol">OMP</abbr>).

- vEdge routers send learned route prefixes from local sites to vSmart controllers via OMP.
- If a vEdge has multiple <abbr title="Data Termination Session">DTLS</abbr> connections to multiple vSmart controllers, it sends OMP information to all of them.
- vEdge routers send TLOC routes and service routes to vSmart via OMP.
- vSmart accepts these route types, tracks them, and uses them to determine VPN membership and topology.
- vSmart then redistributes routes to vEdge routers in the same VPN.

**Control plane impact**

Centralized control policy affects the control plane, not just forwarding behavior. That makes it useful when you need to influence how routes are redistributed or how route attributes are interpreted before they reach the data plane.

Localized control policy can also affect control-plane decisions. One example is checking an OMP tag in an OMP route and changing the metric type for redistribution into OSPF (Open Shortest Path First). This is similar in spirit to route-map style policy logic.

> Centralized policy is not the same as a data policy. Data policies act more like access control lists, while control policies influence route handling and redistribution.

**Policy types to remember**

- **Localized data policy**: used for traffic filtering or steering decisions at a site.
- **Localized control policy**: changes control-plane behavior such as route redistribution.
- **Centralized policy**: managed from a controller and applied across the SD-WAN environment.

A useful exam distinction is that centralized policies rely heavily on reusable lists. That makes them easier to scale across many sites than repeating site-by-site rules.

**Route exchange and topology awareness**

The controller builds a network topology map from the route information it receives. It uses that information to determine the routing path for data traffic and to redistribute routes within the appropriate VPN.

This means the controller is not just a passive repository. It participates in understanding the overlay and in distributing the policy and route information that keeps the fabric consistent.

- Route prefixes come from local site networks.
- TLOC routes identify transport locations.
- Service routes describe reachable services.
- The controller uses these inputs to understand VPN placement and path selection.

**Operational benefits**

SD-WAN centralization supports simpler operations and more consistent policy enforcement. It also helps reduce the need for manual changes at branch locations.

- Remote edge devices can be programmed from a central controller.
- Low-touch or zero-touch provisioning reduces deployment effort.
- Policy-based routing improves application traffic handling.
- Existing WAN links can remain in service under the overlay.

**Study focus for Network+**

For exam prep, focus on the relationship between the controller, the edge devices, and the policy model.

1. Identify whether a question is about the data plane or the control plane.
2. Recognize that centralized policy is pushed from a controller.
3. Remember that OMP carries route information between vEdge routers and vSmart controllers.
4. Distinguish centralized policy from localized policy.
5. Connect SD-WAN to application-aware path selection and transport abstraction.

**Quick comparison**

| Topic | Practical meaning |
|---|---|
| SD-WAN | Software-driven WAN traffic management |
| Centralized policy | Controller-managed rules across the fabric |
| Localized policy | Site-level policy applied closer to the edge |
| OMP | Protocol used for route exchange in the overlay |

**Key takeaway**

SD-WAN centralizes control so you can steer traffic with policy instead of manual per-device configuration. For the exam, know how centralized policy changes route handling, how OMP supports the overlay, and how the controller uses route information to build and maintain the topology.
