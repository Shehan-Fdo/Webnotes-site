## **Course:** CompTIA Network+ (Exam N10-009)

**Segmentation overview**

Network segmentation separates different parts of a network into distinct zones using devices such as firewalls, switches, and routers. This reduces exposure by limiting how far traffic can move if one area is compromised.

Segmentation is a practical security control for modern networks because it helps contain incidents and reduce the severity of security events. It is used in data centers and on-premises environments, where different systems need different levels of access.

- **Network zones** divide traffic into smaller trust boundaries.
- **Firewalls** and routing devices help enforce separation between zones.
- **Incident containment** improves because a compromise in one segment does not automatically expose the rest of the network.

**Microsegmentation**

Microsegmentation applies the same idea at a much finer level. Instead of separating only broad network zones, it isolates machines, applications, and workloads so each segment has its own security perimeter.

This approach reduces the attack surface and limits lateral movement, which is the ability of an attacker to move from one system to another after an initial compromise. If one workload is breached, microsegmentation helps keep the rest of the environment protected.

- **Isolation** is applied to individual workloads or applications.
- **Lateral movement** becomes harder because each segment has tighter controls.
- **Compliance** and operational control improve when access is more precisely defined.

> Successful microsegmentation improves enterprise cybersecurity and availability, but it works best when you plan the boundaries and enforcement points carefully.

**Zero Trust principles**

Zero Trust is a security model built on the idea that you do not automatically trust any user, device, or application. Instead, you verify every request and grant only the access that is needed.

The model is often summarized as “never trust, always verify.” It reflects a shift away from perimeter-based security, where anything inside the network is treated as safe. In Zero Trust, identity is the new perimeter, and continuous verification matters more than location.

- **Least privilege** gives each user or device only the access required for the task.
- **Continuous verification** checks users, devices, and sessions instead of trusting them once.
- **Identity-driven access** becomes central because compromised credentials are a major risk.

**How segmentation supports Zero Trust**

Segmentation and microsegmentation are important building blocks for Zero Trust because they help enforce smaller trust boundaries. They make it easier to verify traffic, restrict access, and reduce the impact of a breach.

Zero Trust also fits modern environments where remote work, cloud services, mobile devices, and hybrid infrastructures make perimeter-only defenses less effective. In these environments, you assume breaches can happen and design controls to limit damage.

- **Policy-based access** helps ensure users reach only approved resources.
- **Encryption** and authentication strengthen trust decisions inside the network.
- **Monitoring** supports verification and helps detect suspicious activity.

**Practical implementation focus**

When you plan segmentation or microsegmentation, start by identifying what needs to be separated and why. Then define the access rules that support business needs without creating unnecessary trust.

1. Identify the network zones, workloads, or applications that need isolation.
2. Decide which devices, users, and services must communicate.
3. Apply controls with firewalls, switches, routers, authentication, and encryption where appropriate.
4. Verify that traffic is limited to the intended paths.
5. Review the design regularly so it continues to match the environment.

**Key takeaways**

Segmentation reduces risk by dividing the network into smaller zones. Microsegmentation goes further by isolating workloads and applications individually. Zero Trust uses verification, least privilege, and continuous checking to remove implicit trust from the network.

Together, these ideas support stronger resilience, better containment, and a more practical security posture for modern environments.
