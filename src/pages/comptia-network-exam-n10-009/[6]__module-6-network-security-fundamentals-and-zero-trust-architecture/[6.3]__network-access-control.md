## **Course:** CompTIA Network+ (Exam N10-009)

**Network Access Control overview**

Network Access Control (**NAC**) is a security approach that manages who and what can connect to a network. It uses predefined policies to evaluate users, devices, and their condition before granting access. In practice, NAC helps you allow authorized endpoints while restricting or blocking non-compliant ones.

NAC is especially useful in environments where many devices connect to wired and wireless networks. It supports stronger security by reducing unauthorized access, limiting malware spread, and helping maintain overall network health.

- **Policy-based access** evaluates connection requests against rules before access is granted.
- **Authorization** can vary by user, device type, location, or system health.
- **Segmentation** helps isolate devices or limit their reach after authentication.
- **Compliance checks** can block or restrict endpoints that do not meet security requirements.

**802.1X and port-based access control**

IEEE 802.1X is a standard for <abbr title="Port-Based Network Access Control">PNAC</abbr> that provides secure authentication for devices connecting to a <abbr title="Local Area Network">LAN</abbr> or <abbr title="Wireless Local Area Network">WLAN</abbr>. It uses a <abbr title="Remote Authentication Dial-In User Service">RADIUS</abbr> server to verify credentials and then grants access based on network policy.

802.1X is commonly used at the switch or wireless edge, where it can authenticate and authorize a device before that device reaches internal resources. This makes the access decision more proactive than relying only on perimeter controls.

- **RADIUS** centralizes authentication decisions for network access.
- **Unique credentials** per user or certificate reduce the risk of a shared password being stolen.
- **Edge enforcement** helps stop unauthorized devices at the point of connection.
- **Policy variation** lets you assign different access levels after verification.

> 802.1X is about verifying identity before access is allowed, not after a device is already trusted on the network.

**How NAC fits into network security**

NAC supports a broader security strategy by controlling access at multiple points in the network. Traditional controls such as access control lists (ACLs) and VLAN segmentation still matter, but NAC adds a more dynamic layer by checking identity and device posture before access is granted.

This aligns well with a zero trust mindset, where users, devices, and applications are not trusted by default. Instead, you verify each request and limit access to only what is needed.

- **Zero trust** treats users and devices as untrusted until verified.
- **Least privilege** limits access to only the resources required.
- **Authentication** confirms identity before network use.
- **Encryption** and other controls can complement NAC inside the network.

**Common NAC decision factors**

NAC policies often consider more than just a username and password. They can use multiple attributes to decide whether to allow access and what level of access to provide.

- **Identity**: who the user is.
- **Device type**: what kind of endpoint is connecting.
- **Location**: where the connection originates.
- **System health**: whether the device meets security requirements.

These checks help you make access decisions that are more precise than a simple allow-or-deny rule. They also help reduce risk in environments with bring your own device (**BYOD**) and remote access needs.

**Practical deployment considerations**

When you deploy NAC on switches or wireless infrastructure, you place authentication close to the access point. That helps protect critical assets by ensuring only approved devices and users can join the network.

A practical NAC design usually includes centralized policy control, endpoint verification, and enforcement at the network edge. If a device fails policy checks, you can block it or place it into a restricted segment.

1. Identify the access point where authentication should occur.
2. Define the policy criteria for users and devices.
3. Configure the authentication method, such as 802.1X with RADIUS.
4. Decide what happens when a device passes or fails policy checks.
5. Verify that authorized devices receive the intended level of access.

**Why NAC matters for Network+**

For Network+ purposes, NAC is important because it connects security hardening with everyday network operations. You should understand how it protects wired and wireless access, how 802.1X supports authentication, and how policy-based decisions improve control over network resources.

NAC is not just a single product or protocol. It is a framework for enforcing access rules, improving visibility, and reducing the chance that unauthorized or unhealthy devices can spread risk across the network.
