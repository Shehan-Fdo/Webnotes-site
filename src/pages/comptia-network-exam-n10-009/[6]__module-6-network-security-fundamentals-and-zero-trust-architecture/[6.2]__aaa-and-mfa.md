## **Course:** CompTIA Network+ (Exam N10-009)

**AAA overview**

<abbr title="Authentication, Authorization, and Accounting">AAA</abbr> is a security framework that controls access to resources, enforces policies, and audits usage. It supports network management and cybersecurity by screening users and tracking what they do while connected.

- **Authentication** verifies who you are by checking submitted credentials against stored records.
- **Authorization** determines what an authenticated user or device can access.
- **Accounting** records activity for auditing, monitoring, and usage analysis.

AAA aligns with broader security goals by helping ensure that only approved users gain access and that their actions are visible for review. In network environments, AAA often appears as a centralized access-control model rather than a single feature on one device.

**How AAA works in practice**

AAA commonly uses a client-server flow. A network device or access server requests a decision from an AAA server, which evaluates identity and policy before allowing access.

- A user presents a username and password or other credentials.
- The AAA system checks those credentials against its database or identity source.
- If the user is valid, the system applies the appropriate access policy.
- The system logs the session for later review.

Protocols such as `RADIUS` and `TACACS+` are used to carry AAA-related requests and responses across the network. These protocols support centralized control in distributed environments.

> Use conservative wording on the exam: AAA is a framework and set of processes, not a single login screen.

**Authentication and identity**

Authentication is the first step in the AAA process. It confirms identity, usually by comparing a claimed identity with evidence such as a password, token, or other credential.

- A username or email address is identification, not proof of access.
- A password alone is a single factor and can be weak if stolen or guessed.
- Stronger authentication methods add more verification before access is granted.

Authentication is often the part users notice most, but it is only one piece of the larger access-control model.

**Multi-factor authentication**

<abbr title="Multi-Factor Authentication">MFA</abbr> requires more than one type of proof before access is granted. It adds layers of security to internet-enabled services and reduces reliance on a password alone.

Common examples include:

- A password plus a one-time code from an authentication app
- A password plus a code sent to email or text
- A password plus a biometric factor such as a fingerprint

MFA is especially useful when you want to reduce the risk of unauthorized access from stolen credentials. It is a practical control for portals, remote access, and other services where identity verification matters.

**AAA and MFA relationship**

AAA and MFA solve different parts of the access problem.

| Concept | Primary purpose | Typical result |
|---|---|---|
| AAA | Control, policy, and auditing | Centralized access management |
| MFA | Stronger identity verification | Reduced risk from password-only access |

AAA provides the framework for deciding who gets in, what they can do, and what gets recorded. MFA strengthens the authentication step inside that framework.

**Zero trust connection**

Zero trust treats every user, device, and application as untrusted until verified. That approach depends heavily on authentication and policy enforcement, which makes AAA and MFA important supporting controls.

- Verify access continuously instead of assuming trust after initial login.
- Apply least privilege so users only reach the resources they need.
- Use authentication and additional checks to reduce implicit trust inside the network.

In practice, zero trust pushes you to validate identity and access more often, not just at the perimeter.

**Exam focus**

For Network+ purposes, focus on the role each term plays:

1. Identify AAA as authentication, authorization, and accounting.
2. Match authentication with identity verification.
3. Match authorization with access rights.
4. Match accounting with logging and auditing.
5. Recognize MFA as multiple verification factors during login.

A simple way to remember the flow is: prove who you are, decide what you can do, and record what happened.
