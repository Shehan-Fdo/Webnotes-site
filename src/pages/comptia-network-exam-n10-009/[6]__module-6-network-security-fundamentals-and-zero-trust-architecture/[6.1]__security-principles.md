## **Course:** CompTIA Network+ (Exam N10-009)

**Security principles overview**

The **CIA triad** is a foundational security model made up of **confidentiality**, **integrity**, and **availability**. It gives you a practical way to think about security goals, identify weaknesses, and choose controls that reduce risk.

A strong security posture does not treat these goals separately in isolation. You use them together to protect data, preserve trust in information, and keep systems usable for authorized users when they need them.

- **Confidentiality** focuses on preventing unauthorized access to data.
- **Integrity** focuses on keeping information accurate and trustworthy.
- **Availability** focuses on keeping systems and data accessible to authorized users.

**Confidentiality**

Confidentiality means only approved users and systems can view sensitive information. When confidentiality fails, data exposure becomes the primary concern.

Common controls that support confidentiality include:
- Access management
- Encryption
- Monitoring for unauthorized exposure

> A confidentiality failure can reveal private data even if the system itself is still running normally.

**Integrity**

Integrity means information stays correct, complete, and unaltered except by authorized changes. If integrity is compromised, you can no longer trust the data or the actions based on it.

Integrity matters because corrupted or manipulated data can lead to bad decisions, unauthorized transactions, or hidden activity. Audit trails and monitoring help you detect and investigate changes.

- Integrity protects the accuracy of records.
- Integrity supports accountability when changes occur.
- Integrity failures often require tracing what changed and when.

**Availability**

Availability means authorized users can access systems and information when needed. A security design that protects data but leaves services unusable still fails an important security goal.

Availability depends on resilience and continuity measures such as:
- Backups
- Monitoring
- Controls that limit the impact of an incident

Security planning should also consider whether defensive measures create enough friction for attackers and whether they reduce the blast radius of an incident.

**Using the CIA triad in security design**

You use the CIA triad as a lens for evaluating security controls and incident impact. It helps you answer which principle failed and what kind of response is needed.

1. Identify the security goal at risk: confidentiality, integrity, or availability.
2. Determine what control or condition failed.
3. Apply corrective actions that restore the affected principle.
4. Review whether the incident exposed a broader weakness in the security design.

This model is useful because many incidents map clearly to one primary failure:
- A misconfigured cloud database that exposes records is a confidentiality issue.
- Deleted or altered records that hide unauthorized activity are an integrity issue.
- A service outage that blocks legitimate access is an availability issue.

**Practical security posture**

A secure workload is resilient to attacks and aligns security controls with business goals. In practice, that means you balance protection, trust, and uptime instead of optimizing only one area.

- Use controls that reduce unauthorized access without making systems unusable.
- Use controls that preserve data accuracy and support investigation.
- Use controls that keep services reachable during normal use and during incidents.

The CIA triad also supports broader security design thinking. When you evaluate a system, ask whether your controls meaningfully slow attackers, limit incident impact, and preserve the core security properties of the workload.

**Key takeaway**

The CIA triad gives you a simple but powerful framework for security analysis. If you can identify whether a problem affects confidentiality, integrity, or availability, you can choose more effective controls and respond with better precision.
