## **Course:** CompTIA Network+ (Exam N10-009)

**Security zones overview**

Security zones divide a network into areas with different trust levels so you can control how traffic moves between them. This segmentation helps reduce exposure, limit lateral movement, and keep public-facing services separate from internal resources.

A firewall commonly enforces the boundaries between zones. In practice, you use zones to separate trusted internal systems, untrusted external networks, and intermediate areas that need tighter control.

- **Trusted zone**: Internal network segment with the highest level of trust.
- **Untrusted zone**: External network, usually the Internet, with no implicit trust.
- **Intermediate zone**: A controlled area between trusted and untrusted networks for exposed services.

**DMZ fundamentals**

A <abbr title="demilitarized zone">DMZ</abbr> is a subnetwork that sits between the Internet and the private internal network. It is also called a perimeter network or screened subnet. You place public-facing services there so external users can reach them without giving direct access to the internal LAN.

The DMZ acts as a buffer zone. If an attacker reaches a service in the DMZ, the internal network remains behind additional firewall controls.

- Host public-facing services in the DMZ, such as web or email services.
- Keep sensitive internal systems off the DMZ.
- Treat the DMZ as isolated from both the Internet and the internal network.

> A DMZ is not the same as a home-router “DMZ host” feature. That feature behaves differently and should not be treated as a true perimeter network.

**Why segmentation matters**

Network segmentation improves security by limiting what can communicate with what. It can also support performance and compliance goals, but the main exam focus is the security benefit: fewer systems share the same trust boundary.

Segmentation can be physical or logical. Physical segmentation uses separate devices, while logical segmentation uses technologies such as virtual networks or VLANs (Virtual Local Area Networks) to separate traffic.

- Separate users from servers when they do not need direct access.
- Isolate applications that should only talk to each other.
- Segment sensitive data, such as payment or health information, from general network traffic.

**Common zone and rule concepts**

Security zones work best when paired with explicit access rules. Access control lists, URL filtering, and content filtering are common ways to enforce what traffic is allowed.

| Concept | Purpose |
|---|---|
| ACL (Access Control List) | Permits or denies traffic based on source, destination, port, or protocol |
| URL filtering | Blocks websites by URL or category |
| Content filtering | Blocks content by file type or data pattern |

These controls help you define what is allowed between zones instead of relying on broad trust.

**Typical DMZ design**

A common design places the DMZ between two firewall boundaries or on a firewall with separate interfaces. The goal is to allow only the traffic needed for public services while keeping the internal network protected.

1. Identify the services that must be reachable from outside.
2. Place those services in the DMZ instead of the internal network.
3. Allow only required ports and protocols through the firewall.
4. Block direct access from the Internet to internal systems.
5. Monitor traffic between zones for unusual activity.

**Zero trust connection**

Traditional perimeter security focuses on the edge of the network, but many internal networks are still relatively open once a user gets inside. Zero trust changes that assumption by treating every user, device, and application as untrusted until verified.

Security zones and DMZs still matter in a zero trust design because they reduce exposure and create smaller trust boundaries. They do not replace authentication, encryption, or least privilege; they support them.

**Key takeaways**

- A **DMZ** is a neutral buffer zone for public-facing services.
- A **firewall** enforces traffic rules between trusted and untrusted zones.
- **Segmentation** reduces exposure and limits access to only what is needed.
- Security rules such as ACLs, URL filtering, and content filtering help control traffic between zones.
- Zero trust strengthens the model by verifying traffic inside and outside the network.
