## **Course:** CompTIA Network+ (Exam N10-009)

**Firewall purpose and placement**

A firewall filters traffic to block unauthorized communication while allowing legitimate traffic. You place it at the network edge or between critical network segments to enforce security policy.

Firewalls commonly support rules that combine multiple criteria, such as source and destination IP address, port number, time of day, and application. This lets you build precise access decisions instead of relying on a single match condition.

- **Access control** defines what traffic is allowed or denied.
- **Security rules** can apply to routers, firewalls, operating systems, and other devices that enforce policy.
- **Implicit deny** means traffic is blocked unless a rule explicitly allows it.

**Traditional firewall features**

Traditional firewalls focus on packet-level decisions and connection awareness. Packet filtering checks packets against rule sets based on addresses, protocols, and ports. Stateful inspection goes further by tracking active connections and allowing return traffic only when the firewall recognizes an established session.

- **Packet filtering** examines packets in isolation.
- **Stateful inspection** tracks connection state in a state table.
- **Layer 3/4** filtering is common for traditional firewalls.
- **Stateless** filtering does not remember prior packets.

> A stateless filter can be fast and simple, but it cannot use connection history to make decisions.

**How stateful inspection works**

Stateful firewalls track the state of TCP and UDP connections so they can distinguish legitimate return traffic from unsolicited traffic. This helps the firewall recognize whether a packet belongs to an existing session.

In TCP, connection state is tied to the handshake and control flags used during communication. Because the firewall remembers the session, it can better identify abnormal patterns such as scans or floods.

- **State table** records active connections.
- **Return traffic** is allowed when it matches an established session.
- **TCP** state tracking is especially important for connection-oriented traffic.
- **UDP** can also be tracked even though it is connectionless at the protocol level.

**Network address translation and logging**

Many firewalls also provide network address translation (NAT), which changes address information as traffic passes through the device. NAT is often part of firewall design because it helps control how internal systems appear to external networks.

Logging and monitoring are also core firewall functions. They help you review allowed and denied traffic, investigate incidents, and validate whether rules behave as expected.

- **NAT** modifies address mappings as traffic crosses the firewall.
- **Logging** records traffic and rule activity.
- **Monitoring** helps you verify policy enforcement and troubleshoot issues.

**Next-generation firewall capabilities**

A next-generation firewall (NGFW) adds more application-aware and threat-focused features than a traditional firewall. It can inspect traffic more deeply and apply controls based on what the traffic is doing, not just where it comes from or which port it uses.

Common NGFW capabilities include advanced threat prevention, advanced URL filtering, DNS security, IoT security, and next-generation cloud access security broker (CASB) features.

- **Advanced threat prevention** helps detect and stop malicious activity.
- **URL filtering** controls access to web destinations.
- **DNS security** helps protect name-resolution traffic.
- **IoT security** supports visibility and control for connected devices.
- **CASB** extends policy enforcement for cloud application use.

**Traditional firewall vs. NGFW**

| Feature | Traditional firewall | NGFW |
|---|---|---|
| Primary focus | Packet and connection rules | Application-aware and threat-aware inspection |
| Typical filtering basis | IP address, port, protocol | Application, content, and threat context |
| State tracking | Often included | Included, plus deeper inspection |
| Extra protections | Basic logging, NAT, access control | Threat prevention, URL filtering, DNS security, IoT security |

Traditional firewalls are still useful when you need straightforward traffic control. NGFWs are better when you need deeper inspection and more granular policy enforcement across modern applications and services.

**Exam focus**

For Network+ scenarios, identify whether the question is asking about simple packet filtering, connection tracking, or deeper inspection features. If the scenario mentions application awareness, URL filtering, or threat prevention, the answer is likely an NGFW.

- **Packet filtering** matches basic header information.
- **Stateful inspection** tracks active sessions.
- **NGFW** adds application and threat controls.
- **ACL** stands for access control list and describes allowed and denied traffic rules.

**Practical review steps**

1. Identify whether the firewall is filtering by packet header, connection state, or application behavior.
2. Check whether the rule set uses source, destination, port, protocol, or additional context.
3. Determine whether the device only enforces basic policy or also provides NGFW features.
4. Match the scenario to the simplest firewall type that satisfies the requirement.
