## **Course:** CompTIA Network+ (Exam N10-009)

**VPN fundamentals**

A <abbr title="Virtual Private Network">VPN</abbr> creates a secure path for data that would otherwise travel in the clear across a public network. It encrypts traffic so you can send sensitive information over the internet with less exposure to eavesdropping.

- A **VPN concentrator** is the device or service that handles encryption and decryption for VPN traffic.
- VPN functionality may run on dedicated hardware or in software on an existing server.
- VPN clients can be installed on many operating systems.

**VPN use cases**

VPNs support secure communication for both users and networks. In practice, you use them when you need protected access to internal resources from outside a trusted network boundary.

- **Client-to-site VPNs** connect an individual device to a private network.
- **Clientless VPNs** provide access without installing a full VPN client.
- **Site-to-site VPNs** connect multiple networks, such as a headquarters and branch offices.
- Site-to-site VPNs are useful when multiple offices need access to shared resources like email servers or data stores.

**Site-to-site VPNs**

A site-to-site VPN creates a secure connection between two or more networks. This is common in organizations with multiple offices spread across different locations.

- A **VPN connection** links your on-premises network and remote network resources.
- A **VPN tunnel** is the encrypted link that carries data between sites.
- AWS describes site-to-site VPN as a connection between a VPC and an on-premises network.
- Routing must be configured so traffic passes through the VPN connection.

> A site-to-site VPN protects traffic between networks, but it does not replace routing design. You still need to direct traffic through the tunnel.

**IPsec basics**

<abbr title="Internet Protocol Security">IPsec</abbr> is a widely used VPN technology for securing IP-layer traffic. It is commonly associated with site-to-site VPNs and secure remote access.

- IPsec provides encryption for traffic at the IP layer.
- It is used by many VPN services and has long-standing support in networking environments.
- IPsec VPNs are often used when you need secure communication across untrusted networks.

**Tunneling concepts**

Tunneling encapsulates traffic so it can travel across another network. The tunnel acts as the protected transport path between endpoints.

- The tunnel carries data from one side of the network to the other.
- Tunneling is the mechanism that makes VPN transport possible.
- Different VPN protocols can use different tunneling methods.

**Protocol notes**

VPNs depend on protocols to move data securely between endpoints. Some protocols provide encryption, while others focus on transport or session behavior.

- **PPP** stands for Point-to-Point Protocol.
- PPP supports authentication, compression, error detection, and multiple links.
- PPP does not include encryption in its base form.
- PPP is best suited for one link at each end in a point-to-point design.

**Operational considerations**

When you deploy or troubleshoot a VPN, focus on the connection type, the tunnel path, and where encryption and decryption occur. These details determine how traffic enters and exits the protected network.

- Verify whether the design is client-to-site or site-to-site.
- Confirm that routing sends the intended traffic through the tunnel.
- Check the concentrator or VPN endpoint if encrypted traffic is not passing correctly.
- Match the protocol choice to the security and topology requirements.

**Quick comparison**

| Topic | Practical meaning |
|---|---|
| Client-to-site VPN | One device connects securely to a private network |
| Clientless VPN | Access without a full VPN client |
| Site-to-site VPN | Two or more networks connect securely |
| VPN tunnel | Encrypted path that carries traffic |
| IPsec | Common protocol suite for secure IP traffic |
