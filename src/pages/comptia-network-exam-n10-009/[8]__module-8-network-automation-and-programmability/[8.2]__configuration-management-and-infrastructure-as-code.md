## **Course:** CompTIA Network+ (Exam N10-009)

**Configuration management and Infrastructure as Code**

Configuration management focuses on keeping systems consistent across their lifecycle. In the networking context, that means you manage performance, functional, physical, and operational settings so the environment stays aligned with the intended design.

Infrastructure as Code (<abbr title="Infrastructure as Code">IaC</abbr>) extends that idea by describing infrastructure in machine-readable files instead of relying on manual configuration in the CLI. For Network+, the key takeaway is that IaC supports repeatable deployment, easier change tracking, and more consistent outcomes across networks, servers, firewalls, switches, routers, and applications.

- **Consistency** is the main goal of configuration management.
- **Declarative files** describe the desired state of infrastructure.
- **Version control** helps you track and manage changes centrally.
- **APIs** automate much of the provisioning and management process.

**What Infrastructure as Code does**

IaC uses code-based definitions to provision and manage infrastructure. Rather than building each component by hand, you define the desired configuration and let automation apply it. This approach is especially useful in cloud-based environments where you need to deploy the same design repeatedly.

IaC is commonly associated with:
- templatized configurations
- a centralized code repository
- source control for change tracking
- automated provisioning through APIs

> <mark>IaC does not replace planning</mark>; it makes the planned design repeatable and easier to manage.

**How configuration management and IaC relate**

Configuration management and IaC work together. IaC describes what the environment should look like, while configuration management helps keep that environment aligned over time. In practice, both support stable deployments and reduce drift between intended and actual states.

A useful way to think about the relationship:
- IaC defines the infrastructure in code.
- Configuration management maintains consistency after deployment.
- Both reduce manual, error-prone changes.
- Both support repeatable operations across environments.

**Network as Code connection**

Network as Code (<abbr title="Network as Code">NaC</abbr>) grows out of IaC. The same principles used to define cloud infrastructure can also describe network behavior and device configuration. For Network+ purposes, the important idea is that network design can be expressed as code to support flexible deployment and automation.

This is useful when you need to:
- build cloud-based network infrastructure
- version infrastructure changes
- redeploy updated configurations
- standardize device and service setup

**Benefits for network operations**

IaC improves operational consistency and makes changes easier to manage. Because the configuration is stored as code, you can review, update, and redeploy it with less manual effort than traditional CLI-based workflows.

Common benefits include:
- repeatable builds
- faster provisioning
- easier rollback planning through version history
- reduced configuration drift
- better collaboration across teams

**Typical workflow**

1. Define the desired infrastructure in a code file.
2. Store the file in a centralized repository with version control.
3. Use automation or APIs to deploy the configuration.
4. Update the code when requirements change.
5. Redeploy the revised configuration to apply the changes.

**Practical exam focus**

For Network+, focus on the purpose and advantages of these concepts rather than memorizing a specific vendor tool. You should be able to explain that configuration management keeps systems consistent, while IaC uses declarative code to provision and manage infrastructure in a repeatable way.

**Key terms to remember**

- **Declarative** means you describe the desired end state.
- **Templatized** means configurations are built from reusable patterns.
- **Version control** means changes are tracked over time in a central source.
- **Automation** means tools or APIs perform repetitive tasks with less manual input.

**Why this matters in modern networking**

CompTIA Network+ emphasizes documentation, lifecycle, change, and configuration management as part of core networking skills. IaC fits directly into that scope because it supports secure, maintainable, and scalable network operations across cloud and virtual environments.
