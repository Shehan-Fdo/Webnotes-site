## **Course:** CompTIA Network+ (Exam N10-009)

**Layer 2 troubleshooting overview**

Layer 2 is the data link layer in the OSI (Open Systems Interconnection) model. It handles communication between devices on the same network segment and is responsible for functions such as flow control and error control. When you troubleshoot at this layer, you focus on switching behavior, trunking, and spanning tree behavior rather than IP routing.

A practical approach is to work bottom-up through the OSI model. If the issue is not in Layer 1 or Layer 2, it is not a Layer 2 problem. This keeps you from chasing symptoms in higher layers before you verify the local link and switching path.

- Check whether devices can communicate on the same segment.
- Confirm switch-to-switch links are up and forwarding as expected.
- Look for signs of degraded performance, no connectivity, or lack of reachability.

**Common Layer 2 symptoms**

Layer 2 problems often show up as connectivity failures between switches or as intermittent network behavior. The evidence highlights degraded performance across a link, no connectivity between switches, and lack of reachability as common troubleshooting starting points.

These symptoms usually point to a switching-path issue rather than an application problem. You should verify the physical link first, then move to VLAN, trunk, and spanning tree checks.

- Degraded performance across a link can indicate a switching or negotiation issue.
- No connectivity between switches suggests a trunk or forwarding problem.
- Lack of reachability can result from a Layer 2 break before traffic ever reaches Layer 3.

**Native VLAN mismatches**

A native VLAN mismatch on a trunk can be detected by Cisco Discovery Protocol (CDP) and by Spanning Tree Protocol (STP) when using a per-VLAN flavor such as PVST+ or RPVST+. The evidence shows that switches can log a mismatched native VLAN on a trunk link, which makes this a useful clue during troubleshooting.

A mismatch matters because trunk traffic is interpreted differently on each side of the link. In practice, that can create confusing forwarding behavior and may contribute to a Layer 2 loop scenario if the trunk is not configured consistently.

> Treat a native VLAN mismatch as a configuration warning on the trunk, not as a routing issue.

- **CDP** can reveal the native VLAN value seen by each switch.
- **PVST+** and **RPVST+** can also detect the mismatch through STP behavior.
- A mismatch on a trunk is a strong indicator that the two ends are not aligned.

**Spanning Tree Protocol issues**

Spanning Tree Protocol (STP) is a core Layer 2 protection mechanism because it prevents loops in switched networks. The troubleshooting guidance in the evidence points to STP as a major area to inspect when Layer 2 behavior is unstable.

When STP is involved, you want to confirm that the topology is converging as expected and that no unexpected loop condition exists. If a trunk or VLAN inconsistency changes how frames are forwarded, STP may react in ways that affect connectivity.

- Verify that STP is active where you expect loop prevention.
- Check for topology changes that coincide with the problem.
- Investigate loop symptoms when traffic storms or unstable forwarding appear.

**Troubleshooting workflow**

Use a structured process so you can isolate the fault quickly. Start with the simplest checks and move toward protocol-specific causes.

1. Confirm the physical link is up and the devices can see each other.
2. Check for basic switch-to-switch connectivity problems.
3. Review trunk configuration and native VLAN settings on both ends.
4. Inspect CDP and STP information for mismatch indicators.
5. Validate whether the issue is limited to one VLAN or affects multiple segments.

This sequence keeps you focused on the Layer 2 path before you move on to higher-layer causes.

**What to remember for exam prep**

For Network+ exam prep, focus on the relationship between Layer 2 symptoms and switching features. The exam expects you to recognize that connectivity problems can come from trunking, VLAN alignment, and STP behavior, not just from bad cables or IP settings.

- Layer 2 is about local network delivery on the same segment.
- Native VLAN mismatches are detectable on trunks.
- CDP and per-VLAN STP variants help expose configuration problems.
- Bottom-up troubleshooting is the safest way to isolate the fault.
