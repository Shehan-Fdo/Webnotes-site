## **Course:** CompTIA Network+ (Exam N10-009)

**Layer 3 troubleshooting overview**

Layer 3 problems usually involve the network layer, where IP addressing and routing determine whether devices can reach each other across networks. When communication fails, you often need to check whether the address plan is valid, whether subnets match, and whether traffic can be routed between networks.

A practical troubleshooting mindset helps you narrow the issue before changing anything. Start by identifying the symptom, then test the most likely causes, and verify the result after each change.

- **IP conflicts** can stop devices from communicating reliably when two endpoints use the same address.
- **Subnet mismatch** issues appear when devices are not configured for the same network range or mask.
- **Routing** problems block traffic between networks even when local connectivity looks normal.
- **NAT** (Network Address Translation) can help isolate overlapping private address spaces when separate networks must coexist.

**Common Layer 3 symptoms**

Layer 3 issues often look like “the network is up, but traffic does not pass.” A device may still link to the switch or access point, yet fail to reach a remote host, server, or shared resource. In overlapping-address environments, the problem can be especially confusing because each local network may work on its own.

> If multiple networks use the same private IP range, do not assume the conflict is harmless just because each segment works in isolation.

- Devices lose access to remote resources even though the local link is active.
- Hosts on different networks cannot communicate because their addressing overlaps.
- A central server cannot distinguish between identical private addresses on separate networks.
- Changing every endpoint address may be impractical in operational environments.

**IP address conflicts and overlapping networks**

An IP conflict occurs when more than one device uses the same IP address on a network path that expects uniqueness. In some environments, separate systems may each use identical private ranges, such as `192.168.1.x`, without issue until they are connected to a shared network.

NAT is a common way to create logical separation without renumbering every device. It translates local addresses into unique addresses on the wider network, which helps prevent collisions between isolated segments.

- Use NAT when you need to connect isolated networks that reuse the same private range.
- Keep the local addressing intact when changing it would disrupt operations.
- Translate each segment to a unique address space on the shared network.
- Treat overlapping private ranges as a design issue, not just a device fault.

**Subnet and routing checks**

Subnet settings tell a device which addresses are local and which require a router. If the subnet mask is wrong, a host may send traffic to the wrong place or fail to recognize that a destination is remote. Routing then determines how packets move between networks once they leave the local subnet.

A useful check is to compare the source and destination addressing against the expected network boundaries. If the addresses do not align, the device may never send traffic to the correct next hop.

- Confirm the IP address and subnet mask on both ends of the path.
- Verify that the default gateway is reachable and correctly configured.
- Check whether the destination is on the same subnet or requires routing.
- Review any translation or segmentation layer that sits between the networks.

**Troubleshooting approach**

Use a simple, repeatable process so you can isolate the fault without guessing. The goal is to prove where communication stops and whether the cause is addressing, translation, or routing.

1. Identify the symptom and the affected devices.
2. Check for duplicate addresses or overlapping private ranges.
3. Verify subnet masks and default gateway settings.
4. Test reachability across the route or translation boundary.
5. Apply the smallest practical fix, then verify communication again.
6. Document the cause and the resolution.

**What to remember for the exam**

Layer 3 issues are about logical delivery, not physical cabling. If the link is up but traffic fails, focus on addressing, subnet boundaries, routing paths, and NAT behavior.

- **Addressing** errors create conflicts and confusion.
- **Subnetting** errors make hosts classify destinations incorrectly.
- **Routing** errors prevent packets from reaching other networks.
- **NAT** can solve overlap problems when separate networks must share connectivity.
