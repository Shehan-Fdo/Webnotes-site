## **Course:** CompTIA Network+ (Exam N10-009)

**Encapsulation overview**

Encapsulation is the process of adding layer-specific information to data as it moves from the sender down through the network stack. In the OSI (Open Systems Interconnection) and TCP/IP models, each layer can add its own header, and the network access layer can also add a trailer.

This extra information helps the receiving system interpret the payload correctly and deliver it to the right application. The model is a reference framework, so the real protocols and technologies do the work while the layers provide a common way to describe it.

- **Header** data is added by layers to identify how the payload should be handled.
- **Trailer** data appears at the lower layer in some cases, especially at the network access layer.
- **Payload** is the data being carried from one layer to the next.

**How encapsulation works**

As data moves downward through the stack, each layer wraps the data it receives and passes it to the next layer. The transport layer may break data into smaller pieces and add information such as source and destination ports and sequence numbers. The internet layer then adds addressing information, such as source and destination IP addresses. The network access layer adds the information needed for delivery on the local link.

> The OSI model is a reference model, not actual software or hardware. Use it to describe what happens to data, not as a literal device or protocol stack.

A simple way to remember the process is that each lower layer sees the data from the layer above as its payload and adds its own control information around it.

- The transport layer focuses on process-to-process delivery.
- The internet layer focuses on logical addressing and routing.
- The network access layer prepares the data for transmission on the physical network.

**Layer-by-layer view**

| Layer role | What it adds | Resulting unit |
|---|---|---|
| Transport | Header with port and sequencing information | Segment |
| Internet | Header with source and destination IP information | Packet |
| Network access | Header and trailer for local delivery | Frame |

The exact names can vary by model and protocol, but the key idea stays the same: each layer wraps the data for its own purpose before passing it down.

**Decapsulation overview**

Decapsulation is the reverse process on the receiving side. Each layer removes the information added by the corresponding sender-side layer, then passes the remaining data upward. This continues until the original application data reaches the destination application.

This reverse flow preserves modularity. Each layer only needs to understand the information relevant to its own job, which keeps communication systems easier to design and troubleshoot.

- The receiving network access layer removes link-layer information.
- The receiving internet layer removes network-layer information.
- The receiving transport layer uses its header information to reassemble and deliver the data correctly.

**Why the process matters**

Encapsulation and decapsulation let communication protocols stay modular. That separation makes it easier to standardize networking behavior across different devices and vendors while still allowing each layer to perform a specific function.

For exam purposes, focus on the direction of travel and the purpose of each step:

1. Data starts at the application side.
2. Each layer adds its own control information as data moves downward.
3. The receiving system removes that information as data moves upward.
4. The original message is delivered to the correct application.

**Quick memory aid**

Think of sending a letter in an envelope. The message is the payload, and each layer adds more addressing or delivery information so the message can reach the right destination. On receipt, each layer removes what it needs to remove before handing the message up.
