## **Course:** CompTIA Network+ (Exam N10-009)

**Overview**

Port numbers help you identify and manage application traffic on a network. In practice, you use them to recognize which service is communicating, choose the right troubleshooting path, and confirm whether traffic is using the expected transport protocol.

CompTIA Network+ expects you to recognize common well-known ports and the services tied to them. The goal is not just memorization; it is understanding how ports map to everyday network applications.

- **TCP** and **UDP** are the transport protocols you see most often in port-based service questions.
- A service may use one port or multiple ports, depending on how it separates control and data traffic.
- Knowing the default port helps you spot misconfiguration and identify unexpected exposure.

**Core port concepts**

A port is a logical endpoint used by an application. When you see a port number, you should think about the service behind it and whether that service is expected on the network.

Some services are especially common in exam and real-world troubleshooting scenarios:

- **FTP** uses TCP port `20` and TCP port `21`.
  - Port `20` is commonly associated with the file transfer process.
  - Port `21` is commonly used for control information.
- **Telnet** uses TCP port `23`.
  - It provides remote console access.
  - It sends data in the clear, so credentials and session data are not encrypted.
- **HTTP** uses TCP port `80`.
  - It is the default port for browser access.
- **HTTPS** uses TCP port `443`.
  - It is the secure web service port.
- **SSH** uses TCP port `22`.
  - It provides secure remote access to Linux and other systems.
- **SMTP** uses TCP port `25`.
  - It is used for sending email to an external mail server.
- **DNS** uses TCP and UDP port `53`.
  - It supports name lookup.
- **SNMP** uses UDP ports `161` and `162`.
  - Port `161` handles inbound requests.
  - Port `162` handles traps.
- **NTP** uses port `123`.
  - It is used for time synchronization.
- **RDP** uses TCP port `3389`.
  - It is a common remote desktop service.
- **LDAP** uses TCP and UDP port `389`.
  - It is commonly associated with directory services.
- **SMB** uses TCP port `445`.
  - It is commonly used for file and printer sharing.
- **DHCP** uses UDP ports `67` and `68`.
  - These ports support address assignment.

**Common ports at a glance**

| Service | Port(s) | Transport | Typical use |
|---|---:|---|---|
| FTP | 20, 21 | TCP | File transfer and control |
| Telnet | 23 | TCP | Remote console access |
| SSH | 22 | TCP | Secure remote access |
| HTTP | 80 | TCP | Web access |
| HTTPS | 443 | TCP | Secure web access |
| SMTP | 25 | TCP | Email sending |
| DNS | 53 | TCP/UDP | Name resolution |
| DHCP | 67, 68 | UDP | Address assignment |
| TFTP | 69 | UDP | Simple file transfer |
| NTP | 123 | TCP | Time synchronization |
| SNMP | 161, 162 | UDP | Monitoring and traps |
| LDAP | 389 | TCP/UDP | Directory services |

| SMB | 445 | TCP | File and printer sharing |

**How to study these ports**

You do best when you connect each port to a service and a use case. That makes it easier to answer exam questions and easier to troubleshoot live traffic.

1. Learn the service name first.
2. Associate the default port number with that service.
3. Note whether the service uses TCP, UDP, or both.
4. Remember the practical purpose of the service, such as web access, remote login, or name resolution.
5. Watch for security implications, especially with unencrypted services like Telnet.

> Telnet is a classic example of why port knowledge matters: it is easy to identify by port `23`, but it is not secure because it transmits data without encryption.

**Troubleshooting mindset**

When a service fails, the port number is one of the first things you verify. If the application is correct but the port is blocked, changed, or misconfigured, the service may appear down even when the host is reachable.

Use port awareness to check:

- Whether the expected service is listening on the correct port.
- Whether a firewall or access control rule is blocking the traffic.
- Whether the application is using the right transport protocol.
- Whether the port matches the service you intended to deploy.

**High-value exam reminders**

- **FTP** is one of the few common services that uses two ports.
- **DNS** commonly uses both TCP and UDP on port `53`.
- **SNMP** uses UDP, not TCP, for its common management traffic.
- **HTTPS** is the secure version of web traffic on port `443`.
- **SSH** is the secure replacement for Telnet-style remote access.
- **DHCP** uses `67` and `68`, which are easy to confuse if you do not tie them to address assignment.

**Quick memory cues**

- Web: `80` and `443`
- Remote access: `22`, `23`, `3389`
- Name and time services: `53`, `123`
- File and sharing services: `20`, `21`, `445`
- Monitoring and management: `161`, `162`, `389`
