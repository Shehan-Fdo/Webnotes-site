## **Course:** CompTIA Network+ (Exam N10-009)

**Transmission modes**

Transmission mode describes how data moves between devices. The core distinction is whether communication is one-way, alternating two-way, or simultaneous two-way. These modes matter because they affect device behavior, link efficiency, and troubleshooting expectations.

- **Simplex** is one-way communication only.
- **Half-duplex** allows two-way communication, but not at the same time.
- **Full-duplex** allows simultaneous two-way communication.

A simple way to remember the difference is to ask whether both sides can talk at once. If they cannot, the link is either simplex or half-duplex. If they can, it is full-duplex.

<dl>
<dt>Simplex</dt>
<dd>Data flows in one direction only; the sender does not receive on the same channel.</dd>
<dt>Half-duplex</dt>
<dd>Data flows in both directions, but only one direction at a time.</dd>
<dt>Full-duplex</dt>
<dd>Data flows in both directions at the same time.</dd>
</dl>

**Practical examples**

- **Simplex** fits devices that only send or only receive in a given direction.
- **Half-duplex** fits push-to-talk style communication, such as a walkie-talkie.
- **Full-duplex** fits modern bidirectional links where sending and receiving happen together.

> Full-duplex is the most efficient of the three for simultaneous communication, while half-duplex requires switching between send and receive states.

**MTU fundamentals**

<abbr title="Maximum Transmission Unit">MTU</abbr> is the largest packet size a network can transmit without fragmentation. If a packet exceeds the MTU on any part of the path, the network may fragment it or drop it, depending on the path and device behavior.

- A common standard Ethernet MTU is `1500` bytes.
- Some environments use larger values such as `9000` bytes for jumbo frames.
- MTU must be consistent enough across the path to avoid packet loss and inefficient fragmentation.

MTU problems often appear as intermittent connectivity issues, performance degradation, or hard-to-trace packet loss. They are especially noticeable when one segment supports larger packets and another segment does not.

**What MTU mismatch causes**

MTU mismatch happens when devices or network segments along a path use different MTU values. That mismatch can create several issues:

- **Fragmentation** increases overhead and can reduce throughput.
- **Packet loss** occurs when a device cannot handle the larger packet size.
- **Latency** rises because fragmented traffic needs extra processing and reassembly.
- **Black holes** can occur when Path MTU Discovery does not work as expected.

Path MTU Discovery, or <abbr title="Path MTU Discovery">PMTUD</abbr>, relies on ICMP (`Internet Control Message Protocol`) messages such as “Fragmentation Needed” to help the sender reduce packet size. If firewalls block those ICMP messages, large packets may keep failing.

**Troubleshooting approach**

1. Check the MTU on each interface along the path.
2. Compare local interface settings with tunnel endpoints, switches, and other hops.
3. Test with packets that reflect the expected path size.
4. Watch for symptoms such as drops, fragmentation, or inconsistent connectivity.
5. Review whether ICMP messages are being blocked if PMTUD appears to fail.

```bash
ip link show
ip link show eth0 | grep mtu
cat /sys/class/net/eth0/mtu
```

**How transmission and MTU relate**

Transmission mode and MTU solve different problems, but both affect network performance. Duplexing controls how devices exchange data on a link, while MTU controls how much data fits into a packet without fragmentation. In troubleshooting, you often check both because a link can be correctly duplexed yet still suffer from MTU-related loss or slow transfers.

- Duplex issues affect how traffic flows on the link.
- MTU issues affect how large each packet can be.
- Both can reduce throughput if they are misconfigured or mismatched.

**Exam focus**

For Network+ questions, be ready to identify the correct transmission mode from a scenario and to recognize MTU mismatch symptoms. The most useful clues are:

- One-way communication points to **simplex**.
- Alternating send/receive points to **half-duplex**.
- Simultaneous two-way communication points to **full-duplex**.
- Fragmentation, packet loss, and PMTUD failures point to **MTU** problems.
