## **Course:** CompTIA Network+ (Exam N10-009)

**Overview**

The OSI (Open Systems Interconnection) and TCP/IP models are logical frameworks for understanding how data moves between devices on a network. They help you describe communication in layers, isolate problems, and map protocols to the functions they perform.

The OSI model is a seven-layer conceptual framework. The TCP/IP model is a more practical protocol stack with fewer layers. In exam scenarios, you often use OSI to explain *where* a problem occurs and TCP/IP to describe how real-world networks are commonly structured.

- **OSI** is protocol-independent and useful for analysis and learning.
- **TCP/IP** is practical and widely used in production networks.
- Both models describe network communication between devices.
- Layered design helps you troubleshoot by narrowing the source of a fault.

**OSI Model Layers**

The OSI model separates network communication into seven layers. Each layer has a distinct role, and the layers work together from physical transmission up to user-facing applications.

1. **Physical**: Sends raw bits over media such as cables and connectors.
2. **Data Link**: Handles local delivery on the same network segment, including hardware addressing.
3. **Network**: Moves packets between networks and supports routing.
4. **Transport**: Provides end-to-end delivery functions such as segmentation and transport behavior.
5. **Session**: Manages communication sessions between systems.
6. **Presentation**: Handles data representation, such as formatting and encoding.
7. **Application**: Supports network services used by applications.

A useful way to remember the OSI model is to think from the wire upward: signals, local delivery, routing, transport, session control, data format, and application access. The exact protocol examples can vary, but the layer purpose stays consistent.

- **Layer 1** focuses on signals and media.
- **Layer 2** focuses on local network delivery.
- **Layer 3** focuses on routing between networks.
- **Layer 4** focuses on transport behavior.
- **Layers 5-7** focus on session handling, data formatting, and application services.

**TCP/IP Model Layers**

The TCP/IP model is a shorter framework that groups network functions into fewer layers. Sources describe it with four or five layers depending on how the lower layers are grouped, but the practical idea is the same: it combines several OSI functions into a smaller stack.

- **Application**: Covers user-facing network services and higher-level protocols.
- **Transport**: Handles end-to-end communication.
- **Network**: Supports internetworking and logical addressing.
- **Network access**: Covers local network delivery and hardware-related functions.

TCP/IP is the model most closely tied to real-world internet communication. It is a protocol suite used in production, so it is more practical than the OSI reference model.

> Use conservative mapping on exams: if a question asks for the conceptual model, answer with OSI; if it asks about the real-world protocol stack, answer with TCP/IP.

**OSI vs. TCP/IP**

The two models overlap in purpose, but they differ in structure and emphasis. OSI is a reference model for understanding and teaching network communication. TCP/IP is the working model behind modern network communication.

| Aspect | OSI | TCP/IP |
|---|---|---|
| Layer count | 7 layers | 4 or 5 layers, depending on grouping |
| Purpose | Conceptual reference framework | Practical protocol stack |
| Focus | Standardizing and explaining communication | Real-world communication and interoperability |
| Use in practice | Common for learning and troubleshooting | Common in production networks |

A key exam point is that TCP/IP compresses multiple OSI functions into fewer layers. For example, OSI separates session and presentation functions, while TCP/IP typically includes those responsibilities within the application layer.

- **OSI** helps you reason about network behavior in a structured way.
- **TCP/IP** reflects how networks are commonly implemented.
- Layer names do not always map one-to-one across the two models.
- The same communication task may appear in different layers depending on the model.

**Troubleshooting with Layers**

Layered models are valuable because they help you isolate where a problem occurs. If a connection fails, you can move from the lower layers upward and verify each stage of communication.

1. Check the physical path first, including cables, connectors, and signal presence.
2. Confirm local network delivery and addressing at the data link layer.
3. Verify routing and logical connectivity at the network layer.
4. Confirm transport behavior if the path exists but the service still fails.
5. Review application-level access if lower layers are working.

This approach keeps troubleshooting practical. Instead of guessing at the cause, you test the layer most likely to contain the fault and then move upward.

**Exam Focus**

For Network+ questions, focus on what each model is for, how many layers each has, and how the layers differ.

- **OSI**: seven-layer reference model.
- **TCP/IP**: practical protocol stack used in real networks.
- **Physical to Application**: the OSI progression from transmission to user services.
- **Layer grouping**: TCP/IP combines several OSI functions into fewer layers.
- **Troubleshooting**: layered thinking helps you isolate failures efficiently.

If you need a quick memory aid, start with the OSI layers from bottom to top: physical, data link, network, transport, session, presentation, application. Then remember that TCP/IP reduces that structure into a smaller, more implementation-focused stack.
