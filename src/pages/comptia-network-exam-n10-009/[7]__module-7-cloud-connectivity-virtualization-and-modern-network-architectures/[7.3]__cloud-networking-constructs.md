## **Course:** CompTIA Network+ (Exam N10-009)

**Cloud networking overview**

Cloud networking is the set of services and design choices that move traffic between applications, users, and infrastructure inside a public cloud and across hybrid environments. In practice, you use cloud networking constructs to control access, segment traffic, and observe what is happening on the network.

A common exam focus is the difference between controls that apply at the **resource** level and controls that apply at the **subnet** level. You also need to recognize which features provide visibility into traffic rather than direct filtering.

- **Virtual private cloud (VPC)**: A logically isolated cloud network where you place resources and apply network controls.
- **Security groups**: Allow specific inbound and outbound traffic at the resource level, such as an instance.
- **Network access control lists (ACLs)**: Allow or deny specific inbound and outbound traffic at the subnet level.
- **Flow logs**: Capture information about IP traffic going to and from network interfaces in the VPC.

**Security groups and subnet filtering**

Security groups are attached to resources, and a single instance can belong to one or more security groups. If you do not specify a security group when launching an instance, the instance is automatically associated with the default security group for its VPC. This makes security groups the first place to look when you need instance-level traffic control.

Network ACLs provide an additional layer of security at the subnet boundary. They can use default or custom rules, and they are useful when you want to control traffic entering or leaving a subnet rather than a specific host.

> Network ACLs and security groups are not the same control point: one filters at the subnet level, while the other filters at the resource level.

- **Security groups**
  - Apply to instances or other resources.
  - Control allowed inbound and outbound traffic.
  - Can differ from one instance to another in the same VPC.
- **Network ACLs**
  - Apply to subnets.
  - Can allow or deny traffic.
  - Add an extra layer of protection for subnet traffic.

**Traffic visibility and monitoring**

Flow logs help you observe network activity by recording IP traffic to and from network interfaces in the VPC. They do not block traffic; instead, they provide visibility for troubleshooting, auditing, and security analysis.

Use flow logs when you need to understand what traffic is moving through a VPC, subnet, or individual network interface. This is especially useful when you are validating whether a rule is working as expected or investigating unexpected connectivity.

**Practical cloud networking distinctions**

The main exam-ready distinction is simple: security groups protect resources, network ACLs protect subnets, and flow logs record traffic. When you design or troubleshoot cloud connectivity, identify the layer where the control applies before changing rules.

| Construct | Scope | Primary purpose |
|---|---|---|
| Security group | Resource | Allow specific inbound and outbound traffic |
| Network ACL | Subnet | Allow or deny specific inbound and outbound traffic |
| Flow log | VPC, subnet, or network interface | Capture traffic information |

**How to reason about cloud traffic control**

1. Identify the resource or subnet you want to protect.
2. Decide whether you need to allow, deny, or observe traffic.
3. Apply the correct construct at the correct scope.
4. Verify the result with traffic behavior or flow log data.

**Key takeaways**

- Cloud networking supports segmentation, access control, and visibility in virtualized environments.
- Security groups are resource-level filters.
- Network ACLs are subnet-level filters.
- Flow logs provide traffic records for analysis rather than enforcement.
- In cloud design, the control point matters as much as the rule itself.
