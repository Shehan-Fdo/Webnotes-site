## **Course:** CompTIA Network+ (Exam N10-009)

**Virtualization in networking**

Virtualization moves networking functions from dedicated hardware into software. In practice, this lets you build and manage logical networks on top of a physical infrastructure. Network virtualization can include services such as IP routing, packet switching, traffic load balancing, and packet filtering.

This approach changes how you think about network design:

- Multiple virtual networks can share one physical network.
- Virtual LANs (VLANs) segment a physical network into multiple logical networks.
- Virtualized environments still need a path to the physical world for external communication.

**Virtual network components**

A virtualized network uses software-based components to connect virtual machines (VMs) and physical devices. These components behave like their physical counterparts, but they are easier to scale and manage.

Key terms to know:

- **Hypervisor**: The platform that hosts and manages virtual machines.
- **Virtual switch** or **vSwitch**: A software-based switch that controls traffic between VMs and between VMs and physical network devices.
- **Virtual network interface card** or **vNIC**: The virtual adapter a VM uses to connect to a network.

A vSwitch can support features such as traffic shaping, VLAN tagging, and Quality of Service (QoS). These features help you control how traffic moves and how it is prioritized inside a virtual environment.

**How virtual switches fit into the network**

A virtual switch lets VMs communicate with other computers on the network. On Hyper-V, you can create and configure a virtual switch with Hyper-V Manager or PowerShell. When you install the Hyper-V role on Windows Server, you can optionally create a virtual switch during setup.

> A virtual switch is software, but it still depends on the underlying physical network for external connectivity.

**Practical implications**

Virtualization gives you flexibility, but it also changes the operational model. Instead of managing only physical devices, you manage logical connections, software-defined traffic paths, and the relationship between virtual and physical segments.

- You can place many virtual servers on one physical host.
- You still need networking services that connect those virtual systems to other systems.
- You may use virtual networking features to isolate traffic or optimize performance.

**Study focus for Network+**

For exam purposes, focus on the relationship between the virtual layer and the physical network. Be able to explain what each component does and why it exists.

- **Virtualization**: software-based implementation of networking services.
- **vSwitch**: connects VMs and physical network devices.
- **vNIC**: gives a VM network access.
- **VLANs**: create logical separation on a physical network.
