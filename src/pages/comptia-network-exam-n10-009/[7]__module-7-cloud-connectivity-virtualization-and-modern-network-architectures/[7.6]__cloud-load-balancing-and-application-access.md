## **Course:** CompTIA Network+ (Exam N10-009)

**Cloud load balancing overview**

A load balancer distributes user traffic across multiple application instances so no single server carries the full demand. In cloud environments, this improves performance, resilience, and scalability while keeping the application accessible during spikes in usage.

Cloud providers offer managed load balancing services that can scale from zero to millions of users. These services are designed to reduce performance issues by spreading requests across backends, often across multiple regions.

- **Traffic distribution** helps prevent overload on any one resource.
- **Availability** improves because the application can continue serving users if one backend becomes unavailable.
- **Scalability** supports growth without requiring manual pre-warming in some managed services.

**Application load balancers**

Application load balancers handle HTTP and HTTPS traffic and can distribute requests across multiple backend instances in multiple regions. They are commonly used when you need content-based routing and a simplified public entry point for the application.

A key advantage is that the entire application can be reached through a single global IP address, which simplifies DNS configuration. For HTTPS traffic, the load balancer can also perform SSL termination, offloading encryption handling from the backend servers.

- **HTTP(S)** load balancing is suited to web applications.
- **Content-based** load balancing routes requests based on application data.
- **SSL termination** moves certificate handling to the load balancer.
- **Single IP** access simplifies client connectivity and DNS records.

**Global and regional behavior**

Cloud load balancing can operate globally or regionally depending on the service design. Global proxy load balancers can distribute millions of requests per second across backends in multiple regions using a single anycast IP address. Regional proxy load balancers keep traffic and backends within a chosen region when jurisdictional control matters.

This distinction matters when you need either broad global reach or tighter geographic control over where traffic is processed.

| Load balancer type | Typical use | Notable behavior |
|---|---|---|
| Global proxy | Worldwide application access | Uses a single anycast IP and can reach backends in multiple regions |
| Regional proxy | Location-specific control | Keeps backend traffic within a region |

**Operational benefits**

Managed cloud load balancing is built to absorb traffic spikes without pre-warming and to provide actionable traffic insights. It also commonly includes integrated security features such as distributed denial-of-service (<abbr title="Distributed Denial of Service">DDoS</abbr>) protection and web application firewall (<abbr title="Web Application Firewall">WAF</abbr>) capabilities.

These features make the load balancer more than a traffic splitter. It becomes a control point for performance, resilience, and basic edge security.

> Use a load balancer when you need to distribute traffic across multiple servers and keep the application available under heavy or uneven demand.

**How load balancing supports application access**

Load balancing sits between users and the server pool, acting as an invisible intermediary. Users connect to the application endpoint, while the load balancer decides which backend should handle each request.

This approach is especially useful for large websites and cloud applications that must serve many simultaneous users. It helps maintain uptime and makes the backend architecture easier to scale horizontally.

- **Invisible mediation** keeps the backend topology hidden from users.
- **Horizontal scaling** adds more servers instead of relying on one larger server.
- **Resilience** improves because traffic can shift away from failing or busy systems.

**Practical selection guidance**

Choose the load balancer based on the traffic type and the access pattern you need. Web applications usually need HTTP(S) support, while other workloads may require network-level load balancing.

When you evaluate a cloud load balancer, focus on:
- **Protocol support** for the traffic you serve.
- **Geographic scope** for global or regional delivery.
- **Security features** such as DDoS protection and WAF integration.
- **Operational simplicity** such as single-IP access and reduced DNS complexity.

**Key takeaways**

Cloud load balancing improves application access by distributing traffic across multiple backends, reducing overload risk, and supporting scalable, resilient service delivery. In managed cloud platforms, it can also simplify DNS, terminate SSL, and provide global reach through a single IP address.
