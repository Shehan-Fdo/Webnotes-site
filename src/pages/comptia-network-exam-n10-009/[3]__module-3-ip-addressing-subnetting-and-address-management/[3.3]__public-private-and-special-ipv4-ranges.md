## **Course:** CompTIA Network+ (Exam N10-009)

**IPv4 range overview**

IPv4 address ranges are often grouped by purpose. For Network+ study, the most important distinction is between **public** addresses, which are routable on the Internet, and **special-use** ranges, which are reserved for local, testing, or control functions.

Classful addressing still appears in older references, but modern networks use Classless Inter-Domain Routing (CIDR). Focus on the range and purpose of an address rather than the old class label.

- **Private** ranges are reserved for internal networks and are not routed on the public Internet.
- **Special-use** ranges support functions such as loopback, link-local communication, multicast, and broadcast.
- **Reserved** addresses are set aside for specific roles and should not be assigned like normal host addresses.

**Private IPv4 ranges**

The three RFC 1918 private IPv4 blocks are the core ranges to memorize. These addresses are used inside organizations, homes, and labs, and they should not appear as public Internet source or destination addresses.

| CIDR block | Address range | Typical use |
|---|---|---|
| `10.0.0.0/8` | `10.0.0.0` – `10.255.255.255` | Large private networks |
| `172.16.0.0/12` | `172.16.0.0` – `172.31.255.255` | Medium private networks |
| `192.168.0.0/16` | `192.168.0.0` – `192.168.255.255` | Small private networks |

These ranges are commonly used behind network address translation (NAT). If you see one of these addresses on a local device, that does not mean the device is directly reachable from the Internet.

> Private IP addresses should not be routed on the public Internet.

**Special IPv4 ranges**

Special ranges are reserved for specific network behaviors. Some are used by hosts, some by routers, and some by network protocols.

| CIDR block | Purpose | Notes |
|---|---|---|
| `0.0.0.0/8` | Local network / "this" network | Used to refer to the current network context |
| `127.0.0.0/8` | Loopback | Used to test the local host; `127.0.0.1` is the common loopback address |
| `169.254.0.0/16` | Link-local | Used when automatic local addressing is needed |
| `224.0.0.0/4` | Multicast | Used for one-to-many delivery |
| `255.255.255.255/32` | Limited broadcast | Broadcasts to all hosts on the local network |

A few special addresses are especially important in troubleshooting:

- `0.0.0.0` can represent an unspecified or local network context.
- `127.0.0.1` always points back to the local machine.
- `169.254.x.x` often indicates a host did not receive a valid address from DHCP (Dynamic Host Configuration Protocol) and self-assigned a link-local address.

**Public IPv4 addresses**

Public IPv4 addresses are the globally routable addresses used on the Internet. They are not part of the private RFC 1918 blocks and are assigned for external communication.

When you troubleshoot connectivity, check whether a device has:
- a valid private address on the local network,
- a gateway that can reach the Internet,
- and a public address path through NAT or another routing device.

**How to identify the range quickly**

Use the first octets and the prefix length to classify the address.

1. Check whether the address falls inside one of the private blocks.
2. If it does not, check whether it matches a special-use range such as loopback, link-local, multicast, or broadcast.
3. If none of those apply, treat it as a public IPv4 address unless another reserved purpose is documented.

**Practical exam reminders**

- **RFC 1918** defines the three private IPv4 blocks.
- **Loopback** addresses stay on the local host and are useful for testing TCP/IP (Transmission Control Protocol/Internet Protocol) stack behavior.
- **Link-local** addresses help devices communicate on the same local segment without manual configuration.
- **Multicast** addresses deliver traffic to multiple interested receivers.
- **Broadcast** addresses target all hosts on the local subnet.

**Quick memory aid**

- `10.0.0.0/8` = large private space
- `172.16.0.0/12` = middle private space
- `192.168.0.0/16` = small private space
- `127.0.0.0/8` = loopback
- `169.254.0.0/16` = link-local
- `224.0.0.0/4` = multicast
- `255.255.255.255` = limited broadcast
