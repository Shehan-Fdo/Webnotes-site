## **Course:** CompTIA Network+ (Exam N10-009)

**NAT and PAT overview**

<abbr title="Network Address Translation">NAT</abbr> translates private IP addresses to public IP addresses so devices on a local network can reach external networks without each device needing its own public address. This helps conserve the limited IPv4 address space and is commonly used in home, corporate, and service-provider environments.

<abbr title="Port Address Translation">PAT</abbr> is a specific form of NAT that also uses transport-layer port numbers. In practice, PAT lets multiple private devices share a single public IP address, which is why it is often called NAT overload. The core idea is the same: hide internal addressing behind a public-facing translation point.

- NAT modifies IP address information in packet headers while traffic passes through a router.
- PAT modifies IP addresses and Layer 4 port numbers to support more simultaneous connections.
- NAT can be one-to-one or many-to-one.
- PAT is used when many internal hosts need to share one public address.
- Both techniques improve address conservation and can add a layer of network abstraction.

**Why NAT and PAT matter**

IPv4 address exhaustion is the main reason these technologies exist. A private network can contain many devices, but the Internet-facing side often has only a small number of public addresses available. NAT and PAT bridge that gap by translating internal addresses into routable public ones.

They are also practical in modern network design because they simplify Internet access for many devices behind a single edge router. In common usage, people may say “NAT” even when the implementation is actually PAT or another NAT variant.

- Use NAT when you need address translation between private and public networks.
- Use PAT when multiple internal hosts must share one public IP address.
- Expect NAT to appear in home routers and enterprise edge devices.
- Treat “NAT” as an umbrella term in many real-world discussions.

**How the translation model works**

NAT sits at the boundary between a private network and the public Internet. When an internal host sends traffic out, the router rewrites the source address so the packet appears to come from a public address. Return traffic is translated back to the original internal host.

PAT extends this by tracking port numbers as well, which allows the router to distinguish between many simultaneous sessions that all use the same public IP. That extra port mapping is what gives PAT its scale advantage.

- Internal hosts keep private addresses on the local network.
- The edge device rewrites traffic as it crosses the boundary.
- Return packets are mapped back to the correct internal device.
- PAT uses port mappings to separate concurrent sessions.

**NAT vs. PAT**

| Feature | NAT | PAT |
|---|---|---|
| Address translation | IP addresses | IP addresses and port numbers |
| Typical mapping | One-to-one or many-to-one | Many-to-one |
| Main benefit | Conserves public IPv4 addresses | Conserves public IPv4 addresses at larger scale |
| Common alias | NAT | NAT overload, NAPT |

**Practical exam takeaways**

For Network+ purposes, focus on the purpose and behavior of each term rather than memorizing vendor-specific configuration details. If a question describes multiple internal devices sharing one public IP, PAT is the best match. If the question only emphasizes translation between private and public addressing, NAT is the broader concept.

> Be careful with terminology: in some sources, NAT is used as a general label that includes PAT/NAPT behavior.

**Quick recall**

- **NAT** translates private IPs to public IPs.
- **PAT** is NAT with port translation.
- **PAT** supports many devices sharing one public IP.
- **IPv4 conservation** is the main reason these technologies exist.
