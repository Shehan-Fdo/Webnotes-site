## **Course:** CompTIA Network+ (Exam N10-009)

**DHCP scope fundamentals**

A <abbr title="Dynamic Host Configuration Protocol">DHCP</abbr> scope is the administrative grouping of IP addresses that a DHCP server can lease to clients on a subnet. It is the core unit you use to control which addresses are available and what network settings clients receive.

A scope does more than hand out addresses. It also lets you define settings such as <abbr title="Domain Name System">DNS</abbr> servers and default gateways for clients in that subnet. In practice, you use scopes to organize address distribution across one or more subnets and to apply policy at the scope level.

- A scope is a pool of addresses on a subnet.
- A server can manage multiple scopes for multiple subnets.
- Scope settings can include DNS servers and network gateways.
- Scope policy can filter hosts by name, MAC address, or operating system before assignment.

**Scope design and address pools**

A DHCP scope is typically a contiguous range of addresses. You define the range, the subnet mask, and the lease duration when you configure it. That range becomes the pool the server uses for automatic assignment.

If you need exceptions, you handle them in the DHCP server rather than changing the whole pool. This keeps the scope predictable while still allowing special cases for specific devices or address ranges.

- Define the IP address range for the subnet.
- Match the subnet mask to the network design.
- Set the lease duration for how long a client keeps an address.
- Use exceptions when a device needs different handling.

**Reservations and fixed addressing**

A DHCP reservation assigns a fixed IP address to a specific device while still using DHCP for management. This is useful when a device needs a consistent address but you still want centralized control.

Reservations help reduce manual configuration and support reliability for devices that should always appear at the same address. They also fit into broader IP address management when you want predictable access without fully static configuration.

- Use a reservation for a device that needs the same IP address every time.
- Keep DHCP in control of the assignment process.
- Use reservations to support consistency and reliability.
- Avoid manual assignment when centralized management is preferred.

**Lease management**

A DHCP lease is the time a client is allowed to keep an assigned IP address. Lease duration is part of scope configuration, so it directly affects how often clients renew addresses and how quickly the server can reclaim unused ones.

Lease management matters because DHCP also handles deallocation of unused addresses. That automatic recovery helps prevent address exhaustion and keeps the pool available for new devices.

- Shorter leases return addresses to the pool sooner.
- Longer leases reduce renewal frequency.
- Lease settings affect address availability across the subnet.
- Automatic deallocation helps reclaim unused IP addresses.

**Troubleshooting scope and lease issues**

When DHCP problems appear, start with the server and the scope. Confirm that the DHCP Server service is running, the server is authorized, and the scope still has available leases for the client’s subnet. Also check for `BAD_ADDRESS` entries in address leases.

If clients cannot obtain addresses, the issue may be simple pool exhaustion or a scope mismatch. A practical check is to verify that the client is on the correct subnet and that the scope statistics show available leases.

> If the scope has no available leases, clients on that subnet cannot receive new addresses until you free or expand the pool.

**Quick review**

1. Identify the subnet and the scope that serves it.
2. Check the address range, subnet mask, and lease duration.
3. Verify whether a reservation is needed for a specific device.
4. Confirm the server is running and authorized.
5. Check scope statistics and address leases for availability or `BAD_ADDRESS` entries.

**Key terms**

- **Scope**: The address pool and related settings for a subnet.
- **Reservation**: A fixed IP assignment for a specific device.
- **Lease**: The time a client can keep an assigned address.
- **Pool**: The range of addresses available for assignment.
