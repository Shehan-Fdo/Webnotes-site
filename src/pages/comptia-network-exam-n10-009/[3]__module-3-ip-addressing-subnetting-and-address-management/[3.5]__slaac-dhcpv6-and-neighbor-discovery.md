## **Course:** CompTIA Network+ (Exam N10-009)

**IPv6 address configuration overview**

IPv6 supports automatic host configuration through **Stateless Address Autoconfiguration** (SLAAC) and **Dynamic Host Configuration Protocol version 6** (DHCPv6). These mechanisms reduce manual setup and help hosts obtain the network parameters they need to communicate.

SLAAC is designed to let an IPv6 host generate its own address with minimal administrator intervention. DHCPv6 is another way to assign IPv6 addresses on a network, and it is commonly discussed alongside SLAAC when planning IPv6 deployment.

- **SLAAC** focuses on automatic address generation.
- **DHCPv6** provides managed IPv6 configuration.
- IPv6 hosts still need a way to learn about routers and on-link prefixes.

**Neighbor Discovery fundamentals**

**Neighbor Discovery** (ND) is the IPv6 protocol family that supports local-link communication and address setup. It combines and improves on several IPv4 functions, including **Address Resolution Protocol** (ARP), **Internet Control Message Protocol** (ICMP), router discovery, and ICMP redirects.

ND helps IPv6 nodes:
- Discover routers on the local link.
- Learn link-layer addresses of neighbors.
- Maintain reachability information for active neighbors.
- Receive prefix and other link information from routers.

Routers use ND to advertise their presence and share information such as on-link prefixes and link settings. Hosts use ND to solicit information from routers and to communicate with other nodes on the same link.

**How SLAAC works with Neighbor Discovery**

SLAAC depends on Neighbor Discovery messages to learn the network prefix and other router-advertised information. A router sends **Router Advertisement** messages, which announce the router and advertise prefixes that are on-link.

That prefix information lets the host form an IPv6 address automatically. In practice, SLAAC gives you a simple way to bring up hosts without manually assigning every address.

- Router advertisements announce available prefixes.
- Hosts use the advertised prefix to build an address.
- SLAAC is intended to be simple and low-touch.

> SLAAC provides automatic address configuration, but it does not inherently provide all network services a host may need.

**DHCPv6 and when it matters**

DHCPv6 is the IPv6 version of DHCP used for automated address assignment and related configuration. It is not just a direct copy of IPv4 DHCP, so you should treat it as a separate protocol with its own behavior and deployment choices.

DHCPv6 appears in two common forms:
- **Stateful DHCPv6** assigns and tracks addresses.
- **Stateless DHCPv6** works with SLAAC and supplies additional configuration without fully managing address assignment.

A practical takeaway is that SLAAC and DHCPv6 are not always mutually exclusive. A network can use SLAAC for address creation and DHCPv6 for other configuration needs.

**Key differences to remember**

| Feature | SLAAC | DHCPv6 |
|---|---|---|
| Address assignment style | Host auto-generates address | Server-based assignment or supplemental config |
| Dependency on server | No dedicated address server required | Requires DHCPv6 infrastructure |
| Typical role | Simple automatic addressing | Managed IPv6 configuration |
| Relationship to ND | Uses router advertisements from ND | Can operate alongside SLAAC |

SLAAC is often described as simpler, while DHCPv6 gives administrators more control. In exam scenarios, focus on whether the question is asking about automatic self-configuration, managed assignment, or additional host settings.

**Practical exam points**

- **Neighbor Discovery** is central to IPv6 local-link operation.
- **Router Advertisement** messages are part of ND and help with prefix discovery.
- **SLAAC** lets a host automatically generate its own IPv6 address.
- **DHCPv6** is used for IPv6 address assignment and configuration.
- **Stateless DHCPv6** can complement SLAAC instead of replacing it.

**Troubleshooting mindset**

If an IPv6 host cannot configure itself, check the path from router discovery to address formation. A missing router advertisement, incorrect prefix advertisement, or absent DHCPv6 service can prevent the host from getting usable IPv6 settings.

A conservative approach is to verify:
1. The router is advertising the expected prefix.
2. The host is receiving Neighbor Discovery messages.
3. DHCPv6 is present if the network design depends on it.
4. The host has the minimum parameters needed to communicate.

**Quick recall**

- **ND** handles discovery, address resolution, and reachability on IPv6 links.
- **SLAAC** automates address creation from router-advertised prefix information.
- **DHCPv6** provides IPv6 configuration through server-managed methods.
- **Stateless DHCPv6** and SLAAC can work together.
