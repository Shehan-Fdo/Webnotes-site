## **Course:** CompTIA Network+ (Exam N10-009)

**IPv6 address fundamentals**

An <abbr title="Internet Protocol version 6">IPv6</abbr> address is a 128-bit identifier used to locate a network interface on an IPv6 network. Unlike IPv4, it uses a much larger address space and is written in hexadecimal.

- An IPv6 address identifies an **interface**, not just a device.
- A node can have more than one interface, and an interface can have more than one IPv6 address.
- If a device has multiple interfaces, you can reach it through any interface that is configured and reachable.

IPv6 addresses are typically written as eight groups of four hexadecimal digits separated by colons. Each group represents 16 bits, so the full address contains eight 16-bit fields.

```text
x:x:x:x:x:x:x:x
```

**Notation and representation**

IPv6 uses hexadecimal characters `0-9` and `A-F`. Hexadecimal is not case sensitive, so uppercase and lowercase forms are equivalent. In practice, lowercase is commonly preferred.

- The full form contains eight groups.
- Each group has four hexadecimal digits.
- Colons separate the groups.
- The preferred written form is the full `x:x:x:x:x:x:x:x` pattern.

A useful way to think about the structure is that the 128 bits are divided into eight segments. Each segment is sometimes described as a nibble-based grouping in the source material, but the practical exam takeaway is that each colon-delimited block is 16 bits wide.

**Address types**

IPv6 defines three primary address types:

- **Unicast**: identifies a single interface.
- **Multicast**: identifies a group of interfaces, usually on different nodes.
- **Anycast**: identifies a group of interfaces, but packets go to the physically closest member of the group.

These types matter because they describe how packets are delivered.

- Unicast targets one interface.
- Multicast targets all members of a group.
- Anycast targets the nearest eligible group member.

> IPv6 addresses are assigned to interfaces rather than nodes, so a single node may have multiple IPv6 addresses across multiple interfaces.

**Practical reading tips**

When you see an IPv6 address in notes or exam questions, focus on three things:

1. Count the address as 128 bits.
2. Recognize the colon-separated hexadecimal format.
3. Identify whether the question is describing unicast, multicast, or anycast behavior.

This topic is mostly about recognition and structure. If you can quickly map the notation to the 128-bit format and match the address type to its delivery behavior, you cover the core exam expectations.
