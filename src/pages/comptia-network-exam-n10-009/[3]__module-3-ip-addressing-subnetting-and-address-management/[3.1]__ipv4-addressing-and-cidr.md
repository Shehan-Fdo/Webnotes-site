## **Course:** CompTIA Network+ (Exam N10-009)

**IPv4 fundamentals**

IPv4 (Internet Protocol version 4) is a 32-bit addressing scheme used to identify devices on a network. An IPv4 address is written as four decimal numbers separated by periods, such as `192.168.1.165`. Each part is an octet, so the valid range for each value is `0` to `255`.

Every device that communicates using IPv4 needs a unique address on its local network. The address works with a subnet mask to show which part identifies the network and which part identifies the host.

- **32-bit** IPv4 space means there are `4,294,967,296` possible addresses in the full pool.
- **Octet** values are limited to `0-255` because each octet is 8 bits.
- **Subnet mask** helps your device determine whether a destination is local or remote.
- Some IPv4 addresses are reserved, so not every address in the pool is assignable.

**Network and host separation**

IPv4 addressing divides an address into a network section and a host section. The network portion identifies the subnet, while the host portion identifies the specific device on that subnet. This division is what lets devices decide whether to send traffic directly or forward it to a router.

A subnet mask is usually paired with the address to define that split. In practice, you use the mask to understand the size of the subnet and the number of usable host addresses it contains.

- The **network section** identifies the subnet.
- The **host section** identifies the device within that subnet.
- A **router** forwards traffic between different networks.
- Local devices use the mask to make the network decision; it is not typically the focus of normal packet transmission.

**CIDR notation**

CIDR (Classless Inter-Domain Routing) notation expresses the network prefix with a slash, such as `/24`, instead of writing the full decimal subnet mask. Many tools and platforms use CIDR notation directly when configuring addresses and networks.

CIDR makes subnet sizing easier to read and compare. The prefix length tells you how many bits belong to the network portion, and the remaining bits belong to hosts.

- `/24` is equivalent to `255.255.255.0`.
- `/16` is equivalent to `255.255.0.0`.
- `/30` is equivalent to `255.255.255.252`.
- `/32` represents a single address.

**Common CIDR examples**

| CIDR | Subnet mask | Total IP addresses | Usable IP addresses |
|---|---:|---:|---:|
| `/32` | `255.255.255.255` | 1 | 1 |
| `/31` | `255.255.255.254` | 2 | 2* |
| `/30` | `255.255.255.252` | 4 | 2 |
| `/29` | `255.255.255.248` | 8 | 6 |
| `/28` | `255.255.255.240` | 16 | 14 |
| `/27` | `255.255.255.224` | 32 | 30 |
| `/26` | `255.255.255.192` | 64 | 62 |
| `/25` | `255.255.255.128` | 128 | 126 |
| `/24` | `255.255.255.0` | 256 | 254 |
| `/23` | `255.255.254.0` | 512 | 510 |
| `/22` | `255.255.252.0` | 1024 | 1022 |
| `/21` | `255.255.248.0` | 2048 | 2046 |
| `/20` | `255.255.240.0` | 4096 | 4094 |

> `*` `/31` is shown in the source table as having 2 usable IP addresses.

**Why CIDR matters**

CIDR replaces rigid class-based thinking with flexible prefix lengths. That flexibility lets you size networks more precisely instead of choosing only broad class defaults. It also supports summarization, which helps reduce routing complexity by grouping address ranges.

For exam purposes, focus on recognizing the prefix, converting it to a subnet mask, and understanding how many addresses are available in the subnet.

- CIDR supports **classless** network design.
- Prefix length directly affects subnet size.
- Smaller host portions mean fewer usable addresses.
- Larger prefix numbers indicate smaller networks.

**Quick conversion habits**

When you see a CIDR prefix, translate it into the subnet mask and then infer the host capacity. This is the fastest way to answer subnetting questions under exam pressure.

1. Identify the prefix length, such as `/24`.
2. Convert it to the decimal subnet mask, such as `255.255.255.0`.
3. Determine the total address count for that subnet.
4. Subtract reserved addresses when you need usable host counts.

**Key takeaways**

IPv4 uses 32-bit addresses, subnet masks define network boundaries, and CIDR notation gives you a compact way to express those boundaries. If you can read the prefix, convert it to a mask, and estimate usable hosts, you cover the core exam skill for this topic.
