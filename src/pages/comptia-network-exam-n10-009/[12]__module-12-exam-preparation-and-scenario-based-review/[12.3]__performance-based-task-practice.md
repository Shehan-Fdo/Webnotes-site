## **Course:** CompTIA Network+ (Exam N10-009)

**Performance-based question overview**

Performance-based questions, or **PBQs**, test your ability to solve problems in real-world settings. CompTIA delivers them as simulations or within virtual environments, so you work through tasks instead of only selecting a multiple-choice answer.

PBQs may present a restricted simulation, such as a firewall, network diagram, terminal window, or operating system. Virtual environments can include full systems with more freedom to take incorrect steps, so you need to stay methodical and verify each change.

- **Simulations** usually limit available actions but still allow multiple valid paths.
- **Virtual environments** run select operating systems and software in a production-like setup.
- **Real-world focus** means you should expect troubleshooting, configuration, and validation tasks.

**How PBQ practice is framed in Network+**

The practice scenario in the evidence centers on a network security analyst working through performance issues and security concerns caused by improper configurations and outdated practices. The tasks include identifying network cable types, interpreting vulnerability scan results, and creating firewall rules to secure the network.

This kind of practice reinforces the habit of checking evidence before changing settings. You first identify what is present, then confirm what is working, and only then apply the fix.

- Identify network components from the scenario details.
- Interpret scan or status results before making changes.
- Apply firewall rules only after you understand the traffic path and the problem.

**Troubleshooting workflow**

A practical PBQ approach is to verify the current state, test connectivity, and isolate the likely cause. In the example scenario from the evidence, the workflow includes checking workstation IP configuration, pinging the default gateway, and then testing access to a target site before reviewing the router ACL.

Use a consistent sequence so you do not skip validation steps.

1. Check the workstation or device configuration.
2. Test the local path to the gateway.
3. Test the remote destination or service.
4. Review the control point, such as an ACL or firewall rule.
5. Make the smallest change that addresses the issue.
6. Retest to confirm the fix.

```bash
ipconfig
ping 10.0.0.1
ping certblaster.com
```

**Firewall and access control tasks**

PBQs often ask you to adjust firewall behavior based on observed connectivity. The evidence shows a scenario where the user cannot reach the Internet through a browser after router reconfiguration, and the task is to inspect the ACL and change the applicable rule.

When you work through these tasks, focus on the relationship between the source, destination, and the rule that blocks or permits traffic. Do not assume the first visible issue is the only issue; confirm the path with tests first.

> Make the change that matches the evidence you can verify, then test again before moving on.

**What to practice before the exam**

PBQ practice is most useful when you rehearse the process, not just the answer. You should be comfortable reading a scenario, extracting the relevant details, and mapping them to a configuration or troubleshooting action.

- Recognize common network components from context clues.
- Read scan or connectivity results carefully.
- Match symptoms to the correct control, such as a firewall rule or ACL.
- Validate your work after each change.

**Exam mindset**

CompTIA describes PBQs as items designed to test problem-solving in realistic settings. That means you should expect multiple possible paths in some simulations and more freedom to make mistakes in virtual environments.

Stay calm, work step by step, and use the evidence in the scenario to guide each action. If a task asks you to secure a network, verify the current state first, then apply the least disruptive fix that satisfies the requirement.
