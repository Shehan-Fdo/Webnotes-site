## **Course:** CompTIA Network+ (Exam N10-009)

**Network device roles**

Network devices are physical appliances that help computers communicate on a network. They can improve performance, extend coverage, and support secure connectivity by moving traffic where it needs to go.

- **Switches** connect devices within the same local network and forward traffic using packet switching.
- **Routers** connect different networks and are commonly used to link a home or office network to the internet.
- **Access points** provide Wi-Fi access so wireless devices can join the network.
- **Firewalls** help protect a network from harmful traffic.
- **Gateways** let different systems communicate when they use different protocols or network environments.

**Switches**

A switch is a multiport bridge that uses <abbr title="Media Access Control">MAC</abbr> addresses to forward frames at the data link layer. Compared with a hub, it sends traffic only where it needs to go, which reduces unnecessary collisions and improves efficiency.

- A switch receives data on one port and forwards it to the destination port.
- It supports packet switching, which helps traffic move in short bursts.
- Some switches also operate at layer 3 and can make routing decisions.
- Switches are the standard replacement for hubs in modern networks.

**Routers**

A router connects separate networks and directs traffic between them. In practical terms, you use a router when you need to move traffic from one network to another, such as from a local network to an external network.

- A router works at the network layer.
- It helps separate broadcast domains by keeping traffic between networks organized.
- It is a common placement point for internet access in homes and offices.
- Some advanced devices combine switching and routing features.

**Access points**

An access point (<abbr title="Wireless Access Point">WAP</abbr>) provides wireless connectivity for devices that need Wi-Fi access. It extends the reach of a wired network into a wireless one.

- An access point bridges wireless clients to the existing network.
- It is useful when you need coverage beyond the range of wired connections.
- Multiple access points can support broader wireless coverage.
- In mesh-style wireless setups, access points can communicate with each other to extend coverage.

**Security devices**

Security devices help control traffic and reduce risk on the network. They are often placed where traffic enters, leaves, or crosses sensitive network boundaries.

- A firewall filters traffic based on security rules.
- A gateway can sit between different systems and help them communicate.
- A demilitarized zone (<abbr title="Demilitarized Zone">DMZ</abbr>) is a network segment used to isolate public-facing services from the internal network.
- Placement matters because the device must inspect or control the traffic it is meant to protect.

> A hub is obsolete and should not be confused with a switch. A hub blindly broadcasts traffic to all ports, which increases collisions and exposes data to every connected device.

**Device placement and selection**

You choose a device based on the traffic problem you need to solve. The exam often tests whether you can identify where a device belongs in a network design.

1. Identify whether the traffic stays inside one network or must cross into another network.
2. Choose a switch for device-to-device communication within the same network.
3. Choose a router when traffic must move between networks.
4. Choose an access point when wireless clients need network access.
5. Add a firewall or DMZ when you need stronger control over exposed services.

**Quick comparison**

| Device | Main role | Typical placement |
|---|---|---|
| Switch | Forwards traffic within a LAN using MAC addresses | Inside the local network |
| Router | Connects different networks | Edge of the network or between subnets |
| Access point | Provides wireless access | Where Wi-Fi coverage is needed |
| Firewall | Filters traffic for security | Network boundary or perimeter |
| Gateway | Enables communication between different systems | Between dissimilar networks or services |

**Exam focus**

Network+ expects you to compare devices by function and placement, not just by name. When you see a scenario, focus on what the device must do: forward within a LAN, route between networks, provide wireless access, or enforce security controls.

- Use **switch** for local forwarding.
- Use **router** for network-to-network communication.
- Use **access point** for wireless connectivity.
- Use **firewall** for traffic filtering.
- Use **gateway** when systems need translation or mediation.
