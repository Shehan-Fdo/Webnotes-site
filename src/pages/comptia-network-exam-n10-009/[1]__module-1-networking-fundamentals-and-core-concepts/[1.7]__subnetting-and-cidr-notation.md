## **Course:** CompTIA Network+ (Exam N10-009)

**Subnetting fundamentals**

Subnetting is the process of dividing a larger IP network into smaller logical networks called subnets. This helps you organize hosts, limit broadcast traffic, and route traffic more efficiently between groups of devices.

An IP address is a unique numerical label assigned to a device on a network. In IPv4, the address is a 32-bit number written as four octets separated by periods, such as `192.168.1.1`. Each device must have a unique address to participate in communication on the network.

- **Subnet** boundaries separate devices into logical groups.
- **Broadcast** traffic stays within the subnet, which reduces unnecessary network-wide traffic.
- **Routing** handles traffic between subnets through a router.
- **Manageability** improves when departments or functions use separate address ranges.

**Why subnetting matters**

Subnetting supports efficient IP address usage and better network control. Instead of placing all devices on one large network, you can split the network into smaller segments that match organizational needs.

This approach is useful when you want to keep traffic organized and reduce the impact of one group on another. It also helps with security policy design because you can isolate departments or device types into separate logical networks.

- Use subnetting to conserve addresses.
- Use subnetting to reduce broadcast scope.
- Use subnetting to separate traffic by department or purpose.
- Use subnetting to improve performance and administration.

**CIDR notation**

<abbr title="Classless Inter-Domain Routing">CIDR</abbr> notation is a flexible way to define subnets. It replaces older class-based thinking with a prefix length that shows how many bits identify the network portion of the address.

CIDR notation appears as an IP address followed by a slash and a number, such as `192.168.1.0/24`. The `/24` tells you how many bits are used for the network portion, which helps define the size of the subnet.

- `CIDR` stands for Classless Inter-Domain Routing.
- The slash value is the prefix length.
- A larger prefix length means a smaller subnet.
- CIDR helps represent IP ranges more flexibly than class A, B, and C addressing.

**Subnet masks and binary structure**

Subnet masks describe which part of an IP address belongs to the network and which part belongs to hosts. In binary, the mask uses `1` bits for the network portion and `0` bits for the host portion.

For example, `255.255.128.0` in binary is:

```text
11111111.11111111.10000000.00000000
```

This binary view is useful because it shows exactly where the network ends and the host portion begins. When you read or calculate subnetting values, the mask tells you how many addresses belong to the subnet and how many remain available for hosts.

- **Network bits** are the `1` bits in the mask.
- **Host bits** are the `0` bits in the mask.
- The mask determines subnet size.
- Binary conversion helps you verify subnet boundaries.

**Practical subnetting workflow**

1. Identify the IP address and prefix length.
2. Determine the subnet mask from the prefix.
3. Separate the network portion from the host portion.
4. Use the subnet to group devices that should communicate locally.
5. Route traffic between subnets when devices need to reach other logical networks.

> A subnet is a logical boundary, not a physical cable boundary. Devices can share the same physical network while still being separated into different subnets.

**Common exam focus**

Network+ questions often test whether you can recognize the purpose of subnetting, interpret CIDR notation, and relate subnet masks to network and host portions. You should be comfortable reading a prefix such as `/24` and understanding that it describes how the address space is divided.

You should also recognize that subnetting is about more than address math. It is a design tool for performance, security, and manageability.

- Know that subnetting divides one network into smaller logical networks.
- Know that CIDR uses slash notation to express the prefix length.
- Know that subnet masks can be read in binary.
- Know that routers move traffic between subnets.

**Quick reference**

| Concept | Meaning |
