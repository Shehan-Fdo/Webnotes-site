## **Course:** CompTIA Network+ (Exam N10-009)

**OSI and TCP/IP overview**

The Open Systems Interconnection (<abbr title="Open Systems Interconnection">OSI</abbr>) model and the <abbr title="Transmission Control Protocol / Internet Protocol">TCP/IP</abbr> model are logical frameworks for describing how data moves between devices on a network. They help you place protocols, understand packet flow, and troubleshoot communication issues.

The OSI model is a seven-layer reference model. The TCP/IP model is a practical protocol suite commonly described with four or five layers, depending on how the lower layers are grouped. Both models describe the same communication process from different viewpoints.

- **OSI** is a conceptual framework used to standardize and explain network communication.
- **TCP/IP** is the protocol stack used in real networks.
- Both models describe how information is transmitted between two devices.
- The OSI model is protocol-independent; TCP/IP relies on standardized protocols.

**OSI model layers**

The OSI model divides communication into seven layers. Each layer has a distinct role and works independently in the reference model. This layered approach makes it easier to isolate where a problem occurs.

1. **Physical**: Sends raw bits over cables, connectors, and signals.
2. **Data Link**: Handles local delivery on the same network segment.
3. **Network**: Handles logical addressing and routing between networks.
4. **Transport**: Supports end-to-end delivery and transport functions.
5. **Session**: Manages communication sessions between systems.
6. **Presentation**: Handles data representation and formatting.
7. **Application**: Provides services directly to user-facing applications.

The OSI model is not actual software or hardware. It is a reference model that gives you a common language for describing what networking devices and protocols are doing.

**TCP/IP model layers**

The TCP/IP model is usually shown with four layers, though some sources describe five by separating the lower network functions more finely. In practical study, focus on the main four-layer view and understand that the lower layers cover physical and link-level delivery.

- **Application**: Combines user-facing network services and higher-level communication functions.
- **Transport**: Provides transport functions between hosts.
- **Internet**: Handles internetworking and IP addressing.
- **Network access**: Covers local network delivery and physical transmission details.

TCP/IP is the model you see in production networks. It groups several OSI functions into fewer layers, which makes it more practical for real protocol use.

**OSI vs. TCP/IP**

The two models are similar because they both explain how data moves across a network, but they differ in scope and structure. OSI is broader as a teaching and troubleshooting framework, while TCP/IP is the protocol suite that networks actually use.

| OSI | TCP/IP | Notes |
|---|---|---|
| 7 layers | 4 or 5 layers | TCP/IP compresses several OSI functions |
| Reference model | Protocol suite | OSI is conceptual; TCP/IP is practical |
| Protocol-independent | Protocol-based | TCP/IP uses standardized protocols |
| Separate session and presentation layers | Combined into application layer | TCP/IP groups higher-level functions |

Use the OSI model when you need a structured way to reason about where a function belongs. Use TCP/IP when you want to map that function to the protocol stack used in real communication.

**Encapsulation**

Encapsulation is the process of adding headers, and sometimes trailers, as data moves down the stack. Each layer adds information needed for delivery, then passes the result to the next layer.

This process helps you understand what a packet contains at each stage. For example, application data becomes transport data with transport information added, then network information is added, and finally local delivery information is added before transmission.

- **Header**: Added by a layer to carry control information.
- **Trailer**: Added at the lower layers when needed for local delivery.
- **Encapsulation**: The wrapping of data with layer-specific information.
- **Packet**: A general term for data with added protocol information.

> Encapsulation is a core troubleshooting concept because it shows where addressing, transport, and delivery information are added or examined.

**How to think about the stack**

When you study the models, focus on function first and memorization second. Ask what each layer is responsible for, then map protocols and devices to that responsibility.

- The upper layers focus on user interaction and data formatting.
- The middle layers focus on delivery between hosts and networks.
- The lower layers focus on moving bits across the physical medium.
- Layered design helps different vendors’ hardware, applications, and protocols work together.

**Practical study tips**

Use the OSI model as a troubleshooting reference and the TCP/IP model as a real-world communication map. If you can identify where a problem occurs in the stack, you can narrow down likely causes faster.

1. Identify whether the issue is local delivery, routing, transport, or application-related.
2. Map the symptom to the most likely layer.
3. Check the protocol or device responsible for that layer.
4. Move up or down the stack until you isolate the fault.

**Key takeaways**

The OSI model gives you a seven-layer reference for understanding network communication. The TCP/IP model groups those ideas into a smaller, practical stack used in real networks. Encapsulation ties the models together by showing how data gains headers and trailers as it moves through the layers.
