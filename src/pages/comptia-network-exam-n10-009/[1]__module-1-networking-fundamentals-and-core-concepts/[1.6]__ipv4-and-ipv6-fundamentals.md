## **Course:** CompTIA Network+ (Exam N10-009)

**IPv4 and IPv6 overview**

Internet Protocol (<abbr title="Internet Protocol">IP</abbr>) addressing is a core part of network communication. Devices use IP addresses to identify endpoints, and the address is often paired with a subnet mask so the local device can determine which subnet it belongs to. The subnet mask is typically used locally rather than transmitted across the network.

IPv4 and IPv6 are both used in modern networks. IPv4 remains widely used, while IPv6 is designed to provide a larger address space and includes address forms that are commonly discussed in troubleshooting and configuration.

- **IPv4** uses a familiar dotted-decimal format such as `192.168.1.165`.
- **Subnet masks** help separate the network portion from the host portion on the local device.
- **IPv6** addresses can be compressed or expanded to improve readability.
- **TCP/IP** networks rely on unique addresses for device communication.

**IPv4 fundamentals**

IPv4 addresses are commonly written as four decimal octets separated by periods. In practice, you usually configure the address together with the subnet mask so the host can determine its local network boundaries. This is essential when you assign an address to a workstation or other endpoint.

IPv4 addressing also includes common categories such as public and private addresses. Public addresses are used for Internet-facing communication, while private addresses are used inside local networks. Understanding that distinction helps you interpret how a device reaches internal resources versus external services.

- **Public IP** addresses are reachable across the Internet.
- **Private IP** addresses are used inside local networks.
- **RFC 1918** defines the private IPv4 address space.
- **IPv4 classes** include A, B, C, D, and E in traditional addressing discussions.

**IPv6 fundamentals**

IPv6 uses a longer address format than IPv4 and is commonly presented in hexadecimal groups separated by colons. Because the full form can be lengthy, IPv6 supports abbreviation so you can shorten repeated or unnecessary zeros and make the address easier to read.

IPv6 also introduces address types that are important in day-to-day networking. These include link-local addresses, global unicast addresses, anycast, and loopback. Knowing the purpose of each type helps you identify how a device communicates on the local link or across larger networks.

- **Link-local** addresses are used on the local network segment.
- **Global unicast** addresses are used for routable communication.
- **Anycast** addresses can be associated with more than one interface.
- **Loopback** addresses let a device refer to itself.

**Common address types and their use**

Some address types appear in both troubleshooting and configuration tasks because they describe where traffic starts, where it can go, and how a device identifies itself. Loopback is especially useful for testing local stack behavior, while link-local addresses are important for local-only communication.

<dl>
<dt><strong>Loopback</strong></dt>
<dd>A virtual interface used to refer to the local machine; in IPv4, <code>127.0.0.1</code> is the common loopback address.</dd>
<dt><strong>Link-local</strong></dt>
<dd>An address used only on the local network segment and not intended for routable communication beyond that link.</dd>
<dt><strong>Global unicast</strong></dt>
<dd>An IPv6 address type used for globally routable communication.</dd>
<dt><strong>Anycast</strong></dt>
<dd>An address assigned to multiple interfaces so traffic can be delivered to the nearest appropriate destination.</dd>
</dl>

> <mark>Critical</mark>: The subnet mask is used locally to interpret the address; it is not usually transmitted across the network.

**Practical comparison**

| Topic | IPv4 | IPv6 |
|---|---|---|
| Address format | Dotted decimal | Colon-separated hexadecimal |
| Common readability aid | Subnet mask | Compression and expansion |
| Typical local-only example | `127.0.0.1` loopback | Link-local address |
| Routing scope | Public and private use | Global unicast and local address types |

**Working with addresses in practice**

When you configure or troubleshoot an address, start by identifying the address type and the scope it is meant to serve. Then confirm the subnet information so you know whether the device should communicate locally or send traffic to a gateway.

1. Identify whether the address is IPv4 or IPv6.
2. Check whether the address is public, private, link-local, loopback, or global unicast.
3. Verify the subnet mask or prefix information.
4. Confirm whether the device should communicate on the local link or across routed networks.

**Key takeaways**

IPv4 and IPv6 both support device communication, but they differ in format, size, and common address types. IPv4 is often paired with a subnet mask, while IPv6 uses address compression and includes types such as link-local, global unicast, anycast, and loopback. For troubleshooting, always start with the address type and the scope it is intended to serve.
