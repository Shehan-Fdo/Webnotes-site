## **Course:** CompTIA Network+ (Exam N10-009)

**Why ports and protocols matter**

Network protocols define how devices exchange information, and ports help direct traffic to the right application or service. For Network+, you need to recognize common port numbers and connect them to their typical uses. This supports security, troubleshooting, and network optimization.

Protocols are often grouped into broad categories:

- **Communication** protocols move data between devices.
- **Management** protocols control and monitor network operations.
- **Security** protocols protect data, authentication, and integrity during transmission.

The OSI (Open Systems Interconnection) model and the TCP/IP (Transmission Control Protocol/Internet Protocol) model provide the context for how these services fit into network communication.

**Common ports to know**

Many exam questions focus on matching a port number with its service. Learn the port, the protocol, and the purpose together.

| Port | Protocol | Common use |
|---:|---|---|
| 20 | FTP | Data transfer |
| 21 | FTP | Control commands |
| 22 | SSH | Secure remote administration and SFTP |
| 23 | Telnet | Remote console access without encryption |
| 25 | SMTP | Sending email |
| 53 | DNS | Resolving domain names to IP addresses |
| 80 | HTTP | Standard web traffic |
| 110 | POP3 | Retrieving email |
| 443 | HTTPS | Encrypted web traffic |

> Telnet sends credentials and session data in the clear, so it is generally replaced by SSH for secure remote access.

**Protocol functions and exam focus**

A port is the endpoint a service listens on, while the protocol defines the rules for communication. In practice, you use both concepts together when identifying traffic or diagnosing connectivity problems.

- **FTP** uses separate ports for control and data, which is why port 20 and port 21 are both associated with it.
- **SSH** provides encrypted remote access and is the secure alternative to Telnet.
- **SMTP** handles outbound email delivery.
- **DNS** translates human-readable names into IP addresses.
- **HTTP** and **HTTPS** support web traffic, with HTTPS adding encryption.
- **POP3** is used for email retrieval.

**How to study these ports**

Focus on recognition, not just memorization. You should be able to identify the service from the port number and explain why that service matters in a real network.

1. Learn the most common port numbers and their associated protocols.
2. Associate each protocol with its primary function.
3. Compare insecure and secure options, such as Telnet versus SSH and HTTP versus HTTPS.
4. Use the port number as a clue when troubleshooting application connectivity.

**Practical troubleshooting mindset**

When an application fails, the port number often tells you where to look first. If a web service is unreachable, check whether traffic to port 80 or 443 is allowed. If remote administration fails, confirm whether the device expects SSH on port 22 rather than Telnet on port 23.

This topic is especially useful because Network+ emphasizes real-world connectivity, monitoring, and troubleshooting. Knowing the common ports helps you interpret traffic, verify service availability, and reduce guesswork during diagnosis.
