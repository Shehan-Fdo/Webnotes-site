## **Course:** CompTIA Network+ (Exam N10-009)

**Email protocol roles**

Email protocols define how messages move between clients and mail servers, and they split into sending and retrieval tasks. In practice, you use one protocol to send mail and another to access it.

- **SMTP** (Simple Mail Transfer Protocol) handles sending and relaying email.
- **IMAP** (Internet Message Access Protocol) and **POP3** (Post Office Protocol 3) handle receiving and mailbox access.
- Email systems involve the sender, recipient, and mail server, so the protocol choice depends on the step in the message flow.

SMTP is the protocol you associate with outbound mail. It sends messages from an email client to a mail server and can relay mail between servers when the sender and recipient use different providers.

**SMTP behavior and ports**

SMTP is a transfer protocol, so it focuses on delivery rather than mailbox synchronization. It commonly uses port `25`, and it may also use ports `587` and `465` for submission or secure delivery scenarios.

- Use SMTP when you need to send email from a client to a server.
- Use SMTP relay when mail must move from one server to another.
- Remember that SMTP does not manage long-term mailbox retrieval.

```text
SMTP default port: 25
Common alternate ports: 587, 465
Primary role: sending and server-to-server transfer
```

**IMAP and POP3 for retrieval**

IMAP and POP3 are the two main retrieval protocols. They differ in how they treat messages after access, which affects device sync and offline use.

- **IMAP** keeps messages on the server and supports access from multiple devices.
- **POP3** downloads messages to a device and is better when you want local, offline access.
- Choose IMAP when you want the mailbox to stay synchronized across clients.
- Choose POP3 when you prefer a download-first workflow.

| Protocol | Main use | Typical behavior |
|---|---|---|
| IMAP | Mail retrieval | Keeps mail on the server for multi-device access |
| POP3 | Mail retrieval | Downloads mail for local access |
| SMTP | Mail sending | Sends mail and relays it between servers |

**How to choose the right protocol**

You do not pick one protocol for every email task. Instead, you match the protocol to the stage of the workflow.

1. Use SMTP to submit or relay outbound mail.
2. Use IMAP if you need synchronized access across devices.
3. Use POP3 if you want messages stored locally after download.

> If you remember one thing, SMTP is for sending and transfer, while IMAP and POP3 are for retrieval behavior.

**Exam focus**

For Network+ purposes, focus on the functional differences rather than memorizing protocol names in isolation. The exam expects you to recognize which protocol fits a scenario.

- Sending mail from a client to a server points to SMTP.
- Accessing the same mailbox from multiple devices points to IMAP.
- Downloading mail for offline reading points to POP3.
- When a question mentions relaying mail between servers, think SMTP.

**Practical memory aid**

A simple way to separate the protocols is to map them to the message lifecycle:

- **SMTP** moves the message out.
- **IMAP** leaves the message in place and syncs it.
- **POP3** pulls the message down.

This distinction helps you answer scenario-based questions quickly and avoid mixing up sending with retrieval.
