## **Course:** CompTIA Network+ (Exam N10-009)

**Core network services overview**

Core network services are the foundational services that help devices resolve names, authenticate users, synchronize time, log events, and manage network infrastructure. In practice, you should recognize both the service purpose and the protocol commonly associated with it.

- **DNS** translates human-readable names into IP addresses so clients can reach the correct host.
- **DHCP** is a core network service you should associate with automatic address assignment.
- **NTP** keeps system clocks synchronized across devices.
- **SNMP** supports network monitoring and management.
- **Syslog** centralizes log messages from network devices and servers.
- **LDAP** provides directory-based access to user and resource information.
- **TACACS+** and **RADIUS** support centralized authentication, authorization, and accounting (AAA).

**Name resolution and addressing**

Domain Name System (DNS) is often described as the internet’s phonebook. You use a name such as a website address, and DNS returns the IP address needed to connect. This service reduces the need to remember numeric addresses and is one of the most visible core services in daily network use.

Dynamic Host Configuration Protocol (DHCP) is another essential service to recognize in exam scenarios. It is commonly tied to automatic network configuration, especially when clients need an address without manual setup. When you see a device joining a network and receiving configuration automatically, DHCP is the service to consider.

**Time, logging, and monitoring**

Time synchronization matters because logs, authentication events, and troubleshooting data are easier to correlate when devices share a consistent clock. Network Time Protocol (NTP) is the standard service to associate with time alignment across systems.

Monitoring and logging services help you observe network health and investigate issues:

- **SNMP** is used for collecting and managing device information.
- **Syslog** sends event messages to a central logging destination.
- These services are often configured alongside other management servers in a centralized policy.

> Be careful not to confuse the purpose of a service with the transport or management method used to configure it. On the exam, focus on what the service does first.

**Authentication and centralized access**

Centralized access services reduce the need to manage credentials on every device individually. The evidence highlights two common protocols:

- **RADIUS** is used for centralized access and is commonly associated with authenticating remote network users.
- **TACACS+** is commonly used for administrator access to network devices such as routers and switches.
- Both protocols provide centralized AAA management.

AAA stands for:

- **Authentication**: who is allowed to gain access
- **Authorization**: what a user can do after access is granted
- **Accounting**: what actions or usage are recorded

A practical exam takeaway is that RADIUS and TACACS+ both support centralized access control, but TACACS+ is especially associated with device administration.

**Common ports and service recognition**

You should know the well-known ports tied to common services and applications. The evidence specifically identifies these examples:

| Service | Common Port | Notes |
|---|---:|---|
| FTP | TCP 20/21 | Uses separate ports for data and control |
| Telnet | TCP 23 | Remote console access, but unencrypted |

FTP uses TCP ports 20 and 21, with one port commonly tied to data transfer and the other to control information. Telnet uses TCP port 23 and sends data in the clear, so it is important to recognize it as an unencrypted remote access protocol.

**Management server concepts**

In some network platforms, management services are grouped into a policy that can include multiple servers. The evidence shows that management server configuration may include:

- DNS
- LDAP
- NTP
- SNMP
- Syslog
- TACACS+

This reinforces a useful study pattern: many core services are deployed together as part of a broader management strategy rather than as isolated features.

**Quick exam recognition cues**

Use these cues to narrow down answers quickly:

- Name to IP lookup points to **DNS**.
- Automatic host configuration points to **DHCP**.
- Clock alignment points to **NTP**.
- Device monitoring points to **SNMP**.
- Central log collection points to **Syslog**.
- Directory-based identity services point to **LDAP**.
- Remote user or device access control points to **RADIUS** or **TACACS+**.
- Remote console access over TCP 23 points to **Telnet**.
- File transfer using TCP 20/21 points to **FTP**.

**Study focus**

For Network+ exam prep, learn each service by its function first, then attach the protocol name and common port where applicable. That approach helps you answer both definition-style and scenario-style questions without relying on memorization alone.
