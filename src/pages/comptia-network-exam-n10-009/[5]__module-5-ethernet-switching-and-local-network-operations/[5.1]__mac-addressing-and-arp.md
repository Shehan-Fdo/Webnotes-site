## **Course:** CompTIA Network+ (Exam N10-009)

**MAC addressing fundamentals**

A <abbr title="Media Access Control">MAC</abbr> address is the hardware address used on a local network to identify a network interface. It is a fixed physical machine address that helps devices deliver traffic to the correct destination on a <abbr title="Local Area Network">LAN</abbr>.

MAC addresses are 48 bits long, or 6 bytes, and are commonly shown in hexadecimal. Because they are used at the data-link layer, they support local frame delivery rather than end-to-end routing.

- **48-bit format**: A MAC address uses 6 bytes.
- **Hexadecimal display**: It is usually written with separators such as colons or periods.
- **Unique identifier**: It identifies a specific network adapter on the local network.
- **Local scope**: It is used for communication within the same broadcast domain.

The first three bytes are the <abbr title="Organizationally Unique Identifier">OUI</abbr>, which identifies the manufacturer. The last three bytes identify the specific interface. This structure helps distinguish the vendor from the device-specific portion of the address.

**ARP purpose and role**

<abbr title="Address Resolution Protocol">ARP</abbr> maps an <abbr title="Internet Protocol">IP</abbr> address to a MAC address so a device can build an Ethernet frame for local delivery. This matters because an IP address and a MAC address are different lengths and serve different roles.

ARP operates on the data-link layer and works within a LAN or broadcast domain. It is used when you know the destination IP address but need the destination MAC address before sending the frame.

- **IP-to-MAC mapping**: ARP resolves a known IP address to the needed MAC address.
- **Local communication**: It supports delivery on the same broadcast domain.
- **Frame construction**: It helps you place the IP packet inside an Ethernet frame.
- **Address translation**: It bridges the 32-bit IPv4 address space and the 48-bit MAC address format.

> ARP is not used for routing across different networks; it supports local delivery on the same LAN.

**How ARP works**

When the destination MAC address is not already known, the device sends an ARP request as a broadcast on the local network. The device that owns the requested IP address responds with an ARP reply, typically as a unicast message.

This exchange lets the sender learn the correct MAC address and continue communication without guessing.

1. You need to send traffic to a device on the local network.
2. You know the destination IP address but not the MAC address.
3. You send an ARP request to the broadcast domain.
4. The device with that IP address sends an ARP reply.
5. You use the learned MAC address to send the Ethernet frame.

**ARP cache**

Devices maintain an ARP cache to store recently resolved IP-to-MAC mappings. This reduces repeated ARP requests and lowers network overhead.

An ARP cache entry is temporary. When the timeout expires, the entry is removed so the device does not keep using an outdated mapping.

- **Cache benefit**: It speeds up future lookups.
- **Temporary storage**: It holds recently learned mappings.
- **Timeout behavior**: Entries expire after a period of validity.
- **Freshness**: Expiration helps prevent stale address use.

**Operational notes and security**

ARP has no built-in authentication, so it can be abused by attackers. One common risk is ARP spoofing, where false mappings are introduced to misdirect traffic.

Practical defenses include static ARP entries, Dynamic ARP Inspection, secure DHCP, and network segmentation. These controls help reduce the chance of forged address mappings affecting local traffic.

- **Static ARP entries**: Use fixed mappings where appropriate.
- **Dynamic ARP Inspection**: Validate ARP behavior on the network.
- **Secure DHCP**: Reduce opportunities for unauthorized address assignment.
- **Segmentation**: Limit the impact of local-layer attacks.

**Key takeaways**

MAC addressing identifies the local hardware destination, while ARP finds the MAC address for a known IP address. Together, they let devices on the same LAN deliver frames correctly and efficiently.

- MAC addresses are fixed, 48-bit hardware addresses.
- ARP resolves IPv4 addresses to MAC addresses.
- ARP uses broadcast requests and replies to learn mappings.
- ARP caches improve efficiency but must expire stale entries.
- ARP security matters because the protocol lacks authentication.
