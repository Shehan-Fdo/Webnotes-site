## **Course:** CompTIA Network+ (Exam N10-009)

**Overview**

Port security and switch hardening focus on the access layer, where users and devices physically connect to the network. This layer is often the weakest point because anyone with access to a wall jack or switch port can attempt to join the network. Hardening reduces that risk by limiting what can connect, restricting unused interfaces, and tightening administrative access.

- **Access layer** ports are exposed to the most device turnover and physical access.
- Unused ports should be secured before production deployment.
- Switch hardening applies to both active and inactive interfaces.

**Port Security Basics**

Port security is a Layer 2 control that limits which devices can use a switch port. It helps prevent unauthorized devices, such as rogue laptops, from connecting to the local network. In practice, you use it on access ports where you expect a known device or a small set of known devices.

A switch without access control forwards traffic for whatever device appears on the port. That default behavior makes the edge easy to use, but also easy to abuse. Port security adds a simple enforcement layer so the switch can reject or restrict unexpected devices.

- **Layer 2** controls operate at the switch frame level.
- Port security is most useful on user-facing access ports.
- It is a practical defense when devices are likely to be moved or swapped.

**Securing Unused Ports**

Unused switch ports should not remain open. A common hardening approach is to place them in access mode, move them to an isolated VLAN, and shut them down. This reduces the chance that someone can plug into an idle port and gain network access.

This is a straightforward baseline control and should be part of initial switch deployment. It is especially important in large environments where many ports may be physically reachable.

- Put unused ports in an isolated **VLAN**.
- Disable ports that do not need to be active.
- Review unused interfaces before production use.

**Administrative and Interface Hardening**

Switch hardening also includes protecting the switch itself, not just the ports. The evidence highlights secure administrative access, restricted remote access, console port protection, and general interface protection as part of a broader hardening strategy. These controls reduce the chance that an attacker can manage the switch or abuse exposed services.

You should treat the switch as a critical infrastructure device. If an attacker gains administrative access, they can alter forwarding behavior, weaken segmentation, or disable protections.

- Restrict **administrative access** to trusted management paths.
- Protect the **console port** and remote access methods.
- Keep switch interfaces limited to required functions.

**Related Layer 2 Protections**

Port security works best alongside other Layer 2 protections. The source material points to VLAN segmentation, DHCP guard, ARP inspection, spanning tree protections, and storm control as additional hardening measures. These controls address different attack paths and help reduce the impact of misbehaving or malicious devices.

Use them as a layered defense rather than relying on a single feature. For example, port security limits who can connect, while other controls help protect the network from spoofing, loops, or excessive traffic.

- **VLAN** segmentation limits broadcast exposure.
- **DHCP guard** helps block unauthorized DHCP behavior.
- **ARP inspection** helps protect against address spoofing.
- **Spanning Tree** protections reduce topology abuse.
- **Storm control** helps contain excessive traffic.

**Practical Exam Takeaways**

For exam purposes, focus on the purpose of each control rather than vendor-specific syntax. The key idea is that access-layer switches need protection because they sit closest to end users and physical access. If a question describes an open wall jack, an unused port, or a rogue device, think about port security and switch hardening.

1. Identify whether the issue is an active user port or an unused port.
2. Apply port security to known access ports that should only accept expected devices.
3. Disable or isolate unused ports.
4. Add supporting Layer 2 protections where appropriate.
5. Restrict switch management access to trusted administrators and paths.
