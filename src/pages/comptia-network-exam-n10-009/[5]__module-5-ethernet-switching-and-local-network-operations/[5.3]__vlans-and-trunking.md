## **Course:** CompTIA Network+ (Exam N10-009)

**VLAN fundamentals**

A <abbr title="Virtual Local Area Network">VLAN</abbr> separates devices into logical networks even when they share the same physical switching infrastructure. This lets you segment traffic by function, department, or device type without moving cables.

The key exam idea is that devices in different VLANs do not communicate by default. If you need communication between VLANs, you must add routing.

- VLANs create logical separation on a switch network.
- Devices in different VLANs stay isolated until routing is configured.
- VLAN design helps you extend network segmentation beyond a single physical layout.

**Trunking and why it matters**

A trunk link carries traffic for multiple VLANs over one physical connection between switches or other network devices. Without a trunk, VLAN traffic stays confined to the local switch segment.

Trunk ports use <abbr title="Institute of Electrical and Electronics Engineers">IEEE</abbr> 802.1Q tagging to label frames so the receiving device knows which VLAN each frame belongs to. This tagging is what allows multiple VLANs to traverse the same link.

- **Trunk port**: carries multiple VLANs.
- **802.1Q**: adds VLAN tags to Ethernet frames.
- No trunk means no VLAN communication between switches.

> A frame that crosses a VLAN-aware link must be identifiable as belonging to exactly one VLAN.

**Inter-VLAN routing**

If you want hosts in separate VLANs to reach each other, you need routing between those VLANs. The evidence highlights two common approaches: router-on-a-stick and Layer 3 switching.

Router-on-a-stick uses one physical router interface to route multiple VLANs through subinterfaces. Layer 3 switching performs inter-VLAN routing on a switch that supports routing functions.

- **Router-on-a-stick** uses one physical interface for multiple VLANs.
- **Layer 3 switching** provides inter-VLAN routing on the switch.
- Routing is required for communication across VLAN boundaries.

**Troubleshooting focus**

When VLANs or trunks are misconfigured, the most common symptom is that traffic does not pass where you expect it to. Start by checking whether the VLAN exists, whether the port is assigned correctly, and whether the trunk is actually carrying the needed VLANs.

The evidence also points to switching loop prevention as part of the broader switching troubleshooting picture. Spanning Tree Protocol (STP) helps stop loops before frames circulate endlessly.

1. Verify the VLAN assignment on the access port.
2. Confirm the trunk link is up and carrying the required VLANs.
3. Check whether inter-VLAN routing is configured if hosts in different VLANs must communicate.
4. Review loop prevention if switching behavior looks abnormal.

**Exam takeaways**

VLANs are about logical separation, trunks are about carrying multiple VLANs across a single link, and routing is what connects VLANs when communication is required. Keep those three ideas distinct when you study.

- VLANs isolate traffic.
- Trunks extend VLANs across links.
- Routing enables communication between VLANs.
- 802.1Q is the tagging method associated with trunking.
