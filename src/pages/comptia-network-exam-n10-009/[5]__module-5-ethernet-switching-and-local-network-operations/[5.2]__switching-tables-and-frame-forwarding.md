## **Course:** CompTIA Network+ (Exam N10-009)

**Switching logic overview**

A switch is a <abbr title="Local Area Network">LAN</abbr> device that forwards Ethernet frames between interfaces using <abbr title="Media Access Control">MAC</abbr> address information. It does not treat every frame the same way; it examines the source and destination MAC addresses and then decides whether to send the frame to one port, several ports, or no additional ports.

The core idea is simple: the switch learns where devices are, stores that information in a MAC address table, and uses that table to make forwarding decisions. This reduces unnecessary traffic compared with older shared-network designs.

- **Learning** happens when the switch reads the source MAC address of an incoming frame and updates its table.
- **Forwarding** happens when the switch uses the destination MAC address to choose an output port.
- **Flooding** happens when the switch does not yet know the destination location.
- **Filtering** happens when the destination is on the same incoming port, so the frame does not need to be sent back out that port.

**MAC address table behavior**

The MAC address table, also called a forwarding database (FDB), maps a device MAC address to the switch port where that device was last seen. This table is the switch’s memory for device locations within the LAN.

When a frame arrives, the switch first learns from the source address. Then it checks the destination address against the table:

- If the destination is known, the switch sends the frame only to the correct port.
- If the destination is unknown, the switch floods a copy out of all ports except the one it entered.
- If the destination is broadcast, the switch sends it to all ports in the broadcast domain.
- If the destination is multicast, the switch may send it to multiple ports that need the traffic.

> A switch forwards based on the destination MAC address inside the frame, so loops can cause the same frame to circulate repeatedly if you do not control them.

**Frame forwarding outcomes**

The forwarding decision depends on what the switch knows at the moment the frame arrives. This is why the first frame to a new destination often gets flooded, while later frames can be sent directly.

| Destination type | Switch action | Result |
|---|---|---|
| Known unicast | Forward to one port | Efficient delivery to the learned port |
| Unknown unicast | Flood to all ports except ingress | Helps discover the destination location |
| Broadcast | Flood to all ports in the broadcast domain | Reaches every device that must see it |
| Multicast | Send to multiple interested ports | Reaches a group of receivers |

This behavior explains why switching improves performance: once the switch learns the path to a device, it avoids sending traffic everywhere.

**Learning and forwarding sequence**

1. The switch receives an Ethernet frame.
2. It examines the source MAC address and updates the MAC address table.
3. It checks the destination MAC address.
4. It forwards, floods, or filters the frame based on what it knows.

This sequence is the practical model you should remember for exam questions. The source address teaches the switch, and the destination address drives the forwarding decision.

**Why loops matter**

Switches forward frames at Layer 2, and there is no built-in frame counter in the MAC-level forwarding process. If you create a loop, frames can keep circulating and multiply across the network. That is why loop prevention is important in switched networks.

Spanning Tree Protocol (STP) is used to stop potential switching loops before they cause repeated frame circulation. In exam scenarios, a loop usually points to excessive flooding, repeated traffic, or unstable switching behavior.

**Exam focus points**

- A switch learns from the **source MAC address**.
- A switch forwards based on the **destination MAC address**.
- A **MAC address table** stores MAC-to-port mappings.
- A **known unicast** goes to one port.
- An **unknown unicast** is flooded out of all other ports.
- A **broadcast** is sent to all ports in the broadcast domain.
- A loop can cause frames to circulate continuously.
- **STP** helps prevent switching loops.

**Quick memory aid**

Think of the switch as doing three jobs in order: learn, look up, and send. If it knows the destination, it sends the frame directly. If it does not know the destination, it floods the frame so the network can reveal where the device is.
