## **Course:** CompTIA Network+ (Exam N10-009)

**Switch management overview**

You manage a switch through either a **GUI** (graphical user interface) or a **CLI** (command-line interface). The web interface is typically easier and more visual, while the CLI is often preferred for direct, scriptable administration.

- **GUI** access is commonly described as a web interface.
- **CLI** access supports command-based management and troubleshooting.
- Equivalent functions may exist in both interfaces, so the choice often depends on preference and available tools.

**Web-based management access**

Switches can provide web-based authentication through built-in web servers. In the evidence, the switch uses both **HTTP** and **HTTPS** servers for user authentication. HTTPS adds encryption, so it is the safer choice when available.

> Use HTTPS instead of HTTP when you need encrypted management traffic.

- **HTTP** is unencrypted web access.
- **HTTPS** is encrypted web access.
- Web management is useful when you want a visual configuration workflow.

**Common management access methods**

Network device management access means logging in to, troubleshooting, and monitoring devices such as switches, routers, and firewalls. For switch administration, the most common access methods include secure remote protocols and web access.

| Method | Typical use | Security note |
|---|---|---|
| **SSH** | Secure remote CLI access | Preferred over Telnet |
| **Telnet** | Legacy remote CLI access | Not encrypted |
| **HTTP** | Web-based management | Not encrypted |
| **HTTPS** | Secure web-based management | Encrypted |

- **SSH** is the secure shell option for remote command-line access.
- **Telnet** is a legacy method and does not protect traffic.
- **HTTPS** is the secure web option for management access.

**Access control and authorization**

Management access can be restricted with **access profiles**. These profiles determine how users authenticate and authorize when they connect through management methods. A profile can limit access by source and by interface.

- Only users who pass both the active access profile and the management authentication checks gain access.
- Only one access profile can be active on a device at a time.
- Rules in a profile run in priority order from top to bottom.
- Rules can permit or deny access based on:
  - **Access method**
  - **Interface**
  - **Source IP address**

**Practical exam takeaways**

Switch management access is about both convenience and control. You choose the interface that fits the task, then restrict who can use it.

1. Identify whether the task calls for **GUI** or **CLI** access.
2. Prefer **HTTPS** or **SSH** when security matters.
3. Avoid **Telnet** and plain **HTTP** when a secure option exists.
4. Use access controls to limit management access by source and interface.

**What to remember**

- Switches are managed through **GUI** or **CLI**.
- Web interfaces commonly use **HTTP** and **HTTPS**.
- **SSH** and **HTTPS** are the secure management choices.
- Access profiles can restrict management by method, interface, and source IP address.
