## **Course:** CompTIA Network+ (Exam N10-009)

**Core concepts**

A **collision domain** is the part of a network where devices share the same communication medium, so simultaneous transmissions can interfere with each other. When two devices send at the same time on a shared segment, the frames collide and must be resent, which reduces efficiency. This is most associated with half-duplex communication and older shared-media designs.

A **broadcast domain** is the set of devices that receive a Layer 2 broadcast. Broadcasts stay within the same broadcast domain, and routers do not forward them between domains. This makes broadcast domains important for controlling how far broadcast traffic spreads.

- Hubs and repeaters create one large collision domain.
- Switches separate collision domains on a per-port basis.
- Routers separate broadcast domains.
- Older devices generally provide less traffic control and fewer ports than switches and routers.

**How common devices affect domains**

Device behavior is the easiest way to remember the difference between the two domain types. A hub acts like a multi-port repeater, so every connected device shares the same collision domain and the same broadcast domain. A switch improves efficiency by isolating collisions per port, but it still forwards broadcasts within the broadcast domain by default. A router stops broadcasts from crossing between networks, so it defines the boundary of a broadcast domain.

| Device | Collision domain behavior | Broadcast domain behavior |
|---|---|---|
| Hub | One shared collision domain | One shared broadcast domain |
| Switch | Separate collision domain per port | Broadcasts stay in the same broadcast domain by default |
| Router | Separate collision domain per interface | Separates broadcast domains |

**Practical exam takeaways**

Use the domain type to predict network behavior:

- If multiple devices share the same medium, collisions are possible.
- If a device forwards Layer 2 broadcasts, it remains inside the same broadcast domain.
- If a device does not forward broadcasts, it creates a broadcast boundary.
- If a device isolates ports from one another, it reduces collisions.

> A common exam trap is mixing up the two terms: collisions are about shared transmission access, while broadcasts are about how far a Layer 2 broadcast can travel.

**Quick memory aid**

- **Collision domain** = where frames can collide.
- **Broadcast domain** = where broadcasts can be heard.
- **Hub** = shared everything.
- **Switch** = fewer collisions.
- **Router** = broadcast boundary.

**Study focus**

When you see a topology question, identify the device first, then infer the domain count. If the diagram includes hubs, expect a larger shared collision domain. If it includes switches, expect collisions to be isolated per port. If it includes routers, expect broadcast traffic to stop at the router interface.
