## **Course:** CompTIA Network+ (Exam N10-009)

**Security monitoring and response tools**

These tools help you detect suspicious activity, block malicious traffic, collect security data, and investigate incidents. In practice, they often work together rather than replacing one another. A firewall filters traffic, while detection and response tools focus on identifying threats, stopping attacks, and giving you visibility into what is happening across the network and endpoints.

- **IDS** stands for Intrusion Detection System and focuses on identifying suspicious activity.
- **IPS** stands for Intrusion Prevention System and focuses on stopping or blocking malicious traffic.
- **SIEM** stands for Security Information and Event Management and focuses on collecting and analyzing security logs and events.
- **EDR** stands for Endpoint Detection and Response and focuses on endpoint activity and threat response.
- **NDR** stands for Network Detection and Response and focuses on network traffic monitoring and anomaly detection.

**IDS and IPS**

An Intrusion Detection System (IDS) watches traffic and alerts you when it sees behavior that looks malicious or unusual. It is a detection tool, so it helps you know something is wrong without necessarily stopping the traffic itself. An Intrusion Prevention System (IPS) adds an active blocking function, so it can prevent or interrupt suspicious traffic before it reaches its target.

> Use conservative wording on exams: IDS detects, while IPS detects and prevents.

- **IDS** is passive from a response standpoint.
- **IPS** is inline and can block traffic.
- Both are commonly discussed alongside firewalls, but they serve different roles.

**SIEM**

Security Information and Event Management (SIEM) platforms collect logs and events from many sources and help you correlate them for investigation and monitoring. This makes SIEM useful for central visibility across systems, users, and network activity. The main value is aggregation and analysis, not endpoint-level prevention.

- **Log collection** brings data into one place.
- **Correlation** helps you connect related events.
- **Monitoring** supports alerting and investigation.
- **Reporting** helps you review security activity over time.

**EDR**

Endpoint Detection and Response (EDR) focuses on endpoints such as laptops, desktops, and servers. It helps you detect suspicious behavior on those devices and respond to threats after they are identified. EDR is endpoint-centric, so it gives you visibility into what is happening on the host rather than only on the network.

- **Endpoint visibility** helps you investigate device activity.
- **Detection** looks for suspicious behavior on the host.
- **Response** supports action after a threat is found.

**NDR**

Network Detection and Response (NDR) focuses on network traffic rather than individual endpoints. It monitors traffic in real time and looks for anomalies that may indicate an attack or unauthorized access. NDR can learn normal traffic patterns and then flag changes that stand out from expected behavior.

- **Real-time monitoring** watches network activity as it happens.
- **Anomaly detection** looks for unusual traffic patterns.
- **Behavioral analysis** helps identify suspicious changes.
- **Network focus** makes it useful when you need visibility across traffic flows.

**How these tools differ**

The easiest way to separate these tools is by where they operate and what they do with what they see. IDS and IPS focus on traffic inspection, SIEM focuses on log aggregation and analysis, EDR focuses on endpoints, and NDR focuses on network behavior.

| Tool | Primary focus | Main function |
|---|---|---|
| IDS | Network traffic | Detects suspicious activity |
| IPS | Network traffic | Detects and blocks suspicious activity |
| SIEM | Logs and events | Collects and correlates security data |
| EDR | Endpoints | Detects and responds on hosts |
| NDR | Network traffic | Monitors traffic and detects anomalies |

**Exam takeaways**

For Network+ questions, match the tool to the problem statement. If the question mentions logs from many systems, think SIEM. If it mentions a laptop or server being monitored for suspicious behavior, think EDR. If it mentions unusual network traffic patterns, think NDR. If it asks about detecting versus preventing malicious traffic, compare IDS and IPS carefully.

- **Detection** means identifying suspicious activity.
- **Prevention** means actively blocking or stopping traffic.
- **Correlation** points to SIEM.
- **Endpoint** points to EDR.
- **Network anomalies** point to NDR.
