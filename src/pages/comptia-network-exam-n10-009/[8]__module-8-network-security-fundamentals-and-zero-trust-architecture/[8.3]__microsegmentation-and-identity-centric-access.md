## **Course:** CompTIA Network+ (Exam N10-009)

**Microsegmentation overview**

Microsegmentation divides a network into smaller, isolated segments so you can control traffic between workloads more precisely. It is commonly described as creating secure zones across cloud and data center environments, with policies that limit east-west traffic and reduce the attack surface.

This approach supports a zero trust security model by assuming traffic is not trusted by default. Instead of allowing broad access across a network, you explicitly allow only the traffic that is needed.

- **Isolation** helps contain a breach to a smaller part of the environment.
- **East-west traffic** is the internal traffic between workloads, not traffic entering or leaving the network.
- **Granular policy** lets you apply controls at a finer level than traditional broad network segmentation.

> Microsegmentation is not a replacement for all other security controls; it is a containment and access-control strategy that works best as part of a broader zero trust design.

**Identity-centric access**

Identity-centric access shifts the control point from the network perimeter to the identity of the user, device, or application. Identity segmentation focuses on identities such as human accounts, service accounts, and privileged accounts, which are key pillars in zero trust.

This model is useful because many attacks use stolen credentials. By tying access decisions to identity and attributes, you reduce the chance that a valid login automatically grants broad network reach.

- **Identity** becomes the basis for access decisions.
- **Attributes** can include the role or type of account being used.
- **Privileged accounts** deserve tighter control because they can expose more of the environment if compromised.

**How the two ideas work together**

Microsegmentation and identity-centric access complement each other. Microsegmentation limits where traffic can go, while identity-centric access limits who or what can initiate that traffic. Together, they help prevent lateral movement after an initial compromise.

In practical terms, you use segmentation to create boundaries and identity controls to decide whether a request should cross those boundaries. This is especially important in distributed environments where users, devices, and applications are no longer protected by a single fixed perimeter.

- **Microsegmentation** controls movement between workloads.
- **Identity-centric access** controls access based on verified identity.
- **Zero trust** ties both ideas together with continuous verification.

**Security benefits**

These controls improve resilience by making it harder for attackers to move through the network after gaining access. They also support compliance goals by narrowing access to only what is required.

Common benefits include:

- Reduced attack surface
- Lower risk of lateral movement
- Better containment of breaches
- More precise access control
- Stronger protection for sensitive resources

**Implementation focus**

When you study this topic, focus on the security outcome rather than a specific vendor method. The key idea is that access is no longer based on a trusted internal network. Instead, you verify identity and restrict traffic to the smallest practical scope.

1. Identify the users, service accounts, devices, and applications that need access.
2. Define the smallest set of allowed communications between workloads.
3. Apply policies that deny everything else by default.
4. Review access regularly so the policy stays aligned with current needs.

**Exam takeaways**

For Network+ purposes, remember these distinctions:

- **Network segmentation** is the broader practice of dividing a network into parts.
- **Microsegmentation** is a more granular form that isolates workloads and limits east-west traffic.
- **Identity-centric access** uses identity and attributes to decide what is allowed.
- **Zero trust** is the guiding model that assumes no implicit trust based on location alone.

A good mental model is: segmentation limits paths, identity limits permission, and zero trust requires verification at every step.
