## **Course:** CompTIA Network+ (Exam N10-009)

**Zscaler and SASE overview**

Secure Access Service Edge (<abbr title="Secure Access Service Edge">SASE</abbr>) is a cloud-delivered approach that combines networking and security services. In the evidence, Zscaler positions Zero Trust SASE as a cloud and mobile architecture that builds on its Secure Service Edge (<abbr title="Secure Service Edge">SSE</abbr>) platform and a fresh approach to software-defined wide area networking (<abbr title="software-defined wide area networking">SD-WAN</abbr>).

The main idea is to move security and connectivity controls closer to users, devices, and applications instead of relying on a traditional perimeter. This matters in hybrid work environments where branches, mobile users, and cloud apps are the norm.

- **Cloud-delivered** architecture follows users and data across environments.
- **Zero trust** reduces implicit trust between locations and devices.
- **Hybrid work** drives the need for flexible, internet-centric connectivity.

**Why traditional SD-WAN creates risk**

Traditional SD-WAN helped reduce the cost of dedicated WAN circuits, but the evidence says it often extends trust between locations. That design can increase exposure when branches, users, and devices connect across the public internet.

Key concerns include:

- **Attack surface** grows when more IP addresses are exposed to the internet.
- **Ransomware risk** increases when attackers can move laterally across the network.
- **Complexity** rises when trust and policy enforcement are spread across locations.

> Traditional network-centric designs assume too much trust inside the environment. Zero trust SASE is presented as a response to that assumption.

**Zero Trust SASE and the security model**

Zero Trust SASE uses a zero trust approach to reduce business risk and network complexity. The evidence describes a zero trust network overlay that connects users and IoT/OT devices to applications through the Zscaler Zero Trust Exchange™, eliminating implicit trust.

This model is useful when you need consistent policy enforcement regardless of where the user or device connects from.

- **Zero trust** means you do not assume internal traffic is safe.
- **Consistent policies** apply across users, devices, and locations.
- **IoT/OT** adoption can happen without widening the attack surface.

**Core technologies in the platform**

The evidence identifies six core technologies in the Zscaler Zero Trust SASE platform. These components work together to replace or reduce dependence on network-based security controls.

| Technology | Purpose |
|---|---|
| **Zero trust SD-WAN** | Optimizes user experience and reduces complexity by selecting the best route for traffic to cloud apps, the internet, and the data center. |
| **Secure web gateway (SWG)** | Prevents unsecured internet traffic from entering the network and helps shield users from cyberthreats and malicious traffic. |
| **Cloud access security broker (CASB)** | Helps prevent data leaks, compliance issues, and malware infection by enforcing safe use of cloud apps and services. |
| **Firewall as a service (FWaaS)** | Replaces physical firewall appliances with cloud-based next-generation firewall capabilities and access controls. |
| **Zero trust network access (ZTNA)** | Provides least-privileged access for users, devices, workloads, and business partners. |
| **Zero Trust Exchange™** | Acts as the zero trust overlay that connects users and devices to applications. |

**How the platform reduces complexity**

Zscaler emphasizes that Zero Trust SASE lowers cost and operational overhead by removing the need for multiple branch appliances and simplifying route management. The platform is described as single-vendor and cloud-delivered, which helps standardize policy enforcement.

Practical benefits include:

- **Plug-and-play appliances** reduce branch deployment effort.
- **Cloud-delivered security** removes some on-premises security hardware.
- **Route table management** becomes simpler when traffic steering is centralized.
- **Least-privileged access** limits what each user or device can reach.

**SASE versus Zero Trust**

SASE and Zero Trust are related but not identical. The evidence frames both as architectural approaches that improve security and user experience, but Zero Trust is the access model while SASE is the broader framework that combines networking and security.

Use this distinction to keep exam answers precise:

- **SASE** = cloud-delivered networking plus security.
- **Zero Trust** = verify access continuously instead of assuming trust.
- **ZTNA** is one way Zero Trust is implemented inside a SASE design.

**Exam focus points**

For Network+ study, focus on the practical outcomes of the architecture rather than vendor branding.

- **Reduced attack surface** through cloud-delivered controls and less trust between sites.
- **Lower lateral movement** risk because access is more tightly controlled.
- **Better support for remote and hybrid work** because the perimeter follows the user.
- **Improved policy consistency** across branches, mobile users, and cloud services.
- **Safer IoT/OT adoption** because access can be extended without broad trust expansion.
