## **Course:** CompTIA Network+ (Exam N10-009)

**VPN security tradeoffs**

A VPN (Virtual Private Network) protects traffic by encrypting the connection between a remote user and the private network. The main security challenge is balancing access, performance, and control. If you open too much access, you increase risk. If you restrict too much, you can slow users down or block legitimate work.

- **Remote access** should be limited to what users need for their role.
- **Encryption** protects traffic in transit, but it does not remove the need for policy and authentication.
- **Performance** matters because remote users still need usable access to business resources.

**Split tunneling**

Split tunneling lets a remote user reach both the VPN and the public Internet at the same time. In this model, only traffic destined for internal subnets goes through the encrypted tunnel. Other traffic uses the user’s normal Internet connection.

This approach can reduce load on the company network and improve performance for remote work. It also creates a security tradeoff because not all traffic is protected by the VPN path.

- **Internal traffic** goes through the VPN tunnel.
- **Non-VPN traffic** uses the local Internet connection.
- **Network load** can be lower because the VPN does not carry every packet.

> Split tunneling improves efficiency, but you should evaluate whether the security impact fits your environment.

**Least privilege and access control**

Least privilege means users get only the access required to do their jobs. For VPN use, that means you should avoid broad network access when a user only needs a small set of resources. This reduces the chance that a compromised account can move freely through the network.

A practical VPN policy should define who can connect, what devices can connect, how much access they receive, and how long idle sessions remain active.

- **Access scope** should match job requirements.
- **Device rules** help control which endpoints may connect.
- **Idle timeout** limits exposure from abandoned sessions.

**Zero trust approach**

Zero trust treats users, devices, and applications as untrusted until they are verified. Instead of assuming that anything inside the network is safe, you check traffic and identity continuously. This is a stronger model for modern remote access because VPN connectivity alone does not guarantee trust.

Zero trust commonly relies on authentication, encryption, and additional verification controls. The goal is to ensure that only the right users reach the right data.

- **Verification** happens before access is granted.
- **Continuous checking** reduces trust in implicit network location.
- **Policy-based access** supports tighter control over sensitive resources.

**VPN security best practices**

A secure VPN deployment is not just about the tunnel. You also need endpoint protection, monitoring, and policy enforcement. Basic defenses help reduce the chance that a remote connection becomes an entry point for malware or unauthorized access.

1. Confirm that company devices have antivirus and firewall protection.
2. Use monitoring or detection services to identify malicious activity.
3. Define a VPN policy for access rights, device types, and session behavior.
4. Review whether split tunneling is appropriate for the workload.
5. Apply least privilege so users only reach the resources they need.

**Exam focus**

For Network+ purposes, focus on the relationship between VPN design and security posture. You should be able to explain why split tunneling can improve performance, why it can also increase exposure, and how least privilege and zero trust reduce risk.

- **Split tunneling** is a performance and security tradeoff.
- **Least privilege** limits unnecessary access.
- **Zero trust** verifies users and devices instead of trusting the internal network by default.
