## **Course:** CompTIA Network+ (Exam N10-009)

**Protocol security basics**

Some protocols are considered insecure because they expose data, credentials, or management traffic in ways that are easier to intercept or abuse. In exam scenarios, you often identify the insecure service first, then match it to the secure alternative that provides the same function with better protection.

A practical way to study this topic is to group protocols by use case:

- **Management** protocols should protect administrative access and credentials.
- **Web** protocols should encrypt browser traffic.
- **File transfer** protocols should avoid cleartext transfers.
- **Monitoring** protocols should use stronger authentication and encryption where available.

**Common insecure-to-secure replacements**

The most testable pairs are the ones that preserve the same job while improving security. For example, `Telnet` is insecure for remote command-line access, while `SSH` provides a secure alternative. `HTTP` is insecure for web traffic, while `HTTPS` adds encryption. `SNMPv1` and `SNMPv2` are weaker versions of the Simple Network Management Protocol, while `SNMPv3` adds stronger security features.

| Insecure protocol | Secure alternative | Typical use |
|---|---|---|
| `Telnet` | `SSH` | Remote administration |
| `HTTP` | `HTTPS` | Web browsing and web apps |
| `SNMPv1` / `SNMPv2` | `SNMPv3` | Network monitoring and management |
| `TFTP` | `FTP` or secure file-transfer options | File transfer use cases |

> In exam questions, choose the secure protocol that performs the same role as the insecure one. Do not confuse a secure protocol with a different service category.

**File transfer protocols**

`SFTP` and `FTPS` are secure file transfer options, but they are not the same protocol family. `SFTP` uses SSH-based protection, while `FTPS` adds security to FTP. When a question asks for a secure replacement, focus on whether the answer preserves the original function and improves confidentiality.

- `TFTP` is a simple transfer protocol and is not secure.
- `FTP` transfers files without the same level of protection as secure alternatives.
- `SFTP` is commonly used when you need secure file transfer over an encrypted channel.
- `FTPS` is another secure option for file transfer workflows.

**Management and monitoring considerations**

Network management protocols are especially important because they often carry sensitive operational data. `SNMPv3` is the secure choice among the SNMP versions listed in the evidence, while older versions are treated as less secure. For remote access, `SSH` is preferred over `Telnet` because it protects the session.

- Use `SSH` for secure remote shell access.
- Use `SNMPv3` when you need stronger protection for monitoring traffic.
- Avoid cleartext administrative protocols when a secure equivalent exists.

**How to answer exam questions**

When you see a question about insecure services on default ports, identify the service and then map it to the secure replacement.

1. Identify the protocol or service name in the prompt.
2. Decide whether the service is insecure because it lacks encryption or stronger authentication.
3. Select the secure alternative that provides the same function.
4. Ignore distractors that are secure technologies but not direct replacements.

**Quick recall**

- `Telnet` → `SSH`
- `HTTP` → `HTTPS`
- `SNMPv1` / `SNMPv2` → `SNMPv3`
- `TFTP` and `FTP` are not secure choices when a secure transfer method is required
- `SFTP` and `FTPS` are secure file transfer options

**Why this matters**

CompTIA Network+ expects you to recognize insecure protocols quickly and choose the safer replacement in real-world troubleshooting and hardening scenarios. This supports secure communication, safer administration, and better protection of network data.
