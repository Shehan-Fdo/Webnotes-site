## **Course:** CompTIA Network+ (Exam N10-009)

**Security principles overview**

Security principles give you a practical way to evaluate whether a network is protected well enough for real-world use. In Network+ terms, you focus on how security hardening, vulnerability reduction, and access control support a stable environment.

The exam emphasizes that security is not just about blocking attacks. It is also about preserving business operations, limiting exposure, and making sure the right people and systems can use the right resources at the right time.

- **Security hardening** reduces attack surface by removing unnecessary exposure.
- **Vulnerabilities** are weaknesses that attackers can exploit.
- **Threats** are potential causes of harm, while **exploits** are the methods used to take advantage of weaknesses.

**CIA triad**

The **CIA triad** is the core security model for this topic. The three pillars are **confidentiality**, **integrity**, and **availability**. Together, they provide a simple way to think about what security must protect.

<dl>
<dt>Confidentiality</dt>
<dd>Protects information from unauthorized access or disclosure.</dd>
<dt>Integrity</dt>
<dd>Protects information from unauthorized modification or deletion.</dd>
<dt>Availability</dt>
<dd>Ensures systems and data remain accessible when needed.</dd>
</dl>

The triad helps you identify what failed when an incident occurs. That makes it easier to trace root causes, assign responsibility, and choose a response that matches the problem.

- A misconfigured cloud database that exposes records is a **confidentiality** issue.
- Unauthorized deletion or tampering with records is an **integrity** issue.
- A service outage or blocked access is an **availability** issue.

> When you analyze a scenario, ask which CIA pillar is affected first. That usually points you toward the correct security control or incident response path.

**How the CIA triad guides security decisions**

The CIA triad is useful because it breaks security into separate concerns instead of treating everything as one broad problem. That separation helps you design controls that match the risk.

Security teams use the model to:
- find vulnerabilities more systematically
- evaluate whether controls protect the right asset
- confirm that a solution improves the overall security posture
- understand which part of the system needs attention after an incident

A strong security posture exists when all three pillars are addressed together. Protecting only one pillar can leave the environment exposed in other ways.

**Zero trust**

**Zero trust** is a security approach that assumes users, devices, and applications are not trusted by default. Instead of relying on the idea that internal network traffic is safe, you verify access continuously.

This model is especially important because many networks are open inside the perimeter. Once an attacker gets in, broad internal access can create major risk. Zero trust reduces that risk by requiring verification before access is granted.

Key ideas include:
- verify every user and device
- limit access to only what is needed
- inspect traffic and requests instead of assuming trust
- combine authentication and encryption with additional controls

**Least privilege**

**Least privilege** means giving users, devices, and applications only the access they need to do their job. It supports zero trust and reduces the damage that can happen if an account or system is compromised.

Use least privilege to:
- limit unnecessary permissions
- reduce lateral movement inside the network
- contain the impact of misused credentials
- make access decisions easier to audit

**Security monitoring and controlled exposure**

Security principles also include watching for attacker behavior and reducing unnecessary entry points. One practical example is the use of a **honeypot**, which is a decoy system designed to attract attackers so you can observe their methods.

A honeypot can help you:
- study attack patterns
- learn what tools or scripts attackers use
- detect malicious activity without exposing production systems
- improve defensive planning based on observed behavior

Port management also supports this goal. Every service uses a port, and an open port can become a vulnerability if it is not needed. Close unused ports and services to reduce exposure.

**Practical exam focus**

For Network+, expect scenario-based questions that ask you to match a security principle to a situation. The best answer usually depends on the business impact and the type of failure described.

- Choose **confidentiality** when the issue is unauthorized disclosure.
- Choose **integrity** when data is altered, deleted, or falsified.
- Choose **availability** when users cannot reach a system or service.
- Choose **zero trust** when the scenario requires continuous verification and reduced implicit trust.
- Choose **least privilege** when access should be minimized to lower risk.

**Quick review**

A secure network is not just one with strong perimeter defenses. It is one that protects data, preserves trust in that data, and keeps services available when users need them.

- **CIA triad** gives you the main security framework.
- **Zero trust** removes implicit trust from internal traffic.
- **Least privilege** limits access and reduces impact.
- **Honeypots** help you observe attacker behavior.
- **Hardening** and port control reduce unnecessary exposure.
