## **Course:** CompTIA Network+ (Exam N10-009)

**Network device roles**

Modern networks use specialized devices to move data between endpoints, network segments, and external networks. Each device has a distinct job, and understanding those jobs helps you choose the right device for forwarding, filtering, or translating traffic.

- **Switches** connect devices within a local network and forward frames based on destination information.
- **Routers** move traffic between IP subnets and direct packets toward different networks.
- **Firewalls** control traffic flow by applying security rules.
- **Gateways** act as entry points between networks and can translate between different networking environments.

A network device can be hardware or software, but the practical goal is the same: manage connectivity, traffic flow, and access. In exam scenarios, focus on what problem the device solves rather than on a single vendor implementation.

**Switches**

A switch is a multiport device that forwards traffic inside a LAN (Local Area Network). It uses MAC (Media Access Control) addresses to send frames to the correct destination instead of broadcasting everything to every port.

- A switch learns which MAC addresses are reachable on which ports.
- It reduces unnecessary traffic compared with a hub.
- It improves efficiency by forwarding frames only where they need to go.
- It operates at Layer 2 in the OSI (Open Systems Interconnection) model in the common case.

Layer 2 switches keep a MAC address table so they can map devices to ports. Some switches also operate at Layer 3 and can forward traffic using IP information, which makes them useful in more complex networks.

> A hub blindly broadcasts traffic to all ports, while a switch learns and forwards more selectively.

**Layer 2 and Layer 3 switching**

Layer 2 and Layer 3 switches differ mainly in what addressing information they use to make forwarding decisions.

| Type | Primary function | Common addressing | Typical use |
|---|---|---|---|
| Layer 2 switch | Forwards frames within the same network | MAC address | Local switching inside a LAN |
| Layer 3 switch | Forwards traffic between networks | IP address | Switching with routing capability |

Layer 2 switching is the standard choice for local device connectivity. Layer 3 switching adds routing behavior, which can help when you need traffic to move between subnets without relying entirely on a separate router.

**Routers**

A router connects different IP subnets and forwards packets between them. It makes decisions based on network-layer information, so it is the device you use when traffic must leave one subnet and reach another.

- A router connects separate networks.
- It forwards packets based on IP addressing.
- It is commonly used at the edge of a network and between internal subnets.
- It helps direct traffic toward the correct next hop.

Routers are central to internetworking because they move traffic beyond a single broadcast domain. In practical terms, if two devices are on different subnets, a router is usually involved in getting traffic between them.

**Firewalls**

A firewall filters traffic according to security policy. It is used to allow, deny, or inspect traffic based on rules that protect a network from unwanted access.

- A firewall enforces access control between trusted and untrusted networks.
- It can block traffic that does not match policy.
- It helps reduce exposure to threats.
- It is often placed at network boundaries, but it can also protect internal segments.

Firewalls are not just about blocking everything. They are about applying rules so that only approved traffic passes. In exam questions, look for language about security policy, permitted connections, or traffic inspection.

**Gateways**

A gateway is an entry point between networks. It can connect different systems and may translate between different networking languages or protocols.

- A gateway serves as a path from one network to another.
- It can act as a translator between different network environments.
- It is commonly associated with access to external networks, including the internet.
- It may be hardware or software.

Gateways are broader in scope than routers because they can handle translation, not just forwarding. When a question mentions connecting dissimilar systems or translating between environments, gateway is often the best fit.

**How to distinguish them quickly**

Use the traffic pattern and the addressing type to identify the device.

1. If the traffic stays inside one LAN and uses MAC addresses, think **switch**.
2. If the traffic must move between IP subnets, think **router**.
3. If the goal is to permit or deny traffic based on security rules, think **firewall**.
4. If the device provides an entry point or translation between networks, think **gateway**.

**Exam focus**

These devices often appear together in network diagrams, so the key is to match function to role.

- Switches handle local forwarding efficiently.
- Routers connect subnets and route packets.
- Firewalls enforce security policy.
- Gateways bridge or translate between networks.

When you see a question about reducing collisions or limiting unnecessary traffic, the answer usually points to a switch. When you see separate subnets, routing is the clue. When you see access control, think firewall. When you see translation or network entry, think gateway.
