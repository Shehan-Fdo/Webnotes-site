## **Course:** CompTIA Network+ (Exam N10-009)

**Overview**

Load balancers and proxy servers sit between clients and servers to improve how traffic moves through a network. They help with scale, availability, and control, but they do not all do the same job. For exam purposes, you should be able to identify what each device does and how it differs from the others.

A **load balancer** distributes incoming requests across multiple backend servers. A **proxy** acts as an intermediary, but the direction of that mediation matters. A forward proxy represents clients, while a reverse proxy represents servers.

- **Load balancing** spreads requests across a pool of servers.
- **Proxies** mediate communication instead of allowing direct client-to-server interaction.
- **Reverse proxies** are commonly associated with server-side traffic handling.
- **Forward proxies** are commonly associated with client-side traffic handling.

**Load Balancing Basics**

Load balancing improves application performance and availability by sharing client access requests across multiple servers. It also helps prevent overload on any one server. In practical terms, this means the system can handle more traffic and respond more consistently.

Load balancers often monitor server health and status before sending traffic. If a server cannot handle requests, the load balancer should avoid sending traffic to it. This makes health awareness part of the balancing function, not just request distribution.

- Distributes traffic across a server pool.
- Uses algorithms and network state to balance requests.
- Checks server health and status.
- Helps maximize throughput and reduce response time.

**Layer 4 and Layer 7 Load Balancing**

Load balancing can operate at different OSI (Open Systems Interconnection) model layers. The most common comparison is Layer 4 versus Layer 7.

Layer 4 load balancing works at the transport layer. It uses TCP (Transmission Control Protocol) and UDP (User Datagram Protocol) information such as IP addresses, ports, and connection details. Layer 7 load balancing works at the application layer and can inspect more detailed request data.

| Type | OSI Layer | Decision Basis | Typical Focus |
|---|---:|---|---|
| Layer 4 | Transport | IPs, TCP/UDP ports, connection data | Fast routing decisions |
| Layer 7 | Application | HTTP/HTTPS headers, content, URL type | Content-aware routing |

- **Layer 4** is simpler and relies on transport-level information.
- **Layer 7** can make decisions using application data.
- **HTTP/HTTPS** details are relevant to Layer 7 decisions.
- The OSI layer helps you predict what information the device can inspect.

> A Layer 7 load balancer is often described as a reverse proxy because it receives client requests and forwards them to backend servers.

**Forward Proxy vs. Reverse Proxy**

A proxy server acts as an intermediary, but the side it represents changes its role. A forward proxy sits in front of clients and makes requests on their behalf. A reverse proxy sits in front of servers and receives client requests before passing them to the appropriate backend system.

This distinction matters on the exam because the same general term, **proxy**, can describe different network placements and purposes. When you see a scenario, ask whether the device is protecting or representing the client side or the server side.

- **Forward proxy** serves client-side requests.
- **Reverse proxy** serves server-side requests.
- Reverse proxies can return resources as if they originated from the proxy itself.
- Forward proxies are useful when the client should not interact directly with the destination server.

**How to Recognize the Right Device**

When you read a scenario, focus on the traffic direction and the goal of the device. If the goal is to distribute incoming traffic across multiple servers, think load balancer. If the goal is to mediate client access to external resources, think forward proxy. If the goal is to sit in front of servers and handle inbound client requests, think reverse proxy.

1. Identify whether the traffic starts from a client or arrives at a server-facing entry point.
2. Determine whether the device is distributing load or acting as an intermediary.
3. Match the OSI layer clues to the likely function.
4. Use health checks, request content, or port-based routing details to narrow the answer.

**Exam Takeaways**

The exam often tests these concepts through short scenarios rather than direct definitions. You should be ready to distinguish between traffic distribution and traffic mediation, and between client-side and server-side proxy behavior.

- Load balancers improve availability and performance by spreading traffic.
- Layer 4 uses transport-layer details such as TCP and UDP.
- Layer 7 uses application-layer details such as headers and URLs.
- Forward proxies represent clients.
- Reverse proxies represent servers.
