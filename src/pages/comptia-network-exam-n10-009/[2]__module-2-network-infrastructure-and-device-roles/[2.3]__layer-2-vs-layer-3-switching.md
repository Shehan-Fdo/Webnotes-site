## **Course:** CompTIA Network+ (Exam N10-009)

**Layer 2 switching basics**

A Layer 2 switch operates at the <abbr title="Data Link layer">Data Link</abbr> layer of the <abbr title="Open Systems Interconnection">OSI</abbr> model. It forwards frames using <abbr title="Media Access Control">MAC</abbr> addresses, which identify devices on the same network segment.

A switch learns where devices are by building a MAC address table. It associates a device’s MAC address with a switch port, then uses that table to send frames to the correct destination instead of flooding the whole network.

- Layer 2 switching stays within the local network.
- The switch uses MAC addresses, not IP addresses, for forwarding decisions.
- A MAC address table maps device addresses to ports.
- Learning may begin by flooding traffic so the switch can populate its table.

**Layer 3 switching basics**

A Layer 3 switch works at the <abbr title="Network layer">Network</abbr> layer. It uses <abbr title="Internet Protocol">IP</abbr> addresses and supports routing between different IP networks.

In practice, Layer 3 switching combines switching behavior with routing capability. This makes it useful when you need to move traffic between subnets or other logical network boundaries without relying only on a separate router.

- Layer 3 switching uses logical addressing.
- It forwards traffic between different IP networks.
- It is associated with routing functions in addition to switching.
- It is commonly compared with routers because both operate at Layer 3.

**Layer 2 vs Layer 3 comparison**

| Feature | Layer 2 Switch | Layer 3 Switch |
|---|---|---|
| OSI layer | Data Link | Network |
| Address type | MAC address | IP address |
| Main role | Forward frames within a LAN | Forward traffic between IP networks |
| Table used | MAC address table | Routing-related forwarding logic |
| Typical scope | Same network segment | Different networks or subnets |

Layer 2 is the better fit when you only need local frame forwarding inside a single network. Layer 3 is the better fit when traffic must cross network boundaries and follow IP-based routing decisions.

**How switching behavior affects traffic**

Switches reduce unnecessary traffic by sending frames only where they need to go. This is different from a hub, which floods traffic to all connected devices.

Because Layer 2 switches forward based on destination MAC address, they must know where a device lives on the network. That is why learning and table maintenance matter so much in normal operation.

> Be careful with switching loops. At the MAC address level, frames can keep circulating if you create a loop, so loop prevention is important in switch design.

**Practical exam takeaways**

- Use **Layer 2** when the question focuses on MAC addresses, frames, or a single LAN.
- Use **Layer 3** when the question focuses on IP addresses, routing, or traffic between networks.
- Remember that Layer 2 is tied to the Data Link layer and Layer 3 is tied to the Network layer.
- If a scenario mentions a switch learning device locations, think of a MAC address table.
