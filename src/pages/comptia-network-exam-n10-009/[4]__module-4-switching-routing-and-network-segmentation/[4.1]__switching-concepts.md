## **Course:** CompTIA Network+ (Exam N10-009)

**Switching fundamentals**

An Ethernet switch forwards traffic based on the **destination MAC address** in the frame. It does not make forwarding decisions using IP addresses for basic Layer 2 switching. The switch also learns where devices live by watching inbound frames and recording the **source MAC address** on the port where the frame arrived.

- A switch forwards **frames** in a LAN.
- It learns a device’s location from the **source MAC** in incoming traffic.
- The switch keeps an active list of known devices in a **MAC address table**.
- The MAC address table is also called the **forwarding database (FDB)**.

**MAC learning and forwarding**

When a frame enters a switch, the switch examines the source MAC address and associates it with the incoming physical interface. That learned entry helps the switch make a later forwarding decision when traffic is sent to that device.

If the destination MAC address is already in the table, the switch forwards the frame out the specific port associated with that entry. This is the normal case for known unicast traffic.

- **MAC learning** happens from inbound frames.
- **Forwarding** uses the destination MAC and the table entry.
- Known destinations go to one specific port.
- The switch updates its table as traffic arrives.

**Unknown destinations and flooding**

If the switch does not yet know the destination MAC address, it floods the frame out all ports except the one it arrived on. This behavior helps the switch discover where the destination device is located when the address is not yet in the table.

> A switch does not send unknown unicast traffic to a default gateway just because the destination is missing from the table.

- Unknown destination MACs trigger **flooding**.
- Flooding sends the frame to all ports except the ingress port.
- The switch still learns from the source MAC while handling the frame.
- Once the destination is learned, later frames can be forwarded directly.

**Aging and table maintenance**

MAC table entries do not stay forever. If an entry is not used for a period of time, the switch ages it out and removes it. This keeps the table current and prevents stale forwarding information from lingering.

- Unused entries are **aged out**.
- Aging helps keep the table accurate.
- Fresh traffic refreshes learned entries.

**Practical switching behavior**

Switches are designed to make fast forwarding decisions at Layer 2. In larger networks, they may also participate in loop prevention with **Spanning Tree Protocol (STP)**, which helps stop switching loops before traffic circulates endlessly.

- Switches forward based on MAC address information.
- Broadcast and multicast traffic can reach multiple devices.
- Loop prevention matters because Layer 2 frames can circulate repeatedly if the network is misconfigured.
- STP helps prevent those loops.

**Quick reference**

| Situation | Switch action |
|---|---|
| Source MAC seen on a port | Learns the MAC-to-port mapping |
| Destination MAC is known | Forwards out the matching port |
| Destination MAC is unknown | Floods out all ports except the incoming port |
| Entry has not been used | Ages out and is removed |

**Key terms**

<dl>
<dt>MAC address table</dt>
<dd>The switch’s list of learned MAC-to-port mappings.</dd>
<dt>FDB</dt>
<dd>Forwarding database; another name for the MAC address table.</dd>
<dt>Flooding</dt>
<dd>Sending a frame out all ports except the one it arrived on.</dd>
