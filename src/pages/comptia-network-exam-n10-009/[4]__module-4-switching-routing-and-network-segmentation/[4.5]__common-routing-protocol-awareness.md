## **Course:** CompTIA Network+ (Exam N10-009)

**Routing protocol fundamentals**

Routing protocols help routers exchange route information so they can forward traffic along the best available path. They automatically share subnet and topology information, which lets the network adapt when links fail or routes change.

A router uses its routing table to decide where to send packets next. In dynamic routing, that table is updated by protocol-driven communication between routers rather than by manual entry alone.

- **Dynamic routing** reduces manual route maintenance by letting routers learn routes from each other.
- **Re-convergence** happens when routers rebuild routes after an outage or link failure.
- **Best path** selection depends on the protocol’s metric or decision method.

**Common routing protocol families**

Routing protocols are commonly grouped by how they evaluate paths. The main families you should recognize are distance vector, link-state, and hybrid.

- **Distance vector** protocols choose routes mainly by hop count.
- **Link-state** protocols use a broader view of network topology.
- **Hybrid** protocols combine ideas from both approaches.

| Family | General idea | Example from evidence |
|---|---|---|
| Distance vector | Selects the path with the fewest hops | RIP |
| Link-state | Uses detailed topology awareness | OSPF |
| Hybrid | Blends distance-vector and link-state ideas | EIGRP |

**Protocol awareness for the exam**

You do not need to memorize every internal detail, but you should recognize how the major protocols are typically categorized. That helps you match a protocol to its routing style and likely behavior during path selection.

- **RIP** is a distance vector protocol and uses hop count as a simple metric.
- **OSPF** is a link-state protocol and is associated with topology-based path selection.
- **EIGRP** is commonly described as a hybrid protocol.
- **BGP** is a major routing protocol name to recognize in enterprise and internet-facing contexts.
- **IS-IS** is another protocol name you should be aware of alongside the others.

> Dynamic routing protocols can adapt when a link goes down, but the exact route choice depends on the protocol’s metric and design.

**How routers use routing information**

When a packet arrives, the router consults its routing table to determine the next hop. Dynamic routing protocols help keep that table current by distributing route information across the network.

This matters because the “best” route is not always the shortest physical distance. The protocol decides based on its own rules, such as hop count or topology awareness.

- **Routing table** entries guide forwarding decisions.
- **Next hop** is the next router or device in the path.
- **Metric** is the value a protocol uses to compare routes.

**Practical study focus**

For Network+ awareness, focus on matching the protocol to its family and core behavior. That gives you enough context to answer questions about route selection, failover, and protocol classification.

1. Identify whether the protocol is distance vector, link-state, or hybrid.
2. Recall the primary metric or decision style associated with it.
3. Connect that behavior to how the network responds to changes or failures.

**Quick recall cues**

- **RIP**: hop count, distance vector.
- **OSPF**: link-state, topology-aware.
- **EIGRP**: hybrid.
- **BGP**: protocol name to recognize.
- **IS-IS**: protocol name to recognize.
