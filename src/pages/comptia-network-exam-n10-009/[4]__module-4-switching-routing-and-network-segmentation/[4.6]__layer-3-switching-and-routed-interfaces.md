## **Course:** CompTIA Network+ (Exam N10-009)

**Layer 3 switching overview**

A Layer 3 switch combines switching and routing functions. At Layer 2, a switch forwards frames within a VLAN using MAC addresses. At Layer 3, the device can also forward packets between different IP networks, which is the basis for inter-VLAN routing.

Layer 3 devices operate at the network layer of the OSI model. The network layer handles packet forwarding and routing through intermediate devices. In practical terms, this means a Layer 3 switch can move traffic between subnets without relying on a separate router for every internal path.

- **VLAN**s separate traffic into distinct broadcast domains.
- A Layer 2 switch stays within one local network and does not read the IP header for routing decisions.
- A Layer 3 switch can route between IP networks at line rate, which supports faster internal forwarding.
- Inter-VLAN routing lets hosts in different VLANs communicate when routing is enabled.

**How Layer 3 switching differs from Layer 2 switching**

A Layer 2 switch forwards traffic based on destination MAC address. It does not make routing decisions between IP networks. That limitation matters because devices in different VLANs are in different logical networks, even if they connect to the same physical switch.

A Layer 3 switch adds IP routing capability. It can examine Layer 3 information and choose the next hop for traffic that must leave the local subnet. This makes it useful when you need both segmentation and internal routing in the same device.

| Capability | Layer 2 switch | Layer 3 switch |
|---|---|---|
| Forwarding basis | MAC address | MAC address and IP routing |
| VLAN awareness | Yes | Yes |
| Inter-VLAN routing | No | Yes |
| Network-layer forwarding | No | Yes |

**Routed interfaces**

A routed interface is a switch interface configured to operate like a Layer 3 interface instead of a Layer 2 switchport. It participates in routing, so it belongs to an IP network rather than a VLAN access domain.

Use routed interfaces when you want the switch to route traffic directly between networks. This is different from a normal switchport, which belongs to a VLAN and forwards frames within that VLAN only.

- A routed interface is associated with an IP subnet.
- It does not function as a typical access port for end devices.
- It supports routing between connected networks on the switch.

**Inter-VLAN routing with Layer 3 switches**

Inter-VLAN routing is the process of allowing traffic to move between VLANs. Cisco Press describes Layer 3 switches as a method for inter-VLAN routing, alongside router-on-a-stick designs. The Layer 3 switch approach keeps routing inside the switch, which simplifies internal traffic flow when multiple VLANs need to communicate.

This design is practical when you want to segment users, servers, or departments into separate VLANs but still allow controlled communication between them. The switch performs the routing function instead of sending all traffic to an external router.

- Separate departments can stay in different VLANs.
- The switch routes traffic only when communication must cross VLAN boundaries.
- The design supports internal segmentation without removing connectivity.

**Operational considerations**

A switch does not automatically become a router just because it supports VLANs. Routing must be enabled or configured through Layer 3 features. If traffic stays within one VLAN, the switch forwards it at Layer 2. If traffic must cross VLANs or subnets, the Layer 3 function handles the forwarding decision.

> A Layer 3 switch still depends on correct VLAN and IP configuration. If the VLANs or routed interfaces are misconfigured, inter-VLAN communication fails even though the hardware supports routing.

**Practical takeaways**

Use a Layer 3 switch when you need both high-speed switching and internal routing in the same device. Use routed interfaces when you want a port to participate in IP routing rather than act as a Layer 2 access port. Keep in mind that Layer 2 switching and Layer 3 routing solve different problems: one moves frames inside a VLAN, and the other moves packets between networks.
