## **Course:** CompTIA Network+ (Exam N10-009)

**Routing overview**

Routing is the process of selecting a path for data packets to travel from a source network to a destination network. It happens at <abbr title="Layer 3">Layer 3</abbr>, also called the network layer, and is performed by devices such as routers and other Layer 3 devices.

A router uses routing information to decide the next hop for a packet when the destination is not on the local network. The goal is to move traffic efficiently while making practical use of available network resources.

- **Routing** connects different networks, not just devices on the same local segment.
- A router makes forwarding decisions based on its routing table.
- The best path is chosen from the routes the device knows about.

**Routing table basics**

A routing table is the router’s map for forwarding traffic. It contains entries that tell the device where to send packets next, including directly connected networks and routes learned in other ways.

Every router builds a Routing Information Base (RIB) from multiple sources. In practical terms, the table helps the router answer a simple question: “Where should this packet go next?”

- **Connected routes** appear automatically when an interface has an IP address and is up.
- **Static routes** are entered manually by an administrator.
- **Dynamic routes** are learned from other routers through a routing protocol.

**Route types**

Connected, static, and dynamic routes are the core route types you need to recognize.

| Route type | How it appears | Typical use |
|---|---|---|
| Connected | Installed automatically when an interface is configured and active | Reachable local subnet |
| Static | Added manually | Predictable forwarding path |
| Dynamic | Learned from routing protocols | Networks that change over time |

Connected routes require no extra configuration beyond assigning the interface address and bringing the interface up. Static routes give you direct control over the path. Dynamic routes reduce manual work because routers exchange route information automatically.

**Static routing**

Static routing uses manually configured routes. It is useful when you want a fixed path and do not need the router to learn alternatives from other devices.

Static routes are practical in small or stable networks, but they require manual updates if the topology changes. That makes them simple, but less flexible than dynamic routing.

- **Static routing** is manually configured.
- It provides predictable forwarding behavior.
- It can become harder to maintain as the network grows.

**Default routing**

A default route is the route a device uses when it does not have a more specific match for the destination. It acts as a catch-all path for unknown destinations.

This is especially useful at the edge of a network, where a router forwards traffic toward an upstream network or gateway when it does not know a more specific route.

- **Default route** handles destinations not covered by other routes.
- It reduces the need to list every possible remote network.
- It is commonly used as an exit path from a local network.

**Dynamic routing**

Dynamic routing uses routing protocols so routers can learn routes from other routers automatically. This is useful when routes change often or when manual configuration would be too time-consuming.

Dynamic routing improves adaptability, but it also adds protocol behavior that you must understand and manage. In practice, it is chosen when scalability and automatic updates matter more than fixed, manual control.

- **Dynamic routing** learns routes from other routers.
- It adapts better to network changes.
- It is better suited to larger or changing environments.

**Forwarding decisions**

When a packet arrives, the router checks whether the destination is local. If it is not, the router consults the routing table and forwards the packet to the next hop.

This process is central to routing behavior: the router does not need to know the full path to the destination, only the next step. That next step is determined by the best available route in the table.

- The router examines the destination address.
- It compares the destination to entries in the routing table.
- It forwards the packet to the next hop that matches the selected route.

**Practical configuration mindset**

When you plan routing, start with the simplest route type that meets the need. Use connected routes for directly attached networks, static routes for fixed paths, and dynamic routing when the network needs automatic route exchange.

> If a destination is not local and no specific route exists, the device uses the default route if one is configured.

**Quick review**

- Routing moves packets between networks at Layer 3.
- Routers use a routing table to choose the next hop.
- Connected routes are learned automatically from active interfaces.
- Static routes are manually configured.
- Dynamic routes are learned from other routers.
- Default routes provide a fallback path for unknown destinations.
