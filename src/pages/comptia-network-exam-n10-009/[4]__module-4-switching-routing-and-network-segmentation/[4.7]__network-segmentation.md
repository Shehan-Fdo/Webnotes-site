## **Course:** CompTIA Network+ (Exam N10-009)

**Network segmentation overview**

Network segmentation divides a large network into smaller, more manageable parts. This reduces crowding, helps traffic move more efficiently, and makes it easier to control how devices communicate. Segmentation also improves security by keeping sensitive systems or data in separate areas.

- **Performance** improves when you split a busy network into smaller segments.
- **Security** improves because devices in one segment do not automatically reach every other segment.
- **Manageability** improves because smaller groups are easier to monitor and troubleshoot.

**Broadcast domains and collision domains**

A **broadcast domain** is the set of devices that receive a broadcast frame. Broadcast traffic can spread widely and slow down a busy network, so limiting broadcast reach is a common design goal. A **collision domain** is the area where devices share the same communication medium and can interfere with each other if they transmit at the same time.

- **Broadcast domains** are controlled with routers and VLANs.
- **Collision domains** are reduced by switches.
- Hubs and repeaters create larger collision domains, which is less efficient for modern networks.
- Routers separate broadcast domains, which helps contain broadcast traffic.

> A single broadcast can reach many devices, so uncontrolled broadcast traffic can reduce network efficiency.

**How segmentation is implemented**

You can segment a network physically or logically. Physical segmentation uses separate devices or separate network paths. Logical segmentation uses technologies such as VLANs to separate traffic without requiring fully separate hardware for every group.

- **Physical segmentation** uses separate switches or other dedicated devices.
- **Logical segmentation** uses VLANs to separate groups on shared infrastructure.
- **802.1Q** VLAN trunking carries VLAN traffic between devices when needed.
- Segmentation can support separate user groups, applications, or sensitive data areas.

**Device behavior and traffic control**

Switches and routers play different roles in segmentation. Switches forward traffic more efficiently than hubs and help create separate collision domains. Routers stop broadcasts from crossing between networks, which makes them useful for separating larger network areas.

| Device | Segmentation effect | Practical result |
|---|---|---|
| Hub / repeater | One large collision domain | More contention and less efficient traffic flow |
| Switch | Separate collision domains | Better traffic handling |
| Router | Separate broadcast domains | Broadcasts stay contained |

**Design and troubleshooting considerations**

Segmentation can improve throughput and security, but it also adds design decisions. A simple physical design is easy to understand, but it can be harder to scale when many separate networks are needed. Logical segmentation is often more flexible for larger environments.

- Use segmentation when you need to limit broadcast impact.
- Use segmentation when different groups should not communicate freely.
- Use segmentation when compliance or data separation matters.
- Check VLAN assignments and switching behavior when troubleshooting connectivity issues.

**Key takeaways**

Network segmentation is about dividing a network into smaller parts to improve performance, reduce unnecessary traffic, and strengthen security. Broadcast domains and collision domains explain why segmentation matters, and switches, routers, and VLANs are the main tools used to implement it.
