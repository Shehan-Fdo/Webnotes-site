## **Course:** CompTIA Network+ (Exam N10-009)

**IPv4 addressing fundamentals**

An IPv4 address is a 32-bit address written as four numbers separated by periods, such as `192.168.1.165`. Devices use it as a unique identifier on an IP network. To communicate correctly, each device needs its own address.

The **subnet mask** works with the IP address to determine which part identifies the network and which part identifies the host. This is how a device knows whether a destination is local or must be sent to another network.

- IPv4 addressing is the basis for host identification on a network.
- The subnet mask divides the address into network and host portions.
- TCP/IP configuration on Windows requires an IP address, a subnet mask, and a default gateway.
- Addressing and subnetting let networks of different sizes connect and operate together.

**Public, private, and special-use addresses**

IPv4 addressing includes public and private address ranges, along with special-purpose addresses used for specific functions. Public addresses are routable across the Internet, while private addresses are used inside local networks.

Common special-use address types include link-local and loopback addresses. These are part of the broader IPv4 addressing model and are useful for local communication and testing.

- **Public IP addresses** are used for Internet-facing communication.
- **Private IP addresses** are used inside internal networks.
- **Link-local addresses** support local network communication.
- **Loopback addresses** support local host testing.

**Subnetting and network division**

Subnetting is the process of taking a larger block of IP addresses and dividing it into smaller chunks. You use it to organize networks, support different departments or VLANs, and allocate address space more efficiently.

This matters because a network rarely needs one flat address block. Subnetting helps you separate traffic logically and manage address space in a practical way.

- Subnetting divides a larger address block into smaller networks.
- Smaller subnets can map to different parts of an organization.
- Subnetting supports efficient address allocation.
- Subnetting is a core skill for IPv4 planning and troubleshooting.

**Route selection and the default gateway**

When a device sends traffic, it checks whether the destination matches a local network. If it does not, the device forwards the packet to the **default gateway**, which is the next hop used for off-subnet traffic.

Routers use routing tables to choose the best next hop. In general, route lookup prefers the **longest prefix match**, meaning the most specific route wins. If no specific route matches, the router can use a default route, also called the **gateway of last resort**.

> A default route in IPv4 is `0.0.0.0/0`. It matches traffic only when no more specific route exists.

- The default gateway handles traffic destined for other networks.
- Routing tables determine the best next hop.
- Longest prefix match selects the most specific route.
- The default route is the fallback when no specific route matches.
- A default route is especially useful when you do not want to learn every remote network.

**How route selection works in practice**

You can think of route selection as a lookup process. The router compares the destination address against entries in the routing table and forwards the packet out the interface associated with the best match.

If the destination is not in the table, the router uses the default route if one exists. Without a default route, the router needs a specific route for every destination it must reach.

1. Check the destination IP address.
2. Compare it against the routing table.
3. Choose the most specific matching route.
4. If no specific route matches, use the default route.
5. Forward the packet to the next hop or out the selected interface.

**Troubleshooting focus**

When IPv4 communication fails, verify the basic configuration first. An incorrect IP address, subnet mask, or default gateway can prevent local communication or stop traffic from reaching other networks.

Routing problems also require checking the routing table on the router. If the table does not contain the needed route, or if the default route is missing, traffic may not leave the network correctly.

- Verify the IP address is valid for the network.
- Confirm the subnet mask matches the intended subnet.
- Check that the default gateway points to the correct next hop.
- Review the routing table when traffic is not reaching remote networks.
- Look for a missing or incorrect default route when off-subnet traffic fails.

**Key takeaways**

IPv4 addressing gives each device a unique identity, while the subnet mask defines the local network boundary. Subnetting divides address space into manageable pieces, and route selection determines where traffic goes when the destination is not local.

The default gateway and default route are central to reaching other networks. Together, they provide the fallback path that keeps traffic moving when no more specific route exists.
