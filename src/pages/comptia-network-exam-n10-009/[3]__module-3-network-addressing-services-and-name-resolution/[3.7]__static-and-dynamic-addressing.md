## **Course:** CompTIA Network+ (Exam N10-009)

**Addressing overview**

An IP (Internet Protocol) address identifies a device on a network so other systems can locate it. In practice, a host also needs related settings such as a subnet mask and a default gateway. Those values can be entered manually or assigned automatically.

Static and dynamic addressing are two ways to provide that configuration. Static addressing keeps the settings fixed until you change them. Dynamic addressing lets the network assign the settings automatically when the device connects.

- **IP address**: the device identifier used for network communication.
- **Subnet mask**: defines which part of the address is the network portion.
- **Default gateway**: the next hop used to reach other networks.

**Static addressing**

With static addressing, you configure the network settings directly on the device. This approach is straightforward and gives you predictable values that do not change unless you edit them.

Use static addressing when you want a device to keep the same network identity over time. That makes it easier to reference the device in documentation and to reach it consistently.

- **Manual configuration**: you enter the address settings yourself.
- **Stable assignment**: the address remains constant until changed.
- **Predictable access**: the device keeps the same network details.

> Static addressing is best used when consistency matters more than convenience.

**Dynamic addressing**

Dynamic addressing assigns network settings automatically when the device boots or joins the network. The evidence identifies **DHCP** (Dynamic Host Configuration Protocol) as the common method for this process, and notes that Linux environments may also use the term **BOOTP** in this context.

This method reduces manual work and helps avoid typing errors. It is useful when many devices need valid settings without individual configuration.

- **DHCP**: automatically provides IP settings to clients.
- **BOOTP**: a related term used in some Linux contexts.
- **Automatic assignment**: the network supplies the configuration.

**Static vs. dynamic comparison**

| Aspect | Static addressing | Dynamic addressing |
|---|---|---|
| Assignment method | Manually entered on the device | Automatically provided by the network |
| Address behavior | Stays the same until changed | Can be assigned when the device connects |
| Best fit | Devices that benefit from consistency | Devices that benefit from automation |

**How the configuration gets applied**

You can think of the process as a choice between typing the settings yourself or letting the network provide them. In both cases, the device still needs valid IP settings before it can communicate normally.

1. Decide whether the device needs a fixed or automatically assigned configuration.
2. Provide the IP address, subnet mask, and default gateway manually, or allow DHCP to supply them.
3. Verify that the device can communicate on the network with the assigned settings.

**Practical selection guidance**

Choose static addressing when you need a device to keep the same settings over time. Choose dynamic addressing when you want simpler administration and automatic assignment for clients.

- **Static** is practical for consistency.
- **Dynamic** is practical for convenience.
- **DHCP** is the standard automatic assignment method referenced in the evidence.

**Key takeaway**

Static addressing gives you direct control and a fixed configuration. Dynamic addressing shifts the work to the network and is better suited to environments where automatic setup is preferred.
