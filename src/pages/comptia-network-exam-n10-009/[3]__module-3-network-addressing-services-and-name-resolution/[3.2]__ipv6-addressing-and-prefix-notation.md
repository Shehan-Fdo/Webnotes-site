## **Course:** CompTIA Network+ (Exam N10-009)

**IPv6 address fundamentals**

IPv6 (<abbr title="Internet Protocol version 6">IPv6</abbr>) is the successor to IPv4 and uses a 128-bit address space. That larger size gives IPv6 a vastly expanded pool of addresses for modern networks and devices.

An IPv6 address identifies a network interface on a host or other network node. It appears in packet headers so routers can make forwarding decisions based on the source and destination addresses.

- IPv6 addresses are 128 bits long.
- IPv4 addresses are 32 bits long.
- The larger address space reduces pressure on address exhaustion.

**Address structure and notation**

An IPv6 address is written as eight groups of four hexadecimal digits, separated by colons. On the exam, you should be able to recognize the standard format quickly and count the groups correctly.

A typical full-form address looks like this:

```text
2001:0db8:0000:0000:0000:ff00:0042:8329
```

- Each group contains hexadecimal values.
- Colons separate the eight groups.
- A valid full address has eight hextets.

**Prefix notation**

IPv6 commonly uses prefix notation to show the network portion of an address. The prefix length follows a slash, such as `/64`, and tells you how many leading bits belong to the network.

This notation is important for planning and assignment because IPv6 networks are typically allocated as short prefixes, leaving room for many `/64` subnets inside a site.

- `2001:db8:abcd::/48` shows a larger site allocation.
- `2001:db8:abcd:1::/64` shows one subnet within that allocation.
- The prefix length is part of the address representation, not a separate setting.

> IPv6 is not managed like IPv4. You generally assign a short prefix to a site so it can create many `/64` subnets without worrying about exhausting the address space.

**Address types**

IPv6 addresses are grouped by how they deliver traffic. The main types are unicast, anycast, and multicast.

- **Unicast** identifies a single network interface.
- **Anycast** is assigned to multiple interfaces, but traffic goes to the nearest one based on routing.
- **Multicast** sends traffic to all members of a multicast group.

These categories help you understand how IPv6 delivers packets and how different hosts receive traffic.

**Global unicast and special allocations**

The Internet Assigned Numbers Authority (<abbr title="Internet Assigned Numbers Authority">IANA</abbr>) allocates only a small portion of the IPv6 space for global unicast use. In practice, globally routed IPv6 prefixes commonly begin with hexadecimal `2` or `3`, because the leading bits are allocated from the `001` range.

Some address ranges are reserved for special purposes, while much of the remaining space is held for future use.

- Global unicast addresses are used on the public Internet.
- Special-purpose ranges exist for non-general use.
- The IPv6 space is large enough that exhaustion is not the same concern as in IPv4.

**Reading and validating addresses**

When you evaluate an IPv6 address, focus on the format first. A valid address should match the expected hextet structure and use colons correctly.

1. Check that the address uses hexadecimal characters only.
2. Confirm the address is divided into eight hextets in full notation.
3. Verify the prefix length if one is present.
4. Identify whether the address is unicast, anycast, or multicast based on context.

**Practical exam takeaways**

IPv6 questions often test recognition more than deep calculation. You should be comfortable reading the address format, identifying the prefix length, and distinguishing address types.

- IPv6 uses 128-bit addressing.
- Prefix notation shows the network boundary.
- Global unicast addresses are the common Internet-routable type.
- Unicast, anycast, and multicast describe delivery behavior.
