## **Course:** CompTIA Network+ (Exam N10-009)

**NAT and PAT overview**

<abbr title="Network Address Translation">NAT</abbr> lets multiple devices on a private network access external networks through one public IPv4 address. This is common in home routers, enterprises, and even some ISP deployments. The main value is practical: you conserve public IPv4 space and keep internal addressing hidden from outside networks.

In common usage, NAT is often used as an umbrella term for related translation methods. Strictly speaking, NAT changes IP addresses, while <abbr title="Port Address Translation">PAT</abbr> also changes transport-layer port numbers to improve scale. PAT is also called <abbr title="Network Address and Port Translation">NAPT</abbr>.

- NAT maps private addresses to a public address and back again.
- PAT adds port translation so many sessions can share the same public IP.
- The overall goal is to let many internal devices communicate externally without needing unique public IPv4 addresses.

**Why NAT matters**

IPv4 has a limited address space, so address sharing is necessary in many environments. NAT helps reduce public IPv4 exhaustion by allowing many private hosts to appear as one public host. It also provides a basic layer of obscurity by hiding internal addresses from external systems.

> NAT is not a replacement for security controls, but it does reduce direct exposure of internal addressing.

**How translation works**

A device sends traffic to the NAT-enabled router or gateway. The router replaces the private source IP with its public IP and, in PAT, assigns a unique port number. The router stores that mapping so return traffic can be sent back to the correct internal device.

1. A private host sends a request to an external destination.
2. The NAT device rewrites the source address, and PAT may rewrite the source port.
3. The NAT device records the mapping in its translation table.
4. Return traffic arrives at the public address and is translated back to the original private host.

```text
Private host 192.168.1.10:51544 -> NAT/PAT device -> Public IP 203.0.113.5:40001
Return traffic -> NAT/PAT device -> 192.168.1.10:51544
```

**NAT versus PAT**

| Feature | NAT | PAT |
|---|---|---|
| Address translation | Yes | Yes |
| Port translation | No | Yes |
| Scale | Lower than PAT | Higher than NAT |
| Common usage | Broad umbrella term in practice | Common for many-to-one internet access |

PAT is the more scalable approach because it can distinguish many simultaneous sessions using port numbers in addition to IP addresses. In practice, many references use NAT to describe both behaviors, so read the context carefully.

**Common use cases**

- Home Wi-Fi routers use NAT/PAT so many devices can share one ISP-assigned public address.
- Enterprises use NAT/PAT to conserve public IPv4 space across many internal hosts.
- Some ISPs use carrier-grade NAT (<abbr title="Carrier-Grade Network Address Translation">CGNAT</abbr>) to share public IPv4 addresses among customers.

**Key takeaways**

NAT and PAT solve the same practical problem: they let private networks communicate with public networks while conserving IPv4 addresses. NAT focuses on address translation, while PAT extends the idea by translating ports for better scaling. For Network+, remember the relationship between private addressing, public address sharing, and translation tables.
