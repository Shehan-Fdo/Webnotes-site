## **Course:** CompTIA Network+ (Exam N10-009)

**Routing table basics**

A router forwards traffic by checking the destination IP address against its routing table. If the router finds a matching route, it sends the packet out the appropriate interface. If it does not find a match, the packet is dropped unless a default route exists.

A routing table can include directly connected networks and manually configured routes. In troubleshooting, you compare the routing table to the expected network path so you can confirm that traffic leaves through the correct interface.

- **Routing table**: the lookup source a router uses to decide where to send packets.
- **Directly connected**: a network attached to the router interface without needing another hop.
- **Static route**: a route you add manually instead of learning it from a routing protocol.

**Static routes**

Static routes are useful when you want traffic to follow a specific path. They are commonly used when a route must avoid the default gateway or when you need a predictable path for security or cost reasons.

Static routing is configured manually, so it gives you control but also requires maintenance. If the network changes, you must update the route yourself.

- Use static routes when you need a fixed path.
- Use them for traffic that should not go through the default gateway.
- Update them manually if the topology changes.

**Default routes**

A default route tells the router where to send packets when no more specific route exists. It acts as the fallback path for unknown destinations.

If a router has no default route and no matching entry in its routing table, it discards the packet. This is why default routes are important for reaching networks outside the local routing table, including Internet-bound traffic.

> A default route is the last resort path. If the router cannot match a destination to a more specific route, it uses the default route or drops the packet if none exists.

**Common Cisco-style default route syntax**

On Cisco routers, a default static route uses `0.0.0.0` for both the destination and mask.

```bash
ip route 0.0.0.0 0.0.0.0
```

This form tells the router to send unmatched traffic toward the configured next hop or exit interface, depending on the full command used in the device configuration.

**How to think about route selection**

The router checks for the most specific match first. If a destination belongs to a directly connected subnet, the router uses that path. If no specific route matches, the default route becomes the fallback.

1. Check whether the destination is on a directly connected network.
2. Check whether a static route exists for that destination.
3. Use the default route if no more specific route matches.
4. Drop the packet if no route exists at all.

**Troubleshooting notes**

When traffic fails to reach an external destination, verify the routing table first. Look for the expected static route or the gateway of last resort. If the default route is missing, packets for unknown destinations will not leave the router.

- Confirm the destination network appears in the routing table.
- Confirm the default route exists when you need fallback forwarding.
- Compare the routing table with the intended network diagram.
- Remember that missing routes usually result in dropped packets, not automatic rerouting.

**Key takeaways**

Static routes give you manual control over packet forwarding. Default routes provide a catch-all path for destinations that do not have a more specific entry. Together, they help the router decide whether to forward traffic, send it to a known next hop, or discard it.
