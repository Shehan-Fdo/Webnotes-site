## **Course:** CompTIA Network+ (Exam N10-009)

**VPN fundamentals**

A <abbr title="Virtual Private Network">VPN</abbr> creates an encrypted tunnel over an untrusted network, usually the internet, so traffic stays confidential and intact in transit. It also helps hide the real source IP address by making the connection appear to come from another location.

- **Encrypted tunnel** protects data as it moves between endpoints.
- **Confidentiality** keeps traffic private from outside observers.
- **Integrity** helps ensure data is not altered in transit.
- **IP masking** can make a user appear to connect from a different location.

**Site-to-site VPN**

A site-to-site VPN connects two or more networks, such as a main office and a branch office. It is also called a router-to-router VPN because the tunnel is typically established between network devices rather than individual users.

This model is useful when you want users at one location to reach resources at another location without each user creating a separate VPN session. In the evidence, site-to-site VPNs are associated with IPsec and with secure communication between networks.

- **Network-to-network** connectivity is the defining trait.
- **IPsec** is used to create the encrypted tunnel in the provided evidence.
- **Branch access** lets one office reach another office’s shared resources.
- **Cost efficiency** can improve when internet pathways replace dedicated private links.

> Site-to-site VPNs are not described as multi-user client connections in the evidence; treat them as network-level links between sites.

**Remote access VPN**

A remote access VPN connects an individual user or device to a private network from outside the organization. It is the common choice when a person needs secure access to internal resources while working remotely.

Compared with site-to-site VPNs, remote access VPNs are user-centric rather than network-centric. The evidence frames them as one of the main VPN categories alongside site-to-site VPNs and notes that VPNs serve different organizational needs.

- **User-to-network** access is the key distinction.
- **Remote work** is a typical use case.
- **Secure access** supports internal resources from outside the office.
- **Different needs** drive the choice between remote access and site-to-site designs.

**Choosing the right VPN type**

You choose the VPN type based on what you are connecting and who needs access. If the goal is to link entire locations, use a site-to-site design. If the goal is to let a person connect securely from elsewhere, use remote access.

| Scenario | Better fit | Why |
|---|---|---|
| Main office to branch office | Site-to-site VPN | Connects two networks through an encrypted tunnel |
| Individual employee working remotely | Remote access VPN | Connects one user or device to the private network |
| Shared access between locations | Site-to-site VPN | Supports network-level resource sharing |
| Temporary off-site access for a person | Remote access VPN | Focuses on user authentication and access |

**Core exam concepts**

VPN questions often test the relationship between the tunnel, the endpoints, and the use case. Keep the distinction simple: site-to-site is for networks, remote access is for users.

- **Tunnel** is the protected path between endpoints.
- **Endpoint** can be a router, gateway, or user device depending on the VPN type.
- **Use case** determines the design you select.
- **Security goal** is to protect traffic over public infrastructure.

**Quick comparison**

| Feature | Site-to-site VPN | Remote access VPN |
|---|---|---|
| Primary connection | Network to network | User to network |
| Typical endpoint | Router or gateway | Client device |
| Main purpose | Link offices or sites | Support individual remote users |
| Access pattern | Shared across locations | Per user or per device |

**Practical takeaway**

When you see VPN in a troubleshooting or design question, identify whether the requirement is location-based or user-based. That single detail usually points you to the correct VPN type.
