## **Course:** CompTIA Network+ (Exam N10-009)

**Routing table fundamentals**

A router forwards traffic by consulting its routing table and choosing the best path for the destination. The table acts like a map of known networks, next hops, and outbound interfaces. When you troubleshoot routing, you start with the table because it determines where packets go.

Routing tables are not limited to routers. Workstations and servers also maintain routing tables, but routers rely on them most heavily for forwarding decisions.

- A routing table stores destination information and forwarding details.
- The router uses the table to decide the **next hop** and outbound interface.
- The best route is the one the router prefers for a given destination.

**Longest prefix match**

<abbr title="Longest Prefix Match">LPM</abbr> is the algorithm routers use to select the most specific matching route for a destination IP address. If multiple routes match, the route with the longest prefix length wins because it matches the most leading bits.

This is the core rule for IPv4 routing table lookups in the provided evidence. A route with `/32` is more specific than `/24`, which is more specific than `/16`, and so on.

- More matching bits means a more specific route.
- The most specific matching route is preferred.
- A host route such as `/32` can override broader network routes when it matches the exact destination.

**Example of prefix specificity**

| Route | Matches | Preference |
|---|---|---|
| `10.0.0.0/8` | `10.x.x.x` | Least specific in this set |
| `10.1.0.0/16` | `10.1.x.x` | More specific |
| `10.1.1.0/24` | `10.1.1.x` | More specific |
| `10.1.1.1/32` | `10.1.1.1` | Most specific |

For a packet destined to `10.1.1.1`, all four routes match, but the `/32` route wins because it has the longest prefix. This is the practical meaning of longest prefix match.

**Route precedence factors**

Routers may compare more than one route to reach the same destination. In the evidence, route precedence is described as a comparison of three attributes:

1. Route specificity
2. Administrative distance
3. Metric

If multiple routes are still identical after those comparisons, the router can load balance across the available paths.

- **Route specificity** is checked first in the provided route precedence model.
- **Administrative distance** helps determine which route source is more trusted.
- **Metric** helps compare path quality within a routing protocol.
- Equal routes can result in **load balancing**.

> If you see multiple routes to the same destination, do not assume the first one listed is chosen. Check specificity first, then administrative distance, then metric.

**Reading routing table output**

Routing table entries tell you how the route was learned and how traffic will be forwarded. The evidence notes that routing protocol codes identify the source of the route, and that the table contains enough information to predict the forwarding path.

When you read a routing table, focus on the destination, the prefix length, and the next hop or interface. Those fields tell you which route is preferred and where the packet goes next.

- Protocol codes show how the route was learned.
- The destination and prefix length show route specificity.
- The next hop or interface shows the forwarding decision.

**Practical troubleshooting approach**

When a packet does not take the path you expect, compare the candidate routes for the destination. Start with the most specific match, then check whether another route has a better administrative distance or metric.

1. Identify the destination IP address.
2. Find all matching routes in the routing table.
3. Choose the route with the longest prefix match.
4. Compare administrative distance if more than one route remains viable.
5. Compare metric if needed.
6. Confirm the selected next hop or outbound interface.

**Key takeaways**

Routing tables are the router’s decision source for forwarding traffic. Longest prefix match selects the most specific route, and route precedence can further refine the choice using administrative distance and metric. If all compared attributes are equal, the router may load balance across paths.
