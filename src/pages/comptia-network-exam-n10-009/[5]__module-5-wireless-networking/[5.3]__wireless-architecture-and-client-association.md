## **Course:** CompTIA Network+ (Exam N10-009)

**Wireless architecture overview**

Wireless networking is organized around how clients discover, join, and move between access points. In 802.11 networks, you should recognize the basic service set concepts that describe whether devices connect through an access point or directly to each other.

- **SSID** is the network name a client uses to identify a wireless network.
- **BSSID** identifies the specific access point radio a client associates with.
- **ESS** describes multiple access points working together as one extended wireless network.
- **IBSS** is an independent basic service set, also called an ad hoc connection, where devices connect directly without an access point.

A wireless design can include multiple network names, login screens, and antenna types. In practice, the architecture determines how clients find the network and what path they use to communicate.

**Client association and connection flow**

When a client joins a wireless network, it performs discovery and then associates with an access point. If security is enabled, the client also authenticates and may derive encryption keys before normal traffic begins.

- On an open SSID, the process is simpler because there is no encryption.
- On secured networks, association includes authentication and key handling.
- The access point becomes the client’s connection point for network access.

> A client does not just “connect” in one step; association, authentication, and security setup are part of the wireless join process.

**Roaming and transition**

Roaming is the movement of a wireless client from one access point to another while maintaining connectivity. The 802.11 standard refers to roaming as a **transition**, although that term is less commonly used in everyday conversation.

- Roaming includes moving the client association from one AP to another.
- It can also involve authentication changes and encryption key derivation.
- Older wireless designs could interrupt service during roaming.
- Modern fast roaming methods reduce or remove that interruption.

Roaming matters when you move through a building or campus and need the connection to follow you. The goal is to keep the session stable while the client changes its attachment point.

**Wireless network types**

Wireless networks are not limited to infrastructure mode. You should distinguish between access-point-based networks and direct peer-to-peer connections.

- **Infrastructure mode** uses an access point as the central connection point.
- **Ad hoc mode** uses IBSS so devices communicate directly.
- IoT devices may use ad hoc-style setup steps during initial configuration before joining the main network.

This distinction helps you understand why some devices connect through a managed wireless network while others communicate directly for setup or special-purpose use.

**Practical exam focus**

For Network+ purposes, focus on the relationship between network naming, access point identity, and client movement. These concepts explain how wireless users stay connected as they move.

1. Identify the wireless network name using the SSID.
2. Identify the specific access point using the BSSID.
3. Recognize whether the network is an ESS or an IBSS.
4. Describe roaming as a client moving between APs with reassociation and security handling.

**Key takeaways**

Wireless architecture defines how the network is built, while client association defines how a device joins it. Roaming extends that idea by letting the client move between access points without losing connectivity when the network supports it.

- **ESS** supports broader coverage through multiple access points.
- **IBSS** supports direct device-to-device communication.
- **Roaming** keeps clients connected as they move across coverage areas.
- **Open SSIDs** have a simpler join process than secured networks.
