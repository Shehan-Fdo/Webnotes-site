## **Course:** CompTIA Network+ (Exam N10-009)

**Wireless security overview**

Wireless networks need both access control and confidentiality. In practice, that means you authenticate users before they join the network and encrypt traffic so nearby devices cannot easily read it. A message integrity check also helps verify that received data has not been altered in transit.

Wi-Fi security has evolved because earlier protection methods had serious weaknesses. The main modes you should recognize are WEP, WPA, WPA2, and WPA3. The Wi-Fi Alliance introduced newer modes to replace weaker ones and improve security over time.

- **Authentication** limits who can join the wireless network.
- **Encryption** protects the contents of wireless traffic.
- **Integrity checks** help confirm that data arrives unchanged.

**WEP, WPA, WPA2, and WPA3**

WEP (Wired Equivalent Privacy) is the earliest wireless encryption protocol in this set. It was designed to secure WLAN (wireless local area network) connections, but later research exposed major weaknesses. Because of those weaknesses, it is not a practical choice for modern wireless security.

WPA (Wi-Fi Protected Access) arrived in 2003 as an intermediate step. It is sometimes associated with TKIP (Temporal Key Integrity Protocol), and it was meant to bridge the gap until a stronger standard was ready. WPA2 followed in 2004 and is commonly used as shorthand for the full IEEE 802.11i standard. WPA3 was announced in 2018 and adds security improvements over WPA2.

| Mode | General role | Practical note |
|---|---|---|
| WEP | Early wireless encryption | Weak and obsolete |
| WPA | Transitional improvement | Introduced as an interim measure |
| WPA2 | Stronger mainstream standard | Commonly tied to IEEE 802.11i |
| WPA3 | Newer security mode | Improves on WPA2 |

**Enterprise authentication**

Enterprise wireless security often uses 802.1X authentication. This approach fits environments where you want stronger user verification before granting network access. In practice, you may see username and password authentication, and sometimes multifactor authentication, before a device is allowed onto the wireless network.

**How to think about the modes**

Use the security mode as a balance between compatibility and protection. Older modes may connect more devices, but they do not provide the same level of security as newer standards. WPA2 remains widely supported, while WPA3 offers stronger protection where both the access point and client support it.

> Do not rely on WEP for modern wireless security. It exists as an early standard, but its weaknesses make it unsuitable for current networks.

**Quick exam reminders**

- **WEP** is the oldest of the common wireless security modes.
- **WPA** was introduced as a temporary improvement before WPA2.
- **WPA2** is closely associated with IEEE 802.11i.
- **WPA3** is the newest of the listed modes and improves security.
- **802.1X** is the enterprise authentication framework to remember.

**Practical selection guidance**

When you configure wireless security, choose the strongest mode that your environment supports. If you need broad compatibility, WPA2 is still a common baseline. If your hardware supports it, WPA3 is the better security choice. For enterprise deployments, pair the wireless security mode with strong authentication so only authorized users can connect.
