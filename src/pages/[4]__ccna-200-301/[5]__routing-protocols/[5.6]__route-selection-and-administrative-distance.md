## **Course:** 200-301 CCNA

**Route selection basics**

- Routers can learn the same destination from multiple sources.
- **Longest prefix match** decides between routes to the same destination network.
- **Administrative distance (AD)** helps choose between routes learned from different sources.
- Lower AD is preferred.
- **Metric** is used after route selection rules compare routes within the same routing process.

**Administrative distance**

- AD is a trust value for a route source.
- Cisco routers prefer the route with the lower AD.
- A route learned from **EIGRP** has a lower default AD than a route learned from **RIP**.
- In that case, the EIGRP route is preferred over the RIP route.
- You can view route codes and route sources with `show ip route`.

**Selection order**

- First, the router looks for the most specific prefix.
- Next, it compares AD when multiple routes match the same prefix.
- Then, it uses the metric when routes are still tied within the same protocol or process.
- This order helps the router pick one best path for the routing table.

**Practical example**

- If the same destination is learned through RIP and EIGRP, the router prefers EIGRP by default.
- If two routes have the same prefix length, AD breaks the tie.
- If routes come from the same protocol, the metric helps choose the better path.

**Key terms**

- **Prefix length**: the number of network bits in a route.
- **Route source**: the protocol or method that learned the route.
- **Routing table**: the list of best routes the router uses for forwarding.
- **Metric**: the protocol-specific cost used to compare routes.

**Verification**

- Use `show ip route` to inspect installed routes.
- Check the route code to identify the source protocol.
- Compare AD values when multiple sources advertise the same destination.
- Confirm the most specific matching prefix before comparing other values.
