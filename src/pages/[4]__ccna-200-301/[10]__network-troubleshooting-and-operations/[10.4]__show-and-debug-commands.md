## **Course:** 200-301 CCNA

**Show commands**

- Use `show` commands to verify configuration, view statistics, and inspect protocol or process state in Cisco IOS.
- Type `show ?` to list available `show` command categories.
- Type `show ip ospf ?` to see OSPF-specific subcommands.
- Use `show version` to view loaded software, hardware, and device information.
- Use `show running-config` to view the current configuration file.
- Use `show startup-config` to view the configuration stored in NVRAM.
- Use `show ip interface brief` to see a summary of interface usability status.
- Use `show ip protocols` to view active routing protocol parameters and current state.
- Use `show ip route` to view the routing table.

**Debug commands**

- Use `debug` commands to view device operations, received or generated traffic, and error messages in real time.
- Use `debug` for short troubleshooting sessions because it can consume significant CPU resources.
- Avoid leaving debugging enabled in production unless you need it briefly.
- Narrow the output by debugging only the event you need.
- Use `debug ip icmp` to trace ICMP events such as ping replies.
- Use `no debug ip icmp` to stop ICMP debugging.
- Use `debug ip rip` to trace RIP messages.

**Troubleshooting use**

- Start with `show` commands to confirm the current state before you enable `debug`.
- Use `show` output to identify whether the issue is with interfaces, routing, or protocol state.
- Use `debug` when you need live detail that `show` output does not provide.
- Focus on one protocol or event at a time to keep output readable.
- Stop debugging as soon as you collect the needed information.

**Practical workflow**

1. Check the device state with `show version`, `show ip interface brief`, and `show ip route`.
2. Confirm the relevant protocol or interface behavior with a more specific `show` command.
3. Enable a targeted `debug` command for the suspected issue.
4. Review the live output for the event you are investigating.
5. Disable debugging with the matching `no debug` command.

> `debug` is a troubleshooting tool, not a routine monitoring command, because it can affect device performance.
