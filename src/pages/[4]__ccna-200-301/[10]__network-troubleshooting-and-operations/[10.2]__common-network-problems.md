## **Course:** 200-301 CCNA

**Common network problems**

- Network issues are inevitable.
- You troubleshoot by using a systematic process of elimination.
- Cisco’s troubleshooting model breaks the work into eight steps.
- Define the problem first.
- Gather detailed information next.
- Consider the probable cause.
- Devise a plan to solve the problem.
- Implement the plan.
- Observe the results.
- Repeat the process if the plan does not resolve the issue.
- Document the changes you make.

**Troubleshooting approach**

- Use the **OSI model** as another way to organize troubleshooting.
- Start with the layer that matches the symptom.
- Narrow the issue by checking one layer at a time.
- Keep your focus on what changed, what failed, and what still works.

**Typical symptoms**

- A switch may not forward traffic.
- An IP address may be wrong.
- A router may advertise the wrong path.
- These symptoms usually point to a gap in fundamentals, not a mystery failure.
- Basic CCNA training covers addressing, switching, routing, services, and security basics.

**Practical mindset**

- Build troubleshooting skill by understanding how the pieces work together.
- Verify the network before guessing.
- Use facts from the live environment.
- Treat each failed test as input for the next step.
- Keep notes on what you changed and what happened after the change.

**CCNA context**

- CCNA training teaches you to install, operate, configure, and verify a basic IPv4 and IPv6 network.
- It also covers switches, routers, wireless LAN controllers, and network device management.
- Troubleshooting fits into that broader skill set because you must verify that the network works as intended.
