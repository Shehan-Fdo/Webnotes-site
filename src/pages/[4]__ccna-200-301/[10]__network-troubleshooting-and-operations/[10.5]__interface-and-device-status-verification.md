## **Course:** 200-301 CCNA

**Interface status checks**

- Use `show ip interface brief` to view a summary of interface usability status.
- Use `show interface type` to inspect detailed interface state and counters.
- Use `show interfaces status` on a switch to check port status quickly.
- Read both the interface state and the line protocol state.
- Treat `up/up` as a healthy operational state.
- Treat `administratively down` as a shutdown condition.
- Treat `line protocol is down` as a link or protocol problem.

**Device and configuration verification**

- Use `show version` to view software, hardware, and device information.
- Use `show running-config` to inspect the current configuration.
- Use `show startup-config` to inspect the configuration stored in NVRAM.
- Use `show ip protocols` to check active routing protocol parameters and state.
- Use `show ip route` to review the current routing table.
- Use `show ?` to discover available `show` command categories.
- Use `show ip ospf ?` to narrow command options for a specific protocol.

**Troubleshooting approach**

- Define the problem before you change anything.
- Gather detailed information about the failure.
- Consider the most likely cause.
- Devise a plan to solve the issue.
- Implement the plan.
- Observe the result after the change.
- Repeat the process if the issue remains.
- Document the changes you make.

**Common status interpretation**

- `administratively down` means the interface is disabled in configuration.
- `line protocol is down` means the data-link state is not established.
- A VLAN interface can also show `administratively down` and `line protocol is down`.
- Verify the interface state before you assume a routing or cabling fault.

**Practical verification flow**

1. Check the interface summary with `show ip interface brief`.
2. Inspect the detailed interface output with `show interface type`.
3. Review configuration with `show running-config` if the status is unexpected.
4. Compare the result with the expected operational state.
5. Apply the smallest safe change and verify again.
