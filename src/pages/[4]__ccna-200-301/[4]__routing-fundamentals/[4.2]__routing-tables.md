## **Course:** 200-301 CCNA

**Routing table basics**

- A routing table stores the routes a router uses to forward packets.
- A router uses the table to choose a path to a destination network.
- Static routing uses administrator-configured paths.
- Dynamic routing lets routers exchange route information and update the table automatically.
- Dynamic routing is more practical when the topology changes often or the network is large.

**Connected and local routes**

- A **connected route** appears when an interface is up and has an IP address.
- The interface must be in `up/up` state for the connected route to be added.
- Connected routes represent subnets directly attached to the router.
- A **local route** is automatically created for the specific interface IP address.
- Local routes use a `/32` prefix length.
- IOS 15 and later create local routes automatically for interface addresses.

**Reading `show ip route`**

- Use `show ip route` to display the current routing table.
- Use `show ip route summary` to view a summary of route sources and counts.
- The `show ip route` output uses codes to identify route sources.
- Common codes include `C` for connected, `S` for static, `R` for RIP, `O` for OSPF, and `B` for BGP.
- The command can filter by address, mask, prefix, or protocol.
- `show ip route summary` can show route source, networks, subnets, overhead, and memory use.

**Route selection notes**

- Routers exchange routes through dynamic routing protocols.
- Route updates help the routing table adapt to topology changes and link failures.
- Longest prefix match is a key rule when multiple routes could match a destination.
- Administrative distance and metric also affect route preference.

**Practical takeaways**

- Check interface status before expecting a connected route.
- Verify the interface has an IP address before troubleshooting missing routes.
- Use `show ip route` first when you need to confirm how a router will forward traffic.
- Use route codes to quickly identify where a route came from.
