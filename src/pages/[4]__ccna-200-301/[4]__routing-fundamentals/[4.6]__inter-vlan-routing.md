## **Course:** 200-301 CCNA

**Inter-VLAN routing overview**

- VLANs segment a switched Layer 2 network into separate broadcast domains.
- Hosts in different VLANs cannot communicate without **Layer 3** routing.
- Inter-VLAN routing forwards traffic from one VLAN to another VLAN.
- A router or Layer 3 switch provides the routing service.
- Each VLAN is typically treated as its own subnet.

**Why you need it**

- Devices in one VLAN do not reach devices in another VLAN by switching alone.
- You use routing to move traffic between the subnets.
- Hosts usually point to the router or Layer 3 switch IP address as the default gateway.
- The routing device needs an IP address in each VLAN subnet.

**Common inter-VLAN routing methods**

- **Legacy routing** uses one physical router interface per VLAN.
- This method does not scale well.
- **Router-on-a-stick** uses one router interface with trunking.
- Router-on-a-stick is suitable for small to medium networks.
- **SVI** means switched virtual interface.
- A Layer 3 switch uses SVIs to route between VLANs.

**Legacy routing**

- You connect a separate router LAN interface to the switch for each VLAN.
- You need one Ethernet interface per VLAN.
- This design is rarely used today because it is not scalable.

**Router-on-a-stick**

- You enable trunking on one router interface.
- The single trunk carries traffic for multiple VLANs.
- This design reduces the number of physical router interfaces needed.
- It is also called **ROAS**.
- It is a common choice for smaller environments.

**Layer 3 switch with SVIs**

- A Layer 3 switch performs both switching and routing.
- You create an SVI for each VLAN that needs routing.
- The switch routes traffic between VLANs through those SVIs.
- This approach is often preferred for better scalability.

**Practical configuration ideas**

- Give each VLAN a unique subnet.
- Assign an IP address to the routing interface for each VLAN.
- Configure hosts to use the correct default gateway.
- Use trunking when one physical link must carry multiple VLANs.
- Use a Layer 3 switch when you want switching and routing in one device.

**Key takeaways**

- Inter-VLAN routing is required for communication between VLANs.
- VLANs isolate traffic at Layer 2.
- Routing restores communication between the isolated VLANs.
- The three main options are legacy routing, router-on-a-stick, and Layer 3 switching.
- The best choice depends on scale and design needs.
