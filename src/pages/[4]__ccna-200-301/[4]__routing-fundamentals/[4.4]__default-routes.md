## **Course:** 200-301 CCNA

**Default route basics**

- A **default route** forwards packets when no more specific route matches the destination.
- It is also called the **gateway of last resort**.
- In IPv4, the default route is `0.0.0.0/0`.
- In IPv6, the default route is `::/0`.
- The prefix length is zero, so it is the shortest possible match.
- Routers still prefer the longest prefix match when a specific route exists.

**Why you use a default route**

- You use a default route when learning every specific route is not practical.
- It is useful in **stub networks**.
- It is useful when a router connects to the Internet.
- Without a default route, the router drops packets for destinations not in its routing table.
- A router would otherwise need routes for many external networks.

**Static default route configuration**

- Use `ip route 0.0.0.0 0.0.0.0` to create a default static route.
- You can point the route to a next-hop IP address or an exit interface.
- After configuration, `show ip route` displays the gateway of last resort.
- If no default route exists, `show ip route` shows that the gateway of last resort is not set.

**Default route behavior**

- Packets to unknown destinations follow the default route if it exists.
- Packets to unknown destinations are dropped if no default route exists.
- The default route acts as a catch-all path for unmatched traffic.
- The route is especially important for Internet-bound traffic.

**Default gateway vs default route**

- A **default gateway** is the next hop a host uses to leave its local subnet.
- A **default route** is a routing table entry on a router or Layer 3 device.
- `ip default-gateway` is generally used on Layer 2 switches for management access.
- `ip route 0.0.0.0 0.0.0.0` is used on routers and Layer 3 switches for forwarding.

**Quick verification**

1. Run `show ip route`.
2. Check whether the gateway of last resort is set.
3. Confirm that `0.0.0.0/0` points to the intended next hop or exit interface.
