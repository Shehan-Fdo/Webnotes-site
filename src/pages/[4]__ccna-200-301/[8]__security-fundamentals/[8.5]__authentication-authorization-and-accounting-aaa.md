## **Course:** 200-301 CCNA

**AAA overview**

- **AAA** stands for Authentication, Authorization, and Accounting.
- It manages user activity on a network by controlling access and tracking use.
- It improves security by allowing only approved users to connect.
- It records activity so you can monitor and log what happens on the network.
- It is commonly used on routers, switches, and firewalls.

**Authentication**

- **Authentication** verifies identity before access is granted.
- It asks for valid credentials to prove the user is legitimate.
- It helps prevent unauthorized entry.

**Authorization**

- **Authorization** defines what an authenticated user can do.
- It applies access rules after identity is confirmed.
- It supports granular control over network resources.
- It helps you restrict actions based on user or role.

**Accounting**

- **Accounting** logs user activity on the network.
- It supports auditing and usage tracking.
- It helps you trace actions back to specific users.
- It provides visibility into network access and activity.

**AAA servers and scalability**

- An external **AAA server** centralizes authentication-related tasks.
- Centralization reduces the need to change settings on every device.
- It helps avoid the time and effort of local-only security management.
- Backup AAA servers improve redundancy.

**Cisco device configuration**

- Cisco IOS devices can use AAA for more granular access control.
- The `aaa new-model` command enables AAA on a device.
- AAA supports configuration with **RADIUS** and **TACACS+**.
- RADIUS and TACACS+ act as AAA protocols for network access control.
- You define the AAA server address on the Cisco device during configuration.

**Practical takeaways**

- Use AAA when you need centralized access control.
- Use AAA when you need user-level accountability.
- Use AAA when local password management does not scale well.
- Use AAA to improve both security and auditability.
