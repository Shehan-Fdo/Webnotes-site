## **Course:** 200-301 CCNA

**Access Control List basics**

- An **ACL** is a sequential list of permit and deny conditions.
- It filters packets as they pass through a device.
- It can restrict network use by users or devices.
- It compares packet fields against applied rules on an interface.
- It can permit or deny traffic crossing specified interfaces.

**ACL types**

- **Standard ACLs** filter mainly by source network information.
- **Extended ACLs** support more control parameters.
- Extended ACLs can filter by network addresses and TCP/UDP applications.
- Use the ACL type that matches the level of control you need.
- Keep the rule set as simple as possible for the task.

**How ACLs work**

- A packet arrives on an interface.
- The device checks the packet against the applied ACL.
- The first matching rule determines the action.
- A permit rule forwards the packet.
- A deny rule blocks the packet.

**Wildcard masks**

- ACL examples commonly use **wildcard masks**.
- A wildcard mask helps match address ranges.
- Cisco documentation includes wildcard mask examples for ACL configuration.
- Use the mask carefully when you want to match specific hosts or groups.

**Configuration and management**

- You can configure ACLs through the **CLI**.
- Cisco also references GUI-based tools for some platforms.
- Some devices support filtering inbound traffic with a security wizard.
- You need to understand the network design before applying filters.
- You need to know how interfaces are used on the filtering device.

**Practical use**

- Use ACLs to limit traffic flow.
- Use ACLs to control access to services.
- Use ACLs to block unwanted packets at a chosen point in the network.
- Place the ACL where it best matches the traffic you want to control.

**Study focus**

- Know the difference between **standard** and **extended** ACLs.
- Know that ACLs use ordered permit and deny statements.
- Know that wildcard masks are part of ACL matching.
- Know that interface placement affects filtering results.
