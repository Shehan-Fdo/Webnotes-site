## **Course:** 200-301 CCNA

**Ethernet frame basics**

- An Ethernet frame is the data link layer protocol data unit used on Ethernet LANs.
- Ethernet is defined by the IEEE 802.3 standard.
- A frame carries data plus control information for delivery.
- Ethernet uses source and destination **MAC** addresses to identify endpoints.
- The frame is the unit that moves data across the Ethernet network.

**Frame placement in transmission**

- A frame is preceded by a preamble and an **SFD**.
- **SFD** means Start Frame Delimiter.
- The preamble helps the receiver synchronize.
- The SFD marks the end of the preamble.
- The frame begins after the SFD.

**Frame fields**

- The header contains the destination MAC address.
- The header contains the source MAC address.
- The Type field identifies the protocol inside the frame, such as IPv4 or IPv6.
- The data field carries the payload.
- Padding is added when needed to meet the minimum data length.
- The frame ends with the **FCS**.
- **FCS** means Frame Check Sequence.
- The FCS uses a 32-bit **CRC**.
- **CRC** means Cyclic Redundancy Check.
- The FCS helps detect corrupted data.

**Practical structure**

- The frame starts with synchronization fields.
- The header identifies sender and receiver.
- The payload carries the encapsulated data.
- The trailer provides error detection.
- Each field has a specific role in reliable delivery.

**CCNA exam focus**

- Know the order of the major frame parts.
- Know the purpose of the destination and source MAC addresses.
- Know that the Type field identifies the upper-layer protocol.
- Know that padding supports the minimum frame data size.
- Know that the FCS checks for corruption.
