## **Course:** 200-301 CCNA

**Switching overview**

- A switch is a **Layer 2** device.
- It forwards frames based on the destination **MAC address**.
- It learns source MAC addresses and builds a MAC address table.
- The MAC address table helps the switch choose an outgoing port.
- If the destination is unknown, the switch can flood the frame within the VLAN or broadcast domain.

**Hubs vs. switches**

- A hub repeats traffic out all ports.
- A switch sends traffic only where it needs to go.
- Hubs create more shared contention than switches.
- Switches improve efficiency by reducing unnecessary frame delivery.

**Collision domains**

- A collision domain is the part of a network where frame collisions can occur.
- Collisions are common in shared-media hub environments.
- Each port on a switch is a separate collision domain.
- Each port on a bridge or router is also a separate collision domain.

**Broadcast domains**

- A broadcast domain is the set of devices that receive a Layer 2 broadcast.
- Switches forward broadcasts by default within the same broadcast domain.
- Routers do not forward broadcasts between broadcast domains.
- All ports on a hub or switch are in the same broadcast domain by default.
- Router interfaces separate broadcast domains.

**Switch operation**

- The switch examines the incoming frame.
- It records the source MAC address and ingress port.
- It checks the destination MAC address against its table.
- It forwards, floods, or filters the frame based on that lookup.
- This behavior is the core of LAN switching.

**Key terms**

- **Frame**: Layer 2 data unit.
- **MAC address**: Hardware address used for local delivery.
- **Flooding**: Sending a frame out multiple ports.
- **Filtering**: Not forwarding a frame out a port.
- **Learning**: Adding source MAC information to the table.
