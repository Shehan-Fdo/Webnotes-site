## **Course:** 200-301 CCNA

**ARP overview**

- **ARP** stands for Address Resolution Protocol.
- It maps an IPv4 address to a hardware, or **MAC**, address.
- You use it when a device needs to communicate on a local network.
- Ethernet communication needs the destination physical address before sending packets.

**How ARP works**

- A sending device knows the destination IP address.
- The sender broadcasts an ARP request on the local network segment.
- The request includes the target IP address.
- All devices on the segment receive the request.
- Only the device with that IP address sends an ARP reply.
- The reply contains the device’s MAC address.
- The sender can then forward the packet to the correct device.

**Broadcast behavior**

- ARP requests use broadcast delivery.
- Ethernet broadcast uses `FF:FF:FF:FF:FF:FF`.
- IP broadcast uses `255.255.255.255`.
- Broadcast lets every host on the local segment see the request.
- The correct host answers with a unicast reply.

**Address resolution purpose**

- ARP translates IP addresses to MAC addresses.
- This translation supports local delivery on Ethernet networks.
- The sender cannot complete Layer 2 forwarding until it learns the MAC address.
- ARP solves that lookup before data transmission.

**Practical flow**

1. You identify the destination IPv4 address.
2. You send an ARP request for that address.
3. You wait for the host with that address to reply.
4. You use the returned MAC address to send the packet.

**Key takeaways**

- ARP works on local network segments.
- It resolves IPv4 to MAC addresses.
- Requests are broadcast.
- Replies come from the host that owns the IP address.
