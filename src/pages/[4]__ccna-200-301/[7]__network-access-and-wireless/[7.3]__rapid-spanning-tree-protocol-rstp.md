## **Course:** 200-301 CCNA

**RSTP overview**

- **RSTP** means Rapid Spanning Tree Protocol.
- It is an evolution of **STP**.
- It was introduced as IEEE `802.1w`.
- It is backwards-compatible with STP.
- IEEE later moved RSTP details into `802.1Q`.

**Why RSTP matters**

- RSTP reduces convergence time after a topology change.
- STP can take `30` to `50` seconds to move from blocking to forwarding.
- RSTP typically reacts in less than `10` seconds after a physical link failure.
- Some references describe convergence in just a couple of seconds.
- Faster convergence helps keep switched networks available.

**Shared behavior with STP**

- RSTP uses the same root bridge election rules as STP.
- RSTP uses the same root port selection rules as STP.
- RSTP uses the same designated port selection rules as STP.
- Both protocols place ports into forwarding or blocking behavior.
- In RSTP, blocking is called the **discarding** state.

**Port states**

- RSTP merges STP listening, blocking, and disabled behavior into the **discarding** state.
- RSTP still uses forwarding behavior for active ports.
- The simplified state model helps speed up convergence.

**Port roles**

- RSTP keeps the **root port** role.
- RSTP keeps the **designated port** role.
- RSTP adds the **alternate port** role.
- RSTP adds the **backup port** role.
- A root port is the forwarding port closest to the root bridge by path cost.
- A designated port is the forwarding port for a LAN segment.
- An alternate port provides a backup path toward the root bridge.
- A backup port provides redundancy on the same switch-side path.

**How RSTP speeds recovery**

- RSTP uses additional port roles to react faster to failures.
- Alternate and backup ports can move to forwarding more quickly than STP waiting behavior.
- RSTP uses synchronization to negotiate port roles and states.
- Proposal and agreement behavior helps speed convergence in active topologies.

**Practical exam focus**

- Know that RSTP is the faster successor to STP.
- Know that `802.1w` is the original IEEE standard for RSTP.
- Know that `discarding` replaces STP blocking-related states in RSTP.
- Know the five common RSTP port roles: root, designated, alternate, backup, and disabled.
- Know that RSTP keeps the same basic root election logic as STP.
