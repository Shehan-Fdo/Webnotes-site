## **Course:** 200-301 CCNA

**WLAN components**

- A **wireless LAN** (WLAN) uses access points to connect wireless clients to the network.
- A lightweight AP can be managed by a **wireless LAN controller** (WLC).
- The AP mode determines whether the AP serves clients or supports network functions.

**Client-serving AP modes**

- **Local mode** is the default mode for lightweight APs.
- Local mode provides **basic service sets** (BSSs) and lets clients associate to wireless LANs.
- When the AP is not sending client frames, it scans other channels.
- Scanning helps measure noise and interference.
- Scanning also helps discover rogue devices.
- Scanning can help match intrusion detection system (IDS) events.

**FlexConnect mode**

- **FlexConnect** lets the AP switch traffic locally between an SSID and a VLAN.
- FlexConnect is useful when the CAPWAP connection to the WLC is down.
- CAPWAP stands for **Control and Provisioning of Wireless Access Points**.
- FlexConnect can still support a remote site during controller loss.
- FlexConnect can also egress traffic at the AP LAN port.

**Bridge and mesh modes**

- **Bridge mode** turns an AP into a dedicated bridge between networks.
- Bridge mode can support point-to-point links.
- Bridge mode can also support point-to-multipoint links.
- **Mesh mode** is listed with bridge mode in Cisco AP mode references.
- Use bridge or mesh behavior when the AP must extend connectivity rather than serve normal clients.

**Practical exam focus**

- Know that AP modes are configured from the controller side for lightweight APs.
- Know the difference between client-serving modes and management-oriented modes.
- Match each mode to its main purpose before you answer exam questions.
- Use conservative wording when a question does not specify the exact Cisco platform or feature set.
