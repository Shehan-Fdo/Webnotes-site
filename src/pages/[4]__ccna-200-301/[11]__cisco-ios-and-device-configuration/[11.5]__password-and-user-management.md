## **Course:** 200-301 CCNA

**Password and user access basics**

- Use **usernames and passwords** to protect Cisco devices from unauthorized CLI access.
- Assign **privilege levels** to control what each user can do.
- Cisco IOS supports 16 privilege levels, from `0` to `15`.
- Lower levels provide less access.
- Higher levels provide more access.

**Default privilege levels**

- `0` is the least privileged level.
- Level `0` allows only a small set of commands.
- `1` is user-level access.
- Level `1` provides limited read-only access.
- `15` is the most privileged level.
- Level `15` provides full administrative access.

**Password types**

- Use `enable password` and `enable secret` for privileged access control.
- Prefer **enable secret** for stronger protection.
- `enable secret` is the recommended choice when both are available.
- Use local usernames for line login when you want per-user access control.

**Local user configuration**

- Create local user accounts on the device.
- Assign a password to each account.
- Map users to the privilege level they need.
- Use local usernames and passwords on VTY lines when you want network logins to use device-local authentication.

**VTY line access**

- Configure **VTY** lines to accept logins with local usernames and passwords.
- Use this approach for remote access to the router or switch.
- Match the login method to the access policy you want.

**Practical security guidance**

- Give each user only the access they need.
- Use lower privilege levels for limited tasks.
- Reserve level `15` for full administrative work.
- Protect device access with both authentication and privilege control.

**Exam focus**

- Know the purpose of local usernames and passwords.
- Know the difference between `enable password` and `enable secret`.
- Know the meaning of Cisco privilege levels `0`, `1`, and `15`.
- Know that privilege levels help implement role-based access control on IOS devices.
