## **Course:** 200-301 CCNA

**Boot sequence overview**

- Cisco routers and switches use a predictable startup flow.
- The device runs **POST** (Power-On Self-Test) first.
- POST checks hardware and verifies installed components.
- Bootstrap code then tells the device where to load the IOS (Internetwork Operating System) image.
- The device loads configuration files after the IOS starts.
- The device then presents a prompt for management or further configuration.

**Memory roles**

- **ROM** stores the bootstrap program and POST code.
- **Flash** stores the IOS image.
- **RAM** stores operational data such as routing tables and the running configuration.
- **NVRAM** stores the startup configuration file.
- RAM contents are lost on reload or power loss.
- Flash and NVRAM retain contents across reloads.

**Configuration files**

- The **running configuration** lives in RAM.
- The **startup configuration** lives in NVRAM.
- The device uses the startup configuration during boot if it is available.
- If the startup configuration is missing or not loaded, the device may enter setup or another fallback behavior.

**Configuration register**

- The **configuration register** helps control boot behavior.
- Cisco documentation notes that the register value affects how the router loads software and configuration.
- You use it when you need to change startup behavior.
- It is part of the boot sequence troubleshooting process.

**Boot troubleshooting focus**

- Check whether the device reaches POST successfully.
- Check whether the IOS image loads from the expected location.
- Check whether the startup configuration loads from NVRAM.
- Check the configuration register when the boot path is unexpected.
- Use fallback behavior awareness to narrow startup problems.

**Practical memory comparison**

| Memory | Stores | Volatile |
|---|---|---|
| `ROM` | Bootstrap and POST | No |
| `Flash` | IOS image | No |
| `RAM` | Running config and tables | Yes |
| `NVRAM` | Startup config | No |

**Key takeaways**

- Boot starts with hardware checks, not configuration loading.
- IOS loading happens before configuration loading.
- RAM is temporary storage for active operation.
- NVRAM preserves the startup configuration.
- The configuration register can change how the device boots.
