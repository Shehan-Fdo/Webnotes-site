## **Course:** 200-301 CCNA

**Licensing and exam context**

- The `200-301 CCNA` exam validates knowledge in network fundamentals, network access, IP connectivity, IP services, security fundamentals, and automation and programmability.
- Passing the exam earns the **CCNA** certification.
- Cisco describes CCNA training as preparation for installing, operating, configuring, and verifying a basic IPv4 and IPv6 network.
- Cisco also includes managing network devices as part of the CCNA training scope.

**IOS image management basics**

- Cisco IOS software images and configuration files can be stored on a central `TFTP` server.
- `TFTP` stands for Trivial File Transfer Protocol.
- Central storage helps control the number of IOS images you maintain.
- Central storage helps control image revisions.
- Central storage also helps manage configuration file backups.
- Production internetworks often span wide areas and include multiple routers.
- Keeping a backup location is a good practice in larger networks.

**Why image management matters**

- Approved software images help keep network equipment up to date.
- Large installations make image consistency harder to maintain.
- Software image management helps standardize image versions across the network.
- Software image management also helps keep configurations consistent.

**Cisco Catalyst Center and SWIM**

- **SWIM** means Software Image Management.
- Cisco Catalyst Center can streamline software image management.
- Cisco Catalyst Center can automate image control across the network.
- Cisco Catalyst Center supports guided management of image versions and configurations.
- Automation reduces manual effort in large deployments.

**Practical takeaways**

- Use a central repository for IOS images when you need easier control and backup.
- Track image versions so you can manage revisions deliberately.
- Keep approved images consistent across devices.
- Use automation tools when manual image management becomes difficult at scale.
