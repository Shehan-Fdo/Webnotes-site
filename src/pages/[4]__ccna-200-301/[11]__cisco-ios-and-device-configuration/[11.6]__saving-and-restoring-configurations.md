## **Course:** 200-301 CCNA

**Configuration backups**

- Keep a backup copy of the device configuration.
- Use the `copy` command to move configurations between locations.
- The first `copy` parameter is the source.
- The second `copy` parameter is the destination.
- Back up both `running-config` and `startup-config` when needed.
- `running-config` is the active configuration in memory.
- `startup-config` is the saved configuration used at boot.

**TFTP backups**

- Use a **TFTP** server for configuration backups.
- TFTP means Trivial File Transfer Protocol.
- TFTP is a client-server protocol for sending and receiving files.
- Set up the TFTP server before you copy files to it.
- In Packet Tracer, you can use a Server device, assign an IP address, and enable the TFTP service.
- A common backup command is `copy running-config tftp:`.
- The device prompts for the remote host address or name.
- The device then prompts for a destination filename.
- The backup writes the configuration to the TFTP server.

**Restoring configurations**

- Restore a saved configuration by copying it back to the device.
- Use the reverse direction of the backup process.
- To restore into the saved boot configuration, use `copy tftp: startup-config`.
- To restore into the active configuration, use `copy tftp: running-config`.
- `copy startup-config running-config` can reload the saved configuration into the current session.
- Be careful when restoring to `running-config` because it affects the active device state.

**Practical workflow**

1. Prepare a reachable TFTP server.
2. Save the current configuration with `copy running-config tftp:`.
3. Confirm the remote host and filename prompts.
4. Store the backup in a known location.
5. Restore the file with the appropriate `copy` command when needed.
6. Choose `startup-config` for boot-time recovery or `running-config` for immediate changes.

**Key reminders**

- Match the source and destination carefully.
- Verify the TFTP server address before copying.
- Use backups before making major configuration changes.
- Keep filenames clear so you can identify the correct version later.
