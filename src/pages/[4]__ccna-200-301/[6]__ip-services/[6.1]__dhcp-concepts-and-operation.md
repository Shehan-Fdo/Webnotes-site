## **Course:** 200-301 CCNA

**DHCP overview**

- **DHCP** stands for Dynamic Host Configuration Protocol.
- It assigns network parameters to devices automatically.
- It reduces the need to configure static settings on each host.
- It uses a client-server model.
- The client requests configuration from a DHCP server.
- The server maintains a pool of available IP addresses.
- The server can also provide:
  - subnet mask
  - default gateway
  - domain name
  - DNS server

**DHCP operation**

- A client starts by sending a broadcast **DHCP Discover** message.
- The Discover message helps locate DHCP servers on the LAN.
- DHCP servers reply with **DHCP Offer** messages.
- An Offer includes IP addressing information.
- If multiple Offers arrive, the client accepts the first one.
- The client then sends a broadcast **DHCP Request** message.
- The Request asks for the selected network parameters.
- The server completes the exchange with a confirmation message.

**Key behavior**

- DHCP simplifies address management in a network.
- Cisco routers can act as both DHCP clients and DHCP servers.
- The initial discovery uses broadcast traffic.
- The process follows a predictable request-and-response flow.

**Practical focus**

- Identify the four-step DHCP exchange:
  1. Discover
  2. Offer
  3. Request
  4. Confirmation
- Match each message to its role in the exchange.
- Remember that the server provides more than just an IP address.
