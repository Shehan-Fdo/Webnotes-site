## **Course:** 200-301 CCNA

**SNMP overview**

- **SNMP** means Simple Network Management Protocol.
- It is an application-layer protocol for network device management.
- You use it to collect and manage information from devices such as routers, switches, servers, and printers.
- It supports network monitoring and device status reporting.

**SNMP components**

- An **NMS** (Network Management Station) is the management software on the administrative computer.
- The NMS gathers SNMP data from managed devices.
- An **agent** runs on the managed device.
- The agent reports information back to the NMS.

**Notifications and traps**

- Devices can send an SNMP alert called a **trap**.
- A trap notifies the NMS about a problem or event.
- Traps help you learn about issues without waiting for a manual poll.
- A device can send a trap when it detects a condition such as a security violation.

**Ports and transport**

- SNMP commonly uses **UDP** (User Datagram Protocol).
- Agents use UDP port `161`.
- The manager uses UDP port `162` for traps.
- Keep port roles clear when you troubleshoot SNMP communication.

**Management data**

- SNMP works with management information on the device.
- Managed systems expose data as variables.
- This data is organized in a **MIB** (Management Information Base).
- The MIB helps structure what the NMS can read or modify.

**SNMP versions**

- The current SNMP version in the evidence is **SNMPv3**.
- Use conservative wording when comparing versions unless you have version-specific requirements.
- SNMPv3 is the secure SNMP version referenced in the sources.

**Practical CCNA takeaways**

- Match the **manager** with the NMS role.
- Match the **agent** with the managed device role.
- Remember `161` for agent communication.
- Remember `162` for trap delivery.
- Use SNMP for monitoring and management, not for end-user data transfer.
