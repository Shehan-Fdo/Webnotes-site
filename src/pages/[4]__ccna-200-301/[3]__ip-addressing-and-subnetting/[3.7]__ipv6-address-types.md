## **Course:** 200-301 CCNA

**IPv6 address categories**

- IPv6 addresses fall into three main categories: **unicast**, anycast, and multicast.
- IPv6 does not use broadcast.
- Multicast provides the functional equivalent of sending to all hosts on a network.

**Unicast**

- A **unicast** address identifies a single interface.
- Packets sent to a unicast address are delivered to one host.
- Global unicast addresses are used on public networks.
- Global unicast addresses are assigned by the Internet Assigned Numbers Authority (IANA).
- Global unicast addresses use the prefix `2000::/3`.
- In practice, Internet IPv6 routing table prefixes commonly start with `2` or `3`.
- Unique local addresses exist as a separate unicast type.
- Link-local addresses are also a unicast type.

**Anycast**

- An **anycast** address identifies one or more interfaces.
- Packets sent to an anycast address go to the nearest matching server.
- Anycast supports load balancing.
- Anycast is often described as a “one-to-nearest” address.

**Multicast**

- A **multicast** address represents a dynamic group of hosts.
- Packets sent to a multicast address are delivered to many interfaces.
- IPv6 multicast serves a similar purpose to IPv4 multicast.

**Address space notes**

- IPv6 addresses are 128 bits long.
- The IPv6 address space is much larger than IPv4.
- IANA allocates only a small portion of the IPv6 space.
- Some addresses that begin with `000` and `111` are reserved for special types.
- The remaining space is reserved for future use.

**Quick comparison**

| Type | Scope | Delivery |
|---|---|---|
| Unicast | One interface | One host |
| Anycast | One or more interfaces | Nearest host |
| Multicast | Group of hosts | Many interfaces |

**Exam focus**

- Remember that IPv6 uses multicast instead of broadcast.
- Remember that global unicast addresses are the public IPv6 addresses you use on routed networks.
- Remember that anycast sends traffic to the nearest available server.
