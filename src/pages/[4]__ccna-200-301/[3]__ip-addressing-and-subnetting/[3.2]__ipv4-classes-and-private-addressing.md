## **Course:** 200-301 CCNA

**IPv4 address classes**

- IPv4 classful addressing groups addresses by the first octet.
- **Class A** uses the first octet range `1-126`.
- **Class B** uses the first octet range `128-191`.
- **Class C** uses the first octet range `192-223`.
- **Class D** is used for multicast.
- **Class E** is reserved for experimental use.
- Classful ranges are a legacy way to describe IPv4 networks.
- Modern network design relies more on subnetting and special-use blocks than on classful assignment.

**Private IPv4 addressing**

- Private IPv4 addresses are not globally unique.
- You can reuse the same private range in different organizations.
- Private addressing helps conserve public IPv4 space.
- RFC 1918 defines private IPv4 address allocation for private internets.
- Private networks can still communicate internally with full network-layer connectivity.
- To reach the Internet, you typically use **NAT** (Network Address Translation).

**RFC 1918 private ranges**

- `10.0.0.0/8` is a private range.
- `172.16.0.0/12` is a private range.
- `192.168.0.0/16` is a private range.
- These ranges are intended for internal use.
- The same private subnet can appear in unrelated networks without conflict inside those separate networks.

**Special-use IPv4 notes**

- Some IPv4 blocks have special purposes beyond normal host addressing.
- RFC 5735 describes global and other specialized IPv4 address blocks assigned by IANA (Internet Assigned Numbers Authority).
- Special-use blocks are separate from private address space.
- Use the address purpose, not just the class label, when identifying how an IPv4 block behaves.

**Practical CCNA takeaways**

- Identify the class from the first octet when classful terminology appears.
- Recognize private ranges as non-routable on the public Internet without translation.
- Use `NAT` when private hosts need Internet access.
- Treat class labels as a reference point, not the full design method.
