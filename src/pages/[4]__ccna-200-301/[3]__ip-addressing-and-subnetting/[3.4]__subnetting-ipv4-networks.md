## **Course:** 200-301 CCNA

**Subnetting overview**

- **Subnetting** divides one IPv4 network into two or more smaller networks.
- It helps improve routing efficiency.
- It reduces the size of the broadcast domain.
- It can improve network security by separating groups of hosts.
- It supports organizational design by grouping devices by department or function.

**Why subnetting matters**

- A single large subnet creates one broadcast domain.
- Broadcast traffic reaches all hosts in that subnet.
- Large shared networks can create unnecessary traffic.
- Sensitive systems should not always share the same subnet with user workstations.
- Separate subnets can limit access to specific resources.

**Subnetting basics**

- An IPv4 subnet uses a network portion and a host portion.
- A subnet mask defines which bits belong to the network.
- Borrowing host bits creates more subnets.
- Borrowing more bits leaves fewer host addresses per subnet.
- The subnet design must balance subnet count and host capacity.

**Host address calculation**

- Use `2^n - 2` to estimate usable host addresses.
- `n` is the number of host bits left after subnetting.
- The `- 2` accounts for the network address and broadcast address.
- A subnet must reserve both of those addresses.
- Plan the host portion before you choose the final mask.

**Subnet planning example**

- A `192.168.116.0/24` network has 8 host bits.
- If you need 30 hosts per subnet, you need 5 host bits.
- `2^5 - 2 = 30`.
- The remaining 3 bits can be borrowed for subnetting.
- This creates smaller subnets from the original `/24` network.

**Practical design goals**

- Choose a subnet size that fits the required number of hosts.
- Keep related devices in the same subnet when appropriate.
- Separate departments when access control or traffic reduction matters.
- Use subnetting to make address management more organized.
- Verify that each subnet still has enough usable host addresses.

**Key terms**

- **Network address**: the identifier for the subnet itself.
- **Broadcast address**: the address used to reach all hosts in the subnet.
- **Usable host address**: an address that can be assigned to a device.
- **Subnet mask**: the value that separates network bits from host bits.
