## **Course:** 200-301 CCNA

**Network components overview**

- Networks use multiple **devices** to move data and support communication.
- Core components include switches, routers, firewalls, access points, servers, and wireless controllers.
- Network fundamentals also cover cables, cloud-based networks, topology, and architecture models.
- You should understand how each component fits into the network design.

**Switches**

- A **switch** connects devices within a local network.
- Switches are a core device in network fundamentals.
- They help build LANs and support internal traffic flow.

**Routers**

- A **router** connects different networks.
- Routers are part of the core device set in CCNA network fundamentals.
- You use routers to move traffic between network segments.

**Firewalls**

- A **firewall** helps control and inspect network traffic.
- Firewalls are listed as a common network component in CCNA fundamentals.
- They support network security and traffic filtering.

**Wireless and access devices**

- An **access point** provides wireless connectivity.
- A **wireless controller** manages wireless infrastructure.
- These devices support wireless network design and operation.

**Infrastructure and supporting components**

- **Servers** provide network services and resources.
- **Copper cables** and **fiber optics** carry network traffic.
- **GBIC** modules appear in network infrastructure discussions.
- Cloud-based networks are part of the broader network environment.

**Network design concepts**

- Network topology describes how devices connect.
- Three-tier architecture is a common design model.
- Spine-and-leaf topology is another design approach mentioned in the evidence.
- Network redundancy improves resilience.

**CCNA study focus**

- Learn what devices do and how data travels through them.
- Connect device roles to the OSI model and network layers.
- Read packet flows to understand how traffic moves across the network.
- Build and test small network setups to reinforce the concepts.
