## **Course:** 200-301 CCNA

**APIs and REST overview**

- Cisco added **programmability** to CCNA to support automation in modern networks.
- Manual CLI-based management can be time-consuming and prone to human error.
- Automation helps with routine tasks, configuration management, and troubleshooting.
- REST APIs are part of the automation and programmability domain.
- REST stands for **Representational State Transfer**.
- An API is a way for software to communicate with a service or device.

**REST API basics**

- REST APIs use standard HTTP requests.
- Common HTTP methods include `GET`, `POST`, `PUT`, and `DELETE`.
- `GET` reads data.
- `POST` creates data.
- `PUT` updates data.
- `DELETE` removes data.
- Not all resources support every HTTP method.
- Some resources may support only `GET`.

**HTTP responses and status codes**

- Check the HTTP `Status-Code` field in the header.
- `2xx` codes indicate success.
- `4xx` codes indicate an error caused by the request data.
- `5xx` codes indicate a server-side error.
- `200 OK` is a common success response.
- `400 Bad Request` can indicate invalid state or missing data.
- `401 Unauthorized` indicates an authentication problem.

**JSON in API work**

- JSON stands for **JavaScript Object Notation**.
- JSON is commonly used with REST APIs.
- JSON helps represent structured data in a readable format.

**Why this matters for CCNA**

- API knowledge supports network automation workflows.
- REST and JSON help you interact with modern Cisco platforms.
- Programmability reduces repetitive manual work.
- Automation can improve deployment speed and network reliability.

**Study focus**

- Learn the purpose of REST APIs.
- Match HTTP methods to their actions.
- Recognize common response code ranges.
- Understand JSON as a data format used in API exchanges.
