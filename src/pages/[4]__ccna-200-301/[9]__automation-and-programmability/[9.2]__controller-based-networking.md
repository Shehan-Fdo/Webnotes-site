## **Course:** 200-301 CCNA

**Controller-Based Networking**

- **Controller-based networking** uses a controller to centralize management and control.
- Cisco describes **SDN** (software-defined networking) as an architecture that makes networks more flexible and easier to manage.
- SDN abstracts the control plane from the data forwarding function in network devices.
- A controller is the core element of an SDN architecture.
- The controller enables centralized management, automation, and policy enforcement.
- SDN supports physical and virtual network environments.
- Controller-based networking is part of CCNA automation and programmability content.
- Cisco Press notes that this topic includes data plane and control plane concepts.
- Cisco Press also highlights examples of network programmability using controllers.
- The topic compares traditional networking with controller-based networking.

**Key Concepts**

- The **control plane** makes forwarding decisions.
- The **data plane** forwards traffic.
- In SDN, the controller centralizes control instead of leaving each device to operate only with local logic.
- **Programmability** lets you interact with the network through software-driven methods.
- **Automation** helps you perform repeated tasks more consistently.
- Automation reduces manual repetition and helps avoid human error.
- Controller-based designs often use APIs (application programming interfaces) for communication.
- Cisco references **southbound APIs** between the controller and network devices.
- Cisco community discussion also points to **northbound APIs** between the controller and applications.
- Northbound and southbound interfaces describe direction of communication in the controller model.

**Benefits**

- Centralized control simplifies management.
- Policy enforcement becomes more consistent.
- Automation improves repeatability.
- Programmability supports faster network changes.
- SDN can make the network easier to manage.
- Controller-based approaches help align network behavior with policy.

**Traditional vs Controller-Based Networks**

- Traditional networks rely more on device-by-device configuration.
- Controller-based networks use a central controller to coordinate devices.
- Traditional management can require more manual work.
- Controller-based management can reduce repetitive configuration.
- Traditional designs keep control logic closer to each device.
- SDN separates control from forwarding to improve flexibility.

**Interfaces and APIs**

- A **southbound API** connects the controller to network devices.
- A **northbound API** connects the controller to applications.
- APIs carry information between software components.
- Interface direction matters when you describe controller communication.
- Use the controller as the reference point when naming northbound and southbound paths.

**Study Focus**

- Know the roles of the controller, control plane, and data plane.
- Know why SDN centralizes management.
- Know how automation and programmability support consistency.
- Know the difference between northbound and southbound communication.
- Know that controller-based networking is a core CCNA automation topic.
