## **Course:** 200-301 CCNA

**Lab focus**

- These labs cover **VLANs**, trunking, and inter-VLAN routing.
- Packet Tracer is used for hands-on practice.
- The lab flow pairs short lectures with lab exercises.
- The goal is to verify connectivity between devices in different VLANs.

**VLAN setup**

- Create VLANs to segment the network.
- Assign switch ports to the correct VLAN.
- Use access ports for end devices in a single VLAN.
- Confirm that each host belongs to the intended broadcast domain.

**Trunk links**

- Configure trunks between switches when VLAN traffic must cross the link.
- Allow the required VLANs on the trunk.
- Check that trunking supports traffic for multiple VLANs.
- Verify the link carries the expected VLAN tags.

**Inter-VLAN routing**

- Use inter-VLAN routing when devices in different VLANs must communicate.
- A router-on-a-stick design uses one router interface with subinterfaces.
- Configure the router and switch so VLAN traffic reaches the router over a trunk.
- Test routing by pinging between hosts in different VLANs.

**Verification**

- Check switch configuration after each change.
- Confirm VLAN membership on access ports.
- Confirm trunk status on inter-switch links.
- Verify end-to-end connectivity after routing is in place.

**Lab approach**

- Work through the configuration in small steps.
- Validate each step before moving on.
- Use the lab to connect theory with device configuration.
- Repeat the exercise until the VLAN path is clear.
