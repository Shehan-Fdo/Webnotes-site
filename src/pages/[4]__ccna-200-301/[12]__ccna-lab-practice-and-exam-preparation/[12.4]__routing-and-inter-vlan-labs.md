## **Course:** 200-301 CCNA

**Lab focus**

- VLANs segment a LAN for **scalability**, security, and management.
- Inter-VLAN routing lets hosts in different VLANs communicate.
- Router-on-a-stick uses one router interface with subinterfaces.
- An 802.1Q trunk carries traffic for multiple VLANs on one link.
- The native VLAN is part of trunk configuration and must match on both ends.

**Typical lab objectives**

- Build the network and set basic device settings.
- Create VLANs and assign switch ports.
- Configure an 802.1Q trunk between switches.
- Configure inter-VLAN routing on the router.
- Verify that inter-VLAN routing works.

**Common addressing pattern**

- Router subinterfaces map to VLANs.
- Example lab addressing uses `192.168.3.1/24` for VLAN 3.
- Example lab addressing uses `192.168.4.1/24` for VLAN 4.
- Switch management interfaces use VLAN 3 in the sample topology.
- End devices use the router subinterface as the default gateway.

**VLAN and port assignment pattern**

- VLAN 3 is labeled **Management** in the sample lab.
- VLAN 4 is labeled **Operations** in the sample lab.
- VLAN 7 is labeled **ParkingLot** for unused access ports.
- VLAN 8 is labeled **Native** for the trunk native VLAN.
- Access ports belong to one VLAN only.
- Parking-lot ports stay assigned to an unused VLAN.

**Router-on-a-stick setup**

- Create a subinterface for each routed VLAN.
- Assign the VLAN ID with `encapsulation dot1Q`.
- Configure the subinterface IP address for the VLAN gateway.
- Leave the physical router interface without an IP address when using subinterfaces.
- Use the router interface connected to the trunk link.

**Verification checks**

- Confirm VLANs exist on the switches.
- Confirm access ports are in the correct VLAN.
- Confirm the trunk is up and carries the expected VLANs.
- Confirm each router subinterface is up and has the correct IP address.
- Test host-to-host communication across VLANs.

**Troubleshooting reminders**

- A VLAN mismatch breaks host connectivity.
- A trunk mismatch prevents VLAN traffic from crossing the link.
- A missing subinterface IP address prevents routing for that VLAN.
- A wrong default gateway stops hosts from reaching other VLANs.
- A native VLAN mismatch can cause trunk issues.

**Practice approach**

- Follow the lab in stages.
- Verify each stage before moving on.
- Use the addressing table to match interfaces and gateways.
- Test connectivity after routing is configured.
- Recheck trunk and VLAN settings if pings fail.
