## **Course:** CompTIA Linux+ Complete Course (Exam XK0-006)

**Command discovery and help**

Linux administration starts with knowing how to find the right command and interpret its documentation. Use the built-in help system before searching the web, because exam tasks often expect you to identify the correct option quickly.

- `man <command>` opens the manual page for a command.
- `man -k <keyword>` searches manual page descriptions by keyword.
- `info <command>` opens GNU Info documentation when available.
- `<command> --help` prints a short usage summary.
- `help <builtin>` shows help for shell built-ins such as `cd`, `export`, and `alias`.
- `type <name>` tells you whether a name is an alias, shell built-in, function, or executable.
- `command -v <name>` prints the path or resolution of a command.
- `which <command>` shows the executable path found in `PATH`, but it does not detect shell built-ins or aliases reliably.
- `whatis <command>` gives a one-line summary from the manual database.

A practical workflow is to inspect the command type first, then read the manual, then test the syntax with a harmless option.

```bash
type cd
command -v ls
man chmod
man -k network
```

**Privilege escalation and sudo**

Administrative work often requires temporary elevated privileges. Use `sudo` to run a single command as another user, usually `root`, instead of logging in directly as the superuser.

- `sudo <command>` runs one command with elevated privileges.
- `sudo -l` lists what the current user is allowed to run.
- `sudo -u <user> <command>` runs a command as a different user.
- `visudo` safely edits the sudoers file and checks syntax before saving.
- `/etc/sudoers` and files in `/etc/sudoers.d/` control sudo policy.

> Avoid editing `/etc/sudoers` with a normal text editor. A syntax error can lock out administrative access.

```bash
sudo -l
sudo systemctl restart sshd
sudo -u apache whoami
visudo
```

**Shell navigation and environment**

Basic shell commands are used constantly in troubleshooting and scripting. Know the difference between the current working directory, the home directory, and environment variables.

- `pwd` prints the current directory.
- `cd` changes directories.
- `ls` lists directory contents.
- `echo $HOME` prints the current user's home directory.
- `env` or `printenv` shows environment variables.
- `export NAME=value` sets an environment variable for child processes.
- `alias ll='ls -alF'` creates a shortcut in the current shell.
- `unalias ll` removes an alias.
- `history` shows previously run commands.
- `clear` clears the terminal screen.

Use `command` when you want to bypass an alias or function and run the real executable.

```bash
alias ll='ls -alF'
command ls
unalias ll
```

**File and text operations**

File management is a core exam area. You should be comfortable creating, copying, moving, deleting, and inspecting files, as well as searching and transforming text.

- `touch file` creates an empty file or updates timestamps.
- `cp source dest` copies files or directories.
- `mv source dest` moves or renames files.
- `rm file` removes files.
- `mkdir dir` creates directories.
- `rmdir dir` removes empty directories.
- `cat file` prints file contents.
- `less file` pages through file contents interactively.
- `head -n 20 file` shows the first 20 lines.
- `tail -n 20 file` shows the last 20 lines.
- `tail -f file` follows a file as it grows.
- `tail -F file` follows a file and attempts to reopen it after rotation, but it is not guaranteed to catch every rotation pattern.
- `wc -l file` counts lines.
- `file name` identifies the file type by content signatures.

For text processing, use the classic Unix filters in pipelines.

- `grep 'pattern' file` searches for matching lines.
- `grep -R 'pattern' /path` searches recursively.
- `grep -i` ignores case.
- `grep -n` shows line numbers.
- `sed` edits or transforms text stream data.
- `awk` extracts fields and performs pattern-based processing.
- `cut -d: -f1 /etc/passwd` extracts fields using a delimiter.
- `sort` orders lines.
- `uniq` removes adjacent duplicates.
- `tr` translates or deletes characters.
- `tee` writes output to both screen and file.

```bash
grep -Rin 'error' /var/log
sed -n '1,5p' /etc/passwd
awk -F: '{print $1, $3}' /etc/passwd
cut -d: -f1 /etc/passwd | sort | uniq
```

**Permissions, ownership, and special bits**

Linux permissions control who can read, write, or execute files. Exam questions often combine standard permissions with ownership and special bits.

- `chmod` changes permissions.
- `chown` changes owner and group.
- `chgrp` changes group ownership.
- `umask` sets default permissions for newly created files.
- `stat file` shows detailed metadata.
- `getfacl file` displays Access Control Lists (ACLs).
- `setfacl` modifies ACLs.
- `ls -l` shows standard permission bits and ownership.

Special bits change how permissions behave:

- **setuid** on an executable runs it with the file owner's effective user ID.
- **setgid** on an executable runs it with the file group's effective group ID.
- **setgid** on a directory makes new files inherit the directory group on many systems.
- **sticky bit** on a directory prevents users from deleting files they do not own, even if the directory is writable.

```bash
chmod 4755 /usr/local/bin/tool
chmod 2755 /srv/shared
chmod 1777 /tmp
chown alice:devops report.txt
getfacl /srv/share
setfacl -m g:qa:r-x /srv/share
setfacl -m d:g:qa:r-x /srv/share
```

> ACLs on a directory can control access to the directory itself, but existing files inside it may still need their own ACLs or recursive changes if you want consistent access.

**Searching for files and content**

Use `find` when you need precise file selection by name, type, size, time, ownership, or permissions. This is more powerful than `locate` because it searches the live filesystem.

- `find /path -name '*.conf'` searches by name.
- `find /path -type f` limits results to files.
- `find /path -type d` limits results to directories.
- `find /path -mtime -1` finds files modified in the last day.
- `find /path -user alice` finds files owned by a user.
- `find /path -perm 0644` matches exact permissions.
- `find /path -exec command {} \;` runs a command on each match.
- `locate filename` searches a prebuilt database and is fast, but the database may be stale.
- `updatedb` refreshes the locate database.

```bash
find /var/log -type f -name '*.log' -mtime -7
find /tmp -type f -size +100M -exec ls -lh {} \;
find /home -user alice -perm -u+w
```

**Process management**

Processes are central to troubleshooting performance, services, and stuck applications. Know how to inspect, signal, and prioritize them.

- `ps` shows running processes.
- `ps aux` is a common full-process listing.
- `ps -ef` is another common format.
- `top` provides a live process view.
- `htop` is an enhanced interactive process viewer when installed.
- `pgrep name` finds process IDs by name.
- `pidof name` returns process IDs for a program.
- `kill PID` sends `SIGTERM` by default.
- `kill -9 PID` sends `SIGKILL` and should be a last resort.
- `pkill name` kills by process name.
- `nice -n 10 command` starts a process with lower priority.
- `renice 5 -p PID` changes an existing process priority.

```bash
ps -ef | grep sshd
pgrep -a nginx
kill -TERM 1234
renice 10 -p 1234
```

**Open files, sockets, and locks**

When a file will not unmount, a port is already in use, or a process is holding a resource, inspect open file descriptors.

- `lsof` lists open files and the processes using them.
- `lsof /path/to/file` shows who has a file open.
- `lsof -i :80` shows processes using port 80.
- `lsof -p PID` shows all open files for a process.
- `fuser /path` identifies processes using a file or filesystem.
- `fuser -k /path` can terminate processes using a resource, so use it carefully.

```bash
lsof -iTCP:22 -sTCP:LISTEN
lsof /var/log/messages
fuser -v /mnt/data
```

**Services and systemd**

`systemd` manages services, sockets, mounts, timers, and other units on most modern Linux distributions. Understand the difference between enabling a unit and starting it.

- `systemctl status <unit>` shows current state and recent logs.
- `systemctl start <unit>` starts a unit now.
- `systemctl stop <unit>` stops a unit now.
- `systemctl restart <unit>` stops and starts a unit.
- `systemctl reload <unit>` asks a service to re-read configuration without a full restart, if supported.
- `systemctl enable <unit>` configures a unit to start at boot.
- `systemctl disable <unit>` prevents a unit from starting at boot.
- `systemctl enable --now <unit>` enables the unit and starts it immediately.
- `systemctl is-enabled <unit>` checks boot-time enablement.
- `systemctl is-active <unit>` checks whether it is running.
- `systemctl list-units` lists loaded units.
- `systemctl list-unit-files` lists installed unit files and their enablement state.
- `systemctl daemon-reload` reloads unit files after changes.

```bash
systemctl status sshd
systemctl enable --now firewalld
systemctl restart nginx
systemctl is-enabled cron
```

**Logging and journal queries**

The systemd journal is a primary source of service and boot logs. Use filters to narrow output quickly during troubleshooting.

- `journalctl` shows journal entries.
- `journalctl -b` shows logs from the current boot.
- `journalctl -b -1` shows logs from the previous boot.
- `journalctl -u <unit>` filters by service or unit.
- `journalctl -f` follows new log entries.
- `journalctl --priority=err` filters by severity.
- `journalctl --grep='pattern'` searches message text.
- `journalctl -n 50` shows the last 50 entries.
- `journalctl --since '1 hour ago'` filters by time range.

```bash
journalctl -u sshd -b
journalctl -p warning --since today
journalctl -f -u nginx
journalctl --grep='failed password'
```

**Package management**

Package tools differ by distribution, but the exam expects you to recognize common query, install, update, and verification tasks.

On RPM-based systems:

- `rpm -q package` checks whether a package is installed.
- `rpm -qi package` shows package information.
- `rpm -ql package` lists installed files.
- `rpm -V package` verifies installed files against package metadata.
- `rpm -i package.rpm` installs a package file.
- `rpm -U package.rpm` upgrades or installs a package file.
- `dnf search term` searches repositories.
- `dnf info package` shows package details.
- `dnf install package` installs a package.
- `dnf remove package` removes a package.
- `dnf update` updates installed packages.
- `dnf check-update` checks for available updates without installing them.
- `dnf repolist` lists enabled repositories.

On Debian-based systems:

- `apt update` refreshes package lists from configured repositories.
- `apt install package` installs a package.
- `apt remove package` removes a package but keeps configuration files.
- `apt purge package` removes a package and its configuration files.
- `apt upgrade` installs available updates for installed packages.
- `apt-cache search term` searches package metadata.
- `apt-cache policy package` shows candidate versions and repository priority.
- `dpkg -l` lists installed packages.
- `dpkg -i package.deb` installs a local package file.
- `dpkg -L package` lists files installed by a package.
- `dpkg -S /path/to/file` identifies which package owns a file.

```bash
rpm -q bash
rpm -V openssh-server
dnf search tcpdump
dnf info vim-enhanced
apt-cache search nginx
apt-cache policy curl
dpkg -S /usr/bin/ssh
```

Repository configuration matters because package managers can only install from enabled sources.

- On RPM systems, repository definitions commonly live in `/etc/yum.repos.d/`.
- On Debian systems, repository entries commonly live in `/etc/apt/sources.list` and `/etc/apt/sources.list.d/`.
- `dnf repolist` and `apt update` help you confirm repository access and metadata freshness.

**Networking basics and troubleshooting**

Networking tasks often require both configuration and verification. Start by checking interface state, IP addressing, routing, and name resolution.

- `ip addr show` displays interface addresses.
- `ip link show` displays link state and interface flags.
- `ip route show` displays the routing table.
- `hostname` shows the system hostname.
- `hostname -I` shows assigned IP addresses on many systems.
- `nmcli` manages NetworkManager from the command line.
- `nmtui` provides a text-based NetworkManager interface.
- `ping host` tests reachability.
- `ss -tulpn` shows listening TCP and UDP sockets with process information.
- `curl URL` retrieves data from a URL and is useful for HTTP troubleshooting.
- `wget URL` downloads files non-interactively.
- `tcpdump` captures packets for low-level network troubleshooting.

```bash
ip addr show
ip route show
hostname -I
nmcli device status
curl -I https://example.com
ss -tulpn
tcpdump -i eth0 port 53
```

Use `curl` when you want to inspect headers, test web services, or verify API responses. Use `tcpdump` when you need to confirm whether traffic is actually leaving or arriving on an interface.

**Name resolution and host configuration**

DNS and local host configuration are common exam topics because many service failures are really name-resolution failures.

- `/etc/hosts` maps hostnames to IP addresses locally.
- `/etc/resolv.conf` contains resolver configuration on many systems, though it may be managed dynamically.
- `getent hosts name` queries the system name service switch and is a good test of resolution.
- `dig name` queries DNS directly when installed.
- `nslookup name` is another DNS query tool, though `dig` is generally preferred for detail.

```bash
getent hosts server1
dig example.com
cat /etc/hosts
```

**Time and timezone management**

Correct time is important for logs, certificates, Kerberos, and scheduled jobs. Know how to inspect and set system time settings.

- `date` shows or sets the system date and time.
- `timedatectl` shows and manages system time, timezone, and NTP status.
- `timedatectl list-timezones` lists available timezones.
- `timedatectl set-timezone Region/City` sets the timezone.
- `hwclock` reads or sets the hardware clock.

```bash
date
timedatectl
timedatectl set-timezone America/New_York
hwclock --show
```

**Storage, partitions, and LVM**

Storage administration often combines device discovery, partitioning, filesystems, mounting, and logical volume management. Understand the sequence: identify the device, partition if needed, create a filesystem, mount it, and make it persistent.

- `lsblk` shows block devices and mount points.
- `blkid` displays filesystem UUIDs and types.
- `fdisk` and `parted` create and modify partitions.
- `mkfs.ext4`, `mkfs.xfs`, and similar commands create filesystems.
- `mount` attaches a filesystem to the directory tree.
- `umount` detaches a mounted filesystem.
- `df -h` shows filesystem usage.
- `du -sh /path` shows directory size.
- `/etc/fstab` defines persistent mounts.

`mount` with no arguments typically lists current mounts on many systems, but output can vary by util-linux version, so `findmnt` or `lsblk` is often clearer for exam and troubleshooting use.

```bash
lsblk -f
blkid
mount /dev/sdb1 /mnt/data
df -h /mnt/data
umount /mnt/data
```

Logical Volume Manager (LVM) is common in enterprise Linux because it simplifies resizing and storage abstraction.

- `pvcreate /dev/sdb` initializes a physical volume.
- `pvs` lists physical volumes.
- `vgcreate vgdata /dev/sdb` creates a volume group.
- `vgs` lists volume groups.
- `lvcreate -n lvshare -L 20G vgdata` creates a logical volume.
- `lvs` lists logical volumes.
- `lvdisplay` shows detailed logical volume information.
- `lvextend -L +10G /dev/vgdata/lvshare` grows a logical volume.
- `resize2fs` grows an ext4 filesystem after extending the volume.
- `xfs_growfs` grows an XFS filesystem after extending the mounted filesystem.

```bash
pvcreate /dev/sdb
vgcreate vgdata /dev/sdb
lvcreate -n lvshare -L 20G vgdata
mkfs.xfs /dev/vgdata/lvshare
mount /dev/vgdata/lvshare /srv/share
lvextend -L +10G /dev/vgdata/lvshare
xfs_growfs /srv/share
```

> For XFS, grow the filesystem while it is mounted. For ext4, `resize2fs` is commonly used after `lvextend`.

**Backup, archive, and integrity checks**

Backups and verification are essential when moving data or validating downloads. Use checksums to confirm integrity before and after transfer.

- `tar -cvf archive.tar files...` creates an archive.
- `tar -xvf archive.tar` extracts an archive.
- `tar -czvf archive.tar.gz files...` creates a compressed gzip archive.
- `tar -tzf archive.tar.gz` lists archive contents.
- `sha256sum file` computes a SHA-256 checksum.
- `md5sum file` computes an MD5 checksum, though SHA-256 is generally preferred for integrity checks.
- `cmp file1 file2` compares two files byte by byte.
- `diff file1 file2` compares text files line by line.

```bash
tar -czvf backup-home.tar.gz /home/alice
tar -tzf backup-home.tar.gz
sha256sum backup-home.tar.gz
sha256sum -c backup-home.tar.gz.sha256
```

**Scheduling tasks**

Use scheduled jobs for recurring maintenance and one-time execution. Know the difference between `cron` and `at`.

- `crontab -e` edits the current user's cron table.
- `crontab -l` lists cron jobs.
- `crontab -r` removes the current user's cron table.
- `/etc/crontab` and `/etc/cron.d/` are system-wide cron locations.
- `at now + 10 minutes` schedules a one-time job.
- `atq` lists queued `at` jobs.
- `atrm <jobid>` removes an `at` job.

```bash
crontab -e
# Run backup every day at 2:30 AM
30 2 * * * /usr/local/bin/backup.sh

echo "systemctl restart nginx" | at now + 1 hour
atq
```

**Security and access control**

Security troubleshooting often involves authentication, authorization, SELinux, and file access controls. Use the right tool for the layer you are changing.

- `passwd` changes a user password.
- `chage` manages password aging.
- `id` shows user and group IDs.
- `groups` shows group membership.
- `su - user` switches to another user and loads their login environment.
- `sudo -l` checks allowed privileged commands.
- `visudo` edits sudo policy safely.
- `getenforce` shows SELinux mode.
- `setenforce 0` temporarily sets SELinux to permissive mode on systems that support it.
- `sestatus` shows SELinux status.
- `semanage` makes persistent SELinux policy changes when installed.
- `restorecon` restores default SELinux file contexts.
- `ls -Z` shows SELinux contexts.

```bash
id alice
sudo -l
getenforce
sestatus
restorecon -Rv /var/www/html
semanage fcontext -a -t httpd_sys_content_t '/srv/web(/.*)?'
restorecon -Rv /srv/web
```

**System identity and hardware overview**

Quick identity commands help you confirm the machine you are working on, especially in multi-host labs.

- `uname -a` shows kernel and system information.
- `hostnamectl` shows and can set the system hostname on systemd systems.
- `lscpu` shows CPU details.
- `free -h` shows memory usage.
- `uptime` shows how long the system has been running and the load average.
- `dmesg` shows kernel ring buffer messages, useful for hardware and driver issues.

```bash
uname -a
hostnamectl
lscpu
free -h
uptime
dmesg | tail
```

**Practical exam patterns**

Many Linux+ questions are scenario-based. You usually need to identify the right command family, then choose the safest option.

- Use `systemctl status` before restarting a service so you can see the failure context.
- Use `journalctl -u <unit> -b` when a service fails after boot.
- Use `lsof -i` or `ss -tulpn` when a port is already in use.
- Use `find` for live filesystem searches and `grep` for content searches.
- Use `rpm -V` or `dpkg -S` when verifying package integrity or ownership.
- Use `timedatectl` when logs appear out of order or authentication fails due to time drift.
- Use `sha256sum` when validating downloads or backup archives.
- Use `nmcli` when you need repeatable network configuration from the CLI.

A compact troubleshooting sequence often looks like this:

1. Confirm the service or resource state with `systemctl status`, `ss`, `ip addr`, or `lsblk`.
2. Inspect logs with `journalctl`, `dmesg`, or application logs.
3. Verify configuration files and permissions with `grep`, `stat`, `ls -l`, or `getfacl`.
4. Apply the smallest safe change.
5. Recheck the service, logs, and expected output.

**High-value command reference**

| Area | Commands | Typical use |
|---|---|---|
| Help and discovery | `man`, `info`, `--help`, `type`, `command -v` | Find syntax and command origin |
| Files and text | `cp`, `mv`, `rm`, `find`, `grep`, `sed`, `awk` | Manage and transform data |
| Permissions | `chmod`, `chown`, `umask`, `setfacl` | Control access |
| Processes | `ps`, `top`, `kill`, `pgrep`, `lsof` | Inspect and manage running tasks |
| Services | `systemctl`, `journalctl` | Control and troubleshoot units |
| Packages | `dnf`, `rpm`, `apt`, `dpkg` | Install, query, verify software |
| Networking | `ip`, `ss`, `nmcli`, `curl`, `tcpdump` | Configure and troubleshoot connectivity |
| Storage | `lsblk`, `fdisk`, `mount`, `lvm` tools | Manage disks and volumes |
| Time | `date`, `timedatectl`, `hwclock` | Verify and set system time |
| Integrity | `sha256sum`, `tar`, `diff` | Validate files and backups |
