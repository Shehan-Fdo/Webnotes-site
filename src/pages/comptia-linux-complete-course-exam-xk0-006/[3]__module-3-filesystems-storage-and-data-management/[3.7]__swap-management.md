## **Course:** CompTIA Linux+ Complete Course (Exam XK0-006)

**Swap fundamentals**

Swap is disk-backed virtual memory that the kernel uses when RAM pressure increases. It helps the system avoid immediate out-of-memory conditions, but it is much slower than RAM, so swap is a safety net rather than a performance resource.

Linux can use swap in two common forms:

- **Swap partition**: a dedicated block device partition marked for swap use.
- **Swap file**: a regular file on a filesystem that the kernel uses as swap.

Swap is useful when:

- You need extra breathing room during memory spikes.
- You want hibernation support, which typically requires swap space at least as large as RAM in many setups.
- You need a flexible way to add memory-backed storage without repartitioning.

Swap does not replace proper memory sizing. Heavy swap activity usually indicates memory pressure and can cause noticeable slowdown.

**How the kernel uses swap**

The kernel moves inactive memory pages from RAM to swap based on memory pressure and page activity. This process is managed by the virtual memory subsystem and is influenced by the **swappiness** setting.

Key ideas:

- Frequently used pages stay in RAM.
- Less active anonymous memory is a candidate for swap.
- File-backed pages can often be dropped and reloaded from disk instead of being swapped out.
- Swap is not a cache you should expect to read from frequently.

A system with swap enabled may still show free RAM usage because Linux uses memory aggressively for cache. That is normal. What matters is whether the system is under sustained memory pressure.

**Inspecting swap status**

Use standard tools to see whether swap is active and how much is in use.

```bash
swapon --show
free -h
cat /proc/swaps
```

Typical output fields include:

- **Filename**: swap device or file path.
- **Type**: partition or file.
- **Size**: total swap capacity.
- **Used**: current swap usage.
- **Priority**: selection order when multiple swap areas exist.

You can also inspect memory behavior with:

```bash
vmstat 1
```

Look at:

- `si`: swap in rate
- `so`: swap out rate

Sustained nonzero values suggest active swapping.

**Creating a swap partition**

A swap partition is a dedicated partition with the Linux swap type. This is common on systems with fixed storage layouts.

Typical workflow:

1. Create a partition with a tool such as `fdisk`, `parted`, or `gdisk`.
2. Set the partition type to Linux swap.
3. Format it with `mkswap`.
4. Enable it with `swapon`.
5. Persist it in `/etc/fstab`.

Example:

```bash
sudo mkswap /dev/sdb3
sudo swapon /dev/sdb3
swapon --show
```

To make it persistent across reboots, add an `/etc/fstab` entry:

```fstab
/dev/sdb3 none swap defaults 0 0
```

Using a UUID is often safer than a device name because device names can change:

```bash
blkid /dev/sdb3
```

Then use the UUID in `/etc/fstab`:

```fstab
UUID=1234abcd-5678-90ef-1234-567890abcdef none swap defaults 0 0
```

**Creating a swap file**

A swap file is often easier to add on an existing system because it does not require repartitioning. It is useful for virtual machines, small systems, and quick capacity changes.

Example procedure:

1. Create the file.
2. Restrict permissions.
3. Mark it as swap.
4. Enable it.
5. Add it to `/etc/fstab`.

```bash
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
swapon --show
```

If `fallocate` is not suitable on a filesystem, you can use `dd`:

```bash
sudo dd if=/dev/zero of=/swapfile bs=1M count=2048 status=progress
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```

Add this to `/etc/fstab`:

```fstab
/swapfile none swap defaults 0 0
```

> Ensure the swap file has restrictive permissions. A world-readable swap file is a security risk because it may expose sensitive memory contents.

**Managing multiple swap areas**

Linux can use more than one swap area at the same time. The kernel distributes usage based on priority.

Rules of thumb:

- Higher priority swap is preferred first.
- Equal-priority swap areas can be used in a balanced way.
- Multiple swap devices can improve resilience and sometimes performance, depending on storage layout and workload.

Example `swapon --show` output conceptually looks like:

```text
NAME      TYPE      SIZE USED PRIO
/swapfile file        2G   0B   -2
/dev/sdb3 partition   4G   0B   -3
```

You can set priority in `/etc/fstab`:

```fstab
/swapfile none swap defaults,pri=100 0 0
/dev/sdb3 none swap defaults,pri=10 0 0
```

**Tuning swappiness**

The `vm.swappiness` kernel parameter controls how aggressively Linux swaps out anonymous memory.

- Lower values reduce swap activity.
- Higher values encourage earlier swapping.
- The default is commonly around `60`, though distributions may vary.

Check the current value:

```bash
sysctl vm.swappiness
```

Set it temporarily:

```bash
sudo sysctl vm.swappiness=10
```

Set it persistently by creating a file such as `/etc/sysctl.d/99-swappiness.conf`:

```conf
vm.swappiness=10
```

Then reload settings:

```bash
sudo sysctl --system
```

Practical guidance:

- Use a lower value on desktops or latency-sensitive systems if swap activity is undesirable.
- Use a moderate value on servers where some swapping is acceptable and memory pressure is expected.
- Do not treat swappiness as a fix for insufficient RAM.

**Adding and removing swap safely**

You can disable swap when needed, such as during maintenance or before resizing storage.

Disable a swap area:

```bash
sudo swapoff /swapfile
```

Enable it again:

```bash
sudo swapon /swapfile
```

To remove a swap file completely:

1. Disable it with `swapoff`.
2. Remove its `/etc/fstab` entry.
3. Delete the file.

```bash
sudo swapoff /swapfile
sudo rm -f /swapfile
```

For a swap partition, remove or comment out the `fstab` entry and then reconfigure the partition if you want to reuse it for another purpose.

**Troubleshooting swap**

Common issues usually involve configuration, permissions, or filesystem support.

- **Swap does not activate at boot**
  - Check `/etc/fstab` syntax.
  - Verify the device or file exists.
  - Confirm the UUID is correct if you use one.
  - Review boot logs with `journalctl -b`.

- **Swap file fails to activate**
  - Ensure permissions are `600`.
  - Confirm the filesystem supports swap files correctly.
  - Verify the file is not sparse in a problematic way on the target filesystem.
  - Recreate the file with `fallocate` or `dd` if needed.

- **Unexpected high swap usage**
  - Check memory pressure with `free -h`, `vmstat`, and `top` or `htop`.
  - Identify memory-hungry processes with `ps`, `top`, or `smem` if available.
  - Consider whether the workload is leaking memory or simply exceeding RAM.

Useful commands:

```bash
free -h
vmstat 1
top
journalctl -b | grep -i swap
```

**Performance and operational considerations**

Swap is slower than RAM because it depends on storage I/O. The impact depends on the underlying device:

- **HDD**: slowest, most noticeable latency.
- **SATA SSD**: much better than HDD, but still slower than RAM.
- **NVMe SSD**: fastest common storage option, but still far behind RAM.

Operational tips:

- Place swap on fast storage when possible.
- Avoid excessive swap on busy database or low-latency systems.
- Monitor memory usage before the system starts thrashing.
- Use swap as a buffer, not as a substitute for capacity planning.

**Hibernation and swap**

Hibernation writes RAM contents to swap so the system can power off and restore later. Because of this, hibernation often requires:

- Sufficient swap space.
- Correct bootloader and resume configuration.
- A stable swap location, often a partition or a reliably referenced swap file.

If hibernation is in scope for a system, verify the resume setup carefully. The exact configuration varies by distribution and boot stack.

**Practical examples**

Check current swap and memory state:

```bash
free -h
swapon --show
cat /proc/swaps
```

Create a 4 GiB swap file and enable it:

```bash
sudo fallocate -l 4G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
echo '/swapfile none swap defaults,pri=50 0 0' | sudo tee -a /etc/fstab
```

Lower swappiness for a desktop system:

```bash
echo 'vm.swappiness=10' | sudo tee /etc/sysctl.d/99-swappiness.conf
sudo sysctl --system
```

Disable all swap temporarily:

```bash
sudo swapoff -a
```

Re-enable all swap entries from `/etc/fstab`:

```bash
sudo swapon -a
```

**Exam focus**

For Linux+ exam purposes, know how to:

- Distinguish between swap partitions and swap files.
- Create swap with `mkswap`.
- Enable and disable swap with `swapon` and `swapoff`.
- Make swap persistent with `/etc/fstab`.
- Inspect usage with `free`, `swapon --show`, and `/proc/swaps`.
- Explain `swappiness` and its effect on memory behavior.
- Troubleshoot common swap file and boot-time activation problems.

**Quick reference**

- `mkswap`: initialize a partition or file for swap use
- `swapon`: activate swap
- `swapoff`: deactivate swap
- `free -h`: view memory and swap summary
- `swapon --show`: list active swap areas
- `vmstat 1`: monitor swap-in and swap-out activity
- `sysctl vm.swappiness`: inspect kernel swap tendency
