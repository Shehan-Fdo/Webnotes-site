## **Course:** Comptia Network Plus

**Routing basics**

- A router forwards traffic by checking the destination IP address against its **routing table**.
- The routing table lists the next hop or outgoing interface for a destination.
- Devices such as workstations and servers can also maintain routing tables.
- A packet may pass through multiple routers before it reaches the destination.

**Static routing**

- **Static routing** is manually configured by an administrator.
- You specify where traffic should go, such as a destination network and next hop.
- Static routes are predictable and simple to configure.
- Static routes do not adapt automatically to failures.
- Static routes require manual updates when the network changes.
- Static routing does not add routing protocol overhead.
- Static routing can be more secure because it does not exchange routing updates.

**Dynamic routing**

- **Dynamic routing** updates routes automatically without human intervention.
- Routers discover routes by exchanging information with neighbors.
- Dynamic routing keeps routing tables in sync when the network changes.
- Dynamic routing reduces manual reconfiguration.
- Dynamic routing is better suited to larger or changing networks.
- Dynamic routing depends on routing protocols.

**Static vs. dynamic routing**

| Static routing | Dynamic routing |
| --- | --- |
| Manual configuration | Automatic route discovery |
| Predictable behavior | Adapts to network changes |
| Low overhead | Uses routing protocol traffic |
| Best for simple or stable paths | Best for larger or changing networks |
| Requires manual maintenance | Reduces manual maintenance |

**Default routes**

- A **default route** gives a router a path to use when no more specific route matches.
- Default routes help forward traffic toward an upstream router or gateway.
- Default routes are useful when a router does not need a full route for every destination.

**Troubleshooting focus**

- Start with the routing table when traffic does not reach the expected destination.
- Check whether the next hop is correct.
- Check whether the route is static, dynamic, or default.
- Verify that the router has a valid path for the destination network.
