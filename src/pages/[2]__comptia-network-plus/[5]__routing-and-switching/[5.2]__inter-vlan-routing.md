## **Course:** Comptia Network Plus

**Inter-VLAN routing overview**

- VLANs (Virtual Local Area Networks) split one network into separate broadcast domains.
- Devices in different VLANs do not communicate directly by default.
- Inter-VLAN routing enables traffic between those isolated VLANs.
- You use routing to connect VLANs while keeping the VLAN separation.

**Why you need it**

- VLANs improve security by separating groups of devices.
- VLANs reduce broadcast traffic.
- VLANs organize network resources logically.
- You need inter-VLAN routing when devices in different VLANs must exchange traffic.

**How routing fits in**

- A router forwards traffic to the correct destination using a routing table.
- The routing table acts like a map of available paths.
- The router checks the table to choose the best next hop.
- Routing decisions depend on the destination network.

**Common implementation idea**

- A router can sit between VLANs and forward traffic between them.
- Older designs often use a separate router connected to switch interfaces.
- The router provides the path that VLANs cannot create on their own.
- In practice, the router or Layer 3 device becomes the point of inter-VLAN communication.

**VLANs and trunking**

- VLANs create separate broadcast domains on a single switch.
- `802.1Q` trunking carries VLAN traffic across a link.
- Trunking helps move multiple VLANs over one connection.
- VLANs can also support voice and video traffic on the same connection.

**Security considerations**

- VLAN separation limits direct communication between groups.
- A router is normally required to cross from one VLAN to another.
- Misconfiguration can weaken the intended separation.
- VLAN hopping attacks may try to move between VLANs without a router.

**Troubleshooting focus**

- Check whether the VLANs are configured as intended.
- Verify that routing exists between the VLANs.
- Confirm the correct interfaces and links are in place.
- Review the routing table when traffic does not reach the destination.

**Key terms**

- **Broadcast domain**: a group of devices that receive each other’s broadcasts.
- **Routing table**: the set of paths a router uses to forward traffic.
- **Next hop**: the next device or router in the path.
- **Trunk**: a link that carries traffic for multiple VLANs.
- **Subinterface**: a logical interface used in some routing designs.
