## **Course:** Comptia Network Plus

**Routing fundamentals**

- Routing selects a path for data packets from source to destination.
- Routers forward traffic to the correct destination by evaluating incoming packets.
- A router sends traffic out the appropriate interface based on its knowledge of the network.
- Packets often pass through multiple intermediate routers before reaching the destination.

**Routing tables**

- A **routing table** stores the information a router uses to decide where to forward traffic.
- The table acts like a map of destinations and next hops.
- A **next hop** is the next router or device in the path.
- Routers consult the routing table when a packet arrives.
- Workstations and servers can also maintain routing tables.

**Route selection**

- Routers use route selection rules to choose the best path.
- **Longest prefix match** wins first.
- **Administrative distance** helps determine how trustworthy a route source is.
- Lower administrative distance is preferred.
- **Metric** helps compare routes from the same protocol.
- Lower metric is preferred when the protocol uses metric-based comparison.
- Static routes can take precedence over dynamic routes in route selection.
- Directly connected networks are typically preferred over learned routes.

**Routing protocols**

- Routing protocols help routers learn routes dynamically.
- Dynamic routing reduces manual route maintenance.
- Static routing uses manually configured routes.
- Routing protocols support path selection across changing networks.
- Common routing discussions include OSPF, BGP, and EIGRP.
- Route preference can vary by protocol and route type.

**Routing technologies**

- Network Address Translation (**NAT**) changes address information for traffic that crosses network boundaries.
- Port Address Translation (**PAT**) is a form of address translation that uses ports to help share addresses.
- First Hop Redundancy Protocol (**FHRP**) helps provide nearly uninterrupted network availability.
- Virtual IP addressing can support redundancy and service continuity.
- Layer 3 subinterfaces let you divide a physical interface into logical interfaces.
- Virtual Local Area Network (**VLAN**) design affects routing and segmentation.
- Spanning Tree Protocol (**STP**) helps prevent switching loops in network designs.
- Maximum Transmission Unit (**MTU**) affects the largest packet size a link can carry.

**Practical troubleshooting focus**

- Start troubleshooting routing issues with the routing table.
- Verify the expected next hop for the destination.
- Check whether the route is static, connected, or learned dynamically.
- Compare route preference when multiple routes exist for the same subnet.
- Confirm interface and subinterface configuration when traffic should cross logical boundaries.

**Key terms**

- **Static route**: a manually configured route.
- **Dynamic route**: a route learned through a routing protocol.
- **Default route**: a route used when no more specific match exists.
- **Default gateway**: the next hop used for off-subnet traffic.
- **Interface**: the network connection a router uses to send traffic onward.
