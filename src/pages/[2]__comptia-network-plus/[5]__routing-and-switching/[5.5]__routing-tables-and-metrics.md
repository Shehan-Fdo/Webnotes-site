## **Course:** Comptia Network Plus

**Routing tables**

- A router evaluates incoming traffic and decides where to send it next.
- A **routing table** gives the router the information it uses to make that decision.
- Most devices have routing tables, including workstations and servers.
- Routers maintain their own routing tables for forwarding decisions.
- Troubleshooting a router often starts and ends with the routing table.
- A routing table can include multiple possible paths to the same destination.

**Next hop and route selection**

- A **next hop** is the next router or device in the path.
- The router uses the routing table to choose the best next hop.
- If multiple routes exist, the router compares them before forwarding traffic.
- The route choice affects ingress and egress traffic flow through the network.
- A hop is a packet passing through a router.

**Routing metrics**

- A **routing metric** helps a router prioritize one route over another.
- Different routing protocols use different metric calculations.
- Different operating systems can also calculate metrics differently.
- Metrics matter when a router has several valid paths to the same destination.
- The router uses the metric to decide which route is best.

**Metric examples**

- Routing Information Protocol (RIP) uses **hop count**.
- EIGRP uses a **composite metric**.
- EIGRP can consider bandwidth, delay, load, reliability, and MTU.
- By default, EIGRP uses minimum bandwidth and delay.
- Load, reliability, and MTU are disabled by default in EIGRP metric calculation.
- EIGRP uses K-values to control which attributes affect the metric.

**Longest prefix match**

- Routers use **longest prefix match** to choose the most specific route.
- The route with the longest matching prefix is preferred.
- More matching bits means a more specific route.
- If several routes match a destination, the most specific one wins.
- This helps the router select the best route for a destination IP address.

**Practical troubleshooting**

- Check the routing table first when traffic does not reach the expected destination.
- Verify the next hop for the destination you are testing.
- Compare available routes when more than one path exists.
- Confirm the metric for each competing route.
- Use the most specific matching route when reviewing forwarding behavior.
