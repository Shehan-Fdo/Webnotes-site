## **Course:** Comptia Network Plus

**IPv4 address basics**

- An IP address is a **logical address** used for routing packets over an IP network.
- A host needs a unique IP address to participate on an IP network.
- IPv4 uses 32-bit addresses written in dotted decimal notation.
- Subnetting uses a subnet mask to divide an address into network and host portions.

**Public addresses**

- A **public address** is globally routable on the Internet.
- Public IPv4 space is assigned through Internet address allocation systems.
- You use public addresses when a device must be reachable from outside a private network.

**Private addresses**

- A **private address** is used inside an enterprise network.
- RFC 1918 defines private IPv4 address space.
- Private address space supports internal connectivity within an organization.
- Private addresses are not meant for direct Internet routing.
- Large private networks can run out of private IPv4 space.
- Renumbering may be required when moving between public and private address plans.

**Special-use addresses**

- **Special-use IPv4** blocks are reserved for specific purposes.
- IANA assigns special-use IPv4 address blocks.
- Shared Address Space is a special-use block intended for service provider networks.
- Shared Address Space is distinct from RFC 1918 private address space.
- Some special-use space supports Carrier-Grade NAT (CGN) devices.

**Address classes**

- IPv4 classes include A, B, C, D, and E.
- Class A, B, and C support different network sizes.
- Class D is used for multicast.
- Class E is reserved.

**Practical takeaways**

- Use public addresses for Internet-facing systems.
- Use private addresses for internal hosts and networks.
- Use special-use ranges only for their intended purpose.
- Use address translation when private or shared space must communicate beyond its local scope.
