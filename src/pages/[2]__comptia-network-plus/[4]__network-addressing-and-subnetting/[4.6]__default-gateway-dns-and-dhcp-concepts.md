## **Course:** Comptia Network Plus

**Core concepts**

- **DNS** stands for Domain Name System.
- DNS translates human-readable names into IP addresses.
- DNS uses a hierarchical database for distributed name resolution.
- The hierarchy starts at the root, shown as `.`.
- Top-level domains include generic TLDs such as `.com`, `.org`, and `.net`.
- Country code TLDs use two-letter country codes such as `.us`, `.ca`, and `.uk`.
- **DHCP** stands for Dynamic Host Configuration Protocol.
- DHCP assigns IP configuration details to devices on a network.
- A DHCP scope defines the address range available for a subnet.
- A DHCP pool is the group of addresses the server leases.
- A scope can include excluded addresses that are not handed out.
- A scope can include the subnet mask.
- A scope can include lease duration.
- A scope can include options such as DNS servers and default gateways.
- A DHCP reservation ties a specific address to a specific device.
- The **default gateway** is a common value in IP configuration.
- The default gateway is the route a device uses to reach other networks.

**How the services work together**

- DHCP can provide the default gateway to clients.
- DHCP can provide DNS server settings to clients.
- DNS helps clients resolve names after they receive IP settings.
- The default gateway supports traffic that leaves the local subnet.
- DNS supports name-based access to hosts and services.

**DHCP configuration details**

- Configure a scope for each subnet you service.
- Set the address range for the subnet.
- Exclude addresses that should stay outside the pool.
- Set lease timers to control how long clients keep addresses.
- Add optional settings when clients need more than an IP address.
- Use reservations when a device needs a consistent address.

**Troubleshooting focus**

- Use `ping` to check whether a host is reachable.
- Use `traceroute` to follow the path to a destination.
- Use `nslookup` or `dig` to test DNS name resolution.
- Check DHCP settings when clients receive the wrong gateway or DNS server.
- Check the default gateway when traffic cannot leave the local network.
- Check DNS when names fail to resolve but IP connectivity still works.

**Key takeaways**

- DNS resolves names to IP addresses.
- DHCP automates IP configuration.
- The default gateway connects a device to other networks.
- These services often work together during client setup.
