## **Course:** Comptia Network Plus

**IPv6 overview**

- IPv6 is the newer Internet Protocol version used for network addressing.
- IPv6 addresses are **128-bit** long.
- IPv4 uses **32-bit** addresses, so IPv6 provides a much larger address space.
- IPv6 addressing is important because IPv4 space is limited.

**Address allocation path**

- The **Internet Assigned Numbers Authority (IANA)** starts the IPv6 allocation process.
- IANA gives address blocks to **Regional Internet Registries (RIRs)**.
- RIRs assign smaller blocks to internet service providers.
- ISPs then assign IPv6 addresses to end users and organizations.
- This hierarchy helps distribute address space in manageable blocks.

**Prefix lengths**

- IPv6 prefix length shows how many bits identify the network.
- A **/64** prefix is the common subnet size for most IPv6 subnets.
- Organizations may receive a **/48** block from an ISP and subnet it further.
- Prefix planning is part of IPv6 address management.

**Addressing concepts**

- An IP address is a logical address used for routing packets over an IP network.
- A host is a device with a unique IP address on the network.
- Subnetting divides an address block into smaller network segments.
- IPv6 subnetting is generally less involved than IPv4 subnetting.

**Practical takeaways**

- You should recognize the allocation chain from IANA to the end user.
- You should identify **/64** as the typical IPv6 subnet prefix.
- You should understand that prefix length defines the network portion of the address.
- You should treat IPv6 as a core topic for modern network design and exam prep.
