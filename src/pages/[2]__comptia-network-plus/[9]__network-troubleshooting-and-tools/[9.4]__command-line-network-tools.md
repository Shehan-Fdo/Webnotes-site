## **Course:** Comptia Network Plus

**Command-line troubleshooting overview**

- Use the command line for detailed troubleshooting.
- Recognize common tools such as `ping`, `traceroute`, `nslookup`, `dig`, `tcpdump`, and `netstat`.
- Expect command-line tools to help you gather facts and confirm or deny troubleshooting theories.
- Learn the purpose of each tool and read its output quickly.

**`ping`**

- Use `ping` to test whether a device is reachable.
- `ping` shows round-trip time between your device and the target.
- `ping` uses `ICMP` (Internet Control Message Protocol).
- Run `ping` early in troubleshooting to check basic connectivity.
- Run `ping` again after troubleshooting to confirm the device responds.

**Path and name-resolution tools**

- Use `traceroute` to see the path traffic takes across the network.
- Use `nslookup` to query name resolution.
- Use `dig` for DNS (Domain Name System) lookups.
- Use these tools when you need to verify where traffic goes or whether names resolve correctly.

**Traffic and session inspection**

- Use `tcpdump` to inspect network traffic.
- Use `netstat` to review active sessions and network-related status.
- Use these tools when you need more detail than a simple reachability test provides.

**Device command familiarity**

- Learn that many network device commands are similar across manufacturers.
- Expect command syntax to vary, but the output often contains similar information.
- Use this familiarity to move between different vendors more easily.
- Recognize common device commands such as `show route`, `show interface`, and `show config`.

**Practical troubleshooting approach**

1. Check basic reachability with `ping`.
2. Trace the path with `traceroute` if connectivity is unclear.
3. Verify name resolution with `nslookup` or `dig`.
4. Inspect sessions or traffic with `netstat` or `tcpdump` when deeper analysis is needed.
5. Re-test with `ping` after changes to confirm the issue is resolved.

> `ping` is often the first command you try and the last command you run again after a fix.
