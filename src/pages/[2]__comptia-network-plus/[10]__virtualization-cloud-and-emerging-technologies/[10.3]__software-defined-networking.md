## **Course:** Comptia Network Plus

**Software-Defined Networking overview**

- **Software-defined networking (SDN)** uses a centralized software system for routing and switching decisions.
- SDN differs from traditional networking, where network devices make path decisions in distributed locations.
- SDN supports virtualized network functions in a more flexible environment.

**Traditional network planes**

- Traditional networking uses three planes.
- The **control plane** decides how traffic moves.
- The **data plane** forwards traffic.
- The **management plane** handles administration tasks.

**Virtualization context**

- SDN fits the move from physical devices to virtualized environments.
- Network functions can be represented in software instead of only in hardware.
- This approach can apply to routers, switches, and other networking devices.

**Network virtualization**

- A **virtual network** is a logical network created through software-based techniques.
- Devices and services can communicate without being physically connected.
- Virtual networks can improve flexibility, isolation, and resource use.

**SDN and related technologies**

- Virtual networks can use technologies such as **VLANs** (Virtual Local Area Networks), **VXLANs** (Virtual Extensible Local Area Networks), or SDN.
- SDN is part of the broader network virtualization space.
- SD-WAN can help optimize cloud-based application traffic.

**Practical takeaways**

- Use SDN when you want centralized control over routing and switching.
- Expect SDN to separate network decision-making from individual hardware devices.
- Look for SDN in environments that need flexible, software-driven network management.
