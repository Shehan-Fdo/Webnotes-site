## **Course:** Comptia Network Plus

**High availability goals**

- Keep the network and applications running.
- Use **fault tolerance** to continue service after a failure.
- Expect added complexity in design, procedures, and cost.
- Accept that some protections can still cause outages or performance degradation.

**Redundancy and failover**

- Duplicate critical components to reduce single points of failure.
- Use **failover** to redirect traffic to alternate components when a failure occurs.
- Apply redundancy to storage, power, and network connections.
- Choose active/passive or active/active designs when the environment supports them.

**Common redundancy methods**

- Use `RAID` for storage redundancy on a single system.
- Add redundant power supplies to keep a device running after a power loss.
- Use load balancing across servers to spread traffic and improve availability.
- Use link aggregation to combine links for more bandwidth and resilience.

**NIC teaming**

- Combine multiple network interface cards in one device for fault tolerance.
- Use **NIC teaming** to keep connectivity if one link fails.
- Use load balancing failover, also called `LBFO`, to support bandwidth and redundancy.
- Install multiple interfaces in a server when you need more throughput and resilience.

**Design tradeoffs**

- Plan for higher cost when you add backup components.
- Expect more configuration and maintenance work.
- Test the design so you know how it behaves during a failure.
- Balance availability goals against complexity and performance impact.

**Practical availability focus**

- Protect critical services first.
- Place redundancy where a failure would cause the most disruption.
- Use alternate components that can take over quickly.
- Keep uptime and availability as the main design target.
