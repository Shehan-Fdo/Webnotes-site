## **Course:** Comptia Network Plus

**Network time basics**

- Network devices keep their own clocks.
- You use time synchronization to keep those clocks aligned across servers, routers, switches, workstations, and other devices.
- Accurate time helps you compare log files across devices.
- Accurate time also helps you track outages and other events.
- Some authentication systems require clocks to stay close together.

**NTP and SNTP**

- **Network Time Protocol (NTP)** synchronizes clocks on networked devices.
- NTP can keep clocks within a few milliseconds of **Universal Coordinated Time (UTC)**.
- Devices request and receive UTC from an NTP server.
- The server can receive precise time from an atomic clock.
- NTP uses a set of rules for how devices transmit, receive, and understand time data.
- **Simple Network Time Protocol (SNTP)** is a lightweight time-sync option.
- NTP and SNTP use **UDP** and port `123`.

**How synchronization is used**

- You configure a device with an NTP server.
- The device checks in with that server to update its clock.
- This process happens automatically.
- Automatic synchronization avoids manual clock updates on every device.
- Automatic synchronization also avoids resetting clocks after reboots.

**Why it matters**

- Synchronized time keeps logs consistent.
- Synchronized time supports troubleshooting.
- Synchronized time supports authentication workflows.
- Synchronized time gives you a reliable record of when events occur.
