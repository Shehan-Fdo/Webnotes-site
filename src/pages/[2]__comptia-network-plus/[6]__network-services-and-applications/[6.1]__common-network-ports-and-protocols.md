## **Course:** Comptia Network Plus

**Ports and services**

- A **port number** helps identify the application or service that should receive packet payloads.
- A port is a 16-bit unsigned number.
- Port numbers range from `0` to `65535`.
- Services use port numbers on transport protocols such as `TCP` and `UDP` (User Datagram Protocol).
- Well-known ports help you map common applications to their default network services.

**Common TCP ports**

- `FTP` uses `TCP 20` and `TCP 21`.
- `SSH` uses `TCP 22`.
- `TELNET` uses `TCP 23`.
- `SMTP` uses `TCP 25`.
- `DNS` uses `TCP 53`.
- `HTTP` uses `TCP 80`.
- `NTP` uses `TCP 123`.
- `HTTPS` uses `TCP 443`.
- `SNMP` uses `TCP 161`.
- `POP3` uses `TCP 110`.
- `IMAP` uses `TCP 143`.

**Common UDP ports**

- `TFTP` uses `UDP 69`.
- `DNS` uses `UDP 53`.
- `DHCP` uses `UDP 67`.
- `SNMP` uses `UDP 161`.

**Secure and non-secure variants**

- `POP3` commonly uses `TCP 110` for non-encrypted mail retrieval.
- `POP3S` uses `TCP 995` for secure mail retrieval.
- `IMAP` commonly uses `TCP 143` for non-encrypted mail access.
- Secure mail services use different ports than their plain-text counterparts.

**Practical exam focus**

- Match the service to its default port number.
- Remember that some services use more than one port.
- Separate `TCP` services from `UDP` services when memorizing defaults.
- Use port numbers to identify traffic during troubleshooting.
