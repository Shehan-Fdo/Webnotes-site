## **Course:** Comptia Network Plus

**Remote access overview**

- Remote access lets you manage devices and use resources from a different physical location.
- It supports a mobile workforce that connects from home, public Wi-Fi, or other offsite locations.
- It often gives you access to web-based applications without installing a local client.
- It helps you reach switches, routers, firewalls, servers, and corporate applications.

**Secure command-line access**

- **SSH** stands for Secure Shell.
- SSH provides an encrypted terminal session to a remote device.
- SSH uses TCP port `22`.
- SSH protects usernames, passwords, and commands in transit.
- **Telnet** provides similar terminal access but no encryption.
- Telnet uses TCP port `23`.
- Use SSH instead of Telnet when security matters.

**Remote management paths**

- A **console** connection gives you direct command-line access to a device.
- A **jump box** is a dedicated server or virtual machine used to reach other systems securely.
- A jump box acts as an intermediate system before you access production resources.
- **In-band management** uses the production network for device administration.
- **Out-of-band management** uses a separate path for administration.

**VPN access**

- A **VPN** is a Virtual Private Network.
- A VPN uses public infrastructure to create a private connection.
- VPNs support remote access to a central site.
- VPNs can also connect two LANs across distant locations.
- **IPsec** is a common VPN security option.
- IPsec provides authentication and encryption at OSI Layer 3.
- IPsec can protect each packet with confidentiality and integrity.
- **SSL VPN** is another common remote access encryption option.
- VPN implementations may use tunneling protocols such as `PPTP`, `L2TP`, and `OpenVPN`.

**Practical security guidance**

- Use encrypted remote access for administrative sessions.
- Avoid Telnet when SSH is available.
- Use a jump box when you need a controlled path to sensitive systems.
- Use VPNs when you need secure access across untrusted networks.
- Choose the access method that matches the task and security requirement.
