## **Course:** Comptia Network Plus

**File services**

- File services let users move data between systems on the network.
- **FTP** stands for File Transfer Protocol.
- FTP uses TCP port `20` and TCP port `21`.
- FTP supports username and password authentication.
- FTP lets you manage folders and files on the server.
- **TFTP** stands for Trivial File Transfer Protocol.
- TFTP uses UDP port `69`.
- TFTP is simple and has no authentication.
- TFTP provides limited file system functions.
- TFTP is designed for basic data transfer.

**Email services**

- Email services support message delivery across the network.
- Email systems commonly use separate protocols for sending and receiving mail.
- **SMTP** stands for Simple Mail Transfer Protocol.
- **POP3** stands for Post Office Protocol version 3.
- **IMAP** stands for Internet Message Access Protocol.
- SMTP handles mail sending.
- POP3 and IMAP handle mail retrieval.
- Use the correct protocol based on whether you need delivery or mailbox access.

**Web services**

- Web servers respond to browser requests.
- Web content is commonly delivered with **HTML** or `HTML5`.
- Browsers request pages from a server and receive them over the network.
- Web pages can be static or dynamic.
- Static pages are created before the request.
- Dynamic pages are created when the client requests them.
- Web services are common in data centers and internet-facing systems.

**Name resolution and supporting services**

- **DNS** stands for Domain Name System.
- DNS converts fully qualified domain names to IP addresses.
- A browser uses DNS before connecting to a web server by name.
- DNS stores lookup information in resource records.
- DNS data is maintained in a text-based lookup file.
- DNS records can include server details, caching information, and mail exchange data.

**Practical exam focus**

- Match each service to its purpose.
- Remember the common ports for file transfer protocols.
- Distinguish between sending mail and reading mail.
- Recognize DNS as the service that maps names to IP addresses.
- Identify web services as browser-facing content delivery.
