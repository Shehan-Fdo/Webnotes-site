## **Course:** Comptia Network Plus

**Network performance metrics**

- **Bandwidth** is the maximum rate a network link can carry data.
- Measure bandwidth in bits per second (`bps`).
- Higher bandwidth supports more simultaneous data transfer.
- Bandwidth describes capacity, not actual delivery.

- **Throughput** is the actual rate of successful data transfer.
- Throughput can be lower than bandwidth.
- Congestion can reduce throughput.
- Hardware limits can also reduce throughput.

- **Latency** is the time a packet takes to travel across the network.
- Lower latency means less delay.
- High latency can hurt interactive traffic.
- Latency affects how quickly data arrives, not how much data can move.

- **Jitter** is the variation in latency over time.
- Stable networks have low jitter.
- High jitter can disrupt real-time traffic.
- Voice over IP (`VoIP`) is sensitive to jitter.

**Common performance problems**

- Congestion happens when too much traffic tries to use the same path at once.
- Queues form in device buffers during congestion.
- Buffers are limited in size.
- When buffers fill, devices may discard packets.

- Packet loss occurs when packets are dropped before delivery.
- Packet loss can follow congestion.
- Packet loss reduces effective communication quality.
- Loss can make performance issues more noticeable.

**Troubleshooting focus**

- Check whether the network link is operating at the expected speed.
- Look for congestion when traffic demand exceeds link capacity.
- Review latency when applications feel slow.
- Review jitter when voice or other real-time traffic sounds uneven.
- Review packet loss when users report missing or incomplete data.

**Key distinctions**

- Bandwidth is potential capacity.
- Throughput is actual delivered rate.
- Latency is delay.
- Jitter is delay variation.
