## **Course:** Comptia Network Plus

**AAA framework**

- **AAA** stands for authentication, authorization, and accounting.
- It supports centralized access control in network environments.
- It helps screen users and track activity while they are connected.
- It is commonly used when you log in to a network, VPN, router, firewall, or switch.

**Authentication**

- Authentication verifies that you are who you claim to be.
- You usually start with identification, such as a username or email address.
- You then prove identity with a password or another authentication factor.
- A centralized authentication server can compare your credentials against stored records.
- If the credentials match, the server approves access.

**Authorization**

- Authorization determines what access you get after authentication succeeds.
- It limits resources based on your role or account.
- A user in one department should only reach systems and data allowed for that role.
- Authorization enforces access boundaries after identity is confirmed.

**Accounting**

- Accounting records what a user does after access is granted.
- It supports logging, auditing, and usage tracking.
- It helps administrators review activity on connected systems.
- It is part of the broader AAA process for network management.

**Centralized AAA services**

- Large networks often use a centralized AAA server.
- Centralized control helps manage many login points consistently.
- Network devices can send login requests to the server for validation.
- This approach is useful when users connect from different locations or devices.

**Common AAA-related technologies**

- **RADIUS** is used for centralized authentication, authorization, and accounting.
- **TACACS+** is also used for AAA functions.
- **LDAP** can support authentication-related directory lookups.
- **SAML** can support single sign-on use cases.
- **SSO** means single sign-on, where one login can provide access to multiple resources.

**Practical flow**

1. You identify yourself with a username or similar account name.
2. You authenticate with a password or additional factor.
3. The system authorizes the resources you can use.
4. The system records activity for accounting and auditing.

**Key takeaways**

- Identification is the claim of identity.
- Authentication is the proof of identity.
- Authorization is the permission decision.
- Accounting is the record of activity.
