## **Course:** Comptia Network Plus

**Network segmentation**

- **Network segmentation** separates parts of a computer network into zones.
- You can use devices like firewalls, switches, and routers to separate those zones.
- Segmentation can be physical, logical, or virtualized.
- You can apply it in data centers and on-premises environments.
- Segmentation helps reduce risk and limit the severity of security incidents.
- Segmentation can also improve performance by isolating high-bandwidth applications.
- You can keep users away from sensitive core systems, such as a database server.

**Why segmentation matters**

- Segmentation limits how far an attacker can move after a compromise.
- It helps you control which protocols and services operate inside a protected zone.
- It supports tighter monitoring of traffic between network areas.
- It is a practical way to protect data in modern networks.

**Zero trust**

- **Zero trust** treats every user, device, and application as untrusted by default.
- You do not grant access until the identity is approved.
- You verify traffic before allowing it to reach resources.
- The model uses authentication, encryption, and additional controls inside the network.
- Zero trust is a holistic security approach that affects users and applications across the environment.
- The goal is to allow only the right users to access the right data.

**Zero trust principles**

- Use **least privilege** so users get only the access they need.
- Verify identity before granting access to resources.
- Monitor user behavior for suspicious activity.
- Apply access controls consistently for employees, vendors, and customers.
- Use the model to reduce exposure as connected devices increase the attack surface.

**Segmentation and zero trust together**

- Segmentation creates boundaries between network zones.
- Zero trust adds verification inside those boundaries.
- Together, they reduce trust assumptions across the network.
- Together, they help protect data, users, and applications from unnecessary exposure.

**Key takeaways**

- Segmentation is about dividing the network.
- Zero trust is about verifying every access request.
- Both approaches improve security by reducing implicit trust.
- Both support stronger control over internal traffic and sensitive resources.
