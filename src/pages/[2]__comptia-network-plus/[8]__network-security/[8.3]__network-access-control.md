## **Course:** Comptia Network Plus

**Network access control overview**

- **NAC** means network access control.
- It helps keep users off the network until they authenticate correctly.
- It applies to physical switch access, not TCP or UDP ports.
- It is often called port-based network access control.

**802.1X authentication**

- The most common NAC standard is IEEE `802.1X`.
- `802.1X` uses port-based authentication on a switch port.
- The port stays closed until the user authenticates.
- `EAP` means Extensible Authentication Protocol.
- `EAP` is commonly used with `802.1X`.
- A `RADIUS` or `TACACS` server usually verifies the authentication.

**Switch port security**

- Disable unused switch interfaces.
- This reduces the chance of unauthorized access through open jacks.
- Use MAC address checking when the switch supports it.
- Treat switch port security as protection for physical interfaces.

**Access control and policy**

- Access control can work inside or outside the network.
- You can base access on the user, department, or location.
- You can enable or disable access quickly.
- If someone leaves the organization, you can remove access immediately.

**BYOD and health checks**

- `BYOD` means bring your own device.
- BYOD increases the need for stronger access control.
- NAC can perform health checks on devices.
- Health checks help block unmanaged devices from the network.

**Practical exam reminders**

- Do not confuse switch port security with firewall port filtering.
- Switch port security protects physical switch ports.
- Firewall rules control traffic at the network edge.
- `802.1X` is the key standard to remember for NAC.
