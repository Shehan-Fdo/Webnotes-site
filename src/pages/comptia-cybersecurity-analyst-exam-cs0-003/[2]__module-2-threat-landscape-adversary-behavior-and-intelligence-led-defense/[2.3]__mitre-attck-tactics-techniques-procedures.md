## **Course:** CompTIA Cybersecurity Analyst (Exam CS0-003)

**MITRE ATT&CK overview**

MITRE ATT&CK is a globally accessible knowledge base of adversary behavior based on real-world observations. It helps you describe how attackers operate and gives you a common language for threat modeling, detection, and response.

ATT&CK is open and available at no charge. It is used across the private sector, government, and the cybersecurity product and service community as a foundation for threat models and methodologies.

- **Adversary behavior** is organized into tactics, techniques, and procedures.
- ATT&CK supports both defensive planning and analysis of observed activity.
- The knowledge base is maintained as a shared reference for the community.

**Tactics, techniques, and procedures**

In ATT&CK, a **tactic** is the tactical goal, a **technique** is how the adversary achieves that goal, and a **procedure** is the specific implementation seen in the real world. This structure helps you move from high-level intent to concrete attacker behavior.

The Enterprise matrix shows common tactics such as Reconnaissance, Resource Development, Initial Access, Execution, Persistence, Privilege Escalation, Defense Evasion, Credential Access, Discovery, Lateral Movement, Collection, and Command and Control.

- **Tactics** describe the attacker’s objective.
- **Techniques** describe the method used to reach that objective.
- **Procedures** describe the observed instance of a technique in a specific campaign or environment.

> ATT&CK content changes over time. For example, MITRE notes planned deprecation of the Enterprise Defense Evasion tactic in a future release, so you should verify current matrix details when studying or mapping activity.

**Enterprise techniques**

Enterprise techniques represent how an adversary achieves a tactical goal by performing an action. ATT&CK includes techniques and sub-techniques, and the Enterprise catalog contains a large set of both.

A practical example from the ATT&CK technique catalog is `T1548 Abuse Elevation Control Mechanism`, which covers adversaries circumventing mechanisms that control privilege elevation. One listed sub-technique is `Setuid and Setgid`, where an adversary abuses Linux configurations that grant code execution in a different user context.

- Use technique names to describe attacker actions precisely.
- Use sub-techniques when you need more specific detail.
- Map observed behavior to the closest ATT&CK entry rather than relying on vague labels.

**Using ATT&CK data and tools**

ATT&CK provides tools for exploring and working with the knowledge base. The ATT&CK Navigator is a web-based tool for annotating and exploring matrices, and it can help you visualize defensive coverage, red/blue team planning, and the frequency of detected techniques.

ATT&CK Workbench lets you explore, create, annotate, and share extensions of the knowledge base. MITRE also provides Python utilities for accessing, querying, and processing ATT&CK data programmatically.

- Use the Navigator to compare coverage across tactics or techniques.
- Use Workbench when you need to manage ATT&CK content and extensions.
- Use programmatic access when you want to query or process ATT&CK data at scale.

**Study focus for analysts**

For exam and operational use, focus on recognizing the relationship between attacker goals and observed actions. ATT&CK is most useful when you can translate an alert, incident note, or threat report into a tactic, technique, and procedure chain.

A simple workflow is to identify the goal, match the method, and then record the specific behavior you observed. That approach makes your analysis more consistent and easier to share.

1. Identify the attacker’s objective from the evidence.
2. Match the observed action to the relevant ATT&CK technique.
3. Add the procedure details that make the behavior specific to the incident.
4. Use the mapping to support detection, reporting, or threat modeling.
