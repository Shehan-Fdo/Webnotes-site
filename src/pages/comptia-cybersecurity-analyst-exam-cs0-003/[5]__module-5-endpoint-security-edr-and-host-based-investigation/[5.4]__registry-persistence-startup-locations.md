## **Course:** CompTIA Cybersecurity Analyst (Exam CS0-003)

**Registry persistence and startup locations**

Windows persistence uses built-in startup behavior to keep a program running after reboot or logon. Attackers abuse these features so malware can launch without user interaction and maintain access beyond a shutdown or restart. In practice, this makes registry-based autostart entries and startup folders important hunting targets.

- **Persistence** means keeping code available across interruptions such as shutdowns, restarts, file deletion, or credential changes.
- **Autostart execution** is the broader behavior of launching a program automatically at boot or logon.
- Defenders focus on these locations because they are common, practical, and often used in real Windows environments.

**Registry run keys**

MITRE ATT&CK identifies `Registry Run Keys / Startup Folder` as technique `T1547.001`. A registry run key is a value that points to a program to execute when a user logs in. The program runs in the context of that user and inherits the account’s permissions.

- Adding a program reference to a **run key** can create user-level persistence.
- Run keys are attractive because they are simple, reliable, and easy to blend into normal startup behavior.
- Monitoring changes to these keys helps reveal unauthorized autostart entries early.

Common defender focus areas include:

- New or modified registry values that point to executables in unusual paths.
- Entries that appear after a compromise or credential change.
- Programs that should not normally launch at user logon.

**Startup folders**

The startup folder is another autostart location that causes referenced programs to run when a user logs in. Like registry run keys, it supports persistence by tying execution to the logon process.

- Startup folder entries are straightforward to abuse for persistence.
- They are useful for attackers who want a simple logon trigger without deeper system changes.
- They are also a practical place for hunters to check during triage.

**Why these locations matter**

Persistence is valuable to attackers because it preserves access even when the environment changes. If a host reboots or a user resets credentials, a persistence mechanism can restore the attacker’s foothold. That is why registry run keys and startup folders remain high-value detection points.

- They support execution without direct user action.
- They can survive routine system use and normal restarts.
- They often appear in Windows environments, so defenders should expect them in both benign and malicious contexts.

**Detection and hunting approach**

You should treat these locations as part of routine persistence hunting. The goal is not just to find malware, but to identify unexpected autostart behavior and confirm whether it is authorized.

1. Review registry run key changes and startup folder contents.
2. Compare entries against known-good software and expected user behavior.
3. Investigate programs that launch at logon but do not match the host’s normal profile.
4. Correlate persistence findings with recent alerts, logon activity, or compromise indicators.

**Practical interpretation**

A registry run key or startup folder entry is not automatically malicious. Many legitimate applications use autostart behavior. The key question is whether the entry is expected, necessary, and consistent with the system’s role.

- Treat unknown executables in autostart locations as suspicious until verified.
- Pay attention to user context, because run key execution inherits that account’s permissions.
- Use persistence findings to support broader incident response and containment decisions.

**Exam focus**

For exam prep, remember the relationship between the mechanism and the effect:

- **Registry run keys** and **startup folders** are Windows autostart locations.
- They are used for **persistence** by launching programs at user logon.
- They are associated with MITRE ATT&CK technique `T1547.001`.
- They are important because they help malware remain active after reboot or interruption.
