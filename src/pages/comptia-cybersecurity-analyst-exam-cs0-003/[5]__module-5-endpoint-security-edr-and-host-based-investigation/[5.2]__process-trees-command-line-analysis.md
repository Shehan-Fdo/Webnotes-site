## **Course:** CompTIA Cybersecurity Analyst (Exam CS0-003)

**Process trees and why they matter**

A process tree shows how a process starts and which child processes it creates. In threat investigation, this helps you understand execution flow on an endpoint and identify relationships that do not fit normal behavior.

Parent-child analysis is especially useful when a process such as `cmd.exe` appears under an unusual parent. Security tools may flag these relationships because they can indicate suspicious execution chains and possible attacker activity.

- **Parent process**: the process that launches another process.
- **Child process**: the process created by the parent.
- **Unusual relationship**: a parent-child pairing that does not match expected system or user activity.
- **Execution context**: the surrounding process chain that helps explain what happened on the host.

**What to look for in command-line analysis**

Command-line analysis gives you more detail than process names alone. It helps you see how a process was started, what arguments were used, and whether the command fits the expected task.

This is useful when you need to distinguish normal administrative activity from suspicious execution. A process may look harmless by name, but its command line can reveal a different intent.

- Review the full command line, not just the executable name.
- Compare the command against expected behavior for that host or user.
- Look for unusual parent processes that launch common shells or utilities.
- Use the command line to connect a process to a broader investigation.

**Investigation workflow**

1. Identify the suspicious process or alert.
2. Trace the parent process and any child processes.
3. Review the command line for arguments, paths, and execution details.
4. Compare the process chain with normal activity on the endpoint.
5. Decide whether the relationship is expected, unusual, or clearly suspicious.

This workflow helps you move from a single alert to a fuller picture of endpoint activity. It also supports faster triage when multiple processes are involved.

**Endpoint tools and live response**

Endpoint detection and response (EDR) tools support this kind of analysis by collecting process activity and helping you investigate suspicious behavior across operating systems. Microsoft Defender for Endpoint live response also includes commands that help you inspect processes directly on a device.

```bash
analyze process 1234
analyze file c:\Users\user\Desktop\malware.txt
connections -output json
dir
```

- `analyze process <PID>` inspects a process by process ID.
- `analyze file <path>` checks a file during live response.
- `connections -output json` lists active connections in JSON format.
- `dir` shows files and subfolders in the current folder.

> Use live response carefully and focus on commands that support investigation and containment. Keep your actions aligned with the incident scope.

**Practical interpretation tips**

Process-tree review works best when you combine structure and context. A suspicious child process may be more meaningful if it comes from an unexpected parent, runs with unusual arguments, or appears alongside other abnormal endpoint activity.

- Treat common utilities with extra care when they appear in odd chains.
- Use process ancestry to explain how execution started.
- Correlate command-line details with the alert source and host context.
- Prefer conservative conclusions when the evidence is incomplete.

**Key takeaway**

Process trees show relationships, and command-line analysis shows intent. Together, they help you identify suspicious execution patterns, validate alerts, and build a clearer endpoint investigation.
