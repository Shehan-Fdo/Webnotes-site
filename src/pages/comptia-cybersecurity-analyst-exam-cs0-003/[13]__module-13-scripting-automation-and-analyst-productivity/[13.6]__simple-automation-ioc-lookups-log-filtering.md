## **Course:** CompTIA Cybersecurity Analyst (Exam CS0-003)

**IOC lookups and log filtering**

Indicators of Compromise (IOCs) are artifacts that suggest malicious activity, such as IP addresses, URLs, and domains. Simple automation helps you check those values at scale, enrich matching events with security context, and filter logs so you can focus on the most relevant activity.

Automation is useful because security teams often face large alert and log volumes. Centralized security data and repeatable enrichment steps help you preserve coverage while reducing manual triage effort.

- **IOC** checks compare event fields against a curated indicator database.
- Log filtering removes low-value noise before deeper analysis.
- Enrichment adds context such as confidence, labels, and update dates.
- Automation supports historical hunting across large log sets.

**Using `ioc:lookup()` for enrichment**

In LogScale, `ioc:lookup()` looks up IOCs in a local copy of CrowdStrike's curated IOC database and annotates events with associated security information. It supports IP addresses, URLs, and domains.

The function uses a required `field` parameter for the event field or fields to check. You can also set `confidenceThreshold` to control which IOC confidence levels are returned.

- `confidenceThreshold` accepts `unverified`, `low`, `medium`, or `high`.
- `high` is the default and returns the lowest confidence level of IOC to consider.
- `field` is required and can contain one or more strings.
- `include` is optional and controls which IOC database columns appear in results.

Common output fields include:
- `indicator`
- `labels`
- `last_updated`
- `malicious_confidence`
- `published_date`
- `type`

```text
ioc:lookup(field=["src_ip", "url"], confidenceThreshold=high, include=["indicator", "labels", "malicious_confidence"])
```

> Use the confidence threshold to balance coverage and noise. Lower thresholds return more results, but they may include less certain matches.

**Filtering logs before or after lookup**

Filtering is a practical way to narrow the dataset before you enrich it. You can reduce unnecessary processing by selecting only the logs, time ranges, or fields that matter to the investigation.

A simple workflow is to filter first, then run IOC matching on the remaining events. In other cases, you may enrich first and then filter on the returned IOC metadata.

- Filter by source, time window, or event type when you already know the scope.
- Filter on enriched fields such as confidence or labels when you need to prioritize matches.
- Keep filters narrow enough to reduce noise, but broad enough to avoid missing relevant events.

**SOC automation context**

Security operations centers use automation for repetitive tasks such as alert triage, enrichment, indicator lookups, containment actions, and report generation. The goal is not to replace analysts, but to free them for higher-value investigation and threat hunting.

In practice, automation works best when it is repeatable and easy to validate. That makes it suitable for SIEM triage, incident response support, and long-term hunting in large data stores.

- Automation reduces manual effort in high-volume environments.
- Centralized data improves long-term visibility for investigations.
- Human oversight remains important for final interpretation and response.
- Clear metrics such as response time and false positives help you measure value.

**Practical workflow**

1. Identify the event fields that may contain IP addresses, URLs, or domains.
2. Filter the log set to the relevant time range or source.
3. Run `ioc:lookup()` against the selected fields.
4. Review returned metadata such as confidence, labels, and update dates.
5. Filter the enriched results to prioritize suspicious matches.
6. Escalate or investigate the highest-value events first.

**Key takeaways**

Simple automation for IOC lookups and log filtering helps you move from raw events to actionable context faster. The main value comes from combining targeted filtering with indicator enrichment so you can reduce noise, preserve coverage, and focus on the most suspicious activity.
