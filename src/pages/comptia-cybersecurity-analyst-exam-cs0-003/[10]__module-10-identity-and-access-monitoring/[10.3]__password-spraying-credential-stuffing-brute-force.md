## **Course:** CompTIA Cybersecurity Analyst (Exam CS0-003)

**Overview**

Password spraying, credential stuffing, and brute force are related password attack methods, but they are not interchangeable. They all aim to gain unauthorized access by trying credentials, and they often succeed when users reuse weak passwords or when organizations lack strong account protections.

- **Brute force** is the broad category for repeated password-guessing attacks.
- **Password spraying** uses one common password, or a small set of common passwords, against many accounts.
- **Credential stuffing** uses credentials stolen or leaked from one account to try other accounts.
- These attacks can lead to unauthorized access, fraudulent transactions, system downtime, data breaches, and financial loss.

**Password spraying**

Password spraying spreads login attempts across many accounts instead of hammering one account with many guesses. This approach helps attackers avoid account lockouts that usually trigger when a single account receives too many failed attempts.

- Attackers use a single common password or a small list of common passwords.
- They try those passwords against many different accounts on the same application or network.
- The password may match a domain complexity policy if it is common enough.
- Commonly targeted services include management services over widely used ports such as SSH, Telnet, FTP, SMB, LDAP, Kerberos, and RDP.

> Password spraying is designed to stay below lockout thresholds by distributing attempts across accounts.

**Credential stuffing**

Credential stuffing depends on password reuse. Attackers take credentials exposed in a breach or leak and test them on other services, expecting users to reuse the same username and password combination elsewhere.

- The attack starts with stolen or leaked credentials from one account.
- The attacker tries those credentials on other accounts and services.
- Success depends heavily on users reusing passwords across sites.
- Like password spraying, it can result in account takeover and downstream abuse.

**Brute force in context**

Brute force is the umbrella idea of trying many credential combinations until one works. Password spraying and credential stuffing are both forms of brute-force password attacks, but each uses a different strategy.

| Attack type | What it tries | How it spreads | Main goal |
|---|---|---|---|
| Password spraying | One common password or a small set of common passwords | Many accounts | Avoid lockouts and find weak passwords |
| Credential stuffing | Known stolen or leaked credentials | Other accounts and services | Exploit password reuse |
| Brute force | Many password guesses | Often one account, but can vary | Discover valid credentials |

**Why these attacks work**

These attacks succeed when credential hygiene is weak. If users choose predictable passwords, reuse passwords, or if organizations do not detect repeated login attempts, attackers can operate with relatively low effort.

- Weak or common passwords increase exposure.
- Password reuse makes breach data reusable.
- Distributed attempts can make detection harder.
- Automated attacks can scale quickly across many accounts.

**Operational impact**

The impact is not limited to a single compromised account. Once attackers gain access, they can move into sensitive systems, trigger fraudulent activity, or disrupt operations.

- Unauthorized access to user or administrative accounts.
- Fraudulent transactions or misuse of services.
- System downtime from incident response or account containment.
- Broader compromise when stolen access is reused internally.

**How to think about the differences**

Use the source of the password and the target pattern to distinguish the attacks.

1. Ask whether the attacker is using a password guessed from common choices.
2. Check whether the same password is being tried across many accounts.
3. Determine whether the attacker is reusing credentials obtained from a breach.
4. Classify the attack based on the strategy, not just the outcome.

**Exam focus**

For exam questions, focus on the pattern of attempts.

- If one common password is tried against many accounts, think **password spraying**.
- If breached credentials are reused on other accounts, think **credential stuffing**.
- If many guesses are made to discover a password, think **brute force**.
- If the scenario mentions avoiding lockouts, that strongly points to password spraying.
